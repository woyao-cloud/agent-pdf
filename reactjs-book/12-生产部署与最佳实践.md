## 12.1 构建优化

### 12.1.1 代码分割

React.lazy 结合 Suspense 是实现代码分割的标准方式。React 19 进一步优化了 Suspense 的行为，使其在服务端渲染和客户端渲染中表现更一致。

```jsx
import { lazy, Suspense } from 'react';

// 按需加载，代码分割点
const Dashboard = lazy(() => import('./pages/Dashboard'));
const UserSettings = lazy(() => import('./pages/UserSettings'));
const Analytics = lazy(() => import('./pages/Analytics'));

// 路由级别的代码分割
const routes = [
  { path: '/dashboard', Component: Dashboard },
  { path: '/settings', Component: UserSettings },
  { path: '/analytics', Component: Analytics },
];

function App() {
  return (
    <Suspense fallback={<PageSkeleton />}>
      <Routes>
        {routes.map(({ path, Component }) => (
          <Route key={path} path={path} element={<Component />} />
        ))}
      </Routes>
    </Suspense>
  );
}
```

**最佳实践：**

- 在路由级别进行代码分割，而非组件级别。路由是用户感知的自然分割点。
- 使用有意义的 fallback（如骨架屏）而非简单的 loading 文字。
- 预加载用户可能访问的下一个页面：

```jsx
import { lazy, startTransition } from 'react';

const ProductDetail = lazy(() => import('./pages/ProductDetail'));

// 在用户悬停链接时预加载
function ProductLink({ productId }) {
  const preloadProduct = () => {
    const ProductDetail = await import('./pages/ProductDetail');
    // 组件已预加载
  };

  return (
    <Link
      to={`/product/${productId}`}
      onMouseEnter={preloadProduct}
      onTouchStart={preloadProduct}
    >
      查看详情
    </Link>
  );
}
```

### 12.1.2 Tree Shaking

Tree Shaking 依赖于 ES Module 的静态结构（`import`/`export`），移除未被使用的导出。

**确保 Tree Shaking 生效：**

```json
// package.json
{
  "sideEffects": [
    "*.css",
    "*.scss",
    "./src/polyfills.ts"
  ]
}
```

- 将 `sideEffects` 设为 `false` 表示所有文件都是"纯"的，可以安全地摇树。
- 如果某些文件有副作用（如导入 CSS、polyfill），在数组中显式列出它们。

**避免破坏 Tree Shaking 的做法：**

```typescript
// 错误：整体导入会导致所有内容被包含
import { Button } from 'antd';

// 正确：按路径导入
import Button from 'antd/es/button';
```

### 12.1.3 Bundle 分析

定期分析 bundle 组成，发现优化机会。


**Bundle 优化的关键指标：**

- **首屏 JS 体积**：目标 < 200KB（gzip 后）
- **首屏 CSS 体积**：目标 < 50KB（gzip 后）
- **最大依赖**：单个依赖不应超过 bundle 总大小的 20%

- 图片裁剪为实际展示尺寸，避免大图缩小显示

## 12.2 性能监控

### 12.2.1 Web Vitals

Web Vitals 是 Google 推出的用户体验量化标准，直接影响 SEO 排名。

| 指标 | 全称 | 目标值 | 测量内容 |
|------|------|--------|---------|
| LCP | Largest Contentful Paint | < 2.5s | 最大内容元素渲染时间 |
| INP | Interaction to Next Paint | < 200ms | 交互响应延迟（取代 FID） |
| CLS | Cumulative Layout Shift | < 0.1 | 视觉稳定性（布局偏移） |

React 19 中监控 Web Vitals：

```jsx
import { onCLS, onINP, onLCP } from 'web-vitals';

function sendToAnalytics(metric) {
  // 发送到监控服务
  const body = {
    name: metric.name,
    value: metric.value,
    rating: metric.rating,
    delta: metric.delta,
    id: metric.id,
    url: window.location.pathname,
  };

  // 使用 navigator.sendBeacon 确保请求不被页面卸载影响
  navigator.sendBeacon('/api/vitals', JSON.stringify(body));
}

function App() {
  useEffect(() => {
    onCLS(sendToAnalytics);
    onINP(sendToAnalytics);
    onLCP(sendToAnalytics);
  }, []);

  return <MainApp />;
}
```

### 12.2.2 React Profiler API

React Profiler 允许以编程方式测量渲染性能。

```jsx
import { Profiler } from 'react';

function onRenderCallback(
  id,                 // Profiler 树的标识
  phase,              // "mount" 或 "update"
  actualDuration,     // 本次渲染耗时
  baseDuration,       // 不使用 memo 的预估耗时
  startTime,          // 渲染开始时间
  commitTime,         // 提交时间
  interactions        // 交互集合
) {
  // 发送到监控服务
  if (actualDuration > 16) { // 超过 16ms（60fps 的帧预算）
    console.warn(`组件 ${id} 渲染耗时 ${actualDuration}ms`);
  }
}

function App() {
  return (
    <Profiler id="MainApp" onRender={onRenderCallback}>
      <Dashboard />
      <Profiler id="DataTable" onRender={onRenderCallback}>
        <DataTable />
      </Profiler>
    </Profiler>
  );
}
```

## 12.3 无障碍

无障碍（Accessibility，a11y）不仅是道德责任，也是法律要求。React 提供了良好的基础，但开发者需要主动关注。

### 12.3.1 语义化 HTML

```jsx
// 错误：div 地狱
function Navigation() {
  return (
    <div className="nav">
      <div className="nav-item" onClick={() => {}}>首页</div>
      <div className="nav-item" onClick={() => {}}>关于</div>
    </div>
  );
}

// 正确：语义化元素
function Navigation() {
  return (
    <nav aria-label="主导航">
      <button onClick={() => {}}>首页</button>
      <button onClick={() => {}}>关于</button>
    </nav>
  );
}
```

### 12.3.2 键盘导航

```jsx
function Modal({ isOpen, onClose, children }) {
  const previousFocus = useRef(null);

  useEffect(() => {
    if (isOpen) {
      previousFocus.current = document.activeElement;
      // 聚焦到模态框
      document.getElementById('modal-content')?.focus();
    } else if (previousFocus.current) {
      // 关闭后恢复焦点
      previousFocus.current.focus();
    }
  }, [isOpen]);

  useEffect(() => {
    const handleKeyDown = (e) => {
      if (e.key === 'Escape') onClose();
      // Tab 循环
      if (e.key === 'Tab') {
        const focusable = modalRef.current.querySelectorAll(
          'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
        );
        const first = focusable[0];
        const last = focusable[focusable.length - 1];
        if (e.shiftKey && document.activeElement === first) {
          e.preventDefault();
          last.focus();
        } else if (!e.shiftKey && document.activeElement === last) {
          e.preventDefault();
          first.focus();
        }
      }
    };

    if (isOpen) {
      document.addEventListener('keydown', handleKeyDown);
      document.body.style.overflow = 'hidden';
    }
    return () => {
      document.removeEventListener('keydown', handleKeyDown);
      document.body.style.overflow = '';
    };
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return createPortal(
    <div
      role="dialog"
      aria-modal="true"
      aria-labelledby="modal-title"
      id="modal-content"
      tabIndex={-1}
    >
      <h2 id="modal-title">{children}</h2>
      <button onClick={onClose} aria-label="关闭">X</button>
    </div>,
    document.getElementById('modal-root')
  );
}
```

```jsx
import { useTranslation } from 'react-i18next';

function LoginPage() {

  const switchLanguage = () => {
    const nextLang = i18n.language === 'zh' ? 'en' : 'zh';
    i18n.changeLanguage(nextLang);

    document.documentElement.dir = nextLang === 'ar' ? 'rtl' : 'ltr';

      <h1>{t('welcome')}</h1>
      <button onClick={switchLanguage}>
        {i18n.language === 'zh' ? 'English' : '中文'}
      </button>
    </div>
  );
}
```

## 12.5 安全

### 12.5.1 XSS 防御

React 默认转义 JSX 中的所有内容，有效防止 XSS 攻击。但有几个例外需要特别小心：

```jsx
// 危险：直接设置 innerHTML
function UnsafeComponent({ htmlContent }) {
  // dangerouslySetInnerHTML 必须配合 sanitize 使用
  return <div dangerouslySetInnerHTML={{ __html: sanitizeHtml(htmlContent) }} />;
}

// 安全：使用专门的 sanitization 库
import DOMPurify from 'dompurify';

function SafeComponent({ htmlContent }) {
  const clean = DOMPurify.sanitize(htmlContent, {
    ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'a'],
    ALLOWED_ATTR: ['href'],
  });
  return <div dangerouslySetInnerHTML={{ __html: clean }} />;
}
```

## 11.5 常见测试陷阱

### 陷阱1：测试实现细节而非用户行为

```typescript
// 错误：直接测试内部状态
expect(component.state('count')).toBe(1);

// 正确：测试用户可见的结果
expect(screen.getByText('1')).toBeInTheDocument();
```

### 陷阱2：未清理 Mock

```typescript
// 错误：Mock 在测试间泄漏
afterAll(() => {
  vi.restoreAllMocks(); // 只在所有测试结束后恢复
});

// 正确：每个测试后清理
afterEach(() => {
  vi.clearAllMocks();
});
```

### 陷阱3：异步测试时序问题

```typescript
// 错误：没有等待异步操作
user.click(button);
expect(screen.getByText('加载完成')).toBeInTheDocument();

// 正确：使用 waitFor 等待
await user.click(button);
await waitFor(() => {
  expect(screen.getByText('加载完成')).toBeInTheDocument();
});
```

### 陷阱4：过度 Mock

**过度 Mock 的问题：** 当你 mock 了太多依赖，测试就变成了在测试你的 mock，而非真实代码。尽量使用真实实现或集成测试。

### 陷阱5：不稳定的测试

常见原因及解决方案：
- **时序依赖**：使用 `waitFor` 和 `findBy` 而非固定 `setTimeout`
- **网络依赖**：使用 MSW 等 mock 库模拟一致的网络响应
- **随机数据**：在测试中使用固定数据，或使用 `faker.seed()` 确保可重复

## 小结

测试金字塔的每一层都有其独特的价值：单元测试提供快速的反馈，组件测试验证用户交互的正确性，E2E 测试确保关键流程的端到端可用性。结合 React DevTools 和 Chrome DevTools 等调试工具，你可以高效地定位和修复问题。

记住测试的核心原则：**测试行为，而非实现。** 遵循这一原则，你的测试套件将更加稳健、可维护，并在重构时真正发挥价值。
