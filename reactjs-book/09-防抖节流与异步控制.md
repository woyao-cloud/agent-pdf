# 第九章：防抖、节流与异步控制

在现代 Web 应用中，异步操作无处不在：搜索建议、表单提交、数据获取、实时更新等。如果不加控制，这些异步操作可能导致性能问题、竞态条件和糟糕的用户体验。本章将系统性地介绍防抖（Debounce）、节流（Throttle）、限流（Rate Limiting）以及各种异步控制策略，帮助你在实际项目中应对复杂的异步场景。

## 9.1 防抖（Debounce）

### 9.1.1 概念与原理

防抖的核心思想是：**将多次连续触发合并为一次执行**。具体来说，当事件被触发后，不会立即执行回调函数，而是等待一段时间。如果在这段时间内事件再次被触发，则重新计时。只有最后一次事件触发且等待时间结束后，回调函数才会执行。

防抖的典型应用场景：
- **搜索输入框**：用户停止输入 300ms 后发起搜索请求
- **自动保存**：用户停止编辑 1 秒后自动保存草稿
- **表单验证**：用户输入完成后验证邮箱格式
- **窗口大小调整**：窗口停止调整后重新计算布局

### 9.1.2 基础实现

```typescript
function debounce<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout> | null = null;

  return (...args: Parameters<T>) => {
    if (timer) {
      clearTimeout(timer);
    }
    timer = setTimeout(() => {
      fn(...args);
      timer = null;
    }, delay);
  };
}

// 使用示例
const handleSearch = debounce((query: string) => {
  fetchSearchResults(query);
}, 300);
```

### 9.1.3 React 中的 useDebounce Hook

在 React 中，我们需要一个专用的 Hook 来管理防抖值。这个 Hook 会返回一个"延迟更新"的值：

```typescript
import { useState, useEffect } from "react";

/**
 * 防抖 Hook
 * @param value 需要防抖的值
 * @param delay 延迟时间（毫秒）
 * @returns 防抖后的值
 */
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);

  useEffect(() => {
    // 设置定时器，在 delay 毫秒后更新防抖值
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    // 如果 value 或 delay 变化，清除之前的定时器
    return () => {
      clearTimeout(timer);
    };
  }, [value, delay]);

  return debouncedValue;
}
```

### 9.1.4 实战：防抖搜索输入框

下面是一个完整的防抖搜索示例，包含加载状态、缓存和错误处理：

```tsx
import { useState, useEffect, useCallback, useRef } from "react";

interface SearchResult {
  id: number;
  title: string;
  description: string;
}

// 防抖 Hook（同上）
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
}

// 搜索 Hook：封装搜索逻辑
function useSearch() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<SearchResult[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  // 使用防抖，300ms 后更新搜索
  const debouncedQuery = useDebounce(query, 300);

  // 缓存上一次请求，用于竞态处理
  const latestRequestRef = useRef(0);

  useEffect(() => {
    if (!debouncedQuery.trim()) {
      setResults([]);
      setError(null);
      return;
    }

    const requestId = ++latestRequestRef.current;

    setIsLoading(true);
    setError(null);

    fetch(`/api/search?q=${encodeURIComponent(debouncedQuery)}`)
      .then((res) => {
        if (!res.ok) throw new Error("搜索请求失败");
        return res.json();
      })
      .then((data: SearchResult[]) => {
        // 竞态检查：只处理最新的请求
        if (requestId === latestRequestRef.current) {
          setResults(data);
          setIsLoading(false);
        }
      })
      .catch((err) => {
        if (requestId === latestRequestRef.current) {
          setError(err.message);
          setIsLoading(false);
        }
      });
  }, [debouncedQuery]);

  return {
    query,
    setQuery,
    results,
    isLoading,
    error,
  };
}

// 搜索组件
function SearchPage() {
  const { query, setQuery, results, isLoading, error } = useSearch();

  return (
    <div className="max-w-2xl mx-auto p-6">
      <div className="relative">
        <input
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="搜索文章..."
          className="w-full px-4 py-3 border rounded-lg text-lg"
        />

        {/* 加载指示器 */}
        {isLoading && (
          <div className="absolute right-3 top-3">
            <Spinner />
          </div>
        )}
      </div>

      {/* 错误提示 */}
      {error && (
        <div className="mt-4 p-3 bg-red-50 text-red-700 rounded">
          搜索出错：{error}
        </div>
      )}

      {/* 搜索结果 */}
      <div className="mt-6 space-y-4">
        {results.length === 0 && !isLoading && query && (
          <p className="text-gray-500">未找到相关结果</p>
        )}

        {results.map((result) => (
          <article
            key={result.id}
            className="p-4 border rounded-lg hover:shadow-md transition-shadow"
          >
            <h3 className="text-lg font-semibold">{result.title}</h3>
            <p className="text-gray-600 mt-1">{result.description}</p>
          </article>
        ))}
      </div>
    </div>
  );
}
```

### 9.1.5 防抖的进阶模式

**前缘防抖（Immediate / Leading Debounce）**：立即执行第一次，然后防抖：

```typescript
function debounceLeading<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout> | null = null;
  let leadingExecuted = false;

  return (...args: Parameters<T>) => {
    if (!leadingExecuted) {
      fn(...args);
      leadingExecuted = true;
    }

    if (timer) {
      clearTimeout(timer);
    }

    timer = setTimeout(() => {
      leadingExecuted = false;
      timer = null;
    }, delay);
  };
}
```

**带取消功能的防抖**：

```typescript
interface DebouncedFunction<T extends (...args: any[]) => any> {
  (...args: Parameters<T>): void;
  cancel: () => void;
  flush: () => void;
}

function debounceWithCancel<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): DebouncedFunction<T> {
  let timer: ReturnType<typeof setTimeout> | null = null;
  let lastArgs: Parameters<T> | null = null;

  const debounced = (...args: Parameters<T>) => {
    lastArgs = args;
    if (timer) clearTimeout(timer);

    timer = setTimeout(() => {
      fn(...args);
      timer = null;
      lastArgs = null;
    }, delay);
  };

  debounced.cancel = () => {
    if (timer) {
      clearTimeout(timer);
      timer = null;
      lastArgs = null;
    }
  };

  debounced.flush = () => {
    if (timer && lastArgs) {
      clearTimeout(timer);
      fn(...lastArgs);
      timer = null;
      lastArgs = null;
    }
  };

  return debounced;
}
```

## 9.2 节流（Throttle）

### 9.2.1 概念与原理

节流的核心思想是：**在指定的时间间隔内，无论事件触发多少次，只执行一次回调函数**。与防抖不同，节流保证在一定时间内至少执行一次，适合需要持续反馈的场景。

节流的典型应用场景：
- **滚动事件**：监听滚动位置以更新导航栏样式
- **窗口大小调整**：调整过程中更新布局
- **拖拽事件**：拖拽过程中更新位置
- **鼠标移动**：跟踪鼠标位置（如自定义光标效果）

### 9.2.2 基础实现

```typescript
function throttle<T extends (...args: any[]) => any>(
  fn: T,
  interval: number
): (...args: Parameters<T>) => void {
  let lastTime = 0;

  return (...args: Parameters<T>) => {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn(...args);
    }
  };
}
```

### 9.2.3 React 中的 useThrottle Hook

```typescript
import { useState, useEffect, useRef } from "react";

/**
 * 节流 Hook
 * @param value 需要节流的值
 * @param interval 时间间隔（毫秒）
 * @returns 节流后的值
 */
function useThrottle<T>(value: T, interval: number): T {
  const [throttledValue, setThrottledValue] = useState<T>(value);
  const lastUpdated = useRef(0);

  useEffect(() => {
    const now = Date.now();

    if (now - lastUpdated.current >= interval) {
      // 距离上次更新超过了间隔，立即更新
      lastUpdated.current = now;
      setThrottledValue(value);
    } else {
      // 否则设置定时器，在间隔到达时更新
      const remaining = interval - (now - lastUpdated.current);
      const timer = setTimeout(() => {
        lastUpdated.current = Date.now();
        setThrottledValue(value);
      }, remaining);

      return () => clearTimeout(timer);
    }
  }, [value, interval]);

  return throttledValue;
}
```

### 9.2.4 实战：滚动监听与阅读进度

```tsx
function useReadingProgress(): number {
  const [progress, setProgress] = useState(0);

  // 使用节流，每 100ms 最多更新一次
  const throttledProgress = useThrottle(progress, 100);

  useEffect(() => {
    function handleScroll() {
      const scrollTop = window.scrollY;
      const docHeight = document.documentElement.scrollHeight;
      const winHeight = window.innerHeight;
      const totalScrollable = docHeight - winHeight;

      if (totalScrollable > 0) {
        const currentProgress = (scrollTop / totalScrollable) * 100;
        setProgress(Math.min(100, Math.max(0, currentProgress)));
      }
    }

    window.addEventListener("scroll", handleScroll, { passive: true });
    return () => window.removeEventListener("scroll", handleScroll);
  }, []);

  return throttledProgress;
}

// 阅读进度条组件
function ReadingProgressBar() {
  const progress = useReadingProgress();

  return (
    <div className="fixed top-0 left-0 w-full h-1 bg-gray-200 z-50">
      <div
        className="h-full bg-blue-500 transition-all duration-150"
        style={{ width: `${progress}%` }}
      />
    </div>
  );
}
```

### 9.2.5 防抖 vs 节流：如何选择

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 搜索输入建议 | 防抖 | 等待用户输入完成再查询 |
| 自动保存草稿 | 防抖 | 用户停止编辑后再保存 |
| 窗口 resize | 节流 | 持续反馈，但限制频率 |
| 滚动加载更多 | 节流 | 保证定期检查是否需要加载 |
| 表单按钮防重复点击 | 防抖或节流 | 防抖更严格，节流更宽容 |
| 实时数据同步 | 节流 | 保证定期发送最新数据 |

简单规则：
- 需要 **最终结果**（搜索、保存） -> 使用 **防抖**
- 需要 **持续反馈**（滚动、拖拽） -> 使用 **节流**

## 9.3 限流（Rate Limiting）

### 9.3.1 概念

限流与防抖和节流不同，它关注的是**在单位时间内最多允许执行多少次操作**。即使操作被频繁调用，超过限制的操作也会被拒绝或排队。

### 9.3.2 自定义 useRateLimiter Hook

```typescript
import { useRef, useCallback } from "react";

interface RateLimiterOptions {
  maxRequests: number;   // 最大请求数
  windowMs: number;      // 时间窗口（毫秒）
}

function useRateLimiter({ maxRequests, windowMs }: RateLimiterOptions) {
  const timestamps = useRef<number[]>([]);

  const isAllowed = useCallback((): boolean => {
    const now = Date.now();
    // 清除过期的时间戳
    timestamps.current = timestamps.current.filter(
      (ts) => now - ts < windowMs
    );

    if (timestamps.current.length >= maxRequests) {
      return false;
    }

    timestamps.current.push(now);
    return true;
  }, [maxRequests, windowMs]);

  const remainingRequests = useCallback((): number => {
    const now = Date.now();
    timestamps.current = timestamps.current.filter(
      (ts) => now - ts < windowMs
    );
    return Math.max(0, maxRequests - timestamps.current.length);
  }, [maxRequests, windowMs]);

  const reset = useCallback(() => {
    timestamps.current = [];
  }, []);

  return { isAllowed, remainingRequests, reset };
}
```

### 9.3.3 实战：防止重复提交

```tsx
import { useActionState } from "react";

function usePreventDoubleSubmit() {
  const submittingRef = useRef(false);

  const withGuard = useCallback(
    async <T,>(action: () => Promise<T>): Promise<T | undefined> => {
      if (submittingRef.current) {
        return; // 已经在提交中，忽略本次请求
      }

      submittingRef.current = true;
      try {
        return await action();
      } finally {
        // 使用 setTimeout 确保在下一个事件循环中释放锁
        setTimeout(() => {
          submittingRef.current = false;
        }, 1000);
      }
    },
    []
  );

  return withGuard;
}

function OrderForm() {
  const withGuard = usePreventDoubleSubmit();
  const [state, formAction, isPending] = useActionState(
    async (prev: unknown, formData: FormData) => {
      return withGuard(async () => {
        await submitOrder(formData);
        return { success: true };
      });
    },
    null
  );

  return (
    <form action={formAction}>
      {/* 表单内容 */}
      <button
        type="submit"
        disabled={isPending}
        className="px-6 py-3 bg-green-600 text-white rounded"
      >
        {isPending ? "提交中..." : "提交订单"}
      </button>
    </form>
  );
}
```

## 9.4 竞态条件处理

### 9.4.1 什么是竞态条件

竞态条件（Race Condition）在异步编程中是指：**多个异步操作的结果到达的顺序与它们发起时的顺序不一致**。

```tsx
// 竞态条件示例
function SearchResults({ query }: { query: string }) {
  const [results, setResults] = useState([]);

  useEffect(() => {
    fetch(`/api/search?q=${query}`)
      .then((res) => res.json())
      .then((data) => {
        // 问题：如果 query 从 "a" 变为 "ab" 再变为 "abc"，
        // 三个请求可能以任意顺序返回
        // 如果 "ab" 的响应在 "abc" 之后到达，搜索结果将是错误的
        setResults(data);
      });
  }, [query]);

  return <ResultsList data={results} />;
}
```

### 9.4.2 解决方案一：AbortController

AbortController 是处理竞态条件最优雅的方式之一。每次发起新请求时，取消上一个未完成的请求：

```tsx
function useSearchWithAbort() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<SearchResult[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  // 使用防抖减少请求次数
  const debouncedQuery = useDebounce(query, 300);

  useEffect(() => {
    if (!debouncedQuery.trim()) {
      setResults([]);
      return;
    }

    // 创建 AbortController
    const abortController = new AbortController();

    setIsLoading(true);
    setError(null);

    fetch(`/api/search?q=${encodeURIComponent(debouncedQuery)}`, {
      signal: abortController.signal,
    })
      .then((res) => {
        if (!res.ok) throw new Error("搜索失败");
        return res.json();
      })
      .then((data) => {
        setResults(data);
        setIsLoading(false);
      })
      .catch((err) => {
        // 忽略取消请求的错误
        if (err.name === "AbortError") return;
        setError(err.message);
        setIsLoading(false);
      });

    // 清理函数：取消未完成的请求
    return () => {
      abortController.abort();
    };
  }, [debouncedQuery]);

  return { query, setQuery, results, isLoading, error };
}
```

### 9.4.3 解决方案二：useRef 标志位

使用 `useRef` 跟踪请求的"新鲜度"：

```tsx
function useLatestRequest() {
  const [data, setData] = useState<Data | null>(null);
  const requestIdRef = useRef(0);

  const fetchData = useCallback(async (url: string) => {
    const requestId = ++requestIdRef.current;

    try {
      const response = await fetch(url);
      const result = await response.json();

      // 只有最新的请求才能更新状态
      if (requestId === requestIdRef.current) {
        setData(result);
      }
    } catch (error) {
      if (requestId === requestIdRef.current) {
        // 只有最新的请求才处理错误
        console.error("请求失败:", error);
      }
    }
  }, []);

  return { data, fetchData };
}
```

### 9.4.4 解决方案三：useReducer 模式

`useReducer` 天然适合管理复杂的状态更新，可以很好地处理竞态条件：

```typescript
type Action =
  | { type: "FETCH_START" }
  | { type: "FETCH_SUCCESS"; payload: Data }
  | { type: "FETCH_ERROR"; error: string };

interface State {
  data: Data | null;
  isLoading: boolean;
  error: string | null;
  latestRequest: number;
}

function reducer(state: State, action: Action): State {
  switch (action.type) {
    case "FETCH_START":
      return {
        ...state,
        isLoading: true,
        error: null,
        latestRequest: Date.now(), // 每次请求都有唯一标识
      };
    case "FETCH_SUCCESS":
      return {
        ...state,
        data: action.payload,
        isLoading: false,
        error: null,
      };
    case "FETCH_ERROR":
      return {
        ...state,
        isLoading: false,
        error: action.error,
      };
    default:
      return state;
  }
}

function useDataFetching() {
  const [state, dispatch] = useReducer(reducer, {
    data: null,
    isLoading: false,
    error: null,
    latestRequest: 0,
  });

  const fetchData = useCallback(async (url: string) => {
    const requestTime = Date.now();
    dispatch({ type: "FETCH_START" });

    try {
      const response = await fetch(url);
      const data = await response.json();

      // 检查是否是当前请求的响应
      dispatch({
        type: "FETCH_SUCCESS",
        payload: data,
        meta: { requestTime }, // 可以携带额外信息
      });
    } catch (error) {
      dispatch({
        type: "FETCH_ERROR",
        error: error instanceof Error ? error.message : "未知错误",
      });
    }
  }, []);

  return { ...state, fetchData };
}
```

### 9.4.5 解决方案四：useTransition

React 19 的 `useTransition` 也可以帮助处理竞态条件，通过将更新标记为非紧急：

```tsx
import { useState, useTransition } from "react";

function SearchWithTransition() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<SearchResult[]>([]);
  const [isPending, startTransition] = useTransition();

  function handleSearch(newQuery: string) {

    startTransition(async () => {
      try {
        const data = await fetchSearchResults(newQuery);
        // 如果在此期间 query 再次变化，此更新将被中断
        setResults(data);
      } catch (error) {
        console.error("搜索失败:", error);
      }
    });
  }

  return (
    <div>
      <input
        value={query}
        onChange={(e) => handleSearch(e.target.value)}
        placeholder="搜索..."
      />
      {isPending && <Spinner />}
      <ResultsList data={results} />
    </div>
  );
}
```

## 9.5 并发请求管理

### 9.5.1 请求去重

当多个组件同时请求相同的数据时，应该避免重复的网络请求：

```typescript
class RequestDeduplicator {
  private pendingRequests = new Map<string, Promise<any>>();

  async fetch<T>(key: string, fetcher: () => Promise<T>): Promise<T> {
    // 如果已有相同的请求在进行中，复用该 Promise
    if (this.pendingRequests.has(key)) {
      return this.pendingRequests.get(key)!;
    }

    const promise = fetcher().finally(() => {
      this.pendingRequests.delete(key);
    });

    this.pendingRequests.set(key, promise);
    return promise;
  }
}

// 全局单例
const deduplicator = new RequestDeduplicator();

// 在 Hook 中使用
function useUserData(userId: string) {
  const [user, setUser] = useState<User | null>(null);

  useEffect(() => {
    deduplicator
      .fetch(`user-${userId}`, () =>
        fetch(`/api/users/${userId}`).then((r) => r.json())
      )
      .then(setUser);
  }, [userId]);

  return user;
}
```

```tsx
import { useState, useEffect, useCallback, useRef } from "react";

// 1. 防抖 Hook
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debouncedValue;
}

// 2. 请求去重器
const requestCache = new Map<string, Promise<SearchResult[]>>();

function deduplicatedFetch(url: string): Promise<SearchResult[]> {
  if (requestCache.has(url)) {
    return requestCache.get(url)!;
  }
  const promise = fetch(url)
    .then((r) => r.json())
    .finally(() => requestCache.delete(url));
  requestCache.set(url, promise);
  return promise;
}

// 3. 完整搜索 Hook
interface SearchState {
  results: SearchResult[];
  isLoading: boolean;
  isFirstLoad: boolean;
  error: string | null;
}

function useRobustSearch() {
  const [query, setQuery] = useState("");
  const [state, setState] = useState<SearchState>({
    results: [],
    isLoading: false,
    isFirstLoad: true,
    error: null,
  });

  const debouncedQuery = useDebounce(query, 300);
  const abortRef = useRef<AbortController | null>(null);

  useEffect(() => {
    // 取消上一次请求
    abortRef.current?.abort();
    abortRef.current = new AbortController();

    if (!debouncedQuery.trim()) {
      setState((prev) => ({
        ...prev,
        results: [],
        isLoading: false,
        isFirstLoad: false,
        error: null,
      }));
      return;
    }

    setState((prev) => ({ ...prev, isLoading: true, error: null }));

    const url = `/api/search?q=${encodeURIComponent(debouncedQuery)}`;

    deduplicatedFetch(url)
      .then((data) => {
        setState({
          results: data,
          isLoading: false,
          isFirstLoad: false,
          error: null,
        });
      })
      .catch((err) => {
        if (err.name === "AbortError") return;
        setState((prev) => ({
          ...prev,
          isLoading: false,
          isFirstLoad: false,
          error: err.message || "搜索失败",
        }));
      });
  }, [debouncedQuery]);

  // 清理
  useEffect(() => {
    return () => abortRef.current?.abort();
  }, []);

  return { query, setQuery, ...state };
}

// 4. 搜索 UI 组件
function RobustSearch() {
  const { query, setQuery, results, isLoading, isFirstLoad, error } =
    useRobustSearch();

  return (
    <div className="max-w-3xl mx-auto p-4">
      <div className="relative mb-6">
        <input
          type="search"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="输入关键词搜索..."
          className="w-full px-4 py-3 pr-12 border-2 border-gray-200 rounded-xl
                     focus:border-blue-500 focus:outline-none transition-colors
                     text-lg"
          aria-label="搜索输入框"
        />
        {isLoading && (
          <div className="absolute right-4 top-1/2 -translate-y-1/2">
            <LoadingSpinner />
          </div>
        )}
      </div>

      {/* 错误状态 */}
      {error && (
        <div className="mb-4 p-4 bg-red-50 border border-red-200 rounded-lg">
          <p className="text-red-700">{error}</p>
        </div>
      )}

      {/* 首次加载提示 */}
      {isFirstLoad && !query && (
        <div className="text-center py-12 text-gray-400">
          <SearchIcon className="w-16 h-16 mx-auto mb-4" />
          <p>输入关键词开始搜索</p>
        </div>
      )}

      {/* 空结果 */}
      {!isLoading && !isFirstLoad && query && results.length === 0 && (
        <div className="text-center py-12 text-gray-500">
          <p>未找到与 "{query}" 相关的结果</p>
        </div>
      )}

      {/* 搜索结果 */}
      <div className="space-y-3">
        {results.map((result) => (
          <SearchResultCard key={result.id} result={result} />
        ))}
      </div>
    </div>
  );
}
```

## 9.7 本章小结

本章全面介绍了 React 应用中异步控制的四种核心模式：

| 模式 | 适用场景 | 核心思想 |
|------|---------|---------|
| **防抖（Debounce）** | 搜索输入、自动保存 | 等待稳定后再执行 |
| **节流（Throttle）** | 滚动、拖拽、resize | 定期执行，保证频率 |
| **限流（Rate Limiting）** | API 调用、表单提交 | 单位时间内限制次数 |
| **竞态处理** | 搜索、分页、筛选 | 保证最新请求的结果 |

选择策略的决策流程：

1. **这个操作需要最终结果还是持续反馈？**
   - 最终结果 -> 防抖
   - 持续反馈 -> 节流

2. **是否有外部 API 调用限制？**
   - 有 -> 防抖 + 限流

3. **是否可能有竞态条件？**
   - 是 -> AbortController + useRef 标志位

4. **是否需要缓存和去重？**
   - 是 -> SWR / TanStack Query / 手动缓存

异步控制是构建高质量 React 应用的必备技能。掌握这些模式不仅能解决实际问题，还能帮助你写出更健壮、用户体验更好的代码。
