## 11.1 Milvus

### 11.1.1 概述与架构

Milvus是全球最受欢迎的开源向量数据库，专为十亿级向量检索设计，由Zilliz公司开发并捐赠给LF AI & Data基金会。

```python
# ===== Milvus 完整实战示例 =====

from pymilvus import (
    connections, Collection, FieldSchema, CollectionSchema,
    DataType, utility, AnnSearchRequest, WeightedRanker
)
import numpy as np

# ---- 1. 连接 ----
# Milvus Lite（嵌入式，自动启动）
connections.connect("default", uri="milvus_demo.db")

# 或连接远程Milvus
# connections.connect("default", host="localhost", port=19530)

# ---- 2. 定义Schema ----
fields = [
    FieldSchema(name="id", dtype=DataType.INT64,
                is_primary=True, auto_id=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR,
                dim=1024),
    FieldSchema(name="content", dtype=DataType.VARCHAR,
                max_length=65535),
    FieldSchema(name="source", dtype=DataType.VARCHAR,
                max_length=256),
    FieldSchema(name="page", dtype=DataType.INT32),
    FieldSchema(name="chapter", dtype=DataType.VARCHAR,
                max_length=128),
]

schema = CollectionSchema(
    fields=fields,
    description="产品文档知识库"
)

# ---- 3. 创建集合 ----
collection_name = "product_docs"

# 删除已存在的同名集合（开发时常用）
if utility.has_collection(collection_name):
    utility.drop_collection(collection_name)

collection = Collection(
    name=collection_name,
    schema=schema
)

# ---- 4. 创建索引 ----
index_params = {
    "index_type": "HNSW",
    "metric_type": "COSINE",
    "params": {
        "M": 32,
        "efConstruction": 200
    }
}

collection.create_index(
    field_name="embedding",
    index_params=index_params
)

# 为元数据字段创建标量索引（加速过滤）
collection.create_index(field_name="source")
collection.create_index(field_name="chapter")

# ---- 5. 插入数据 ----
# 模拟数据
num_docs = 1000
embeddings = np.random.randn(num_docs, 1024).astype('float32')
# 归一化
embeddings = embeddings / np.linalg.norm(embeddings, axis=1, keepdims=True)

contents = [f"这是第{i}个文档段落的文本内容" for i in range(num_docs)]
sources = [f"manual_{i % 5}.pdf" for i in range(num_docs)]
pages = [i % 200 for i in range(num_docs)]
chapters = [f"第{i % 10}章" for i in range(num_docs)]

data = [
    embeddings.tolist(),
    contents,
    sources,
    pages,
    chapters,
]

collection.insert(data)
collection.flush()  # 确保数据持久化

print(f"插入 {collection.num_entities} 条数据")

# ---- 6. 向量搜索 ----
collection.load()  # 加载集合到内存

# 生成查询向量
query_embedding = np.random.randn(1, 1024).astype('float32')
query_embedding = query_embedding / np.linalg.norm(query_embedding)

# 基础向量搜索
search_params = {
    "metric_type": "COSINE",
    "params": {"ef": 64}  # HNSW搜索参数
}

results = collection.search(
    data=query_embedding.tolist(),
    anns_field="embedding",
    param=search_params,
    limit=5,
    output_fields=["content", "source", "page"]
)

print("=== 向量搜索结果 ===")
for hits in results:
    for hit in hits:
        print(f"  ID: {hit.id}, 距离: {hit.distance:.4f}, "
              f"来源: {hit.entity.get('source')}, "
              f"页码: {hit.entity.get('page')}")
        print(f"  内容: {hit.entity.get('content')[:50]}...")

# ---- 7. 带元数据过滤的搜索 ----
results_filtered = collection.search(
    data=query_embedding.tolist(),
    anns_field="embedding",
    param=search_params,
    limit=5,
    expr='source == "manual_0.pdf" and page > 50',
    output_fields=["content", "source", "page", "chapter"]

print("\n=== 过滤搜索结果 (manual_0.pdf, page>50) ===")
for hits in results_filtered:
    for hit in hits:
        print(f"  ID: {hit.id}, 距离: {hit.distance:.4f}, "
              f"来源: {hit.entity.get('source')}, "
              f"页码: {hit.entity.get('page')}")

# ---- 8. 混合搜索（多向量字段） ----
# Milvus 2.4+ 支持多向量字段和混合搜索
# 添加第二个向量字段用于关键词嵌入（如BM25稀疏向量）
sparse_field = FieldSchema(
    name="sparse_embedding",
    dtype=DataType.SPARSE_FLOAT_VECTOR  # 稀疏向量

# 注意：需要重新创建集合以添加稀疏向量字段
```

## 11.2 Weaviate

### 11.2.1 概述与架构

Weaviate是一个原生支持向量化的向量数据库，其独特之处在于**内置向量化模块**——可以直接存储文本，由Weaviate自动调用嵌入模型生成向量，无需用户自行计算嵌入。

```
Weaviate 架构：
┌──────────────────────────────────────────────────────────────┐
│                      Weaviate                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌───────────────────────────────────────────────────────┐   │
│  │                   GraphQL / REST API                   │   │
│  └───────────────────────┬───────────────────────────────┘   │
│                          │                                   │
│  ┌───────────────────────▼───────────────────────────────┐   │
│  │                    核心引擎                            │   │
│  │  ─────────────────────────────────────────────────── │   │
│  │  向量索引 (HNSW) │ 对象存储 │ 过滤引擎               │   │
│  └───────────────────────┬───────────────────────────────┘   │
│                          │                                   │
│  ┌───────────────────────▼───────────────────────────────┐   │
│  │                    模块系统                            │   │
│  │  ─────────────────────────────────────────────────── │   │
│  │  向量化模块:  OpenAI │ Cohere │ HuggingFace │ 本地   │   │
│  │  生成模块:    OpenAI │ Cohere │ 自定义                   │   │
│  │  重排序模块:  Cohere Rerank │ Jina Reranker            │   │
│  └───────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

**核心特色**：
- **GraphQL API**：所有操作通过GraphQL完成
- **内置向量化**：存储文本时自动生成嵌入
- **生成式搜索**：RAG原生集成，检索+生成一步完成
- **多租户**：原生支持SaaS多租户场景

### 11.2.3 代码实战

```python
# ===== Weaviate 完整实战示例 =====

import weaviate
from weaviate.classes.config import Configure, Property, DataType
from weaviate.classes.query import MetadataQuery

# ---- 1. 连接 ----
# 连接本地Weaviate（使用OpenAI向量化模块）
client = weaviate.connect_to_local(
    host="localhost",
    port=8080,
    headers={"X-OpenAI-Api-Key": "your-openai-api-key"}
)

# ---- 2. 创建集合 ----
# Weaviate的独特之处：可以指定向量化模块，存储时自动生成嵌入
collection_name = "ProductDoc"

# 检查并删除已存在的集合
if client.collections.exists(collection_name):
    client.collections.delete(collection_name)

collection = client.collections.create(
    name=collection_name,
    vectorizer_config=Configure.Vectorizer.text2vec_openai(
        model="text-embedding-3-small",
        dimensions=1536
    ),
    generative_config=Configure.Generative.openai(
        model="gpt-4o-mini"
    ),
    properties=[
        Property(name="content", data_type=DataType.TEXT),
        Property(name="source", data_type=DataType.TEXT),
        Property(name="page", data_type=DataType.INT),
        Property(name="chapter", data_type=DataType.TEXT),
    ]
)

# ---- 3. 插入数据 ----
# 注意：只需提供文本，Weaviate自动调用OpenAI生成嵌入！
docs = [
    {"content": "要配置网络连接，请前往设置页面，选择网络选项，输入WiFi密码。",
     "source": "manual_v2.pdf", "page": 15, "chapter": "网络配置"},
    {"content": "重置密码的步骤：1.点击登录页面的'忘记密码' 2.输入注册邮箱 3.查收重置邮件",
     "source": "manual_v2.pdf", "page": 22, "chapter": "账户管理"},
    {"content": "系统支持批量导入数据，最大文件大小为500MB，支持CSV和JSON格式。",
     "source": "admin_guide.pdf", "page": 8, "chapter": "数据管理"},
    {"content": "网络故障排查：首先检查物理连接，然后运行ping命令测试网络连通性。",
     "source": "troubleshoot.pdf", "page": 45, "chapter": "故障排查"},
]

with collection.batch.dynamic() as batch:
    for doc in docs:
        batch.add_object(properties=doc)

print(f"插入 {len(docs)} 条数据")

# ---- 4. 语义搜索 ----
results = collection.query.hybrid(
    query="如何设置网络",
    alpha=0.7,  # 0=纯关键词(BM25), 1=纯向量, 0.7=混合
    limit=3,
    return_metadata=MetadataQuery(score=True),
    include_vector=False
)

print("=== 混合搜索结果 ===")
for obj in results.objects:
    print(f"  分数: {obj.metadata.score:.4f}")
    print(f"  内容: {obj.properties['content'][:50]}...")
    print(f"  来源: {obj.properties['source']}, "
          f"页码: {obj.properties['page']}")
    print()

# ---- 5. 元数据过滤搜索 ----
results_filtered = collection.query.hybrid(
    query="网络配置",
    alpha=0.7,
    filters=weaviate.classes.query.Filter.by_property("source").equal("manual_v2.pdf"),
    limit=3,
    return_metadata=MetadataQuery(score=True)
)

print("=== 过滤搜索结果 (manual_v2.pdf) ===")
for obj in results_filtered.objects:
    print(f"  来源: {obj.properties['source']}, "
          f"分数: {obj.metadata.score:.4f}")

# ---- 6. 生成式搜索（RAG原生支持） ----
# Weaviate内置了RAG能力：检索+生成一步完成
generative_results = collection.generate.hybrid(
    query="如何配置网络连接？",
    alpha=0.7,
    limit=3,
    single_prompt="基于以下文档内容回答问题。问题: {query} 文档: {content}"
)

print("=== 生成式搜索结果 ===")
for obj in generative_results.objects:
    print(f"  生成回答: {obj.generated}")
    print(f"  引用来源: {obj.properties['source']}, "
          f"页码: {obj.properties['page']}")
    print()

# ---- 7. 纯向量搜索（近邻搜索） ----
near_vector_results = collection.query.near_text(
    query="网络连接设置",
    limit=3,
    return_metadata=MetadataQuery(distance=True),
    filters=weaviate.classes.query.Filter.by_property("page").greater_than(10)
)

print("=== 近邻搜索结果 (page > 10) ===")
for obj in near_vector_results.objects:
    print(f"  距离: {obj.metadata.distance:.4f}, "
          f"页码: {obj.properties['page']}")

# ---- 8. 多租户示例 ----
# 创建多租户集合
tenant_collection = client.collections.create(
    name="TenantDocs",
    multi_tenancy_config=Configure.multi_tenancy(enabled=True),
    vectorizer_config=Configure.Vectorizer.text2vec_openai()
)

# 为每个租户创建独立的向量空间
tenant_collection.multi_tenancy.create(tenant="company_a")
tenant_collection.multi_tenancy.create(tenant="company_b")

# 插入数据到特定租户
tenant_collection.data.insert(
    properties={"content": "Company A的内部文档..."},
    tenant="company_a"
)

# 搜索特定租户的数据
results = tenant_collection.query.hybrid(
    query="内部文档",
    tenant="company_a",  # 只搜索company_a的数据
    limit=5
)

# ---- 9. 清理 ----
client.collections.delete(collection_name)
client.close()
```

## 11.3 Qdrant

### 11.3.1 概述与架构

Qdrant是用Rust编写的向量数据库，以**高性能和低资源占用**著称。其核心优势在于Rust的零成本抽象和内存安全，以及在量化压缩方面的优秀支持。

```
Qdrant 架构：
┌──────────────────────────────────────────────────────────────┐
│                       Qdrant                                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌───────────────────────────────────────────────────────┐   │
│  │                    REST / gRPC API                     │   │
│  └───────────────────────┬───────────────────────────────┘   │
│                          │                                   │
│  ┌───────────────────────▼───────────────────────────────┐   │
│  │                   存储层                               │   │
│  │  ─────────────────────────────────────────────────── │   │
│  │  Collection (集合)                                    │   │
│  │  ├── Points (数据点)                                  │   │
│  │  │   ├── Vector (向量)                                │   │
│  │  │   └── Payload (元数据)                             │   │
│  │  ├── HNSW Index (向量索引)                            │   │
│  │  ├── WAL (预写日志)                                   │   │
│  │  └── Quantization (量化)                              │   │
│  │       ├── Scalar (标量量化)                            │   │
│  │       ├── Product (乘积量化)                           │   │
│  │       └── Binary (二值量化)                            │   │
│  └───────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

### 11.3.3 代码实战

```python
# ===== Qdrant 完整实战示例 =====

from qdrant_client import QdrantClient
from qdrant_client.models import (
    Distance, VectorParams, PointStruct,
    Filter, FieldCondition, MatchValue,
    SearchRequest, ScalarQuantization, ScalarType,
    ProductQuantization, BinaryQuantization
)
import numpy as np

# ---- 1. 连接 ----
client = QdrantClient(host="localhost", port=6333)
# 嵌入式: client = QdrantClient(":memory:")

# ---- 2. 创建集合 ----
collection_name = "product_docs"

# 删除已有集合
client.delete_collection(collection_name=collection_name)

client.create_collection(
    collection_name=collection_name,
    vectors_config=VectorParams(
        size=1024,
        distance=Distance.COSINE
    ),
    # 启用标量量化（减少内存占用）
    quantization_config=ScalarQuantization(
        scalar=ScalarType.INT8,
        quantile=0.99,  # 99%的值在量化范围内
        always_ram=True  # 量化向量始终在RAM中
    )
)

# ---- 3. 插入数据 ----
num_docs = 1000
embeddings = np.random.randn(num_docs, 1024).astype('float32')
embeddings = embeddings / np.linalg.norm(embeddings, axis=1, keepdims=True)

points = [
    PointStruct(
        id=i,
        vector=embeddings[i].tolist(),
        payload={
            "content": f"第{i}个文档段落的文本内容",
            "source": f"manual_{i % 5}.pdf",
            "page": i % 200,
            "chapter": f"第{i % 10}章",
        }
    )
    for i in range(num_docs)
]

# 批量插入
client.upsert(
    collection_name=collection_name,
    points=points
)

print(f"插入 {num_docs} 条数据")

# ---- 4. 向量搜索 ----
query_embedding = np.random.randn(1024).astype('float32')
query_embedding = query_embedding / np.linalg.norm(query_embedding)

results = client.query_points(
    collection_name=collection_name,
    query=query_embedding.tolist(),
    limit=5,
    with_payload=True
)

print("=== 向量搜索结果 ===")
for point in results.points:
    print(f"  ID: {point.id}, 分数: {point.score:.4f}, "
```

## 11.4 Chroma

### 11.4.1 概述

Chroma（chromadb）是一个**轻量级、Python原生**的向量数据库，专为AI应用开发和原型验证设计。它的设计哲学是"零配置、开箱即用"，用最少的代码完成向量检索。

### 11.5.3 代码实战

```python
# ===== pgvector 完整实战示例 =====

import psycopg2
from pgvector.psycopg2 import register_vector
import numpy as np

# ---- 1. 连接 ----
conn = psycopg2.connect(
    host="localhost",
    port=5432,
    dbname="ragdb",
    user="raguser",
    password="ragpass"
)

# 注册vector类型
register_vector(conn)

cur = conn.cursor()

# ---- 2. 启用pgvector扩展 ----
cur.execute("CREATE EXTENSION IF NOT EXISTS vector")
conn.commit()

# ---- 3. 创建表 ----
cur.execute("DROP TABLE IF EXISTS product_docs")

cur.execute("""
    CREATE TABLE product_docs (
        id SERIAL PRIMARY KEY,
        content TEXT NOT NULL,
        source VARCHAR(256),
        page INTEGER,
        chapter VARCHAR(128),
        embedding vector(1024)
    )
""")
conn.commit()

# ---- 4. 创建索引 ----
# IVFFlat索引（适合中等规模数据）
# 注意：IVFFlat索引需要先有足够数据才能训练
# 先插入数据，再建索引

# HNSW索引（推荐，无需训练，召回率更高）
cur.execute("""
    CREATE INDEX ON product_docs
    USING hnsw (embedding vector_cosine_ops)
    WITH (m = 32, ef_construction = 200)
""")

# 索引参数说明：
# m = 32: HNSW每个节点的连接数
# ef_construction = 200: 建索引时的搜索宽度
# vector_cosine_ops: 余弦距离操作符

# IVFFlat索引（备选）
# cur.execute("""
#     CREATE INDEX ON product_docs
#     USING ivfflat (embedding vector_cosine_ops)
#     WITH (lists = 100)
# """)

conn.commit()

# ---- 5. 插入数据 ----
num_docs = 1000
embeddings = np.random.randn(num_docs, 1024).astype('float32')
embeddings = embeddings / np.linalg.norm(embeddings, axis=1, keepdims=True)

for i in range(num_docs):
    cur.execute(
        """
        INSERT INTO product_docs (content, source, page, chapter, embedding)
        VALUES (%s, %s, %s, %s, %s)
        """,
        (
            f"第{i}个文档段落的文本内容",
            f"manual_{i % 5}.pdf",
            i % 200,
            f"第{i % 10}章",
            embeddings[i].tolist()
        )
    )

conn.commit()
print(f"插入 {num_docs} 条数据")

# ---- 6. 向量搜索（余弦距离） ----
query_embedding = np.random.randn(1024).astype('float32')
query_embedding = query_embedding / np.linalg.norm(query_embedding)

# pgvector距离操作符：
# <=> : 余弦距离 (1 - cosine_similarity)
# <-> : L2距离 (欧氏距离)
# <#> : 负内积 (负点积，用于最大内积搜索)
# <~> : 汉明距离 (二值向量)
# <+> : Jaccard距离 (二值向量)

cur.execute(
    """
    SELECT id, content, source, page, chapter,
           1 - (embedding <=> %s) AS cosine_similarity
    FROM product_docs
    ORDER BY embedding <=> %s
    LIMIT 5
    """,
    (query_embedding.tolist(), query_embedding.tolist())
)

print("=== 余弦距离搜索结果 ===")
for row in cur.fetchall():
    doc_id, content, source, page, chapter, similarity = row
    print(f"  ID: {doc_id}, 相似度: {similarity:.4f}, "
          f"来源: {source}, 页码: {page}")

# ---- 7. 带SQL过滤的向量搜索 ----
# pgvector的巨大优势：向量搜索 + SQL过滤 = 强大的混合查询
cur.execute(
    """
    SELECT id, content, source, page,
           1 - (embedding <=> %s) AS cosine_similarity
    FROM product_docs
    WHERE source = %s AND page > %s
    ORDER BY embedding <=> %s
    LIMIT 5
    """,
    (query_embedding.tolist(), "manual_0.pdf", 50,
     query_embedding.tolist())
)

print("\n=== SQL过滤搜索结果 (manual_0.pdf, page>50) ===")
for row in cur.fetchall():
    doc_id, content, source, page, similarity = row
    print(f"  ID: {doc_id}, 相似度: {similarity:.4f}, "
          f"页码: {page}")

# ---- 8. 复杂混合查询 ----
# 结合向量搜索、全文搜索、元数据过滤
# 先创建全文搜索索引
cur.execute("""
    CREATE EXTENSION IF NOT EXISTS pg_trgm
""")
cur.execute("""
    CREATE INDEX idx_content_trgm ON product_docs
    USING gin (content gin_trgm_ops)
""")
conn.commit()

# 向量搜索 + 全文搜索 + 元数据过滤
cur.execute(
    """
    SELECT id, content, source, page,
           1 - (embedding <=> %s) AS vector_score,
           similarity(content, %s) AS text_score
    FROM product_docs
    WHERE source = %s
    ORDER BY
        0.7 * (1 - (embedding <=> %s)) +
        0.3 * similarity(content, %s) DESC
    LIMIT 5
    """,
    (query_embedding.tolist(), "网络", "manual_0.pdf",
     query_embedding.tolist(), "网络")
)

print("\n=== 混合搜索结果 (向量70% + 文本30%) ===")
for row in cur.fetchall():
    doc_id, content, source, page, vec_score, txt_score = row
    print(f"  ID: {doc_id}, 向量分: {vec_score:.4f}, "
          f"文本分: {txt_score:.4f}")

# ---- 9. 批量搜索 ----
# 使用UNION ALL实现多查询向量搜索
query_vectors = [np.random.randn(1024).astype('float32') for _ in range(3)]
for v in query_vectors:
    v /= np.linalg.norm(v)

cur.execute(
    """
    SELECT q.query_id, d.id, d.content,
           1 - (d.embedding <=> q.query_vec) AS similarity
    FROM (
        SELECT 1 AS query_id, %s::vector AS query_vec
        UNION ALL SELECT 2, %s::vector
        UNION ALL SELECT 3, %s::vector
    ) q
    CROSS JOIN LATERAL (
        SELECT id, content, embedding
        FROM product_docs
        ORDER BY embedding <=> q.query_vec
        LIMIT 3
    ) d
    ORDER BY q.query_id, similarity DESC
    """,
    (query_vectors[0].tolist(), query_vectors[1].tolist(),
     query_vectors[2].tolist())
)

print("\n=== 批量搜索结果 ===")
current_query = None
for row in cur.fetchall():
    query_id, doc_id, content, similarity = row
    if query_id != current_query:
        current_query = query_id
        print(f"  查询 {query_id}:")
    print(f"    ID: {doc_id}, 相似度: {similarity:.4f}")
```

## 11.7 总结

本章通过完整的代码示例，实战演练了五大向量数据库：

1. **Milvus** 是大规模向量检索的王者，适合十亿级数据和企业级部署。如果你的RAG系统需要处理海量数据，Milvus是首选。

2. **Weaviate** 的差异化在于内置向量化和生成式搜索。它让RAG开发变得极其简洁——存文本、搜文本、生成回答，三步完成。适合追求开发效率和功能丰富度的团队。

3. **Qdrant** 以Rust的高性能和全面的量化支持著称。在内存敏感或性能要求极高的场景中，Qdrant是最佳选择。

4. **Chroma** 是原型开发的利器。5分钟跑通一个RAG demo，Chroma做到了极简。但生产环境需要考虑更成熟的方案。

5. **pgvector** 是PostgreSQL用户的最优路径。如果你已有PostgreSQL基础设施，pgvector的迁移成本几乎为零。

选型的核心原则是：**从当前需求出发，不过度设计**。大多数RAG应用在初期数据量不大，Chroma或pgvector足以应对。当数据量增长到百万级时，迁移到Qdrant或Weaviate。到达十亿级时，Milvus是唯一可靠的选择。向量数据库的切换成本不高（核心操作都是插入+搜索），因此可以渐进式升级。
