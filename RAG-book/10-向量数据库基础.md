## 10.1 向量表示

### 10.1.1 什么是向量嵌入

向量嵌入（Vector Embedding）是将文本、图像等非结构化数据映射到高维数值空间的过程。在这个空间中，语义相近的内容在几何距离上更接近。

以文本为例，一个嵌入模型将一段文字转换为一个固定长度的浮点数数组：

```
"猫坐在垫子上" → [0.023, -0.156, 0.891, ..., 0.034]  # 768维向量
"猫躺在垫子上" → [0.025, -0.148, 0.887, ..., 0.031]  # 非常接近
"股票市场上涨" → [-0.412, 0.567, -0.234, ..., 0.178]  # 相距较远
```

这个映射不是随机的——嵌入模型经过训练，使得语义相似的文本映射到向量空间中相近的位置。这就是向量检索的基础：**通过向量距离近似衡量语义相似度**。

### 10.1.2 嵌入模型的工作原理

现代嵌入模型通常基于Transformer架构，其工作流程如下：

```
输入文本 → Token化 → Transformer编码器 → 池化 → 归一化 → 嵌入向量

详细流程：
┌──────────────────────────────────────────────────────────┐
│  "如何重置密码？"                                        │
│       │                                                  │
│       ▼                                                  │
│  Token化: [如何, 重置, 密码, ？] → [1284, 5678, 3456, 89] │
│       │                                                  │
│       ▼                                                  │
│  Transformer编码器 (12层注意力机制)                        │
│  ┌──────────────────────────────────┐                     │
│  │  自注意力层 → 前馈层 → 层归一化   │ ×12层              │
│  └──────────────────────────────────┘                     │
│       │                                                  │
│       ▼                                                  │
│  池化策略:                                               │
│  - CLS池化: 取[CLS]token的输出 → [768d]                 │
│  - 平均池化: 所有token的平均 → [768d]                    │
│  - 最后层: 取最后一层的隐藏状态 → [768d]                  │
│       │                                                  │
│       ▼                                                  │
│  L2归一化: v = v / ||v||₂                                │
│       │                                                  │
│       ▼                                                  │
│  [0.023, -0.156, 0.891, ..., 0.034]  # 单位向量          │
└──────────────────────────────────────────────────────────┘
```

**关键步骤解释**：

1. **Token化**：将文本分割为模型可处理的token序列
2. **Transformer编码**：通过多层自注意力机制捕获token间的语义关系
3. **池化**：将变长序列压缩为固定维度向量
4. **归一化**：将向量投影到单位超球面，使余弦相似度等价于内积

### 10.1.4 维度权衡

嵌入维度是核心设计参数，涉及多个权衡：

| 维度 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| 384 (MiniLM) | 速度快，内存小 | 语义表达力有限 | 原型、嵌入式设备 |
| 768 (mpnet) | 平衡 | 中等 | 中小规模应用 |
| 1024 (BGE/E5) | 表达力强 | 内存较大 | 生产环境推荐 |
| 1536 (OpenAI) | 表达力很强 | 成本高 | 高精度要求 |
| 3072+ (OpenAI large) | 最高表达力 | 成本最高 | 极高精度要求 |

**选择原则**：
- **原型验证**：从384或768维开始
- **生产环境**：1024维是性价比最优选择
- **跨语言检索**：选择多语言模型（BGE-m3、multilingual-e5）
- **纯英文检索**：BGE-large-en-v1.5或E5-large-v2
- **纯中文检索**：BGE-large-zh-v1.5

### 10.1.5 Matryoshka嵌入

Matryoshka嵌入（又称"俄罗斯套娃嵌入"）是一种自适应维度技术，允许在同一个嵌入向量中使用不同长度的前缀来获得不同精度的表示：

```python
# Matryoshka嵌入示例（OpenAI text-embedding-3-large）
from openai import OpenAI

client = OpenAI()

# 生成3072维嵌入
full_response = client.embeddings.create(
    model="text-embedding-3-large",
    input="如何重置密码？"
)
full_embedding = full_response.data[0].embedding  # 3072维

# 可以直接截断到更低维度，无需重新计算
# 因为模型训练时使用了Matryoshka损失函数
embedding_1024 = full_embedding[:1024]  # 截断到1024维
embedding_512 = full_embedding[:512]     # 截断到512维
embedding_256 = full_embedding[:256]     # 截断到256维

# 不同维度的质量对比（OpenAI官方数据）：
# 3072维: MTEB 64.2
# 1536维: MTEB 63.5  （仅损失0.7）
# 1024维: MTEB 62.8  （损失1.4）
# 512维:  MTEB 60.5  （损失3.7）
# 256维:  MTEB 56.5  （损失7.7）
```

Matryoshka嵌入的价值：
- **存储灵活性**：存储3072维向量，按需截断使用
- **检索优化**：先在低维空间快速粗筛，再在高维空间精排
- **带宽节省**：传输低维前缀即可

## 10.2 距离度量

### 10.2.1 余弦相似度（Cosine Similarity）

余弦相似度衡量两个向量之间的夹角，是最常用的文本相似度度量：

$$\cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{||\mathbf{A}|| \cdot ||\mathbf{B}||} = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \cdot \sqrt{\sum_{i=1}^{n} B_i^2}}$$

**取值范围**：[-1, 1]
- 1：方向完全相同（最相似）
- 0：正交（无关）
- -1：方向完全相反（最不相似）

**为什么余弦相似度适合文本**：
- 文本嵌入经过归一化后，余弦相似度只关注方向（语义），忽略幅度
- 对文本长度不敏感：短文本和长文本如果语义相同，余弦相似度仍然很高
- 计算效率高：对于单位向量，余弦相似度退化为点积

```python
import numpy as np

def cosine_similarity(a, b):
    """计算余弦相似度"""
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

# 示例
a = np.array([0.1, 0.9, 0.3])
b = np.array([0.2, 0.8, 0.4])

print(f"余弦相似度: {cosine_similarity(a, b):.4f}")
# 输出: 余弦相似度: 0.9762

# 对于归一化向量，余弦相似度 = 点积
a_normalized = a / np.linalg.norm(a)
b_normalized = b / np.linalg.norm(b)
print(f"点积（归一化后）: {np.dot(a_normalized, b_normalized):.4f}")
# 输出: 点积（归一化后）: 0.9762
```

### 10.2.2 欧氏距离（Euclidean Distance / L2）

欧氏距离衡量向量空间中两点之间的直线距离：

$$d_{L2}(\mathbf{A}, \mathbf{B}) = \sqrt{\sum_{i=1}^{n} (A_i - B_i)^2}$$

**取值范围**：[0, +∞)，0表示完全相同

```python
def euclidean_distance(a, b):
    """计算欧氏距离"""
    return np.sqrt(np.sum((a - b) ** 2))

# 示例
print(f"欧氏距离: {euclidean_distance(a, b):.4f}")
# 输出: 欧氏距离: 0.1732
```

**注意**：对于归一化向量，余弦相似度和欧氏距离有单调关系：

$$d_{L2}^2 = 2(1 - \cos(\theta))$$

因此，归一化向量使用余弦相似度和L2距离的排序结果完全一致。

### 10.2.3 内积（Inner Product / Dot Product）

内积是最简单的度量方式：

$$\mathbf{A} \cdot \mathbf{B} = \sum_{i=1}^{n} A_i B_i$$

**关键性质**：
- 对于归一化向量，内积 = 余弦相似度
- 计算最快（无归一化步骤）
- 取值范围取决于向量值：(-∞, +∞)

```python
def dot_product(a, b):
    """计算内积"""
    return np.dot(a, b)

print(f"内积: {dot_product(a, b):.4f}")
# 输出: 内积: 0.8300

# 对于归一化向量
print(f"归一化内积: {dot_product(a_normalized, b_normalized):.4f}")
# 输出: 归一化内积: 0.9762  (等于余弦相似度)
```

### 10.2.4 汉明距离（Hamming Distance）

汉明距离用于**二值嵌入**（Binary Embedding），衡量两个等长二进制串中不同位的数量：

$$d_H(\mathbf{A}, \mathbf{B}) = \sum_{i=1}^{n} \mathbb{1}[A_i \neq B_i]$$

```python
def hamming_distance(a, b):
    """计算汉明距离（二值向量）"""
    return np.sum(a != b)

# 二值嵌入示例
a_binary = np.array([1, 0, 1, 1, 0, 1, 0, 0])
b_binary = np.array([1, 1, 1, 0, 0, 1, 1, 0])

print(f"汉明距离: {hamming_distance(a_binary, b_binary)}")
# 输出: 汉明距离: 3  (有3个位不同)
```

二值嵌入的价值：极大的存储和计算节省——一个1024维浮点向量需要4096字节，而二值嵌入只需128字节（32倍压缩）。

### 10.2.5 度量选择指南

| 度量方式 | 适用场景 | 计算成本 | 是否需要归一化 |
|----------|----------|----------|----------------|
| 余弦相似度 | 文本语义检索（最常用） | 中 | 否（但归一化后可简化为内积） |
| 内积 | 归一化向量、最大吞吐量 | 低 | 是 |
| 欧氏距离 | 量级重要的场景 | 中 | 视情况 |
| 汉明距离 | 二值嵌入、极致存储 | 极低 | 不适用 |

**实践建议**：
1. **大多数文本检索场景选择余弦相似度**——这是嵌入模型最常用的训练目标
2. **确认向量已归一化后可以使用内积替代**——计算更快，结果等价
3. **不要在归一化向量上混用欧氏距离和余弦相似度**——排序结果一致，但数值含义不同
4. **二值嵌入只在极致存储/速度场景使用**——精度损失明显

### 10.2.6 归一化的重要性

归一化将向量投影到单位超球面上：

$$\hat{\mathbf{v}} = \frac{\mathbf{v}}{||\mathbf{v}||_2}$$

```python
def normalize(v):
    """L2归一化"""
    norm = np.linalg.norm(v)
    if norm == 0:
        return v
    return v / norm

# 归一化的好处
v = np.array([3.0, 4.0])  # 原始向量
v_norm = normalize(v)       # [0.6, 0.8]

# 归一化后：余弦相似度 = 内积（无需额外归一化步骤）
# 这在向量数据库中意味着可以用更快的内积搜索替代余弦搜索
```

**归一化的三大好处**：
1. **简化计算**：余弦相似度退化为内积，无需额外归一化步骤
2. **数值稳定**：消除向量量级的影响，避免浮点溢出
3. **公平比较**：不同长度的文本生成相同量级的相似度分数

## 10.3 索引算法

当向量数量从几千增长到数百万甚至数十亿时，精确搜索（暴力搜索）变得不可接受。索引算法通过牺牲少量精度换取数量级的速度提升。

### 10.3.1 暴力搜索（Brute Force / Flat Index）

暴力搜索是最直接的方法：计算查询向量与数据库中每个向量的距离，返回最近的K个。

```
查询: q
数据库: [v₁, v₂, ..., vₙ]

对每个 vᵢ:
  distance[i] = cosine_similarity(q, vᵢ)

返回 distance 最小的 K 个向量
```

| 特性 | 说明 |
|------|------|
| 时间复杂度 | O(n·d)，n为向量数，d为维度 |
| 召回率 | 100%（精确搜索） |
| 内存 | 存储所有原始向量 |
| 适用规模 | < 10万向量 |

```python
# 暴力搜索实现
import numpy as np

class BruteForceSearch:
    def __init__(self, metric="cosine"):
        self.vectors = None
        self.metric = metric

    def index(self, vectors):
        """索引所有向量"""
        self.vectors = vectors
        if self.metric == "cosine":
            # 预归一化
            norms = np.linalg.norm(vectors, axis=1, keepdims=True)
            self.vectors = vectors / norms

    def search(self, query, k=5):
        """搜索最近的k个向量"""
        if self.metric == "cosine":
            query = query / np.linalg.norm(query)
            similarities = np.dot(self.vectors, query)
            indices = np.argsort(-similarities)[:k]
        else:  # l2
            distances = np.linalg.norm(self.vectors - query, axis=1)
            indices = np.argsort(distances)[:k]

        return indices
```

### 10.3.2 IVF（倒排文件索引）

IVF（Inverted File Index）将向量空间划分为多个聚类（Voronoi单元），搜索时只计算查询向量所属聚类及邻近聚类中的向量。

```
建索引阶段：
1. 对所有向量运行K-Means，得到nlist个聚类中心
2. 将每个向量分配到最近的聚类中心

搜索阶段：
1. 计算查询向量到所有聚类中心的距离
2. 选择最近的nprobe个聚类
3. 在这nprobe个聚类中做暴力搜索
```

| 参数 | 说明 | 权衡 |
|------|------|------|
| nlist | 聚类数量 | 越大越细分，建索引越慢 |
| nprobe | 搜索时探测的聚类数 | 越大越精确，越慢 |

```
                  nprobe=1（快但可能漏）
                  │
        ●───查询───◆───◆
       ╱    │      ╲   ╲
      ╱  ○  │  ○    ╲   ○
     ╱   ○  │  ○     ○   ○
    ╱    ○  │  ○    ╱   ○
   ○───────◆───◆───◆───◆─── 聚类中心
    ╲    ○  │  ○    ╲
     ╲   ○  │  ○     ╲
      ╲  ○  │  ○      ╲
                  │
                  nprobe=5（慢但更精确）
```

```python
# IVF索引示例（使用faiss）
import faiss
import numpy as np

# 生成示例数据
np.random.seed(42)
d = 128  # 维度
n = 100000  # 向量数
vectors = np.random.randn(n, d).astype('float32')

# 构建IVF索引
nlist = 100  # 聚类数
quantizer = faiss.IndexFlatL2(d)  # 内层索引（用于聚类中心搜索）
index = faiss.IndexIVFFlat(quantizer, d, nlist)

# 训练（K-Means聚类）
index.train(vectors)

# 添加向量
index.add(vectors)

# 搜索
index.nprobe = 10  # 搜索10个聚类
query = np.random.randn(1, d).astype('float32')
distances, indices = index.search(query, k=5)
```

### 10.3.3 HNSW（分层可导航小世界图）

HNSW（Hierarchical Navigable Small World）是当前最流行的近似最近邻（ANN）索引算法之一，基于图结构：

```
HNSW结构（3层示意）：

层2（稀疏）:  ●───────────────────●
              │                    │
层1（中等）:  ●───────●───────●   │
              │   ╱   │   ╲    │   │
层0（密集）:  ●──●──●──●──●──●───●──●
              │╱ │╱ │╱ │╱ │╲ │╲ │╲ │

搜索过程：
1. 从层2的最高层入口开始
2. 在当前层贪心搜索最近邻
3. 下降到下一层，以上一层结果为起点
4. 在最底层（层0）做精确搜索
```

| 参数 | 说明 | 推荐值 | 影响 |
|------|------|--------|------|
| M | 每个节点的连接数 | 16-64 | 越大召回率越高，内存越大 |
| ef_construction | 建索引时搜索宽度 | 100-200 | 越大建索引越慢，质量越高 |
| ef_search | 搜索时搜索宽度 | 50-200 | 越大越精确，越慢 |

```python
# HNSW索引示例（使用faiss）
import faiss
import numpy as np

d = 128
n = 100000
vectors = np.random.randn(n, d).astype('float32')

# 构建HNSW索引
M = 32  # 每个节点的连接数
index = faiss.IndexHNSWFlat(d, M)

# 设置参数
index.hnsw.efConstruction = 200  # 建索引时的搜索宽度
index.hnsw.efSearch = 50         # 搜索时的搜索宽度

# 添加向量（HNSW不需要train）
index.add(vectors)

# 搜索
query = np.random.randn(1, d).astype('float32')
distances, indices = index.search(query, k=5)

print(f"最近的5个向量: {indices}")
print(f"距离: {distances}")
```

**HNSW的优势**：
- 搜索速度快：通常在微秒级完成
- 召回率高：在合适参数下可达99%+
- 支持增删：可以动态添加和删除向量
- 无需训练：不需要预处理步骤

**HNSW的劣势**：
- 内存占用大：每个节点需要存储M个连接的指针
- 建索引慢：特别是ef_construction较大时
- 不支持压缩：向量以原始精度存储

### 10.3.4 PQ（乘积量化）

乘积量化（Product Quantization）将高维向量切分为多个子空间，对每个子空间独立量化：

```
原始向量（128维）:
[v₁, v₂, ..., v₃₂ | v₃₃, v₃₄, ..., v₆₄ | v₆₅, ..., v₉₆ | v₉₇, ..., v₁₂₈]
 └──── 子空间1 ────┘ └──── 子空间2 ────┘ └──── 子空间3 ────┘ └──── 子空间4 ────┘

每个子空间：
1. 对子空间内向量做K-Means（256个中心）
2. 每个向量用中心索引表示（1字节）
3. 128维 × 4字节 = 512字节 → 32 × 1字节 = 32字节

压缩比：16倍！
```

| 特性 | 说明 |
|------|------|
| 压缩比 | 8-32倍（取决于子空间数） |
| 精度损失 | 5-15%召回率下降 |
| 内存 | 大幅降低 |
| 搜索方式 | 非对称距离计算（ADC） |

```python
# PQ索引示例（使用faiss）
import faiss
import numpy as np

d = 128
n = 100000
vectors = np.random.randn(n, d).astype('float32')

# 构建PQ索引
m = 32  # 子空间数量（必须是d的因子）
# 每个子空间256个中心（8bit = 1byte）
# 压缩后每个向量：32 × 1byte = 32byte
index = faiss.IndexPQ(d, m, 8)  # d维度, m子空间, 8bit每个编码

# 训练
index.train(vectors)
index.add(vectors)

# 搜索
query = np.random.randn(1, d).astype('float32')
distances, indices = index.search(query, k=5)
```

### 10.3.5 IVF-PQ（混合索引）

IVF-PQ结合了IVF的聚类粗筛和PQ的压缩存储，是处理亿级数据的常用方案：

```
搜索过程：
┌──────────────────────────────────────────────────┐
│  1. 查询向量 → 找到最近的nprobe个聚类（IVF粗筛） │
│  2. 在每个聚类中用PQ做近似距离计算（PQ细排）     │
│  3. 返回最近的K个向量                            │
└──────────────────────────────────────────────────┘

存储对比（100万条128维向量）：
- 原始存储: 100万 × 128 × 4字节 = 512MB
- IVF-PQ:  100万 × 32 × 1字节 = 32MB （16倍压缩）
```

```python
# IVF-PQ索引示例
import faiss
import numpy as np

d = 128
n = 1000000
vectors = np.random.randn(n, d).astype('float32')

nlist = 1000  # 聚类数
m = 32        # 子空间数

# 构建IVF-PQ索引
quantizer = faiss.IndexFlatL2(d)
index = faiss.IndexIVFPQ(quantizer, d, nlist, m, 8)

# 训练和添加
index.train(vectors[:100000])  # 用子集训练
index.add(vectors)

# 搜索
index.nprobe = 50  # 探测50个聚类
query = np.random.randn(1, d).astype('float32')
distances, indices = index.search(query, k=5)
```

### 10.3.6 DiskANN

DiskANN是微软提出的磁盘ANN索引方案，用于在普通硬件上处理十亿级向量：

```
DiskANN架构：
┌──────────────────────────────────────────────────┐
│  内存中：                                        │
│  - PQ压缩向量（用于粗排）                        │
│  - HNSW图结构（导航图）                          │
│                                                  │
│  磁盘中：                                        │
│  - 原始向量（用于精排）                           │
│  - 邻居列表                                      │
└──────────────────────────────────────────────────┘

搜索流程：
1. 在内存中的PQ压缩向量上做HNSW搜索
2. 按需从磁盘读取原始向量做精排
3. 仅需4-16GB内存即可处理10亿向量
```

| 特性 | 说明 |
|------|------|
| 规模 | 十亿级 |
| 内存需求 | 低（4-16GB） |
| 延迟 | 毫秒级（SSD） |
| 召回率 | 95%+ |
| 硬件要求 | 普通服务器 + SSD |

### 10.3.7 索引算法对比

| 算法 | 召回率 | 延迟 | 内存 | 建索引 | 增删 | 适用规模 |
|------|--------|------|------|--------|------|----------|
| Flat | 100% | O(n) | 高 | 无 | 支持 | <10万 |
| IVF | 90-99% | 低 | 高 | 中 | 支持 | 10万-100万 |
| HNSW | 95-99.5% | 极低 | 高 | 慢 | 支持 | 10万-1000万 |
| PQ | 85-95% | 低 | 极低 | 中 | 支持 | 100万-10亿 |
| IVF-PQ | 85-95% | 低 | 低 | 中 | 支持 | 100万-10亿 |
| DiskANN | 95%+ | 中 | 极低 | 慢 | 支持 | 10亿+ |

**选择决策树**：

```
你的数据规模是多少？
│
├── < 10万
│   └── Flat（精确搜索，无需索引）
│
├── 10万 - 100万
│   ├── 延迟优先 → HNSW
│   ├── 内存优先 → IVF-PQ
│   └── 平衡 → IVF
│
├── 100万 - 1000万
│   ├── 延迟优先 → HNSW（需要足够内存）
│   └── 内存优先 → IVF-PQ
│
└── > 1亿
    ├── 有足够内存 → HNSW-PQ
    └── 内存有限 → DiskANN
```

## 10.4 向量数据库核心概念

### 10.4.1 集合/索引（Collection/Index）

集合是向量的逻辑容器，类似于关系数据库中的表：

```
Collection: "产品手册"
├── 向量字段: embedding (1024维, cosine)
├── 元数据字段:
│   ├── doc_id (string)
│   ├── chapter (string)
│   ├── page (integer)
│   └── source (string)
├── 索引类型: HNSW
└── 距离度量: cosine
```

设计原则：
- 按业务实体划分集合（如产品文档、用户画像、日志）
- 同一集合内的向量维度和度量方式必须一致
- 合理规划集合数量（过多会增加管理复杂度）

### 10.4.2 Schema设计

Schema定义了集合中向量和元数据的结构：

```python
# Milvus Schema 示例
from pymilvus import CollectionSchema, FieldSchema, DataType

# 定义字段
pk_field = FieldSchema(
    name="id",
    dtype=DataType.INT64,
    is_primary=True,
    auto_id=True

vector_field = FieldSchema(
    name="embedding",
    dtype=DataType.FLOAT_VECTOR,
    dim=1024  # 必须与嵌入模型维度一致

# 元数据字段
content_field = FieldSchema(
    name="content",
    dtype=DataType.VARCHAR,
    max_length=65535
)

    name="source",
    dtype=DataType.VARCHAR,
    max_length=256
)

page_field = FieldSchema(
    name="page",
    dtype=DataType.INT32
)

# 创建Schema
schema = CollectionSchema(
    fields=[pk_field, vector_field, content_field, source_field, page_field],
    description="产品手册向量集合"

```

### 10.4.4 混合搜索

混合搜索（Hybrid Search）结合向量检索和关键词检索，是RAG系统中最常用的检索策略：

```
混合搜索流程：
┌──────────────────────────────────────────────────────────────┐
│  用户查询: "如何配置网络连接"                                  │
│                                                              │
│  ┌──────────────────┐     ┌──────────────────┐              │
│  │  向量检索         │     │  关键词检索       │              │
│  │  (语义相似度)     │     │  (BM25/TF-IDF)   │              │
│  │                  │     │                  │              │
│  │  结果1: 0.92     │     │  结果3: 8.5      │              │
│  │  结果2: 0.87     │     │  结果5: 7.2      │              │
│  │  结果3: 0.83     │     │  结果1: 6.8      │              │
│  │  结果4: 0.78     │     │  结果7: 5.1      │              │
│  └────────┬─────────┘     └────────┬─────────┘              │
│           │                        │                         │
│           └────────┬───────────────┘                         │
│                    │                                         │
│           ┌────────▼────────┐                               │
│           │   融合与重排序    │                               │
│           │   (RRF / 加权)  │                               │
│           └────────┬────────┘                               │
│                    │                                         │
│           ┌────────▼────────┐                               │
│           │  最终排序结果    │                               │
│           │  结果1 (向量+关键词)                              │
│           │  结果3 (关键词+向量)                              │
│           │  结果2 (仅向量)                                  │
│           └─────────────────┘                               │
└──────────────────────────────────────────────────────────────┘
```

**融合算法——倒数排名融合（RRF）**：

$$\text{RRF}(d) = \sum_{i=1}^{k} \frac{1}{\lambda + \text{rank}_i(d)}$$

其中λ是常数（通常取60），rank_i(d)是文档d在第i个排序列表中的排名。

```python
# RRF融合示例
def reciprocal_rank_fusion(ranked_lists, k=60):
    """倒数排名融合"""
    scores = {}
    for ranked_list in ranked_lists:
        for rank, doc_id in enumerate(ranked_list, start=1):
            if doc_id not in scores:
                scores[doc_id] = 0
            scores[doc_id] += 1 / (k + rank)

    # 按融合分数排序
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)

# 示例
vector_results = ["doc1", "doc3", "doc5", "doc2", "doc8"]  # 向量检索结果
keyword_results = ["doc3", "doc7", "doc1", "doc5", "doc9"]  # 关键词检索结果

fused = reciprocal_rank_fusion([vector_results, keyword_results])
print("融合排序:", fused)
# 融合排序: [('doc1', 0.032), ('doc3', 0.031), ...]
```

### 10.4.5 元数据过滤

元数据过滤是向量搜索的重要补充，允许在语义相似度检索的同时应用精确的属性筛选：

```python
# 元数据过滤示例（Milvus语法）

# 过滤特定来源的文档
expr_source = 'source == "product_manual.pdf"'

# 过滤特定页码范围
expr_page = 'page >= 10 and page <= 50'

# 过滤特定章节和来源
expr_combined = 'chapter == "网络配置" and source != "obsolete.pdf"'

# 在搜索中使用元数据过滤
results = collection.search(
    data=[query_vector],
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"nprobe": 10}},
    limit=10,
    expr=expr_combined  # 先过滤，再在满足条件的向量中搜索
)
```

元数据过滤的执行顺序：
1. **预过滤**（Pre-filtering）：先按元数据过滤，再在子集中做向量搜索。适用于过滤比例高的场景（过滤掉大部分数据）
2. **后过滤**（Post-filtering）：先做向量搜索，再在结果中按元数据过滤。适用于过滤比例低的场景
3. 大多数向量数据库默认使用预过滤

### 10.4.6 分片与复制

```
分片（Sharding）：水平扩展
┌──────────────────────────────────────────────────────┐
│  Collection: "产品文档" (1000万向量)                    │
│                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│  │  Shard 1     │  │  Shard 2     │  │  Shard 3 │  │
│  │  0-333万     │  │  333-666万   │  │  666-1000万│  │
│  └──────────────┘  └──────────────┘  └──────────┘  │
│                                                      │
│  每个分片独立搜索，结果合并排序                         │
└──────────────────────────────────────────────────────┘

复制（Replication）：高可用
┌──────────────────────────────────────────────────────┐
│  Shard 1:                                            │
│  ┌──────────────┐  ┌──────────────┐                 │
│  │  Replica 1   │  │  Replica 2   │  读请求负载均衡   │
│  │  (Leader)    │  │  (Follower) │                 │
│  └──────────────┘  └──────────────┘                 │
│                                                      │
│  写请求 → Leader → 同步到 Follower                    │
│  读请求 → Leader 或 Follower（负载均衡）               │
└──────────────────────────────────────────────────────┘
```

### 10.4.7 一致性模型

向量数据库通常支持多种一致性级别：

| 级别 | 说明 | 延迟 | 适用场景 |
|------|------|------|----------|
| Strong | 写入立即可读 | 高 | 金融交易、实时更新 |
| Bounded Staleness | 最多延迟N秒 | 中 | 大多数业务场景 |
| Session | 同一客户端读己之写 | 低 | 用户体验优先 |
| Eventual | 最终一致，可能有延迟 | 最低 | 日志分析、大规模批处理 |

```python
# Milvus 一致性级别示例
from pymilvus import Collection

collection = Collection("product_manual")

# 强一致性搜索（确保读到最新写入的数据）
results = collection.search(
    data=[query_vector],
    anns_field="embedding",
    param={"metric_type": "COSINE"},
    limit=5,
    consistency_level="Strong"  # Strong / Bounded / Session / Eventually
)

# 最终一致性搜索（更高吞吐，可能读到稍旧的数据）
results = collection.search(
    data=[query_vector],
    anns_field="embedding",
    param={"metric_type": "COSINE"},
    limit=5,
    consistency_level="Eventually"
)
```

## 10.5 总结

本章系统性地讲解了向量数据库的基础知识：

1. **向量嵌入**是将文本映射到高维数值空间的技术，是RAG检索的基础。选择合适的嵌入模型需要考虑维度、语言支持、精度和成本。

2. **距离度量**决定了相似性的计算方式。余弦相似度是文本检索的默认选择；归一化向量可以将余弦搜索简化为内积搜索。

3. **索引算法**是大规模向量检索的关键。Flat适用于小规模精确搜索，HNSW在中等规模下表现最佳，IVF-PQ和DiskANN为超大规模提供了内存和延迟的平衡方案。

4. **核心概念**包括集合Schema设计、CRUD操作、混合搜索、元数据过滤、分片复制和一致性模型——这些都是构建生产级RAG系统必须掌握的知识。
