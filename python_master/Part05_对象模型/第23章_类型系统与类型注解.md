# 第23章 类型系统与类型注解

> Python 的类型系统是动态的，但并非无类型。从 duck typing 到 PEP 484 Type Hints，再到 PEP 544 Protocol，Python 的类型系统在不断演进——在不牺牲灵活性的前提下，获得更多的安全性和工具支持。

---

## 23.1 概念与语法

### 23.1.1 动态类型原理

Python 采用**动态类型**（dynamic typing）：变量没有类型，只有对象有类型。变量仅仅是绑定到对象的名称。

```python
# 变量没有类型——它只是一个名字标签
x = 42        # x 绑定到 int 对象
x = "hello"   # x 重新绑定到 str 对象——完全合法
x = [1, 2]    # x 绑定到 list 对象

# 类型检查发生在运行时
def add(a, b):
    return a + b  # 只要 a 和 b 支持 + 运算就行

print(add(1, 2))       # 3 — int + int
print(add("a", "b"))    # "ab" — str + str
print(add([1], [2]))    # [1, 2] — list + list
# add("a", 1)           # TypeError: can only concatenate str to str
```

动态类型的 C 层面实现：变量不是内存盒子，而是字典中的键值对。

```c
// CPython 源码: Python/ceval.c (简化)
// 赋值语句 x = 42 的底层执行
TARGET(STORE_NAME) {
    PyObject *name = GETITEM(names, oparg);  // 变量名 "x"
    PyObject *value = POP();                   // 从栈顶弹出值 42
    err = PyObject_SetItem(locals, name, value); // 存入局部变量字典
    Py_DECREF(value);
    DISPATCH();
}
```

**局部变量的优化**：CPython 3.12+ 引入 specializing adaptive interpreter，对于频繁执行的局部变量访问，会将字典查找优化为数组索引访问：

```c
// CPython 源码: Python/ceval.c
// 优化后的局部变量存储（快速路径）
// locals 是一个数组而非字典，变量名编译为索引
TARGET(LOAD_FAST) {
    PyObject *value = GETLOCAL(oparg);  // 直接通过索引访问
    if (value == NULL) {
        // 未初始化变量错误
    }
    Py_INCREF(value);
    PUSH(value);
    DISPATCH();
}
```

### 23.1.2 Duck Typing

Duck typing 的核心思想："如果它走起来像鸭子、叫起来像鸭子，那它就是鸭子。"——关注对象的行为（协议/方法），而非对象的类型。

```python
# Duck typing 的经典示例
class Duck:
    def quack(self):
        return "Quack!"

    def swim(self):
        return "Swimming like a duck"


class Person:
    def quack(self):
        return "I'm pretending to quack!"

    def swim(self):
        return "Swimming like a human"


def make_sound(entity):
    # 不检查类型，只检查行为
    return entity.quack()


d = Duck()
p = Person()
print(make_sound(d))  # Quack!
print(make_sound(p))  # I'm pretending to quack!
```

Python 内置了大量基于 duck typing 的协议：

| 协议 | 需要实现的方法 | 典型用途 |
|------|---------------|---------|
| 迭代器协议 | `__iter__` + `__next__` | `for` 循环、解包 |
| 上下文管理器协议 | `__enter__` + `__exit__` | `with` 语句 |
| 序列协议 | `__getitem__` + `__len__` | 索引、切片、`len()` |
| 映射协议 | `__getitem__` + `__len__` + `__iter__` | 字典式访问 |
| 描述符协议 | `__get__` + `__set__`/`__delete__` | 属性管理 |
| 可调用协议 | `__call__` | 函数调用 `()` |
| 比较协议 | `__eq__`, `__lt__` 等 | 比较运算 |

### 23.1.3 Type Hints 语法

PEP 484（2014）引入了 Type Hints——一种在 Python 中标注类型的标准语法，但类型标注**不影响运行时行为**，仅供静态分析工具使用。

```python
# 基础类型标注
def greet(name: str) -> str:
    return f"Hello, {name}"

# 容器类型（Python 3.9+ 使用内置泛型）
def process(items: list[str]) -> dict[str, int]:
    return {item: len(item) for item in items}

# 可选类型
from typing import Optional
def find_user(user_id: int) -> Optional[str]:
    # 返回 str 或 None
    ...

# Union 类型
from typing import Union
def process(value: Union[int, str]) -> str:
    return str(value)

# Python 3.10+ 可以使用 | 语法
def process(value: int | str) -> str:
    return str(value)

# 可调用类型
from typing import Callable
def apply(func: Callable[[int, int], int], a: int, b: int) -> int:
    return func(a, b)

# 元组类型
def get_point() -> tuple[float, float]:
    return (1.0, 2.0)

# 不定长元组
def head(items: tuple[int, ...]) -> int:
    return items[0]
```

### 23.1.4 Protocol（PEP 544）

PEP 544（Python 3.8+）引入了 `Protocol`——结构化类型（structural typing）的核心机制。与 `ABC` 的名义类型（nominal typing）不同，Protocol 关注对象的结构（方法签名），而非继承关系。

```python
from typing import Protocol, runtime_checkable

# 定义 Protocol——只要实现了 close() 方法就符合此协议
@runtime_checkable  # 允许在运行时用 isinstance() 检查
class Closable(Protocol):
    def close(self) -> None:
        ...

# 任何有 close() 方法的类都自动满足 Closable 协议
# 无需显式继承 Closable
class FileHandle:
    def __init__(self, path: str):
        self.path = path

    def close(self) -> None:
        print(f"Closing {self.path}")

class DatabaseConnection:
    def close(self) -> None:
        print("Closing database connection")

# 两者都是 Closable 的"隐式子类型"
fh = FileHandle("/tmp/data")
db = DatabaseConnection()

print(isinstance(fh, Closable))   # True
print(isinstance(db, Closable))   # True

# 在类型标注中使用
def cleanup(resource: Closable) -> None:
    resource.close()

cleanup(fh)   # 正常工作
cleanup(db)   # 正常工作
```

### 23.1.5 泛型——TypeVar 与 Generic

```python
from typing import TypeVar, Generic, Sequence

# TypeVar：定义类型变量
T = TypeVar('T')          # 任意类型
T_co = TypeVar('T_co', covariant=True)  # 协变类型变量
Num = TypeVar('Num', int, float)         # 约束类型变量

# 泛型函数
def first(items: Sequence[T]) -> T:
    return items[0]

result: str = first(["a", "b", "c"])  # T 推断为 str

# 泛型类
class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def is_empty(self) -> bool:
        return len(self._items) == 0

# Python 3.12+ 新语法（PEP 695）
class StackNew[T]:
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

# 多类型变量
K = TypeVar('K')
V = TypeVar('V')

class Pair(Generic[K, V]):
    def __init__(self, key: K, value: V):
        self.key = key
        self.value = value

p: Pair[str, int] = Pair("age", 30)
```

## 23.2 原理与机制

### 23.2.1 动态类型的底层实现

Python 变量不是 C 语言中的"有类型的内存区域"，而是一个**指向对象的引用**。在 CPython 的字节码执行中，局部变量存储在帧对象的 `f_localsplus` 数组中，每个元素是一个 `PyObject *` 指针。

```c
// CPython 源码: Include/frameobject.h (简化)
typedef struct _PyInterpreterFrame {
    PyObject *f_funcobj;     // 函数对象
    PyObject *f_globals;     // 全局命名空间
    PyObject *f_builtins;    // 内建命名空间
    PyObject *f_locals;       // 局部命名空间
    _Py_CODEUNIT *prev_instr; // 上一条指令
    int stacktop;             // 操作数栈顶
    // ...
    PyObject *localsplus[];   // 局部变量 + 单元变量 + 自由变量 + 操作数栈
} _PyInterpreterFrame;
```

**名字绑定过程**：

```
Python 代码: x = 42

编译后的字节码:
  LOAD_CONST               0 (42)
  STORE_FAST               0 (x)

执行过程:
  1. LOAD_CONST: 将常量 42 的 PyObject* 压入操作数栈
  2. STORE_FAST: 从操作数栈弹出 PyObject*，存入 localsplus[0]
     — 变量名 "x" 在编译时已映射为索引 0
     — 42 的引用计数 +1（新绑定）
     — 旧值的引用计数 -1（如果 x 之前有绑定）
```

**类型检查的延迟**：Python 不在编译时检查类型，而是在运行时，由具体操作查找对象的方法来决定行为：

```c
// CPython 源码: Python/ceval.c
// a + b 的执行过程（简化）
TARGET(BINARY_OP) {
    PyObject *right = POP();
    PyObject *left = TOP();
    PyObject *res;
    // 从 left 的类型对象中查找 __add__ 方法
    // 即 left->ob_type->tp_as_number->nb_add
    // 或 left->ob_type->tp_as_sequence->sq_concat
    res = PyNumber_Add(left, right);
    // ...
}
```

### 23.2.2 类型标注的运行时表示

类型标注在运行时存储在 `__annotations__` 属性中，但 CPython 3.10+ 引入了 PEP 563（`from __future__ import annotations`）和 PEP 649（延迟注解求值），改变了注解的存储方式。

```python
# Python 3.10 之前：注解在定义时求值
class MyClass:
    name: str                    # 存入 __annotations__
    value: int = 0               # 存入 __annotations__ 和类命名空间

print(MyClass.__annotations__)    # {'name': <class 'str'>, 'value': <class 'int'>}

# Python 3.10+ with PEP 563：注解存储为字符串
from __future__ import annotations

class MyClass2:
    name: str
    value: list[str]             # 不会在定义时报错！

print(MyClass2.__annotations__)  # {'name': 'str', 'value': 'list[str]'}
                                  # 字符串形式，延迟求值
```

**`typing` 模块的底层机制**：

```python
# typing 模块中的泛型类型是 _GenericAlias 对象
import typing

# list[str] 创建一个 _GenericAlias 对象
t = list[str]
print(type(t))                    # <class 'types.GenericAlias'>
print(t.__origin__)               # <class 'list'>
print(t.__args__)                 # (<class 'str'>,)

# Union[int, str] 创建 _UnionGenericAlias 对象
u = int | str                     # Python 3.10+ 语法
print(type(u))                    # <class 'types.UnionType'>
print(u.__args__)                 # (<class 'int'>, <class 'str'>)

# Optional[str] 等价于 str | None
from typing import Optional
o = Optional[str]
print(o)                          # str | None
print(o == (str | None))          # True
```

### 23.2.3 Protocol 的实现原理

Protocol 的核心是**结构化子类型**（structural subtyping）。与传统的名义子类型（nominal subtyping，即必须显式继承）不同，Protocol 只要求对象满足方法签名，无需声明继承关系。

```python
from typing import Protocol, runtime_checkable, TYPE_CHECKING

@runtime_checkable
class Drawable(Protocol):
    def draw(self) -> None: ...

class Circle:
    def draw(self) -> None:
        print("Drawing circle")

class Square:
    def draw(self) -> None:
        print("Drawing square")

# Circle 和 Square 没有继承 Drawable
# 但它们"结构上兼容"——都有 draw() 方法
c = Circle()
print(isinstance(c, Drawable))  # True — 运行时检查通过

# mypy 静态分析也能识别
def render(shape: Drawable) -> None:
    shape.draw()

render(Circle())  # mypy: OK
render(Square())   # mypy: OK
```

**`@runtime_checkable` 的实现**：

在 CPython 源码中，`@runtime_checkable` 装饰器为 Protocol 类注入了 `__instancecheck__` 方法：

```python
# CPython 源码: Lib/typing.py (简化)
def _proto_hook(cls, other):
    """Protocol 的 __instancecheck__ 实现"""
    if not getattr(cls, '_is_protocol', False):
        return NotImplemented
    # 检查 other 是否拥有 Protocol 中定义的所有方法
    for attr in cls._protocol_attrs:
        # 检查方法是否存在且可调用
        for base in other.__mro__:
            if attr in base.__dict__:
                if callable(base.__dict__[attr]):
                    break
        else:
            return False
    return True

# 实际实现更复杂，涉及 _check_proto_meth 等辅助函数
```

### 23.2.4 泛型的实现原理

`TypeVar` 在运行时是一个轻量级对象，主要用于 mypy 等静态分析工具：

```python
from typing import TypeVar

T = TypeVar('T')

# TypeVar 对象本身几乎不做任何事
print(type(T))         # <class 'typing.TypeVar'>
print(T.__name__)      # 'T'
print(T.__constraints__)  # ()

# 泛型类的运行时行为
from typing import Generic

class Box(Generic[T]):
    def __init__(self, value: T):
        self.value = value

# 运行时，Box[int] 和 Box[str] 是同一个类
b1 = Box(42)
b2 = Box("hello")
print(type(b1) is type(b2))  # True！运行时不区分泛型参数
print(type(b1))               # <class '__main__.Box'>

# Box[int] 创建的是 _GenericAlias 对象
print(Box[int])               # __main__.Box[int]
print(type(Box[int]))         # <class 'types.GenericAlias'>
print(Box[int].__origin__)    # <class '__main__.Box'>
print(Box[int].__args__)      # (<class 'int'>,)
```

**Python 3.12 的 PEP 695 新语法**从根本上改变了泛型的实现：

```python
# PEP 695 新语法：类型参数在类/函数定义时声明
class Stack[T]:
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

def first[T](items: list[T]) -> T:
    return items[0]

# 新语法自动创建 TypeVar，不再需要在模块级别定义
# 且支持上界约束和多约束
class Number[T: (int, float)]:
    value: T

class Comparable[T: SupportsLessThan]:
    items: list[T]
```

## 23.3 使用场景

### 23.3.1 用 Protocol 实现多态而不依赖继承

```python
"""Protocol 实现真正的 duck typing 类型安全"""
from typing import Protocol

class Sized(Protocol):
    """任何有 __len__ 方法的类型"""
    def __len__(self) -> int: ...

class Iterable(Protocol[T]):
    """任何可迭代的类型"""
    def __iter__(self): ...

def total_length(items: Sized) -> int:
    """接受任何有 __len__ 方法的对象"""
    return len(items)

# 多种类型都兼容 Sized 协议
print(total_length([1, 2, 3]))      # 3 — list
print(total_length("hello"))          # 5 — str
print(total_length({"a": 1}))         # 1 — dict
print(total_length(range(10)))        # 10 — range
```

### 23.3.2 用泛型实现类型安全的容器

```python
"""类型安全的泛型容器"""
from typing import TypeVar, Generic

T = TypeVar('T')

class Result(Generic[T]):
    """类型安全的结果容器，类似 Rust 的 Result"""

    def __init__(self, value: T | None, error: Exception | None):
        self._value = value
        self._error = error

    @classmethod
    def ok(cls, value: T) -> 'Result[T]':
        return cls(value, None)

    @classmethod
    def err(cls, error: Exception) -> 'Result[T]':
        return cls(None, error)

    @property
    def is_ok(self) -> bool:
        return self._error is None

    @property
    def is_err(self) -> bool:
        return self._error is not None

    def unwrap(self) -> T:
        if self._error is not None:
            raise self._error
        return self._value  # type: ignore

    def unwrap_or(self, default: T) -> T:
        if self._error is not None:
            return default
        return self._value  # type: ignore

# 使用
def divide(a: int, b: int) -> Result[float]:
    try:
        return Result.ok(a / b)
    except ZeroDivisionError as e:
        return Result.err(e)

r1 = divide(10, 2)
r2 = divide(10, 0)

if r1.is_ok:
    print(f"10 / 2 = {r1.unwrap()}")  # 5.0

print(f"10 / 0 = {r2.unwrap_or(0.0)}")  # 0.0
```

### 23.3.3 用 mypy 进行静态类型检查

```python
# 文件: typed_module.py
"""mypy 静态类型检查示例"""

from typing import Protocol

class Sortable(Protocol):
    def __lt__(self, other: object) -> bool: ...
    def __gt__(self, other: object) -> bool: ...

def find_min(values: list[int]) -> int:
    """mypy 能检查出类型错误"""
    if not values:
        raise ValueError("empty list")
    minimum = values[0]
    for v in values:
        if v < minimum:
            minimum = v
    return minimum

# mypy 会报错：
# result = find_min(["a", "b"])  # error: Argument 1 has incompatible type "list[str]"
#                                    expected "list[int]"

def safe_divide(a: int, b: int) -> float | None:
    if b == 0:
        return None
    return a / b

result: float | None = safe_divide(10, 0)
# mypy 强制处理 None 的情况
if result is not None:
    double: float = result * 2  # OK
else:
    double = 0.0
```

## 23.4 示例代码

### 示例1：类型系统的运行时探索

```python
"""
深入探索 Python 类型系统的运行时行为
验证类型标注在运行时的表示
"""
import typing
import types
import sys

def explore_type_system():
    """探索类型系统的内部结构"""

    # ---- 1. 基础类型标注的运行时表示 ----
    print("=== 基础类型标注 ===")

    def func(x: int, y: str) -> bool:
        return str(x) == y

    print(f"__annotations__: {func.__annotations__}")
    # {'x': <class 'int'>, 'y': <class 'str'>, 'return': <class 'bool'>}

    # ---- 2. 泛型类型的运行时结构 ----
    print("\n=== 泛型类型的运行时结构 ===")

    generic_list = list[str]
    print(f"list[str]: {generic_list}")
    print(f"  type: {type(generic_list)}")
    print(f"  __origin__: {generic_list.__origin__}")
    print(f"  __args__: {generic_list.__args__}")

    generic_dict = dict[str, int]
    print(f"dict[str, int]: {generic_dict}")
    print(f"  __origin__: {generic_dict.__origin__}")
    print(f"  __args__: {generic_dict.__args__}")

    # ---- 3. Union 类型 ----
    print("\n=== Union 类型 ===")

    # Python 3.10+ 语法
    u1 = int | str
    print(f"int | str: {u1}")
    print(f"  type: {type(u1)}")
    print(f"  __args__: {u1.__args__}")

    # typing.Union（旧语法）
    from typing import Union
    u2 = Union[int, str]
    print(f"Union[int, str]: {u2}")
    # 两种写法在 Python 3.10+ 等价
    print(f"int|str == Union[int,str]: {u1 == u2}")  # True

    # ---- 4. TypeVar ----
    print("\n=== TypeVar ===")

    T = typing.TypeVar('T')
    print(f"TypeVar('T'): {T}")
    print(f"  type: {type(T)}")
    print(f"  __name__: {T.__name__}")
    print(f"  __constraints__: {T.__constraints__}")
    print(f"  __bound__: {T.__bound__}")

    B = typing.TypeVar('B', bound=int)
    print(f"TypeVar('B', bound=int): {B}")
    print(f"  __bound__: {B.__bound__}")

    # ---- 5. Callable 类型 ----
    print("\n=== Callable 类型 ===")

    c1 = typing.Callable[[int, str], bool]
    print(f"Callable[[int, str], bool]: {c1}")

    # ---- 6. 类型别名的运行时检查 ----
    print("\n=== 类型别名 ===")

    from typing import TypeAlias

    Vector: TypeAlias = list[float]
    print(f"Vector: {Vector}")
    print(f"  isinstance: Vector is list[float = {Vector == list[float]}")

    # ---- 7. isinstance 与类型标注 ----
    print("\n=== isinstance 与类型标注 ===")

    # isinstance 支持 runtime_checkable Protocol
    from typing import Protocol, runtime_checkable

    @runtime_checkable
    class HasLen(Protocol):
        def __len__(self) -> int: ...

    print(f"isinstance([1,2,3], HasLen): {isinstance([1, 2, 3], HasLen)}")
    print(f"isinstance('hello', HasLen): {isinstance('hello', HasLen)}")
    print(f"isinstance(42, HasLen): {isinstance(42, HasLen)}")

    # isinstance 支持泛型（Python 3.10+）
    print(f"isinstance([1,2,3], list): {isinstance([1, 2, 3], list)}")

    # 但不支持参数化泛型
    # isinstance([1,2,3], list[str])  # TypeError!

explore_type_system()
```

### 示例2：Protocol 与 ABC 的对比

```python
"""
Protocol vs ABC：结构化类型 vs 名义类型
"""
from typing import Protocol, runtime_checkable
from abc import ABC, abstractmethod


# ============================================================
# 方案1：ABC（名义类型——必须显式继承）
# ============================================================

class ShapeABC(ABC):
    @abstractmethod
    def area(self) -> float:
        ...

    @abstractmethod
    def perimeter(self) -> float:
        ...


class CircleABC(ShapeABC):  # 必须显式继承 ShapeABC
    def __init__(self, radius: float):
        self.radius = radius

    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2

    def perimeter(self) -> float:
        import math
        return 2 * math.pi * self.radius


# 以下类没有继承 ShapeABC，即使接口兼容也无法通过类型检查
class TriangleLike:
    """没有继承 ShapeABC，但接口兼容"""
    def __init__(self, a: float, b: float, c: float):
        self.a, self.b, self.c = a, b, c

    def area(self) -> float:
        import math
        s = (self.a + self.b + self.c) / 2
        return math.sqrt(s * (s - self.a) * (s - self.b) * (s - self.c))

    def perimeter(self) -> float:
        return self.a + self.b + self.c


# def process_shape_abc(shape: ShapeABC) -> float:
#     return shape.area()
#
# process_shape_abc(TriangleLike(3, 4, 5))  # mypy 报错！
#                                              # 不兼容类型


# ============================================================
# 方案2：Protocol（结构化类型——只需满足接口）
# ============================================================

@runtime_checkable
class ShapeProtocol(Protocol):
    def area(self) -> float: ...
    def perimeter(self) -> float: ...


class CircleProto:  # 无需显式继承 ShapeProtocol
    def __init__(self, radius: float):
        self.radius = radius

    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2

    def perimeter(self) -> float:
        import math
        return 2 * math.pi * self.radius


def process_shape_proto(shape: ShapeProtocol) -> float:
    return shape.area()

# CircleProto 自动满足 ShapeProtocol
print(f"isinstance(CircleProto(1), ShapeProtocol): "
      f"{isinstance(CircleProto(1), ShapeProtocol)}")

# TriangleLike 也自动满足 ShapeProtocol
t = TriangleLike(3, 4, 5)
print(f"isinstance(TriangleLike(3,4,5), ShapeProtocol): "
      f"{isinstance(t, ShapeProtocol)}")

print(f"TriangleLike area: {process_shape_proto(t):.2f}")
```

### 示例3：完整的泛型数据结构

```python
"""
完整的泛型数据结构实现——类型安全的二叉搜索树
"""
from typing import TypeVar, Generic, Optional, Iterator

T = TypeVar('T')


class TreeNode(Generic[T]):
    """二叉搜索树节点"""

    def __init__(self, value: T):
        self.value: T = value
        self.left: Optional[TreeNode[T]] = None
        self.right: Optional[TreeNode[T]] = None

    def __repr__(self) -> str:
        return f"TreeNode({self.value!r})"


class BST(Generic[T]):
    """类型安全的二叉搜索树"""

    def __init__(self):
        self._root: Optional[TreeNode[T]] = None
        self._size: int = 0

    def insert(self, value: T) -> None:
        """插入值"""
        if self._root is None:
            self._root = TreeNode(value)
        else:
            self._insert_recursive(self._root, value)
        self._size += 1

    def _insert_recursive(self, node: TreeNode[T], value: T) -> None:
        if value < node.value:
            if node.left is None:
                node.left = TreeNode(value)
            else:
                self._insert_recursive(node.left, value)
        elif value > node.value:
            if node.right is None:
                node.right = TreeNode(value)
            else:
                self._insert_recursive(node.right, value)
        # value == node.value: 不插入重复值

    def search(self, value: T) -> bool:
        """搜索值"""
        return self._search_recursive(self._root, value)

    def _search_recursive(self, node: Optional[TreeNode[T]], value: T) -> bool:
        if node is None:
            return False
        if value < node.value:
            return self._search_recursive(node.left, value)
        elif value > node.value:
            return self._search_recursive(node.right, value)
        return True

    def inorder(self) -> list[T]:
        """中序遍历（返回排序后的列表）"""
        result: list[T] = []
        self._inorder_recursive(self._root, result)
        return result

    def _inorder_recursive(self, node: Optional[TreeNode[T]], result: list[T]) -> None:
        if node is not None:
            self._inorder_recursive(node.left, result)
            result.append(node.value)
            self._inorder_recursive(node.right, result)

    def __len__(self) -> int:
        return self._size

    def __contains__(self, value: T) -> bool:
        return self.search(value)

    def __iter__(self) -> Iterator[T]:
        return iter(self.inorder())


# 使用
tree: BST[int] = BST()
values = [50, 30, 70, 20, 40, 60, 80]
for v in values:
    tree.insert(v)

print(f"树的大小: {len(tree)}")
print(f"中序遍历: {tree.inorder()}")
print(f"搜索 40: {40 in tree}")
print(f"搜索 45: {45 in tree}")

# 泛型保证类型安全
str_tree: BST[str] = BST()
for name in ["banana", "apple", "cherry", "date"]:
    str_tree.insert(name)

print(f"字符串树中序遍历: {str_tree.inorder()}")
print(f"搜索 'cherry': {'cherry' in str_tree}")
```

## 23.5 常见陷阱与最佳实践

### 陷阱1：类型标注不强制运行时类型

```python
def add(a: int, b: int) -> int:
    return a + b

# Python 运行时不检查类型！
result = add("hello", " world")  # 不会报错！
print(result)  # "hello world" — 字符串拼接而非加法

# 类型标注只是提示，不产生任何运行时效果
# 需要运行时类型检查，使用 pydantic 或 TypeGuard
```

### 陷阱2：泛型类型擦除

```python
from typing import Generic, TypeVar

T = TypeVar('T')

class Box(Generic[T]):
    def __init__(self, value: T):
        self.value = value

# 运行时 Box[int] 和 Box[str] 是同一个类
b1: Box[int] = Box(42)
b2: Box[str] = Box("hello")

print(type(b1) is type(b2))  # True — 泛型参数在运行时被擦除
print(type(b1))               # <class '__main__.Box'>

# 无法在运行时区分 Box[int] 和 Box[str]
# isinstance(b1, Box[int])  # TypeError: isinstance() doesn't support generics
```

### 陷阱3：Protocol 方法签名不匹配

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class Closeable(Protocol):
    def close(self) -> None: ...

class BadCloseable:
    def close(self, force: bool = False) -> None:
        """参数签名不匹配"""
        pass

# 运行时 isinstance 检查通过（只检查方法是否存在）
print(isinstance(BadCloseable(), Closeable))  # True

# 但 mypy 会报错：
# Argument 1 to "close" of "Closeable" has incompatible type
```

### 陷阱4：循环引用的类型标注

```python
# 错误：前向引用问题
class Node:
    def __init__(self, value: int, next: Node):  # NameError!
        self.value = value
        self.next = next

# 方案1：使用字符串注解
class Node:
    def __init__(self, value: int, next: 'Node'):
        self.value = value
        self.next = next

# 方案2：使用 from __future__ import annotations（推荐）
from __future__ import annotations  # Python 3.7+

class Node:
    def __init__(self, value: int, next: Node):  # 现在可以了
        self.value = value
        self.next = next
```

### 陷阱5：Optional 误用

```python
from typing import Optional

# Optional[T] 等价于 T | None，不是"可选参数"
# 错误理解
def func(x: Optional[int] = None):  # 正确：参数默认为 None
    if x is not None:
        return x + 1
    return 0

# Optional 不等于"可以不传"
# 以下写法表示参数是必须的，但可以是 None
def func2(x: Optional[int]):  # 调用时必须传参！
    ...

# 正确：可省略的参数应该有默认值
def func3(x: Optional[int] = None):  # 可以不传
    ...
```

### 最佳实践总结

| 场景 | 推荐做法 | 原因 |
|------|----------|------|
| 定义接口 | 优先用 `Protocol` | 结构化类型更灵活，减少耦合 |
| 需要运行时检查 | 用 `@runtime_checkable` | 默认 Protocol 不支持 `isinstance` |
| 泛型容器 | 用 `Generic[T]` 或 `class X[T]` | 保持类型安全 |
| 类型别名 | 用 `TypeAlias` 或 PEP 695 语法 | 提高可读性 |
| 前向引用 | 用 `from __future__ import annotations` | 避免循环引用 |
| 函数签名泛型 | 用 `ParamSpec` | 保留参数签名信息 |
| 返回自身类型 | 用 `Self`（PEP 673） | 替代 `TypeVar` 的自引用 |
| 重写方法 | 用 `@override`（PEP 698） | 显式声明，避免拼写错误 |
| 静态检查 | 使用 mypy strict 模式 | `--strict` 提供最强保证 |
| 类型标注与运行时 | 理解标注不等于检查 | 需要运行时检查用 pydantic |
