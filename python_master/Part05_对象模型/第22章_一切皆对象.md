# 第22章 一切皆对象

> "In Python, everything is an object." 这不仅是一句口号，而是 CPython 实现层面的铁律。理解这一点，是理解 Python 对象模型的基石。

---

## 22.1 概念与语法

### 22.1.1 对象的三大要素

Python 中每个对象都拥有三个根本属性：

| 要素 | 内建函数 | 含义 | C 层面对应 |
|------|----------|------|-----------|
| identity | `id()` | 对象的唯一标识，生命周期内不变 | 对象在内存中的地址（`PyObject *` 指针值） |
| type | `type()` | 对象的类型，决定了对象支持的操作 | `ob_type` 指针指向的 `PyTypeObject` |
| value | 直接访问 | 对象存储的数据 | `ob_size` + 紧随 `PyObject` 之后的特化数据 |

```python
# 三大要素的直观演示
s = "hello"
print(f"id: {id(s)}")       # 例如 140234865746480 — 即对象在内存中的地址
print(f"type: {type(s)}")    # <class 'str'>
print(f"value: {s}")         # hello

# id() 的返回值等价于对象的内存地址
import ctypes
addr = id(s)
# 通过 ctypes 可以间接证实 id 即地址（仅限 CPython）
raw_ptr = ctypes.cast(addr, ctypes.py_object).value
print(f"通过地址取回对象: {raw_ptr}")  # hello
```

**identity 的不变性**：一个对象的 `id()` 在其整个生命周期内绝不变更。即使对象被移动（如某些 GC 实现），CPython 保证 `id()` 的语义稳定——因为它就是对象指针的整数值。

**type 的恒定性**：在 CPython 中，对象的类型一旦创建就不可更改（`ob_type` 指针在对象创建时设定，此后不再变动）。唯一的例外是 `__class__` 赋值，但这是在 Python 层面的操作，底层仍然通过 `heapreplace` 等机制完成。

**value 的可变性**：取决于类型——不可变类型（int, str, tuple）的 value 不可变，可变类型（list, dict, set）的 value 可变。

### 22.1.2 "一切皆对象"的完整含义

"一切皆对象"意味着 Python 中没有"原始类型"（primitive types）的概念。与 Java 中 `int` 和 `Integer` 的区分不同，Python 中的整数就是对象，没有装箱/拆箱（boxing/unboxing）的开销差异——因为一开始就是对象。

```python
# 整数是对象
x = 42
print(type(x))        # <class 'int'>
print(x.__class__)    # <class 'int'>
print(x.real)         # 42  — 对象有属性
print(x.bit_length()) # 6   — 对象有方法
print(x.numerator)    # 42  — 对象有属性

# 函数是对象
def greet(name):
    return f"Hello, {name}"

print(type(greet))              # <class 'function'>
print(greet.__name__)           # 'greet'
print(greet.__code__.co_code)   # 函数的字节码

# 类本身也是对象
class MyClass:
    pass

print(type(MyClass))            # <class 'type'>
print(isinstance(MyClass, type)) # True — 类是 type 的实例

# 模块是对象
import sys
print(type(sys))                # <class 'module'>

# even None 是对象
print(type(None))               # <class 'NoneType'>
print(id(None))                 # 有确定的 id
```

## 22.2 原理与机制

### 22.2.1 PyObject 结构体——一切对象的根基

在 CPython 的 C 源码中，**所有** Python 对象都以 `PyObject` 结构体开头。这是整个对象模型的根基。

```c
// CPython 源码: Include/object.h (Python 3.12)

// 前向声明
typedef struct _object PyObject;

// 对象的核心结构体
struct _object {
    Py_ssize_t ob_refcnt;          // 引用计数
    PyTypeObject *ob_type;         // 指向类型对象的指针
};

// 带变长数据的对象头部
typedef struct {
    PyObject ob_base;               // 继承 PyObject
    Py_ssize_t ob_size;            // 变长部分的元素数量
} PyVarObject;
```

**关键设计**：

1. **`ob_refcnt`**：引用计数，CPython 内存管理的核心机制。每次赋值、传参时递增；变量离开作用域或被重新赋值时递减。归零即触发回收。

2. **`ob_type`**：指向类型对象的指针。这根指针决定了"这个对象是什么"，以及"这个对象能做什么"。CPython 的方法查找、运算符分派全依赖此指针。

3. **`PyVarObject`**：对于 list、str、tuple 等变长对象，在 `PyObject` 之后追加 `ob_size` 字段，记录元素个数。这是一种 C 语言的"继承"手法——`PyVarObject` 的前两个成员与 `PyObject` 完全一致，因此任何接受 `PyObject *` 的函数也能接受 `PyVarObject *`。

### 22.2.2 具体类型的 C 结构体

每种内置类型都在 `PyObject`（或 `PyVarObject`）之上扩展了自己的字段：

```c
// CPython 源码: Include/cpython/longintrep.h (简化)
// int 对象 — 任意精度整数
typedef struct _longobject {
    PyObject ob_base;          // 继承 PyObject
    Py_ssize_t ob_size;       // 数字的符号和大小（用补码表示符号）
    digit ob_digit[1];         // 变长数组，存储数字的"位"（30-bit digits）
} PyLongObject;

// CPython 源码: Include/cpython/listobject.h (简化)
// list 对象
typedef struct {
    PyObject ob_base;          // 继承 PyObject（实际是 PyVarObject）
    Py_ssize_t ob_size;       // 列表中元素的数量
    PyObject **ob_item;       // 指向元素指针数组的指针
    Py_ssize_t allocated;     // 已分配的空间大小
} PyListObject;

// CPython 源码: Include/cpython/dictobject.h (简化)
// dict 对象
typedef struct {
    PyObject ob_base;
    Py_ssize_t ma_used;        // 已使用的条目数
    uint64_t ma_version_tag;  // 版本标签，用于快速检测字典是否变化
    PyDictKeysObject *ma_keys; // 键的存储结构
    PyObject **ma_values;     // 值的存储（split dict 优化）
} PyDictObject;
```

**所有类型都以 `PyObject` 开头**，这意味着 CPython 中可以安全地将任何 Python 对象的指针强制转换为 `PyObject *`，并访问 `ob_refcnt` 和 `ob_type`。这是 CPython 多态的实现基础。

### 22.2.3 类型的类型——PyTypeObject 与元类体系

在 Python 中，`type(obj)` 返回对象的类型，而 `type(type(obj))` 返回 `type`——类型的类型。这一切在 C 层面如何实现？

```c
// CPython 源码: Include/cpython/object.h (简化)
// 类型对象本身的结构体
typedef struct _typeobject {
    PyObject_VAR_HEAD                  // 包含 ob_refcnt, ob_type, ob_size
    const char *tp_name;               // 类型名称，如 "int", "str"
    Py_ssize_t tp_basicsize;           // 该类型实例的基本大小
    Py_ssize_t tp_itemsize;            // 变长部分的元素大小
    // ... 超过 70 个槽位（slot），定义了该类型的全部行为
    destructor tp_dealloc;             // 析构函数
    Py_ssize_t tp_vectorcall_offset;    // 向量调用偏移量
    getattrfunc tp_getattr;            // __getattr__
    setattrfunc tp_setattr;            // __setattr__
    PyAsyncMethods *tp_as_async;       // __await__, __aiter__
    reprfunc tp_repr;                   // __repr__
    PyNumberMethods *tp_as_number;     // 数值运算方法集
    PySequenceMethods *tp_as_sequence;  // 序列方法集
    PyMappingMethods *tp_as_mapping;   // 映射方法集
    // ... 还有几十个
    PyObject *tp_dict;                  // 类的 __dict__
    PyObject *tp_bases;                 // 类的基类元组
    // ...
} PyTypeObject;
```

**`PyTypeObject` 自身也是 `PyObject`**（通过 `PyObject_VAR_HEAD` 宏展开），所以类型对象也有自己的 `ob_type`。这个 `ob_type` 指向谁？答案是 `PyType_Type`——即 Python 中的 `type` 本身。

```
对象的 ob_type ──→ PyTypeObject (如 &PyLong_Type)
                          │
                          │ ob_type
                          ↓
                    PyTypeObject (&PyType_Type)
                          │
                          │ ob_type
                          ↓
                    PyTypeObject (&PyType_Type)  ← 指向自身！
```

这就是著名的"鸡与蛋"问题：`type` 的类型是 `type` 本身。在 CPython 源码中，`PyType_Type` 的 `ob_type` 被硬编码为指向自身：

```c
// CPython 源码: Objects/typeobject.c
PyTypeObject PyType_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0)   // ob_type 指向自身！
    "type",                                     /* tp_name */
    sizeof(PyHeapTypeObject),                   /* tp_basicsize */
    // ...
};
```

**元类链的完整证明**：

```python
# 元类链：obj -> type(obj) -> type(type(obj)) -> ...
x = 42
print(type(x))              # <class 'int'>
print(type(type(x)))         # <class 'type'>
print(type(type(type(x))))   # <class 'type'> — 链到此终止，type 的类型是 type 自身

# 验证 type 的 ob_type 指向自身
print(type is type(type))    # True

# 所有内置类型的类型链
for obj in [42, "hello", 3.14, True, None, [], {}, set(), (1,)]:
    t = type(obj)
    print(f"{obj!r:12} -> type: {t.__name__:10} -> type of type: {type(t).__name__}")
# 输出：
# 42           -> type: int         -> type of type: type
# 'hello'      -> type: str         -> type of type: type
# 3.14         -> type: float       -> type of type: type
# True         -> type: bool        -> type of type: type
# None         -> type: NoneType    -> type of type: type
# []           -> type: list        -> type of type: type
# {}           -> type: dict        -> type of type: type
# set()        -> type: set         -> type of type: type
# (1,)         -> type: tuple       -> type of type: type
```

### 22.2.4 对象的创建过程

以 `x = 42` 为例，追踪 CPython 从源码到对象的完整路径：

1. **编译阶段**：Python 编译器将 `42` 编译为 `LOAD_CONST` 字节码，常量 `42` 存入 `co_consts` 元组。

2. **执行阶段**：
   ```c
   // CPython 源码: Python/ceval.c (简化)
   TARGET(LOAD_CONST) {
       PyObject *value = GETITEM(consts, oparg);  // 从 co_consts 取出
       Py_INCREF(value);                           // 引用计数 +1
       PUSH(value);                                // 压入操作数栈
       DISPATCH();
   }
   ```

3. **小整数缓存**：对于 `-5` 到 `256` 的整数，CPython 在启动时预先创建：
   ```c
   // CPython 源码: Objects/longobject.c
   #define NSMALLPOSINTS           257
   #define NSMALLNEGINTS           5
   static PyLongObject small_ints[NSMALLNEGINTS + NSMALLPOSINTS];
   ```
   因此 `x = 42` 不会创建新对象，而是返回缓存的 `small_ints[42 + 5]`。

4. **大整数创建**：对于超出缓存范围的整数，CPython 调用 `PyLong_FromLong` 或 `PyLong_FromString` 动态分配 `PyLongObject`。

### 22.2.5 引用计数的实现

```c
// CPython 源码: Include/object.h
#define Py_INCREF(op)  (                         \
    _PyObject_CAST(op)->ob_refcnt++)             // 引用计数 +1

#define Py_DECREF(op)                            \
    do {                                          \
        PyObject *_py_decref_op = _PyObject_CAST(op); \
        if (--_py_decref_op->ob_refcnt == 0) {   // 引用计数 -1，若为 0
            _PyObject_DEALLOC(_py_decref_op);     // 触发析构
        }                                         \
    } while (0)
```

引用计数的致命缺陷是**循环引用**。CPython 通过循环垃圾收集器（cyclic GC）解决：

```c
// CPython 源码: Include/objimpl.h
// 支持 GC 的对象必须以此头部开始
typedef union _gc_head {
    struct {
        union _gc_head *gc_next;  // 双向链表前驱
        union _gc_head *gc_prev;  // 双向链表后继
        Py_uintptr_t gc_refs;     // GC 使用的引用标记
    } gc;
    double dummy;  // 确保对齐
} PyGC_Head;
```

容器对象（list, dict, set, 自定义类实例等）在创建时会被注册到 GC 的双向链表中。当 GC 运行时，通过 `gc_refs` 标记-清除算法检测循环引用。

### 22.2.6 函数也是对象

Python 中的函数是 `function` 类的实例，其 C 层面结构体为：

```c
// CPython 源码: Include/cpython/funcobject.h
typedef struct {
    PyObject ob_base;               // 继承 PyObject
    PyObject *func_globals;        // 全局命名空间（__globals__）
    PyObject *func_builtins;       // 内建命名空间
    PyObject *func_name;           // 函数名（__name__）
    PyObject *func_qualname;       // 限定名（__qualname__）
    PyObject *func_code;           // 代码对象（__code__）
    PyObject *func_defaults;       // 默认参数值元组（__defaults__）
    PyObject *func_kwdefaults;    // 关键字参数默认值字典（__kwdefaults__）
    PyObject *func_closure;        // 闭包单元格元组（__closure__）
    PyObject *func_doc;            // 文档字符串（__doc__）
    PyObject *func_dict;           // 函数属性字典（__dict__）
    PyObject *func_weakreflist;    // 弱引用列表
    PyObject *func_annotations;    // 类型注解字典（__annotations__）
    PyObject *func_typeparams;     // 类型参数（PEP 695）
    int func_flags;                 // 各种标志位
    uint8_t func_version;          // 函数版本号（用于快速验证缓存）
} PyFunctionObject;
```

**函数作为对象的关键证明**：

```python
def add(a, b):
    """Return the sum of a and b."""
    return a + b

# 1. 函数有 id、type、value（三大要素齐全）
print(f"id: {id(add)}")              # 有唯一标识
print(f"type: {type(add)}")          # <class 'function'>

# 2. 函数有属性（对象的特征）
print(add.__name__)                   # 'add'
print(add.__doc__)                    # 'Return the sum of a and b.'
print(add.__code__.co_varnames)       # ('a', 'b')

# 3. 函数可以作为参数传递（一等公民）
def apply(f, x, y):
    return f(x, y)
print(apply(add, 3, 4))               # 7

# 4. 函数可以作为返回值
def make_adder(n):
    def inner(x):
        return x + n
    return inner
add5 = make_adder(5)
print(add5(10))                       # 15

# 5. 函数可以存储在数据结构中
ops = [add, lambda x, y: x * y]
for op in ops:
    print(op(3, 4))                   # 7, 12
```

### 22.2.7 模块也是对象

```c
// CPython 源码: Include/moduleobject.h
typedef struct {
    PyObject ob_base;               // 继承 PyObject
    PyObject *md_dict;              // 模块的命名空间字典
} PyModuleObject;
```

模块对象的 `md_dict` 就是模块的 `__dict__`，存储了模块内所有定义的变量、函数、类。

```python
import math

# 模块是对象——三大要素齐全
print(f"id: {id(math)}")          # 有唯一标识
print(f"type: {type(math)}")      # <class 'module'>

# 模块的 __dict__ 即模块命名空间
print(type(math.__dict__))         # <class 'mappingproxy'>

# 模块可以动态添加属性（因为它是对象）
import math
math.my_custom_attr = "hello"
print(math.my_custom_attr)         # hello

# 模块可以作为参数传递
def describe_module(mod):
    return f"Module {mod.__name__} has {len(dir(mod))} attributes"

print(describe_module(math))       # Module math has 75 attributes

# 模块可以被存储在数据结构中
import sys
import json
import collections

modules = [sys, json, collections]
for m in modules:
    print(f"{m.__name__}: {type(m)}")
```

## 22.3 使用场景

### 22.3.1 利用"一切皆对象"实现策略模式

```python
"""策略模式——利用函数即对象"""
from typing import Callable

def discount_regular(price: float) -> float:
    return price * 0.95

def discount_vip(price: float) -> float:
    return price * 0.8

def discount_svip(price: float) -> float:
    return price * 0.6

# 策略映射：将函数作为值存入字典
STRATEGIES: dict[str, Callable[[float], float]] = {
    "regular": discount_regular,
    "vip": discount_vip,
    "svip": discount_svip,
}

def calculate_price(price: float, level: str) -> float:
    strategy = STRATEGIES[level]  # 取出函数对象
    return strategy(price)         # 调用函数对象

print(calculate_price(100, "regular"))  # 95.0
print(calculate_price(100, "vip"))       # 80.0
print(calculate_price(100, "svip"))      # 60.0
```

### 22.3.2 利用类型对象动态分发

```python
"""类型对象即对象——动态类型分发"""
import math

# 类型对象本身可以作为字典的键、函数的参数
SHAPE_AREA: dict[type, callable] = {}

def register(cls):
    """装饰器：将类注册到面积计算表"""
    def decorator(func):
        SHAPE_AREA[cls] = func
        return func
    return decorator

class Circle:
    def __init__(self, radius):
        self.radius = radius

class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

@register(Circle)
def circle_area(c: Circle) -> float:
    return math.pi * c.radius ** 2

@register(Rectangle)
def rect_area(r: Rectangle) -> float:
    return r.width * r.height

def area(shape) -> float:
    return SHAPE_AREA[type(shape)](shape)

print(area(Circle(5)))      # 78.53981633974483
print(area(Rectangle(3, 4))) # 12.0
```

### 22.3.3 内省与元编程

```python
"""利用对象的 __dict__ 实现对象序列化"""
import json

def to_json(obj) -> str:
    """将任意对象转为 JSON（利用一切皆对象的特性）"""
    if isinstance(obj, (str, int, float, bool)) or obj is None:
        return json.dumps(obj)
    if isinstance(obj, (list, tuple)):
        return json.dumps([to_json(item) for item in obj])
    if isinstance(obj, dict):
        return json.dumps({k: to_json(v) for k, v in obj.items()})
    # 对于自定义对象，利用 __dict__ 内省
    if hasattr(obj, '__dict__'):
        return json.dumps({k: to_json(v) for k, v in obj.__dict__.items()})
    return json.dumps(str(obj))

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(3, 4)
print(to_json(p))  # {"x": 3, "y": 4}
```

## 22.4 示例代码

### 示例1：深入探索对象的 C 层结构

```python
"""
深入探索 Python 对象的内部结构
通过 ctypes 模拟访问 PyObject 结构体
"""
import ctypes
import sys

# ============================================================
# 模拟 PyObject 结构体的内存布局
# CPython 3.12 中：
#   struct _object {
#       Py_ssize_t ob_refcnt;    // 8 bytes (64-bit)
#       PyTypeObject *ob_type;   // 8 bytes (64-bit)
#   };
# ============================================================

class PyObjectStruct(ctypes.Structure):
    """对应 CPython 的 PyObject 结构体"""
    _fields_ = [
        ("ob_refcnt", ctypes.c_ssize_t),  # 引用计数
        ("ob_type", ctypes.c_void_p),     # 类型指针
    ]

def inspect_object_header(obj):
    """读取 Python 对象的 PyObject 头部"""
    addr = id(obj)
    py_obj = PyObjectStruct.from_address(addr)
    refcnt = py_obj.ob_refcnt
    type_ptr = py_obj.ob_type

    # 获取类型对象的信息
    type_obj = type(obj)
    type_addr = id(type_obj)

    return {
        "address": hex(addr),
        "refcnt": refcnt,
        "type_ptr": hex(type_ptr) if type_ptr else "NULL",
        "type_name": type_obj.__name__,
        "type_address": hex(type_addr),
        "sys_refcnt": sys.getrefcount(obj),  # 包含传参的临时引用
    }

# ---- 测试各种类型的对象头部 ----

# 1. 小整数（缓存）
a = 42
info = inspect_object_header(a)
print(f"整数 42: 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

# 2. 字符串
s = "hello"
info = inspect_object_header(s)
print(f"字符串 'hello': 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

# 3. 列表
lst = [1, 2, 3]
info = inspect_object_header(lst)
print(f"列表 [1,2,3]: 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

# 4. 函数
def my_func():
    pass
info = inspect_object_header(my_func)
print(f"函数 my_func: 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

# 5. None
info = inspect_object_header(None)
print(f"None: 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

# 6. True
info = inspect_object_header(True)
print(f"True: 地址={info['address']}, 引用计数={info['refcnt']}, 类型={info['type_name']}")

print("\n--- 引用计数变化演示 ---")
x = []                    # 创建列表
```

### 示例4：自定义元类——控制类型的创建

```python
"""
元类：类型的类型
演示如何通过元类拦截和控制类的创建过程
"""

class Meta(type):
    """自定义元类，记录所有通过它创建的类"""

    # 类的注册表
    _registry = {}

    def __new__(mcs, name, bases, namespace, **kwargs):
        print(f"  [Meta.__new__] 创建类 {name}")
        print(f"    bases = {bases}")
        print(f"    namespace 的键 = {list(namespace.keys())}")

        # 可以修改命名空间
        namespace['_created_by_meta'] = True

        # 调用 type.__new__ 创建类对象
        cls = super().__new__(mcs, name, bases, namespace, **kwargs)
        return cls

    def __init__(cls, name, bases, namespace, **kwargs):
        print(f"  [Meta.__init__] 初始化类 {name}")
        # 注册到元类的注册表
        Meta._registry[name] = cls
        super().__init__(name, bases, namespace, **kwargs)

    def __call__(cls, *args, **kwargs):
        print(f"  [Meta.__call__] 实例化 {cls.__name__}，args={args}, kwargs={kwargs}")
        # 这里可以拦截实例化过程
        instance = super().__call__(*args, **kwargs)
        return instance


# 使用元类创建类
print("=== 创建 Animal 类 ===")
class Animal(metaclass=Meta):
    """动物基类"""

    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError

print("\n=== 创建 Dog 类 ===")
class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

print("\n=== 实例化 Dog ===")
d = Dog("Rex")

print(f"\nDog 的类型: {type(Dog)}")
print(f"Animal 的类型: {type(Animal)}")
print(f"Meta 的类型: {type(Meta)}")

# 元类链
print(f"\ntype(Dog) = {type(Dog)}")
print(f"type(type(Dog)) = {type(type(Dog))}")
print(f"type(type(type(Dog))) = {type(type(type(Dog)))}")

# 注册表
print(f"\n元类注册表: {list(Meta._registry.keys())}")

# 类对象本身是元类的实例
print(f"\nisinstance(Dog, Meta) = {isinstance(Dog, Meta)}")
print(f"isinstance(Dog, type) = {isinstance(Dog, type)}")
print(f"isinstance(Animal, type) = {isinstance(Animal, type)}")
```

## 22.5 常见陷阱与最佳实践

### 陷阱1：误用 `is` 比较值

```python
# 错误：用 is 比较整数/字符串的值
a = 257
b = 257
# a is b 可能是 False！（超出小整数缓存范围）

# 正确：用 == 比较值
assert a == b

# is 仅用于判断对象身份（如与 None 比较）
x = None
if x is None:       # 正确：与单例比较
    pass
if x == None:        # 可以工作但不推荐
    pass
```

### 陷阱2：默认参数的可变对象

```python
# 错误：默认参数是同一个对象
def append_to(item, lst=[]):  # lst 在函数定义时创建一次
    lst.append(item)
    return lst

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] — 不是 [2]！因为 lst 是同一个对象

# 正确：使用 None 作为哨兵值
def append_to(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

print(append_to(1))  # [1]
print(append_to(2))  # [2]
```

### 陷阱3：混淆类型检查方式

```python
# 不推荐：type() 比较
if type(x) == int:    # 不考虑子类
    pass

# 不推荐：type() 比较（更差的写法）
if type(x) is int:     # 同上，且过于依赖实现细节
    pass

# 推荐：isinstance() 检查
if isinstance(x, int):  # 考虑子类（如 bool 是 int 的子类）
    pass

# 但注意 bool 是 int 的子类
print(isinstance(True, int))   # True
print(type(True) == int)       # False

# 如果需要排除子类
if type(x) is int:             # 严格匹配
    pass
```

### 陷阱4：循环引用导致内存泄漏

```python
import gc
import weakref

class Node:
    def __init__(self, name):
        self.name = name
        self.parent = None
        self.children = []

    def add_child(self, child):
        self.children.append(child)
        child.parent = self  # 循环引用！

# 创建循环引用
parent = Node("root")
child = Node("child")
parent.add_child(child)

# 即使删除变量，对象也不会被回收（循环引用）
del parent
del child

# CPython 的循环 GC 会检测并回收循环引用
collected = gc.collect()
print(f"GC 回收了 {collected} 个对象")

# 最佳实践：使用弱引用打破循环
class BetterNode:
    def __init__(self, name):
        self.name = name
        self._parent_ref = None  # 弱引用
        self.children = []

    @property
    def parent(self):
        if self._parent_ref is not None:
            return self._parent_ref()
        return None

    @parent.setter
    def parent(self, node):
        self._parent_ref = weakref.ref(node)

    def add_child(self, child):
        self.children.append(child)
        child.parent = self  # 使用弱引用，不增加引用计数

parent = BetterNode("root")
child = BetterNode("child")
parent.add_child(child)

print(f"child.parent = {child.parent.name}")  # root
del parent  # 没有循环引用，child.parent 变为 None
```

### 陷阱5：误解 `__class__` 赋值

```python
# Python 3 允许修改对象的 __class__
class A:
    x = 1

class B:
    x = 2

obj = A()
print(type(obj))    # <class 'A'>
print(obj.x)        # 1

obj.__class__ = B   # 在 Python 层面改变对象的类型！
print(type(obj))    # <class 'B'>
print(obj.x)        # 2

# 但这仅限内存布局兼容的类型
# 以下操作会抛出 TypeError:
# (1, 2).__class__ = list  # TypeError: __class__ assignment only supported
#                            # for heap types or subtype of heap type
```

### 最佳实践总结

| 场景 | 推荐做法 | 原因 |
|------|----------|------|
| 值比较 | 用 `==` | `is` 比较身份，不保证值相同 |
| 身份比较（None等单例） | 用 `is` | PEP 8 推荐，语义更清晰 |
| 类型检查 | 用 `isinstance()` | 支持继承，更灵活 |
| 精确类型匹配 | 用 `type(x) is T` | 排除子类 |
| 打破循环引用 | 用 `weakref` | 避免内存泄漏 |
| 默认参数 | 用 `None` 作哨兵 | 避免共享可变对象 |
| 属性访问 | 优先用 `getattr(obj, attr, default)` | 比 `hasattr` + `getattr` 更高效 |
| 动态属性 | 谨慎使用 `__class__` 赋值 | 仅限内存布局兼容的类型 |
