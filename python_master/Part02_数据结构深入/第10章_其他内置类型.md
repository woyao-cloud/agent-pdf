# 第10章 其他内置类型

Python 的类型体系远不止 `list`、`dict`、`str` 这些高频容器。`bytes`/`bytearray` 处理二进制数据，`memoryview` 实现零拷贝缓冲区协议，`frozenset` 提供不可变集合，`collections` 模块在基础容器之上构建了更专用的数据结构，`enum` 为有限状态空间赋予类型安全，`dataclasses` 则用声明式语法消灭样板代码。本章将从 CPython 源码层逐一剖析这些类型的实现原理与使用边界。

---

## 1. 概念与语法

### 1.1 bytes 与 bytearray

`bytes` 是不可变的字节序列，`bytearray` 是可变的字节序列。二者共同构成 Python 处理二进制数据的基础设施。

```python
# 字节面量 (bytes literal)
b1 = b'hello'          # ASCII 字符直接书写
b2 = b'\x00\x01\xff'   # 十六进制转义
b3 = bytes([72, 101])   # 从可迭代整数构造

# bytearray 构造
ba1 = bytearray(b'hello')
ba2 = bytearray(10)       # 10 个零字节
ba3 = bytearray.fromhex('00 01 ff')
```

**语法要点**：
- 字节面量 `b'...'` 只能包含 ASCII 可见字符（0x20-0x7E），其余必须用转义序列
- `bytes` 和 `bytearray` 的构造函数签名相同：`bytes(source, encoding, errors)`
- `bytearray` 支持索引赋值：`ba[0] = 65`，而 `bytes` 不支持

### 1.2 memoryview

`memoryview` 创建一个引用原始缓冲区的视图对象，无需拷贝底层数据即可对缓冲区进行切片和访问。

```python
mv = memoryview(b'abcdefgh')
print(mv[2:5])       # <memory at 0x...>
print(mv[2:5].tobytes())  # b'cde'
print(mv[0])         # 97  (返回整数)

# 对 bytearray 创建可写 memoryview
ba = bytearray(b'hello')
mv_writable = memoryview(ba)
mv_writable[0] = 74  # 直接修改底层 bytearray
```

### 1.3 frozenset

`frozenset` 是不可变、可哈希的集合类型，一旦创建不可增删元素。

```python
fs = frozenset([1, 2, 3])
print(fs & frozenset([2, 3, 4]))  # frozenset({2, 3})
print(hash(fs))  # 可哈希，能作为 dict key

# 作为 dict key
cache = {frozenset(['a', 'b']): 'value'}
```

### 1.4 collections 模块

| 类型 | 基类 | 核心特性 |
|------|------|----------|
| `Counter` | `dict` | 计数器，支持算术运算 |
| `deque` | object | 双端队列，O(1) 首尾操作 |
| `defaultdict` | `dict` | 自动填充缺失 key |
| `OrderedDict` | `dict` | 保持插入顺序（3.7+ 普通 dict 也有序，但 OrderedDict 语义更强） |
| `ChainMap` | object | 多映射逻辑合并 |
| `namedtuple` | tuple | 具名元组，属性访问 |

## 2. 原理与机制

### 2.1 bytes 与 bytearray 的 CPython 实现

#### PyBytesObject 结构（Include/cpython/bytesobject.h）

```c
typedef struct {
    PyObject_VAR_HEAD        // ob_refcnt, ob_type, ob_size
    Py_hash_t ob_shash;      // 缓存的哈希值，-1 表示未计算
    char ob_sval[1];          // 柔性数组，实际长度 = ob_size + 1（含 \0）
} PyBytesObject;
```

关键设计：
- **柔性数组（flexible array member）**：`ob_sval[1]` 是 C 语言技巧，`ob_sval` 实际紧跟在结构体头部之后，长度由 `ob_size` 决定
- **尾零保证**：`ob_sval[ob_size]` 始终为 `'\0'`，使得 `bytes` 对象可以安全传递给期望 C 字符串的函数
- **哈希缓存**：`ob_shash` 在首次计算后缓存，避免重复计算。初始值 `-1` 表示未计算
- **内存开销**：一个 `bytes` 对象的额外开销为 `PyBytesObject` 头部大小（约 33 字节，64 位系统上）+ 1 字节尾零。对于极小的 `bytes` 对象，这个开销占比显著

#### PyByteArrayObject 结构（Include/cpython/bytearrayobject.h）

```c
typedef struct {
    PyObject_VAR_HEAD
    Py_hash_t ob_shash;       // 缓存哈希（但 bytearray 不可哈希，此字段保留）
    char *ob_start;           // 指向堆分配的字节数组
    Py_ssize_t ob_exports;    // 导出计数（memoryview 引用计数）
    Py_ssize_t ob_alloc;      // 已分配容量
} PyByteArrayObject;
```

关键区别：
- `bytes` 的数据内联在结构体内（柔性数组），而 `bytearray` 的数据通过指针 `ob_start` 指向独立堆分配的缓冲区
- `ob_exports` 跟踪有多少 `memoryview` 正在引用此 `bytearray`。如果 `ob_exports > 0`，则调整大小操作（`resize`）会被禁止，必须先释放所有 `memoryview`
- `ob_alloc` 实现了超额分配策略：增长时按 `(newsize + (newsize >> 3) + 6) & ~3` 计算新容量，避免频繁 `realloc`

#### bytes 拼接的优化

CPython 对 `bytes` 拼接有专门优化（Python/bytesobject.c `bytes_concat`）：

```c
/* 当其中一个操作数为空时直接返回另一个的引用（共享底层数据） */
if (a_size == 0) { Py_INCREF(b); return b; }
if (b_size == 0) { Py_INCREF(a); return a; }
```

对 `%` 格式化和 `join()` 方法，CPython 会预计算总长度然后一次性分配，比循环 `+=` 高效得多。

### 2.2 memoryview 与 buffer protocol

#### buffer protocol（Include/buffer.h）

`buffer protocol` 是 Python 对象暴露原始内存缓冲区的标准接口，定义在 `PyBufferProcs` 中：

```c
typedef struct {
    getbufferproc bf_getbuffer;    // 获取缓冲区视图
    releasebufferproc bf_releasebuffer;  // 释放缓冲区视图
} PyBufferProcs;
```

支持 `buffer protocol` 的内置类型包括：`bytes`、`bytearray`、`array.array`、`memoryview`、`numpy.ndarray` 等。

#### memoryview 的 CPython 实现（Modules/_io/memoryview）

`memoryview` 对象的核心字段：

```c
typedef struct {
    PyObject_HEAD
    PyObject *base;           // 被引用的原始对象
    Py_buffer view;           // 缓冲区视图描述符
    int flags;                // 可读/可写标志
    Py_ssize_t start;         // 视图起始偏移
    Py_ssize_t end;           // 视图结束偏移
    // ... 多维数组支持字段
} PyMemoryViewObject;
```

**零拷贝机制**：
- `memoryview` 切片操作（`mv[start:stop]`）仅创建新的 `PyMemoryViewObject`，调整 `start`/`end` 偏移，绝不拷贝底层数据
- 与 `bytes` 切片对比：`b[start:stop]` 会分配新 `PyBytesObject` 并拷贝数据
- 对于大缓冲区的子区域访问，`memoryview` 的内存优势可达数量级

#### memoryview 的引用计数与安全

- 创建 `memoryview` 时调用 `base` 对象的 `bf_getbuffer`，增加 `ob_exports`（对于 `bytearray`）
- 释放 `memoryview` 时调用 `bf_releasebuffer`，减少 `ob_exports`
- 当 `ob_exports > 0` 时，`bytearray` 拒绝 `resize` 操作，抛出 `BufferError`

### 2.3 frozenset 的实现

`frozenset` 与 `set` 共享几乎全部底层实现（Objects/setobject.c），仅类型标志不同：

```c
// frozenset 的 tp_as_mapping->mp_ass_subscript = NULL（禁止赋值）
// set 的 tp_as_mapping->mp_ass_subscript = set_ass_sub
```

**可哈希性**的实现：
- `frozenset` 的 `tp_hash` 指向 `frozenset_hash` 函数
- `set` 的 `tp_hash` 指向 `_Py_HashPointer`（不可哈希，基于对象地址）
- `frozenset_hash` 的算法：遍历所有元素，对每个元素的哈希值执行 `xor` 和位旋转操作，确保与元素顺序无关

```c
// 简化的 frozenset 哈希计算逻辑
static Py_hash_t frozenset_hash(PyObject *self) {
    Py_hash_t h = 0;
    // 遍历 set 内所有元素
    for each element e:
        h ^= PyObject_Hash(e);
        h = _PyHASH_BITS_ROTATION(h);  // 位旋转
    // 加上元素数量以区分 {1, 2} 和 {3}
    h ^= (len * _PyHASH_BITS);
    if (h == -1) h = -2;
    return h;
}
```

### 2.4 collections 模块深入

#### Counter 的实现

`Counter` 是 `dict` 的子类，核心增量极小：

```python
class Counter(dict):
    def __init__(self, iterable=None, /, **kwds):
        super().__init__()
        self.update(iterable, **kwds)
    
    def __missing__(self, key):
        return 0  # 关键：缺失 key 返回 0 而非 KeyError
```

**算术运算**的实现原理：
- `+`：取两个 Counter 中所有 key，值取 `max(0, c1[k] + c2[k])`
- `-`：取所有 key，值取 `max(0, c1[k] - c2[k])`
- `|`：并集，取每个 key 在两个 Counter 中的 `max` 值
- `&`：交集，取每个 key 在两个 Counter 中的 `min` 值
- 结果中值为 0 或负数的 key 会被自动移除

#### deque 的块状链表实现

`deque` 不使用简单链表，而采用块状链表（block-based linked list），定义在 `Modules/_collectionsmodule.c`：

```c
#define BLOCKLEN 64  // 每个块的大小

typedef struct BLOCK {
    PyObject *data[BLOCKLEN];
    struct BLOCK *leftlink;
    struct BLOCK *rightlink;
} block;

typedef struct {
    PyObject_VAR_HEAD
    block *leftblock;     // 最左边的块
    block *rightblock;    // 最右边的块
    Py_ssize_t leftindex;  // 左块中的起始索引
    Py_ssize_t rightindex; // 右块中的结束索引
    Py_ssize_t maxlen;    // 最大长度（0 表示无限制）
    Py_ssize_t state;     // 修改计数器（用于迭代安全检测）
} dequeobject;
```

**性能特征**：
- `append`/`appendleft`：O(1)，只在右块/左块不满时修改索引；块满时分配新块（O(1) 均摊）
- `pop`/`popleft`：O(1)
- 索引访问 `d[i]`：O(n)，需从 `leftblock` 开始遍历，每 64 个元素跳一个块
- 中间插入/删除：O(n)
- `BLOCKLEN = 64` 是缓存友好性和内存浪费之间的平衡：64 个指针（512 字节）接近 L1 缓存行大小

**maxlen 的实现**：当 `len > maxlen` 时，从对侧丢弃元素。此操作在 `append` 内部完成，不需要额外遍历。

#### defaultdict 的实现

```python
class defaultdict(dict):
    def __missing__(self, key):
        if self.default_factory is None:
            raise KeyError(key)
        self[key] = self.default_factory()  # 调用工厂函数
        return self[key]
```

`default_factory` 可以为 `int`（返回 0）、`list`（返回空列表）、`set`（返回空集合）或任何无参可调用对象。

#### OrderedDict 的实现

在 Python 3.7+，普通 `dict` 已保持插入顺序，但 `OrderedDict` 仍有独特语义：

```c
// OrderedDict 内部维护双向链表
typedef struct {
    PyObject_HEAD
    PyObject *key;
    PyObject *value;
    PyObject *prev;   // 前驱节点
    PyObject *next;   // 后继节点
    Py_hash_t hash;
} _ODictNode;
```

与普通 `dict` 的关键区别：
- `popitem(last=False)` 支持 FIFO 语义（普通 `dict.popitem()` 只有 LIFO）
- `move_to_end(key)` 可将元素移到链表尾部
- `__eq__` 同时比较顺序（普通 `dict` 不比较顺序）
- 重新插入已有 key 会将其移到末尾

#### ChainMap 的实现

```python
class ChainMap(collections.abc.MutableMapping):
    def __init__(self, *maps):
        self.maps = list(maps) or [{}]  # 存储映射列表

    def __getitem__(self, key):
        for mapping in self.maps:
            try:
                return mapping[key]
            except KeyError:
                pass
        raise KeyError(key)
    
    def __setitem__(self, key, value):
        self.maps[0][key] = value  # 写入第一个映射
```

**关键设计**：
- 查找按 `maps` 列表顺序，第一个命中即返回
- 写入只修改 `maps[0]`（最前端的映射）
- `new_child(m=None)` 创建新的 `ChainMap`，将 `m` 插入 `maps[0]`
- `parents` 属性返回去掉第一个映射的 `ChainMap`

#### namedtuple 的实现

`namedtuple` 使用模板字符串在运行时动态生成类：

```python
# 简化版 namedtuple 实现
def namedtuple(typename, field_names, *, rename=False):
    # 1. 验证字段名
    # 2. 构建类定义的源代码字符串
    class_definition = _class_template.format(...)  # 包含 __new__, _make, _asdict, _replace 等
    # 3. exec 编译源代码
    namespace = dict(__name__=f'namedtuple_{typename}')
    exec(class_definition, namespace)
    # 4. 返回创建的类
    cls = namespace[typename]
    return cls
```

生成的 `__new__` 方法只接受位置参数或关键字参数，并调用 `tuple.__new__`：
- 属性访问通过 `property` 描述符实现，`fget = itemgetter(i)`
- 内存布局与 `tuple` 完全相同，没有额外字段开销
- `__slots__ = ()` 防止添加实例属性

### 2.5 enum 的 metaclass 实现

`Enum` 使用 `EnumMeta`（即 `EnumType`）作为元类，核心逻辑在 `enum.py` 中：

```python
class EnumMeta(type):
    def __new__(metacls, cls, bases, classdict, **kwds):
        # 1. 收集所有非 _descriptor 的属性作为枚举成员
        # 2. 为每个成员创建 Enum 实例（单例）
        # 3. 将成员注册到 _value2member_map_ 和 _member_map_
        # 4. 处理 auto() —— 调用 _generate_next_value_
        ...
```

**关键机制**：
- **成员单例**：每个枚举值在类创建时实例化一次，之后通过 `_value2member_map_` 查找返回同一对象
- **`__new__` vs `__init__`**：`Enum` 类通常重写 `__new__` 而非 `__init__`，因为实例化被元类拦截
- **`auto()` 机制**：调用 `_generate_next_value_(name, start, count, last_values)`，默认返回 `last_values[-1] + 1`
- **`IntEnum`** 同时继承 `int` 和 `Enum`，使得枚举值可与 `int` 直接比较和运算
- **`Flag`** 使用位运算语义：`__or__` 返回组合标志，未命名的组合值不会成为类属性

### 2.6 dataclasses 的实现

`@dataclass` 装饰器在类创建时执行，本质是一个类转换函数（`dataclasses.py` 中的 `_process_class`）：

```python
def _process_class(cls, init, repr, eq, order, unsafe_hash, frozen, ...):
    # 1. 收集所有带注解的字段
    # 2. 将注解转换为 Field 对象（通过 fields() 函数）
    # 3. 按规则生成 __init__, __repr__, __eq__, __lt__ 等方法
    # 4. 将生成的方法注入到 cls 的命名空间
    return cls
```

**`Field` 对象**（`dataclasses.Field`）：
```python
class Field:
    __slots__ = ('name', 'type', 'default', 'default_factory',
                 'repr', 'hash', 'init', 'compare', 'metadata',
                 '_field_type')
```

**`field()` 配置项**：
- `default`：默认值（不可与 `default_factory` 同时指定）
- `default_factory`：可调用对象，每次实例化时调用产生默认值
- `repr`：是否出现在 `__repr__` 中
- `hash`：是否参与 `__hash__` 计算
- `init`：是否出现在 `__init__` 参数中
- `compare`：是否参与 `__eq__`/`__lt__` 比较

**`frozen=True` 的实现**：
- 生成 `__setattr__` 和 `__delattr__`，调用时抛出 `FrozenInstanceError`
- 生成 `__hash__`（默认情况下 `frozen=True` 会启用哈希）
- 底层使用 `object.__setattr__` 在 `__init__` 中完成初始赋值，绕过冻结检查

**继承行为**：
- 父类的字段按 MRO 顺序收集，父类字段排在前面
- 子类字段不能有与父类同名字段（除非父类该字段 `default_factory` 设为 `MISSING`，即父类只声明了类型注解）
- `__init__` 参数顺序：先父类字段，再子类字段，有默认值的字段必须在无默认值的字段之后

## 3. 使用场景

### 3.1 bytes 与 bytearray

| 场景 | 推荐类型 | 原因 |
|------|----------|------|
| 网络协议数据包 | `bytes` | 不可变，安全传递，可哈希 |
| 二进制数据拼接 | `bytearray` | 可变，避免反复创建新对象 |
| 文件 I/O 缓冲区 | `bytearray` | 可原地修改，支持 `readinto()` |
| 加密哈希输入 | `bytes` | `hashlib` 要求 `bytes`-like 对象 |
| 与 C 扩展交互 | `bytes` 或 `bytearray` | 均支持 buffer protocol |

### 3.2 memoryview

- **大文件切片分析**：对 GB 级二进制文件做 `mmap` 后用 `memoryview` 切片访问
- **协议解析**：网络协议头、TLV 结构的零拷贝解析
- **numpy 互操作**：`numpy` 数组支持 buffer protocol，通过 `memoryview` 可直接共享内存
- **避免数据拷贝**：当需要将同一缓冲区的不同区域传递给多个消费者时

### 3.3 frozenset

- **dict key**：用 `frozenset` 作为缓存键，如函数参数集合去重
- **集合的集合**：`set` 中包含 `set` 需要改用 `frozenset`
- **不可变约束**：当 API 契约要求集合不被修改时，返回 `frozenset` 向调用者表达语义

### 3.4 collections 类型选择

| 类型 | 典型场景 |
|------|----------|
| `Counter` | 词频统计、投票计数、多集合运算 |
| `deque` | BFS 队列、滑动窗口、回文检测、循环缓冲区 |
| `defaultdict` | 分组（group by）、树/图邻接表、多值映射 |
| `OrderedDict` | LRU 缓存（配合 `move_to_end`）、需 FIFO 弹出的有序映射 |
| `ChainMap` | 配置层级叠加、作用域链（CLI 参数 > 环境变量 > 默认值） |
| `namedtuple` | 轻量级不可变数据传输对象、CSV 行映射、函数返回多值 |

### 3.5 enum

- **状态机**：用 `Enum` 表示有限状态
- **配置常量**：替代模块级整数常量，获得类型检查和自动文档
- **位标志**：用 `Flag`/`IntFlag` 表示权限、选项组合
- **类型安全的分支**：`match` 语句中枚举比字符串更安全

### 3.6 dataclasses

- **数据传输对象（DTO）**：API 请求/响应模型
- **配置对象**：替代手写 `__init__` 的样板代码
- **不可变值对象**：`frozen=True` 创建可哈希、不可变的值类型
- **ORM 模型基类**：许多框架的 model 基于或兼容 dataclass 语义

## 4. 示例代码

### 4.1 bytes 与 bytearray

```python
"""bytes 与 bytearray 的核心操作与性能对比"""
import timeit

def demonstrate_bytes_basics():
    """bytes 基本操作"""
    # 构造方式
    b1 = b'hello'
    b2 = bytes([0x48, 0x65, 0x6c, 0x6c, 0x6f])  # 从整数列表
    b3 = bytes('hello', encoding='utf-8')        # 从字符串
    b4 = b1 * 3                                   # 重复
    assert b1 == b2 == b3
    assert b4 == b'hellohellohello'

    # 索引与切片（注意：索引返回整数，切片返回 bytes）
    assert b1[0] == 104           # int

    def concat_plus():
        result = b''
        for _ in range(n):
            result += chunk
        return result

        return b''.join(chunk for _ in range(n))

        for _ in range(n):
            ba.extend(chunk)
        return bytes(ba)

    t2 = timeit.timeit(concat_join, number=3)
    t3 = timeit.timeit(concat_bytearray, number=3)
```

```python
def demonstrate_deque_advanced():

    # maxlen 实现滑动窗口
    def sliding_max(iterable, k):
        """滑动窗口最大值（单调队列法）"""
        dq = deque()  # 存储索引
        result = []
        for i, val in enumerate(iterable):
            while dq and iterable[dq[-1]] <= val:
                dq.pop()
            dq.append(i)
            if dq[0] <= i - k:
                dq.popleft()
            if i >= k - 1:
                result.append(iterable[dq[0]])
        return result

    assert first == ('a', 1)

    # 实现简单的 LRU 缓存
    class LRUCache:
        def __init__(self, capacity):
            self.capacity = capacity
            self.cache = OrderedDict()

        def get(self, key):
            if key not in self.cache:
                return -1
            self.cache.move_to_end(key)  # 访问时移到末尾
            return self.cache[key]

        def put(self, key, value):
            if key in self.cache:
                self.cache.move_to_end(key)
            self.cache[key] = value
            if len(self.cache) > self.capacity:
                self.cache.popitem(last=False)  # 淘汰最久未用

    lru = LRUCache(2)
    lru.put(1, 'a')
    lru.put(2, 'b')
    assert lru.get(1) == 'a'
    lru.put(3, 'c')  # 淘汰 key=2
    assert lru.get(2) == -1

    # OrderedDict 的 __eq__ 比较顺序
    od1 = OrderedDict([('a', 1), ('b', 2)])
    od2 = OrderedDict([('b', 2), ('a', 1)])
    assert od1 != od2  # 顺序不同则不等（与普通 dict 不同）
```

```python
    # 枚举与 match 语句（Python 3.10+）
    def describe(color):
        match color:
            case Color.RED:
                return "红色"
            case Color.GREEN:
                return "绿色"
            case Color.BLUE:
                return "蓝色"
            case _:
                return "未知"

    class Color(Enum):
        RED = 1
        GREEN = 2
        BLUE = 3

    assert describe(Color.GREEN) == "绿色"


if __name__ == '__main__':
    print("=== Enum 基本用法 ===")
    demonstrate_enum_basics()

    print("=== auto() ===")
    demonstrate_auto()
    print("=== Enum 高级特性 ===")
```

### 4.6 dataclasses

```python
"""dataclasses 深入"""
from dataclasses import dataclass, field, asdict, astuple, fields, replace
from dataclasses import InitVar, FrozenInstanceError
import hashlib

def demonstrate_dataclass_basics():
    """dataclass 基本用法"""
    @dataclass
    class Point:
        x: float
        y: float
        z: float = 0.0  # 有默认值的字段必须在无默认值之后

    p = Point(1.0, 2.0)
    assert p.x == 1.0
    assert p.z == 0.0
    assert repr(p) == "Point(x=1.0, y=2.0, z=0.0)"

    # asdict / astuple
    assert asdict(p) == {'x': 1.0, 'y': 2.0, 'z': 0.0}
    assert astuple(p) == (1.0, 2.0, 0.0)

    # fields：获取字段元信息
    fs = fields(p)
    assert len(fs) == 3
    assert fs[0].name == 'x'
    assert fs[2].default == 0.0

    # replace：创建副本并修改部分字段
    p2 = replace(p, x=10.0)
    assert p2 == Point(10.0, 2.0, 0.0)
    assert p == Point(1.0, 2.0, 0.0)  # 原实例不变


def demonstrate_field_config():
    """field() 配置详解"""
    @dataclass
    class Config:
        name: str
        tags: list = field(default_factory=list)      # 可变默认值必须用 factory
        internal_id: int = field(default=0, repr=False)  # 不出现在 repr 中
        hash_key: str = field(default='', hash=True, compare=False)
        skip_init: str = field(default='computed', init=False)  # 不在 __init__ 中

    c = Config(name='test')
    assert c.tags == []
    assert 'internal_id' not in repr(c)
    assert c.skip_init == 'computed'

    # 验证两个实例的 tags 是独立列表
    c1 = Config(name='a')
    c2 = Config(name='b')
    c1.tags.append('x')
    assert c1.tags == ['x']
    assert c2.tags == []  # 独立，不会共享


def demonstrate_frozen():
    """frozen=True 不可变 dataclass"""
    @dataclass(frozen=True)
    class Coordinate:
        lat: float
        lon: float

    c = Coordinate(39.9, 116.4)
    try:
        c.lat = 40.0
        assert False, "Should raise FrozenInstanceError"
    except FrozenInstanceError:
        pass

    # frozen dataclass 自动生成 __hash__
    assert hash(Coordinate(39.9, 116.4)) == hash(Coordinate(39.9, 116.4))

    # 可作为 dict key 和 set 元素
    locations = {Coordinate(39.9, 116.4): 'Beijing'}


def demonstrate_order():
    """order=True 比较方法"""
    @dataclass(order=True)
    class Version:
        major: int
        minor: int
        patch: int

    v1 = Version(1, 0, 0)
    v2 = Version(1, 1, 0)
    v3 = Version(2, 0, 0)

    assert v1 < v2 < v3
    assert sorted([v3, v1, v2]) == [v1, v2, v3]

    # sort_index: field 配置控制排序依据
    @dataclass(order=True)
    class Task:
        priority: int = field(compare=True)
        name: str = field(compare=False)
        description: str = field(compare=False)

    tasks = [Task(3, 'low'), Task(1, 'urgent'), Task(2, 'normal')]
    assert [t.name for t in sorted(tasks)] == ['urgent', 'normal', 'low']


def demonstrate_post_init():
    """__post_init__ 钩子"""
    @dataclass
    class Rectangle:
        width: float
        height: float
        area: float = field(init=False)  # 不参与 __init__
        perimeter: float = field(init=False)

        def __post_init__(self):
            self.area = self.width * self.height
            self.perimeter = 2 * (self.width + self.height)

    r = Rectangle(3, 4)
    assert r.area == 12.0
    assert r.perimeter == 14.0

    # InitVar：只在 __init__ 中传递，不存储为字段
    @dataclass
    class User:
        name: str
        password_hash: str = field(init=False, repr=False)
        password: InitVar[str] = None  # InitVar 不成为字段

        def __post_init__(self, password):
            if password is not None:
                self.password_hash = hashlib.sha256(
                    password.encode()
                ).hexdigest()[:16]

    u = User('alice', 'secret123')
    assert u.password_hash != ''
    assert not hasattr(u, 'password')  # password 不是字段
    assert 'password' not in asdict(u)


def demonstrate_inheritance():
    """dataclass 继承行为"""
    @dataclass
    class Base:
        x: int
        y: int = 0

    @dataclass
    class Child(Base):
        z: int = 0
        # 不能重写父类字段，但可以给父类无默认值的字段加默认值

    c = Child(x=1)
    assert c == Child(x=1, y=0, z=0)

    # 字段顺序：先父类，后子类
    assert [f.name for f in fields(c)] == ['x', 'y', 'z']

    # 多继承的 MRO 合并
    @dataclass
    class A:
        a: int = 1

    @dataclass
    class B:
        b: int = 2

    @dataclass
    class C(A, B):
        c: int = 3

    obj = C()
    assert obj == C(a=1, b=2, c=3)


def demonstrate_metadata():
    """field metadata 元数据"""
    @dataclass
    class Person:
        name: str = field(metadata={'max_length': 50, 'pattern': r'^\w+$'})
        age: int = field(metadata={'min': 0, 'max': 150})
        email: str = field(metadata={'format': 'email'})

    for f in fields(Person):
        print(f"  {f.name}: {f.metadata}")

    # 实际应用：ORM 验证
    def validate(instance):
        for f in fields(instance):
            value = getattr(instance, f.name)
            meta = f.metadata
            if 'min' in meta and value < meta['min']:
                raise ValueError(f"{f.name} too small: {value}")
            if 'max' in meta and value > meta['max']:
                raise ValueError(f"{f.name} too large: {value}")

    p = Person('Alice', 30, 'alice@example.com')
    validate(p)  # 通过


if __name__ == '__main__':
    print("=== dataclass 基本用法 ===")
    demonstrate_dataclass_basics()
    print("=== field() 配置 ===")
    demonstrate_field_config()
    print("=== frozen=True ===")
    demonstrate_frozen()
    print("=== order=True ===")
    demonstrate_order()
    print("=== __post_init__ ===")
    demonstrate_post_init()
    print("=== 继承行为 ===")
    demonstrate_inheritance()
    print("=== field metadata ===")
    demonstrate_metadata()
```

## 5. 常见陷阱与最佳实践

### 5.1 bytes 与 bytearray

**陷阱 1：bytes 索引返回 int，切片返回 bytes**

```python
b = b'hello'
assert b[0] == 104        # int，不是 b'h'
assert b[0:1] == b'h'     # bytes
# 错误：if b[0] == b'h'  → 104 == b'h' 永远为 False
```

**陷阱 2：字符串与 bytes 混淆**

```python
# 错误
# b'hello' + ' world'  → TypeError
# 正确
result = b'hello' + ' world'.encode()
# 或者
result = 'hello'.encode() + b' world'
```

**陷阱 3：bytearray 被 memoryview 持有时无法 resize**

```python
ba = bytearray(b'hello')
mv = memoryview(ba)
try:
    ba.append(33)  # BufferError: cannot resize
except BufferError:
    print("memoryview 持有期间不能 resize bytearray")
mv.release()      # 释放视图后才能 resize
ba.append(33)     # 现在可以了
```

**最佳实践**：
- 网络 I/O 优先用 `bytes`，拼接场景用 `bytearray`
- 大量拼接用 `b''.join(iterable)` 而非 `+=`
- 使用 `memoryview` 避免大缓冲区的切片拷贝
- 始终明确编码：`s.encode('utf-8')` / `b.decode('utf-8')`

### 5.2 memoryview

**陷阱 1：memoryview 的内存布局依赖原始对象**

```python
import array
arr = array.array('i', [1, 2, 3])  # 4 字节整数
mv = memoryview(arr)
assert mv.itemsize == 4
assert mv[0] == 1  # 返回 Python int，但底层是 4 字节

# cast 到不同格式可能对齐问题
# 错误示例：将奇数长度字节数组 cast 为 'h'（2 字节短整型）
ba = bytearray(5)  # 5 字节，不能整除 2
try:
    mv = memoryview(ba).cast('h')  # TypeError
except TypeError:
    print("cast 要求总字节数能整除 itemsize")
```

**陷阱 2：memoryview 不支持所有 bytes 方法**

```python
mv = memoryview(b'hello world')
# mv.split(b' ')  →  TypeError: memoryview doesn't have split
# mv.replace(b'h', b'H')  →  AttributeError
# 需要 tobytes() 后操作
parts = mv.tobytes().split(b' ')
```

**最佳实践**：
- 对 GB 级二进制数据始终使用 `memoryview`，不要用 `bytes` 切片
- 用 `mmap` + `memoryview` 处理大文件，避免将整个文件读入内存
- `memoryview` 操作完成后及时释放或离开作用域，以免阻止原始 `bytearray` resize

### 5.3 frozenset

**陷阱 1：frozenset 元素也必须是可哈希的**

```python
# 错误
try:
    fs = frozenset([1, [2, 3]])  # list 不可哈希
except TypeError as e:
    print(f"frozenset 元素必须可哈希: {e}")

# 正确：嵌套使用 frozenset
fs = frozenset([1, frozenset([2, 3])])
```

**陷阱 2：frozenset 运算返回 frozenset，而非 set**

```python
fs1 = frozenset([1, 2])
fs2 = frozenset([2, 3])
result = fs1 | fs2
assert isinstance(result, frozenset)  # 不是 set

# 混合运算返回左边类型
mixed = fs1 | {2, 3}     # frozenset | set → frozenset
mixed2 = {1, 2} | fs2   # set | frozenset → set
```

**陷阱 3：frozenset 的哈希值与 set 不同**

```python
# frozenset 的哈希与内容相关
# 不同内容的 frozenset 理论上可能哈希碰撞（概率极低）
fs = frozenset(range(1000000))
# 哈希计算 O(n)，对极大 frozenset 可能成为性能瓶颈
```

**最佳实践**：
- 用 `frozenset` 作为 dict key 时，确保元素数量可控
- API 返回集合时优先返回 `frozenset` 表达不可变语义
- 使用 `frozenset({k1, k2, ...})` 作为函数签名的缓存键

### 5.4 collections 模块

**陷阱 1：Counter 的负值行为**

```python
c = Counter(a=3, b=-1)
assert c['b'] == -1         # Counter 允许负值
assert c['c'] == 0          # __missing__ 返回 0

c2 = c - Counter(a=5)
assert c2 == Counter()      # 减法运算移除零和负值
# 但 c 本身仍保留 b=-1
```

**陷阱 2：deque 的索引访问是 O(n)**

```python
d = deque(range(100000))
# d[50000] 需要 O(n) 遍历
# 需要随机访问时用 list，需要首尾操作时用 deque
```

**陷阱 3：defaultdict 的工厂函数在缺失时才调用**

```python
dd = defaultdict(list)
# 访问缺失 key 会创建空列表并插入
_ = dd['new_key']
assert 'new_key' in dd  # 意外地创建了键
# 如果只想检查而不创建，用 dd.get('key', default)
```

**陷阱 4：ChainMap 的写入只影响第一个映射**

```python
defaults = {'a': 1, 'b': 2}
overrides = {'a': 10}
cm = ChainMap(overrides, defaults)
cm['b'] = 20  # 写入 overrides，不是 defaults
assert defaults == {'a': 1, 'b': 2}  # defaults 不变
assert overrides == {'a': 10, 'b': 20}
```

**陷阱 5：namedtuple 没有默认 __dict__**

```python
from collections import namedtuple
Point = namedtuple('Point', 'x y')
p = Point(1, 2)
# p.z = 3  → AttributeError: 无法添加属性
# 但可以定义子类添加方法，必须设置 __slots__ = ()
```

**最佳实践**：
- `Counter` 用于计数，`defaultdict(int)` 用于累加——语义更清晰
- `deque` 配合 `maxlen` 实现固定大小缓冲区
- `ChainMap` 适合多层配置叠加，但不适合大量随机写入
- 需要 `namedtuple` + 默认值 + 方法时，考虑 `dataclass`（更现代的替代）

### 5.5 enum

**陷阱 1：Enum 成员不是简单的整数**

```python
class Color(Enum):
    RED = 1
    GREEN = 2

# 错误用法
# if color == 1:  →  永远不会为 True（除非是 IntEnum）
# 正确用法
# if color == Color.RED:
# 或者
# if color.value == 1:
```

**陷阱 2：枚举值重复导致别名**

```python
class Status(Enum):
    PENDING = 1
    WAITING = 1  # 这不是新成员，而是 PENDING 的别名
    APPROVED = 2

assert Status.WAITING is Status.PENDING
assert len(Status) == 2  # 只有 2 个成员，不是 3
# 用 @unique 装饰器避免意外别名
```

**陷阱 3：Flag 的空值**

```python
class Permission(Flag):
    READ = auto()
    WRITE = auto()

# 没有任何标志时的值
empty = Permission(0)
assert empty == Permission(0)  # 存在但不是任何命名成员
# 检查"无标志"应使用：
assert bool(empty) == False
```

**陷阱 4：IntEnum 破坏类型安全**

```python
class Priority(IntEnum):
    LOW = 1
    HIGH = 2

class Status(IntEnum):
    PENDING = 1

# 意外的比较
assert Priority.LOW == Status.PENDING  # True！可能不是期望行为
# 解决方案：优先使用 Enum 而非 IntEnum
```

**最佳实践**：
- 优先使用 `Enum` 而非 `IntEnum`，除非确实需要与 `int` 互操作
- 始终使用 `@unique` 装饰器防止值重复
- 用 `auto()` 避免手动维护数值
- 枚举类名用单数（`Color` 而非 `Colors`）
- 需要位运算组合时使用 `Flag`/`IntFlag`

### 5.6 dataclasses

**陷阱 1：可变默认值**

```python
# 错误
@dataclass
class Bad:
    items: list = []  # TypeError: mutable default not allowed

# 正确
@dataclass
class Good:
    items: list = field(default_factory=list)
```

**陷阱 2：frozen dataclass 中无法在 __post_init__ 中赋值**

```python
# 错误：frozen=True 时 __post_init__ 中的赋值也会报错
@dataclass(frozen=True)
class Bad:
    x: int
    double: int = field(init=False)

    def __post_init__(self):
        # self.double = self.x * 2  → FrozenInstanceError
        # 必须使用 object.__setattr__
        object.__setattr__(self, 'double', self.x * 2)

b = Bad(5)
assert b.double == 10
```

**陷阱 3：继承中字段顺序冲突**

```python
@dataclass
class Parent:
    x: int = 0  # 有默认值

# 错误：子类无默认值字段不能排在父类有默认值字段之后
# @dataclass
# class Child(Parent):
#     y: int  # TypeError: non-default argument follows default argument

# 正确方案 1：给子类字段也加默认值
@dataclass
class Child1(Parent):
    y: int = 0

# 正确方案 2：给父类字段去掉默认值
@dataclass
class Parent2:
    x: int

@dataclass
class Child2(Parent2):
    y: int = 0
```

**陷阱 4：__eq__ 与继承**

```python
@dataclass
class Base:
    x: int

@dataclass
class Child(Base):
    y: int

b = Base(1)
c = Child(1, 2)
# b == c → False（类型不同，dataclass __eq__ 检查类型）
# 这与普通类继承的 __eq__ 行为一致，但容易被忽略
```

**陷阱 5：hash 与 eq 的配合**

```python
# 不设 frozen=True 但设 eq=True（默认）→ 不可哈希
@dataclass
class NotHashable:
    x: int
    # hash(NotHashable(1)) → TypeError

# 需要哈希时：
@dataclass(frozen=True)     # 方案 1：frozen 自动生成 __hash__
class Hashable1:
    x: int

@dataclass(eq=True, unsafe_hash=True)  # 方案 2：手动启用（有风险）
class Hashable2:
    x: int
    # 注意：如果实例被修改，哈希值会变化，导致 dict/set 行为异常
```

**最佳实践**：
- 可变默认值始终用 `field(default_factory=...)`
- 需要 `dict key`/`set` 元素时用 `frozen=True`
- `__post_init__` 中计算衍生字段时使用 `field(init=False)`
- 用 `InitVar` 传递初始化参数但不存储为字段
- 继承 `dataclass` 时注意字段顺序和默认值冲突
- 优先用 `dataclass` 替代手写 `__init__`/`__repr__`/`__eq__`
- 对简单不可变值对象用 `@dataclass(frozen=True, order=True)` 替代 `namedtuple`
