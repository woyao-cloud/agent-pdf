### 52.1.2 src layout vs flat layout

```text
# Flat layout（传统）
myproject/
├── mypackage/
├── tests/
└── setup.py

# src layout（推荐）
myproject/
├── src/
│   └── mypackage/
├── tests/
└── pyproject.toml
```

src layout 的优势：
1. **防止意外导入**：`pip install -e .` 前无法直接 `import mypackage`
2. **确保测试安装版本**：测试必须使用安装后的包
3. **避免 sys.path 问题**：不会因为项目根目录在 PATH 中而导入未安装的包

### 52.1.4 `__init__.py` 的作用

```python
# __init__.py 的三个作用：
# 1. 标记目录为 Python 包
# 2. 控制包级别的导入
# 3. 执行包的初始化代码

# mypackage/__init__.py

# 方式1：空文件（最简单）
# （文件为空）

# 方式2：重新导出公共 API
from .core import MyClass, main_function
from .models import User, Product

__all__ = ['MyClass', 'main_function', 'User', 'Product']

# 方式3：懒加载（避免循环导入）
def __getattr__(name):
    if name == 'MyClass':
        from .core import MyClass
        return MyClass
    raise AttributeError(f'module {__name__!r} has no attribute {name!r}')

# 方式4：包级常量
__version__ = '1.0.0'
__author__ = 'Author Name'
```

## 52.2 原理与机制

### 52.2.1 Python 的模块解析机制

```python
# import mypackage.api.routes 的解析过程：
# 1. 查找 mypackage：
#    - sys.modules 缓存检查
#    - sys.path 中的每个目录
#    - 查找 mypackage/__init__.py 或 mypackage.py
# 2. 查找 mypackage.api：
#    - mypackage 路径下的 api/__init__.py
# 3. 查找 mypackage.api.routes：
#    - mypackage/api/routes.py

# 模块搜索顺序：
# 1. sys.modules（已导入模块缓存）
# 2. 内置模块（sys.builtin_module_names）
# 3. sys.path 中的目录：
#    a. 脚本所在目录（'' 或脚本路径）
#    b. PYTHONPATH 环境变量
#    c. 安装路径的 site-packages
#    d. 标准库路径

# 查看搜索路径
import sys
print(sys.path)

# 相对导入 vs 绝对导入
# 绝对导入（推荐）
from mypackage.api import routes

# 相对导入（包内部使用）
from . import routes         # 同级模块
from .. import models        # 上级模块
from .auth import validate   # 同级子模块
```

### 52.2.2 循环导入的解决

```python
# 循环导入：A 导入 B，B 导入 A

# 解决方案1：重构代码，提取共享逻辑到第三个模块
# shared.py
def common_function():
    pass

# a.py
from .shared import common_function

# b.py
from .shared import common_function

# 解决方案2：延迟导入
# a.py
def function_a():
    from .b import function_b  # 函数内部导入
    return function_b()

# 解决方案3：使用 TYPE_CHECKING
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from .b import B  # 仅类型检查时导入

class A:
    def process(self, b: B) -> None:  # 注解可用
        ...

# 解决方案4：__init__.py 中的懒加载
# mypackage/__init__.py
def __getattr__(name):
    if name == 'A':
        from .a import A
        return A
    if name == 'B':
        from .b import B
        return B
```

### 52.3.2 插件式架构

```python
"""插件式架构：动态加载模块"""
import importlib
from pathlib import Path


class PluginManager:
    def __init__(self, plugin_dir='plugins'):
        self.plugin_dir = Path(plugin_dir)
        self.plugins = {}

    def discover(self):
        """发现并加载所有插件。"""
        for path in self.plugin_dir.glob('*.py'):
            if path.name.startswith('_'):
                continue
            module_name = f'plugins.{path.stem}'
            try:
                module = importlib.import_module(module_name)
                if hasattr(module, 'register'):
                    plugin = module.register()
                    self.plugins[plugin.name] = plugin
            except Exception as e:
                print(f"Failed to load plugin {path.name}: {e}")

    def get_plugin(self, name):
        return self.plugins.get(name)
```

## 52.4 示例代码

### 52.4.1 配置管理模块

```python
"""统一配置管理"""
import os
from pathlib import Path
from typing import Any


class Settings:
    """从环境变量和配置文件加载设置。"""

    def __init__(self):
        self._values: dict = {}

    def load_from_env(self, prefix='APP_'):
        """从环境变量加载（APP_ 前缀）。"""
        for key, value in os.environ.items():
            if key.startswith(prefix):
                name = key[len(prefix):].lower()
                self._values[name] = self._convert(value)

    def load_from_dict(self, data: dict):
        """从字典加载。"""
        self._values.update(data)

    def get(self, key: str, default: Any = None) -> Any:
        return self._values.get(key, default)

    def __getattr__(self, name: str) -> Any:
        if name.startswith('_'):
            raise AttributeError(name)
        return self._values.get(name)

    @staticmethod
    def _convert(value: str):
        if value.lower() in ('true', 'yes', '1'):
            return True
        if value.lower() in ('false', 'no', '0'):
            return False
        try:
            return int(value)
        except ValueError:
            try:
                return float(value)
            except ValueError:
                return value
```

## 52.5 常见陷阱与最佳实践

### 52.5.1 常见架构陷阱

| 陷阱 | 最佳实践 |
|------|---------|
| 循环导入 | 提取共享逻辑或使用延迟导入 |
| 上帝模块（一个文件包含所有逻辑） | 按职责拆分为子包 |
| 配置散落各处 | 统一配置管理（Settings 模式） |
| 硬编码路径 | 使用 pathlib 和配置 |
| 测试依赖生产配置 | 使用测试专用配置和 fixture |
