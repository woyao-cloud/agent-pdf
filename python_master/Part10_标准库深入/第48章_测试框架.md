## 48.1 概念与语法

### 48.1.1 unittest 框架

```python
import unittest

class TestStringMethods(unittest.TestCase):
    """字符串方法测试示例。"""

    def test_upper(self):
        self.assertEqual('hello'.upper(), 'HELLO')

    def test_isupper(self):
        self.assertTrue('HELLO'.isupper())
        self.assertFalse('hello'.isupper())

    def test_split(self):
        s = 'hello world'
        self.assertEqual(s.split(), ['hello', 'world'])
        with self.assertRaises(TypeError):
            s.split(2)  # split() 不接受整数参数

    def setUp(self):
        """每个测试方法前执行。"""
        self.test_data = [1, 2, 3]

    def tearDown(self):
        """每个测试方法后执行。"""
        del self.test_data

    @classmethod
    def setUpClass(cls):
        """所有测试前执行一次。"""
        cls.shared_resource = create_expensive_resource()

    @classmethod
    def tearDownClass(cls):
        """所有测试后执行一次。"""
        destroy_resource(cls.shared_resource)


if __name__ == '__main__':
    unittest.main()
```

**断言方法**：

| 方法 | 含义 |
|------|------|
| `assertEqual(a, b)` | a == b |
| `assertNotEqual(a, b)` | a != b |
| `assertTrue(x)` | bool(x) is True |
| `assertFalse(x)` | bool(x) is False |
| `assertIs(a, b)` | a is b |
| `assertIsNone(x)` | x is None |
| `assertIn(a, b)` | a in b |
| `assertIsInstance(a, b)` | isinstance(a, b) |
| `assertRaises(exc)` | 上下文管理器，断言抛出异常 |
| `assertAlmostEqual(a, b)` | round(a-b, 7) == 0 |
| `assertGreater(a, b)` | a > b |
| `assertRegex(text, regex)` | re.search(regex, text) |

### 48.1.3 mock 模块

```python
from unittest.mock import Mock, patch, MagicMock, call

# Mock 对象
mock = Mock()
mock.some_method('arg')
mock.some_method.assert_called_once_with('arg')
mock.some_method.return_value = 42
print(mock.some_method())  # 42

# MagicMock — 支持魔术方法的 Mock
mock = MagicMock()
len(mock)        # 调用 __len__，默认返回 0
mock[0]          # 调用 __getitem__
mock == 1        # 调用 __eq__

# 属性自动创建
mock = Mock()
mock.nested.deep.method()  # 自动创建中间 Mock 对象

# patch — 替换目标对象
import requests

class MyService:
    def fetch_data(self, url):
        response = requests.get(url)
        return response.json()

# 方式1：装饰器
@patch('requests.get')
def test_fetch_data(self, mock_get):
    mock_get.return_value.json.return_value = {'key': 'value'}
    service = MyService()
    result = service.fetch_data('http://example.com')
    self.assertEqual(result, {'key': 'value'})
    mock_get.assert_called_once_with('http://example.com')

# 方式2：上下文管理器
def test_with_context(self):
    with patch('requests.get') as mock_get:
        mock_get.return_value.json.return_value = {'key': 'value'}
        service = MyService()
        result = service.fetch_data('http://example.com')

# 方式3：手动 start/stop
patcher = patch('requests.get')
mock_get = patcher.start()
# ... 测试 ...
patcher.stop()

# side_effect — 模拟副作用
mock = Mock()
mock.side_effect = [1, 2, 3]  # 依次返回
mock()  # 1
mock()  # 2
mock()  # 3

mock.side_effect = ValueError('error')  # 抛出异常
mock()  # raises ValueError

# call 对象 — 检查调用参数
mock = Mock()
mock(1, 2, key='value')
print(mock.call_args)         # call(1, 2, key='value')
print(mock.call_args_list)    # [call(1, 2, key='value')]
```

## 48.2 原理与机制

### 48.2.1 unittest 的测试加载与执行体系

```python
"""
unittest 的核心类关系：

TestLoader
  ├── loadTestsFromTestCase(cls) — 从 TestCase 类加载
  ├── loadTestsFromModule(module) — 从模块加载
  ├── discover() — 自动发现测试
  └── 返回 TestSuite

TestSuite
  ├── 包含多个 TestCase 或 TestSuite
  ├── run(result) — 执行所有测试
  └── 支持嵌套组合

TestRunner
  ├── run(suite) — 执行测试套件
  └── 返回 TestResult

TestResult
  ├── 记录测试结果（成功、失败、错误）
  ├── failures — 断言失败列表
  ├── errors — 异常错误列表
  └── skipped — 跳过测试列表

执行流程：
1. TestLoader.loadTestsFromTestCase() 扫描以 test_ 开头的方法
2. 创建 TestSuite 包含所有测试方法
3. TestRunner.run(suite) 依次执行每个测试
4. 每个测试执行：setUp → test_method → tearDown
5. 断言失败 → 记录到 TestResult.failures
6. 异常 → 记录到 TestResult.errors
7. 生成报告
"""
```

### 48.2.2 Mock 的实现原理

```python
from unittest.mock import Mock, MagicMock

# Mock 的核心实现机制：
# 1. __getattr__ — 自动创建属性 Mock
# 2. __call__ — 使 Mock 可调用，记录调用
# 3. assert_called_with — 检查调用记录

"""
class Mock(Base):
    def __getattr__(self, name):
        # 每次访问不存在的属性，自动创建并缓存一个新的 Mock
        result = self._get_child_mock(name=name)
        setattr(self, name, result)
        return result

    def __call__(self, *args, **kwargs):
        # 记录调用
        self.call_args = Call(args, kwargs)
        self.call_count += 1
        self.call_args_list.append(self.call_args)

        # 返回 return_value 或触发 side_effect
        if self.side_effect is not None:
            return self.side_effect(*args, **kwargs)
        return self.return_value

    def assert_called_once_with(self, *args, **kwargs):
        # 检查调用记录
        if self.call_count != 1:
            raise AssertionError(...)
        if self.call_args != Call(args, kwargs):
            raise AssertionError(...)
"""

# MagicMock vs Mock 的区别
# MagicMock 预定义了所有魔术方法的 Mock 实现
# Mock 访问 __len__ 等魔术方法会返回 Mock 对象（不可用于内置函数）
mock = Mock()
print(len(mock))   # TypeError: object of type Mock has no len()

magic = MagicMock()
print(len(magic))  # 0 — __len__ 返回 0
print(hash(magic)) # 有返回值 — __hash__ 已定义
```

### 48.2.3 patch 的替换机制

```python
from unittest.mock import patch

# patch 的原理：
# 1. 导入目标模块
# 2. 使用 setattr() 替换目标属性为 Mock
# 3. 在 with 块结束时使用 setattr() 恢复原始值

"""
@patch('module.ClassName')
等效于：
original = module.ClassName
module.ClassName = Mock(...)
# ... 测试 ...
module.ClassName = original
"""

# patch 位置的重要性
# 错误：patch 定义模块的位置
@patch('requests.get')  # 如果 requests 在 my_module 中导入
# 正确：patch 使用模块的位置
@patch('my_module.requests.get')  # 替换 my_module 中的 requests.get 引用

# 原因：Python 的 import 创建的是绑定关系
# my_module.py 中: from requests import get
# 这时 my_module.get 和 requests.get 是不同的引用
# 必须替换 my_module.get 而非 requests.get
```

## 48.3 使用场景

### 48.3.1 Mock 外部依赖

```python
"""Mock 数据库层进行单元测试"""
import unittest
from unittest.mock import patch, Mock


class UserRepository:
    def __init__(self, db_connection):
        self.db = db_connection

    def get_user(self, user_id):
        cursor = self.db.cursor()
        cursor.execute('SELECT * FROM users WHERE id = %s', (user_id,))
        row = cursor.fetchone()
        if row is None:
            raise ValueError(f'User {user_id} not found')
        return {'id': row[0], 'name': row[1]}

    def create_user(self, name, email):
        cursor = self.db.cursor()
        cursor.execute(
            'INSERT INTO users (name, email) VALUES (%s, %s)',
            (name, email)
        )
        self.db.commit()
        return cursor.lastrowid


class TestUserRepository(unittest.TestCase):
    def setUp(self):
        self.mock_db = Mock()
        self.repo = UserRepository(self.mock_db)

    def test_get_user_success(self):
        mock_cursor = Mock()
        mock_cursor.fetchone.return_value = (1, 'Alice', 'alice@example.com')
        self.mock_db.cursor.return_value = mock_cursor

        user = self.repo.get_user(1)
        self.assertEqual(user['id'], 1)
        self.assertEqual(user['name'], 'Alice')

    def test_get_user_not_found(self):
        mock_cursor = Mock()
        mock_cursor.fetchone.return_value = None
        self.mock_db.cursor.return_value = mock_cursor

        with self.assertRaises(ValueError):
            self.repo.get_user(999)

    def test_create_user(self):
        mock_cursor = Mock()
        mock_cursor.lastrowid = 42
        self.mock_db.cursor.return_value = mock_cursor

        user_id = self.repo.create_user('Bob', 'bob@example.com')
        self.assertEqual(user_id, 42)
        self.mock_db.commit.assert_called_once()
```

## 48.4 示例代码

### 48.4.1 完整测试套件

```python
"""完整的测试套件示例：测试一个计算器类"""
import unittest
from unittest.mock import patch


class Calculator:
    """简单计算器。"""

    def add(self, a, b):
        return a + b

    def divide(self, a, b):
        if b == 0:
            raise ValueError('Cannot divide by zero')
        return a / b

    def sqrt(self, a):
        if a < 0:
            raise ValueError('Cannot take sqrt of negative number')
        return a ** 0.5

    def log(self, a, base=10):
        import math
        if a <= 0:
            raise ValueError('Cannot take log of non-positive number')
        return math.log(a, base)


class TestCalculator(unittest.TestCase):
    def setUp(self):
        self.calc = Calculator()

    # --- 加法测试 ---
    def test_add_positive(self):
        self.assertEqual(self.calc.add(2, 3), 5)

    def test_add_negative(self):
        self.assertEqual(self.calc.add(-1, -2), -3)

    def test_add_zero(self):
        self.assertEqual(self.calc.add(0, 0), 0)

    def test_add_float(self):
        self.assertAlmostEqual(self.calc.add(0.1, 0.2), 0.3, places=7)

    # --- 除法测试 ---
    def test_divide_normal(self):
        self.assertEqual(self.calc.divide(6, 3), 2.0)

    def test_divide_by_zero(self):
        with self.assertRaises(ValueError) as ctx:
            self.calc.divide(1, 0)
        self.assertIn('Cannot divide by zero', str(ctx.exception))

    # --- 平方根测试 ---
    def test_sqrt_normal(self):
        self.assertAlmostEqual(self.calc.sqrt(4), 2.0)

    def test_sqrt_negative(self):
        with self.assertRaises(ValueError):
            self.calc.sqrt(-1)

    # --- 对数测试 ---
    @patch('math.log')
    def test_log_with_mock(self, mock_log):
        mock_log.return_value = 1.0
        result = self.calc.log(10)
        self.assertEqual(result, 1.0)
        mock_log.assert_called_once_with(10, 10)


# 运行所有测试
if __name__ == '__main__':
    unittest.main(verbosity=2)
```

### 48.4.2 TDD 工作流示例

```python
"""
TDD 工作流演示：开发一个 Stack 类

步骤1：RED — 写失败测试
步骤2：GREEN — 写最简实现使测试通过
步骤3：REFACTOR — 重构代码
"""

import unittest


# 步骤1: 先写测试（此时 Stack 类不存在，测试会失败）
class TestStack(unittest.TestCase):
    def test_push_and_pop(self):
        stack = Stack()
        stack.push(1)
        stack.push(2)
        self.assertEqual(stack.pop(), 2)
        self.assertEqual(stack.pop(), 1)

    def test_is_empty(self):
        stack = Stack()
        self.assertTrue(stack.is_empty())
        stack.push(1)
        self.assertFalse(stack.is_empty())

    def test_pop_empty(self):
        stack = Stack()
        with self.assertRaises(IndexError):
            stack.pop()

    def test_peek(self):
        stack = Stack()
        stack.push(1)
        stack.push(2)
        self.assertEqual(stack.peek(), 2)
        self.assertEqual(stack.size(), 2)  # peek 不弹出

    def test_size(self):
        stack = Stack()
        self.assertEqual(stack.size(), 0)
        stack.push(1)
        self.assertEqual(stack.size(), 1)


# 步骤2: 最简实现（使所有测试通过）
class Stack:
    def __init__(self):
        self._items = []

    def push(self, item):
        self._items.append(item)

    def pop(self):
        if self.is_empty():
            raise IndexError('pop from empty stack')
        return self._items.pop()

    def peek(self):
        if self.is_empty():
            raise IndexError('peek at empty stack')
        return self._items[-1]

    def is_empty(self):
        return len(self._items) == 0

    def size(self):
        return len(self._items)


# 步骤3: 重构（添加 __len__、迭代支持等）
# 保持测试通过的同时改善代码
```

## 48.5 常见陷阱与最佳实践

### 48.5.1 测试隔离

```python
import unittest

# 陷阱：测试之间共享可变状态
shared_list = []  # 模块级变量

class TestBad(unittest.TestCase):
    def test_a(self):
        shared_list.append(1)
        self.assertEqual(len(shared_list), 1)  # 可能失败

    def test_b(self):
        shared_list.append(2)
        self.assertEqual(len(shared_list), 1)  # 取决于执行顺序

# 最佳实践：每个测试独立创建数据
class TestGood(unittest.TestCase):
    def setUp(self):
        self.data = []  # 每个测试方法独立的实例变量

    def test_a(self):
        self.data.append(1)
        self.assertEqual(len(self.data), 1)

    def test_b(self):
        self.data.append(2)
        self.assertEqual(len(self.data), 1)  # 总是正确
```

### 48.5.2 Mock 滥用

```python
from unittest.mock import Mock

# 陷阱：过度 Mock 导致测试与实现强耦合
class BadTest(unittest.TestCase):
    @patch('module.Database.query')
    @patch('module.Database.connect')
    @patch('module.Database.close')
    def test_service(self, mock_close, mock_connect, mock_query):
        # Mock 了太多内部实现细节
        # 重构 Database 类时所有测试都会失败
        pass

# 最佳实践：只 Mock 外部边界
class GoodTest(unittest.TestCase):
    @patch('module.Database')  # 只 Mock 整个外部依赖
    def test_service(self, MockDB):
        db = MockDB.return_value
        db.query.return_value = [{'id': 1}]
        # 测试业务逻辑，不测试 Database 内部
```

### 48.5.3 测试金字塔

```python
"""
测试金字塔（Test Pyramid）：

          /\
         /  \         E2E 测试（少量）
        / E2E\        - 慢、脆弱、昂贵
       /------\
      /        \      集成测试（适量）
     /Integration\    - 测试模块交互
    /--------------\
   /                \  单元测试（大量）
  /   Unit Tests     \ - 快速、稳定、便宜
 /--------------------\

最佳实践：
- 70% 单元测试
- 20% 集成测试
- 10% E2E 测试
"""
```

### 48.5.4 总结

| 陷阱 | 最佳实践 |
|------|---------|
| 测试之间共享状态 | 使用 setUp 创建独立数据 |
| 过度 Mock 内部实现 | 只 Mock 外部边界 |
| 测试代码与实现耦合 | 测试行为而非实现 |
| 忽略边界条件 | 测试空值、极端值、异常 |
| 缺少负面测试 | 测试错误路径和异常 |
| 测试运行过慢 | Mock 外部 I/O，使用测试金字塔 |
| 不测试测试本身 | 交叉验证、变异测试 |
