# 第28章 自定义异常体系

## 1. 概念与语法

### 1.1 为什么需要自定义异常体系

Python 内置异常是面向通用场景设计的——`ValueError`、`TypeError`、`RuntimeError` 等无法表达领域特定的语义。自定义异常体系解决以下问题：

| 问题 | 内置异常 | 自定义异常 |
|------|---------|-----------|
| 语义清晰度 | `raise ValueError("user not found")` | `raise UserNotFoundError(user_id=42)` |
| 精确捕获 | `except ValueError` 捕获所有值错误 | `except UserNotFoundError` 精确捕获 |
| 错误分类 | 无层级关系 | `class UserNotFoundError(AppNotFoundError)` |
| 错误信息 | 仅字符串 | 结构化属性（error_code、context 等） |
| 框架集成 | 无 | HTTP 状态码映射、日志分级、i18n |

### 1.2 自定义异常的基本语法

```python
# 最简形式
class AppError(Exception):
    """应用程序异常基类"""
    pass

# 带自定义属性
class DatabaseError(AppError):
    """数据库操作异常"""
    def __init__(self, message, host=None, query=None):
        self.host = host
        self.query = query
        super().__init__(message)

# 带错误码
class BusinessError(AppError):
    """业务逻辑异常"""
    error_code = "BIZ_000"

    def __init__(self, message, error_code=None, context=None):
        self.error_code = error_code or self.error_code
        self.context = context or {}
        super().__init__(message)
```

### 1.3 异常与错误码的对比

```python
# 方式1：错误码（C/Go 风格）
def divide_error_code(a, b):
    if b == 0:
        return None, "ERR_DIV_ZERO"
    return a / b, None

result, err = divide_error_code(10, 0)
if err:
    handle_error(err)

# 方式2：异常（Python 风格）
class DivisionByZeroError(ArithmeticError):
    pass

def divide_exception(a, b):
    if b == 0:
        raise DivisionByZeroError("division by zero")
    return a / b

try:
    result = divide_exception(10, 0)
except DivisionByZeroError:
    handle_error()
```

**对比分析**：

| 维度 | 错误码 | 异常 |
|------|-------|------|
| 语义 | 值需要显式检查 | 控制流自动跳转 |
| 强制性 | 调用方可能忽略 | 不捕获则传播 |
| 组合性 | 需要逐层传递 | 自动沿调用栈传播 |
| 性能 | 正常路径零开销 | 正常路径几乎零开销，异常路径开销大 |
| 可读性 | 每层都需要 if 检查 | 正常路径清晰，错误处理集中 |
| 适用场景 | 简单函数、性能关键路径 | 业务逻辑、框架设计 |

## 2. 原理与机制

### 2.1 异常类的 C 层结构

自定义异常类在 CPython 中仍然是 `PyTypeObject` 实例，但继承了 `PyExc_Exception` 的类型对象：

```c
// Objects/exceptions.c
// 内置异常的初始化过程
// PyExc_Exception 是所有非系统级异常的基类
PyTypeObject *PyExc_Exception = &PyExc_ExceptionType;

// 当用户定义 class AppError(Exception): pass
// CPython 执行以下步骤：
// 1. 调用 type.__new__ 创建新的 PyTypeObject
// 2. 设置 tp_base = &PyExc_ExceptionType
// 3. 继承 BaseException 的 tp_init, tp_repr 等方法
// 4. tp_basicsize = sizeof(PyBaseExceptionObject)
//    （所有 Exception 子类共享相同的实例结构）
```

**异常实例的内存布局**（所有 `Exception` 子类共享）：

```c
// Include/cpython/exceptions.h
typedef struct {
    PyObject_HEAD
    PyObject *dict;          // __dict__：存储自定义属性
    PyObject *args;          // 构造参数元组
    PyObject *notes;         // Python 3.11+ 的 __notes__
    PyObject *message;       // Python 3.11+ 的 __message__
    PyObject *traceback;     // __traceback__
    PyObject *cause;         // __cause__（显式异常链）
    PyObject *context;       // __context__（隐式异常链）
    char suppress_context;   // __suppress_context__
} PyBaseExceptionObject;
```

自定义属性（如 `error_code`、`context`）通过 `__dict__` 字段存储——这就是为什么可以在异常实例上动态添加属性。

### 2.2 `__init__` 与 `__new__` 的交互

当创建自定义异常时，CPython 的调用链如下：

```
AppError("message", code=500)
    │
    ▼
type.__call__(AppError, "message", code=500)
    │
    ├── type.__new__(AppError, "message", code=500)
    │       │
    │       ▼
    │   BaseException.__new__()              // C 层：BaseException_new()
    │       ├── 创建 PyBaseExceptionObject
    │       ├── 初始化 dict, args, notes 为 NULL
    │       └── 返回实例
    │
    └── AppError.__init__(instance, "message", code=500)
            │
            ▼
        super().__init__("message")          // 调用 BaseException.__init__()
            │
            ▼
        BaseException.__init__()             // C 层：BaseException_init()
            ├── 设置 self.args = ("message",)
            └── 不影响 self.dict 或其他属性
```

**关键洞察**：`BaseException.__init__()` 只设置 `self.args`，所有自定义属性必须在子类 `__init__` 中显式设置。

### 2.3 `__str__` 与 `__repr__` 的 C 层实现

```c
// Objects/exceptions.c

// BaseException_str —— str(e) 的实现
static PyObject *
BaseException_str(PyBaseExceptionObject *self)
{
    // 如果 self.args 只有一个元素，返回该元素的 str
    // 如果 self.args 为空，返回空字符串
    // 否则返回 str(self.args)
    if (Py_SIZE(self->args) == 1)
        return PyObject_Str(((PyTupleObject *)self->args)->ob_item[0]);
    else
        return PyObject_Str(self->args);
}

// BaseException_repr —— repr(e) 的实现
static PyObject *
BaseException_repr(PyBaseExceptionObject *self)
{
    // 返回 "ClassName(args...)"
    // 例如：ValueError('invalid literal')
}
```

这意味着自定义异常如果不在 `args` 中传递信息，`str(e)` 可能返回空字符串：

```python
class BadError(Exception):
    def __init__(self):
        self.message = "something went wrong"
        # 忘记调用 super().__init__！

e = BadError()
print(str(e))     # 输出: ""  （空字符串！）
print(repr(e))    # 输出: "BadError()"
```

### 2.4 异常匹配的 MRO 机制

`except` 子句使用 `isinstance()` 语义匹配异常，而 `isinstance()` 依赖 MRO（方法解析顺序）：

```c
// Python/ceval.c (CHECK_EXC_MATCH)
// 等价于 PyObject_IsInstance(exc, exc_type)
// 而 PyObject_IsInstance 遍历 exc 的 MRO 检查是否匹配
```

```python
class AppError(Exception): pass
class DomainError(AppError): pass
class UserError(DomainError): pass
class UserNotFoundError(UserError): pass

# MRO 决定匹配顺序
print(UserNotFoundError.__mro__)
# (<class 'UserNotFoundError'>,
#  <class 'UserError'>,
#  <class 'DomainError'>,
#  <class 'AppError'>,
#  <class 'Exception'>,
#  <class 'BaseException'>,
#  <class 'object'>)

try:
    raise UserNotFoundError(user_id=42)
except UserNotFoundError:      # 匹配 ✓
    print("caught as UserNotFoundError")
except UserError:              # 不会执行（已被前面的 except 捕获）
    print("caught as UserError")
```

### 2.5 `__reduce__` 与异常的序列化

异常对象需要支持 pickle 序列化，`__reduce__` 控制序列化行为：

```python
class AppError(Exception):
    def __init__(self, message, error_code=None):
        self.message = message
        self.error_code = error_code or "UNKNOWN"
        super().__init__(message)

    def __reduce__(self):
        # 序列化时恢复所需参数
        return (self.__class__, (self.message, self.error_code))

import pickle
e = AppError("test", error_code="ERR_001")
data = pickle.dumps(e)
restored = pickle.loads(data)
assert str(restored) == "test"
assert restored.error_code == "ERR_001"
```

CPython 内部，`BaseException` 的 pickle 支持依赖 `__reduce__` 和 `args` 属性：

```c
// Objects/exceptions.c
// BaseException_reduce 的实现
// 返回 (type(exc), exc.args) 用于重建异常
// 如果有 __dict__，还包括 __dict__ 的内容
```

### 2.6 异常的 `__notes__` 机制（Python 3.11+）

Python 3.11 引入了 `BaseException.add_note()` 和 `__notes__` 属性：

```c
// Objects/exceptions.c (Python 3.11+)
// BaseException_add_note 的实现
static PyObject *
BaseException_add_note(PyBaseExceptionObject *self, PyObject *note)
{
    // 1. 检查 note 是否为 str 类型
    // 2. 如果 self->notes 为 NULL，创建空列表
    // 3. 将 note 追加到 self->notes 列表
    // 4. 返回 Py_None

    if (self->notes == NULL) {
        self->notes = PyList_New(0);
        if (self->notes == NULL)
            return NULL;
    }
    return PyList_Append(self->notes, note);
}
```

### 3.3 重试与降级策略

```python
class RetryableError(Exception):
    """可重试的异常——框架自动重试"""
    max_retries = 3
    retry_delay = 1.0

class TransientError(RetryableError):
    """临时性错误（网络抖动等）"""
    max_retries = 3

class RateLimitError(RetryableError):
    """速率限制——需要等待后重试"""
    max_retries = 5
    retry_delay = 2.0

def retry_on_error(func):
    """自动重试装饰器"""
    import time
    def wrapper(*args, **kwargs):
        last_error = None
        for attempt in range(3):
            try:
                return func(*args, **kwargs)
            except RetryableError as e:
                last_error = e
                time.sleep(e.retry_delay * (attempt + 1))
        raise last_error
    return wrapper
```

## 4. 示例代码

### 4.1 完整的异常体系设计

```python
#!/usr/bin/env python3
"""完整的自定义异常体系设计——电商领域"""

import traceback
from datetime import datetime


# ============================================================
# 基础设施层异常
# ============================================================

class AppError(Exception):
    """应用程序异常基类

    所有自定义异常的根。提供：
    - 结构化错误信息
    - 错误码
    - 时间戳
    - 上下文信息
    - 异常链支持
    """

    # 类级别默认值
    error_code = "APP_ERROR"
    http_status = 500
    user_message = "An unexpected error occurred"

    def __init__(self, message=None, error_code=None, context=None,
                 user_message=None, cause=None):
        # 调用 super().__init__() 确保 self.args 被正确设置
        self.message = message or self.user_message
        self.error_code = error_code or self.error_code
        self.context = context or {}
        self.user_message = user_message or self.user_message
        self.timestamp = datetime.now()

        # 初始化 BaseException 的 args
        super().__init__(self.message)

        # 异常链
        if cause is not None:
            self.__cause__ = cause

    def __str__(self):
        parts = [f"[{self.error_code}]"]
        if self.message:
            parts.append(str(self.message))
        if self.context:
            parts.append(f"context={self.context}")
        return " ".join(parts)

    def __repr__(self):
        return (f"{self.__class__.__name__}("
                f"message={self.message!r}, "
                f"error_code={self.error_code!r}, "
                f"http_status={self.http_status})")

    def to_dict(self):
        """序列化为字典（用于 API 响应）"""
        result = {
            "error_code": self.error_code,
            "message": self.user_message,
            "timestamp": self.timestamp.isoformat(),
        }
        if self.context:
            result["context"] = self.context
        if self.__cause__:
            result["root_cause"] = str(self.__cause__)
        return result

    def __reduce__(self):
        """支持 pickle 序列化"""
        return (self.__class__, (
            self.message,
            self.error_code,
            self.context,
            self.user_message,
        ))

class ExceptionRegistry:
    """异常注册表——集中管理异常类"""

    _registry = {}

    @classmethod
    def register(cls, exception_class):
        """注册异常类"""
        cls._registry[exception_class.error_code] = exception_class
        return exception_class

    @classmethod
    def get(cls, error_code):
        """通过错误码查找异常类"""
        return cls._registry.get(error_code)

    @classmethod
    def all_error_codes(cls):
        """返回所有已注册的错误码"""
        return dict(cls._registry)


# 装饰器方式注册异常

def demonstrate_exception_hierarchy():
    """演示异常继承与匹配"""
    print("=== 异常继承层级 ===")

    # MRO 检查
    print(f"UserNotFoundError MRO: {[c.__name__ for c in UserNotFoundError.__mro__]}")

    # isinstance 匹配
    exc = UserNotFoundError(user_id=42)
    print(f"\nisinstance(exc, UserNotFoundError): {isinstance(exc, UserNotFoundError)}")
    print(f"isinstance(exc, UserError): {isinstance(exc, UserError)}")
    print(f"isinstance(exc, DomainError): {isinstance(exc, DomainError)}")
    print(f"isinstance(exc, AppError): {isinstance(exc, AppError)}")
    print(f"isinstance(exc, Exception): {isinstance(exc, Exception)}")

    # except 匹配顺序
    print("\n=== except 匹配顺序 ===")
    try:
        raise UserNotFoundError(user_id=42)
    except UserNotFoundError as e:
        print(f"最精确匹配: {type(e).__name__}")
    except UserError:
        print("不会执行")
    except DomainError:
        print("不会执行")

    # 精确匹配 vs 宽泛匹配
    print("\n=== 宽泛匹配 ===")
    try:
        raise UserNotFoundError(user_id=42)
    except DomainError as e:
        print(f"宽泛匹配: {type(e).__name__} caught as DomainError")
```

```python
# ============================================================
# 异常体系
# ============================================================

class AppError(Exception):
    """应用异常基类"""
    error_code: str = "APP_ERROR"
    http_status: int = 500
    user_message: str = "Internal server error"

    def __init__(self, message=None, **kwargs):
        self.message = message or self.user_message
        self.context = kwargs
        super().__init__(self.message)

    def to_dict(self):
        return {
            "error_code": self.error_code,
            "message": self.user_message,
            "detail": str(self.message),
            **self.context,
        }


class NotFoundError(AppError):

    user_message = "Resource not found"

    http_status = 400
    user_message = "Validation failed"

    def to_dict(self):
        result = super().to_dict()
        result["errors"] = self.errors
        return result

@dataclass
class Response:
    """模拟 HTTP 响应"""
    status_code: int
    body: Dict[str, Any]

class ExceptionMiddleware:
    """异常处理中间件——将异常转换为 HTTP 响应"""

    def __init__(self):
        # 异常类到 HTTP 响应的映射
        self._handlers: Dict[Type[Exception], Any] = {}
        self._register_default_handlers()

    def _register_default_handlers(self):
        """注册默认处理器"""
        self._handlers[AppError] = self._handle_app_error
        self._handlers[ValidationError] = self._handle_validation_error
        self._handlers[AuthenticationError] = self._handle_auth_error
        self._handlers[NotFoundError] = self._handle_not_found
        self._handlers[RateLimitError] = self._handle_rate_limit
        self._handlers[DatabaseError] = self._handle_database_error

    def _handle_app_error(self, exc: AppError) -> Response:
        """处理应用异常"""
        logger.error(f"AppError: {exc}", exc_info=True)
        return Response(
            status_code=exc.http_status,
            body=exc.to_dict(),
            headers={"X-Error-Code": exc.error_code}
        )

    def _handle_validation_error(self, exc: ValidationError) -> Response:
        """处理验证异常"""
        logger.warning(f"ValidationError: {exc}")
        return Response(
            status_code=exc.http_status,
            body=exc.to_dict(),
            headers={"X-Error-Code": exc.error_code}
        )

    def _handle_auth_error(self, exc: AuthenticationError) -> Response:
        """处理认证异常"""
        logger.warning(f"AuthError: {exc}")
        return Response(
            status_code=exc.http_status,
            body=exc.to_dict(),
            headers={
                "X-Error-Code": exc.error_code,
                "WWW-Authenticate": 'Bearer realm="api"',
            }
        )

    def _handle_not_found(self, exc: NotFoundError) -> Response:
        """处理 404"""
        logger.info(f"NotFound: {exc}")
        return Response(
            status_code=404,
            body=exc.to_dict(),
        )

    def _handle_rate_limit(self, exc: RateLimitError) -> Response:
        """处理限流"""
        logger.warning(f"RateLimit: {exc}")
        return Response(
            status_code=429,
            body=exc.to_dict(),
            headers={"Retry-After": "60"}
        )

    def _handle_database_error(self, exc: DatabaseError) -> Response:
        """处理数据库异常"""
        logger.error(f"DatabaseError: {exc}", exc_info=True)
        return Response(
            status_code=503,
            body=exc.to_dict(),
            headers={"Retry-After": "30"}
        )

    def handle(self, exc: Exception) -> Response:
        """根据异常类型路由到对应处理器"""
        # 按 MRO 顺序查找最精确的处理器
        for exc_class in type(exc).__mro__:
            handler = self._handlers.get(exc_class)
            if handler is not None:
                return handler(exc)

        # 未知异常——返回 500
        logger.critical(f"Unhandled exception: {exc}", exc_info=True)
        return Response(
            status_code=500,
            body={
                "error_code": "INTERNAL_ERROR",
                "message": "Internal server error",
            }
        )

def translate_exceptions(*mappings):
    """异常转换装饰器——将底层异常转换为领域异常

    用法：
        @translate_exceptions((sqlite3.OperationalError, DatabaseError))
        def query_db():
            ...
    """
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            try:
                return func(*args, **kwargs)
            except tuple(m[0] for m in mappings) as e:
                for source_exc, target_exc in mappings:
                    if isinstance(e, source_exc):
                        raise target_exc(str(e)) from e
                raise  # 不应该到达这里
        return wrapper
    return decorator


def retry_on(exception_types, max_retries=3, delay=1.0, backoff=2.0):
    """重试装饰器

    Args:
        exception_types: 需要重试的异常类型元组
        max_retries: 最大重试次数
        delay: 初始延迟（秒）
        backoff: 延迟倍数
    """
    import time

    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exception = None
            current_delay = delay
            for attempt in range(max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except exception_types as e:
                    last_exception = e
                    if attempt < max_retries:
                        logger.info(
                            f"Retry {attempt + 1}/{max_retries} "
                            f"after {current_delay:.1f}s: {e}"
                        )
                        time.sleep(current_delay)
                        current_delay *= backoff
                    else:
                        logger.error(f"All {max_retries} retries exhausted: {e}")
            raise last_exception
        return wrapper
    return decorator
```

```python
#!/usr/bin/env python3
"""traceback 定制与异常诊断——C 源码级分析"""

import sys
import traceback
import linecache
import types


# ============================================================
# traceback 对象的内部结构
# ============================================================

class TracebackAnalyzer:
    """分析 traceback 对象——对应 CPython PyTracebackObject 结构

    PyTracebackObject 结构（Include/cpython/frame.h）：
    typedef struct _traceback {
        PyObject_HEAD
        PyTracebackObject *tb_next;     // 上一级 traceback（调用者）
        PyFrameObject *tb_frame;         // 栈帧对象
        int tb_lasti;                    // 最后执行的字节码偏移
        int tb_lineno;                    // 源码行号
    } PyTracebackObject;
    """

    @staticmethod
    def analyze_traceback(exc):
        """深度分析异常的 traceback"""
        tb = exc.__traceback__
        if tb is None:
            print("No traceback attached to this exception")
            return

        print(f"=== Traceback 分析: {type(exc).__name__}: {exc} ===\n")

        # 遍历 traceback 链
        depth = 0
        current = tb
        while current is not None:
            frame = current.tb_frame
            code = frame.f_code

            print(f"--- 帧 {depth} ---")
            print(f"  文件: {code.co_filename}")
            print(f"  函数: {code.co_name}")
            print(f"  行号: {current.tb_lineno}")
            print(f"  字节码偏移: {current.tb_lasti}")

            # 帧的局部变量（可能敏感，生产环境慎用）
            print(f"  局部变量: {list(frame.f_locals.keys())}")

            # 源码行
            line = linecache.getline(code.co_filename, current.tb_lineno)
            print(f"  源码: {line.strip() if line else '<unavailable>'}")

            # 字节码信息
            print(f"  代码对象名称: {code.co_name}")
            print(f"  代码对象参数: {code.co_varnames[:code.co_argcount]}")

            current = current.tb_next
            depth += 1

        print(f"\n总帧数: {depth}")

# 自定义 traceback 格式化
# ============================================================

class CompactTracebackFormatter:
    """紧凑的 traceback 格式化器"""

    @staticmethod
    def format(exc, limit=None):
        """格式化异常信息为紧凑形式"""
        lines = []

        # 格式化异常链
        current = exc
        depth = 0
        while current is not None:
            if depth > 0:
                lines.append("")
                cause_type = "caused by" if current.__cause__ is not None else "context"
                lines.append(f"The above exception was the {cause_type} of:")

            # 格式化 traceback
            tb = current.__traceback__
            if tb is not None:
                frames = traceback.extract_tb(tb, limit=limit)
                for frame in frames:
                    lines.append(
                        f"  {frame.filename}:{frame.lineno} "
                        f"in {frame.name}"
                    )

            # 格式化异常类型和消息
            lines.append(f"{type(current).__name__}: {current}")

            # 处理 notes (Python 3.11+)
            if hasattr(current, '__notes__') and current.__notes__:
                for note in current.__notes__:
                    lines.append(f"  Note: {note}")

            # 沿异常链继续
            current = current.__cause__ or (
                current.__context__
                if not current.__suppress_context__
                else None
            )
            depth += 1

        return "\n".join(lines)


class JsonTracebackFormatter:
    """JSON 格式的 traceback 格式化器——用于日志系统"""

    @staticmethod
    def format(exc):
        """格式化异常信息为 JSON 兼容的字典"""
        result = {
            "type": type(exc).__name__,
            "message": str(exc),
            "traceback": [],
        }

        tb = exc.__traceback__
        if tb is not None:
            for frame in traceback.extract_tb(tb):
                result["traceback"].append({
                    "file": frame.filename,
                    "line": frame.lineno,
                    "function": frame.name,
                    "code": frame.line,
                })

        # 处理异常链
        if exc.__cause__:
            result["cause"] = JsonTracebackFormatter.format(exc.__cause__)
        elif exc.__context__ and not exc.__suppress_context__:
            result["context"] = JsonTracebackFormatter.format(exc.__context__)

        return result
```

### 5.2 陷阱：异常类属性与实例属性混淆

```python

e = AmbiguousError("test")
print(e.error_code)                # "OVERRIDDEN"（实例属性覆盖了类属性）

# GOOD——使用不同的命名或明确的意图
class ClearError(Exception):
    default_code = "CLEAR_001"      # 类属性用不同名称

    def __init__(self, message, error_code=None):
        self.error_code = error_code or self.default_code
        super().__init__(message)
```

### 5.3 陷阱：异常继承过深导致匹配混乱

```python
# BAD——继承层级过深
class AppError(Exception): pass
class DomainError(AppError): pass
class UserError(DomainError): pass
class UserAuthError(UserError): pass
class UserPasswordError(UserAuthError): pass
class UserPasswordTooShortError(UserPasswordError): pass
# 6 层继承！except 匹配变得复杂

# GOOD——扁平化异常体系
class AppError(Exception): pass
class DomainError(AppError): pass
class UserNotFoundError(DomainError): pass
class UserAlreadyExistsError(DomainError): pass
class UserPasswordError(DomainError): pass
class UserPermissionError(DomainError): pass
# 最多 3 层，except 匹配简单
```

### 5.4 陷阱：过度使用异常链导致 traceback 冗长

```python
# BAD——每一层都包装异常
try:
    try:
        try:
            try:
                raise OSError("connection refused")
            except OSError as e:
                raise DatabaseError(str(e)) from e        # 链1
        except DatabaseError as e:
            raise RepositoryError(str(e)) from e          # 链2
    except RepositoryError as e:
        raise ServiceError(str(e)) from e                 # 链3
except ServiceError as e:
    raise APIError(str(e)) from e                         # 链4
# traceback 有 4 层异常链，难以阅读

# GOOD——只在抽象边界转换异常
try:
    raise OSError("connection refused")
except OSError as e:
    # 只在基础设施层→应用层边界转换一次
    raise AppError("Service unavailable") from e
```

### 5.5 陷阱：异常中的可变状态

```python
# BAD——异常包含可变状态
class MutableError(Exception):
    details = []    # 类属性是可变的！在所有实例间共享！

e1 = MutableError("first")
e1.details.append("error detail")
e2 = MutableError("second")
print(e2.details)  # ['error detail'] —— 被其他实例修改了！

# GOOD——使用实例属性或不可变类型
class ImmutableError(Exception):
    def __init__(self, message, details=None):
        self.details = tuple(details or [])   # 不可变元组
        super().__init__(message)

e1 = ImmutableError("first", ["detail"])
e2 = ImmutableError("second")
print(e2.details)  # () —— 独立的
```

### 5.6 最佳实践：异常体系设计原则

```python
# 原则1：按领域/模块组织异常层级
# 每个模块/领域一个基类，最多 3 层继承
class AppError(Exception):
    """应用级基类"""
    pass

class DomainError(AppError):
    """领域层基类"""
    pass

class UserError(DomainError):    # 不超过3层
    """用户领域基类"""
    pass

# 原则2：异常名称以 Error 结尾
class UserNotFoundError(UserError): pass    # ✓
class UserNotFound(UserError): pass         # ✗ 不直观

# 原则3：异常携带结构化信息
class UserNotFoundError(UserError):
    def __init__(self, user_id):
        self.user_id = user_id              # 结构化属性
        super().__init__(f"User {user_id} not found")

# 原则4：提供 to_dict() 用于 API 响应
class AppError(Exception):
    def to_dict(self):
        return {
            "error_code": self.error_code,
            "message": self.user_message,
        }

# 原则5：异常类应有清晰的 docstring
class UserNotFoundError(UserError):
    """当指定 ID 的用户不存在时抛出。

    Args:
        user_id: 未找到的用户 ID

        user_id: 未找到的用户 ID
    """
```

### 5.7 最佳实践：使用异常抽象边界

```python
# 原则：只在抽象层边界转换异常

# 数据访问层
class DataAccessException(Exception):
    pass

def get_user(user_id):
    try:
        return db.query("SELECT * FROM users WHERE id = %s", user_id)
    except db.ConnectionError as e:
        # 在层边界转换：数据库异常 → 数据访问异常
        raise DataAccessException(f"Failed to get user {user_id}") from e

# 业务逻辑层
class BusinessRuleException(Exception):
    pass

def activate_user(user_id):
    try:
        user = get_user(user_id)
    except DataAccessException as e:
        # 在层边界转换：数据访问异常 → 业务异常
        raise BusinessRuleException(f"Cannot activate user {user_id}") from e
    # 业务逻辑...

# 表示层（API 层）
@app.route("/users/<user_id>/activate")
def handle_activate(user_id):
    try:
        activate_user(user_id)
    except BusinessRuleException as e:
        # 在层边界转换：业务异常 → HTTP 异常
        raise HttpError(status=400, message=str(e)) from e
```

### 5.8 最佳实践：异常与日志的配合

```python
import logging

logger = logging.getLogger(__name__)

# 原则：在异常发生点记录日志，在异常处理点决定响应

def data_access_layer():
    try:
        result = db.query("SELECT ...")
    except db.Error as e:
        # 记录完整堆栈（包含底层细节）
        logger.exception("Database query failed")  # 自动包含 traceback
        # 抛出领域异常（不暴露底层细节）
        raise DataAccessException("Data access failed") from e

def api_layer():
    try:
        data_access_layer()
    except DataAccessException as e:
        # 记录领域异常
        logger.error(f"Data access error: {e.error_code}")
        # 返回用户友好的错误响应
        return {"error": e.user_message, "code": e.error_code}

# 日志级别选择
# DEBUG: 开发调试信息
# INFO: 正常业务流程
# WARNING: 可恢复的异常情况
# ERROR: 需要关注的异常
# CRITICAL: 系统级故障
```
