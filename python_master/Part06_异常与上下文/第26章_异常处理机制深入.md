# 第26章 异常处理机制深入

## 1. 概念与语法

### 1.1 异常层级体系：BaseException 继承树

Python 的所有异常都源自 `BaseException`，其继承树如下：

```
BaseException
 ├── SystemExit                     # sys.exit() 触发
 ├── KeyboardInterrupt               # Ctrl+C 触发
 ├── GeneratorExit                   # 生成器 close() 触发
 └── Exception                       # 所有用户可捕获异常的基类
      ├── StopIteration
      ├── ArithmeticError
      │    ├── FloatingPointError
      │    ├── OverflowError
      │    └── ZeroDivisionError
      ├── LookupError
      │    ├── IndexError
      │    └── KeyError
      ├── OSError
      │    ├── FileNotFoundError
      │    ├── PermissionError
      │    ├── IsADirectoryError
      │    └── NotADirectoryError
      ├── TypeError
      ├── ValueError
      ├── AttributeError
      ├── NameError
      ├── RuntimeError
      │    └── NotImplementedError
      ├── SyntaxError
      │    └── IndentationError
      │         └── TabError
      ├── ImportError
      │    └── ModuleNotFoundError
      └── ... (其他内置异常)
```

**关键设计原则**：`BaseException` 的直接子类（`SystemExit`、`KeyboardInterrupt`、`GeneratorExit`）不应被常规 `except Exception` 捕获——它们代表进程级中断信号，而非程序逻辑错误。

### 1.2 try/except/else/finally 完整语义

```python
try:
    # 执行可能抛出异常的代码
    risky_operation()
except ValueError as e:
    # 捕获特定异常，绑定异常对象到 e
    handle_value_error(e)
except (TypeError, AttributeError) as e:
    # 捕获多个异常类型
    handle_type_or_attr_error(e)
except Exception as e:
    # 捕获所有用户级异常（不推荐用于顶层兜底）
    handle_generic(e)
else:
    # 仅当 try 块没有抛出任何异常时执行
    # 适合放置"成功后"的逻辑，避免意外捕获非预期异常
    on_success()
finally:
    # 无论是否发生异常，无论异常是否被捕获，都必定执行
    # 用于资源清理
    cleanup()
```

**执行顺序规则**：

| try 是否异常 | except 是否匹配 | else 是否执行 | finally 执行时机 |
|---|---|---|---|
| 无异常 | — | 执行 | try 之后立即 |
| 有异常 | 匹配 | 不执行 | except 之后 |
| 有异常 | 不匹配 | 不执行 | 异常传播前 |

### 1.3 异常链（Exception Chaining）

```python
# 显式链：raise ... from ...
try:
    raw = parse_config(path)
except ValueError as e:
    raise ConfigError(f"Invalid config: {path}") from e
    # e 成为新异常的 __cause__，traceback 中显示：
    # "The above exception was the direct cause of the following exception:"

# 隐式链：异常处理中再次抛出
try:
    process()
except ValueError:
    raise RuntimeError("processing failed")
    # 原始 ValueError 绑定到 __context__
    # traceback 中显示：
    # "During handling of the above exception, another exception occurred:"

# 抑制链：raise ... from None
try:
    legacy_parse(data)
except LegacyError:
    raise ModernError("parse failed") from None
    # __cause__ 被设为 None，__suppress_context__ 被设为 True
    # traceback 中不显示前导异常
```

### 1.4 异常信息与 traceback 定制

```python
import traceback

try:
    1 / 0
except ZeroDivisionError as e:
    # 获取格式化的 traceback 字符串
    tb_str = traceback.format_exc()

    # 获取 traceback 对象
    import sys
    exc_type, exc_value, exc_tb = sys.exc_info()

    # 手动遍历栈帧
    for frame in traceback.extract_tb(exc_tb):
        print(f"{frame.filename}:{frame.lineno} in {frame.name}")
        print(f"  {frame.line}")
```

## 2. 原理与机制

### 2.1 CPython 中异常对象的内存布局

在 CPython 源码（`Objects/exceptions.c`、`Include/cpython/exceptions.h`）中，异常对象的底层结构如下：

```c
// Include/cpython/exceptions.h (简化)
typedef struct {
    PyObject_HEAD                    // ob_refcnt, ob_type
    PyObject *dict;                  // __dict__ 属性字典
    PyObject *args;                  // 参数元组
    PyObject *notes;                 // __notes__ 列表 (Python 3.11+)
    PyObject *message;               // __message__ (Python 3.11+)
} PyBaseExceptionObject;

typedef struct {
    PyBaseExceptionObject base;      // 继承 BaseException
    PyObject *msg;                   // 错误消息
    PyObject *filename;              // 文件名 (OSError 系)
    PyObject *lineno;                // 行号
    PyObject *offset;                // 列偏移
    PyObject *end_lineno;            // 结束行号 (Python 3.10+)
    PyObject *end_offset;            // 结束列偏移 (Python 3.10+)
    PyObject *text;                  // 源码行文本
} PySyntaxErrorObject;
```

**关键细节**：

- `BaseException` 实例的 `args` 属性即构造时传入的参数元组，这是 `str(e)` 的输出来源
- `__dict__` 允许动态添加属性（`e.custom_attr = value`），但推荐在子类中声明具名属性
- `__traceback__` 属性在异常被 `except` 捕获时自动绑定，指向 `PyTracebackObject`

### 2.2 异常传播链路的 C 层实现

当 Python 虚拟机执行字节码时遇到错误，传播过程如下：

```
Python VM 字节码执行
    │
    ▼ 遇到错误（如除零）
PyEval_EvalFrameDefault()  ← Python/ceval.c
    │  设置异常到当前线程状态
    ▼
PyThreadState.exc_type / exc_value / exc_traceback
    │  保存到 tstate->curexc_*
    ▼
帧栈展开 (frame stack unwinding)
    │  逐帧向上查找 except 子句
    ▼
找到匹配的 except 块
    │  执行 except 块体
    ▼
如果 except 块自身抛出异常
    │  原异常被保存为 __context__
    ▼
如果使用 raise ... from ...
    │  原异常被保存为 __cause__
    ▼
最终未被捕获的异常到达 sys.excepthook
```

**核心 C 函数**：

```c
// Python/errors.c
void PyErr_SetObject(PyObject *exception, PyObject *value) {
    PyThreadState *tstate = _PyThreadState_GET();
    // 设置 curexc_type, curexc_value, curexc_traceback
    _PyErr_SetObject(tstate, exception, value);
}

// Python/ceval.c 中的异常处理逻辑
// 当 SETUP_FINALLY / SETUP_EXCEPT 字节码指令设置帧的异常处理块时：
// f_blockstack[0] = { SETUP_EXCEPT, target_addr }
// 帧栈展开时，每帧检查 f_blockstack 是否有匹配的 handler
```

### 2.3 try/except 字节码级分析

```python
def demo():
    try:
        x = 1 / 0
    except ZeroDivisionError:
        x = 0
```

编译后的字节码：

```
  2           0 PUSH_EXC_INFO         # 保存当前异常信息到栈
              2 LOAD_CONST            0 (1)
              4 LOAD_CONST            1 (0)
              6 BINARY_OP            11 (/)   # 触发除零异常
             10 STORE_NAME            0 (x)
             12 POP_EXCEPT            0       # 正常路径：弹出异常栈
             14 JUMP_FORWARD         10 (to 36)

  4          16 PUSH_EXC_INFO         # except 路径入口
             18 LOAD_NAME             1 (ZeroDivisionError)
             20 CHECK_EXC_MATCH                  # 匹配异常类型
             22 POP_TOP
  5          24 LOAD_CONST            2 (0)
             26 STORE_NAME            0 (x)
             28 POP_EXCEPT            0
             30 JUMP_FORWARD          2 (to 36)
             32 POP_EXCEPT            1       # 不匹配时恢复异常
             34 RERAISE               0       # 重新抛出

  2          36 NOP
             38 LOAD_CONST            0 (None)
             40 RETURN_VALUE
```

**关键字节码指令**：

| 指令 | 作用 |
|------|------|
| `PUSH_EXC_INFO` | 将当前异常压入异常栈，为 except 做准备 |
| `CHECK_EXC_MATCH` | 检查栈顶异常是否匹配指定类型（使用 `PyObject_IsInstance`） |
| `POP_EXCEPT` | 弹出异常栈，恢复正常异常状态 |
| `RERAISE` | 重新抛出当前异常（用于 except 不匹配时） |
| `SETUP_FINALLY` | （旧版）注册 finally 块的清理处理程序 |

### 2.4 finally 的无异常路径优化

CPython 对 `finally` 块做了重要优化——当 `try` 块没有异常时，`finally` 的执行不需要经过完整的异常栈操作：

```c
// Python/ceval.c 中的逻辑（伪代码）
// 如果 try 块正常完成，直接跳转到 finally 块
// 不经过 PUSH_EXC_INFO / POP_EXCEPT 流程
// 但 finally 块中如果发生 return/break/continue，
// 仍然会确保 finally 代码先执行
```

这意味着 `finally` 块的"必定执行"保证是通过字节码层面的控制流保证的，而非通过异常机制本身。

### 2.5 异常链的内部结构

```c
// BaseException 的完整 C 结构（简化）
typedef struct {
    PyObject_HEAD
    PyObject *dict;
    PyObject *args;
    PyObject *notes;
    PyObject *message;
    // 以下为异常链核心字段
    PyObject *traceback;          // __traceback__: PyTracebackObject*
    PyObject *cause;              // __cause__: 显式链（raise ... from）
    PyObject *context;           // __context__: 隐式链
    char suppress_context;        // __suppress_context__: from None 时为 1
} PyBaseExceptionObject;
```

**`raise ... from` 的 C 层实现**：

```c
// Python/ceval.c 中 RAISE_VARSTAR 字节码处理
// 当检测到 from 子句时：
//   1. 将 cause 异常赋值给 value->cause
//   2. 设置 value->suppress_context = 0
//
// 当检测到 from None 时：
//   1. 设置 value->cause = Py_None (None)
//   2. 设置 value->suppress_context = 1
//   这样 traceback 打印时会跳过 __context__ 的显示
```

### 2.6 异常匹配的 isinstance 语义

`except` 子句的匹配使用 `isinstance()` 语义，这在 C 层通过 `PyObject_IsInstance` 实现：

```c
// Python/ceval.c (CHECK_EXC_MATCH 处理)
// 等价于 Python 层面的：
// isinstance(raised_exception, caught_type)

// 这意味着：
# 1. 子类异常会被父类 except 捕获
class MyValueError(ValueError): pass
try:
    raise MyValueError("oops")
except ValueError:       # 捕获到！因为 MyValueError 是 ValueError 的子类
    pass

# 2. except 元组匹配使用 OR 语义
except (TypeError, ValueError):   # 等价于 isinstance(e, (TypeError, ValueError))
    pass
```

### 3.2 资源清理保障——finally 的核心价值

```python
def process_with_cleanup(resource):
    """确保资源释放，无论是否发生异常"""
    resource.acquire()
    try:
        result = resource.process()
    except ProcessingError as e:
        resource.mark_failed()
        raise  # 重新抛出，但不影响清理
    else:
        resource.commit()
        return result
    finally:
        resource.release()  # 必定执行
```

### 3.3 异常链——封装底层实现细节

### 3.5 控制流——EAFP 风格

```python
def get_nested(data, *keys, default=None):
    """EAFP 风格：用异常处理替代逐层判断"""
```

## 4. 示例代码

### 4.1 完整的异常层级探索器

```python
#!/usr/bin/env python3
"""探索 Python 异常层级体系——C 源码级分析"""

import builtins
import inspect


def build_exception_tree():
    """构建完整的异常继承树并统计"""
    tree = {}
    all_exceptions = []

    for name in dir(builtins):
        obj = getattr(builtins, name)
        if isinstance(obj, type) and issubclass(obj, BaseException):
            all_exceptions.append(obj)

    # 按 MRO 深度排序
    all_exceptions.sort(key=lambda c: len(c.__mro__))

    for exc in all_exceptions:
        parent = exc.__bases__[0] if exc.__bases__ else None
        parent_name = parent.__name__ if parent else None
        tree[exc.__name__] = {
            "class": exc,
            "parent": parent_name,
            "mro_depth": len(exc.__mro__),
            "is_leaf": not any(issubclass(e, exc) and e is not exc
                               for e in all_exceptions),
        }

    return tree


def print_exception_tree(tree, root="BaseException", indent=0):
    """递归打印异常树"""
    children = [name for name, info in tree.items()
                if info["parent"] == root]
    prefix = "  " * indent + ("├── " if indent else "")
    for child in sorted(children):
        info = tree[child]
        leaf_marker = " [leaf]" if info["is_leaf"] else ""
        mro = f" (MRO depth: {info['mro_depth']})"
        print(f"{prefix}{child}{leaf_marker}{mro}")
        print_exception_tree(tree, child, indent + 1)


def analyze_exception_internals():
    """分析异常对象的内部属性——对应 CPython 结构体字段"""
    exc = ValueError("test error", 42, {"key": "val"})

    print("\n=== 异常对象内部属性 ===")
    print(f"类型: {type(exc).__name__}")
    print(f"__module__: {exc.__module__}")       # 对应 C 结构的 ob_type->tp_name
    print(f"args: {exc.args}")                    # 对应 PyBaseExceptionObject.args
    print(f"str(e): {str(exc)}")                  # 调用 BaseException_str()
    print(f"repr(e): {repr(exc)}")                # 调用 BaseException_repr()

    # __dict__ 对应 C 结构的 dict 字段
    exc.custom_attr = "dynamic"                    # 动态属性存入 __dict__
    print(f"__dict__: {exc.__dict__}")            # {'custom_attr': 'dynamic'}

    # 检查异常链字段（对应 C 结构的 traceback/cause/context/suppress_context）
    print(f"\n=== 异常链字段（CPython PyBaseExceptionObject） ===")
    print(f"__traceback__: {exc.__traceback__}")  # 对应 traceback 字段
    print(f"__cause__: {exc.__cause__}")           # 对应 cause 字段
    print(f"__context__: {exc.__context__}")       # 对应 context 字段
    print(f"__suppress_context__: {exc.__suppress_context__}")  # 对应 suppress_context


def demonstrate_exception_chaining():
    """演示三种异常链模式的完整语义"""

    print("\n=== 1. 显式链：raise ... from ... ===")
    try:
        try:
            raise ValueError("original error")
        except ValueError as e:
            raise RuntimeError("wrapper error") from e
    except RuntimeError as e:
        print(f"__cause__: {e.__cause__}")             # ValueError 实例
        print(f"__context__: {e.__context__}")          # ValueError 实例（同 __cause__）
        print(f"__suppress_context__: {e.__suppress_context__}")  # False

    print("\n=== 2. 隐式链：except 块中 raise ===")
    try:
        try:
            raise ValueError("original error")
        except ValueError:
            raise RuntimeError("wrapper error")     # 没有 from，自动链
    except RuntimeError as e:
        print(f"__cause__: {e.__cause__}")             # None
        print(f"__context__: {e.__context__}")         # ValueError 实例
        print(f"__suppress_context__: {e.__suppress_context__}")  # False

    print("\n=== 3. 抑制链：raise ... from None ===")
    try:
        try:
            raise ValueError("original error")
        except ValueError:
            raise RuntimeError("wrapper error") from None
    except RuntimeError as e:
        print(f"__cause__: {e.__cause__}")             # None
        print(f"__context__: {e.__context__}")         # ValueError 实例（仍存在）
        print(f"__suppress_context__: {e.__suppress_context__}")  # True


def demonstrate_try_semantics():
    """演示 try/except/else/finally 的完整执行语义"""

    results = []

    # 场景1：无异常
    print("\n=== 场景1：try 块无异常 ===")
    results.clear()
    try:
        results.append("try")
    except ValueError:
        results.append("except")
    else:
        results.append("else")       # 执行
    finally:
        results.append("finally")    # 执行
    print(f"执行顺序: {' → '.join(results)}")
    # 输出: try → else → finally

    # 场景2：有异常被捕获
    print("\n=== 场景2：异常被捕获 ===")
    results.clear()
    try:
        results.append("try")
        raise ValueError("boom")
    except ValueError:
        results.append("except")     # 执行
    else:
        results.append("else")
    finally:
        results.append("finally")    # 执行
    print(f"执行顺序: {' → '.join(results)}")
    # 输出: try → except → finally

    # 场景3：异常未被捕获
    print("\n=== 场景3：异常未被捕获 ===")
    results.clear()
    try:
        try:
            results.append("try")
            raise TypeError("boom")
        except ValueError:
            results.append("except")
        else:
            results.append("else")
        finally:
            results.append("finally")  # 仍然执行
    except TypeError:
        results.append("outer-except")
    print(f"执行顺序: {' → '.join(results)}")
    # 输出: try → finally → outer-except

    # 场景4：finally 中的 return 覆盖异常
    print("\n=== 场景4：finally 中 return 吞掉异常 ===")
    def swallow_exception():
        try:
            raise ValueError("swallowed")
        finally:
            return "finally-returned"     # 异常被丢弃！

    result = swallow_exception()
    print(f"返回值: {result}")   # "finally-returned"

    # 场景5：finally 中 raise 替换原异常
    print("\n=== 场景5：finally 中 raise 替换异常 ===")
    try:
        try:
            raise ValueError("original")
        finally:
            raise RuntimeError("from finally")   # 替换原异常
    except RuntimeError as e:
        print(f"捕获到: {type(e).__name__}: {e}")
        print(f"__context__: {type(e.__context__).__name__}: {e.__context__}")


def demonstrate_traceback_customization():
    """traceback 定制与深度分析"""
    import sys
    import traceback

    print("\n=== traceback 对象结构 ===")

    def level3():
        return 1 / 0                          # 最底层：异常起源

    def level2():
        return level3()                        # 中间层

    def level1():
        return level2()                        # 顶层调用

    try:
        level1()
    except ZeroDivisionError as e:
        tb = e.__traceback__

        # 遍历 traceback 链（对应 CPython PyTracebackObject 结构）
        print("traceback 链（从内到外）：")
        frames = []
        current = tb
        while current is not None:
            # 每个 PyTracebackObject 包含：
            # tb_frame: PyFrameObject*（栈帧）
            # tb_lasti: int（最后执行的字节码偏移）
            # tb_lineno: int（源码行号）
            # tb_next: PyTracebackObject*（上一级 traceback）
            frame = current.tb_frame
            print(f"  {frame.f_code.co_filename}:{current.tb_lineno}"
                  f" in {frame.f_code.co_name}"
                  f" (lasti={current.tb_lasti})")
            current = current.tb_next

        # 使用 traceback 模块的高级 API
        print("\n=== 格式化输出 ===")
        print("1. traceback.format_exc():")
        print(traceback.format_exc())

        print("2. 自定义格式（提取关键信息）：")
        for frame_info in traceback.extract_tb(e.__traceback__):
            print(f"  {frame_info.filename}:{frame_info.lineno}"
                  f" in {frame_info.name}")
            print(f"    {frame_info.line}")


if __name__ == "__main__":
    print("=" * 60)
    print("Python 异常层级体系")
    print("=" * 60)

    tree = build_exception_tree()
    print(f"\n内置异常总数: {len(tree)}")
    print("\n异常继承树：")
    print_exception_tree(tree)

    analyze_exception_internals()
    demonstrate_exception_chaining()
    demonstrate_try_semantics()
    demonstrate_traceback_customization()
```

### 4.2 异常传播与帧栈展开的完整模拟

```python
#!/usr/bin/env python3
"""模拟 CPython 异常传播与帧栈展开过程"""

import sys
import dis
import traceback


def simulate_exception_propagation():
    """展示异常如何沿调用栈逐帧传播"""

    def inner():
        frame = sys._getframe()
        print(f"  [inner] 帧信息: {frame.f_code.co_name}, 行 {frame.f_lineno}")
        raise ValueError("originated in inner()")

    def middle():
        frame = sys._getframe()
        print(f"  [middle] 帧信息: {frame.f_code.co_name}, 行 {frame.f_lineno}")
        inner()  # 异常将传播到此帧

    def outer():
        frame = sys._getframe()
        print(f"  [outer] 帧信息: {frame.f_code.co_name}, 行 {frame.f_lineno}")
        middle()  # 异常将传播到此帧

    print("=== 异常传播模拟 ===")
    print("调用链: outer() → middle() → inner()")
    print("异常在 inner() 产生，沿帧栈逐层向上传播\n")

    try:
        outer()
    except ValueError as e:
        print("\n捕获到异常后的 traceback 链：")
        tb = e.__traceback__
        depth = 0
        while tb is not None:
            frame = tb.tb_frame
            print(f"  [深度 {depth}] {frame.f_code.co_filename}:"
                  f"{tb.tb_lineno} in {frame.f_code.co_name}"
                  f" (字节码偏移: {tb.tb_lasti})")
            tb = tb.tb_next
            depth += 1


def analyze_bytecode_exception_handling():
    """分析异常处理的字节码实现"""

    def try_except_else_finally():
        try:
            x = 1 / 0
        except ZeroDivisionError:
            x = 0
        else:
            y = x + 1
        finally:
            z = x

    print("\n=== 异常处理字节码 ===")
    dis.dis(try_except_else_finally)


def demonstrate_nested_exception_context():
    """展示嵌套异常处理中的 __context__ 行为"""

    print("\n=== 嵌套异常中的 context 链 ===")

    try:
        try:
            try:
                raise ValueError("Level 1")
            except ValueError:
                raise TypeError("Level 2")     # context = ValueError
        except TypeError:
            raise RuntimeError("Level 3")       # context = TypeError
    except RuntimeError as e:
        # 沿 context 链回溯
        print("context 链（从外到内）：")
        current = e
        depth = 0
        while current is not None:
            print(f"  [深度 {depth}] {type(current).__name__}: {current}")
            current = current.__context__
            depth += 1
            if depth > 10:  # 防止无限循环
                break


def show_exception_state_in_thread():
    """展示线程级异常状态（对应 CPython tstate->curexc_*）"""

    import sys

    print("\n=== 线程级异常状态 ===")
    print("sys.exc_info() 在 Python 3.12+ 中返回 (None, None, None)")
    print("异常信息现在通过异常对象的 __traceback__ 属性获取")
    print(f"当前线程异常状态: {sys.exc_info()}")

    # 在 except 块内部，sys.exc_info() 仍然可用
    try:
        raise RuntimeError("test")
    except RuntimeError:
        exc_type, exc_value, exc_tb = sys.exc_info()
        print(f"except 块内: type={exc_type.__name__}, value={exc_value}")


if __name__ == "__main__":
    simulate_exception_propagation()
    analyze_bytecode_exception_handling()
    demonstrate_nested_exception_context()
```

### 4.3 自定义异常的元类级操控

```python
#!/usr/bin/env python3
"""自定义异常的高级元编程技术"""

import traceback


class ExceptionMeta(type):
    """异常元类：自动注册异常类到全局注册表"""
    _registry = {}

    def __new__(mcs, name, bases, namespace):
        cls = super().__new__(mcs, name, bases, namespace)
        # 跳过基类自身
        if name != "DomainException":
            mcs._registry[name] = cls
        return cls

    @classmethod
    def get_registry(mcs):
        return dict(mcs._registry)


class DomainException(Exception, metaclass=ExceptionMeta):
    """领域异常基类——所有业务异常的统一入口"""
    pass


class ValidationError(DomainException):
    """验证错误"""
    def __init__(self, field, message, value=None):
        self.field = field
        self.message = message
        self.value = value
        super().__init__(f"{field}: {message}" + (f" (got {value!r})" if value else ""))


class NotFoundError(DomainException):
    """资源未找到"""
    def __init__(self, resource_type, resource_id):
        self.resource_type = resource_type
        self.resource_id = resource_id
        super().__init__(f"{resource_type} with id={resource_id} not found")


class ConflictError(DomainException):
    """资源冲突"""
    def __init__(self, resource_type, resource_id, reason=""):
        self.resource_type = resource_type
        self.resource_id = resource_id
        self.reason = reason
        super().__init__(f"{resource_type} conflict on id={resource_id}: {reason}")


def demonstrate_exception_metaclass():
    """演示元类自动注册功能"""

    print("=== 异常元类自动注册 ===")
    registry = ExceptionMeta.get_registry()
    print(f"已注册异常: {list(registry.keys())}")

    # 可以通过名称查找异常类
    exc_cls = ExceptionMeta._registry.get("ValidationError")
    if exc_cls:
        print(f"通过名称找到: {exc_cls}")


def demonstrate_rich_exception_attrs():
    """演示异常的自定义属性（对应 C 结构的 dict 字段）"""

    print("\n=== 异常自定义属性 ===")

    try:
        raise ValidationError("email", "invalid format", "not-an-email")
    except ValidationError as e:
        # 自定义属性存储在 __dict__ 中
        print(f"异常类: {type(e).__name__}")
        print(f"args: {e.args}")          # ('email: invalid format (got \'not-an-email\')',)
        print(f"field: {e.field}")         # 'email'
        print(f"message: {e.message}")     # 'invalid format'
        print(f"value: {e.value}")         # 'not-an-email'
        print(f"__dict__: {e.__dict__}")
        print(f"str(e): {e}")              # 'email: invalid format (got \'not-an-email\')'


def demonstrate_exception_notes():
    """Python 3.11+ 的 __notes__ 特性"""

    print("\n=== Python 3.11+ __notes__ 特性 ===")

    try:
        exc = ValueError("validation failed")
        # add_note() 向 __notes__ 列表添加注释
        exc.add_note("Field 'age' must be positive")
        exc.add_note("Field 'name' cannot be empty")
        raise exc
    except ValueError as e:
        print(f"__notes__: {e.__notes__}")
        # traceback 输出中将包含 notes


if __name__ == "__main__":
    demonstrate_exception_metaclass()
    demonstrate_rich_exception_attrs()
    demonstrate_exception_notes()
```

## 5. 常见陷阱与最佳实践

### 5.1 陷阱：裸 except 捕获过多

```python
# BAD——捕获所有异常包括 SystemExit 和 KeyboardInterrupt
try:
    do_something()
except:                    # 等价于 except BaseException
    pass                   # 吞掉 Ctrl+C、sys.exit() 等

# GOOD——至少指定 Exception
try:
    do_something()
except Exception as e:     # 不捕获 SystemExit / KeyboardInterrupt
    logger.error(f"Unexpected error: {e}")
    raise                  # 重新抛出，不要静默吞掉

# BEST——捕获具体异常
try:
    do_something()
except (ValueError, TypeError) as e:
    handle_error(e)
```

### 5.2 陷阱：finally 中的 return 吞掉异常

```python
# BAD——finally 中的 return 静默吞掉异常
def bad_func():
    try:
        raise ValueError("important error")
    finally:
        return 42          # 异常被丢弃！调用方完全不知道发生了错误

# GOOD——finally 只做清理，不返回值
def good_func():
    result = None
    error = None
    try:
        result = risky_operation()
    except ValueError as e:
        error = e
    finally:
        cleanup()
    if error:
        raise error
    return result
```

### 5.3 陷阱：异常匹配顺序错误

```python
# BAD——子类在前，父类永远无法到达
try:
    do_something()
except Exception:           # 先匹配，所有 Exception 子类都被捕获
    pass
except ValueError:          # 永远不会执行！ValueError 是 Exception 的子类
    pass

# GOOD——从具体到宽泛
try:
    do_something()
except ValueError:           # 先匹配具体类型
    pass
except Exception:            # 最后匹配宽泛类型
    pass
```

### 5.4 陷阱：异常对象在循环中引用

```python
# BAD——在 Python 3 中，except 子句结束后异常引用被清除
# 但在循环中可能意外保留引用
def bad_loop():
    exceptions = []
    for i in range(3):
        try:
            1 / i
        except ZeroDivisionError as e:
            exceptions.append(e)    # e 在 except 块结束后被清除
    # exceptions[0] 的 __traceback__ 可能引用已清除的帧

# GOOD——显式保存异常信息
def good_loop():
    exceptions = []
    for i in range(3):
        try:
            1 / i
        except ZeroDivisionError as e:
            # 创建新异常或保存必要信息
            exceptions.append((type(e), str(e), traceback.format_exc()))
```

### 5.5 最佳实践：异常信息的丰富性

```python
# BAD——信息不足
raise ValueError("invalid input")

# GOOD——包含上下文信息
raise ValueError(f"invalid input: expected positive number, got {value!r}")

# BEST——使用自定义异常，携带结构化数据
class InvalidInputError(ValueError):
    def __init__(self, field, value, constraint):
        self.field = field
        self.value = value
        self.constraint = constraint
        super().__init__(
            f"Field '{field}' validation failed: "
            f"expected {constraint}, got {value!r}"
        )
```

### 5.6 最佳实践：避免异常用于控制流

```python
# BAD——用异常做流程控制（慢且不清晰）
def get_item(lst, index):
    try:
        return lst[index]
    except IndexError:
        return None

# GOOD——使用条件判断（EAFP 仅适用于合理场景）
def get_item(lst, index):
    if 0 <= index < len(lst):
        return lst[index]
    return None

# 但 EAFP 在某些场景是 Pythonic 的（如 dict.get 不够用时）
def get_config(key):
    try:
        return os.environ[key]           # 可能抛出 KeyError
    except KeyError:
        return DEFAULT_CONFIG[key]       # 再试另一个来源
```

### 5.7 最佳实践：合理使用异常链

```python
# BAD——丢失原始异常信息
try:
    db.connect()
except ConnectionError:
    raise AppError("database connection failed")  # 丢失了底层错误细节

# GOOD——使用 from 保留因果链
try:
    db.connect()
except ConnectionError as e:
    raise AppError("database connection failed") from e

# 适当使用 from None 隐藏实现细节
try:
    legacy_parser.parse(data)
except LegacyError:
    raise ModernError("parse failed") from None  # 用户不需要看到遗留异常
```

### 5.8 最佳实践：__str__ 与 __repr__ 的区分

```python
class AppError(Exception):
    def __init__(self, message, error_code, details=None):
        self.error_code = error_code
        self.details = details or {}
        super().__init__(message)

    def __str__(self):
        # 面向用户的简洁消息
        return f"[{self.error_code}] {self.args[0]}"

    def __repr__(self):
        # 面向开发者的详细信息
        return (f"{self.__class__.__name__}("
                f"message={self.args[0]!r}, "
                f"error_code={self.error_code!r}, "
                f"details={self.details!r})")
```

### 5.9 性能考量：异常处理的开销

CPython 中异常处理的性能特征：

- **正常路径（无异常）**：`try` 块几乎零开销——`PUSH_EXC_INFO` 和 `POP_EXCEPT` 都是轻量操作
- **异常路径（抛出异常）**：开销较大——需要遍历帧栈、创建 traceback 对象、填充异常结构体
- **最佳实践**：异常应仅用于真正的异常情况，不应用于常规控制流

```python
import timeit

# 正常路径——try 几乎零开销
normal_with_try = timeit.timeit(
    "try:\n    x = 1 + 1\nexcept:\n    pass",
    number=10_000_000
)

# 异常路径——开销显著
exception_path = timeit.timeit(
    "try:\n    raise ValueError()\nexcept ValueError:\n    pass",
    number=10_000_000
)

print(f"正常路径 (try): {normal_with_try:.3f}s")
print(f"异常路径 (raise): {exception_path:.3f}s")
# 典型结果：异常路径慢 10-50 倍
```
