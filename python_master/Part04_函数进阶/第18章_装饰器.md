# 第18章 装饰器

## 18.1 概念与语法

### 18.1.1 装饰器的本质

装饰器（Decorator）是一种设计模式，在 Python 中通过语法糖 `@decorator` 实现。其本质是一个 **接受函数作为参数并返回新函数的高阶函数**。`@decorator` 语法等价于 `func = decorator(func)`。

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before call")
        result = func(*args, **kwargs)
        print("After call")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")

# 等价于: say_hello = my_decorator(say_hello)
say_hello("World")
# Before call
# Hello, World!
# After call
```

### 18.1.2 函数装饰器

最基础的装饰器形式，接收一个函数，返回一个增强后的函数：

```python
def validate_non_negative(func):
    """验证所有参数为非负数"""
    def wrapper(*args, **kwargs):
        for arg in args:
            if isinstance(arg, (int, float)) and arg < 0:
                raise ValueError(f"Negative value not allowed: {arg}")
        return func(*args, **kwargs)
    return wrapper

@validate_non_negative
def compute_area(radius):
    return 3.14159 * radius ** 2

print(compute_area(5))    # 78.53975
# compute_area(-1)        # ValueError: Negative value not allowed: -1
```

### 18.1.3 类装饰器

类装饰器使用 `__call__` 方法实现可调用实例，适合需要维护状态的场景：

```python
class CountCalls:
    """统计函数调用次数的类装饰器"""
    def __init__(self, func):
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Call #{self.count} of {self.func.__name__}")
        return self.func(*args, **kwargs)

@CountCalls
def greet(name):
    return f"Hello, {name}!"

print(greet("Alice"))  # Call #1 of greet
print(greet("Bob"))    # Call #2 of greet
print(greet.count)     # 2
```

### 18.1.4 带参数的装饰器

带参数的装饰器实际上是 **装饰器工厂**——一个返回装饰器的函数：

```python
def retry(max_attempts=3, delay=1.0, exceptions=(Exception,)):
    """重试装饰器工厂"""
    import time

    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts:
                        raise
                    print(f"Attempt {attempt}/{max_attempts} failed: {e}")
                    time.sleep(delay)
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.5, exceptions=(ValueError,))
def unstable_operation(x):
    import random
    if random.random() < 0.7:
        raise ValueError("Random failure")
    return x * 2

result = unstable_operation(5)
```

### 18.1.5 装饰器链

多个装饰器可以叠加使用，执行顺序为 **从下到上**（最靠近函数定义的装饰器先执行包装），调用时则 **从上到下**：

```python
@decorator_a   # 第二层包装
@decorator_b   # 第一层包装
def func():
    pass

# 等价于: func = decorator_a(decorator_b(func))
```

```python
def trace(prefix):
    def decorator(func):
        def wrapper(*args, **kwargs):
            print(f"{prefix} -> {func.__name__}")
            result = func(*args, **kwargs)
            print(f"{prefix} <- {func.__name__}")
            return result
        return wrapper
    return decorator

@trace("A")
@trace("B")
def hello():
    print("  hello!")

hello()
# A -> B_wrapper  （注意：内层装饰后函数名变为 B_wrapper）
# B -> hello
#   hello!
# B <- hello
# A <- B_wrapper
```

## 18.2 原理与机制

### 18.2.1 装饰器的编译期处理

当 Python 编译器遇到 `@decorator` 语法时，它在编译期将其转换为函数调用和赋值：

```python
# 源代码
@decorator
def func():
    pass

# 编译后等价于
func = decorator(func)
```

带参数的装饰器编译过程：

```python
# 源代码
@decorator(arg1, arg2)
def func():
    pass

# 编译后等价于
func = decorator(arg1, arg2)(func)
# 即: 先调用 decorator(arg1, arg2) 返回真正的装饰器，
#     再用真正的装饰器包装 func
```

通过 `dis` 模块可以观察到装饰器语法糖展开的字节码：

```python
import dis

code_str = """
@my_decorator
def func():
    pass
"""
dis.dis(compile(code_str, '<test>', 'exec'))
# 可以看到 LOAD_NAME 'my_decorator'
#          LOAD_CONST 'func' (函数对象)
#          CALL_FUNCTION 1   <-- 调用 my_decorator(func)
#          STORE_NAME 'func'
```

### 18.2.2 functools.wraps 的原理

装饰器默认会丢失原始函数的元信息（`__name__`、`__doc__`、`__module__` 等）。`functools.wraps` 通过将这些元信息从原始函数复制到包装函数来解决这个问题。

`functools.wraps` 的源码核心逻辑（`Lib/functools.py`）：

```python
WRAPPER_ASSIGNMENTS = ('__module__', '__name__', '__qualname__',
                       '__annotations__', '__doc__')
WRAPPER_UPDATES = ('__dict__',)

def wraps(wrapped,
          assigned=WRAPPER_ASSIGNMENTS,
          updated=WRAPPER_UPDATES):
    """装饰器工厂，将 wrapped 的属性更新到 wrapper 上"""
    def decorator(wrapper):
        # 复制属性
        for attr in assigned:
            try:
                value = getattr(wrapped, attr)
            except AttributeError:
                pass
            else:
                setattr(wrapper, attr, value)
        # 更新 __dict__
        for attr in updated:
            getattr(wrapper, attr).update(getattr(wrapped, attr, {}))
        # 设置 __wrapped__ 属性
        wrapper.__wrapped__ = wrapped
        return wrapper
    return decorator
```

使用与不使用 `@wraps` 的对比：

```python
def without_wraps(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

def with_wraps(func):
    import functools
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@without_wraps
def func_a():
    """This is func_a"""
    pass

@with_wraps
def func_b():
    """This is func_b"""
    pass

print(func_a.__name__)  # 'wrapper'  —— 丢失了原始名称
print(func_a.__doc__)   # None       —— 丢失了原始文档
print(func_b.__name__)  # 'func_b'   —— 保留了原始名称
print(func_b.__doc__)   # 'This is func_b' —— 保留了原始文档
```

### 18.2.3 `__wrapped__` 属性

`functools.wraps` 会在包装函数上设置 `__wrapped__` 属性，指向原始函数。这提供了直接访问原始函数的途径：

```python
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Before")
        result = func(*args, **kwargs)
        print("After")
        return result
    return wrapper

@my_decorator
def original():
    """Original docstring"""
    return 42

# 通过 __wrapped__ 访问原始函数
print(original.__name__)              # 'original'
print(original.__wrapped__)           # <function original at 0x...>
print(original.__wrapped__())         # 42（跳过装饰器）
print(original.__wrapped__.__doc__)   # 'Original docstring'
```

当多个装饰器叠加时，`__wrapped__` 链：

```python
@decorator_a
@decorator_b
def func():
    pass

# func.__wrapped__ -> decorator_b 包装后的函数
# func.__wrapped__.__wrapped__ -> 原始 func
```

### 18.2.4 类装饰器的 `__call__` 机制

类装饰器通过 `__call__` 协议使实例可调用。CPython 中 `__call__` 的调用路径：

1. `instance(*args, **kwargs)` 触发 `type(instance).__call__(instance, *args, **kwargs)`
2. 在 `__call__` 方法内部，`self.func` 保存了原始函数的引用
3. 每次调用实例时，`__call__` 被触发，执行增强逻辑

```python
class TimerDecorator:
    def __init__(self, func):
        self.func = func
        self.total_time = 0.0
        self.call_count = 0
        functools.update_wrapper(self, func)  # 复制元信息到实例

    def __call__(self, *args, **kwargs):
        import time
        start = time.perf_counter()
        result = self.func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        self.total_time += elapsed
        self.call_count += 1
        return result

    def average_time(self):
        return self.total_time / self.call_count if self.call_count else 0

@TimerDecorator
def slow_function():
    import time
    time.sleep(0.01)
    return "done"

slow_function()
slow_function()
print(f"Average: {slow_function.average_time():.6f}s")
```

### 18.2.5 装饰器与描述符协议

当装饰器作用于方法时，需要特别处理 `self` 参数。更高级的装饰器可以通过描述符协议（`__get__`）实现更精细的控制：

```python
class MethodDecorator:
    """使用描述符协议的装饰器，正确处理方法绑定"""
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        import functools
        return functools.partial(self.func, obj)

class MyClass:
    @MethodDecorator
    def method(self, x):
        return x * 2

obj = MyClass()
print(obj.method(5))   # 10（正确绑定了 self）
```

## 18.3 使用场景

### 18.3.1 路由注册（Web 框架）

Flask 风格的路由注册是装饰器的经典应用：

```python
class Router:
    def __init__(self):
        self._routes = {}

    def route(self, path, methods=None):
        methods = methods or ["GET"]
        def decorator(func):
            for method in methods:
                key = (method.upper(), path)
                self._routes[key] = func
            return func
        return decorator

    def dispatch(self, method, path):
        key = (method.upper(), path)
        if key not in self._routes:
            raise ValueError(f"No route for {method} {path}")
        return self._routes[key]()

router = Router()

@router.route("/", methods=["GET"])
def index():
    return "Home page"

@router.route("/api/users", methods=["GET", "POST"])
def users():
    return "Users list"

print(router.dispatch("GET", "/"))           # Home page
print(router.dispatch("POST", "/api/users"))  # Users list
```

### 18.3.2 权限检查

```python
def require_role(*required_roles):
    """权限检查装饰器"""
    def decorator(func):
        import functools
        @functools.wraps(func)
        def wrapper(user, *args, **kwargs):
            if not hasattr(user, 'role'):
                raise PermissionError("User has no role attribute")
            if user.role not in required_roles:
                raise PermissionError(
                    f"Role '{user.role}' not in {required_roles}"
                )
            return func(user, *args, **kwargs)
        return wrapper
    return decorator

class User:

        self.role = role

@require_role("admin", "moderator")
def delete_post(user, post_id):

admin = User("Alice", "admin")
guest = User("Bob", "guest")

print(delete_post(admin, 42))      # Alice deleted post 42
# delete_post(guest, 42)           # PermissionError
```

### 18.3.3 缓存（Memoization）

```python
def memoize(func):
    """通用缓存装饰器，支持位置参数和关键字参数"""
    import functools
    cache = {}

    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        # 构建缓存键
        key = (args, frozenset(kwargs.items()))
        if key not in cache:
            cache[key] = func(*args, **kwargs)
        return cache[key]

    wrapper.cache = cache
    wrapper.cache_clear = lambda: cache.clear()
    return wrapper

@memoize
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(100))  # 瞬间完成
print(f"缓存条目数: {len(fibonacci.cache)}")
fibonacci.cache_clear()
print(f"清空后缓存条目数: {len(fibonacci.cache)}")
```

### 18.3.4 重试与容错

```python
def retry(max_attempts=3, backoff_base=2, exceptions=(Exception,)):
    """指数退避重试装饰器"""
    import time
    import functools

    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exception = None
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    last_exception = e
                    if attempt < max_attempts:
                        delay = backoff_base ** (attempt - 1)
                        print(f"  Attempt {attempt} failed: {e}, "
                              f"retrying in {delay}s...")
                        time.sleep(delay)
            raise last_exception
        return wrapper
    return decorator

@retry(max_attempts=3, backoff_base=2, exceptions=(ConnectionError,))
def fetch_data(url):
    import random
    if random.random() < 0.8:
        raise ConnectionError(f"Failed to connect to {url}")
    return f"Data from {url}"

# fetch_data("http://example.com")
```

### 18.3.5 类型检查与契约

```python
def typecheck(**expected_types):
    """运行时类型检查装饰器"""
    import functools

    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # 检查关键字参数类型
            import inspect
            sig = inspect.signature(func)
            bound = sig.bind(*args, **kwargs)
            bound.apply_defaults()

            for param_name, expected_type in expected_types.items():
                if param_name in bound.arguments:
                    value = bound.arguments[param_name]
                    if not isinstance(value, expected_type):
                        raise TypeError(
                            f"Parameter '{param_name}' expected "
                            f"{expected_type.__name__}, got "
                            f"{type(value).__name__}"
                        )
            return func(*args, **kwargs)
        return wrapper
    return decorator

@typecheck(name=str, age=int)
def create_user(name, age):
    return {"name": name, "age": age}

print(create_user("Alice", 30))       # {'name': 'Alice', 'age': 30}
# create_user("Alice", "thirty")      # TypeError
```

## 18.4 示例代码

### 示例 1：完整的带参数装饰器——性能计时器

```python
"""
示例：带参数的性能计时器装饰器，
支持多种输出格式和统计信息
"""
import functools
import time
import statistics


def timer(output="print", precision=6, collect_stats=False):
    """
    性能计时装饰器

    参数:
        output: 'print' 打印结果, 'return' 返回元组, 'log' 记录日志
        precision: 时间精度小数位数
        collect_stats: 是否收集多次调用的统计信息
    """
    def decorator(func):
        call_times = [] if collect_stats else None

        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            start = time.perf_counter()
            result = func(*args, **kwargs)
            elapsed = time.perf_counter() - start

            if collect_stats:
                call_times.append(elapsed)

            formatted = f"{elapsed:.{precision}f}s"

            if output == "print":
                print(f"{func.__name__} took {formatted}")
            elif output == "return":
                if collect_stats:
                    return result, elapsed, statistics.mean(call_times)
                return result, elapsed

            return result

        if collect_stats:
            wrapper.get_stats = lambda: {
                "count": len(call_times),
                "total": sum(call_times),
                "avg": statistics.mean(call_times),
                "min": min(call_times),
                "max": max(call_times),
            }

        return wrapper
    return decorator


# 使用示例
@timer(output="print", collect_stats=True)
def compute_sum(n):
    return sum(range(n))

for _ in range(5):
    compute_sum(1_000_000)

print(f"\n统计信息: {compute_sum.get_stats()}")


@timer(output="return", precision=4)
def fast_compute(n):
    return n ** 2

result, elapsed = fast_compute(100)
print(f"结果: {result}, 耗时: {elapsed:.4f}s")
```

### 示例 2：类装饰器——带状态的速率限制器

```python
"""
示例：使用类装饰器实现速率限制器，
展示类装饰器维护状态的天然优势
"""
import time
import functools


class RateLimiter:
    """滑动窗口速率限制器装饰器"""

    def __init__(self, max_calls=10, period=1.0):
        self.max_calls = max_calls
        self.period = period
        self.call_times = []

    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            now = time.time()
            # 移除窗口外的调用记录
            self.call_times = [
                t for t in self.call_times
                if now - t < self.period
            ]

            if len(self.call_times) >= self.max_calls:
                wait_time = self.period - (now - self.call_times[0])
                raise RuntimeError(
                    f"Rate limit exceeded: {self.max_calls} calls per "
                    f"{self.period}s. Retry after {wait_time:.2f}s"
                )

            self.call_times.append(now)
            return func(*args, **kwargs)

        # 附加控制方法
        wrapper.reset = self.call_times.clear
        wrapper.remaining = lambda: self.max_calls - len(self.call_times)
        return wrapper


@RateLimiter(max_calls=3, period=1.0)
def send_request(url):
    return f"Response from {url}"

# 测试速率限制
for i in range(3):
    print(send_request(f"http://api.example.com/{i}"))

try:
    send_request("http://api.example.com/overflow")
except RuntimeError as e:
    print(f"Rate limited: {e}")

print(f"剩余调用次数: {send_request.remaining()}")
```

### 示例 3：装饰器链——组合多个装饰器

```python
"""
示例：装饰器链的组合效果，
展示多个装饰器的叠加行为
"""
import functools
import time


def log(level="INFO"):
    """日志装饰器"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            print(f"[{level}] Calling {func.__name__}")
            result = func(*args, **kwargs)
            print(f"[{level}] {func.__name__} returned {result!r}")
            return result
        return wrapper
    return decorator


def validate(*param_validators):
    """参数验证装饰器"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for i, validator in enumerate(param_validators):
                if i < len(args) and not validator(args[i]):
                    raise ValueError(
                        f"Validation failed for arg {i}: {args[i]!r}"
                    )
            return func(*args, **kwargs)
        return wrapper
    return decorator


def cache(maxsize=128):
    """简单缓存装饰器"""
    def decorator(func):
        cache_dict = {}

        @functools.wraps(func)
        def wrapper(*args):
            if args in cache_dict:
                print(f"  [CACHE HIT] {func.__name__}{args}")
                return cache_dict[args]
            result = func(*args)
            if len(cache_dict) >= maxsize:
                cache_dict.pop(next(iter(cache_dict)))
            cache_dict[args] = result
            return result

        wrapper.cache = cache_dict
        wrapper.cache_clear = lambda: cache_dict.clear()
        return wrapper
    return decorator


# 装饰器链：从下到上包装
@log(level="DEBUG")           # 第3层（最外层）
@validate(                     # 第2层
    lambda x: isinstance(x, (int, float)) and x >= 0,
    lambda y: isinstance(y, (int, float)) and y >= 0,
)
@cache(maxsize=10)             # 第1层（最内层）
def add(a, b):
    """Add two numbers."""
    return a + b

# 调用链: log -> validate -> cache -> add
print(add(1, 2))
# [DEBUG] Calling add
# [DEBUG] add returned 3
# 3

print(add(1, 2))  # 缓存命中
# [DEBUG] Calling add
#   [CACHE HIT] add(1, 2)
# [DEBUG] add returned 3
# 3

print(f"函数名: {add.__name__}")     # 'add'（@wraps 保留）
print(f"文档: {add.__doc__}")         # 'Add two numbers.'
```

### 示例 4：`__wrapped__` 属性与装饰器内省

```python
"""
示例：演示 __wrapped__ 属性的工作原理，
以及如何利用它进行装饰器内省
"""
import functools


def decorator_a(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Decorator A: before")
        result = func(*args, **kwargs)
        print("Decorator A: after")
        return result
    return wrapper


def decorator_b(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Decorator B: before")
        result = func(*args, **kwargs)
        print("Decorator B: after")
        return result
    return wrapper


@decorator_a
@decorator_b
def target_function(x, y):
    """Original target function"""
    return x + y


# 调用装饰后的函数
print("--- 正常调用 ---")
result = target_function(3, 4)

# 通过 __wrapped__ 访问
print("\n--- __wrapped__ 链 ---")
print(f"target_function: {target_function}")
print(f"__wrapped__: {target_function.__wrapped__}")
print(f"__wrapped__.__wrapped__: {target_function.__wrapped__.__wrapped__}")

# 跳过装饰器直接调用
print("\n--- 跳过所有装饰器 ---")
original = target_function
while hasattr(original, '__wrapped__'):
    original = original.__wrapped__
print(f"原始函数: {original}")
print(f"直接调用: {original(3, 4)}")

# 装饰器内省工具
def unwrap(func):
    """递归展开所有装饰器，找到原始函数"""
    while hasattr(func, '__wrapped__'):
        func = func.__wrapped__
    return func

original_func = unwrap(target_function)
print(f"\n原始函数名: {original_func.__name__}")
print(f"原始函数文档: {original_func.__doc__}")
```

### 示例 5：框架级装饰器——插件注册系统

```python
"""
示例：使用类装饰器实现插件注册系统，
模拟 Web 框架中的路由、中间件注册机制
"""
import functools


class PluginRegistry:
    """插件注册系统，模拟框架的装饰器注册机制"""

    def __init__(self):
        self._routes = {}      # 路径 -> 处理函数
        self._middleware = []   # 中间件链
        self._hooks = {        # 生命周期钩子
            "before_request": [],
            "after_request": [],
            "on_error": [],
        }

    def route(self, path, methods=None):
        """路由注册装饰器"""
        methods = methods or ["GET"]

        def decorator(func):
            @functools.wraps(func)
            def wrapper(*args, **kwargs):
                return func(*args, **kwargs)

            for method in methods:
                self._routes[(method.upper(), path)] = wrapper
            wrapper.path = path
            wrapper.methods = methods
            return wrapper
        return decorator

    def middleware(self, func):
        """中间件注册装饰器"""
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            return func(*args, **kwargs)
        self._middleware.append(wrapper)
        return wrapper

    def hook(self, event):
        """钩子注册装饰器"""
        def decorator(func):
            @functools.wraps(func)
            def wrapper(*args, **kwargs):
                return func(*args, **kwargs)
            self._hooks[event].append(wrapper)
            return wrapper
        return decorator

    def handle_request(self, method, path, *args, **kwargs):
        """模拟请求处理"""
        method = method.upper()

        # 执行 before_request 钩子
        for hook_fn in self._hooks["before_request"]:
            hook_fn(method, path)

        # 执行中间件
        for mw in self._middleware:
            mw(method, path)

        # 查找路由处理函数
        key = (method, path)
        if key not in self._routes:
            # 执行 on_error 钩子
            for hook_fn in self._hooks["on_error"]:
                result = hook_fn(method, path, "Route not found")
                if result:
                    return result
            raise ValueError(f"No route: {method} {path}")

        handler = self._routes[key]
        result = handler(*args, **kwargs)

        # 执行 after_request 钩子
        for hook_fn in self._hooks["after_request"]:
            hook_fn(method, path, result)

        return result


# 创建应用
app = PluginRegistry()


@app.route("/users", methods=["GET"])
def list_users():
    return ["Alice", "Bob", "Charlie"]


@app.route("/users/<id>", methods=["GET", "POST"])
def get_user(user_id=None):
    return {"id": user_id, "name": "Alice"}


@app.middleware
def log_middleware(method, path):
    print(f"  [Middleware] {method} {path}")


@app.hook("before_request")
def before(method, path):
    print(f"  [Before] Processing {method} {path}")


@app.hook("after_request")
def after(method, path, result):
    print(f"  [After] Completed {method} {path}")


# 模拟请求处理
print("--- 处理 GET /users ---")
result = app.handle_request("GET", "/users")
print(f"Result: {result}")

print("\n--- 处理 GET /users/<id> ---")
result = app.handle_request("GET", "/users/<id>", user_id=42)
print(f"Result: {result}")
```

## 18.5 常见陷阱与最佳实践

### 18.5.1 丢失函数元信息

**陷阱**：装饰器返回的 wrapper 函数会覆盖原始函数的 `__name__`、`__doc__` 等元信息。

```python
# 陷阱
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def my_function():
    """My docstring"""
    pass

print(my_function.__name__)  # 'wrapper' —— 不是 'my_function'
print(my_function.__doc__)   # None —— 文档丢失

# 修复：使用 @functools.wraps
import functools

def my_decorator_fixed(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator_fixed
def my_function2():
    """My docstring"""
    pass

print(my_function2.__name__)  # 'my_function2'
print(my_function2.__doc__)   # 'My docstring'
```

### 18.5.2 装饰器顺序错误

**陷阱**：装饰器的叠加顺序至关重要，错误的顺序会导致非预期行为。

```python
# 错误顺序：缓存在外层，权限检查在内层
# 结果：未授权的请求也可能命中缓存！
@cache()           # 外层
@require_auth()    # 内层
def sensitive_operation():
    pass

# 正确顺序：权限检查在外层
@require_auth()   # 外层 —— 先检查权限
@cache()           # 内层 —— 再查缓存
def sensitive_operation():
    pass
```

### 18.5.3 装饰器与类方法的 `self` 问题

**陷阱**：类装饰器或不当的函数装饰器可能破坏方法绑定：

```python
class BuggyDecorator:
    def __init__(self, func):
        self.func = func

    def __call__(self, *args, **kwargs):
        return self.func(*args, **kwargs)

class MyClass:
    @BuggyDecorator
    def method(self, x):
        return x * 2

obj = MyClass()
# obj.method(5)  # TypeError: method() missing 1 required positional argument: 'x'
# 原因：BuggyDecorator 实例没有 __get__ 方法，不会被绑定方法协议处理

# 修复：添加描述符协议
import functools

class FixedDecorator:
    def __init__(self, func):
        self.func = func

    def __call__(self, *args, **kwargs):
        return self.func(*args, **kwargs)

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return functools.partial(self.func, obj)

class MyClass2:
    @FixedDecorator
    def method(self, x):
        return x * 2

obj2 = MyClass2()
print(obj2.method(5))  # 10
```

### 18.5.4 装饰器中的异常处理陷阱

**陷阱**：装饰器吞掉异常或改变异常类型：

```python
# 陷阱：装饰器吞掉所有异常
def bad_error_handler(func):
    def wrapper(*args, **kwargs):
        try:
            return func(*args, **kwargs)
        except Exception:  # 吞掉所有异常！
            return None
    return wrapper

# 修复：只捕获预期的异常，其他异常向上传播
def good_error_handler(*expected_exceptions):
    import functools
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            try:
                return func(*args, **kwargs)
            except expected_exceptions as e:
                print(f"Handled expected error: {e}")
                return None
        return wrapper
    return decorator

@good_error_handler(ValueError, TypeError)
def safe_divide(a, b):
    return a / b

print(safe_divide(10, 0))  # ZeroDivisionError 仍然传播
```

### 18.5.5 装饰器与 pickling/序列化

**陷阱**：装饰后的函数无法被 pickle 序列化，因为闭包包含了不可序列化的引用：

```python
import pickle
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def my_func():
    return 42

# pickle.dumps(my_func)  # AttributeError: Can't pickle local object 'wrapper'

# 修复方案：使用 functools.partial 或在模块级别定义装饰器
# 或者使用支持序列化的装饰器库如 dill
```

### 18.5.6 最佳实践总结

| 实践 | 说明 |
|------|------|
| **始终使用 `@functools.wraps`** | 保留原始函数的 `__name__`、`__doc__`、`__module__` 等元信息 |
| **注意装饰器叠加顺序** | 先权限检查、再缓存、最后日志 |
| **类装饰器添加 `__get__`** | 使装饰器正确支持方法绑定 |
| **只捕获预期异常** | 避免吞掉未知异常 |
| **保持装饰器单一职责** | 每个装饰器只做一件事 |
| **装饰器应该是透明的** | 不改变被装饰函数的输入输出类型 |
| **使用 `__wrapped__` 提供内省** | 允许用户访问原始函数 |
| **避免在装饰器中持有大量状态** | 使用类装饰器时注意内存管理 |
| **带参数装饰器使用三层嵌套** | `def decorator_factory(args) -> def decorator(func) -> def wrapper(*args)` |
