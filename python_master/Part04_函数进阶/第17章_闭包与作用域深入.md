# 第17章 闭包与作用域深入

## 17.1 概念与语法

### 17.1.1 闭包的定义

闭包（Closure）是一个函数对象，它记住了定义时所在作用域的变量，即使该作用域已经结束执行。简而言之，闭包 = 函数 + 其引用的外部变量（自由变量，free variable）。

```python
def make_counter(start=0):
    count = start  # 自由变量
    def counter():
        nonlocal count
        count += 1
        return count
    return counter

c = make_counter()
print(c())  # 1
print(c())  # 2
```

### 17.1.2 LEGB 作用域规则

Python 变量查找遵循 LEGB 规则：

| 层级 | 作用域 | 说明 |
|------|--------|------|
| L | Local | 函数内部的局部作用域 |
| E | Enclosing | 外层嵌套函数的作用域 |
| G | Global | 模块级别的全局作用域 |
| B | Built-in | Python 内置名称空间 |

```python
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)    # Local 优先
    inner()

outer()  # 输出: local
```

### 17.1.3 global 与 nonlocal 声明

- **global**：在函数内声明变量为全局作用域，赋值会修改全局变量
- **nonlocal**：在嵌套函数中声明变量为外层（Enclosing）作用域，赋值会修改闭包变量

```python
x = 10

def outer():
    y = 20
    def inner():
        nonlocal y   # 引用外层函数的 y
        global x      # 引用全局的 x
        x = 100
        y = 200
    inner()
    print(f"outer: y={y}")  # y 被修改为 200
    return inner

f = outer()  # outer: y=200
print(f"x={x}")  # x=100
```

### 17.1.4 闭包的形成条件

闭包形成的三个必要条件：

1. 存在嵌套函数（函数内定义函数）
2. 内层函数引用了外层函数的变量
3. 内层函数被返回（或被传递到外层函数之外）

## 17.2 原理与机制

### 17.2.1 cell 对象：闭包的底层实现

当 Python 编译器检测到一个变量既在内层函数中被引用，又在外层函数中被赋值或引用时，它会将该变量标记为 **cell variable**，并在运行时创建一个 `cell` 对象来共享这个变量。

CPython 源码中 cell 对象的定义（`Objects/cellobject.c`）：

```c
typedef struct {
    PyObject_HEAD
    PyObject *ob_ref;   // cell 引用的实际对象
} PyCellObject;
```

cell 对象是一个间接引用层：外层函数和内层函数都持有对同一个 cell 对象的引用，而 cell 对象内部持有对实际值对象的引用。这种设计使得闭包变量可以在多个作用域间共享修改。

用 `__closure__` 属性验证：

```python
def make_adder(n):
    def adder(x):
        return x + n
    return adder

f = make_adder(10)
print(f.__closure__)      # (<cell at 0x...: int object at 0x...>,)
print(f.__closure__[0])   # <cell at 0x...: int object at 0x...>
print(f.__closure__[0].cell_contents)  # 10
```

### 17.2.2 闭包变量的字节码分析

通过 `dis` 模块可以观察闭包变量在字节码层面的操作：

```python
import dis

def make_adder(n):
    def adder(x):
        return x + n
    return adder

dis.dis(make_adder)
```

关键字节码指令：

| 指令 | 含义 |
|------|------|
| `LOAD_DEREF` | 从 cell 对象加载闭包变量 |
| `STORE_DEREF` | 将值存储到 cell 对象 |
| `LOAD_CLOSURE` | 加载 cell 对象本身（用于构建内层函数的闭包） |

```python
# adder 函数的字节码中会出现：
# LOAD_FAST    0 (x)
# LOAD_DEREF   0 (n)     <-- 闭包变量通过 LOAD_DEREF 访问
# BINARY_ADD
# RETURN_VALUE
```

### 17.2.3 函数对象的 `__code__` 属性

函数对象包含多个与闭包相关的属性：

```python
def outer(x):
    y = x * 2
    def inner():
        return y
    return inner

f = outer(5)
print(f.__code__.co_freevars)    # ('y',) - 自由变量名
print(f.__code__.co_cellvars)     # () - 内层无嵌套，无 cell 变量
print(outer.__code__.co_cellvars) # ('y',) - 外层函数的 cell 变量
print(outer.__code__.co_freevars) # () - 外层函数无自由变量
```

- **co_freevars**：内层函数引用但不属于自己局部作用域的变量名元组
- **co_cellvars**：外层函数中被内层函数引用的局部变量名元组

### 17.2.4 作用域与闭包的编译期决策

Python 的作用域决策发生在 **编译期**，而非运行期。编译器通过符号表分析确定每个变量的作用域类型：

1. 如果变量在函数体内被赋值（包括 `=` 赋值、`for` 循环变量、`def` 定义等），编译器将其标记为局部变量
2. 如果变量被内层函数引用且在本函数中被赋值/引用，标记为 cell 变量
3. 如果变量只被引用但从未被赋值，编译器将其视为自由变量（free variable），从外层作用域查找

```python
def demo():
    x = 1       # 局部变量
    y = 2       # cell 变量（被内层函数引用）
    def inner():
        print(y) # 自由变量 y
    return inner
```

### 17.2.5 nonlocal 的底层机制

`nonlocal` 声明告诉编译器：不要将变量当作局部变量，而是从 Enclosing 作用域中查找。在字节码层面，`nonlocal` 变量的访问使用 `LOAD_DEREF` / `STORE_DEREF`，与闭包变量共享同一条访问路径。

```python
import dis

def outer():
    count = 0
    def inner():
        nonlocal count
        count += 1
        return count
    return inner

dis.dis(inner := outer())
# LOAD_DEREF   0 (count)    <-- 通过 cell 对象读取
# LOAD_CONST    1 (1)
# BINARY_ADD
# STORE_DEREF  0 (count)    <-- 通过 cell 对象写入
```

`nonlocal` 和闭包的 cell 机制是同一套底层实现——没有 `nonlocal`，内层函数只能读取闭包变量而不能赋值（赋值会创建新的局部变量，遮蔽外层变量）。

## 17.3 使用场景

### 17.3.1 装饰器基础

闭包是装饰器的实现基础。装饰器本质上就是一个返回闭包的高阶函数：

```python
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} took {elapsed:.6f}s")
        return result
    return wrapper

@timer
def slow_add(a, b):
    import time
    time.sleep(0.1)
    return a + b

result = slow_add(1, 2)  # slow_add took 0.1...s
```

### 17.3.2 回调函数与事件处理

闭包天然适合封装回调函数的上下文信息：

```python
def create_button_handler(button_id, callback):
    """工厂函数，为不同按钮创建独立的点击处理器"""
    click_count = [0]  # 用列表包装可变状态

    def handler(event):
        click_count[0] += 1
        print(f"Button {button_id}: click #{click_count[0]}")
        if callback:
            callback(event, click_count[0])

    return handler

# 每个按钮有独立的 click_count
button_a_handler = create_button_handler("A", lambda e, c: None)
button_b_handler = create_button_handler("B", lambda e, c: None)
```

### 17.3.3 工厂模式

闭包可以替代类来实现轻量级的工厂模式：

```python
def make_validator(min_len=0, max_len=100, pattern=None):
    """创建一个配置化的字符串验证器"""
    import re
    regex = re.compile(pattern) if pattern else None

    def validator(value):
        if not isinstance(value, str):
            return False, "value must be a string"
        if len(value) < min_len:
            return False, f"length must be >= {min_len}"
        if len(value) > max_len:
            return False, f"length must be <= {max_len}"
        if regex and not regex.match(value):
            return False, f"value does not match pattern {pattern}"
        return True, "valid"

    return validator

# 创建不同的验证器
email_validator = make_validator(5, 100, r'^[^@]+@[^@]+\.[^@]+$')
username_validator = make_validator(3, 20, r'^[a-zA-Z0-9_]+$')

print(email_validator("user@example.com"))   # (True, 'valid')
print(username_validator("ab"))               # (False, 'length must be >= 3')
```

### 17.3.4 延迟计算与惰性求值

闭包可以实现延迟计算，只在真正需要时才执行计算：

```python
def lazy_compute(compute_fn, *args, **kwargs):
    """延迟计算：只在访问 .value 时才执行"""
    _result = [None]
    _computed = [False]

    def get_value():
        if not _computed[0]:
            _result[0] = compute_fn(*args, **kwargs)
            _computed[0] = True
        return _result[0]

    class LazyValue:
        @property
        def value(self):
            return get_value()

        def __repr__(self):
            return f"LazyValue(computed={_computed[0]})"

    return LazyValue()

import time
expensive = lazy_compute(time.sleep, 1)  # 尚未执行
print(expensive)                           # LazyValue(computed=False)
result = expensive.value                   # 此时才执行 sleep(1)
print(expensive)                           # LazyValue(computed=True)
```

### 17.3.5 状态机与有状态计算

闭包可以封装状态机逻辑，不需要定义完整的类：

```python
def make_traffic_light(initial="red"):
    """交通灯状态机"""
    transitions = {
        "red": "green",
        "green": "yellow",
        "yellow": "red",
    }
    state = initial

    def advance():
        nonlocal state
        state = transitions[state]
        return state

    def current():
        return state

    def reset(new_state="red"):
        nonlocal state
        state = new_state

    class LightController:
        advance = advance
        current = current
        reset = reset

    return LightController()

light = make_traffic_light()
print(light.current())   # red
print(light.advance())   # green
print(light.advance())   # yellow
print(light.advance())   # red
light.reset("green")
print(light.current())   # green
```

### 示例 2：nonlocal 与可变闭包状态

```python
"""
示例：使用 nonlocal 修改闭包状态，
对比不使用 nonlocal 时的行为差异
"""

import dis

def counter_with_nonlocal(start=0):
    """使用 nonlocal 修改闭包变量"""
    count = start

    def increment():
        nonlocal count
        count += 1
        return count

    return increment


def counter_without_nonlocal(start=0):
    """不使用 nonlocal——会导致 UnboundLocalError"""
    count = start

    def increment():
        # count += 1  # 这行会导致 UnboundLocalError
        # 因为编译器看到赋值，将 count 视为局部变量，
        # 但读取时局部变量尚未赋值
        pass

    return increment


# 正常工作的计数器
c = counter_with_nonlocal()
print(f"c() = {c()}")  # 1
print(f"c() = {c()}")  # 2
print(f"c() = {c()}")  # 3

# 观察字节码中 nonlocal 变量的访问方式
print("\n--- increment 函数字节码（使用 nonlocal）---")
dis.dis(counter_with_nonlocal.__code__.co_consts[1])
# LOAD_DEREF + LOAD_CONST + BINARY_ADD + STORE_DEREF
```

### 示例 3：循环变量延迟绑定陷阱与修复

```python
"""
示例：经典的循环变量延迟绑定陷阱及其多种修复方案
"""

# ---- 陷阱：延迟绑定 ----
def create_buggy_funcs():
    """所有函数都返回 4——延迟绑定的经典问题"""
    funcs = []
    for i in range(5):
        def func():
            return i  # 闭包捕获的是变量 i，而非 i 的值
        funcs.append(func)
    return funcs

buggy = create_buggy_funcs()
print("--- 延迟绑定陷阱 ---")
for f in buggy:
    print(f"  {f()}", end="")
# 输出: 4 4 4 4 4（全部返回最终值）

# ---- 修复方案 1：默认参数立即绑定 ----
def create_fixed_funcs_default_arg():
    """使用默认参数在定义时立即绑定值"""
    funcs = []
    for i in range(5):
        def func(i=i):  # 默认参数在函数定义时求值
            return i
        funcs.append(func)
    return funcs

fixed1 = create_fixed_funcs_default_arg()
print("\n--- 修复方案 1：默认参数 ---")
for f in fixed1:
    print(f"  {f()}", end="")
# 输出: 0 1 2 3 4

# ---- 修复方案 2：工厂函数 ----
def create_fixed_funcs_factory():
    """使用工厂函数创建独立的闭包"""
    funcs = []
    for i in range(5):
        def make_func(val):
            def func():
                return val  # val 是 make_func 的局部变量
            return func
        funcs.append(make_func(i))
    return funcs

fixed2 = create_fixed_funcs_factory()
print("\n--- 修复方案 2：工厂函数 ---")
for f in fixed2:
    print(f"  {f()}", end="")
# 输出: 0 1 2 3 4

# ---- 修复方案 3：lambda + 默认参数 ----
def create_fixed_funcs_lambda():
    """lambda 表达式 + 默认参数"""
    return [lambda i=i: i for i in range(5)]

fixed3 = create_fixed_funcs_lambda()
print("\n--- 修复方案 3：lambda + 默认参数 ---")
for f in fixed3:
```

### 示例 4：闭包与内存——引用追踪

```python
"""
示例：观察闭包对变量的引用保持，
以及 del/弱引用对内存的影响
"""

import sys
import weakref
import gc


def create_closure_with_large_data():
    """闭包持有对大对象的引用"""
    large_data = list(range(1_000_000))  # 约 8MB

    def process():
        return len(large_data)

    return process, large_data


# 闭包保持引用
func, data = create_closure_with_large_data()
print(f"数据引用计数: {sys.getrefcount(data) - 1}")  # 2（func闭包 + data变量）

# 删除一个引用后，闭包仍然持有
del data
gc.collect()
print(f"闭包仍可访问数据，函数返回: {func()}")

# 检查闭包中 cell 对象的内容
print(f"闭包 cell 内容长度: {len(func.__closure__[0].cell_contents)}")


# ---- 弱引用方案：避免闭包强引用 ----
def create_weakref_closure(data_ref):
    """使用弱引用避免闭包强持有对象"""
    def process():
        data = data_ref()
        if data is None:
            raise ValueError("Referenced object has been garbage collected")
        return len(data)
    return process


large_list = list(range(1_000_000))
weak_ref = weakref.ref(large_list)
weak_closure = create_weakref_closure(weak_ref)

print(f"\n弱引用闭包可用: {weak_closure()}")
del large_list
gc.collect()
try:
    weak_closure()
except ValueError as e:
    print(f"弱引用闭包检测到对象已回收: {e}")
```

### 示例 5：闭包在装饰器/回调/工厂模式中的综合应用

```python
"""
示例：使用闭包实现一个简单的任务调度器，
综合演示装饰器、回调、工厂模式
"""

import time
from collections import defaultdict


def create_task_scheduler():
    """工厂模式：创建任务调度器"""
    tasks = defaultdict(list)        # 闭包状态 1
    results = {}                      # 闭包状态 2
    task_counter = [0]                 # 闭包状态 3

    def register(task_type):
        """装饰器：注册任务处理器"""
        def decorator(func):
            tasks[task_type].append(func)
            func._task_type = task_type
            return func
        return decorator

    def execute(task_type, *args, **kwargs):
        """执行指定类型的所有任务"""
        if task_type not in tasks:
            raise ValueError(f"Unknown task type: {task_type}")

        task_results = []
        for task in tasks[task_type]:
            task_id = task_counter[0]
            task_counter[0] += 1

            start = time.perf_counter()
            result = task(*args, **kwargs)
            elapsed = time.perf_counter() - start

            task_results.append({
                "task_id": task_id,
                "result": result,
                "elapsed": elapsed,
            })
            results[task_id] = result

        return task_results

    def get_results():
        """获取所有执行结果"""
        return dict(results)

    # 返回具有命名方法的命名空间对象
    class Scheduler:
        register = staticmethod(register)
        execute = execute
        get_results = get_results

    return Scheduler()


# 使用调度器
scheduler = create_task_scheduler()


@scheduler.register("math")
def add(a, b):
    return a + b


@scheduler.register("math")
def multiply(a, b):
    return a * b


@scheduler.register("string")
def concatenate(a, b):
    return f"{a}{b}"


# 执行任务
print("--- 执行 math 任务 ---")
math_results = scheduler.execute("math", 3, 4)
for r in math_results:
    print(f"  Task {r['task_id']}: result={r['result']}, time={r['elapsed']:.6f}s")

print("\n--- 执行 string 任务 ---")
str_results = scheduler.execute("string", "hello", "world")
for r in str_results:
    print(f"  Task {r['task_id']}: result={r['result']}, time={r['elapsed']:.6f}s")

print(f"\n所有结果: {scheduler.get_results()}")
```

## 17.5 常见陷阱与最佳实践

### 17.5.1 循环变量延迟绑定

**陷阱**：在循环中创建闭包时，所有闭包共享同一个循环变量的最终值。

```python
# 陷阱
funcs = [lambda: i for i in range(5)]
print([f() for f in funcs])  # [4, 4, 4, 4, 4]

# 修复：默认参数立即求值
funcs = [lambda i=i: i for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]
```

**原理**：闭包捕获的是变量的引用（cell 对象），而非值的副本。循环结束后，所有 lambda 都引用同一个 `i`，其最终值为 4。

### 17.5.2 闭包与可变默认参数的交互

**陷阱**：可变默认参数与闭包结合时，行为可能出人意料：

```python
def make_appender(value, target=[]):
    """可变默认参数与闭包的交互"""
    def append():
        target.append(value)
        return target
    return append

# 所有 appender 共享同一个 list！
a = make_appender(1)
b = make_appender(2)
print(a())  # [1]
print(b())  # [1, 2] —— 共享同一个 list！

# 修复：不使用可变默认参数
def make_appender_fixed(value, target=None):
    if target is None:
        target = []
    def append():
        target.append(value)
        return target
    return append
```

### 17.5.3 闭包导致的内存泄漏

**陷阱**：闭包会持有对其引用的所有变量的强引用，阻止垃圾回收。

```python
import sys

def create_leaky_closure():
    huge_data = "x" * 10_000_000
    result = len(huge_data)

    def small_func():
        return result  # 只需要 result，但闭包持有整个帧

    return small_func

f = create_leaky_closure()
# huge_data 无法被回收——闭包的 __closure__ 通过 cell 引用了整个帧
```

**最佳实践**：只捕获必要的值，避免闭包意外持有大数据：

```python
def create_efficient_closure():
    huge_data = "x" * 10_000_000
    result = len(huge_data)  # 立即计算
    del huge_data             # 显式删除不再需要的大对象

    def small_func():
        return result

    return small_func
```

### 17.5.4 nonlocal 与多层嵌套的歧义

**陷阱**：多层嵌套时，`nonlocal` 只向上查找一层 Enclosing 作用域：

```python
def level1():
    x = 1
    def level2():
        x = 2
        def level3():
            nonlocal x  # 修改的是 level2 的 x，而非 level1 的
            x = 3
        level3()
        print(f"level2: x = {x}")  # x = 3
    level2()
    print(f"level1: x = {x}")      # x = 1（未被修改！）

level1()
# 输出:
# level2: x = 3
# level1: x = 1
```

**最佳实践**：避免深层嵌套，必要时使用类或字典替代多层 nonlocal。

### 17.5.5 闭包中的变量遮蔽

**陷阱**：在内层函数中对变量赋值，会创建新的局部变量而非修改闭包变量：

```python
def outer():
    value = 10
    def inner():
        value = 20    # 创建了新的局部变量，遮蔽了外层的 value
        print(f"inner: {value}")  # 20
    inner()
    print(f"outer: {value}")      # 10（未被修改）

outer()

# 修复：使用 nonlocal
def outer_fixed():
    value = 10
    def inner():
        nonlocal value
        value = 20
        print(f"inner: {value}")  # 20
    inner()
    print(f"outer: {value}")      # 20

outer_fixed()
```

### 17.5.6 最佳实践总结

| 实践 | 说明 |
|------|------|
| **优先使用默认参数绑定** | 循环中创建闭包时，用 `lambda i=i: i` 立即绑定 |
| **避免深层嵌套** | 超过 2 层嵌套时，考虑重构为类 |
| **显式删除不需要的大对象** | `del` 不再需要的大数据，防止闭包意外持有 |
| **使用 `__closure__` 调试** | 检查闭包捕获了哪些变量 |
| **理解编译期决策** | 变量作用域在编译期确定，运行时无法改变 |
| **用类替代复杂状态** | 当闭包需要管理多个可变状态时，使用类更清晰 |
| **使用 `nonlocal` 时保持谨慎** | 确保 nonlocal 变量的生命周期和意图清晰 |
| **优先使用不可变数据** | 闭包中优先使用不可变数据，减少意外的状态修改 |
