# 第20章 协程演进

## 20.1 概念与语法

### 20.1.1 协程的定义

协程（Coroutine）是一种比线程更轻量的并发原语。与线程由操作系统内核调度不同，协程在用户空间通过协作式调度实现切换——协程主动让出控制权（`yield` / `await`），而非被抢占。

Python 协程演进经历了三个阶段：

| 阶段 | 版本 | 机制 | 特点 |
|------|------|------|------|
| 基于生成器的协程 | 2.5+ | `yield` + `send()` | 与生成器语法混用，容易混淆 |
| `yield from` 协程 | 3.3+ | `yield from` 委托 | 改进子生成器委托，但仍与生成器共用类型 |
| 原生协程 | 3.5+ | `async`/`await` | 独立类型 `coroutine`，语法清晰 |

### 20.1.2 基于生成器的协程（Generator-based Coroutine）

Python 2.5 引入 `send()` 和 `yield` 表达式，使生成器可以接收外部值，从而具备了协程的基本能力：

```python
def coroutine_based():
    """基于生成器的协程"""
    print("Coroutine started")
    value = None
    while True:
        # yield 暂停执行，send() 恢复并传入值
        value = yield value
        if value is None:
            break
        print(f"Received: {value}")
        value = value * 2
    print("Coroutine finished")

# 使用
co = coroutine_based()
next(co)           # 启动协程（prime），输出 "Coroutine started"
print(co.send(10)) # Received: 10 → 20
print(co.send(20)) # Received: 20 → 40
co.close()
```

### 20.1.3 yield from 委托（PEP 380）

Python 3.3 引入 `yield from`，解决了生成器协程的委托问题：

```python
def sub_coroutine():
    """子协程"""
    result = 0
    while True:
        x = yield result
        if x is None:
            break
        result += x
    return result  # 返回值传递给 yield from

def main_coroutine():
    """主协程通过 yield from 委托给子协程"""
    total = yield from sub_coroutine()  # 接收子协程的返回值
    print(f"Total from sub: {total}")
    yield "done"

co = main_coroutine()
next(co)        # 启动
co.send(10)     # 子协程收到 10
co.send(20)     # 子协程收到 20
co.send(30)     # 子协程收到 30
try:
    co.send(None)  # 子协程结束，返回 60
except StopIteration:
    pass
# 输出: Total from sub: 60
```

### 20.1.4 async/await 语法（PEP 492）

Python 3.5 引入原生协程语法，使协程与生成器彻底分离：

```python
import asyncio

async def fetch_data(url):
    """原生协程——使用 async def 定义"""
    print(f"Fetching {url}")
    await asyncio.sleep(1)  # await 暂停协程，等待异步操作
    print(f"Done: {url}")
    return f"data from {url}"

async def main():
    """原生协程入口"""
    result = await fetch_data("http://example.com")
    print(result)

# 运行协程
asyncio.run(main())
```

### 20.1.5 原生协程 vs 基于生成器的协程

| 特性 | 生成器协程 | 原生协程 |
|------|-----------|----------|
| 定义方式 | `def` + `yield` | `async def` |
| 暂停方式 | `yield` / `yield from` | `await` |
| 返回类型 | `generator` | `coroutine` |
| 是否可迭代 | 是（`for x in gen`） | 否 |
| 调试 | 与普通生成器混淆 | 类型独立，易于区分 |
| 异常传播 | `throw()` / `close()` | 自动传播 |

```python
# 生成器协程
def gen_coroutine():
    yield 1

# 原生协程
async def native_coroutine():
    await asyncio.sleep(1)

print(type(gen_coroutine()))   # <class 'generator'>
print(type(native_coroutine())) # <class 'coroutine'>

# 原生协程不可迭代
# for x in native_coroutine():  # TypeError: 'coroutine' object is not iterable
```

## 20.2 原理与机制

### 20.2.1 协程的 CPython 底层数据结构

原生协程在 CPython 中由 `PyCoroObject` 实现（`Include/cpython/coroutine.h`）：

```c
typedef struct {
    PyObject_HEAD                  // PyObject 头部
    PyFrameObject *cr_frame;       // 协程帧（执行上下文）
    PyObject *cr_code;              // 代码对象
    PyObject *cr_name;              // 协程名称
    PyObject *cr_qualname;          // 限定名
    PyObject *cr_origin;            // 创建位置信息
    int cr_running;                  // 是否正在执行
    PyObject *cr_await;             // 当前 await 的对象
    PyObject *cr_exc_state;         // 异常状态
} PyCoroObject;
```

与 `PyGenObject`（生成器）的关键区别：

| 字段 | 生成器 | 协程 |
|------|--------|------|
| 帧字段 | `gi_frame` | `cr_frame` |
| 运行标志 | `gi_running` | `cr_running` |
| 委托目标 | `gi_yieldfrom` | `cr_await` |
| 类型标记 | `PyGenObject` | `PyCoroObject` |
| 可迭代 | 是 | 否 |

```python
async def demo_coro():
    await asyncio.sleep(0.1)
    return 42

coro = demo_coro()
print(type(coro))          # <class 'coroutine'>
print(coro.cr_frame)       # <frame at ...>
print(coro.cr_running)     # False
print(coro.cr_code)        # <code object demo_coro ...>
print(coro.cr_await)       # None（尚未 await）

# 必须通过 asyncio.run() 或 await 执行
try:
    next(coro)  # TypeError: 'coroutine' object is not an iterator
except TypeError as e:
    print(e)
```

### 20.2.2 async/await 的字节码

`async def` 和 `await` 有专属的字节码指令：

```python
import dis

async def demo():
    result = await some_async_op()
    return result

dis.dis(demo)
# RESUME                       0
# LOAD_GLOBAL                  some_async_op
# CALL_FUNCTION                0
# GET_AWAITABLE                0
# LOAD_CONST                   None
# YIELD_VALUE                  2       <-- 暂停协程
# RESUME                       3
# STORE_FAST                   result
# LOAD_FAST                    result
# RETURN_VALUE
```

关键指令对比：

| 指令 | 生成器 | 协程 |
|------|--------|------|
| 暂停 | `YIELD_VALUE` | `YIELD_VALUE`（相同机制） |
| 获取可等待对象 | — | `GET_AWAITABLE` |
| 委托 | `YIELD_FROM` | `YIELD_VALUE` + `RESUME` |
| 恢复 | `RESUME` | `RESUME` |

`GET_AWAITABLE` 的作用：将 `await` 的对象转换为可等待对象。如果对象是协程，直接返回；如果对象有 `__await__` 方法，调用它返回迭代器；否则抛出 `TypeError`。

### 20.2.3 asyncio 事件循环机制

`asyncio` 的核心是事件循环（Event Loop），它负责调度和执行协程。事件循环的工作流程：

```
1. 将协程封装为 Task
2. 运行 Task → 协程执行到 await
3. await 挂起协程，注册回调（如 I/O 完成回调）
4. 事件循环检查其他就绪的 Task
5. 当 await 的操作完成，恢复挂起的协程
6. 重复直到所有 Task 完成
```

CPython 中 `Task` 的核心逻辑（简化自 `Lib/asyncio/tasks.py`）：

```python
class Task(futures._PyFuture):
    def __init__(self, coro, *, loop=None):
        super().__init__(loop=loop)
        self._coro = coro  # 存储协程对象

    def __step(self, exc=None):
        """执行协程的一步"""
        try:
            if exc is None:
                result = self._coro.send(None)  # 恢复协程
            else:
                result = self._coro.throw(type(exc), exc, exc.__traceback__)
        except StopIteration as si:
            # 协程正常完成
            self.set_result(si.value)
        except CancelledError:
            # 协程被取消
            self.cancel()
        except Exception as e:
            # 协程抛出异常
            self.set_exception(e)
        else:
            # result 是一个 Future，注册回调等待其完成
            result.add_done_callback(self.__step)
```

### 20.2.4 awaitable 协议

`await` 操作符要求对象实现 `__await__()` 方法（awaitable 协议）：

```python
class AwaitableWrapper:
    """自定义 awaitable 对象"""

    def __init__(self, value):
        self.value = value

    def __await__(self):
        """__await__ 必须返回一个迭代器"""
        yield  # 暂停点
        return self.value

async def use_awaitable():
    result = await AwaitableWrapper(42)
    print(f"Got: {result}")

asyncio.run(use_awaitable())
# 输出: Got: 42
```

CPython 中 `GET_AWAITABLE` 指令的逻辑：

```c
// 简化自 Python/ceval.c
PyObject *awaitable = TOP();
if (PyCoro_CheckExact(awaitable)) {
    // 原生协程，直接返回
} else if (Py_TYPE(awaitable)->tp_as_async &&
           Py_TYPE(awaitable)->tp_as_async->am_await) {
    // 有 __await__ 方法，调用并返回迭代器
    iter = Py_TYPE(awaitable)->tp_as_async->am_await(awaitable);
} else {
    // 既不是协程也没有 __await__，TypeError
    PyErr_Format(PyExc_TypeError,
                 "object %s can't be used in 'await' expression",
                 Py_TYPE(awaitable)->tp_name);
}
```

### 20.2.5 协程的创建、调度与销毁

完整生命周期：

```
创建 → 首次 await → 暂停 → 恢复 → ... → 完成/异常
  │                    │         │
  │                    │         └─ Task.__step() 调用 coro.send()
  │                    └─ await 将控制权交给事件循环
  └─ asyncio.create_task() 将协程封装为 Task
```

```python
import asyncio

async def lifecycle_demo():
    """演示协程生命周期"""
    print("1. 协程启动")
    await asyncio.sleep(0)  # 让出控制权
    print("2. 第一次恢复")
    await asyncio.sleep(0)  # 再次让出
    print("3. 第二次恢复")
    return "完成"

async def main():
    task = asyncio.create_task(lifecycle_demo())
    print(f"Task 状态: {task.done()}")  # False

    result = await task
    print(f"Task 结果: {result}")
    print(f"Task 状态: {task.done()}")  # True

asyncio.run(main())
```

### 20.2.6 从 yield from 到 await 的语义对应

`await` 是 `yield from` 在异步上下文中的等价物：

```python
# 基于生成器的协程（Python 3.3-3.4）
@asyncio.coroutine
def old_style():
    result = yield from some_async_op()
    return result

# 原生协程（Python 3.5+）
async def new_style():
    result = await some_async_op()
    return result
```

`yield from` 和 `await` 在 CPython 层面共享 `YIELD_VALUE` 指令，但 `await` 增加了 `GET_AWAITABLE` 校验，确保被等待的对象是 awaitable。

## 20.3 使用场景

### 20.3.1 异步 I/O 操作

协程最常见的场景是处理高并发 I/O：

```python
import asyncio
import time

async def fetch_url(url, delay=1.0):
    """模拟异步 HTTP 请求"""
    print(f"开始请求: {url}")
    await asyncio.sleep(delay)  # 模拟网络延迟
    print(f"完成请求: {url}")
    return f"<html>{url}</html>"

async def sequential_fetch():
    """串行请求——总耗时等于所有延迟之和"""
    start = time.perf_counter()
    r1 = await fetch_url("http://a.com", 1.0)
    r2 = await fetch_url("http://b.com", 1.0)
    r3 = await fetch_url("http://c.com", 1.0)
    elapsed = time.perf_counter() - start
    print(f"串行总耗时: {elapsed:.2f}s")
    return [r1, r2, r3]

async def concurrent_fetch():
    """并发请求——总耗时等于最长延迟"""
    start = time.perf_counter()
    results = await asyncio.gather(
        fetch_url("http://a.com", 1.0),
        fetch_url("http://b.com", 1.0),
        fetch_url("http://c.com", 1.0),
    )
    elapsed = time.perf_counter() - start
    print(f"并发总耗时: {elapsed:.2f}s")
    return results

# 串行: ~3s，并发: ~1s
asyncio.run(sequential_fetch())
asyncio.run(concurrent_fetch())
```

```python
class AsyncDBConnection:
```

### 20.3.3 异步迭代器

```python
import asyncio

class AsyncPageReader:
    """异步分页读取器"""

    def __init__(self, data, page_size=3):
        self.data = data
        self.page_size = page_size
        self.index = 0

    def __aiter__(self):
        return self

    async def __anext__(self):
        await asyncio.sleep(0.01)  # 模拟异步读取
        if self.index >= len(self.data):
            raise StopAsyncIteration
        page = self.data[self.index:self.index + self.page_size]
        self.index += self.page_size
        return page

async def main():
    data = list(range(1, 16))  # [1, 2, ..., 15]
    async for page in AsyncPageReader(data, page_size=5):
        print(f"Page: {page}")

# Page: [6, 7, 8, 9, 10]
# Page: [11, 12, 13, 14, 15]
```

### 20.3.4 超时与取消

```python
import asyncio

async def slow_operation():
    """模拟一个可能超时的慢操作"""
    await asyncio.sleep(10)
    return "done"

async def main():
    # asyncio.wait_for 设置超时

    task = asyncio.create_task(slow_operation())

    try:
        await task
    except asyncio.CancelledError:
        print("任务已取消")

asyncio.run(main())
```

### 20.3.5 协程在 Web 框架中的应用

```python
import asyncio

class SimpleAsyncWebFramework:
    """极简异步 Web 框架演示"""

    def __init__(self):
        self._routes = {}
        self._middleware = []

    def route(self, path, methods=None):
        methods = methods or ["GET"]
        def decorator(func):
            for method in methods:
                self._routes[(method.upper(), path)] = func
            return func
        return decorator

    def middleware(self, func):
        self._middleware.append(func)
        return func

    async def handle_request(self, method, path, **kwargs):
        """异步请求处理"""
        # 执行中间件
        for mw in self._middleware:
            result = await mw(method, path)
            if result is not None:
                return result

        # 查找路由
        handler = self._routes.get((method.upper(), path))
        if handler is None:
            return {"status": 404, "body": "Not Found"}

        # 执行异步处理函数
        result = await handler(**kwargs)
        return {"status": 200, "body": result}


app = SimpleAsyncWebFramework()


@app.middleware
async def log_middleware(method, path):
    print(f"[Middleware] {method} {path}")
    return None  # 返回 None 继续处理


@app.route("/api/users", methods=["GET"])
async def list_users():
    await asyncio.sleep(0.01)  # 模拟异步数据库查询
    return ["Alice", "Bob", "Charlie"]


@app.route("/api/users/<id>", methods=["GET"])
async def get_user(user_id):
    await asyncio.sleep(0.01)
    return {"id": user_id, "name": "Alice"}


async def main():
    response = await app.handle_request("GET", "/api/users")
    print(f"Response: {response}")

    response = await app.handle_request("GET", "/api/users/<id>", user_id=42)
    print(f"Response: {response}")

asyncio.run(main())
```

## 20.4 示例代码

### 示例 1：从生成器到原生协程的完整演进

```python
"""
示例：展示同一个任务用三种方式实现，
对比生成器协程、yield from 协程、原生协程
"""
import asyncio
import time


# ---- 方式 1：基于生成器的协程（Python 2.5+）----
def gen_coroutine_task(name, delay):
    """最原始的生成器协程"""
    print(f"[{name}] 开始")
    start = time.time()
    while time.time() - start < delay:
        time.sleep(0.1)
        yield  # 让出控制权
    print(f"[{name}] 完成")
    return f"{name} done"


def gen_scheduler(*coros):
    """简单的生成器调度器"""
    results = {}
    while coros:
        remaining = []
        for co in coros:
            try:
                next(co)
                remaining.append(co)
            except StopIteration as e:
                results[co] = e.value
        coros = remaining
    return results


print("=== 生成器协程 ===")
results = gen_scheduler(
    gen_coroutine_task("A", 0.3),
    gen_coroutine_task("B", 0.2),
)
print(f"结果: {results}")


# ---- 方式 2：yield from 协程（Python 3.3+）----
def yield_from_task(name, delay):
    """使用 yield from 的协程"""
    print(f"[{name}] 开始")
    yield from asyncio.sleep(delay)  # 委托给 asyncio
    print(f"[{name}] 完成")
    return f"{name} done"


# ---- 方式 3：原生协程（Python 3.5+）----
async def native_coroutine_task(name, delay):
    """原生协程——最推荐的写法"""
    print(f"[{name}] 开始")
    await asyncio.sleep(delay)
    print(f"[{name}] 完成")
    return f"{name} done"


async def main():
    print("\n=== 原生协程 ===")
    results = await asyncio.gather(
        native_coroutine_task("A", 0.3),
        native_coroutine_task("B", 0.2),
    )
    print(f"结果: {results}")


asyncio.run(main())
```

### 示例 2：协程调度器底层实现

```python
"""
示例：手动实现一个简单的协程调度器，
理解事件循环的核心机制
"""
import time
from collections import deque


class SimpleEventLoop:
    """极简事件循环——理解协程调度的核心原理"""

    def __init__(self):
        self._ready = deque()    # 就绪队列
        self._current = None     # 当前运行的协程

    def create_task(self, coro):
        """将协程封装为 Task 并加入就绪队列"""
        task = Task(coro, loop=self)
        self._ready.append(task)
        return task

    def run(self):
        """运行事件循环直到所有任务完成"""
        while self._ready:
            task = self._ready.popleft()
            self._current = task
            task.step()
            if not task.done:
                self._ready.append(task)


class Task:
    """简化版 Task——管理协程的执行状态"""

    def __init__(self, coro, loop):
        self._coro = coro
        self._loop = loop
        self._result = None
        self._done = False
        self._exception = None

    @property
    def done(self):
        return self._done

    def step(self):
        """执行协程的一步"""
        try:
            # 发送 None 恢复协程
            result = self._coro.send(None)
            # result 是 SleepCommand，稍后再恢复
            if isinstance(result, SleepCommand):
                pass  # 简化处理：直接放回队列
        except StopIteration as e:
            self._result = e.value
            self._done = True
            print(f"  Task 完成: result = {self._result}")
        except Exception as e:
            self._exception = e
            self._done = True
            print(f"  Task 异常: {e}")


class SleepCommand:
    """模拟 asyncio.sleep 的命令对象"""
    def __await__(self):
        yield self  # 暂停点
        return None


async def simple_task(name, count):
    """简单的异步任务"""
    for i in range(count):
        print(f"  [{name}] 步骤 {i+1}/{count}")
        await SleepCommand()  # 让出控制权
    return f"{name} done"


# 运行
loop = SimpleEventLoop()
loop.create_task(simple_task("Task-A", 3))
loop.create_task(simple_task("Task-B", 2))
loop.run()
```

### 示例 3：asyncio 核心并发模式

```python
"""
示例：asyncio 的核心并发模式——
gather、wait、create_task、Queue
"""
import asyncio
import random


async def worker(worker_id, queue):
    """异步工作协程——从队列获取任务"""
    while True:
        task_id = await queue.get()
        if task_id is None:
            break
        print(f"  Worker-{worker_id} 处理 Task-{task_id}")
        await asyncio.sleep(random.uniform(0.05, 0.2))  # 模拟处理
        queue.task_done()
        print(f"  Worker-{worker_id} 完成 Task-{task_id}")


async def producer(queue, count):
    """异步生产者——向队列添加任务"""
    for i in range(count):
        print(f"  Producer 添加 Task-{i}")
        await queue.put(i)
        await asyncio.sleep(0.02)


async def gather_demo():
    """asyncio.gather：并发等待多个协程"""
    print("=== gather Demo ===")
    results = await asyncio.gather(
        asyncio.sleep(0.1, result="A"),
        asyncio.sleep(0.2, result="B"),
        asyncio.sleep(0.05, result="C"),
    )
    print(f"  gather 结果: {results}")


async def wait_demo():
    """asyncio.wait：更灵活的等待"""
    print("=== wait Demo ===")
    tasks = [
        asyncio.create_task(asyncio.sleep(i * 0.1, result=f"Task-{i}"))
        for i in range(3)
    ]
    done, pending = await asyncio.wait(
        tasks,
        timeout=0.25,          # 超时
        return_when=asyncio.ALL_COMPLETED  # 全部完成
    )
    print(f"  完成数: {len(done)}, 待定数: {len(pending)}")
    for task in done:
        print(f"    结果: {task.result()}")


async def queue_demo():
    """asyncio.Queue：生产者-消费者模式"""
    print("\n=== Queue Demo ===")
    queue = asyncio.Queue(maxsize=5)
    producers = [asyncio.create_task(producer(queue, 8))]
    workers = [
        asyncio.create_task(worker(i, queue))
        for i in range(3)
    ]

    # 等待所有生产者完成
    await asyncio.gather(*producers)
    # 向每个 worker 发送停止信号
    for w in workers:
        await queue.put(None)
    # 等待所有 worker 完成
    await asyncio.gather(*workers)


async def main():
    await gather_demo()
    await wait_demo()
    await queue_demo()


asyncio.run(main())
```

### 示例 4：协程异常处理与取消

```python
"""
示例：协程的异常处理、取消机制和超时控制
"""
import asyncio


async def task_with_exception():
    """可能抛出异常的协程"""
    await asyncio.sleep(0.05)
    raise ValueError("Something went wrong!")


async def cancellable_task():
    """可取消的协程"""
    try:
        print("  可取消任务: 开始")
        await asyncio.sleep(10)  # 长时间等待
        print("  可取消任务: 完成（不应该看到这行）")
    except asyncio.CancelledError:
        print("  可取消任务: 收到取消信号，清理资源...")
        raise  # 重新抛出，让调用者知道已取消
    finally:
        print("  可取消任务: finally 块执行")


async def timeout_task():
    """会被超时的协程"""
    await asyncio.sleep(10)
    return "should not reach here"


async def main():
    # ---- 异常处理 ----
    print("=== 异常处理 ===")
    try:
        await task_with_exception()
    except ValueError as e:
        print(f"  捕获异常: {e}")

    # gather 的异常处理
    print("\n=== gather 异常处理 ===")
    results = await asyncio.gather(
        asyncio.sleep(0.05, result="success"),
        task_with_exception(),
        return_exceptions=True,  # 异常作为结果返回而非抛出
    )
    for result in results:
        if isinstance(result, Exception):
            print(f"  异常结果: {type(result).__name__}: {result}")
        else:
            print(f"  正常结果: {result}")

    # ---- 取消 ----
    print("\n=== 取消 ===")
    task = asyncio.create_task(cancellable_task())
    await asyncio.sleep(0.05)  # 让任务开始
    task.cancel()
    try:
        await task
    except asyncio.CancelledError:
        print("  任务已被取消")

    # ---- 超时 ----
    print("\n=== 超时 ===")
    try:
        await asyncio.wait_for(timeout_task(), timeout=0.1)
    except asyncio.TimeoutError:
        print("  操作超时！")


asyncio.run(main())
```

### 示例 5：异步生成器与 async for

```python
"""
示例：异步生成器的定义与使用，
async yield 与 async for 的配合
"""
import asyncio
import random


async def async_range(start, end, delay=0.01):
    """异步生成器——模拟异步数据源"""
    current = start
    while current < end:
        await asyncio.sleep(delay)  # 模拟异步 I/O
        yield current
        current += 1


async def async_filter(aiterable, predicate):
    """异步过滤管道"""
    async for item in aiterable:
        if predicate(item):
            yield item


async def async_map(aiterable, transform):
    """异步映射管道"""
    async for item in aiterable:
        result = transform(item)
        if asyncio.iscoroutine(result):
            result = await result
        yield result


async def async_enumerate(aiterable, start=0):
    """异步枚举"""
    index = start
    async for item in aiterable:
        yield index, item
        index += 1


async def async_reduce(aiterable, func, initial):
    """异步归约"""
    result = initial
    async for item in aiterable:
        result = func(result, item)
    return result


async def main():
    # 基本异步生成器
    print("=== 异步生成器 ===")
    async for num in async_range(0, 5):
        print(f"  {num}", end=" ")
    print()

    # 异步管道
    print("\n=== 异步管道 ===")
    pipeline = async_map(
        async_filter(
            async_range(0, 20, delay=0.001),
            lambda x: x % 2 == 0
        ),
        lambda x: x ** 2
    )

    async for index, value in async_enumerate(pipeline):
        print(f"  [{index}] = {value}")

    # 异步归约
    print("\n=== 异步归约 ===")
    result = await async_reduce(
        async_range(0, 10, delay=0.001),
        lambda acc, x: acc + x,
        0
    )
    print(f"  Sum: {result}")


asyncio.run(main())
```

## 20.5 常见陷阱与最佳实践

### 20.5.1 忘记 await 协程

**最常见的陷阱**：调用 `async def` 函数但不 `await`，导致协程从未执行：

```python
# 陷阱：协程未被 await，只是创建了协程对象
async def fetch_data():
    await asyncio.sleep(1)
    return "data"

async def buggy():
    result = fetch_data()  # 忘记 await！result 是 coroutine 对象
    print(type(result))    # <class 'coroutine'>
    # Python 会发出 RuntimeWarning: coroutine 'fetch_data' was never awaited

# 修复：始终 await 协程
async def fixed():
    result = await fetch_data()  # 正确：await 等待协程完成
    print(result)                 # 'data'
```

Python 3.8+ 会对未 await 的协程发出 `RuntimeWarning`，3.12+ 可能抛出 `RuntimeError`。

### 20.5.2 在异步代码中使用阻塞 I/O

**陷阱**：在协程中调用阻塞 I/O（如 `time.sleep()`、`requests.get()`、`open()`），会阻塞整个事件循环：

```python
import asyncio
import time

# 陷阱：阻塞整个事件循环
async def buggy_concurrent():
    start = time.perf_counter()
    await asyncio.gather(
        blocking_task(),  # 阻塞！其他协程无法运行
        blocking_task(),
    )
    print(f"耗时: {time.perf_counter() - start:.2f}s")  # ~4s 而非 ~2s

async def blocking_task():
    time.sleep(2)  # 同步阻塞！
    return "done"

# 修复：使用 asyncio 的异步版本或在线程池中运行
async def fixed_concurrent():
    start = time.perf_counter()
    await asyncio.gather(
        async_task(),
        async_task(),
    )
    print(f"耗时: {time.perf_counter() - start:.2f}s")  # ~2s

async def async_task():
    await asyncio.sleep(2)  # 异步非阻塞
    return "done"

# 修复：将阻塞代码放入线程池
async def run_blocking():
    loop = asyncio.get_running_loop()
    result = await loop.run_in_executor(None, lambda: time.sleep(2))
    return result
```

### 20.5.3 协程与生成器混用

**陷阱**：`await` 生成器会导致 `TypeError`：

```python
async def bad():
    # await 不能用于普通生成器
    # for x in async_gen():  # 这是可以的，但不是 await
    pass

# 正确用法：
# 1. 用 async for 迭代异步生成器
# 2. 用 await 等待协程
# 3. 用 asyncio.gather() 并发运行多个协程
```

### 20.5.4 Task 创建时机

**陷阱**：在 `asyncio.gather()` 之外手动创建 Task 可能导致非预期行为：

```python
# 陷阱：Task 在 gather 之前就开始运行
async def buggy():
    task1 = asyncio.create_task(slow_op("A", 1.0))  # 立即开始！
    task2 = asyncio.create_task(slow_op("B", 1.0))  # 立即开始！
    await asyncio.sleep(0.5)
    # task1 和 task2 已经运行了 0.5s
    results = await asyncio.gather(task1, task2)  # 不是从 gather 开始计时
    return results

# 修复：直接传协程给 gather
async def fixed():
    results = await asyncio.gather(
        slow_op("A", 1.0),
        slow_op("B", 1.0),
    )
    return results
```

### 20.5.5 异步生成器资源泄漏

**陷阱**：未消费完的异步生成器可能导致资源泄漏：

```python
async def db_cursor(query):
    """模拟数据库游标"""
    conn = await acquire_connection()
    try:
        for i in range(100):
            await asyncio.sleep(0.01)
            yield f"row_{i}"
    finally:
        await release_connection(conn)

# 陷阱：提前退出，连接未释放
async def buggy():
    async for row in db_cursor("SELECT *"):
        if row == "row_5":
            break  # finally 不保证执行！

# 修复：使用 async with + asynccontextmanager
from contextlib import asynccontextmanager

@asynccontextmanager
async def db_session():
    conn = await acquire_connection()
    try:
        yield conn
    finally:
        await release_connection(conn)
```

### 20.5.6 最佳实践总结

| 实践 | 说明 |
|------|------|
| **始终 await 协程** | 调用 `async def` 函数后必须 `await`，否则协程不执行 |
| **使用 `asyncio.gather` 并发** | 需要并发运行多个协程时使用 `gather`，而非串行 `await` |
| **避免阻塞 I/O** | 在协程中使用 `asyncio.sleep` 而非 `time.sleep`，使用 `aiohttp` 而非 `requests` |
| **使用 `asyncio.create_task`** | 需要后台任务时使用 `create_task`，而非直接 `await` |
| **处理 `CancelledError`** | 在 `try/finally` 中清理资源，`CancelledError` 后重新抛出 |
| **使用 `asynccontextmanager`** | 管理异步资源时使用 `async with` |
| **设置超时** | 使用 `asyncio.wait_for` 防止协程永远挂起 |
| **不要混用同步与异步** | 不要在异步代码中直接调用同步阻塞函数 |
| **理解 Task vs 协程** | `create_task` 立即调度，协程只是定义，`gather` 统一调度 |
| **使用 `return_exceptions=True`** | `gather` 中一个协程失败不应影响其他协程 |
