# 第21章 函数式编程范式

## 21.1 概念与语法

### 21.1.1 函数式编程核心思想

函数式编程（Functional Programming, FP）是一种编程范式，其核心思想是：

- **函数是一等公民**：函数可以赋值给变量、作为参数传递、作为返回值
- **纯函数**：相同输入总是产生相同输出，没有副作用
- **不可变数据**：数据一旦创建就不修改，而是创建新的数据
- **声明式**：描述"做什么"而非"怎么做"
- **组合**：通过组合简单函数构建复杂功能

```python
# 命令式风格
result = []
for x in range(10):
    if x % 2 == 0:
        result.append(x ** 2)

# 函数式风格
result = list(map(lambda x: x ** 2, filter(lambda x: x % 2 == 0, range(10))))
```

### 21.1.2 高阶函数

高阶函数是接受函数作为参数或返回函数的函数。Python 内置了三个经典高阶函数：

```python
# map(function, iterable, ...) —— 映射
squares = list(map(lambda x: x ** 2, range(5)))
print(squares)  # [0, 1, 4, 9, 16]

# 多 iterable 参数
sums = list(map(lambda x, y: x + y, [1, 2, 3], [10, 20, 30]))
print(sums)  # [11, 22, 33]

# filter(function, iterable) —— 过滤
evens = list(filter(lambda x: x % 2 == 0, range(10)))
print(evens)  # [0, 2, 4, 6, 8]

# functools.reduce(function, iterable[, initial]) —— 归约
from functools import reduce
total = reduce(lambda acc, x: acc + x, range(1, 6), 0)
print(total)  # 15
```

### 21.1.3 lambda 表达式

`lambda` 是匿名函数的语法糖，只能包含单个表达式：

```python
# 基本语法: lambda parameters: expression
add = lambda x, y: x + y
print(add(3, 5))  # 8

# 作为排序键
pairs = [(1, 'b'), (3, 'a'), (2, 'c')]
pairs.sort(key=lambda p: p[1])
print(pairs)  # [(3, 'a'), (1, 'b'), (2, 'c')]

# 作为回调
numbers = [3, 1, 4, 1, 5, 9, 2, 6]
sorted_desc = sorted(numbers, key=lambda x: -x)
print(sorted_desc)  # [9, 6, 5, 4, 3, 2, 1, 1]
```

### 21.1.4 列表/字典/集合推导式

推导式是 Python 中最常用的函数式风格特性：

```python
# 列表推导式
squares = [x ** 2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]

# 字典推导式
word_lengths = {word: len(word) for word in ["hello", "world", "python"]}

# 集合推导式

# 生成器表达式（惰性）

```

### 21.1.5 函数组合

函数式编程的核心操作是组合——将简单函数组合成复杂函数：

```python
def compose(*funcs):
    """从右到左组合函数: compose(f, g, h)(x) == f(g(h(x)))"""
    def composed(x):
        result = x
        for func in reversed(funcs):
            result = func(result)
        return result
    return composed

def pipe(*funcs):
    """从左到右组合函数: pipe(f, g, h)(x) == h(g(f(x)))"""
    def piped(x):
        result = x
        for func in funcs:
            result = func(result)
        return result
    return piped

# 使用
double = lambda x: x * 2
add_one = lambda x: x + 1
square = lambda x: x ** 2

transform = compose(square, add_one, double)  # double → add_one → square
print(transform(3))  # square(add_one(double(3))) = square(7) = 49

transform2 = pipe(double, add_one, square)  # 同样的顺序
print(transform2(3))  # square(add_one(double(3))) = 49
```

## 21.2 原理与机制

### 21.2.1 map/filter/reduce 的 CPython 实现

#### map 的底层实现

`map` 在 CPython 中由 C 实现（`Python/bltinmodule.c`），核心结构：

```c
typedef struct {
    PyObject_HEAD
    PyObject *iters;       // 存储所有输入迭代器的元组
    PyObject *func;         // 映射函数
    int num_iterables;      // 迭代器数量
} mapobject;
```

`map_next()` 的核心逻辑：

```c
static PyObject *
map_next(mapobject *lz)
{
    PyObject *val, *result;
    // 对每个迭代器调用 next()
    // 将所有值打包为元组
    // 调用 func(*args)
    result = PyObject_Call(lz->func, args, NULL);
    return result;
}
```

关键特性：
- `map` 返回迭代器（Python 3），惰性求值
- 多个 `iterable` 参数时，以最短的为准（类似 `zip`）
- 当 `func` 为 `None` 时，等价于 `zip()`

```python
# map 的惰性特性
m = map(lambda x: x ** 2, range(5))
print(type(m))  # <class 'map'> —— 迭代器，不是列表
print(list(m))   # [0, 1, 4, 9, 16] —— 一次性消费

# func=None 等价于 zip
result = list(map(None, [1, 2, 3], ['a', 'b', 'c']))
# TypeError in Python 3! (map(None, ...) was removed)
# Python 2 会返回 [(1, 'a'), (2, 'b'), (3, 'c')]
```

#### filter 的底层实现

`filter` 同样由 C 实现，核心逻辑：

```c
static PyObject *
filter_next(filterobject *lz)
{
    PyObject *val, *result;
    val = PyIter_Next(lz->it);        // 获取下一个值
    if (lz->func == Py_None) {
        result = val;                   // None → 保留真值
    } else {
        result = PyObject_CallFunctionObjArgs(lz->func, val, NULL);
        if (PyObject_IsTrue(result)) {
            result = val;               // 函数返回真 → 保留
        } else {
            Py_DECREF(result);
            continue;                   // 函数返回假 → 跳过
        }
    }
    return result;
}
```

```python
# filter 的惰性特性
f = filter(lambda x: x % 2 == 0, range(10))
print(type(f))  # <class 'filter'>

# func=None 过滤假值
truthy = list(filter(None, [0, 1, "", "hello", [], [1, 2], None]))
print(truthy)  # [1, 'hello', [1, 2]]
```

#### reduce 的底层实现

`reduce` 在 Python 3 中移至 `functools` 模块，其 Python 实现核心逻辑（`Lib/functools.py`）：

```python
def reduce(function, iterable, initializer=None):
    it = iter(iterable)
    if initializer is None:
        value = next(it)  # 无初始值时，取第一个元素作为初始值
    else:
        value = initializer
    for element in it:
        value = function(value, element)  # 累积计算
    return value
```

```python
from functools import reduce

# 无初始值——取第一个元素作为初始值
total = reduce(lambda acc, x: acc + x, [1, 2, 3, 4, 5])
print(total)  # 15 —— (((1 + 2) + 3) + 4) + 5

# 有初始值
total = reduce(lambda acc, x: acc + x, [1, 2, 3, 4, 5], 0)
print(total)  # 15 —— ((((0 + 1) + 2) + 3) + 4) + 5

# 空序列必须提供初始值
# reduce(lambda acc, x: acc + x, [])  # TypeError
result = reduce(lambda acc, x: acc + x, [], 0)
print(result)  # 0
```

### 21.2.2 functools.partial 的实现原理

`functools.partial` 创建一个部分应用（偏函数），固定函数的部分参数：

CPython 中 `partial` 的 C 实现（`Modules/_functoolsmodule.c`）：

```c
typedef struct {
    PyObject_HEAD
    PyObject *fn;           // 原始函数
    PyObject *args;          // 固定的位置参数元组
    PyObject *kw;            // 固定的关键字参数字典
    PyObject *dict;          // __dict__
    PyObject *weakreflist;   // 弱引用列表
} partialobject;
```

`partial` 调用时的核心逻辑：

```c
static PyObject *
partial_call(partialobject *pto, PyObject *args, PyObject *kw)
{
    // 1. 合并固定参数和新参数
    // pto->args + args → 最终位置参数
    // pto->kw + kw    → 最终关键字参数

    // 2. 调用原始函数
    result = PyObject_Call(pto->fn, final_args, final_kw);
    return result;
}
```

```python
from functools import partial

# 偏函数示例
def power(base, exponent):
    return base ** exponent

square = partial(power, exponent=2)    # 固定 exponent=2
cube = partial(power, exponent=3)       # 固定 exponent=3

print(square(5))  # 25 —— power(5, exponent=2)
print(cube(3))    # 27 —— power(3, exponent=3)

# partial 保存了原始函数和固定参数
print(square.func)        # <function power at 0x...>
print(square.args)        # ()
print(square.keywords)    # {'exponent': 2}
```

### 21.2.3 functools.lru_cache 的实现原理

`lru_cache` 是一个基于 LRU（Least Recently Used）策略的缓存装饰器。CPython 实现使用双向链表 + 字典：

核心数据结构：

```
            ┌─────────────────────────────────────────┐
            │              Cache Dict                  │
            │  key → (result, hash, link_node)         │
            └─────────────────────────────────────────┘

            ┌─────────────────────────────────────────┐
            │           Doubly-Linked List             │
            │   root ↔ newest ↔ ... ↔ oldest ↔ root   │
            └─────────────────────────────────────────┘
```

工作流程：

1. **缓存命中**：查找字典 → 将节点移动到链表头部（最近使用）
2. **缓存未命中**：计算结果 → 存入字典 → 插入链表头部 → 若超出 `maxsize`，移除链表尾部节点
3. **淘汰策略**：当缓存满时，移除最久未使用的条目（链表尾部）

```python
from functools import lru_cache
import time

@lru_cache(maxsize=128)
def expensive_computation(n):
    """带有 LRU 缓存的昂贵计算"""
    print(f"  Computing for n={n}")
    time.sleep(0.01)
    return n ** 2

# 第一次调用——缓存未命中
print(expensive_computation(5))  # Computing for n=5 → 25

# 第二次调用——缓存命中
print(expensive_computation(5))  # 25（无 "Computing" 输出）

# 查看缓存信息
print(expensive_computation.cache_info())
# CacheInfo(hits=1, misses=1, maxsize=128, currsize=1)

# 清空缓存
expensive_computation.cache_clear()
print(expensive_computation.cache_info())
# CacheInfo(hits=0, misses=0, maxsize=128, currsize=0)
```

### 21.2.4 functools.singledispatch 的实现原理

`singledispatch` 实现了单分派泛型函数（single-dispatch generic function），根据第一个参数的类型选择不同的实现：

底层机制：
1. 维护一个 `registry` 字典：`{type: implementation_function}`
2. 调用时，根据第一个参数的类型查找 `registry`
3. 若未找到精确匹配，按 MRO（方法解析顺序）查找父类实现

```python
from functools import singledispatch

@singledispatch
def process(value):
    """默认实现"""
    raise TypeError(f"Unsupported type: {type(value)}")

@process.register(int)
def _(value):
    return f"Processing integer: {value * 2}"

@process.register(str)
def _(value):
    return f"Processing string: {value.upper()}"

@process.register(list)
def _(value):
    return f"Processing list: {len(value)} items"

print(process(42))          # Processing integer: 84
print(process("hello"))     # Processing string: HELLO
print(process([1, 2, 3]))  # Processing list: 3 items
# process(3.14)             # TypeError: Unsupported type: <class 'float'>

# 注册更多类型
@process.register(float)
def _(value):
    return f"Processing float: {value:.2f}"

print(process(3.14))  # Processing float: 3.14

# 查看 registry
print(process.registry.keys())
# dict_keys([<class 'object'>, <class 'int'>, <class 'str'>, <class 'list'>, <class 'float'>])
```

### 21.2.5 operator 模块的底层

`operator` 模块提供了与 Python 运算符对应的函数，这些函数在 C 层实现，比 `lambda` 更快：

```python
import operator
import timeit

# operator.add vs lambda
print(timeit.timeit("add(1, 2)", globals={"add": operator.add}))
# ~0.08μs

print(timeit.timeit("add(1, 2)", globals={"add": lambda x, y: x + y}))
# ~0.12μs —— lambda 更慢（需要 Python 函数调用开销）
```

常用 operator 函数与运算符对照：

| operator 函数 | 运算符 | 说明 |
|---------------|--------|------|
| `operator.add(a, b)` | `a + b` | 加法 |
| `operator.sub(a, b)` | `a - b` | 减法 |
| `operator.mul(a, b)` | `a * b` | 乘法 |
| `operator.truediv(a, b)` | `a / b` | 除法 |
| `operator.floordiv(a, b)` | `a // b` | 整除 |
| `operator.mod(a, b)` | `a % b` | 取模 |
| `operator.neg(a)` | `-a` | 取负 |
| `operator.eq(a, b)` | `a == b` | 等于 |
| `operator.lt(a, b)` | `a < b` | 小于 |
| `operator.gt(a, b)` | `a > b` | 大于 |
| `operator.itemgetter(k)` | `obj[k]` | 索引取值 |
| `operator.attrgetter(name)` | `obj.name` | 属性取值 |
| `operator.methodcaller(name)` | `obj.name()` | 方法调用 |

## 21.3 使用场景

### 21.3.1 数据转换管道

使用函数组合构建数据处理管道：

```python
from functools import reduce

def compose(*funcs):
    """从右到左组合函数"""
    return reduce(lambda f, g: lambda x: f(g(x)), funcs)

def pipe(*funcs):
    """从左到右组合函数（更自然的阅读顺序）"""
    return reduce(lambda f, g: lambda x: g(f(x)), funcs)

# 数据清洗管道
strip = lambda s: s.strip()
lower = lambda s: s.lower()
remove_digits = lambda s: ''.join(c for c in s if not c.isdigit())
capitalize = lambda s: s.capitalize()

clean_pipeline = pipe(strip, lower, remove_digits, capitalize)

raw_data = ["  Hello123  ", "WORLD456  ", "  python789  "]
cleaned = [clean_pipeline(s) for s in raw_data]
print(cleaned)  # ['Hello', 'World', 'Python']
```

### 21.3.3 缓存与记忆化

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fibonacci(n):
    """无限制缓存的斐波那契数列"""
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# 瞬间完成——缓存使得递归斐波那契变为 O(n)
print(fibonacci(100))  # 354224848179261915075

# 递归关系保持——每个值只计算一次
print(fibonacci.cache_info())
# CacheInfo(hits=98, misses=101, maxsize=None, currsize=101)

# 缓存键包含所有参数
@lru_cache(maxsize=128)
def expensive_db_query(table, conditions):
    """缓存数据库查询结果"""
    print(f"  Querying {table} with {conditions}")
    return f"data_from_{table}"

# 相同参数命中缓存
expensive_db_query("users", "active=True")
expensive_db_query("users", "active=True")  # 缓存命中
expensive_db_query("users", "active=False")  # 缓存未命中

print(expensive_db_query.cache_info())
# CacheInfo(hits=1, misses=2, maxsize=128, currsize=2)
```

### 21.3.4 多类型分发

```python
from functools import singledispatch
import json

@singledispatch
def to_json(value):
    """将不同类型转换为 JSON 兼容格式"""
    raise TypeError(f"Cannot serialize {type(value).__name__}")

@to_json.register(str)
def _(value):
    return value

@to_json.register(int)
def _(value):
    return value

@to_json.register(float)
def _(value):
    return value

@to_json.register(bool)
def _(value):
    return value

@to_json.register(list)
def _(value):
    return [to_json(item) for item in value]

@to_json.register(dict)
def _(value):
    return {str(k): to_json(v) for k, v in value.items()}

@to_json.register(tuple)
def _(value):
    return [to_json(item) for item in value]

@to_json.register(set)
def _(value):
    return sorted([to_json(item) for item in value])

# 使用
data = {
    "name": "Alice",
    "scores": (95, 87, 92),
    "tags": {"python", "functional"},
    "active": True,
}

print(json.dumps(to_json(data), indent=2))
```

### 21.3.5 不可变数据结构实践

```python
from collections import namedtuple
from types import MappingProxyType
from dataclasses import dataclass
from functools import partial

# ---- namedtuple：不可变序列 ----
Point = namedtuple('Point', ['x', 'y'])
p1 = Point(3, 4)
# p1.x = 5  # AttributeError: 不可修改

# "修改"返回新实例
p2 = p1._replace(x=5)
print(p1)  # Point(x=3, y=4)  —— 原实例不变
print(p2)  # Point(x=5, y=4)  —— 新实例

# ---- MappingProxyType：不可变字典 ----
config = MappingProxyType({
    "host": "localhost",
    "port": 5432,
    "database": "myapp",
})
# config["host"] = "remote"  # TypeError: 不可修改

# ---- frozen dataclass ----
@dataclass(frozen=True)
class User:
    name: str
    age: int
    email: str

u1 = User("Alice", 30, "alice@example.com")
# u1.name = "Bob"  # FrozenInstanceError: 不可修改

# "修改"创建新实例
u2 = User("Bob", u1.age, u1.email)

# ---- 不可变数据转换 ----
def immutable_update(mapping, **kwargs):
    """返回新的不可变映射，原映射不变"""
    data = dict(mapping)
    data.update(kwargs)
    return MappingProxyType(data)

new_config = immutable_update(config, port=3306)
print(new_config["port"])  # 3306
print(config["port"])      # 5432 —— 原映射不变
```

## 21.4 示例代码

### 示例 1：map/filter/reduce 完整示例

```python
"""
示例：使用 map、filter、reduce 处理数据，
对比函数式风格与命令式风格
"""
from functools import reduce
import operator
import math

# 数据集
students = [
    {"name": "Alice", "scores": [95, 87, 92, 88]},
    {"name": "Bob", "scores": [78, 82, 75, 90]},
    {"name": "Charlie", "scores": [92, 95, 89, 94]},
    {"name": "Diana", "scores": [85, 88, 82, 91]},
    {"name": "Eve", "scores": [70, 65, 72, 68]},
]

# ---- map：计算每个学生的平均分 ----
# 函数式
averages = map(
    lambda s: {"name": s["name"], "average": sum(s["scores"]) / len(s["scores"])},
    students
)
print("--- 学生平均分 ---")
for item in averages:
    print(f"  {item['name']}: {item['average']:.1f}")

# ---- filter：筛选优秀学生（平均分 >= 85） ----
# 函数式
honor_students = filter(
    lambda s: sum(s["scores"]) / len(s["scores"]) >= 85,
    students
)
print("\n--- 优秀学生 ---")
for s in honor_students:
    avg = sum(s["scores"]) / len(s["scores"])
    print(f"  {s['name']}: {avg:.1f}")

# ---- reduce：计算所有分数的总和 ----
all_scores = reduce(
    lambda acc, s: acc + s["scores"],
    students,
    []
)
total = reduce(operator.add, all_scores, 0)
print(f"\n--- 总分: {total} ---")

# ---- 组合使用：优秀学生的最高分 ----
honor_max = reduce(
    max,
    map(
        lambda s: max(s["scores"]),
        filter(
            lambda s: sum(s["scores"]) / len(s["scores"]) >= 85,
            students
        )
    )
)
print(f"\n--- 优秀学生最高分: {honor_max} ---")

# ---- 与命令式风格对比 ----
# 命令式
honor_max_imperative = float('-inf')
for s in students:
    avg = sum(s["scores"]) / len(s["scores"])
    if avg >= 85:
        max_score = max(s["scores"])
        if max_score > honor_max_imperative:
            honor_max_imperative = max_score

print(f"--- 命令式结果: {honor_max_imperative} ---")
print(f"--- 两种方式结果一致: {honor_max == honor_max_imperative} ---")
```

### 示例 2：functools.partial 与偏函数

```python
"""
示例：偏函数的多种应用场景
"""
from functools import partial
import json

# ---- 基本偏函数 ----
def power(base, exponent):
    return base ** exponent

# 固定第二个参数
square = partial(power, exponent=2)
```

### 示例 3：functools.lru_cache 深度示例

```python
"""
示例：lru_cache 的多种用法，
包括缓存分析、缓存失效、参数限制
"""
from functools import lru_cache
import time


# ---- 基本缓存 ----
@lru_cache(maxsize=128)
def fibonacci(n):
    """缓存斐波那契计算"""
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)


print("--- 斐波那契缓存 ---")
start = time.perf_counter()
result = fibonacci(50)
elapsed = time.perf_counter() - start
print(f"fibonacci(50) = {result}")
print(f"耗时: {elapsed:.6f}s")
print(f"缓存信息: {fibonacci.cache_info()}")

# ---- 缓存矩阵乘法 ----
@lru_cache(maxsize=1024)
def matrix_multiply_hash(a_hash, b_hash, rows, cols):
    """用哈希作为缓存键，避免缓存大型矩阵"""
    # 实际计算在缓存未命中时进行
    import random
    random.seed(a_hash + b_hash)
    result = [[random.random() for _ in range(cols)] for _ in range(rows)]
    return tuple(tuple(row) for row in result)  # 返回不可变结果


# ---- 缓存与缓存失效 ----
class CachedDataService:
    """带缓存失效机制的数据服务"""

    def __init__(self):
        self._version = 0

    @lru_cache(maxsize=32)
    def get_data(self, key, version=None):
        """version 参数用于缓存失效"""
        print(f"  [Cache Miss] Loading data for key={key}")
        time.sleep(0.01)
        return f"data_for_{key}"

    def invalidate_cache(self):
        """通过改变 version 使缓存失效"""
        self._version += 1
        self.get_data.cache_clear()
        print(f"  Cache invalidated, version={self._version}")

    def query(self, key):
        return self.get_data(key, version=self._version)


service = CachedDataService()
print("\n--- 缓存服务 ---")
print(service.query("user:1"))   # Cache Miss
print(service.query("user:1"))   # Cache Hit
print(service.query("user:2"))   # Cache Miss
service.invalidate_cache()
print(service.query("user:1"))   # Cache Miss（版本变了）


# ---- 缓存递归组合计算 ----
@lru_cache(maxsize=None)
def combinator(n, k):
    """缓存组合数计算 C(n, k)"""
    if k == 0 or k == n:
        return 1
    return combinator(n - 1, k - 1) + combinator(n - 1, k)


print("\n--- 组合数计算 ---")
print(f"C(30, 15) = {combinator(30, 15)}")
print(f"缓存信息: {combinator.cache_info()}")

# ---- 注意：lru_cache 的键必须是可哈希的 ----
# @lru_cache(maxsize=128)
# def bad_cache(data):  # dict 不可哈希！
#     return len(data)

# 修复：使用 frozenset 或元组作为键
@lru_cache(maxsize=128)
def good_cache(key_tuple):
    """使用元组作为缓存键"""
    return sum(key_tuple)

print(f"\ngood_cache: {good_cache((1, 2, 3, 4, 5))}")
print(f"缓存信息: {good_cache.cache_info()}")
```

### 示例 4：functools.singledispatch 与 operator

```python
"""
示例：singledispatch 实现多态函数，
operator 模块替代 lambda
"""
from functools import singledispatch
import operator
from collections import Counter

# ---- singledispatch：类型驱动的渲染器 ----
@singledispatch
def render(data):
    """根据数据类型选择渲染方式"""
    return str(data)

@render.register(str)
def _(data):
    return f'"{data}"'

@render.register(int)
def _(data):
    return f"{data:,}"

@render.register(float)
def _(data):
    return f"{data:.2f}"

@render.register(list)
def _(data):
    items = ", ".join(render(item) for item in data)
    return f"[{items}]"

@render.register(dict)
def _(data):
    items = ", ".join(
        f"{render(k)}: {render(v)}"
        for k, v in data.items()
    )
    return f"{{{items}}}"

@render.register(tuple)
def _(data):
    items = ", ".join(render(item) for item in data)
    return f"({items})"

@render.register(set)
def _(data):
    items = ", ".join(render(item) for item in sorted(data, key=str))
    return f"{{{items}}}"

@render.register(bool)
def _(data):
    return "✓" if data else "✗"


# 测试
print("--- 渲染器示例 ---")
print(render("hello"))                        # "hello"
print(render(1234567))                        # 1,234,567
print(render(3.14159))                         # 3.14
print(render([1, "two", 3.0]))                 # [1, "two", 3.14]
print(render({"key": "value"}))                # {"key": "value"}
print(render({1, 2, 3}))                       # {1, 2, 3}
print(render(True))                            # ✓


# ---- operator 模块替代 lambda ----
print("\n--- operator 模块 ---")

# 排序——使用 itemgetter
students = [
    {"name": "Alice", "score": 95},
    {"name": "Bob", "score": 82},
    {"name": "Charlie", "score": 91},
    {"name": "Diana", "score": 88},
]

# 使用 lambda
by_score_lambda = sorted(students, key=lambda s: s["score"], reverse=True)
# 使用 operator.itemgetter
by_score_op = sorted(students, key=operator.itemgetter("score"), reverse=True)

print("按分数排序:")
for s in by_score_op:
    print(f"  {s['name']}: {s['score']}")

# 归约——使用 operator
total_score = reduce(operator.add, map(operator.itemgetter("score"), students))
print(f"\n总分: {total_score}")

# 累乘
product = reduce(operator.mul, range(1, 6), 1)
print(f"5! = {product}")

# attrgetter——按属性排序
from collections import namedtuple
Person = namedtuple('Person', ['name', 'age', 'height'])

people = [
    Person("Alice", 30, 165),
    Person("Bob", 25, 180),
    Person("Charlie", 35, 172),
]

by_age = sorted(people, key=operator.attrgetter('age'))
print("\n按年龄排序:")
for p in by_age:
    print(f"  {p.name}: {p.age}")

# methodcaller——调用方法
words = ["Hello", "world", "Python"]
upper_words = list(map(operator.methodcaller('upper'), words))
print(f"\n大写: {upper_words}")

title_words = list(map(operator.methodcaller('title'), words))
print(f"首字母大写: {title_words}")
```

### 示例 5：函数式数据处理综合示例

```python
"""
示例：综合使用函数式编程技巧处理数据，
展示管道、组合、柯里化等模式
"""
from functools import reduce, partial, lru_cache
import operator
import re

# ---- 工具函数 ----
def pipe(*funcs):
    """从左到右组合函数"""
    return lambda x: reduce(lambda v, f: f(v), funcs, x)

def compose(*funcs):
    """从右到左组合函数"""
    return lambda x: reduce(lambda v, f: f(v), reversed(funcs), x)

def curry(func, *args, **kwargs):
    """柯里化：将多参数函数转换为一系列单参数函数"""
    return partial(func, *args, **kwargs)

def trace(label, value):
    """调试辅助：打印并返回值"""
    print(f"  [{label}] {value}")
    return value

# ---- 函数式数据管道 ----
# 模拟日志数据
log_lines = [
    '192.168.1.1 - - [10/Oct/2023:13:55:36] "GET /index.html HTTP/1.1" 200 2326',
    '10.0.0.1 - - [10/Oct/2023:13:55:37] "POST /login HTTP/1.1" 401 128',
    '192.168.1.2 - - [10/Oct/2023:13:55:38] "GET /about.html HTTP/1.1" 200 2048',
    '172.16.0.1 - - [10/Oct/2023:13:55:39] "GET /admin HTTP/1.1" 403 256',
    '192.168.1.1 - - [10/Oct/2023:13:55:40] "GET /data.json HTTP/1.1" 200 5120',
    '10.0.0.1 - - [10/Oct/2023:13:55:41] "GET /index.html HTTP/1.1" 200 2326',
    '192.168.1.2 - - [10/Oct/2023:13:55:42] "POST /api/data HTTP/1.1" 500 64',
]

# 解析函数
LOG_PATTERN = re.compile(
    r'(?P<ip>\S+) \S+ \S+ \[(?P<time>[^\]]+)\] '
    r'"(?P<method>\w+) (?P<path>\S+) \S+" (?P<status>\d+) (?P<size>\d+)'
)

def parse_log(line):
    match = LOG_PATTERN.match(line)
    return match.groupdict() if match else None

def is_success(entry):
    return entry is not None and int(entry['status']) < 400

def is_error(entry):
    return entry is not None and int(entry['status']) >= 400

# 使用管道处理数据
success_ips = pipe(
    lambda lines: map(parse_log, lines),           # 解析
    partial(filter, lambda e: e is not None),        # 过滤 None
    partial(filter, is_success),                      # 过滤成功请求
    partial(map, operator.itemgetter('ip')),          # 提取 IP
    set,                                              # 去重
)

error_ips = pipe(
    lambda lines: map(parse_log, lines),
    partial(filter, lambda e: e is not None),
    partial(filter, is_error),
    partial(map, operator.itemgetter('ip')),
    set,
)

status_counts = pipe(
    lambda lines: map(parse_log, lines),
    partial(filter, lambda e: e is not None),
    partial(map, operator.itemgetter('status')),
    Counter,
)

print("--- 成功请求的 IP ---")
print(success_ips(log_lines))

print("\n--- 错误请求的 IP ---")
print(error_ips(log_lines))

print("\n--- 状态码统计 ---")
print(status_counts(log_lines))

# ---- 柯里化与偏函数 ----
print("\n--- 柯里化与偏函数 ---")

# 柯里化的加法
add = lambda x: lambda y: x + y
add5 = add(5)
print(f"add5(3) = {add5(3)}")  # 8

# 柯里化的乘法
multiply = lambda x: lambda y: x * y
double = multiply(2)
triple = multiply(3)
print(f"double(4) = {double(4)}")  # 8
print(f"triple(4) = {triple(4)}")  # 12

# ---- 不可变数据转换 ----
from types import MappingProxyType

def immutable_update(d, **kwargs):
    """不可变更新字典——返回新字典"""
    new_dict = dict(d)
    new_dict.update(kwargs)
    return MappingProxyType(new_dict)

config = MappingProxyType({"host": "localhost", "port": 5432})
new_config = immutable_update(config, port=3306)

print(f"\n原配置: {dict(config)}")
print(f"新配置: {dict(new_config)}")
```

## 21.5 常见陷阱与最佳实践

### 21.5.1 lambda 中的变量捕获

**陷阱**：lambda 捕获的是变量的引用，而非值的副本——与闭包相同的延迟绑定问题：

```python
# 陷阱：lambda 捕获变量的引用
funcs = [lambda: i for i in range(5)]
print([f() for f in funcs])  # [4, 4, 4, 4, 4]

# 修复 1：默认参数立即绑定
funcs = [lambda i=i: i for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]

# 修复 2：使用偏函数
from functools import partial
funcs = [partial(lambda i: i, i=i) for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]
```

### 21.5.2 map/filter 的惰性与多次迭代

**陷阱**：`map` 和 `filter` 返回迭代器，只能迭代一次：

```python
# 陷阱：迭代器耗尽
squares = map(lambda x: x ** 2, range(5))
print(list(squares))  # [0, 1, 4, 9, 16]
print(list(squares))  # [] —— 迭代器已耗尽！

# 修复：如果需要多次迭代，转为列表
squares_list = list(map(lambda x: x ** 2, range(5)))
print(squares_list)  # [0, 1, 4, 9, 16]
print(squares_list)  # [0, 1, 4, 9, 16]

# 或者重新创建迭代器
def squares_iter():
    return map(lambda x: x ** 2, range(5))

print(list(squares_iter()))  # [0, 1, 4, 9, 16]
print(list(squares_iter()))  # [0, 1, 4, 9, 16]
```

### 21.5.3 reduce 与初始值

**陷阱**：`reduce` 不提供初始值时，空序列会抛出 `TypeError`：

```python
from functools import reduce

# 陷阱：空序列无初始值
# reduce(lambda a, b: a + b, [])  # TypeError: reduce() of empty sequence with no initial value

# 修复：提供初始值
result = reduce(lambda a, b: a + b, [], 0)
print(result)  # 0

# 陷阱：单元素序列
result = reduce(lambda a, b: a + b, [42])
print(result)  # 42 —— 无初始值时，返回唯一元素

result = reduce(lambda a, b: a + b, [42], 0)
print(result)  # 42 —— 有初始值时，正常计算
```

### 21.5.4 lru_cache 的限制

**陷阱**：`lru_cache` 要求所有参数都是可哈希的：

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def process(data, key):
    return f"processed_{key}"

# process({"a": 1}, "x")  # TypeError: unhashable type: 'dict'

# 修复 1：使用可哈希的键类型
@lru_cache(maxsize=128)
def process_v2(key_tuple):
    return f"processed_{key_tuple}"

process_v2((1, 2, 3))  # OK

# 修复 2：使用 frozenset 代替 set，使用 tuple 代替 list
@lru_cache(maxsize=128)
def process_v3(items_tuple, flag):
    return f"processed_{items_tuple}_{flag}"

process_v3((1, 2, 3), True)  # OK

# 陷阱：缓存无限增长
@lru_cache(maxsize=None)  # 无限制缓存——可能导致内存泄漏
def grow_forever(n):
    return n ** 2

# 修复：设置合理的 maxsize
@lru_cache(maxsize=1024)  # 限制缓存大小
def bounded_cache(n):
    return n ** 2
```

### 21.5.5 过度使用函数式风格

**陷阱**：过度嵌套的函数式代码可读性极差：

```python
# 陷阱：难以阅读的函数式代码
result = reduce(
    lambda acc, x: acc + [x ** 2],
    filter(
        lambda x: x % 2 == 0,
        map(
            lambda x: x + 1,
            filter(
                lambda x: x > 0,
                [-5, -3, -1, 0, 1, 2, 3, 4, 5]
            )
        )
    ),
    []
)

# 修复：使用推导式——更 Pythonic
result = [x ** 2 for x in (x + 1 for x in [-5, -3, -1, 0, 1, 2, 3, 4, 5] if x > 0) if x % 2 == 0]

# 最佳：拆分为多步，每步清晰可读
numbers = [-5, -3, -1, 0, 1, 2, 3, 4, 5]
positives = [x for x in numbers if x > 0]
incremented = [x + 1 for x in positives]
even_incremented = [x for x in incremented if x % 2 == 0]
result = [x ** 2 for x in even_incremented]

print(result)  # [4, 16]
```

### 21.5.6 singledispatch 的继承限制

**陷阱**：`singledispatch` 只根据第一个参数的类型分派，不支持多分派：

```python
from functools import singledispatch

@singledispatch
def add(a, b):
    raise TypeError(f"Unsupported types: {type(a)}, {type(b)}")

@add.register(int)
def _(a, b):
    # b 的类型无法自动分派
    if isinstance(b, int):
        return a + b
    elif isinstance(b, float):
        return a + b
    else:
        return NotImplemented

# 如果需要根据多个参数类型分派，需要手动检查
# 或使用 functools.singledispatchmethod（Python 3.8+）+ 多次注册
```

### 21.5.7 最佳实践总结

| 实践 | 说明 |
|------|------|
| **推导式优先于 map/filter** | 简单场景用推导式，更 Pythonic、更可读 |
| **operator 优先于 lambda** | `operator.add` 比 `lambda x, y: x + y` 更快更清晰 |
| **lru_cache 缓存纯函数** | 只缓存无副作用的纯函数，避免缓存有状态操作 |
| **提供 reduce 初始值** | 避免空序列异常，使行为更可预测 |
| **partial 优先于 lambda** | `partial(func, arg)` 比 `lambda x: func(arg, x)` 更清晰 |
| **可读性优先** | 不要为了函数式而函数式，选择最清晰的风格 |
| **不可变数据优先** | 使用 `tuple`、`frozenset`、`MappingProxyType` 替代可变类型 |
| **singledispatch 用于类型分派** | 替代 `isinstance` 链和 `if-elif-else` 类型检查 |
| **限制 lambda 复杂度** | lambda 只用于简单表达式，复杂逻辑用 `def` |
| **注意迭代器耗尽** | `map`/`filter` 返回迭代器，只能消费一次 |
