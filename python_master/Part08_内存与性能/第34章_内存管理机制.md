# 第34章 内存管理机制

## 1. 概念与语法

### 1.1 Python 内存管理全景

Python 的内存管理采用分层架构，从底层操作系统内存到上层 Python 对象，形成四级结构：

```
┌─────────────────────────────────────────┐
│         Python 应用层 (Objects)          │  PyObject, PyListObject, etc.
├─────────────────────────────────────────┤
│         对象分配器 (pymalloc)             │  小对象专用分配器 (≤512 bytes)
├─────────────────────────────────────────┤
│         C 运行时内存分配                  │  malloc/calloc/realloc/free
├─────────────────────────────────────────┤
│         操作系统内核                      │  brk/mmap 系统调用
└─────────────────────────────────────────┘
```

关键源码文件（CPython 源码树）：
- `Objects/obmalloc.c` — pymalloc 核心实现，arena/pool/block 三级结构
- `Include/obmalloc.h` — pymalloc 数据结构定义
- `Include/cpython/pymem.h` — Python 内存分配宏与接口
- `Objects/object.c` — PyObject 通用分配/释放逻辑

### 1.2 核心语法与 API

```python
import sys

# 查看对象的浅层大小（不含引用对象）
```

## 2. 原理与机制

### 2.1 pymalloc 小对象分配器

CPython 为小于等于 512 字节的对象设计了专用分配器 pymalloc，避免频繁调用系统 malloc 带来的碎片和性能开销。

#### 2.1.1 Arena / Pool / Block 三级结构

```
Arena (256 KB)
┌───────────────────────────────────────────────────┐
│  Pool Header (48 bytes)  │  Block │ Block │ ... │
│  ─────────────────────────────────────────────── │
│  Pool Header (48 bytes)  │  Block │ Block │ ... │
│  ─────────────────────────────────────────────── │
│  ... (最多 64 个 Pool)                             │
└───────────────────────────────────────────────────┘

Pool (4 KB = 系统页大小)
┌──────────────────┬──────────────────────────────────┐
│  pool_header     │  Block │ Block │ Block │ ...    │
│  (48 bytes)      │  (固定大小, 对齐)                 │
└──────────────────┴──────────────────────────────────┘
```

**源码定义**（`Objects/obmalloc.c`）：

```c
// Arena 结构
struct arena {
    uintptr_t address;      // arena 起始地址
    poolp freepools;        // 空闲 pool 链表头
    struct arena* nextarena; // 双向链表
    struct arena* prevarena;
    uint nfreepools;        // 空闲 pool 数量
    uint ntotalpools;       // 总 pool 数量
};

// Arena 大小 = 256 KB (2^18)
#define ARENA_SIZE      (1 << 18)  // 262144 bytes
#define ARENA_SIZE_MASK (ARENA_SIZE - 1)

// Pool 大小 = 4 KB (系统页大小)
#define SYSTEM_PAGE_SIZE        (4 * 1024)
#define SYSTEM_PAGE_SIZE_MASK   (SYSTEM_PAGE_SIZE - 1)
#define POOL_SIZE               SYSTEM_PAGE_SIZE
#define POOL_SIZE_MASK          SYSTEM_PAGE_SIZE_MASK
```

#### 2.1.2 Block 大小分级

pymalloc 将请求大小对齐到 8 字节的倍数，按尺寸分为 64 个 size class：

```c
// 对齐粒度
#define ALIGNMENT              8
#define ALIGNMENT_MASK         (ALIGNMENT - 1)

// Block 最大尺寸（含对齐后）
#define SMALL_REQUEST_THRESHOLD 512
#define NB_SMALL_SIZE_CLASSES   (SMALL_REQUEST_THRESHOLD / ALIGNMENT)  // 64

// size class 索引计算
#define SIZECLASS_INDEX(size)   (((size) + ALIGNMENT_MASK) >> 3)
```

| 请求大小范围 | size class | Block 大小 | 每个 Pool 的 Block 数 |
|------------|-----------|------------|---------------------|
| 1 - 8      | 0         | 8          | 496 (4048/8 - 6)    |
| 9 - 16     | 1         | 16         | 248                  |
| 17 - 24    | 2         | 24         | 165                  |
| ...        | ...       | ...        | ...                  |
| 505 - 512  | 63        | 512        | 7                    |

#### 2.1.3 分配流程源码分析

```c
// Objects/obmalloc.c — pymalloc 核心分配逻辑（简化）
static void*
_PyObject_Malloc(void *ctx, size_t nbytes)
{
    block *p;
    poolp pool;
    uint size;

    // 超过小对象阈值，直接走系统 malloc
    if (nbytes > SMALL_REQUEST_THRESHOLD) {
        return PyMem_RawMalloc(nbytes);
    }

    size = (uint)(nbytes + ALIGNMENT_MASK) & ~ALIGNMENT_MASK;
    // 等价于: size = ((nbytes + 7) >> 3) << 3

    // 查找当前可用的 pool（usedpools[size class index]）
    pool = usedpools[size_class_index(size)];

    if (pool != pool->nextpool) {
        // pool 有空闲 block
        ++pool->ref.count;
        p = pool->freeblock;
        pool->freeblock = *(block **)p;  // 链表弹出
        return (void *)p;
    }

    // 当前无可用 pool，从 arenas 分配新 pool
    if (usable_arenas == NULL) {
        // 需要创建新 arena
        arena = new_arena();
        // ...
    }

    // 从 arena 中取一个空闲 pool
    pool = usable_arenas->freepools;
    // ... 初始化 pool 并分配 block
}
```

#### 2.1.4 Pool 内部结构

```
Pool Header (48 bytes):
┌────────────┬──────────┬───────────┬──────────┬──────────┐
│ nextpool   │ prevpool │ ref.count │ freeblock │ next    │
│ (4 bytes)  │ (4 bytes)│ (4 bytes) │ (4 bytes) │ ...     │
└────────────┴──────────┴───────────┴──────────┴──────────┘

freeblock 指向第一个空闲 block，空闲 block 内部形成隐式链表：
┌──────┐     ┌──────┐     ┌──────┐
│ next │────▶│ next │────▶│ NULL │   (空闲链表)
└──────┘     └──────┘     └──────┘
```

Pool 的三种状态：
- **used** — 部分 block 被使用，挂在 `usedpools[]` 链表中
- **full** — 所有 block 被使用，不在任何链表中
- **empty** — 所有 block 空闲，挂在 arena 的 freepools 链表中

### 2.3 内存释放与 Arena 回收

```c
// 释放 block
static void
_PyObject_Free(void *ctx, void *p)
{
    poolp pool;
    block *lastfree;
    pool = POOL_OF_PTR(p);   // 通过地址对齐找到所属 pool

    if (pool->ref.count == 1) {
        // pool 从 "full" 变为 "used"，加入 usedpools
    }
    --pool->ref.count;

    if (pool->ref.count == 0) {
        // pool 变为 "empty"
        // 检查 arena 是否可以整体释放
        if (arena->nfreepools == arena->ntotalpools) {
            // 整个 arena 空闲，释放给操作系统
            _PyObject_ArenaFree(arena->address);
        }
    }
}
```

**关键细节**：Arena 不会在所有 pool 空闲时立即释放，而是保留一定数量的 arena 作为缓存，避免反复向 OS 请求内存。只有当空闲 arena 数量超过 `MAX_UNUSED_ARENAS` 阈值时才真正释放。

### 2.4 对象头与实际内存消耗

每个 Python 对象都包含一个 `PyObject_HEAD`：

```c
// Include/object.h
typedef struct _object {
    Py_ssize_t ob_refcnt;   // 引用计数: 8 bytes (64-bit)
    PyTypeObject *ob_type;  // 类型指针:  8 bytes (64-bit)
} PyObject;                 // 合计: 16 bytes

// 具体类型继承 PyObject
typedef struct {
    PyObject_HEAD           // 16 bytes
    Py_ssize_t ob_shash;    // 缓存哈希: 8 bytes
    int ob_sstate;          // 内部状态: 4 bytes (对齐后8bytes)
    char ob_sval[1];        // 变长数据: 至少 1 byte + NULL
} PyBytesObject;            // 空 bytes 对象至少占 34 bytes
```

**实际测量**：

```python
import sys

# 基础类型开销
print(sys.getsizeof(0))           # 28 — int (值为0)
print(sys.getsizeof(1))           # 28 — int (值为1)
print(sys.getsizeof(2**30))       # 32 — int (需要更多位存储)
print(sys.getsizeof(2**60))       # 36 — int (继续增长)
print(sys.getsizeof(True))        # 28 — bool (实际是 int 子类)
print(sys.getsizeof(''))          # 49 — 空 str
print(sys.getsizeof('a'))         # 50 — 1字符str (每字符+1byte, UTF-8: 1-4 bytes/char)
print(sys.getsizeof([]))          # 56 — 空列表
print(sys.getsizeof(()))          # 40 — 空元组
print(sys.getsizeof({}))          # 232 — 空字典 (预分配哈希表)
print(sys.getsizeof(set()))       # 216 — 空集合
```

### 2.5 内存碎片处理

Python 通过以下机制缓解内存碎片：

1. **pymalloc 的 pool 对齐**：每个 pool 4KB 对齐，block 8 字节对齐，减少内部碎片
2. **size class 分级**：相同大小的对象从同一 pool 分配，避免外部碎片
3. **Arena 整体分配/释放**：256KB 为单位与 OS 交互，减少系统调用

但外部碎片仍然存在：当大量小对象长期存活时，pool 无法被回收，即使其他 block 已释放。

```python
"""
tracemalloc 内存追踪示例
追踪内存分配的精确位置和大小
"""
import tracemalloc
import linecache


def create_data():
    """制造一些内存分配"""
    data = [i ** 2 for i in range(10000)]
    return data


def process_data(data):
    """进一步处理"""
    result = {k: v for k, v in enumerate(data)}
    return result


def main():
    # 启动 tracemalloc，只追踪最近 25 帧的 traceback
    tracemalloc.start(25)

    # 第一次快照：基准
    snapshot1 = tracemalloc.take_snapshot()

    # 执行业务代码
    data = create_data()
    result = process_data(data)

    # 第二次快照：对比
    snapshot2 = tracemalloc.take_snapshot()

    # 计算两次快照之间的差异
    top_stats = snapshot2.compare_to(snapshot1, 'lineno')

    print("=== 内存分配增量 TOP 10 ===")
    for stat in top_stats[:10]:
        print(stat)

    # 查看单个分配的完整调用栈
    print("\n=== 最大分配的调用栈 ===")
    top_stats = snapshot2.statistics('traceback')
    stat = top_stats[0]
    print(f"{stat}")
    for line in stat.traceback.format():
        print(line)

    # 获取特定文件的内存统计
    print("\n=== 按文件统计 ===")
    snapshot_stats = snapshot2.statistics('filename')
    for stat in snapshot_stats[:5]:
        print(f"{stat.traceback}: {stat.size / 1024:.1f} KiB")

    tracemalloc.stop()


if __name__ == '__main__':
    main()
```

### 4.2 pymalloc 行为观察

```python
"""
通过实验观察 pymalloc 的分配行为
间接验证 arena/pool/block 机制
"""
import sys
import tracemalloc


def measure_allocation_pattern():
    """观察不同大小对象的内存分配模式"""
    tracemalloc.start()

    results = {}

    # 测试不同 size class 的分配
    test_sizes = [8, 16, 32, 64, 128, 256, 512, 1024, 2048, 4096]

    for size in test_sizes:
        tracemalloc.clear_traces()
        before = tracemalloc.take_snapshot()

        # 创建指定大小的 bytes 对象
        # bytes 对象头约 33 bytes (PyBytesObject) + 数据
        # 实际 pymalloc 分配大小 = (size + 33 + 7) & ~7（对齐到8）
        objs = [bytes(size) for _ in range(1000)]

        after = tracemalloc.take_snapshot()
        diff = after.compare_to(before, 'filename')

        total_allocated = sum(stat.size_diff for stat in diff)
        results[size] = {
            'count': len(objs),
            'total_allocated': total_allocated,
            'per_object': total_allocated // len(objs),
            'sys_sizeof': sys.getsizeof(bytes(size)),
        }

        del objs  # 释放，避免影响下一轮

    print(f"{'Size':>6} | {'总分配':>12} | {'每对象分配':>10} | {'sys.getsizeof':>14}")
    print("-" * 55)
    for size, info in results.items():
        print(f"{size:>6} | {info['total_allocated']:>10} B | "
              f"{info['per_object']:>8} B | {info['sys_sizeof']:>12} B")


def measure_arena_effect():
    """观察 arena 缓冲效果"""
    import gc
    gc.collect()

    # 分配大量小对象
    before = tracemalloc.take_snapshot()
    small_objs = [[] for _ in range(10000)]
    after_alloc = tracemalloc.take_snapshot()

    # 逐步释放
    batch_size = 2000
    for i in range(0, len(small_objs), batch_size):
        small_objs[i:i+batch_size] = [None] * batch_size
        gc.collect()

    after_free = tracemalloc.take_snapshot()

    alloc_stats = after_alloc.compare_to(before, 'lineno')
    free_stats = after_free.compare_to(before, 'lineno')

    print("分配后内存增量:")
    for s in alloc_stats[:5]:
        print(f"  {s}")

    print("\n部分释放后内存增量（注意：可能未完全释放给OS）:")
    for s in free_stats[:5]:
        print(f"  {s}")


if __name__ == '__main__':
    print("=== pymalloc 分配模式观察 ===\n")
    measure_allocation_pattern()
    print("\n=== Arena 缓冲效果 ===\n")
    measure_arena_effect()
```

### 4.3 内存布局分析工具

```python
"""
深入分析 Python 对象的内存布局
包括对齐、padding、内联存储等细节
"""
import sys
import struct


def analyze_int_layout():
    """分析 int 对象的内存布局"""
    # CPython int 实现结构（3.12+）:
    # PyLongObject:
    #   PyObject_HEAD (16 bytes: ob_refcnt + ob_type)
    #   ob_size (8 bytes): 数值位数（正/负表示符号）
    #   ob_digit[] (每项4 bytes): 30-bit digit 数组

    values = [0, 1, 127, 128, 255, 256, 2**30 - 1, 2**30, 2**60, -1]
    print("=== int 对象大小 ===")
    print(f"{'值':>20} | {'sys.getsizeof':>14} | {'digit数':>6}")
    print("-" * 50)

    for v in values:
        size = sys.getsizeof(v)
        # 每个 digit 存储 30 bits，计算需要的 digit 数
        if v == 0:
            digits = 0
        else:
            import math
            bits = v.bit_length()
            digits = (bits + 29) // 30
        print(f"{v!r:>20} | {size:>12} B | {digits:>4}")


def analyze_str_layout():
    """分析 str 对象的内存布局"""
    # CPython str 实现（compact ascii）:
    # PyASCIIObject:
    #   PyObject_HEAD (16 bytes)
    #   Py_hash_t ob_shash (8 bytes)
    #   unsigned int ob_length (内嵌在 state 字段中)
    #   ... state bitfield (1 byte)
    #   ... padding
    #   char data[] (内联数据)

    print("\n=== str 对象大小 ===")
    print(f"{'内容':>20} | {'长度':>4} | {'大小':>8} | {'每字符开销':>10}")
    print("-" * 55)

    for s in ['', 'a', 'ab', 'abc', 'hello world',
              'a' * 100, '中文测试']:
        length = len(s)
        size = sys.getsizeof(s)
        per_char = (size - sys.getsizeof('')) / max(length, 1)
        print(f"{s!r:>20} | {length:>4} | {size:>6} B | {per_char:>8.1f} B/char")


def analyze_container_layout():
    """分析容器对象的内存布局"""
    print("\n=== 容器对象大小（空 vs 含元素） ===")

    # 空容器
    containers = {
        'list': ([], [1], [1, 2], list(range(8)), list(range(100))),
        'tuple': ((), (1,), (1, 2), tuple(range(8)), tuple(range(100))),
        'dict': ({}, {1: 1}, dict(zip(range(5), range(5))),
                 dict(zip(range(8), range(8)))),
        'set': (set(), {1}, set(range(5)), set(range(8))),
    }

    for name, instances in containers.items():
        print(f"\n{name}:")
        for obj in instances:
            length = len(obj)
            size = sys.getsizeof(obj)
            print(f"  len={length:>3}, size={size:>6} B, "
                  f"per_item={size / max(length, 1):.1f} B")


def analyze_reference_cycle():
    """展示引用计数与内存释放的关系"""
    print("\n=== 引用计数与内存释放 ===")

    import weakref
    import gc

    class Node:
        def __init__(self, name):
            self.name = name
            self.next = None
        def __repr__(self):
            return f"Node({self.name!r})"

    # 创建循环引用
    a = Node('A')
    b = Node('B')
    a.next = b
    b.next = a

    weak_a = weakref.ref(a)
    weak_b = weakref.ref(b)

    print(f"创建循环引用前: a={weak_a()}, b={weak_b()}")

    # 删除强引用
    del a
    del b

    # 循环引用使对象无法通过引用计数释放
    print(f"del后（gc前）: a={weak_a()}, b={weak_b()}")

    # 手动触发 GC
    collected = gc.collect()
    print(f"GC回收了 {collected} 个对象")
    print(f"gc后: a={weak_a()}, b={weak_b()}")


if __name__ == '__main__':
    analyze_int_layout()
    analyze_str_layout()
    analyze_container_layout()
    analyze_reference_cycle()
```

### 4.4 自定义内存分配器调试

```python
"""
使用 tracemalloc 进行细粒度内存泄漏检测
模拟真实场景中的内存泄漏诊断
"""
import tracemalloc
import gc
import time
from collections import defaultdict


class DataCache:
    """模拟一个有内存泄漏的缓存类"""

    def __init__(self):
        self._cache = {}
        self._access_log = []  # 泄漏点：持续增长的日志列表

    def get(self, key):
        if key not in self._cache:
            self._cache[key] = list(range(1000))
            self._access_log.append(f"Cache miss for key: {key}")
            # 泄漏：每次访问都追加日志，从不清理
        return self._cache[key]


def simulate_workload(cache, iterations=100):
    """模拟工作负载"""
    for i in range(iterations):
        key = f"key_{i % 50}"  # 50个不同的key
        cache.get(key)


def diagnose_leak():
    """使用 tracemalloc 诊断内存泄漏"""
    tracemalloc.start(25)  # 存储25帧traceback

    cache = DataCache()

    # 基准快照
    gc.collect()
    baseline = tracemalloc.take_snapshot()

    # 运行工作负载
    simulate_workload(cache, 100)
    gc.collect()
    snap1 = tracemalloc.take_snapshot()

    # 继续运行
    simulate_workload(cache, 500)
    gc.collect()
    snap2 = tracemalloc.take_snapshot()

    # 分析增量
    print("=== 第一阶段内存增量 TOP 5 ===")
    for stat in snap1.compare_to(baseline, 'lineno')[:5]:
        print(f"  {stat}")

    print("\n=== 第二阶段内存增量 TOP 5 ===")
    for stat in snap2.compare_to(snap1, 'lineno')[:5]:
        print(f"  {stat}")

    # 按traceback追踪（最详细的定位方式）
    print("\n=== 持续增长分配的调用栈 ===")
    diff_by_trace = snap2.compare_to(snap1, 'traceback')
    for stat in diff_by_trace[:3]:
        print(f"\n  分配大小: {stat.size_diff / 1024:.1f} KiB")
        for line in stat.traceback.format():
            print(f"    {line}")

    # 检测持续增长的文件
    print("\n=== 按文件分组的增长趋势 ===")
    diff_by_file = snap2.compare_to(snap1, 'filename')
    for stat in diff_by_file[:5]:
        filename = stat.traceback[0].filename
        # 只显示项目文件，跳过标准库
        if 'site-packages' not in filename and 'lib/python' not in filename:
            print(f"  {filename}: {stat.size_diff / 1024:.1f} KiB 增长")

    tracemalloc.stop()


if __name__ == '__main__':
    diagnose_leak()
```

### 4.5 内存池行为实验

```python
"""
通过 sys.getsizeof 和实际分配测量
验证 pymalloc 的 size class 行为
"""
import sys
import gc


def measure_actual_overhead():
    """测量 Python 对象的实际内存开销"""
    print("=== Python 对象内存开销详细分析 ===\n")

    # int 对象：值越大，内存越多
    print("--- int 对象 ---")
    for exp in range(0, 70, 10):
        val = 2 ** exp if exp > 0 else 0
        size = sys.getsizeof(val)
        print(f"  2^{exp:>2} = {val if exp <= 30 else f'2^{exp}':>15}, "
              f"sizeof = {size} bytes")

    # list 对象：预分配策略
    print("\n--- list 对象（预分配策略）---")
    last_capacity = 0
    for target_len in [0, 1, 2, 3, 4, 5, 8, 16, 32, 64, 128]:
        lst = list(range(target_len))
        size = sys.getsizeof(lst)
        # CPython list 的预分配公式:
        # new_allocated = (newsize + (newsize >> 3) + 6) & ~3
        # 但实际容量可以通过大小变化推断
        print(f"  len={target_len:>4}, sizeof={size:>6} bytes")

    # dict 对象：哈希表预分配
    print("\n--- dict 对象（哈希表预分配）---")
    for target_len in [0, 1, 2, 3, 4, 5, 8, 16, 32]:
        d = {i: i for i in range(target_len)}
        size = sys.getsizeof(d)
        print(f"  len={target_len:>4}, sizeof={size:>6} bytes")

    # 比较内置类型与 __slots__
    print("\n--- __slots__ 内存节省 ---")

    class RegularPoint:
        def __init__(self, x, y):
            self.x = x
            self.y = y

    class SlotPoint:
        __slots__ = ('x', 'y')

        def __init__(self, x, y):
            self.x = x
            self.y = y

    regular = RegularPoint(1, 2)
    slotted = SlotPoint(1, 2)

    print(f"  RegularPoint: {sys.getsizeof(regular)} bytes "
          f"(+ __dict__: {sys.getsizeof(regular.__dict__)} bytes)")
    print(f"  SlotPoint:    {sys.getsizeof(slotted)} bytes")
    print(f"  节省: {sys.getsizeof(regular) + sys.getsizeof(regular.__dict__) - sys.getsizeof(slotted)} bytes")


def observe_pymalloc_threshold():
    """观察 pymalloc 512 字节阈值的行为"""
    print("\n\n=== pymalloc 分配阈值观察 ===")

    # 小于等于 512 字节走 pymalloc，大于 512 字节走系统 malloc
    # 我们通过 tracemalloc 来观察分配行为的差异

    import tracemalloc
    tracemalloc.start()

    # 测试不同大小
    sizes = [8, 64, 128, 256, 500, 512, 520, 1024, 4096]

    for size in sizes:
        # bytes(size) 创建指定大小的字节串
        # 实际对象大小 = PyBytesObject头(约33bytes) + size + 1(NULL)
        obj = bytes(size)
        obj_size = sys.getsizeof(obj)
        in_pymalloc = obj_size <= 512 + 33 + 8  # 头 + 对齐
        print(f"  bytes({size:>5}): 对象大小={obj_size:>4}B, "
              f"可能走pymalloc={in_pymalloc}")
        del obj

    tracemalloc.stop()


if __name__ == '__main__':
    measure_actual_overhead()
    observe_pymalloc_threshold()
```

## 5. 常见陷阱与最佳实践

### 5.1 陷阱：误解 sys.getsizeof 的含义

```python
# 错误：认为 getsizeof 返回容器及其内容的总大小
import sys

lst = [1, 'hello', [3, 4, 5]]
print(sys.getsizeof(lst))  # 只返回列表容器本身的大小，不含元素！

# 正确：递归计算总大小
def total_size(obj, seen=None):
    """递归计算对象及其引用的总大小"""
    if seen is None:
        seen = set()
    obj_id = id(obj)
    if obj_id in seen:
        return 0
    seen.add(obj_id)
    size = sys.getsizeof(obj)

    if isinstance(obj, (str, bytes, int, float, bool, type(None))):
        return size
    if isinstance(obj, (tuple, list, set, frozenset)):
        return size + sum(total_size(item, seen) for item in obj)
    if isinstance(obj, dict):
        return size + sum(
            total_size(k, seen) + total_size(v, seen)
            for k, v in obj.items()
        )
    # 自定义对象
    return size + sum(
        total_size(getattr(obj, slot), seen)
        for slot in getattr(obj, '__slots__', obj.__dict__)
        if hasattr(obj, slot)
    )

print(f"列表容器: {sys.getsizeof(lst)} bytes")
print(f"完整大小: {total_size(lst)} bytes")
```

### 5.2 陷阱：小整数缓存与字符串驻留

```python
# CPython 缓存了 -5 到 256 的整数
a = 256
b = 256
print(a is b)  # True — 使用缓存

c = 257
d = 257
print(c is d)  # False（在交互模式下；脚本中可能 True 因常量折叠）

# 字符串驻留（intern）
s1 = "hello"
s2 = "hello"
print(s1 is s2)  # True — 字面量字符串自动驻留

s3 = "".join(["he", "llo"])
print(s1 is s3)   # False — 运行时构造的字符串不自动驻留

# 显式驻留
import sys
s4 = sys.intern(s3)
s5 = sys.intern("hello")
print(s4 is s5)   # True — 驻留后相同

# 最佳实践：永远用 == 比较值，用 is 比较 None/True/False
```

### 5.3 陷阱：可变默认参数导致内存膨胀

```python
# 错误：可变默认参数在函数定义时创建，所有调用共享
def append_to(element, target=[]):
    target.append(element)
    return target

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] — 不是预期的 [2]！

# 正确：使用 None 作为哨兵
def append_to(element, target=None):
    if target is None:
        target = []
    target.append(element)
    return target
```

### 5.4 陷阱：__slots__ 的误用

```python
# __slots__ 确实省内存，但有限制：
class Good:
    __slots__ = ('x', 'y')
    # 优点：每个实例节省约 56 bytes（__dict__ 的大小）
    # 缺点：无法动态添加属性

class Bad:
    __slots__ = ('x', 'y')
    # 如果子类不定义 __slots__，子类实例仍然有 __dict__
    # 继承时 __slots__ 需要每个子类重新声明

class Parent:
    __slots__ = ('x',)

class Child(Parent):
    # 必须重新声明 __slots__，否则子类实例会有 __dict__
    __slots__ = ('y',)

# 最佳实践：
# 1. 仅在大量实例场景下使用 __slots__（数万以上）
# 2. 继承链中每个子类都要声明 __slots__
# 3. 如果需要动态属性，不要使用 __slots__
# 4. 使用 dataclass(frozen=True) 作为替代方案
```

### 5.5 最佳实践总结

| 实践 | 说明 |
|------|------|
| **使用 `__slots__`** | 大量实例时可节省 40-50% 内存 |
| **使用生成器** | `yield` 代替返回列表，避免一次性加载 |
| **字符串驻留** | 大量重复字符串时用 `sys.intern()` |
| **`del` 及时释放** | 大对象用完后 `del` 并 `gc.collect()` |
| **避免全局列表** | 全局列表导致所有元素无法回收 |
| **使用 `array` 模块** | 纯数值数组比 list 省内存 4-8 倍 |
| **NumPy 数组** | 科学计算用 ndarray 代替 Python list |
| **tracemalloc 定位** | 生产环境用 tracemalloc 而非手动估算 |
| **`weakref` 打破循环** | 缓存用弱引用避免循环引用泄漏 |
| **分批处理** | 大数据集分批处理，避免峰值内存过高 |
