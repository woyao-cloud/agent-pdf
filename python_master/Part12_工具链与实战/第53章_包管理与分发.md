# 第53章 包管理与分发

Python 的包管理生态历经演进，从 distutils 到 setuptools 再到 pyproject.toml。本章从包的构建原理、依赖解析到发布流程，深入剖析 Python 包管理的底层机制。

---

## 53.1 概念与语法

### 53.1.1 pip 包管理器

```bash
# 安装包
pip install requests
pip install requests==2.31.0        # 指定版本
pip install requests>=2.28,<3.0     # 版本范围
pip install -e .                    # 可编辑模式安装
pip install -r requirements.txt     # 从文件安装
pip install ./local-package         # 本地包

# 卸载
pip uninstall requests

# 查询
pip list                            # 已安装包列表
pip show requests                   # 包详细信息
pip freeze                          # 导出依赖（格式：package==version）
pip freeze > requirements.txt       # 保存到文件
pip check                           # 检查依赖冲突

# 升级
pip install --upgrade requests

# 从 PyPI 安装特定索引
pip install --index-url https://pypi.org/simple/ requests
pip install --extra-index-url https://my-pypi.example.com/simple/ internal-pkg
```

### 53.1.2 requirements.txt vs pyproject.toml

```text
# requirements.txt — 仅列出依赖（无元数据）
requests>=2.28
pydantic>=2.0
pytest>=7.0

# pyproject.toml — 完整的项目配置（推荐）
[project]
name = "myapp"
version = "1.0.0"
dependencies = [
    "requests>=2.28",
    "pydantic>=2.0",
]

[project.optional-dependencies]
dev = ["pytest>=7.0", "ruff>=0.1"]

# pyproject.toml 的优势：
# 1. 统一配置入口（构建、测试、格式化等）
# 2. 声明式元数据（不需要执行 setup.py）
# 3. PEP 621 标准化
# 4. 不再需要 setup.py/setup.cfg
```

### 53.1.3 构建与分发

```bash
# 安装构建工具
pip install build

# 构建 sdist 和 wheel
python -m build

# 生成文件在 dist/ 目录：
# myapp-1.0.0.tar.gz      — 源码分发包（sdist）
# myapp-1.0.0-py3-none-any.whl — 二进制包（wheel）

# sdist vs wheel
# sdist: 源码打包，安装时需要构建
# wheel: 预构建二进制包，安装更快（只需解压）

# 发布到 PyPI
pip install twine
twine upload dist/*
```

## 53.2 原理与机制

### 53.2.1 pip 的依赖解析

```python
# pip 的依赖解析流程：
# 1. 解析 pyproject.toml 或 setup.py 中的依赖声明
# 2. 从 PyPI 下载包的元数据（METADATA 文件）
# 3. 构建依赖图
# 4. 求解满足所有版本约束的解
# 5. 下载并安装

# 依赖解析是 NP 困难问题（类似 SAT 问题）
# pip 使用回溯算法求解
# 新版 pip（22.3+）使用新的解析器，更正确但可能更慢

# 版本说明符（PEP 440）
# ~=1.4.2   — 兼容版本（>=1.4.2, <1.5.0）
# ==1.0     — 精确匹配
# !=1.0     — 排除
# >=1.0     — 大于等于
# <2.0      — 小于

# 环境标记（PEP 508）
# requests[security]>=2.28; python_version>='3.10'
# pywin32; sys_platform == 'win32'
```

### 53.2.2 wheel 格式

```python
# Wheel 文件名格式：
# {distribution}-{version}(-{build tag})?-{python tag}-{abi tag}-{platform tag}.whl

# 示例：
# numpy-1.26.0-cp312-cp312-manylinux_2_17_x86_64.whl
#   cp312 — CPython 3.12
#   cp312 — ABI 版本
#   manylinux_2_17_x86_64 — 平台标签

# 纯 Python wheel：
# myapp-1.0.0-py3-none-any.whl
#   py3 — Python 3
#   none — 无 C 扩展
#   any — 任何平台

# Wheel 的安装过程：
# 1. 解压 .whl（本质是 zip 文件）
# 2. 将文件复制到 site-packages
# 3. 写入 .dist-info 元数据
# 4. 不需要执行 setup.py（比 sdist 快得多）
```

### 53.2.3 安装路径

```python
import site
print(site.getsitepackages())  # site-packages 路径
print(site.getusersitepackages())  # 用户级 site-packages

# 安装位置的查找顺序：
# 1. 虚拟环境的 site-packages（如果激活）
# 2. 用户级 site-packages（--user）
# 3. 系统 site-packages
```

## 53.3 使用场景

### 53.3.1 私有 PyPI 服务器

```bash
# 使用 pypiserver 搭建私有 PyPI
pip install pypiserver
mkdir /path/to/packages
cp dist/* /path/to/packages/
pypi-server run -p 8080 /path/to/packages

# 从私有源安装
pip install --extra-index-url http://localhost:8080/simple/ myapp

# 使用 devpi（更完整的方案）
pip install devpi-server devpi-web
devpi-server --init
devpi-server --start
```

### 53.3.2 锁定依赖版本

```bash
# pip freeze — 简单但不精确
pip freeze > requirements-lock.txt

# pip-compile（pip-tools）— 推荐
pip install pip-tools
echo 'requests>=2.28' > requirements.in
echo 'pydantic>=2.0' >> requirements.in
pip-compile requirements.in
# 生成 requirements.txt，包含精确版本和子依赖

# poetry.lock — Poetry 项目的锁文件
poetry lock
```

## 53.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 不固定依赖版本 | 使用 pip-compile 或 poetry lock |
| setup.py 中执行任意代码 | 迁移到 pyproject.toml 声明式配置 |
| 依赖冲突 | 使用 pip check 检查，虚拟环境隔离 |
| 发布包含敏感信息 | 使用 .gitignore，检查 sdist 内容 |
| 缺少版本说明符 | 明确指定 >= 和 < 上限 |
