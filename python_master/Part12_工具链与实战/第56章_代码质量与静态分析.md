### 56.1.1 类型检查：mypy

```bash

# 常见错误
# error: Incompatible types in assignment (expression has type "str", variable has type "int")

# error: Argument 1 to "foo" has incompatible type "str"; expected "int"
```

```python
# 类型注解示例
from typing import Optional, List, Dict, Union, Callable, TypeVar, Generic, Protocol

# 基本类型
def greet(name: str, times: int = 1) -> str:
    return (f"Hello, {name}! " * times).strip()

# Optional
def find_user(user_id: int) -> Optional[dict]:
    ...  # 返回 dict 或 None

# 容器类型
names: List[str] = []
scores: Dict[str, int] = {}

# Union
def process(value: Union[str, int]) -> str:
    return str(value)

# Callable
def apply(func: Callable[[int, int], int], a: int, b: int) -> int:
    return func(a, b)

# 泛型
T = TypeVar('T')

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: List[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

# Protocol（结构化子类型/鸭子类型）
class Drawable(Protocol):
    def draw(self) -> None: ...

class Circle:
    def draw(self) -> None:
        print("drawing circle")

def render(shape: Drawable) -> None:  # Circle 满足 Drawable 协议
    shape.draw()

# Python 3.10+ 简化语法
# def process(value: str | int) -> str:
# names: list[str] = []
# from collections.abc import Callable
# def apply(func: Callable[[int, int], int]) -> int:
```

## 56.2 原理与机制

### 56.2.1 mypy 的类型检查原理

```python
# mypy 的类型检查流程：
# 1. 解析源代码，构建 AST
# 2. 提取类型注解，构建类型环境
# 3. 逐函数进行类型检查
# 4. 对每个操作检查类型兼容性
# 5. 报告类型错误

# mypy 使用子类型（subtyping）关系：
# - 显式子类型：继承关系（class B(A) → B 是 A 的子类型）
# - 结构化子类型：Protocol（满足协议方法即视为子类型）

# 类型兼容性规则：
# 1. 子类型可以赋给父类型（Subtype → Supertype）
# 2. 返回类型是协变的（covariant）
# 3. 参数类型是逆变的（contravariant）
# 4. 不可变类型（如 List）是不变的（invariant）

# 示例：
class Animal: pass
class Dog(Animal): pass

# 协变：Dog → Animal（子类型关系保持）
# 逆变：Callable[[Animal], None] 是 Callable[[Dog], None] 的子类型
# 不变：List[Dog] 不是 List[Animal] 的子类型

# TypeVar 的变体
from typing import TypeVar

T_co = TypeVar('T_co', covariant=True)    # 协变
T_contra = TypeVar('T_contra', contravariant=True)  # 逆变
```

### 56.2.2 AST 分析原理

```python
# Linter 工具的核心是 AST（抽象语法树）分析
import ast

# 示例：检测未使用的导入
code = """
import os
import sys
from pathlib import Path

os.path.join('a', 'b')
"""

tree = ast.parse(code)

# 收集导入
imports = set()
used = set()

for node in ast.walk(tree):
    if isinstance(node, ast.Import):
        for alias in node.names:
            imports.add(alias.name.split('.')[0])
    elif isinstance(node, ast.ImportFrom):
        for alias in node.names:
            imports.add(alias.name)
    elif isinstance(node, ast.Name):
        used.add(node.id)

unused = imports - used
print(f"未使用的导入: {unused}")  # {'sys', 'Path'}

# Ruff 的实现使用 Rust 编写的 AST 解析器
# 速度比纯 Python 实现快 10-100 倍
# 使用 Ruff 的 Python-AFL（Abstract Syntax Tree）替代标准 ast 模块
```

## 56.3 使用场景

### 56.3.1 CI/CD 集成

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install tools
        run: pip install ruff mypy pytest

      - name: Lint
        run: ruff check src/

      - name: Format check
        run: ruff format --check src/

      - name: Type check
        run: mypy src/

      - name: Test
        run: pytest
```

### 56.3.2 pre-commit 钩子

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.1.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.0.0
    hooks:
      - id: mypy
```

```bash

pre-commit run --all-files
```

### 56.4.1 自定义类型检查规则

```python
"""自定义 mypy 插件示例"""
from typing import Callable
from mypy.plugin import Plugin, MethodContext
from mypy.types import Type


class CustomPlugin(Plugin):
    def get_method_hook(self, fullname: str) -> Callable[[MethodContext], Type] | None:
        if fullname == 'mylib.require_positive':
            return self._check_positive
        return None

    def _check_positive(self, ctx: MethodContext) -> Type:
        # 自定义类型检查逻辑
        return ctx.default()


def plugin(version: str):
    return CustomPlugin
```

## 56.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| mypy 忽略所有类型错误 | 渐进式启用：先 `--no-strict`，逐步收紧 |
| # type: ignore 滥用 | 只在必要时使用，添加具体错误码 |
| 多个 Linter 规则冲突 | 使用 Ruff 统一替代 flake8/pylint/isort |
| 类型注解影响运行时性能 | 类型注解在运行时被忽略（可用 `from __future__ import annotations` 延迟求值） |
| CI 中不运行质量检查 | 集成到 CI/CD，使用 pre-commit 钩子 |
