## 55.2 原理与机制

### 55.2.1 pdb 的底层实现

```python
# pdb 基于 bdb（Breakpoint Debugger）模块构建
# bdb 模块使用 sys.settrace() 实现断点拦截

"""
执行流程：
1. pdb.set_trace() 调用 sys.settrace(tracefunc)
2. Python 解释器在每行代码执行前调用 tracefunc
3. tracefunc 检查当前行号是否匹配任何断点
4. 如果匹配，设置 trace_dispatch 暂停执行
5. 进入交互式命令循环，等待用户输入
6. 用户输入 continue 时，恢复 trace_dispatch 继续执行
"""

# sys.settrace 的性能影响
# 启用 settrace 后，解释器每行代码都要调用 trace 函数
# 性能下降约 10-100 倍
# 因此 pdb 只适合调试，不适合生产环境

# bdb.Breakpoint 类
"""
class Breakpoint:
    number: int       # 断点编号
    file: str         # 文件路径
    line: int         # 行号
    temporary: bool   # 临时断点
    enabled: bool     # 是否启用
    cond: str         # 条件表达式
    ignore: int       # 忽略次数
    hits: int         # 命中次数
"""
```

### 55.2.2 调试器的栈帧操作

```python
import sys

# pdb 使用 sys._getframe() 操作调用栈
# 每个栈帧包含：
# f_code     — 代码对象
# f_locals   — 局部变量
# f_globals  — 全局变量
# f_back     — 上一帧
# f_lineno   — 当前行号

# pdb 的 up/down 命令实现
def up(self, count=1):
    """移动到调用栈的上层帧。"""
    frame = self.curframe
    for _ in range(count):
        if frame.f_back:
            frame = frame.f_back
    self.curframe = frame
    self.print_stack_entry()

# pdb 的交互式执行
# 在 pdb 中输入表达式时，使用 eval() 在当前帧的上下文中执行
# exec() 用于执行语句（如赋值）
# (Pdb) x = 10           # 赋值（exec）
# (Pdb) print(x + 5)     # 表达式（eval）
```

## 55.4 示例代码

### 55.4.1 调试工具类

```python
"""常用调试工具集合"""
import sys
import traceback
import time
from contextlib import contextmanager
from functools import wraps


@contextmanager
def debug_on_error():
    """异常时自动进入 pdb。"""
    try:
        yield
    except Exception:
        import pdb
        pdb.post_mortem(sys.exc_info()[2])


@contextmanager
def timer(name='block'):
    """计时上下文管理器。"""
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f'{name}: {elapsed:.4f}s')


def trace_calls(func):
    """函数调用追踪装饰器。"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f'→ {func.__name__}({args}, {kwargs})')
        try:
            result = func(*args, **kwargs)
            print(f'← {func.__name__} → {result!r}')
            return result
        except Exception as e:
            print(f'✕ {func.__name__} raised {type(e).__name__}: {e}')
            raise
    return wrapper


def dump_locals(depth=1):
    """打印调用栈中的局部变量。"""
    frame = sys._getframe(depth)
    print(f'--- Locals in {frame.f_code.co_name} ---')
    for name, value in frame.f_locals.items():
        print(f'  {name} = {value!r}')
```

## 55.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 调试后忘记移除 pdb | 使用 breakpoint() + PYTHONBREAKPOINT 控制 |
| print 调试无法追踪复杂问题 | 使用 pdb 交互式调试 |
| cProfile 开销影响结果 | 多次运行取平均，或用 py-spy 采样 |
| 多线程调试困难 | 使用 rpdb 远程调试 |
| 生产环境启用 settrace | 只在开发环境使用，生产用 logging |
