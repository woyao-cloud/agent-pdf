# 第15章 元类与类创建

> "元类就是类的类，是类的模板。如果说类是对象的蓝图，那么元类就是蓝图的蓝图。" —— Tim Peters
>
> "元类是 99% 的程序员都不需要理解的深奥魔法。如果你在犹豫是否需要它，那就是不需要。" —— Tim Peters

Python 的元类（metaclass）机制是整个对象系统中最深刻、最强大的特性之一。理解元类不仅意味着掌握 `type` 的三参数调用形式，更意味着理解 Python 从 C 层面如何将命名空间、描述符、类体执行串联为一整套完整的类创建流程。`class` 语句本身不过是一段语法糖——真正的类创建逻辑深藏在 `type.__call__`、`__build_class__`、`_PyType_FromMetaclass` 等 CPython 内部函数中。

本章将从 CPython 源码层面剖析类创建的每一步，覆盖 `type` 双重身份、`__new__`/`__init__` 在元类中的角色、`__prepare__` 自定义命名空间、`__init_subclass__` 替代方案、`__classcell__` 与 `super()` 的关联，以及元类在 ORM、路由注册、单例模式等场景中的实战应用。

## 15.1 概念与语法

### 15.1.1 type 既是类也是元类

在 Python 中，一切皆对象，类本身也是对象。既然类是对象，那么类也必须有自己的类——这就是**元类（metaclass）**。

```python
# 类是创建实例的模板
class MyClass:
    pass

obj = MyClass()
print(type(obj))     # <class '__main__.MyClass'>
print(type(MyClass))  # <class 'type'>
```

`MyClass` 是 `obj` 的类，而 `type` 是 `MyClass` 的类。因此 `type` 就是 `MyClass` 的元类。

但 `type` 的特殊之处在于它的双重身份——它既是内置函数（用于查询对象类型），又是所有类的默认元类（用于创建类对象）：

```python
# type 的双重身份
print(type(42))        # <class 'int'>      —— 单参数：返回对象的类型
print(type(int))       # <class 'type'>     —— int 的元类是 type
print(type(type))      # <class 'type'>     —— type 的元类是 type 自身！
print(type(object))    # <class 'type'>     —— object 的元类也是 type
```

这是一个自引用循环：`type(type) is type`。在 CPython 的 C 源码中，`type` 对象在解释器启动阶段通过特殊手法初始化——先创建 `PyType_Type` 结构体，再手动将 `PyType_Type.ob_type` 指向自身来打破无限递归。

**CPython 源码级分析**—— `Objects/typeobject.c` 中 `type` 对象的引导过程：

```c
/* Objects/typeobject.c - type 对象的定义与初始化 */

/* type 对象的 C 结构体定义 */
PyTypeObject PyType_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0)  /* ob_type 指向自身 */
    "type",                                     /* tp_name */
    sizeof(PyHeapTypeObject),                   /* tp_basicsize */
    /* ... 大量槽位 ... */
    type_new,                                   /* tp_new */
    type_init,                                  /* tp_init */
    /* ... */
};

/*
 * 在 PyType_Ready 中完成类型体系的自举：
 * 1. PyType_Ready(&PyType_Type) 先执行
 * 2. 此时 PyType_Type.ob_type 已经指向 &PyType_Type（自引用）
 * 3. PyType_Ready(&PyBaseObject_Type) 设置 object 的类型为 type
 * 4. 所有内置类型（int, str, list, ...）的类型均指向 type
 *
 * type 的自引用是整个类型体系的基石：
 *   type.__class__ is type           # True
 *   isinstance(type, type)           # True（type 是 type 的实例）
 *   issubclass(type, type)           # True（type 是 type 的子类）
 *   isinstance(object, type)         # True（object 是 type 的实例）
 *   isinstance(type, object)         # True（type 是 object 的子类）
 */
```

整个类型体系的层次关系：

```
type ←────────────────────────┐
  │                            │
  ├── int ──→ 42              │
  ├── str ──→ "hello"          │
  ├── list ──→ [1, 2, 3]      │
  ├── MyClass ──→ obj          │
  ├── object                   │
  │     └── 所有新式类的基类    │
  └── type ──→ 自身 ───────────┘

核心关系：
  type(object) is type            # object 的类是 type
  type(type) is type               # type 的类是 type（自引用）
  type.__bases__ == (object,)      # type 继承自 object
  object.__bases__ == ()           # object 是继承链的顶端
  isinstance(type, object)         # True: type 是 object 的实例（因为 type 继承自 object）
  isinstance(object, type)         # True: object 是 type 的实例（因为 object 是一个类）
```

**`type` 与 `object` 的哲学关系**：

```python
# type 和 object 互为彼此的实例/子类
print(isinstance(object, type))   # True  —— object 是 type 的实例
print(isinstance(type, object))   # True  —— type 是 object 的子类
print(issubclass(type, object))   # True  —— type 继承自 object
print(issubclass(object, type))   # False —— object 不继承自 type

# 这不是一个循环论证，而是 CPython 自举时的刻意设计：
# type 的 C 结构体中 tp_base 指向 object（type 继承自 object）
# object 的 C 结构体中 ob_type 指向 type（object 的元类是 type）
```

### 15.1.2 type(name, bases, dict) 创建类

`type` 的三参数调用形式是理解元类的基石。当传入三个参数时，`type` 不再返回对象的类型，而是动态创建一个新类：

```python
# type 的两种调用方式
# 1. 单参数：返回对象的类型
print(type(42))  # <class 'int'>

# 2. 三参数：动态创建类
MyClass = type('MyClass', (), {'x': 1, 'greet': lambda self: 'hello'})

# 等价于：
class MyClass:
    x = 1
    def greet(self):
        return 'hello'
```

三参数 `type` 的签名：`type(name, bases, dict)`

| 参数    | 类型          | 说明                                    |
|---------|--------------|----------------------------------------|
| `name`  | `str`         | 类的 `__name__`，决定类在运行时的名称    |
| `bases` | `tuple`       | 基类元组，决定 MRO 和继承关系，默认 `(object,)` |
| `dict`  | `mapping`     | 类的命名空间，包含属性和方法             |

```python
# 动态创建带继承关系的类
class Base:
    def base_method(self):
        return "from base"

DynamicChild = type(
    'DynamicChild',           # 类名
    (Base,),                  # 基类元组
    {                         # 命名空间
        'x': 10,
        'child_method': lambda self: "from child",
    }
)

obj = DynamicChild()
print(obj.base_method())   # from base
print(obj.child_method())  # from child
print(DynamicChild.__name__)  # DynamicChild
print(DynamicChild.__bases__)  # (<class '__main__.Base'>,)
```

**CPython 源码级分析**—— `type_call` 中的三参数路径：

```c
/* Objects/typeobject.c - type_call 简化版 */
static PyObject *
type_call(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    PyObject *obj;

    /* 单参数形式：type(obj) -> 返回 obj 的类型 */
    if (PyTuple_GET_SIZE(args) == 1 && kwds == NULL) {
        obj = PyTuple_GET_ITEM(args, 0);
        return PyObject_Type(obj);
    }

    /* 三参数形式：type(name, bases, dict) -> 创建新类 */
    if (PyTuple_GET_SIZE(args) == 3 && kwds == NULL) {
        PyObject *name = PyTuple_GET_ITEM(args, 0);
        PyObject *bases = PyTuple_GET_ITEM(args, 1);
        PyObject *dict = PyTuple_GET_ITEM(args, 2);
        return type_new_impl(type, name, bases, dict);
    }

    /* 其他情况：创建 type 的实例 */
    /* 即调用元类创建类对象（当 type 本身作为元类被调用时）*/
    obj = type->tp_new(type, args, kwds);
    if (obj == NULL)
        return NULL;

    if (!PyType_IsSubtype(Py_TYPE(obj), type)) {
        return obj;  /* __new__ 返回了非 type 实例，跳过 __init__ */
    }

    if (type->tp_init != NULL) {
        if (type->tp_init(obj, args, kwds) < 0) {
            Py_DECREF(obj);
            return NULL;
        }
    }

    return obj;
}
```

三参数形式的完整执行流程在 `type_new_impl` 中：

```c
/* Objects/typeobject.c - type_new_impl 简化版 */
static PyObject *
type_new_impl(PyTypeObject *metatype, PyObject *name,
              PyObject *bases, PyObject *dict)
{
    /* 1. 确定正确的元类（最派生元类规则） */
    PyObject *metatype_winner = calculate_metaclass(metatype, bases);
    /* 如果 metatype 不是最派生的元类，委托给真正的元类 */

    /* 2. 调用 metatype.__new__ 创建类对象 */
    /*    即 type.__new__(metatype_winner, name, bases, dict) */
    PyTypeObject *new_type = type_new(metatype_winner, name, bases, dict);

    /* 3. 调用 metatype.__init__ 初始化类对象 */
    /*    即 metatype_winner.__init__(new_type, name, bases, dict) */
    type_init(new_type, args, kwds);

    return new_type;
}
```

### 15.1.3 最派生元类规则

元类的查找遵循**最派生元类规则**（most-derived metaclass rule）：如果一个类的所有基类的元类中，存在一个元类是其他元类的子类，那么它就是最终元类；否则，元类冲突。

```python
class MetaA(type): pass
class MetaB(type): pass

class A(metaclass=MetaA): pass
class B(metaclass=MetaB): pass

# 以下会报错，因为 MetaA 和 MetaB 没有继承关系
# TypeError: metaclass conflict: the metaclass of a derived class
# must be a (non-strict) subclass of the metaclasses of all its bases
# class C(A, B): pass
```

解决元类冲突的方式：

```python
# 方式一：创建兼容的元类
class MetaCombined(MetaA, MetaB):
    """同时继承两个元类，解决冲突"""
    pass

class C(A, B, metaclass=MetaCombined):
    pass

print(type(C))  # <class '__main__.MetaCombined'>

# 方式二：显式使用 type 创建类（绕过元类推导，但可能丢失元类行为）
C = type('C', (A, B), {})
print(type(C))  # <class 'type'> —— 注意：丢失了 MetaA/MetaB 的行为
```

**元类推导的 CPython 实现**：

```c
/* Objects/typeobject.c - calculate_metaclass 简化版 */
static PyObject *
calculate_metaclass(PyTypeObject *metatype, PyObject *bases)
{
    PyTypeObject *winner = metatype;
    Py_ssize_t nbase = PyTuple_GET_SIZE(bases);

    for (Py_ssize_t i = 0; i < nbase; i++) {
        PyTypeObject *base_type = (PyTypeObject *)PyTuple_GET_ITEM(bases, i);
        PyTypeObject *base_meta = base_type->ob_type;

        if (PyType_IsSubtype(base_meta, winner)) {
            winner = base_meta;  /* 更派生的元类获胜 */
        } else if (!PyType_IsSubtype(winner, base_meta)) {
            /* 不兼容：两个元类没有继承关系 */
            PyErr_SetString(PyExc_TypeError,
                "metaclass conflict: the metaclass of a derived class "
                "must be a (non-strict) subclass of the metaclasses "
                "of all its bases");
            return NULL;
        }
    }
    return (PyObject *)winner;
}
```

推导逻辑的 Python 等价实现：

```python
def calculate_metaclass(metatype, bases):
    """最派生元类推导的 Python 等价"""
    winner = metatype
    for base in bases:
        base_meta = type(base)
        if issubclass(base_meta, type(winner)):
            # base 的元类比当前 winner 更派生
            winner = base_meta
        elif not issubclass(winner, base_meta):
            # 两者没有继承关系 -> 冲突
            raise TypeError(
                f"metaclass conflict: cannot combine "
                f"{winner.__name__} and {base_meta.__name__}"
            )
    return winner
```

### 15.1.4 自定义元类的基本语法

```python
class MyMeta(type):
    """自定义元类：在类创建时自动添加 created_at 属性"""
    def __new__(mcs, name, bases, namespace, **kwargs):
        cls = super().__new__(mcs, name, bases, namespace)
        cls.created_at = __import__('time').time()
        return cls

class Foo(metaclass=MyMeta):
    pass

print(hasattr(Foo, 'created_at'))  # True
print(type(Foo))  # <class '__main__.MyMeta'>
```

Python 3 中 `metaclass` 关键字参数的处理流程：

1. 解释器扫描类定义中的 `metaclass=XXX`
2. 收集所有基类的元类，执行最派生元类推导
3. 调用推导出的元类的 `__prepare__` 获取命名空间映射
4. 在该命名空间中执行类体代码
5. 调用元类的 `__new__` 创建类对象
6. 调用元类的 `__init__` 初始化类对象
7. 调用基类的 `__init_subclass__`
8. 调用描述符的 `__set_name__`
9. 解析 `__classcell__` 支持 `super()` 闭包

### 15.1.5 `types.new_class()` 与 `types.prepare_class()` —— 编程式创建类

除了 `class` 语句和 `type(name, bases, dict)` 之外，Python 还提供了 `types.new_class()` 作为编程式创建类的标准 API：

```python
import types

# 方式一：class 语句（声明式）
class MyClass:
    x = 10

# 方式二：type 三参数调用（动态式，但跳过了 __prepare__）
MyClass2 = type('MyClass2', (), {'x': 10})

# 方式三：types.new_class（编程式，完整走类创建流程）
def class_body(ns):
    """类体函数：接收命名空间字典，在其中定义类属性"""
    ns['x'] = 10

MyClass3 = types.new_class(
    'MyClass3',       # 类名
    (),               # 基类
    {'metaclass': type},  # 关键字参数（含元类）
    class_body        # 类体函数
)

print(MyClass3.x)  # 10
print(type(MyClass3))  # <class 'type'>
```

`types.new_class()` 与 `type()` 的关键区别：

| 特性 | `type(name, bases, dict)` | `types.new_class()` |
|------|--------------------------|---------------------|
| 是否调用 `__prepare__` | 否（直接传入 dict） | 是 |
| 是否调用 `__init_subclass__` | 否 | 是 |
| 是否调用 `__set_name__` | 否 | 是 |
| 是否支持 `__classcell__` | 否 | 是 |
| 是否执行类体代码 | 否（dict 已就绪） | 是（通过回调函数） |
| 适用场景 | 简单动态创建 | 需要完整类创建流程 |

```python
import types

# types.prepare_class() 可以单独获取元类和命名空间
# 这是 new_class 的底层步骤拆解
metaclass, namespace, kwds = types.prepare_class(
    'PreparedClass',
    (object,),
    {'metaclass': MyMeta}
)
print(f"元类: {metaclass}")          # <class '__main__.MyMeta'>
print(f"命名空间类型: {type(namespace).__name__}")  # dict
print(f"关键字参数: {kwds}")         # {}

# 然后可以手动执行类体并创建类
namespace['x'] = 10
PreparedClass = metaclass('PreparedClass', (object,), namespace, **kwds)
```

**CPython 源码级分析**—— `types.new_class()` 的实现路径：

```c
/* Python/bltinmodule.c - builtin___build_class__ 简化版 */
/* 这是 class 语句背后的核心函数 */

static PyObject *
builtin___build_class__(PyObject *self, PyObject *args, PyObject *kwds)
{
    PyObject *func;      /* 类体函数 */
    PyObject *name;      /* 类名 */
    PyObject *bases;      /* 基类元组 */
    PyObject *metatype;   /* 元类 */
    PyObject *prep;       /* __prepare__ 方法 */
    PyObject *ns;         /* 命名空间 */
    PyObject *cls;        /* 创建的类 */

    /* 1. 从关键字参数中提取 metaclass */
    metatype = _PyDict_GetItemWithError(kwds, &_Py_ID(metaclass));
    if (metatype == NULL) {
        metatype = (PyObject *)&PyType_Type;  /* 默认元类 */
    }

    /* 2. 计算最派生元类 */
    metatype = calculate_metaclass(metatype, bases);

    /* 3. 调用 __prepare__ 获取命名空间 */
    prep = PyObject_GetAttr(metatype, &_Py_ID(__prepare__));
    if (prep != NULL) {
        ns = PyObject_Call(prep, ...);  /* 调用 __prepare__ */
    } else {
        ns = PyDict_New();  /* 没有 __prepare__ 则用普通 dict */
    }

    /* 4. 执行类体函数，将结果写入命名空间 */
    /*    类体中的所有赋值操作（x = 10, def method, ...）
     *    都会通过 STORE_NAME 指令写入 ns */
    result = _PyEval_EvalCode(func, globals, ns);

    /* 5. 调用元类创建类对象 */
    /*    等价于 metatype(name, bases, ns, **kwds) */
    cls = PyObject_Call(metatype, args_for_type_call, kwds);

    return cls;
}
```

## 15.2 原理与机制

### 15.2.1 `__new__` 与 `__init__` 在元类中的角色

元类中的 `__new__` 和 `__init__` 承担完全不同的职责：

| 方法        | 调用时机           | 主要职责                          | 返回值       | 参数            |
|------------|-------------------|----------------------------------|-------------|----------------|
| `__new__`  | 类对象被创建之前    | 创建并返回类对象本身                | 类对象       | `mcs, name, bases, namespace, **kwargs` |
| `__init__` | 类对象已创建之后    | 初始化已创建的类对象属性             | None        | `cls, name, bases, namespace, **kwargs` |

注意参数命名的区别：`__new__` 的第一个参数是 `mcs`（metaclass，即元类本身），而 `__init__` 的第一个参数是 `cls`（刚创建的类对象）。

```python
class VerboseMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        print(f"  [__new__] 即将创建类: {name}")
        print(f"  [__new__] bases = {bases}")
        print(f"  [__new__] namespace keys = {list(namespace.keys())}")
        # __new__ 负责创建类对象
        cls = super().__new__(mcs, name, bases, namespace)
        print(f"  [__new__] 类对象已创建: {cls}")
        print(f"  [__new__] 类的 id = {id(cls)}")
        return cls  # 必须返回类对象

    def __init__(cls, name, bases, namespace, **kwargs):
        print(f"  [__init__] 初始化类: {name}")
        print(f"  [__init__] cls 的 id = {id(cls)}")
        # __init__ 负责初始化已创建的类对象
        super().__init__(name, bases, namespace)
        print(f"  [__init__] 初始化完成")

class MyClass(metaclass=VerboseMeta):
    x = 10
    def method(self):
        return self.x

# 输出：
#   [__new__] 即将创建类: MyClass
#   [__new__] bases = ()
#   [__new__] namespace keys = ['__module__', '__qualname__', 'x', 'method']
#   [__new__] 类对象已创建: <class '__main__.MyClass'>
#   [__new__] 类的 id = 140...
#   [__init__] 初始化类: MyClass
#   [__init__] 类的 id = 140...  （与 __new__ 中相同）
#   [__init__] 初始化完成
```

**关键细节：`__new__` 的返回值决定类对象**

```python
class TrickyMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        print(f"  [TrickyMeta.__new__] 尝试创建 {name}")
        # 返回一个已存在的类，而非创建新类
        # Python 不会调用 __init__，因为返回的类不是 mcs 的实例
        return int  # 返回 int 类对象！

class FakeClass(metaclass=TrickyMeta):
    x = 10

# 输出：
#   [TrickyMeta.__new__] 尝试创建 FakeClass
# FakeClass 实际上是 int！
print(FakeClass)      # <class 'int'>
print(FakeClass(42))  # 42
print(type(FakeClass))  # <class 'type'>  （int 的元类，不是 TrickyMeta）
```

这个例子揭示了一个重要事实：`__new__` 的返回值如果不是该元类的实例，Python 会跳过 `__init__` 的调用。

**CPython 源码验证**—— `type_call` 中的判断逻辑：

```c
/* Objects/typeobject.c - type_call 简化版 */
static PyObject *
type_call(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    PyObject *obj;

    /* 调用 tp_new (即 __new__) */
    obj = type->tp_new(type, args, kwds);
    if (obj == NULL)
        return NULL;

    /* 关键判断：只有当返回的对象是 type 的实例时才调用 __init__ */
    if (!PyType_IsSubtype(Py_TYPE(obj), type)) {
        return obj;  /* 返回类型不匹配，跳过 __init__ */
    }

    /* 调用 tp_init (即 __init__) */
    if (type->tp_init != NULL) {
        if (type->tp_init(obj, args, kwds) < 0) {
            Py_DECREF(obj);
            return NULL;
        }
    }

    return obj;
}
```

**`__new__` 控制类创建的深层机制**

`__new__` 是类创建过程中最强大的钩子，因为它可以：

1. **修改命名空间**：在类对象创建前增删属性
2. **修改基类**：动态调整继承关系
3. **拦截创建**：返回缓存的类或完全不同的对象
4. **验证约束**：确保类符合特定接口要求

```python
class ValidatingMeta(type):
    """验证类必须实现特定方法，否则拒绝创建"""

    REQUIRED_METHODS = ('save', 'delete')

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 在类创建前进行验证
        for method_name in mcs.REQUIRED_METHODS:
            # 检查命名空间或基类中是否存在该方法
            if method_name not in namespace:
                # 检查是否从基类继承
                found = False
                for base in bases:
                    if hasattr(base, method_name):
                        found = True
                        break
                if not found:
                    raise TypeError(
                        f"Class '{name}' must implement '{method_name}'"
                    )
        return super().__new__(mcs, name, bases, namespace)

# 以下会抛出 TypeError
# class BadRepo(metaclass=ValidatingMeta):
#     pass

class GoodRepo(metaclass=ValidatingMeta):
    def save(self):
        return "saved"
    def delete(self):
        return "deleted"

print(GoodRepo().save())  # saved
```

**`__new__` 中修改命名空间的注意事项**：

```python
class NamespaceModifierMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        # 注意：此时 namespace 是一个 dict（或 __prepare__ 返回的映射）
        # 修改 namespace 必须在 super().__new__ 之前进行

        # 1. 添加属性（可以在 super().__new__ 之后做，也可以之前做）
        namespace['auto_field'] = 'generated'

        # 2. 修改已有属性（必须在 super().__new__ 之前做）
        for key, value in list(namespace.items()):
            if isinstance(value, str) and value.startswith('_private:'):
                # 将字符串标记为私有，生成 property
                private_name = value.split(':')[1]
                namespace[key] = property(lambda self, n=private_name: n)

        cls = super().__new__(mcs, name, bases, namespace)

        # 3. 创建类之后也可以修改（但不如在 namespace 中修改干净）
        # cls.after_creation = 'added later'

        return cls

class MyModel(metaclass=NamespaceModifierMeta):
    name = '_private:secret'

obj = MyModel()
print(obj.name)        # 'secret' (property)
print(MyModel.auto_field)  # 'generated'
```

### 15.2.2 `__call__` 在元类中的角色 —— 控制实例创建

元类中的 `__call__` 控制着类实例化的整个流程。当你写 `MyClass()` 时，实际上是调用了 `type.__call__(MyClass)`。这就是元类实现单例模式的核心机制：

```python
class TracingMeta(type):
    """追踪实例创建的元类"""

    def __call__(cls, *args, **kwargs):
        print(f"  [__call__] 创建 {cls.__name__} 的实例")
        print(f"  [__call__] args = {args}, kwargs = {kwargs}")

        # 1. 调用 __new__ 创建实例
        instance = cls.__new__(cls, *args, **kwargs)
        print(f"  [__call__] __new__ 返回: {instance}")

        # 2. 如果 __new__ 返回的是 cls 的实例，调用 __init__
        if isinstance(instance, cls):
            cls.__init__(instance, *args, **kwargs)
            print(f"  [__call__] __init__ 完成")

        return instance

class TrackedClass(metaclass=TracingMeta):
    def __new__(cls, value):
        print(f"    [TrackedClass.__new__] value = {value}")
        return super().__new__(cls)

    def __init__(self, value):
        print(f"    [TrackedClass.__init__] value = {value}")
        self.value = value

obj = TrackedClass(42)
# 输出：
#   [__call__] 创建 TrackedClass 的实例
#   [__call__] args = (42,), kwargs = {}
#   [TrackedClass.__new__] value = 42
#   [__call__] __new__ 返回: <__main__.TrackedClass object at 0x...>
#   [TrackedClass.__init__] value = 42
#   [__call__] __init__ 完成
```

**`type.__call__` 的完整 CPython 实现**：

```c
/* Objects/typeobject.c - type_call 完整版简化 */
static PyObject *
type_call(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    PyObject *obj;
    PyObject *restype;

    /* 1. 调用 tp_new (即 __new__) */
    obj = type->tp_new(type, args, kwds);
    if (obj == NULL)
        return NULL;

    /* 2. 检查 __new__ 的返回类型 */
    restype = (PyObject *)Py_TYPE(obj);
    if (!PyType_IsSubtype((PyTypeObject *)restype, type)) {
        /* 返回类型不是 type 的子类，跳过 __init__ */
        return obj;
    }

    /* 3. 调用 tp_init (即 __init__) */
    if (type->tp_init != NULL) {
        if (type->tp_init(obj, args, kwds) < 0) {
            Py_DECREF(obj);
            return NULL;
        }
    }

    return obj;
}
```

### 15.2.3 `__prepare__` 方法

`__prepare__` 是 Python 3 引入的元类钩子，它在类体执行之前被调用，用于**控制类命名空间的类型**。这是类创建流程中**最先被调用**的钩子。

```python
class MyMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        """返回一个映射对象，用于类的命名空间"""
        print(f"  [__prepare__] 准备命名空间: {name}")
        return {}  # 默认返回普通 dict
```

**执行时机与流程**：

```
class Foo(metaclass=MyMeta, option='value'):
    x = 10

等价于：
1. ns = MyMeta.__prepare__('Foo', (), option='value')    # 步骤1
2. 在 ns 中执行类体代码（x = 10 写入 ns）                # 步骤2
3. Foo = MyMeta('Foo', (), ns, option='value')            # 步骤3
```

**经典应用一：有序命名空间**

在 Python 3.6 之前，`dict` 是无序的，元类需要返回 `OrderedDict` 来保持属性定义顺序：

```python
from collections import OrderedDict

class OrderedMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        return OrderedDict()

    def __new__(mcs, name, bases, namespace, **kwargs):
        # namespace 是 OrderedDict，保持定义顺序
        print(f"属性定义顺序: {list(namespace.keys())}")
        return super().__new__(mcs, name, bases, namespace)

class Struct(metaclass=OrderedMeta):
    name: str
    age: int
    email: str

# 输出：属性定义顺序: ['__module__', '__qualname__', 'name', 'age', 'email']
# 注意：Python 3.7+ dict 默认有序，OrderedDict 在此场景下不再必需
```

**经典应用二：自定义映射类型**

`__prepare__` 可以返回任意实现了映射协议的对象，用于拦截类体中的所有赋值操作：

```python
class TracingDict(dict):
    """记录所有属性赋值操作的命名空间"""

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.assignments = []  # 记录赋值历史

    def __setitem__(self, key, value):
        import traceback
        # 记录赋值操作和调用栈
        self.assignments.append({
            'key': key,
            'value': value,
            'stack': ''.join(traceback.format_stack()[-4:-1])
        })
        super().__setitem__(key, value)

class TracingMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        return TracingDict()

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 打印赋值历史
        if hasattr(namespace, 'assignments'):
            for a in namespace.assignments:
                print(f"  赋值: {a['key']} = {a['value']}")
        return super().__new__(mcs, name, bases, dict(namespace))

class Debugged(metaclass=TracingMeta):
    x = 10
    y = 20

# 输出：
#   赋值: __module__ = __main__
#   赋值: __qualname__ = Debugged
#   赋值: x = 10
#   赋值: y = 20
```

**`__prepare__` 接收关键字参数**：

```python
class ConfigurableMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, *, case='snake', **kwargs):
        """根据配置返回不同的命名空间映射"""
        if case == 'upper':
            return UpperCaseDict()
        return {}

class UpperCaseDict(dict):
    """将所有键转为大写的命名空间"""
    def __setitem__(self, key, value):
        if not key.startswith('__') or not key.endswith('__'):
            key = key.upper()
        super().__setitem__(key, value)

class MyModel(metaclass=ConfigurableMeta, case='upper'):
    field_name = 'value'

print(hasattr(MyModel, 'FIELD_NAME'))  # True
print(hasattr(MyModel, 'field_name'))  # False
```

**CPython 源码级分析**—— `__prepare__` 的调用位置：

```c
/* Python/bltinmodule.c - builtin___build_class__ 简化版 */
static PyObject *
builtin___build_class__(PyObject *self, PyObject *args, PyObject *kwds)
{
    PyObject *func, *name, *bases, *metatype;
    PyObject *prep, *ns;

    /* 确定元类 */
    metatype = ...;  /* 从 bases 和 metaclass 关键字参数推导 */

    /* 调用 __prepare__ */
    prep = PyObject_GetAttr(metatype, &_Py_ID(__prepare__));
    if (prep != NULL) {
        /* 存在 __prepare__，调用它获取命名空间 */
        ns = PyObject_Call(prep, ...);  /* 调用 __prepare__ */
        Py_DECREF(prep);
    } else {
        /* 没有 __prepare__ 则用普通 dict */
        PyErr_Clear();
        ns = PyDict_New();
    }

    /* 执行类体（func 是类体的代码对象） */
    /* 类体中的赋值操作会直接写入 ns */
    PyObject *result = _PyEval_EvalCode(func, globals, ns);

    /* 调用元类创建类对象 */
    cls = PyObject_Call(metatype, ...);

    return cls;
}
```

**Python 3.7+ 与 3.12 的变化**：

从 Python 3.7 开始，`dict` 保证插入顺序。Python 3.12 更进一步，移除了为类命名空间使用特殊有序 `dict` 的内部机制（PEP 412 的部分优化被移除），类命名空间现在统一使用普通 `dict`。这意味着 `__prepare__` 返回 `OrderedDict` 在 3.7+ 中不再必需（对于保持顺序这一目的），但 `__prepare__` 的其他用途（如拦截赋值、自定义键处理）仍然有效。

### 15.2.4 `__init_subclass__` 钩子

Python 3.6 引入了 `__init_subclass__`，这是一个**不需要元类**就能在子类创建时获得通知的机制。它的设计目标是减少元类的使用频率——大多数情况下，你只需要在子类创建时做一些简单的注册或初始化，而不需要控制类创建过程本身。

```python
class PluginBase:
    """插件基类：自动注册所有子类"""

    _registry = {}

    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)  # 重要：必须调用 super()
        print(f"  [__init_subclass__] 注册子类: {cls.__name__}")
        PluginBase._registry[cls.__name__] = cls

print("创建 PluginA:")
class PluginA(PluginBase):
    pass
# 输出：[__init_subclass__] 注册子类: PluginA

print("\n创建 PluginB:")
class PluginB(PluginBase):
    pass
# 输出：[__init_subclass__] 注册子类: PluginB

print(f"\n注册表: {PluginBase._registry}")
# {'PluginA': <class '__main__.PluginA'>, 'PluginB': <class '__main__.PluginB'>}
```

**`__init_subclass__` 接收关键字参数**：

```python
class Describable:
    def __init_subclass__(cls, description='', **kwargs):
        super().__init_subclass__(**kwargs)
        cls.description = description

class Animal(Describable, description='A living creature'):
    pass

print(Animal.description)  # A living creature
```

**`__init_subclass__` 的调用时机**——它发生在元类的 `__new__` 和 `__init__` **之后**：

```python
class OrderMeta(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        print(f"  [1] {name}: Meta.__new__")
        return super().__new__(mcs, name, bases, ns)

    def __init__(cls, name, bases, ns, **kwargs):
        print(f"  [2] {name}: Meta.__init__")
        super().__init__(name, bases, ns)

class OrderBase(metaclass=OrderMeta):
    def __init_subclass__(cls, **kwargs):
        print(f"  [3] {cls.__name__}: __init_subclass__")
        super().__init_subclass__(**kwargs)

class Child(OrderBase):
    pass

# 输出：
#   [1] Child: Meta.__new__
#   [2] Child: Meta.__init__
#   [3] Child: __init_subclass__
```

**CPython 源码级分析**—— `__init_subclass__` 的调用位置：

```c
/* Objects/typeobject.c - type_new_impl 中的 __init_subclass__ 调用 */
/* 简化版：展示调用链 */

static PyObject *
type_new_impl(PyTypeObject *metatype, PyObject *name,
              PyObject *bases, PyObject *dict)
{
    /* ... 创建类对象（通过 __new__） ... */
    /* ... 初始化类对象（通过 __init__） ... */

    /* 调用 __init_subclass__ */
    /* 遍历所有基类，查找 __init_subclass__ 并调用 */
    PyObject *init_subclass_result = call_init_subclass(type, bases, kwds);

    /* 调用 __set_name__ */
    set_names(type);

    return type;
}

/*
 * __init_subclass__ 的调用顺序：
 * 1. type.__new__ 创建类对象
 * 2. type.__init__ 初始化类对象
 * 3. 调用基类的 __init_subclass__（按 MRO 顺序）
 * 4. 调用描述符的 __set_name__
 * 5. 设置 __classcell__
 */
```

**`__init_subclass__` vs 元类对比**：

| 特性                          | `__init_subclass__` | 元类             |
|-------------------------------|--------------------|-----------------|
| 控制类创建前的命名空间          | 不可以              | 可以（`__prepare__`） |
| 修改类创建过程                | 不可以              | 可以（`__new__`） |
| 初始化已创建的类               | 可以               | 可以（`__init__`） |
| 控制实例创建过程               | 不可以              | 可以（`__call__`） |
| 复杂度                        | 低                 | 高              |
| 调试难度                      | 低                 | 高              |
| 多重继承兼容性                | 好                 | 需要注意元类冲突  |
| 适用场景                      | 简单钩子、注册     | 深层控制、ORM    |

**经验法则**：如果你不需要 `__prepare__` 或 `__new__`，就不需要元类，用 `__init_subclass__` 就够了。

### 15.2.5 `__set_name__` 描述符钩子

`__set_name__` 在类创建完成后被调用，用于通知描述符它所属的类和属性名。这个钩子解决了描述符需要知道自己在类中的名称这一常见问题。

```python
class ValidatedField:
    """描述符：在类创建时获知自己的名称"""

    def __init__(self, validator, error_msg=''):
        self.validator = validator
        self.error_msg = error_msg

    def __set_name__(self, owner, name):
        """在类创建时自动调用，告知描述符所属的类和属性名"""
        print(f"  [__set_name__] {owner.__name__}.{name} 描述符被设置")
        self.name = name
        self.private_name = f'_{name}'

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return getattr(obj, self.private_name, None)

    def __set__(self, obj, value):
        if not self.validator(value):
            raise ValueError(
                f'{self.name}: {self.error_msg} (got {value!r})')
        setattr(obj, self.private_name, value)

class Person:
    name = ValidatedField(str.istitle, '必须为标题大小写格式')
    age = ValidatedField(lambda x: isinstance(x, int) and 0 <= x < 150,
                        '必须为 0-149 的整数')

# 输出：
#   [__set_name__] Person.name 描述符被设置
#   [__set_name__] Person.age 描述符被设置

p = Person()
p.name = 'Alice'
p.age = 30
print(p.name)  # Alice
print(p.age)   # 30

# 以下会抛出 ValueError
# p.age = 200
```

**`__set_name__` 的 CPython 调用逻辑**：

```c
/* Objects/typeobject.c - set_names 简化版 */
/* 在类创建的最后阶段，遍历类字典调用 __set_name__ */

static int
set_names(PyTypeObject *type)
{
    PyObject *names = PyDict_Keys(type->tp_dict);
    for (Py_ssize_t i = 0; i < PyList_GET_SIZE(names); i++) {
        PyObject *name = PyList_GET_ITEM(names, i);
        PyObject *value = PyDict_GetItem(type->tp_dict, name);

        /* 检查是否有 __set_name__ 方法 */
        PyObject *set_name = PyObject_GetAttr(value, &_Py_ID(__set_name__));
        if (set_name != NULL) {
            /* 调用 value.__set_name__(type, name) */
            PyObject_Call(set_name, (type, name), NULL);
            Py_DECREF(set_name);
        }
    }
    return 0;
}
```

### 15.2.6 `__classcell__` 与 `super()` 的关联

`__classcell__` 是 Python 类创建过程中的一个隐藏机制，用于支持零参数 `super()` 语法。理解它对避免元类中的隐蔽 Bug 至关重要。

```python
# Python 3 中零参数 super() 的工作原理
class Parent:
    def greet(self):
        return "Hello from Parent"

class Child(Parent):
    def greet(self):
        # super() 无参数，但 Python 如何知道要调用 Parent？
        # 答案：编译器在类体中看到 super() 时，
        # 会创建一个 __classcell__ 引用来传递类对象
        return super().greet() + " and Child"

print(Child().greet())  # Hello from Parent and Child
```

**`__classcell__` 的 CPython 实现原理**：

```c
/*
 * Python 3 的零参数 super() 依赖 __classcell__：
 *
 * 1. 编译器在编译类体时，如果检测到使用 super()（无参数形式），
 *    会在类体的局部变量中插入一个 __classcell__ 单元格变量。
 *
 * 2. 类体执行时，__classcell__ 被创建为一个 Cell 对象（闭包引用）。
 *
 * 3. 类创建完成后（type.__new__ 之后），CPython 会将 __classcell__
 *    指向刚创建的类对象。
 *
 * 4. super() 在运行时通过 __classcell__ 获取当前类，从而确定 MRO。
 *
 * 如果元类在 __new__ 中过滤了 __classcell__，super() 将无法工作！
 */
```

**`__classcell__` 陷阱演示**：

```python
class BreakSuperMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        # 危险：移除 __classcell__ 会导致 super() 失败
        # namespace.pop('__classcell__', None)  # 千万不要这样做！
        # 正确做法：传递完整的 namespace 给 super().__new__
        return super().__new__(mcs, name, bases, namespace)

class Base:
    def method(self):
        return "base"

class Child(Base, metaclass=BreakSuperMeta):
    def method(self):
        # super() 依赖 __classcell__
        return super().method() + " -> child"

print(Child().method())  # base -> child（正常工作）
```

如果移除了 `__classcell__`：

```python
class BreakSuperMetaBad(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        # 危险：移除 __classcell__
        namespace.pop('__classcell__', None)  # 导致 super() 崩溃！
        return super().__new__(mcs, name, bases, namespace)

class ChildBad(Base, metaclass=BreakSuperMetaBad):
    def method(self):
        return super().method()  # RuntimeError: super(): __class__ cell not found
```

### 15.2.7 类创建的完整流程

当 Python 解释器遇到一个 `class` 语句时，完整的创建流程如下：

```

                    │    1. 确定元类（最派生规则）    │
                    └──────────────┬──────────────┘
                                   │
                    ┌──────────────▼──────────────┐
                    │ 2. 调用 __prepare__ 获取命名空间  │

                                   │
                    ┌──────────────▼──────────────┐
                    │ 3. 执行类体，填充命名空间       │

                    └──────────────┬──────────────┘

                    ┌──────────────▼──────────────┐
                    │ 4. 调用 __new__ 创建类对象      │

                    ┌──────────────▼──────────────┐
                    │ 5. 调用 __init__ 初始化类对象    │
                    │    (metaclass.__init__)        │
                    └──────────────┬──────────────┘
                                   │
                    ┌──────────────▼──────────────┐
                    │ 6. 调用 __init_subclass__     │
                    │    (在父类上，通知子类创建)       │

                    │ 7. 调用 __set_name__           │
                    │    (通知描述符它们所属的类和名称)  │

                    ┌──────────────▼──────────────┐
                    │ 8. 设置 __classcell__           │

```

**完整的可运行示例**——追踪类创建的每一个步骤：

```python
import sys

class StepTracker(type):
    """逐步追踪类创建的完整流程"""

    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        print(f"[步骤1] __prepare__ 被调用")
        print(f"  name = {name!r}")
        print(f"  bases = {bases}")
        print(f"  kwargs = {kwargs}")
        print(f"  返回命名空间映射对象")
        namespace = {}
        namespace['_created_by_prepare'] = True
        return namespace

    def __new__(mcs, name, bases, namespace, **kwargs):
        print(f"\n[步骤2] __new__ 被调用")
        print(f"  mcs = {mcs}")
        print(f"  name = {name!r}")
        print(f"  bases = {bases}")
        print(f"  namespace keys = {list(namespace.keys())}")
        print(f"  kwargs = {kwargs}")
        cls = super().__new__(mcs, name, bases, namespace)
        print(f"  类对象已创建: {cls}")
        return cls

    def __init__(cls, name, bases, namespace, **kwargs):
        print(f"\n[步骤3] __init__ 被调用")
        print(f"  cls = {cls}")
        print(f"  name = {name!r}")
        super().__init__(name, bases, namespace)
        print(f"  初始化完成")

class TrackedBase(metaclass=StepTracker):

    def __init_subclass__(cls, version='0.0', **kwargs):

        super().__init_subclass__(**kwargs)

        self.name = name
```

### 15.3.1 ORM 模型注册

元类在 ORM 框架中最经典的应用是模型注册——自动将模型类注册到全局注册表中，而无需手动注册。

```python
class ModelRegistry(type):
    """ORM 模型注册元类"""

    _registry = {}  # {表名: 模型类}

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 抽象基类不需要注册
        if namespace.get('__abstract__', False):
            return super().__new__(mcs, name, bases, namespace)

        cls = super().__new__(mcs, name, bases, namespace)

        # 收集字段定义
        fields = {}
        for key, value in list(namespace.items()):
            if isinstance(value, Field):
                fields[key] = value
        cls._fields = fields

        # 确定表名
        table_name = getattr(cls, '__table__', name.lower())
        cls._table = table_name

        # 注册模型
        mcs._registry[table_name] = cls
        print(f"  [注册] 模型 {name} -> 表 {table_name}, 字段: {list(fields.keys())}")

        return cls

    @classmethod
    def get_model(mcs, table_name):
        return mcs._registry.get(table_name)

    @classmethod
    def get_all_models(mcs):
        return dict(mcs._registry)

class Field:
    """字段描述符基类"""
    def __init__(self, field_type, primary_key=False, nullable=True, default=None):
        self.field_type = field_type
        self.primary_key = primary_key
        self.nullable = nullable
        self.default = default
        self.name = None

    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get(self.name, self.default)

    def __set__(self, obj, value):
        if value is None and not self.nullable:
            raise ValueError(f"{self.name} 不能为 None")
        obj.__dict__[self.name] = value

class Model(metaclass=ModelRegistry):
    """ORM 模型基类"""
    __abstract__ = True  # 抽象基类，不会注册

    def __init__(self, **kwargs):
        for name, field in self._fields.items():
            value = kwargs.get(name, field.default)
            setattr(self, name, value)

    def __repr__(self):
        fields = ', '.join(
            f'{k}={v!r}' for k, v in self.__dict__.items()
            if not k.startswith('_')
        )
        return f'{self.__class__.__name__}({fields})'

class User(Model):
    __table__ = 'users'
    id = Field(int, primary_key=True, nullable=False)
    name = Field(str, nullable=False)
    email = Field(str, nullable=False)
    age = Field(int, default=0)

class Product(Model):
    id = Field(int, primary_key=True, nullable=False)
    name = Field(str, nullable=False)
    price = Field(float, default=0.0)

# 输出：
#   [注册] 模型 User -> 表 users, 字段: ['id', 'name', 'email', 'age']
#   [注册] 模型 Product -> 表 product, 字段: ['id', 'name', 'price']

# 使用注册表
print(ModelRegistry.get_all_models())
# {'users': <class 'User'>, 'product': <class 'Product'>}

user = User(id=1, name='Alice', email='alice@example.com', age=25)
print(user)  # User(id=1, name='Alice', email='alice@example.com', age=25)
```

### 15.3.2 API 路由注册

Web 框架中，元类可以自动收集路由定义并注册到应用中：

```python
class RouteMeta(type):
    """API 路由注册元类"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        cls = super().__new__(mcs, name, bases, namespace)

        # 收集所有带 _route 前缀的方法
        routes = {}
        for attr_name, attr_value in list(namespace.items()):
            route_info = getattr(attr_value, '_route', None)
            if route_info:
                method, path = route_info
                routes[(method, path)] = attr_value

        # 继承父类的路由
        for base in bases:
            base_routes = getattr(base, '_routes', {})
            for key, handler in base_routes.items():
                if key not in routes:
                    routes[key] = handler

        cls._routes = routes

        if routes and not namespace.get('__abstract__', False):
            print(f"  [路由注册] {name}:")
            for (method, path), handler in routes.items():
                print(f"    {method} {path} -> {handler.__name__}")

        return cls

def route(method, path):
    """路由装饰器"""
    def decorator(func):
        func._route = (method, path)
        return func
    return decorator

class ControllerBase(metaclass=RouteMeta):
    """控制器基类"""
    __abstract__ = True

class UserController(ControllerBase):
    """用户控制器"""

    @route('GET', '/users')
    def list_users(self):
        return [{'id': 1, 'name': 'Alice'}]

    @route('GET', '/users/{user_id}')
    def get_user(self, user_id):
        return {'id': int(user_id), 'name': 'Alice'}

    @route('POST', '/users')
    def create_user(self):
        return {'id': 2, 'name': 'Bob'}

class AdminController(ControllerBase):
    """管理员控制器"""

    @route('GET', '/admin/dashboard')
    def dashboard(self):
        return {'stats': 'admin dashboard'}

# 输出：
#   [路由注册] UserController:
#     GET /users -> list_users
#     GET /users/{user_id} -> get_user
#     POST /users -> create_user
#   [路由注册] AdminController:
#     GET /admin/dashboard -> dashboard

# 查找路由处理器
handler = UserController._routes[('GET', '/users')]
print(handler(UserController()))  # [{'id': 1, 'name': 'Alice'}]
```

### 15.3.3 单例模式

元类实现的单例模式是线程安全的，且在类定义时就保证了单例语义。相比 `__new__` 实现，元类实现控制了整个实例创建流程（`__call__`），避免了 `__init__` 重复调用的问题。

```python
import threading

class SingletonMeta(type):
    """线程安全的单例元类"""

    _instances = {}
    _lock = threading.Lock()

    def __call__(cls, *args, **kwargs):
        # 双重检查锁定模式（Double-Checked Locking）
        if cls not in cls._instances:
            with cls._lock:
                if cls not in cls._instances:
                    print(f"  [SingletonMeta] 创建 {cls.__name__} 的唯一实例")
                    instance = super().__call__(*args, **kwargs)
                    cls._instances[cls] = instance
        return cls._instances[cls]

    @classmethod
    def reset(mcs, cls):
        """重置单例（仅用于测试）"""
        with mcs._lock:
            mcs._instances.pop(cls, None)

class DatabaseConnection(metaclass=SingletonMeta):
    """数据库连接（单例）"""

    def __init__(self, host='localhost', port=5432):
        self.host = host
        self.port = port
        self.connected = False
        print(f"  连接到 {self.host}:{self.port}")

    def connect(self):
        self.connected = True
        return f"已连接到 {self.host}:{self.port}"

# 多次创建只会产生一个实例
db1 = DatabaseConnection(host='localhost', port=5432)
db2 = DatabaseConnection(host='other-host', port=3306)  # 参数被忽略

print(db1 is db2)  # True
print(db1.host)     # localhost（第一次创建时的参数）
print(db1.connect())  # 已连接到 localhost:5432

# 线程安全性测试
import concurrent.futures

def create_instance():
    return DatabaseConnection()

with concurrent.futures.ThreadPoolExecutor(max_workers=10) as executor:
    futures = [executor.submit(create_instance) for _ in range(10)]
    instances = [f.result() for f in futures]

# 所有实例都是同一个对象
print(all(inst is instances[0] for inst in instances))  # True
```

**为什么元类实现比 `__new__` 实现更好？**

```python
# 方式一：在 __new__ 中实现单例（有严重问题）
class SingletonByNew:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    # 问题：__init__ 每次都会被调用！
    def __init__(self, value=0):
        self.value = value  # 每次调用 SingletonByNew() 都会重置 value

s1 = SingletonByNew(value=10)
s2 = SingletonByNew(value=20)
print(s1.value)  # 20 —— 被第二次调用覆盖了！

# 方式二：元类实现（正确）
# 元类的 __call__ 控制了整个实例创建流程
# __init__ 只在第一次创建时调用
# 后续调用直接返回缓存实例，不再调用 __init__
```

### 15.3.4 接口验证（抽象基类增强）

元类可以强制子类实现特定接口，比 `abc.ABC` 提供更丰富的验证能力：

```python
class InterfaceMeta(type):
    """接口验证元类：强制子类实现所有抽象方法"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        cls = super().__new__(mcs, name, bases, namespace)

        # 接口本身不需要验证
        if namespace.get('__is_interface__', False):
            cls.__abstract_methods__ = frozenset(
                name for name, value in namespace.items()
                if getattr(value, '__is_abstract__', False)
            )
            return cls

        # 收集所有需要实现的抽象方法
        abstract_methods = set()
        for base in cls.__mro__:
            if hasattr(base, '__abstract_methods__'):
                abstract_methods.update(base.__abstract_methods__)

        # 检查是否所有抽象方法都已实现
        missing = abstract_methods - set(namespace.keys())
        # 也检查继承的方法
        missing -= {name for name in abstract_methods
                     if any(hasattr(b, name) and not getattr(
                         getattr(b, name), '__is_abstract__', False
                     ) for b in cls.__mro__ if b is not object)}

        if missing and not namespace.get('__is_interface__', False):
            raise TypeError(
                f"类 '{name}' 必须实现以下抽象方法: {sorted(missing)}"
            )

        return cls

def abstract_method(func):
    """标记抽象方法"""
    func.__is_abstract__ = True
    return func

class IRepository(metaclass=InterfaceMeta):
    """仓储接口"""
    __is_interface__ = True

    @abstract_method
    def find_by_id(self, id):
        pass

    @abstract_method
    def save(self, entity):
        pass

    @abstract_method
    def delete(self, id):
        pass

# 以下会抛出 TypeError
# class IncompleteRepo(IRepository):
#     def find_by_id(self, id):
#         pass
# TypeError: 类 'IncompleteRepo' 必须实现以下抽象方法: ['delete', 'save']

class CompleteRepo(IRepository):
    def find_by_id(self, id):
        return {'id': id}

    def save(self, entity):
        return True

    def delete(self, id):
        return True

repo = CompleteRepo()
print(repo.find_by_id(1))  # {'id': 1}
```

### 15.3.5 使用 `__init_subclass__` 的替代方案

Python 3.6 引入 `__init_subclass__` 后，许多原来需要元类的场景可以用更简单的方式实现：

```python
# 场景：自动注册子类 —— 使用 __init_subclass__ 替代元类

class Shape:
    """图形基类：自动注册所有子类"""

    _registry = {}

    def __init_subclass__(cls, register=True, **kwargs):
        super().__init_subclass__(**kwargs)
        if register and not getattr(cls, '__abstract__', False):
            name = getattr(cls, 'shape_type', cls.__name__.lower())
            Shape._registry[name] = cls
            print(f"  [注册] {cls.__name__} -> {name}")

    @classmethod
    def create(cls, shape_type, **kwargs):
        """工厂方法：根据类型名创建实例"""
        klass = cls._registry.get(shape_type)
        if klass is None:
            raise ValueError(f"未知的图形类型: {shape_type}")
        return klass(**kwargs)

class Circle(Shape, shape_type='circle'):
    def __init__(self, radius=1.0):
        self.radius = radius

    def area(self):
        import math
        return math.pi * self.radius ** 2

class Rectangle(Shape, shape_type='rectangle'):
    def __init__(self, width=1.0, height=1.0):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

# 输出：
#   [注册] Circle -> circle
#   [注册] Rectangle -> rectangle

# 使用工厂方法创建
shape = Shape.create('circle', radius=5)
print(shape.area())  # 78.53981633974483

print(Shape._registry)
# {'circle': <class 'Circle'>, 'rectangle': <class 'Rectangle'>}
```

### 15.3.6 `__mro_entries__` —— 动态基类替换

Python 3.7 引入了 `__mro_entries__`，允许类在继承时动态替换自身。这是 `typing.Generic` 等特性的底层机制：

```python
class DynamicBase:
    """动态基类：根据条件替换实际基类"""

    def __init_subclass__(cls, base_type='default', **kwargs):
        super().__init_subclass__(**kwargs)
        cls._base_type = base_type

    def __class_getitem__(cls, params):
        """支持 Generic 风格的参数化"""
        return ParameterizedBase(cls, params)

class ParameterizedBase:
    """参数化的基类代理"""

    def __init__(self, origin, params):
        self.origin = origin
        self.params = params

    def __mro_entries__(self, bases):
        """在继承时动态替换基类"""
        # 返回一个元组，用于替换 bases 中的自身
        print(f"  [__mro_entries__] 参数: {self.params}")
        if self.params == 'list':
            return (list,)
        elif self.params == 'dict':
            return (dict,)
        return (self.origin,)

# __mro_entries__ 让 bases 中的对象在计算 MRO 时被替换
class MyList(DynamicBase['list']):
    pass

class MyDict(DynamicBase['dict']):
    pass

print(MyList.__mro__)  # (<class 'MyList'>, <class 'list'>, <class 'object'>)
print(MyDict.__mro__)  # (<class 'MyDict'>, <class 'dict'>, <class 'object'>)
print(isinstance(MyList(), list))  # True
print(isinstance(MyDict(), dict))  # True
```

### 15.4.2 使用元类实现 Python 风格的声明式 DSL

```python
"""
使用元类实现声明式 DSL
模拟 SQLAlchemy 风格的模型定义
"""

class Column:
    """列定义描述符"""

    def __init__(self, col_type, primary_key=False, nullable=True,
                 default=None, unique=False):
        self.col_type = col_type
        self.primary_key = primary_key
        self.nullable = nullable
        self.default = default
        self.unique = unique
        self.name = None
        self.table = None

    def __set_name__(self, owner, name):
        self.name = name
        self.table = owner.__name__

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj._data.get(self.name, self.default)

    def __set__(self, obj, value):
        if value is None and not self.nullable:
            raise ValueError(f"{self.table}.{self.name} 不允许为 None")
        if value is not None and not isinstance(value, self.col_type):
            raise TypeError(
                f"{self.table}.{self.name} 期望 {self.col_type.__name__}, "
                f"得到 {type(value).__name__}"
            )
        obj._data[self.name] = value

    def __repr__(self):
        pk = ' PRIMARY KEY' if self.primary_key else ''
        null = ' NOT NULL' if not self.nullable else ''
        return f'{self.col_type.__name__}{pk}{null}'


class TableMeta(type):
    """表元类：收集列定义并生成 SQL"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 收集列定义
        columns = {}
        # 从基类继承列
        for base in bases:
            if hasattr(base, '_columns'):
                columns.update(base._columns)

        # 从当前命名空间收集列
        for attr_name, attr_value in list(namespace.items()):
            if isinstance(attr_value, Column):
                columns[attr_name] = attr_value

        cls = super().__new__(mcs, name, bases, namespace)
        cls._columns = columns
        cls._table_name = getattr(cls, '__tablename__', name.lower())

        # 生成 SQL 工具方法
        if columns and not namespace.get('__abstract__', False):
            cls._create_sql = mcs._generate_create_sql(cls)
            print(f"  [TableMeta] 注册表: {cls._table_name}")
            print(f"    列: {list(columns.keys())}")

        return cls

    @staticmethod
    def _generate_create_sql(cls):
        cols = []
        for name, col in cls._columns.items():
            type_map = {int: 'INTEGER', str: 'TEXT', float: 'REAL', bool: 'INTEGER'}
            sql_type = type_map.get(col.col_type, 'TEXT')
            col_def = f"{name} {sql_type}"
            if col.primary_key:
                col_def += " PRIMARY KEY"
            if not col.nullable:
                col_def += " NOT NULL"
            if col.unique:
                col_def += " UNIQUE"
            cols.append(col_def)
        return f"CREATE TABLE {cls._table_name} ({', '.join(cols)})"


class Model(metaclass=TableMeta):
    """声明式模型基类"""
    __abstract__ = True

    def __init__(self, **kwargs):
        self._data = {}
        for name, col in self._columns.items():
            value = kwargs.get(name, col.default)
            if value is not None:
                setattr(self, name, value)
            elif not col.nullable:
                raise ValueError(f"缺少必填字段: {name}")

    def __repr__(self):
        fields = ', '.join(
            f'{k}={v!r}' for k, v in self._data.items()
        )
        return f'{self.__class__.__name__}({fields})'

    def save(self):
        """模拟保存操作"""
        print(f"  [SAVE] INSERT INTO {self._table_name} VALUES {self._data}")


class User(Model):
    __tablename__ = 'users'

    id = Column(int, primary_key=True, nullable=False)
    username = Column(str, nullable=False, unique=True)
    email = Column(str, nullable=False, unique=True)
    age = Column(int, nullable=True, default=0)


class Post(Model):
    __tablename__ = 'posts'

    id = Column(int, primary_key=True, nullable=False)
    title = Column(str, nullable=False)
    content = Column(str, nullable=True)
    author_id = Column(int, nullable=False)
```

### 15.4.3 元类实现属性访问控制

```python
"""
使用元类实现细粒度的属性访问控制
类似于 Python 的 __slots__ 但更灵活
"""

class AccessControlMeta(type):
    """属性访问控制元类"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 收集访问控制规则
        readonly_fields = set()
        private_fields = set()
        allowed_fields = set()

        # 从命名空间收集规则
        for key, value in list(namespace.items()):
            if isinstance(value, ReadOnly):
                readonly_fields.add(key)
                namespace[key] = value.default
            elif isinstance(value, PrivateField):
                private_fields.add(key)
                namespace[key] = value.default
            elif isinstance(value, AllowedField):
                allowed_fields.add(key)
                namespace[key] = value.default

        # 从基类继承规则
        for base in bases:
            readonly_fields.update(getattr(base, '_readonly_fields', set()))
            private_fields.update(getattr(base, '_private_fields', set()))
            allowed_fields.update(getattr(base, '_allowed_fields', set()))

        cls = super().__new__(mcs, name, bases, namespace)
        cls._readonly_fields = frozenset(readonly_fields)
        cls._private_fields = frozenset(private_fields)
        cls._allowed_fields = frozenset(allowed_fields)

        return cls


class ReadOnly:
    """只读字段标记"""
    def __init__(self, default=None):
        self.default = default


class PrivateField:
    """私有字段标记"""
    def __init__(self, default=None):
        self.default = default


class AllowedField:
    """允许的字段标记"""
    def __init__(self, default=None):
        self.default = default


class SecureModel(metaclass=AccessControlMeta):
    """安全模型基类：支持属性访问控制"""

    def __setattr__(self, name, value):
        if name.startswith('_'):
            return super().__setattr__(name, value)
        if name in self._readonly_fields:
            raise AttributeError(f"'{name}' 是只读字段")
        if name in self._private_fields:
            raise AttributeError(f"'{name}' 是私有字段")
        if self._allowed_fields and name not in self._allowed_fields:
            raise AttributeError(
                f"'{name}' 不是允许的字段，"
                f"允许: {sorted(self._allowed_fields)}"
            )
        super().__setattr__(name, value)

    def __getattr__(self, name):
        if name in self._private_fields:
            raise AttributeError(f"'{name}' 是私有字段")
        raise AttributeError(
            f"'{self.__class__.__name__}' 对象没有属性 '{name}'"
        )


class UserAccount(SecureModel):
    """用户账户模型"""

    id = ReadOnly(default=None)
    username = AllowedField(default='')
    password_hash = PrivateField(default='')
    email = AllowedField(default='')
    created_at = ReadOnly(default=None)

    def __init__(self, **kwargs):
        for name in self._allowed_fields | self._readonly_fields | self._private_fields:
            if name in kwargs:
                # 只读字段只能在创建时设置
                super().__setattr__(name, kwargs[name])

    def __repr__(self):
        public = {k: v for k, v in self.__dict__.items()
                   if k not in self._private_fields and not k.startswith('_')}
        return f'{self.__class__.__name__}({public})'


# 测试
account = UserAccount(id=1, username='alice', password_hash='abc123',
                       email='alice@example.com', created_at='2024-01-01')
print(account)

# 只读字段不可修改
try:
    account.id = 999
except AttributeError as e:
    print(f"只读错误: {e}")

# 私有字段不可访问
try:
    print(account.password_hash)
except AttributeError as e:
    print(f"私有错误: {e}")

# 允许的字段可以修改
account.username = 'alice_new'
print(f"修改后: {account}")
```

### 15.4.4 `__init_subclass__` 实现类型安全的 Builder 模式

```python
"""
使用 __init_subclass__ 实现类型安全的 Builder 模式
不需要元类，利用 Python 3.6+ 的钩子机制
"""

class BuilderBase:
    """类型安全的构建器基类"""

    _required_fields = set()
    _optional_fields = {}

    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)

        # 收集所有字段信息
        required = set()
        optional = {}
        annotations = getattr(cls, '__annotations__', {})

        for field_name, field_type in annotations.items():
            if hasattr(cls, field_name):
                # 有默认值的是可选字段
                optional[field_name] = getattr(cls, field_name)
            else:
                # 无默认值的是必填字段
                required.add(field_name)

        cls._required_fields = required
        cls._optional_fields = optional

        # 为每个字段生成 setter 方法
        for field_name in required | set(optional.keys()):
            setter = cls._make_setter(field_name)
            setattr(cls, f'with_{field_name}', setter)

    @classmethod
    def _make_setter(cls, field_name):
        def setter(self, value):
            self._data[field_name] = value
            return self  # 支持链式调用
        setter.__name__ = f'with_{field_name}'
        return setter

    def __init__(self):
        self._data = {}

    def build(self):
        missing = self._required_fields - set(self._data.keys())
        if missing:
            raise ValueError(
                f"缺少必填字段: {sorted(missing)}"
            )
        # 合并默认值
        result = dict(self._optional_fields)
        result.update(self._data)
        return result


class HttpRequestBuilder(BuilderBase):
    """HTTP 请求构建器"""
    url: str           # 必填
    method: str        # 必填
    headers: dict = {}  # 可选
    timeout: int = 30   # 可选
    verify_ssl: bool = True  # 可选


# 使用构建器
request = (
    HttpRequestBuilder()
    .with_url('https://api.example.com/data')
    .with_method('POST')
    .with_headers({'Content-Type': 'application/json'})
    .with_timeout(60)
    .build()
)
print(f"构建的请求: {request}")

# 缺少必填字段会报错
try:
    HttpRequestBuilder().with_url('https://example.com').build()
except ValueError as e:
    print(f"验证错误: {e}")
```

### 15.4.5 元类组合：多层元类协作

```python
"""
元类组合：多个元类特性通过继承组合
"""

class LoggingMeta(type):
    """日志元类：记录所有方法调用"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        cls = super().__new__(mcs, name, bases, namespace)

        # 包装所有非私有方法
        for attr_name, attr_value in list(namespace.items()):
            if (callable(attr_value)
                    and not attr_name.startswith('__')
                    and not isinstance(attr_value, (classmethod, staticmethod))):
                wrapped = mcs._wrap_method(attr_value, attr_name)
                setattr(cls, attr_name, wrapped)

        return cls

    @staticmethod
    def _wrap_method(method, method_name):
        import functools
        @functools.wraps(method)
        def wrapper(*args, **kwargs):
            print(f"  [LOG] 调用 {method_name}")
            result = method(*args, **kwargs)
            print(f"  [LOG] {method_name} 返回 {result!r}")
            return result
        return wrapper


class SingletonMeta(type):
    """单例元类"""

    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]


class CombinedMeta(SingletonMeta, LoggingMeta):
    """组合元类：同时具有单例和日志功能"""
    pass


class ConfigService(metaclass=CombinedMeta):
    """配置服务：单例 + 日志"""

    def __init__(self):
        self._config = {}

    def set_value(self, key, value):
        self._config[key] = value
        return value

    def get_value(self, key, default=None):
        return self._config.get(key, default)

# 测试

svc1.set_value('db_host', 'localhost')

print(svc2.get_value('db_host'))  # localhost（共享配置）
```

### 15.5.1 陷阱：元类冲突

```python
# 陷阱：多个基类使用不兼容的元类
class MetaA(type): pass
class MetaB(type): pass

class A(metaclass=MetaA): pass
class B(metaclass=MetaB): pass

# TypeError: metaclass conflict
# class C(A, B): pass

# 解决方案：创建兼容的子元类
class MetaAB(MetaA, MetaB):
    """组合元类：解决冲突"""
    pass

class C(A, B, metaclass=MetaAB):
    pass

print(type(C))  # <class '__main__.MetaAB'>
print(isinstance(C, MetaA))  # True
print(isinstance(C, MetaB))  # True
```

**元类方法解析顺序（MRO）的影响**：

```python
class MetaA(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        print(f"  MetaA.__new__ for {name}")
        return super().__new__(mcs, name, bases, ns)

class MetaB(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        print(f"  MetaB.__new__ for {name}")
        return super().__new__(mcs, name, bases, ns)

class MetaAB(MetaA, MetaB):
    """MRO: MetaAB -> MetaA -> MetaB -> type"""
    def __new__(mcs, name, bases, ns, **kwargs):
        print(f"  MetaAB.__new__ for {name}")
        return super().__new__(mcs, name, bases, ns)

class X(metaclass=MetaAB):
    pass

# 输出：
#   MetaAB.__new__ for X
#   MetaA.__new__ for X
#   MetaB.__new__ for X
# （按照 MetaAB 的 MRO 顺序调用 super().__new__）
```

### 15.5.3 陷阱：命名空间中的特殊键

```python
# 陷阱：类命名空间中的特殊键可能被忽略或覆盖
class TrickyMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        # __module__ 和 __qualname__ 总是存在于命名空间中
        # __doc__ 可能为 None
        # __dict__ 不在 namespace 中（它是类属性，不是命名空间项）
        print(f"  __module__: {namespace.get('__module__')}")
        print(f"  __qualname__: {namespace.get('__qualname__')}")
        print(f"  __doc__: {namespace.get('__doc__')}")

        # 注意：__classcell__ 是一个特殊的内部键
        if '__classcell__' in namespace:
            print(f"  __classcell__ 存在（用于 super() 支持）")

        return super().__new__(mcs, name, bases, namespace)

class TrickyClass(metaclass=TrickyMeta):
    """这是一个文档字符串"""
    pass

# 输出：
#   __module__: __main__
#   __qualname__: 'TrickyClass'
#   __doc__: 这是一个文档字符串
```

**命名空间中的完整键列表**：

| 键名 | 说明 | 是否总是存在 |
|------|------|-------------|
| `__module__` | 类所在的模块名 | 是 |
| `__qualname__` | 类的限定名 | 是（Python 3.4+） |
| `__doc__` | 类的文档字符串 | 是（可能为 None） |
| `__classcell__` | super() 闭包引用 | 仅当类体中使用 super() 时 |
| 用户自定义属性 | 类体中定义的所有属性和方法 | 视情况而定 |

### 15.5.4 陷阱：`__classcell__` 与 `super()` 的致命关联

```python
# 如果在元类 __new__ 中过滤了 __classcell__，
# super() 将无法工作！

class BreakSuperMeta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        # 危险：移除 __classcell__ 会导致 super() 失败
        # namespace.pop('__classcell__', None)  # 千万不要这样做！

        # 正确做法：传递完整的 namespace 给 super().__new__
        return super().__new__(mcs, name, bases, namespace)

class Base:
    def method(self):
        return "base"

class Child(Base, metaclass=BreakSuperMeta):
    def method(self):
        # super() 依赖 __classcell__
        return super().method() + " -> child"

print(Child().method())  # base -> child（正常工作）
```

**`__classcell__` 的工作原理**：

```python
# Python 3 的零参数 super() 编译为：
# super().__class__  通过 __classcell__ 引用获取
# 而非 super(Child, self) 的显式形式

# 当类体编译器检测到 super() 调用时：
# 1. 在类体的局部作用域中创建一个 Cell 变量 __classcell__
# 2. 类体执行时，__classcell__ 被写入命名空间
# 3. 类创建完成后，CPython 将 __classcell__ 指向类对象
# 4. super() 通过 __classcell__ 找到当前类，确定 MRO

# 深层原因：
# super() 需要知道当前类（不是 self 的类，而是定义 super() 的类）
# 这是因为 super() 需要从 MRO 中当前类之后查找
# __classcell__ 正是传递这个信息的机制
```

**检测和防御 `__classcell__` 问题**：

```python
class SafeMeta(type):
    """安全的元类：保留 __classcell__"""

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 保存 __classcell__ 的引用
        classcell = namespace.get('__classcell__')

        # 在此处可以安全地修改其他命名空间内容
        namespace['added_by_meta'] = True

        cls = super().__new__(mcs, name, bases, namespace)

        # 验证 __classcell__ 是否正确设置
        if classcell is not None:
            # 类创建后，__classcell__ 应该指向类对象
            # 如果不正确，super() 将无法工作
            pass

        return cls
```

### 15.5.5 陷阱：`__prepare__` 返回的映射类型转换

```python
from collections import OrderedDict

class CustomNamespaceMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        return OrderedDict()

    def __new__(mcs, name, bases, namespace, **kwargs):
        # 重要：namespace 的类型是 __prepare__ 返回的类型
        print(f"  namespace 类型: {type(namespace).__name__}")

        # 如果需要修改 namespace，必须在 super().__new__ 之前做
        # 因为 super().__new__ 之后 namespace 已经被"消费"了

        # 安全的做法：将自定义映射转为普通 dict
        cls = super().__new__(mcs, name, bases, dict(namespace))

        # 此时 cls.__dict__ 是一个特殊的 mappingproxy
        # 不能直接修改 cls.__dict__
        print(f"  cls.__dict__ 类型: {type(cls.__dict__).__name__}")

        return cls

class MyClass(metaclass=CustomNamespaceMeta):
    x = 1
    y = 2

# 输出：
#   namespace 类型: OrderedDict
#   cls.__dict__ 类型: mappingproxy
```

**`__prepare__` 返回值必须满足的条件**：

| 条件 | 说明 | 违反后果 |
|------|------|---------|
| 必须是可变映射 | 类体需要写入属性 | 类体执行时报错 |
| 必须支持 `__setitem__` | 用于属性赋值 | 类体赋值报错 |
| 可以是任意映射子类 | dict, OrderedDict, 自定义映射 | 正常工作 |
| 传给 `__new__` 时类型不变 | 保留 `__prepare__` 返回的类型 | `__new__` 收到原始类型 |

### 15.5.6 陷阱：元类中的 `super()` 调用

```python
# 陷阱：元类中忘记调用 super()

class ForgetfulMeta(type):

        # 在元类继承链中尤其重要
        super().__init__(name, bases, namespace)

    # __init_subclass__ 中也必须调用 super()
    # 错误示范：
    # def __init_subclass__(cls, **kwargs):
    #     # 忘记 super().__init_subclass__(**kwargs)
    #     pass

    # def __init_subclass__(cls, **kwargs):
```

### 15.5.7 性能对比：元类 vs `__init_subclass__` vs 装饰器

### 15.5.8 最佳实践总结

**原则一：能用 `__init_subclass__` 就不用元类**

```python
# 不推荐：用元类做简单的子类通知
class RegistrationMeta(type):
    _registry = {}
    def __new__(mcs, name, bases, ns, **kwargs):
        cls = super().__new__(mcs, name, bases, ns)
        mcs._registry[name] = cls
        return cls

# 推荐：用 __init_subclass__ 替代
class Base:
    _registry = {}
    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        Base._registry[cls.__name__] = cls
```

**原则二：元类中始终调用 `super()`**

```python
class GoodMeta(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        # 修改命名空间
        ns['added_by_meta'] = True
        # 必须调用 super().__new__()
        cls = super().__new__(mcs, name, bases, ns)
        return cls

    def __init__(cls, name, bases, ns, **kwargs):
        # 必须调用 super().__init__()
        super().__init__(name, bases, ns)
        # 初始化类对象
        cls._initialized = True
```

**原则三：不要在元类中做过多的逻辑**

```python
# 不推荐：元类中做太多事
class GodMeta(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        # 注册、验证、修改、日志... 全在 __new__ 中
        mcs._registry[name] = ...  # 注册
        mcs._validate(ns)           # 验证
        mcs._transform(ns)          # 转换
        mcs._log(name)              # 日志
        return super().__new__(mcs, name, bases, ns)

# 推荐：分离关注点，元类只做调度
class CleanMeta(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        cls = super().__new__(mcs, name, bases, ns)
        # 元类只负责调度，具体逻辑委托给类方法
        cls._register()
        cls._validate()
        return cls
```

**原则四：`__prepare__` 返回的映射必须是可变的**

```python
# 错误：__prepare__ 返回不可变映射
class BadMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        return {}  # 正确：dict 是可变的
        # return MappingProxyType({})  # 错误：不可变，类体无法写入！

# 正确：如果需要自定义映射，确保实现了 __setitem__
class GoodNamespace(dict):
    """自定义命名空间：必须实现 __setitem__"""
    def __setitem__(self, key, value):
        # 可以在这里做拦截
        super().__setitem__(key, value)

class GoodMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        return GoodNamespace()
```

**原则五：避免在元类的 `__new__` 中修改基类元组**

```python
# 危险：在 __new__ 中修改 bases
class DangerousMeta(type):
    def __new__(mcs, name, bases, ns, **kwargs):
        # 添加基类——这会导致意想不到的 MRO 变化
        bases = bases + (SomeMixin,)
        return super().__new__(mcs, name, bases, ns)

# 如果确实需要添加基类，考虑使用 __init_subclass__ 或类装饰器
# 并明确文档说明基类变化
```

**原则六：使用 `__init_subclass__` 传递关键字参数时务必调用 `super()`**

```python
class Base:
    def __init_subclass__(cls, debug=False, **kwargs):
        # 必须调用 super()，否则继承链中的其他 __init_subclass__ 不会被调用
        super().__init_subclass__(**kwargs)
        cls._debug = debug

class Middle(Base):
    def __init_subclass__(cls, log_level='INFO', **kwargs):
        super().__init_subclass__(**kwargs)  # 传递 kwargs
        cls._log_level = log_level

class Leaf(Middle, debug=True, log_level='DEBUG'):
    pass

print(Leaf._debug)      # True
print(Leaf._log_level)  # DEBUG
```

**原则七：元类应该是单继承或显式组合，避免深度继承**

```python
# 不推荐：深层元类继承链
class Meta1(type): pass
class Meta2(Meta1): pass
class Meta3(Meta2): pass
class Meta4(Meta3): pass
# 理解成本高，调试困难

# 推荐：单一元类或显式组合
class CombinedMeta(Feature1Meta, Feature2Meta):
    """明确组合多个元类特性"""
    pass
```

**原则八：优先使用 `__set_name__` 替代手动设置属性名**

```python
# 不推荐：在 __init__ 中手动设置属性名
class FieldOld:
    def __init__(self, name=None):
        self.name = name  # 需要外部传入名称

class ModelOld:
    field = FieldOld(name='field')  # 重复写了两次 'field'

# 推荐：使用 __set_name__ 自动获取名称
class FieldNew:
    def __set_name__(self, owner, name):
        self.name = name  # 自动获取，无需重复

class ModelNew:
    field = FieldNew()  # __set_name__ 会自动调用
```

### 15.5.9 边界情况全面覆盖

**边界情况一：抽象基类不注册**

```python
class ModelRegistry(type):
    _registry = {}

    def __new__(mcs, name, bases, ns, **kwargs):
        cls = super().__new__(mcs, name, bases, ns)
        # 抽象基类不注册
        if not ns.get('__abstract__', False):
            mcs._registry[name] = cls
        return cls

class AbstractModel(metaclass=ModelRegistry):
    __abstract__ = True  # 不注册

class ConcreteModel(AbstractModel):
    pass  # 注册

print(ModelRegistry._registry)
# {'ConcreteModel': <class 'ConcreteModel'>}
# AbstractModel 不在注册表中
```

**边界情况二：元类与 `__slots__` 的交互**

```python
class SlotMeta(type):
    """元类与 __slots__ 的交互"""

    def __new__(mcs, name, bases, ns, **kwargs):
        # 注意：__slots__ 在 ns 中定义
        # 元类必须正确处理 __slots__
        slots = ns.get('__slots__')
        if slots:
            print(f"  [SlotMeta] {name} 定义了 __slots__: {slots}")
        return super().__new__(mcs, name, bases, ns)

class Point(metaclass=SlotMeta):
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
print(p.x, p.y)  # 1 2

# 注意：有 __slots__ 的类，实例没有 __dict__
# print(p.__dict__)  # AttributeError: 'Point' object has no attribute '__dict__'
print(hasattr(p, '__dict__'))  # False
```

**边界情况三：元类继承时 `__init__` 的调用顺序**

```python
class MetaA(type):

    def __init__(cls, name, bases, ns, **kwargs):

class MetaB(MetaA):

class MyClass(metaclass=MetaB):
```

**边界情况四：元类与类装饰器的组合**

```python
class TracingMeta(type):

        print(f"  [Meta] 创建 {name}")
        return super().__new__(mcs, name, bases, ns)

def add_greeting(cls):

# 注意：元类先执行，装饰器后执行
```

**边界情况五：`type()` 创建类与 `class` 语句的微妙差异**

```python
# class 语句会调用 __prepare__, __init_subclass__, __set_name__
# type() 不会调用这些钩子

class HookMeta(type):
    @classmethod
    def __prepare__(mcs, name, bases, **kwargs):
        print(f"  __prepare__ 被调用")
        return {}

    def __new__(mcs, name, bases, ns, **kwargs):
        print(f"  __new__ 被调用")
        return super().__new__(mcs, name, bases, ns)

print("使用 class 语句：")
class A(metaclass=HookMeta):
    pass
# 输出：__prepare__ 被调用
#       __new__ 被调用

print("\n使用 type() 创建：")
B = HookMeta('B', (), {})
# 输出：__new__ 被调用
# 注意：没有 __prepare__！因为 type() 直接传入命名空间

# 使用 types.new_class() 可以完整走类创建流程：
import types

print("\n使用 types.new_class()：")

    pass

C = types.new_class('C', (), {'metaclass': HookMeta}, body)

#       __new__ 被调用
```

```python
# Python 2 语法（已废弃）

    __metaclass__ = MyMeta
    pass

# Python 3 语法（当前）
class Foo(Base, metaclass=MyMeta):
    pass

# Python 3 动态创建类
```

**Python 3.12 的重要变化**：

```python
# Python 3.12 之前：type.__init__ 忽略多余的关键字参数
class OldMeta(type):
    pass

class OldClass(metaclass=OldMeta, extra='ignored'):
    pass
# 在 3.11 及之前：extra 被静默忽略
# 在 3.12+：extra 会传递给 __init_subclass__

# Python 3.12+ 行为
class NewMeta(type):
    def __init__(cls, name, bases, ns, **kwargs):
        super().__init__(name, bases, ns)
        print(f"  kwargs in __init__: {kwargs}")

class NewBase(metaclass=NewMeta):
    def __init_subclass__(cls, extra=None, **kwargs):
        super().__init_subclass__(**kwargs)
        print(f"  extra in __init_subclass__: {extra}")

class NewChild(NewBase, extra='hello'):
    pass
# 输出：
#   kwargs in __init__: {'extra': 'hello'}
#   extra in __init_subclass__: hello
```

**Python 3.12 引入的 `_PyType_FromMetaclass` API**：

```c

 * Python 3.12 引入了一个新的 C API：_PyType_FromMetaclass
 * 它是创建类的底层入口点，比直接调用 type.__new__ 更高效

 * 主要改进：
 * 1. 避免了通过 Python 层面调用元类的开销
 * 2. 直接在 C 层面处理元类查找和类创建
```

## 本章小结

元类是 Python 最强大的特性之一，也是最容易被滥用的特性。

**核心要点回顾**：

1. **`type` 是所有类的元类**——理解 `type` 的三参数形式是掌握元类的基础；`type` 与 `object` 互为彼此的实例和子类，这是 CPython 自举的基石
2. **类创建流程**——`__prepare__` → 类体执行 → `__new__` → `__init__` → `__init_subclass__` → `__set_name__` → `__classcell__`
3. **`__new__` 控制创建，`__init__` 控制初始化**——`__new__` 的返回值决定了后续流程；如果返回值不是元类的实例，`__init__` 会被跳过
4. **`__call__` 控制实例化**——元类的 `__call__` 方法控制 `ClassName()` 的整个流程，包括 `__new__` 和 `__init__` 的调用
5. **`__prepare__` 控制命名空间**——Python 3.7+ dict 已有序，此方法主要用于自定义映射（拦截赋值、键转换等）
6. **优先使用 `__init_subclass__`**——只在需要 `__prepare__`、`__new__` 或 `__call__` 时才使用元类
7. **始终调用 `super()`**——无论是元类方法还是 `__init_subclass__`，忘记调用 `super()` 会导致继承链断裂
8. **注意 `__classcell__`**——不要在元类中过滤掉这个特殊键，否则 `super()` 会崩溃
9. **注意性能开销**——元类比 `__init_subclass__` 慢约 70%，比类装饰器慢约 50%，但实例创建无差异
10. **`types.new_class()` 是编程式创建类的标准 API**——比 `type()` 更完整，会走完整的类创建流程

掌握元类的关键不是知道什么时候用，而是知道什么时候不用。当你觉得需要元类时，先考虑以下替代方案：

| 需求                | 优先方案                      | 元类方案                     |
|--------------------|-------------------------------|------------------------------|
| 子类注册/通知       | `__init_subclass__`           | 元类 `__new__`/`__init__`   |
| 属性验证            | `__set_name__` + 描述符       | 元类 `__new__`              |
| 类修改              | 类装饰器                      | 元类 `__new__`              |
| 自定义命名空间      | 无替代方案                    | 元类 `__prepare__`           |
| 控制实例创建        | 元类 `__call__`               | 无替代方案                   |
| 接口验证            | `abc.ABC` + `__init_subclass__` | 元类 `__new__`           |
