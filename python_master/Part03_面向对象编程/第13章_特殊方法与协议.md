# 第13章 特殊方法与协议

Python 的特殊方法（special methods），又称"双下划线方法"或"dunder methods"（double underscore 的缩写），是 Python 数据模型的核心机制。它们不是被直接调用的，而是被 Python 解释器在特定操作中隐式触发——当你写 `a + b` 时，解释器实际执行的是 `a.__add__(b)`；当你写 `len(obj)` 时，解释器调用的是 `obj.__len__()`。这套机制构成了 Python 的"协议"（protocol）体系：一组约定好的特殊方法，只要你的类实现了它们，就能无缝融入 Python 的内置操作中。

本章将完整梳理 Python 数据模型中的核心协议，深入 CPython 源码分析其底层机制，揭示特殊方法的查找规则与隐式调用的微妙差异，并给出性能对比与边界情况的全覆盖。

## 13.1 概念与语法

### 13.1.1 特殊方法总览

Python 的特殊方法按照功能域可以分为以下几类：

| 功能域 | 核心特殊方法 | 对应的内置操作 |
|--------|-------------|---------------|
| 对象创建与初始化 | `__new__`, `__init__` | `ClassName()` |
| 对象销毁 | `__del__` | 垃圾回收 |
| 对象表示 | `__repr__`, `__str__`, `__format__` | `repr()`, `str()`, `format()` |
| 比较与排序 | `__eq__`, `__ne__`, `__lt__`, `__le__`, `__gt__`, `__ge__` | `==`, `!=`, `<`, `<=`, `>`, `>=` |
| 哈希 | `__hash__` | `hash()`, `dict` 键, `set` 元素 |
| 数值运算 | `__add__`, `__radd__`, `__iadd__` 等 | `+`, `+=` 等 |
| 一元运算 | `__neg__`, `__pos__`, `__abs__`, `__invert__` | `-x`, `+x`, `abs(x)`, `~x` |
| 容器协议 | `__len__`, `__getitem__`, `__setitem__` 等 | `len()`, `[]`, `in` 等 |
| 迭代协议 | `__iter__`, `__next__` | `for`, `iter()` |
| 可调用协议 | `__call__` | `obj()` |
| 上下文管理 | `__enter__`, `__exit__` | `with` 语句 |
| 属性访问 | `__getattr__`, `__getattribute__`, `__setattr__`, `__delattr__` | `obj.attr` |
| 描述符协议 | `__get__`, `__set__`, `__delete__` | 属性访问拦截 |

### 13.1.2 对象创建：`__new__` 与 `__init__`

`__new__` 是类方法，负责创建实例对象；`__init__` 是实例方法，负责初始化已创建的对象：

```python
class MyClass:
    def __new__(cls, *args, **kwargs):
        """创建实例。必须返回一个实例对象。
        如果返回的不是 cls 的实例，__init__ 不会被调用。"""
        print(f"__new__ called with cls={cls}, args={args}")
        instance = super().__new__(cls)  # 委托给 object.__new__
        return instance

    def __init__(self, value):
        """初始化实例。self 已经是创建好的对象。"""
        print(f"__init__ called with self={self}, value={value}")
        self.value = value


obj = MyClass(42)
# __new__ called with cls=<class '__main__.MyClass'>, args=(42,)
# __init__ called with self=<__main__.MyClass object at 0x...>, value=42
```

**关键区别**：

| 特性 | `__new__` | `__init__` |
|------|----------|-----------|
| 类型 | 静态方法（自动） | 实例方法 |
| 参数 | `cls` + 构造参数 | `self` + 构造参数 |
| 返回值 | 必须返回实例对象 | 无返回值（返回 `None`） |
| 调用时机 | 先于 `__init__` | 在 `__new__` 之后 |
| 主要职责 | 分配内存、创建对象 | 初始化属性 |

### 13.1.3 对象销毁：`__del__`

`__del__` 是析构方法，在对象被垃圾回收前调用：

```python
class Resource:
    def __init__(self, name):
        self.name = name
        print(f"Resource '{self.name}' created")

    def __del__(self):
        print(f"Resource '{self.name}' destroyed")


r = Resource("db_connection")
del r  # 可能触发 __del__，但不保证立即触发
# Resource 'db_connection' destroyed
```

**重要警告**：`__del__` 的调用时机不可控，不适合做关键资源回收。应优先使用上下文管理器（`with` 语句）。

### 13.1.4 对象表示：`__repr__`、`__str__` 与 `__format__`

```python
import math

class Vector2D:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        """机器友好表示，应尽量做到 eval(repr(obj)) == obj"""
        return f"Vector2D({self.x!r}, {self.y!r})"

    def __str__(self):
        """人类友好表示"""
        return f"({self.x}, {self.y})"

    def __format__(self, format_spec):
        """format() 函数和 f-string 的协议"""
        if format_spec == 'polar':
            r = math.hypot(self.x, self.y)
            theta = math.degrees(math.atan2(self.y, self.x))
            return f"r={r:.2f}, θ={theta:.2f}°"
        elif format_spec == 'int':
            return f"({int(self.x)}, {int(self.y)})"
        # 默认回退到 __str__
        return format(str(self), format_spec)


v = Vector2D(3, 4)
print(repr(v))     # Vector2D(3, 4)
print(str(v))      # (3, 4)
print(f"{v:polar}") # r=5.00, θ=53.13°
print(f"{v:int}")   # (3, 4)
print(format(v, 'polar'))  # r=5.00, θ=53.13°
```

三种表示方法的调用优先级：

| 操作 | 调用链 |
|------|--------|
| `repr(obj)` | `obj.__repr__()` |
| `str(obj)` | `obj.__str__()` → 回退 `obj.__repr__()` |
| `format(obj, spec)` | `obj.__format__(spec)` → 回退 `obj.__str__()` |
| `f"{obj}"` | `obj.__format__('')` → `obj.__str__()` → `obj.__repr__()` |
| `f"{obj!r}"` | `obj.__repr__()` |
| `f"{obj!s}"` | `obj.__str__()` |

### 13.1.5 比较协议

Python 提供六个比较特殊方法，对应六种比较运算符：

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    def __ne__(self, other):
        # Python 3 默认从 __eq__ 派生，但可以显式定义以改变行为
        result = self.__eq__(other)
        if result is NotImplemented:
            return NotImplemented
        return not result

    def __lt__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) < (other.x, other.y)

    def __le__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) <= (other.x, other.y)

    def __gt__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) > (other.x, other.y)

    def __ge__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) >= (other.x, other.y)


p1 = Point(1, 2)
p2 = Point(1, 2)
p3 = Point(3, 4)

print(p1 == p2)  # True
print(p1 < p3)   # True
print(p3 > p1)   # True
```

**`functools.total_ordering` 简化比较协议**：

```python
from functools import total_ordering

@total_ordering
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)

    def __lt__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) < (other.x, other.y)

    # total_ordering 自动生成 __le__, __gt__, __ge__


p1 = Point(1, 2)
p2 = Point(3, 4)
print(p1 < p2)   # True
print(p1 <= p2)  # True（自动生成）
print(p2 > p1)   # True（自动生成）
print(p2 >= p1)  # True（自动生成）
```

### 13.1.6 比较与 hash 的一致性

如果重写了 `__eq__`，必须同时重写 `__hash__`，以保持一致性规则：

> **规则**：如果 `a == b` 为 `True`，则 `hash(a)` 必须等于 `hash(b)`。

```python
class GoodPoint:
    """正确：同时定义 __eq__ 和 __hash__"""
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, GoodPoint):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)

    def __hash__(self):
        return hash((self.x, self.y))


class BadPoint:
    """错误：定义了 __eq__ 但没有 __hash__，Python 会自动将 __hash__ 设为 None"""
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, BadPoint):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)


gp = GoodPoint(1, 2)
print(hash(gp))           # 正常工作
s = {gp}                  # 可以放入集合

bp = BadPoint(1, 2)
# hash(bp)                # TypeError: unhashable type: 'BadPoint'
# s = {bp}                # TypeError: unhashable type: 'BadPoint'
```

**`__hash__` 的三条规则**：

| 规则 | 说明 |
|------|------|
| 规则一 | 如果 `a == b`，则 `hash(a) == hash(b)` 必须成立 |
| 规则二 | 定义 `__eq__` 但不定义 `__hash__`，`__hash__` 被自动设为 `None`，对象不可哈希 |
| 规则三 | 不定义 `__eq__` 也不定义 `__hash__`，使用 `id()` 作为哈希源，对象可哈希但按身份比较 |

### 13.1.7 数值运算协议

Python 的数值运算协议采用**双分派**（double dispatch）机制，允许不同类型之间的运算：

```python
class Vector:
    def __init__(self, *components):
        self.components = tuple(components)

    def __add__(self, other):
        """正向运算：self + other"""
        if isinstance(other, Vector):
            if len(self.components) != len(other.components):
                return NotImplemented
            return Vector(*(a + b for a, b in zip(self.components, other.components)))
        return NotImplemented

    def __radd__(self, other):
        """反向运算：other + self（当 other 不知道如何与 self 运算时调用）"""
        if isinstance(other, (int, float)):
            return Vector(*(other + c for c in self.components))
        return NotImplemented

    def __iadd__(self, other):
        """增量运算：self += other"""
        if isinstance(other, Vector):
            if len(self.components) != len(other.components):
                return NotImplemented
            self.components = tuple(a + b for a, b in zip(self.components, other.components))
            return self  # 增量运算应返回 self
        return NotImplemented

    def __repr__(self):
        return f"Vector{self.components}"


v1 = Vector(1, 2, 3)
v2 = Vector(4, 5, 6)

print(v1 + v2)    # Vector(5, 7, 9)  — 调用 __add__
print(v1 + 10)    # NotImplemented → TypeError（本例中 int 不支持与 Vector 相加）

# 反向运算示例
class Scalar:
    def __add__(self, other):
        """Scalar 不知道如何加 Vector"""
        return NotImplemented

s = Scalar()
# s + v1  # Scalar.__add__ 返回 NotImplemented，然后尝试 v1.__radd__(s)
```

**`NotImplemented` 与 `TypeError` 的区别**：

| 概念 | 含义 | 来源 |
|------|------|------|
| `NotImplemented` | 单值常量，表示当前类型不知道如何处理该运算 | 由特殊方法返回 |
| `TypeError` | 异常类型，表示运算完全不可行 | 由解释器抛出 |

当一个运算返回 `NotImplemented` 时，解释器会尝试反射方法（reflected method）。只有两个方向都返回 `NotImplemented` 时，才会抛出 `TypeError`。

### 13.1.9 容器协议

容器协议让自定义对象表现得像内置的序列或映射：

```python
class SimpleList:
    """模拟内置列表的核心容器协议"""

    def __init__(self, *items):
        self._data = list(items)

    def __len__(self):
        """len() 函数调用"""
        return len(self._data)

    def __getitem__(self, key):
        """obj[key] 访问 — 同时支持索引和切片"""
        if isinstance(key, slice):
            # 切片：返回新实例
            return SimpleList(*self._data[key])
        return self._data[key]

    def __setitem__(self, key, value):
        """obj[key] = value 赋值"""
        self._data[key] = value

    def __delitem__(self, key):
        """del obj[key] 删除"""
        del self._data[key]

    def __contains__(self, item):
        """item in obj — 成员检测"""
        return item in self._data

    def __iter__(self):
        """for item in obj — 迭代协议"""
        return iter(self._data)

    def __reversed__(self):
        """reversed(obj) — 反转迭代"""
        return SimpleList(*reversed(self._data))

    def __repr__(self):
        return f"SimpleList{tuple(self._data)}"


sl = SimpleList(10, 20, 30, 40, 50)

print(len(sl))          # 5
print(sl[0])            # 10
print(sl[1:3])          # SimpleList(20, 30)
sl[0] = 100
print(sl[0])            # 100
print(30 in sl)         # True
print(list(sl))         # [100, 20, 30, 40, 50]
print(list(reversed(sl)))  # [50, 40, 30, 20, 100]
```

### 13.1.10 可切片对象实现

`__getitem__` 接收的 `key` 参数可以是整数索引，也可以是 `slice` 对象。实现可切片对象需要正确处理这两种情况：

```python
class Matrix:
    """二维矩阵，支持多维切片"""

    def __init__(self, data):
        self._data = [list(row) for row in data]
        self._rows = len(self._data)
        self._cols = len(self._data[0]) if self._data else 0

    def __getitem__(self, key):
        """支持单索引和切片访问"""
        if isinstance(key, tuple):
            # 多维索引/切片：m[1:3, 0:2]
            row_key, col_key = key
            rows = self._data[row_key]
            if isinstance(row_key, slice):
                # 行切片
                result = [row[col_key] for row in rows]
                return Matrix(result)
            else:
                # 单行，列可能切片
                return rows[col_key]
        else:
            # 单维索引/切片：m[1] 或 m[1:3]
            if isinstance(key, slice):
                return Matrix(self._data[key])
            return self._data[key]

    def __repr__(self):
        rows = ['[' + ', '.join(f'{x:3}' for x in row) + ']'
                for row in self._data]
        return 'Matrix([\n' + '\n'.join(rows) + '\n])'


m = Matrix([[1, 2, 3], [4, 5, 6], [7, 8, 9]])
print(m[1])         # [4, 5, 6] — 第 2 行
print(m[0:2])       # Matrix — 前两行组成的子矩阵
print(m[1, 2])      # 6 — 第 2 行第 3 列
```

### 13.1.11 可调用协议：`__call__`

通过 `__call__` 方法，实例对象可以像函数一样被调用：

```python
class Multiplier:
    """可调用对象：工厂函数风格"""
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, x):
        return self.factor * x

    def __repr__(self):
        return f"Multiplier({self.factor})"


double = Multiplier(2)
triple = Multiplier(3)

print(double(5))    # 10
print(triple(5))    # 15
print(callable(double))  # True
```

**函数对象 vs 可调用对象**的对比：

```python
import functools

# 方式 1：闭包函数
def make_multiplier_closure(factor):
    def multiplier(x):
        return factor * x
    return multiplier

# 方式 2：可调用对象
class MakeMultiplierCallable:
    def __init__(self, factor):
        self.factor = factor
    def __call__(self, x):
        return self.factor * x

# 方式 3：functools.partial
def multiply(a, b):
    return a * b
double_partial = functools.partial(multiply, 2)

# 三者行为等价
f1 = make_multiplier_closure(2)
f2 = MakeMultiplierCallable(2)
f3 = double_partial

print(f1(5))   # 10
print(f2(5))   # 10
print(f3(5))   # 10

# 但可调用对象有状态和继承的优势
print(hasattr(f2, 'factor'))  # True — 可以检查和修改状态
f2.factor = 3
print(f2(5))   # 15 — 状态可变

# partial 的属性
print(f3.func)    # <function multiply at 0x...>
print(f3.args)    # (2,)
print(f3.keywords)  # {}
```

### 13.1.12 上下文管理协议概要

上下文管理器协议由 `__enter__` 和 `__exit__` 两个方法构成，用于 `with` 语句。第 27 章将深入讲解，此处仅做概要介绍：

```python
class ManagedResource:
    """简单的上下文管理器示例"""
    def __enter__(self):
        print("获取资源")
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        print(f"释放资源（异常：{exc_type}）")
        return False  # 不抑制异常

    def do_work(self):
        print("使用资源")


with ManagedResource() as r:
    r.do_work()
# 获取资源
# 使用资源
# 释放资源（异常：None）
```

## 13.2 原理与机制

### 13.2.1 特殊方法的查找规则：类型优先于实例

这是 Python 数据模型中最微妙但也最关键的规则之一：**当 Python 解释器通过隐式语法触发特殊方法时，它直接在对象的类型（type）上查找，而不是在实例上查找**。

```python
class Sneaky:
    """尝试在实例上覆盖特殊方法 —— 不会生效！"""
    def __init__(self):
        # 在实例上设置 __add__ 属性
        self.__add__ = lambda other: "instance __add__"

    # 类上的 __add__ 才会被隐式调用
    def __add__(self, other):
        return "type __add__"


s = Sneaky()
print(s.__add__(s))   # "instance __add__" — 显式调用，查找实例属性
print(s + s)           # "type __add__" — 隐式调用，查找类型属性！
```

这个规则的 CPython 实现原理如下：

在 CPython 中，`PyNumber_Add`（对应 `+` 运算符）的执行流程为：

```c
// Objects/abstract.c（简化版逻辑）
PyObject *
PyNumber_Add(PyObject *v, PyObject *w)
{
    // 1. 直接从 v 的类型对象上查找 nb_add（即 __add__ 的 C 层对应）
    //    注意：不是从 v 的 __dict__ 中查找！
    PyNumberMethods *v_methods = Py_TYPE(v)->tp_as_number;
    if (v_methods && v_methods->nb_add) {
        result = (*v_methods->nb_add)(v, w);
        if (result != Py_NotImplemented)
            return result;
        Py_DECREF(result);
    }

    // 2. 如果正向方法返回 NotImplemented，尝试 w 的反向方法
    PyNumberMethods *w_methods = Py_TYPE(w)->tp_as_number;
    if (w_methods && w_methods->nb_add) {
        // 注意：对 w 尝试的是 nb_radd（即 __radd__）
        ...
    }
}
```

关键点在于 `Py_TYPE(v)->tp_as_number`：它直接从类型的结构体中读取函数指针，完全绕过了实例的 `__dict__`。

**隐式调用 vs 显式调用**：

| 调用方式 | 查找位置 | 示例 |
|---------|---------|------|
| 隐式调用（语法糖触发） | 类型（`type(obj)`） | `a + b`, `len(a)`, `str(a)` |
| 显式调用（手动调用方法） | 实例 → 类型（普通属性查找顺序） | `a.__add__(b)`, `a.__len__()` |

```python
class Demo:
    def __len__(self):
        return 42

d = Demo()
d.__len__ = lambda: 99  # 在实例上覆盖

print(len(d))         # 42 — 隐式调用，查找类型
print(d.__len__())    # 99 — 显式调用，查找实例
```

### 13.2.2 `__new__` 与 `__init__` 的 CPython 调用链

当我们执行 `MyClass(arg)` 时，CPython 的调用链如下：

```
MyClass(arg)
  → PyObject_Call(MyClass, (arg,), {})           # Objects/abstract.c
    → type_call(MyClass, (arg,), {})              # Objects/typeobject.c
      → MyClass.__new__(MyClass, arg)             # 创建实例
      → 如果返回的是 MyClass 的实例：
          → MyClass.__init__(instance, arg)       # 初始化实例
          → 返回 instance
      → 如果返回的不是 MyClass 的实例：
          → 跳过 __init__，直接返回 __new__ 的返回值
```

在 CPython 源码 `Objects/typeobject.c` 中的关键实现：

```c
// Objects/typeobject.c（简化版）
static PyObject *
type_call(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    PyObject *obj;

    // 步骤1：调用 __new__
    obj = type->tp_new(type, args, kwds);
    if (obj == NULL)
        return NULL;

    // 步骤2：检查 __new__ 返回值是否为 type 的实例
    // 如果不是，跳过 __init__
    if (!PyType_IsSubtype(Py_TYPE(obj), type)) {
        return obj;  // 直接返回，不调用 __init__
    }

    // 步骤3：调用 __init__
    type = Py_TYPE(obj);
    if (type->tp_init != NULL) {
        if (type->tp_init(obj, args, kwds) < 0) {
            Py_DECREF(obj);
            return NULL;
        }
    }

    return obj;
}
```

**`__new__` 返回非当前类型实例的场景**：

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, value):
        print(f"__init__ called with value={value}")
        self.value = value


# 注意：每次 Singleton() 都会调用 __init__，即使对象已存在！
s1 = Singleton(1)  # __init__ called with value=1
s2 = Singleton(2)  # __init__ called with value=2 —— 再次初始化！
print(s1.value)     # 2 — 被第二次初始化覆盖了！
print(s1 is s2)     # True — 同一个对象
```

这是单例模式中常见的陷阱：`__new__` 返回已有实例时，`__init__` 仍会被调用。正确做法是在 `__init__` 中加保护或使用模块级变量。

### 13.2.3 `__del__` 的垃圾回收交互

`__del__` 的调用时机与垃圾回收器紧密相关。CPython 使用引用计数为主、分代回收为辅的 GC 策略：

```python
import gc
import weakref

class Cycle:
    def __init__(self, name):
        self.name = name

    def __del__(self):
        print(f"Cycle('{self.name}').__del__ called")

    def __repr__(self):
        return f"Cycle('{self.name}')"


# 情况1：引用计数归零 —— 立即回收
a = Cycle("no_cycle")
del a  # 立即输出：Cycle('no_cycle').__del__ called

# 情况2：循环引用 —— 延迟到 GC 运行
print("--- 循环引用 ---")
x = Cycle("x")
y = Cycle("y")
x.ref = y  # x → y
y.ref = x  # y → x，形成循环引用
del x
del y
# 此时没有 __del__ 输出！循环引用导致对象无法立即回收

gc.collect()  # 手动触发 GC
# Cycle('x').__del__ called
# Cycle('y').__del__ called
```

**`__del__` 与循环引用的死锁**：

如果对象参与了循环引用且定义了 `__del__`，CPython 的 GC 在 Python 3.4 之前无法回收它们（因为无法确定 `__del__` 的调用顺序）。Python 3.4（PEP 442）引入了安全终结器机制，解决了这个问题：

```python
# Python 3.4+ 可以回收带 __del__ 的循环引用对象
import gc

class Node:
    def __init__(self, name):
        self.name = name
        self.next = None

    def __del__(self):
        print(f"Node('{self.name}').__del__ called")


a = Node("A")
b = Node("B")
a.next = b
b.next = a
del a
del b
gc.collect()  # Python 3.4+ 可以正确回收
# Node('A').__del__ called
# Node('B').__del__ called
```

### 13.2.4 `__repr__` 与 `__str__` 的 CPython 实现路径

`repr()` 和 `str()` 的调用路径在 CPython 中是不同的：

```c
// Objects/object.c（简化）
PyObject *
PyObject_Repr(PyObject *v)
{
    // 1. 从 Py_TYPE(v)->tp_repr 获取函数指针
    // 2. 直接调用，不经过实例 __dict__
    PyObject *res;
    res = (*Py_TYPE(v)->tp_repr)(v);
    return res;
}

PyObject *
PyObject_Str(PyObject *v)
{
    // 1. 先尝试 tp_str（对应 __str__）
    // 2. 如果 tp_str 为 NULL，回退到 tp_repr（对应 __repr__）
    PyObject *res;
    if (Py_TYPE(v)->tp_str != NULL)
        res = (*Py_TYPE(v)->tp_str)(v);
    else
        res = PyObject_Repr(v);
    return res;
}
```

这解释了为什么只定义 `__repr__` 时 `str()` 也能工作——因为 `tp_str` 为空时回退到 `tp_repr`。

### 13.2.5 比较协议的双分派机制

当执行 `a < b` 时，CPython 的执行流程：

```
a < b
  → 尝试 a.__lt__(b)
    → 如果返回 NotImplemented
      → 尝试 b.__gt__(a)（反射方法）
        → 如果也返回 NotImplemented
          → 抛出 TypeError
```

**特殊情况：同一类型的比较**。当 `a` 和 `b` 是同一类型时，`b.__gt__(a)` 不会被尝试（因为与 `a.__lt__(b)` 逻辑对称，没必要重复调用）。

```python
class A:
    def __lt__(self, other):
        print("A.__lt__ called")
        if isinstance(other, A):
            return NotImplemented
        return NotImplemented

    def __gt__(self, other):
        print("A.__gt__ called")
        if isinstance(other, A):
            return NotImplemented
        return NotImplemented


class B:
    def __lt__(self, other):
        print("B.__lt__ called")
        if isinstance(other, B):
            return NotImplemented
        return NotImplemented

    def __gt__(self, other):
        print("B.__gt__ called")
        if isinstance(other, B):
            return NotImplemented
        return NotImplemented


a = A()
b = B()

# a < b 的调用流程：
# 1. 尝试 a.__lt__(b) → 输出 "A.__lt__ called"，返回 NotImplemented
# 2. 尝试 b.__gt__(a) → 输出 "B.__gt__ called"，返回 NotImplemented
# 3. 抛出 TypeError
try:
    result = a < b
except TypeError as e:
    print(f"TypeError: {e}")
# A.__lt__ called
# B.__gt__ called
# TypeError: '<' not supported between instances of 'A' and 'B'
```

### 13.2.6 `functools.total_ordering` 的实现原理

`@total_ordering` 装饰器的工作原理是在类创建后，检查已定义的比较方法，然后用已有方法推导出缺失的方法：

```python
# functools.total_ordering 的核心逻辑（简化版）
def total_ordering(cls):
    """根据已有的一个比较方法，自动生成其他比较方法"""
    # 需要至少定义 __eq__ 和一个比较方法
    if not hasattr(cls, '__eq__'):
        raise TypeError("must define __eq__")

    # 查找已定义的比较方法
    for name in ('__lt__', '__le__', '__gt__', '__ge__'):
        if hasattr(cls, name) and getattr(cls, name) is not getattr(object, name, None):
            root = name
            break
    else:
        raise TypeError("must define at least one ordering method")

    # 根据根方法生成其他方法
    # 例如，如果定义了 __lt__：
    #   __le__ = lambda self, other: self < other or self == other
    #   __gt__ = lambda self, other: not (self < other or self == other)
    #   __ge__ = lambda self, other: not (self < other)
    def _le_from_lt(self, other):
        if self is other:
            return True
        result = self.__lt__(other)
        if result is NotImplemented:
            return NotImplemented
        return result or self == other

    def _gt_from_lt(self, other):
        if self is other:
            return False
        result = self.__lt__(other)
        if result is NotImplemented:
            return NotImplemented
        return not result and self != other

    def _ge_from_lt(self, other):
        if self is other:
            return True
        result = self.__lt__(other)
        if result is NotImplemented:
            return NotImplemented
        return not result

    # 将生成的方法添加到类上（仅当类未定义时）
    if not hasattr(cls, '__le__') or cls.__le__ is object.__le__:
        cls.__le__ = _le_from_lt
    if not hasattr(cls, '__gt__') or cls.__gt__ is object.__gt__:
        cls.__gt__ = _gt_from_lt
    if not hasattr(cls, '__ge__') or cls.__ge__ is object.__ge__:
        cls.__ge__ = _ge_from_lt

    return cls
```

**性能注意**：`@total_ordering` 生成的比较方法依赖多次调用根方法。在性能敏感场景下，应手动实现所有比较方法。

### 13.2.7 数值运算的双分派与反射机制

Python 的数值运算采用双分派（double dispatch）机制。以 `a + b` 为例：

```
a + b 的完整调用流程：
  1. 检查 type(a) 是否与 type(b) 相同，或者是 type(b) 的子类
     → 如果是，且 b 的类型定义了 __radd__，先检查是否需要优先调用 b.__radd__

  2. 如果 a 有更高优先级（或同类）：
     → 调用 a.__add__(b)
       → 如果返回 NotImplemented：
         → 调用 b.__radd__(a)
           → 如果也返回 NotImplemented：
             → 抛出 TypeError

  3. 如果 b 有更高优先级（b 的类型是 a 类型的子类且定义了 __radd__）：
     → 先调用 b.__radd__(a)
       → 如果返回 NotImplemented：
         → 调用 a.__add__(b)
           → 如果也返回 NotImplemented：
             → 抛出 TypeError
```

**子类优先规则**：

```python
class MyInt:
    """自定义整数，演示反射方法优先级"""
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        print(f"MyInt.__add__({self.value}, {other})")
        if isinstance(other, int):
            return MyInt(self.value + other)
        return NotImplemented

    def __radd__(self, other):
        print(f"MyInt.__radd__({self.value}, {other})")
        if isinstance(other, int):
            return MyInt(other + self.value)
        return NotImplemented

    def __repr__(self):
        return f"MyInt({self.value})"


class SuperInt(MyInt):
    """MyInt 的子类，覆盖了 __radd__"""
    def __radd__(self, other):
        print(f"SuperInt.__radd__({self.value}, {other})")
        if isinstance(other, int):
            return SuperInt(other + self.value)
        return NotImplemented

    def __repr__(self):
        return f"SuperInt({self.value})"


# int + MyInt：int 不知道如何加 MyInt，所以尝试 MyInt.__radd__
m = MyInt(10)
print(5 + m)
# MyInt.__radd__(10, 5)
# MyInt(15)

# int + SuperInt：SuperInt 是 MyInt 的子类，子类的 __radd__ 优先于父类的 __add__
s = SuperInt(20)
print(5 + s)
# SuperInt.__radd__(20, 5)  — 子类的反射方法优先
# SuperInt(25)
```

### 13.2.8 `__len__` 与 `__bool__` 的隐式调用路径

`__len__` 有一个特殊的隐式调用路径：当对象出现在布尔上下文中且未定义 `__bool__` 时，Python 会回退到 `__len__`：

```c
// Objects/object.c（简化）
int
PyObject_IsTrue(PyObject *v)
{
    // 1. 尝试 tp_as_async->am_await（不相关）
    // 2. 尝试 tp_as_number->nb_bool（即 __bool__）
    // 3. 如果 nb_bool 为 NULL，尝试 tp_as_sequence->sq_length（即 __len__）
    // 4. 如果 __len__ 返回 0 → False，非零 → True
    // 5. 都没有 → 默认 True
}
```

```python
class Container:
    def __len__(self):
        print("__len__ called")
        return 0

    # 没有定义 __bool__


class BoolContainer:
    def __bool__(self):
        print("__bool__ called")
        return False

    def __len__(self):
        print("__len__ called")
        return 5


c = Container()
print(bool(c))  # __len__ called → False

bc = BoolContainer()
print(bool(bc))  # __bool__ called → False（__bool__ 优先于 __len__）
```

### 13.2.9 容器协议的 CPython 调用路径

以 `len(obj)` 为例，CPython 的实现为：

```c
// Objects/abstract.c（简化版）
Py_ssize_t
PyObject_Length(PyObject *o)
{
    // 1. 尝试 mp_length（映射协议，即 __len__）
    // 2. 尝试 sq_length（序列协议，也是 __len__）
    // 3. 如果都没有，抛出 TypeError
    PySequenceMethods *sq = Py_TYPE(o)->tp_as_sequence;
    PyMappingMethods *mp = Py_TYPE(o)->tp_as_mapping;

    if (mp && mp->mp_length)
        return mp->mp_length(o);
    else if (sq && sq->sq_length)
        return sq->sq_length(o);
    else {
        PyErr_Format(PyExc_TypeError,
                     "object of type '%.200s' has no len()",
                     Py_TYPE(o)->tp_name);
        return -1;
    }
}
```

而 `obj in container` 的调用路径则涉及 `__contains__` 和 `__iter__` 的回退关系：

```
obj in container
  1. 优先调用 container.__contains__(obj)
  2. 如果没有 __contains__：
     → 迭代 container 的每个元素，逐个与 obj 比较（等价于 any(obj == item for item in container)）
```

```python
class SlowContainer:
    """没有 __contains__，in 操作会退化为线性扫描"""
    def __init__(self, data):
        self._data = data

    def __iter__(self):
        return iter(self._data)


class FastContainer:
    """有 __contains__，可以优化查找"""
    def __init__(self, data):
        self._set = set(data)

    def __contains__(self, item):
        return item in self._set  # O(1) 查找

    def __iter__(self):
        return iter(self._set)
```

### 13.2.10 `__call__` 的 CPython 实现

`__call__` 在 CPython 中的实现是通过类型对象的 `tp_call` 槽位：


### 13.2.11 `__format__` 的回退机制

`format()` 函数和 f-string 的格式化调用路径：

```
format(obj, spec)
  → type(obj).__format__(obj, spec)
    → 如果 __format__ 不存在或返回 NotImplemented：
      → 如果 spec 为空：回退到 str(obj)
      → 如果 spec 非空：抛出 TypeError
```

```python
class OnlyRepr:
    """只定义了 __repr__，没有 __format__"""
    def __repr__(self):
        return "OnlyRepr()"


# Python 3 中，如果没有定义 __format__，会回退到 __str__ → __repr__
obj = OnlyRepr()

```

Python 3 对象默认有 `__format__`，它会委托给 `__str__`（进而委托给 `__repr__`），但仅当格式说明符为空时。

## 13.3 使用场景

### 13.3.1 场景一：值对象与不可变类型

使用 `__new__` 实现真正的不可变类型：

```python
class ImmutablePoint:
    """不可变点类型，利用 __new__ 控制实例创建"""

    __slots__ = ('_x', '_y')

    def __new__(cls, x, y):
        # 在创建阶段就可以做验证
        if not isinstance(x, (int, float)) or not isinstance(y, (int, float)):
            raise TypeError("坐标必须是数字")
        instance = super().__new__(cls)
        # 在 __new__ 中设置属性（因为 __slots__ 限制在 __init__ 外赋值）
        object.__setattr__(instance, '_x', x)
        object.__setattr__(instance, '_y', y)
        return instance

    @property
    def x(self):
        return self._x

    @property
    def y(self):
        return self._y

    def __setattr__(self, name, value):
        raise AttributeError("ImmutablePoint 对象不可变")

    def __repr__(self):
        return f"ImmutablePoint({self._x}, {self._y})"

    def __eq__(self, other):
        if not isinstance(other, ImmutablePoint):
            return NotImplemented
        return (self._x, self._y) == (other._x, other._y)

    def __hash__(self):
        return hash((self._x, self._y))


p1 = ImmutablePoint(3, 4)
p2 = ImmutablePoint(3, 4)
print(p1 == p2)     # True
print(hash(p1))     # 与 hash(p2) 相同
# p1.x = 5          # AttributeError: ImmutablePoint 对象不可变
s = {p1, p2}
print(len(s))       # 1 — 哈希相等且相等，去重
```

### 13.3.2 场景二：自定义数值类型

实现一个完整的二维向量类型，覆盖所有数值运算：

```python
import math

class Vector2D:
    """完整的二维向量，实现数值运算协议"""

    __slots__ = ('_x', '_y')

    def __init__(self, x=0, y=0):
        self._x = float(x)
        self._y = float(y)

    @property
    def x(self):
        return self._x

    @property
    def y(self):
        return self._y

    # ---- 对象表示 ----

    def __repr__(self):
        return f"Vector2D({self._x}, {self._y})"

    def __str__(self):
        return f"({self._x:.2f}, {self._y:.2f})"

    def __format__(self, format_spec):
        if format_spec == 'polar':
            r = abs(self)
            theta = math.degrees(math.atan2(self._y, self._x))
            return f"(r={r:.2f}, θ={theta:.2f}°)"
        return f"({format(self._x, format_spec)}, {format(self._y, format_spec)})"

    def __abs__(self):
        return math.hypot(self._x, self._y)

    def __bool__(self):
        return self._x != 0 or self._y != 0

    # ---- 比较协议 ----

    def __eq__(self, other):
        if isinstance(other, Vector2D):
            return (self._x, self._y) == (other._x, other._y)
        return NotImplemented

    def __hash__(self):
        return hash((self._x, self._y))

    def __lt__(self, other):
        if isinstance(other, Vector2D):
            return abs(self) < abs(other)
        return NotImplemented

    def __le__(self, other):
        if isinstance(other, Vector2D):
            return abs(self) <= abs(other)
        return NotImplemented

    # ---- 一元运算 ----

    def __neg__(self):
        return Vector2D(-self._x, -self._y)

    def __pos__(self):
        return Vector2D(self._x, self._y)

    # ---- 二元运算 ----

    def __add__(self, other):
        if isinstance(other, Vector2D):
            return Vector2D(self._x + other._x, self._y + other._y)
        if isinstance(other, (int, float)):
            return Vector2D(self._x + other, self._y + other)
        return NotImplemented

    def __radd__(self, other):
        return self.__add__(other)

    def __sub__(self, other):
        if isinstance(other, Vector2D):
            return Vector2D(self._x - other._x, self._y - other._y)
        if isinstance(other, (int, float)):
            return Vector2D(self._x - other, self._y - other)
        return NotImplemented

    def __mul__(self, other):
        """向量 * 标量（标量积）或向量 * 向量（内积）"""
        if isinstance(other, (int, float)):
            return Vector2D(self._x * other, self._y * other)
        if isinstance(other, Vector2D):
            return self._x * other._x + self._y * other._y  # 内积
        return NotImplemented

    def __rmul__(self, other):
        if isinstance(other, (int, float)):
            return Vector2D(self._x * other, self._y * other)
        return NotImplemented

    def __matmul__(self, other):
        """矩阵乘法运算符 @（此处用作内积）"""
        if isinstance(other, Vector2D):
            return self._x * other._x + self._y * other._y
        return NotImplemented

    def __rmatmul__(self, other):
        return self.__matmul__(other)

    # ---- 增量运算 ----

    def __iadd__(self, other):
        if isinstance(other, Vector2D):
            self._x += other._x
            self._y += other._y
            return self
        if isinstance(other, (int, float)):
            self._x += other
            self._y += other
            return self
        return NotImplemented

    # ---- 类型转换 ----

    def __complex__(self):
        return complex(self._x, self._y)

    def __float__(self):
        return abs(self)

    def __round__(self, ndigits=None):
        if ndigits is None:
            return Vector2D(round(self._x), round(self._y))
        return Vector2D(round(self._x, ndigits), round(self._y, ndigits))
```

```python
import math
from functools import total_ordering


@total_ordering
class MyFraction:
    """自定义分数类型，完整实现特殊方法协议"""

    def __init__(self, numerator, denominator=1):
        if denominator == 0:
            raise ZeroDivisionError("分母不能为零")
        if denominator < 0:
            numerator, denominator = -numerator, -denominator
        g = math.gcd(abs(numerator), denominator)
        self._num = numerator // g
        self._den = denominator // g

    @property
    def numerator(self):
        return self._num

    @property
    def denominator(self):
        return self._den

    # ---- 对象表示 ----

    def __repr__(self):
        if self._den == 1:
            return f"MyFraction({self._num})"
        return f"MyFraction({self._num}, {self._den})"

    def __str__(self):
        if self._den == 1:
            return str(self._num)
        return f"{self._num}/{self._den}"

    def __format__(self, format_spec):
        if format_spec == 'decimal':
            return f"{self._num / self._den:.6f}"
        if format_spec == 'mixed':
            whole = self._num // self._den
            rem = self._num % self._den
            if whole == 0:
                return str(self)
            if rem == 0:
                return str(whole)
            return f"{whole} {rem}/{self._den}"
        return format(str(self), format_spec)

    # ---- 比较协议 ----

    def __eq__(self, other):
        if isinstance(other, MyFraction):
            return (self._num, self._den) == (other._num, other._den)
        if isinstance(other, (int, float)):
            return self._num / self._den == other
        return NotImplemented

    def __lt__(self, other):
        if isinstance(other, MyFraction):
            return self._num * other._den < other._num * self._den
        if isinstance(other, (int, float)):
            return self._num / self._den < other
        return NotImplemented

    def __hash__(self):
        return hash((self._num, self._den))

    # ---- 一元运算 ----

    def __neg__(self):
        return MyFraction(-self._num, self._den)

    def __pos__(self):
        return MyFraction(self._num, self._den)

    def __abs__(self):
        return MyFraction(abs(self._num), self._den)

    def __invert__(self):
        return MyFraction(self._den, self._num)

    # ---- 二元运算 ----

    def _ensure_fraction(self, other):
        """将 other 转换为 MyFraction"""
        if isinstance(other, MyFraction):
            return other
        if isinstance(other, int):
            return MyFraction(other)
        return NotImplemented

    def __add__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        return MyFraction(
            self._num * other._den + other._num * self._den,
            self._den * other._den
        )

    def __radd__(self, other):
        return self.__add__(other)

    def __sub__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        return MyFraction(
            self._num * other._den - other._num * self._den,
            self._den * other._den
        )

    def __rsub__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        return other - self

    def __mul__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        return MyFraction(self._num * other._num, self._den * other._den)

    def __rmul__(self, other):
        return self.__mul__(other)

    def __truediv__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        if other._num == 0:
            raise ZeroDivisionError("除以零")
        return MyFraction(self._num * other._den, self._den * other._num)

    def __rtruediv__(self, other):
        other = self._ensure_fraction(other)
        if other is NotImplemented:
            return NotImplemented
        return other / self

    def __pow__(self, other):
        if isinstance(other, int):
            if other >= 0:
                return MyFraction(self._num ** other, self._den ** other)
            else:
                return MyFraction(self._den ** (-other), self._num ** (-other))
        return NotImplemented

    # ---- 增量运算 ----

    def __iadd__(self, other):
        result = self + other
        if result is NotImplemented:
            return NotImplemented
        self._num = result._num
        self._den = result._den
        return self

    # ---- 类型转换 ----

    def __int__(self):
        return self._num // self._den

    def __float__(self):
        return self._num / self._den

    def __bool__(self):
        return self._num != 0

    # ---- 容器协议（分子分母作为元素）----

    def __len__(self):
        return 2  # 分子、分母

    def __getitem__(self, key):
        if key == 0:
            return self._num
        if key == 1:
            return self._den
        raise IndexError(f"索引 {key} 超出范围（0=分子, 1=分母）")
```

```python
import numbers

class SortedList:
    """始终有序的列表，展示完整序列协议"""

    def __init__(self, iterable=None):
        self._data = []
        if iterable is not None:
            self._data = sorted(iterable)

    # ---- 序列协议 ----

    def __len__(self):
        return len(self._data)

    def __getitem__(self, key):
        if isinstance(key, slice):
            return SortedList(self._data[key])
        if isinstance(key, numbers.Integral):
            return self._data[int(key)]
        raise TypeError(f"索引类型必须是整数或切片，不是 {type(key).__name__}")

    def __contains__(self, item):
        """利用有序性做二分查找，O(log n)"""
        import bisect
        idx = bisect.bisect_left(self._data, item)
        return idx < len(self._data) and self._data[idx] == item

    def __iter__(self):
        return iter(self._data)

    def __reversed__(self):
        return SortedList(reversed(self._data))

    def __eq__(self, other):
        if isinstance(other, SortedList):
            return self._data == other._data
        if isinstance(other, list):
            return self._data == other
        return NotImplemented

    def __hash__(self):
        return hash(tuple(self._data))

    def __repr__(self):
        return f"SortedList({self._data!r})"

    def __str__(self):
        return str(self._data)

    def __add__(self, other):
        if isinstance(other, SortedList):
            return SortedList(self._data + other._data)
        if isinstance(other, list):
            return SortedList(self._data + other)
        return NotImplemented

    def __mul__(self, other):
        if isinstance(other, int):
            return SortedList(self._data * other)
        return NotImplemented

    # ---- 可变序列协议 ----

        import bisect
        bisect.insort(self._data, item)

    def remove(self, item):
        """删除元素"""
        self._data.remove(item)

    def count(self, item):
        return self._data.count(item)

    def index(self, item):
        try:
            return self._data.index(item)
        except ValueError:
            raise ValueError(f"{item!r} 不在 SortedList 中")
```

## 13.5 常见陷阱与最佳实践

### 13.5.1 陷阱一：`__eq__` 与 `__hash__` 不一致

```python
# ---- 错误：只定义 __eq__，忘记定义 __hash__ ----

class BadHash:
    def __init__(self, x):
        self.x = x

    def __eq__(self, other):
        if not isinstance(other, BadHash):
            return NotImplemented
        return self.x == other.x

    # 没有定义 __hash__！Python 自动将 __hash__ 设为 None

# bh = BadHash(1)
# hash(bh)  # TypeError: unhashable type: 'BadHash'
# {bh}      # TypeError: unhashable type: 'BadHash'

# ---- 正确：同时定义 __eq__ 和 __hash__ ----

class GoodHash:
    def __init__(self, x):
        self.x = x

    def __eq__(self, other):
        if not isinstance(other, GoodHash):
            return NotImplemented
        return self.x == other.x

    def __hash__(self):
        return hash(self.x)

gh = GoodHash(1)
print(hash(gh))  # 正常工作
s = {gh}          # 可以放入集合
```

**规则**：

| 定义了 `__eq__` | 定义了 `__hash__` | 行为 |
|----------------|-------------------|------|
| 否 | 否 | 默认按 `id()` 哈希，按 `id()` 比较 |
| 是 | 否 | `__hash__` 被设为 `None`，对象不可哈希 |
| 是 | 是 | 按自定义逻辑哈希和比较 |
| 否 | 是 | 允许但危险，可能导致不一致 |

### 13.5.2 陷阱二：`__new__` 中返回非当前类型实例

```python
class QuestionableNew:
    def __new__(cls, value):
        # 返回 int 而非 QuestionableNew 实例
        # __init__ 不会被调用！
        if isinstance(value, int):
            return value  # 返回 int 实例，跳过 __init__
        return super().__new__(cls)

    def __init__(self, value):
        print(f"__init__ called with {value}")
        self.value = value


q1 = QuestionableNew(42)
print(type(q1))   # <class 'int'> — 不是 QuestionfulNew！
print(q1)          # 42

q2 = QuestionableNew("hello")
# __init__ called with hello
print(type(q2))   # <class 'QuestionableNew'>
```

**最佳实践**：`__new__` 应该总是返回当前类的实例。如果需要返回其他类型，应该使用工厂函数而非 `__new__`。

### 13.5.3 陷阱三：`__del__` 的不可控性

```python
# ---- 错误：依赖 __del__ 做资源清理 ----

class DatabaseConnection:
    """错误示范：依赖 __del__ 关闭连接"""
    def __init__(self, url):
        self.url = url
        self.conn = f"connected to {url}"  # 模拟连接

    def __del__(self):
        # 问题1：不确定何时调用
        # 问题2：如果 __del__ 引用了已回收的对象，会出错
        # 问题3：异常处理中的 __del__ 可能不执行
        print(f"Closing connection to {self.url}")
        self.conn = None


# ---- 正确：使用上下文管理器 ----

class DatabaseConnectionFixed:
    """正确做法：使用上下文管理器"""
    def __init__(self, url):
        self.url = url
        self._conn = None

    def __enter__(self):
        self._conn = f"connected to {self.url}"
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        if self._conn is not None:
            print(f"Closing connection to {self.url}")
            self._conn = None
        return False

    def query(self, sql):
        if self._conn is None:
            raise RuntimeError("连接未打开")
        return f"result of {sql}"


with DatabaseConnectionFixed("localhost:5432") as db:
    print(db.query("SELECT 1"))
# 确定性地关闭连接
```

### 13.5.4 陷阱四：增量运算符不返回 `self`

```python
class BuggyVector:
    """错误：__iadd__ 返回新对象而非 self"""
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __iadd__(self, other):
        # 错误！返回新对象而非修改 self
        return BuggyVector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"BuggyVector({self.x}, {self.y})"


v = BuggyVector(1, 2)
print(f"before: {v}")     # BuggyVector(1, 2)
v += BuggyVector(3, 4)    # v 被重新绑定到新对象！
print(f"after: {v}")      # BuggyVector(4, 6) — 看似正确
# 但如果有其他引用指向 v，它们不会被更新


# ---- 正确做法 ----

class FixedVector:
    """正确：__iadd__ 修改 self 并返回 self"""
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __iadd__(self, other):
        self.x += other.x
        self.y += other.y
        return self  # 关键：必须返回 self

    def __repr__(self):
        return f"FixedVector({self.x}, {self.y})"


v = FixedVector(1, 2)
alias = v          # 另一个引用指向 v
v += FixedVector(3, 4)
print(f"v: {v}")       # FixedVector(4, 6)
print(f"alias: {alias}")  # FixedVector(4, 6) — alias 也看到了修改
print(v is alias)    # True — 同一个对象
```

### 13.5.5 陷阱五：比较方法返回 `NotImplemented` vs 抛出异常

```python
# ---- 错误：在比较方法中抛出 TypeError ----

class WrongComparison:
    def __lt__(self, other):
        # 错误！应该返回 NotImplemented 让 Python 尝试反射方法
        raise TypeError("'<' not supported")


class RightComparison:
    def __lt__(self, other):
        # 正确！返回 NotImplemented
        return NotImplemented


# 错误做法：抛异常会中断反射机制
# w = WrongComparison()
# w < 42  # TypeError: '<' not supported — 没有机会让 int 尝试比较

# 正确做法：返回 NotImplemented 允许反射
# r = RightComparison()
# r < 42  # 尝试 RightComparison.__lt__(42) → NotImplemented
#         # 然后尝试 int.__gt__(r) → 可能也返回 NotImplemented
#         # 最终 TypeError: '<' not supported between instances of ...
#         # 但至少给了反射机制一个机会
```

### 13.5.6 陷阱六：`__repr__` 与 `__str__` 的选择

```python
class PoorRepr:
    """只定义 __str__，交互式输出不友好"""
    def __init__(self, value):
        self.value = value

    def __str__(self):
        return f"值是 {self.value}"


class GoodRepr:
    """同时定义 __repr__ 和 __str__"""
    def __init__(self, value):
        self.value = value

    def __repr__(self):
        return f"GoodRepr({self.value!r})"

    def __str__(self):
        return f"值是 {self.value}"


# 在列表中，使用 __repr__
items = [PoorRepr(1), PoorRepr(2)]
print(items)  # [<__main__.PoorRepr object at 0x...>, ...] — 不友好！

items2 = [GoodRepr(1), GoodRepr(2)]
print(items2)  # [GoodRepr(1), GoodRepr(2)] — 清晰！
```

**最佳实践**：

| 场景 | 应该定义的方法 | 原因 |
|------|--------------|------|
| 所有类 | `__repr__` | 调试、日志、容器显示 |
| 需要用户友好输出 | `__str__` | `print()`, `str()` 场景 |
| 需要格式化 | `__format__` | `format()`, f-string |

**`__repr__` 的目标**：`eval(repr(obj)) == obj`（在不切实际时，返回 `<ClassName ...>` 形式的描述性字符串）。

### 13.5.7 陷阱七：可变对象的 `__hash__`

```python
class MutablePoint:
    """错误：可变对象定义了 __hash__"""
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, MutablePoint):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)

    def __hash__(self):
        return hash((self.x, self.y))


p = MutablePoint(1, 2)
s = {p}
d = {p: "value"}

# 修改对象后哈希值改变！
p.x = 3
print(p in s)       # False — 哈希值已变，集合找不到
print(p in d)        # False — 同理
# s 中和 d 中的 p 已经"丢失"了！

# ---- 正确做法：可变对象不应定义 __hash__ ----

class MutablePointFixed:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, MutablePointFixed):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)

    __hash__ = None  # 显式声明不可哈希


# mp = MutablePointFixed(1, 2)
# hash(mp)  # TypeError: unhashable type
# {mp}      # TypeError: unhashable type
```

### 13.5.8 陷阱八：`__getattr__` 与特殊方法不兼容

```python
class DynamicAttrs:
    """使用 __getattr__ 实现动态属性访问"""
    def __getattr__(self, name):
        if name.startswith('get_'):
            return lambda: f"dynamic {name}"
        raise AttributeError(f"'{type(self).__name__}' 对象没有属性 '{name}'")


da = DynamicAttrs()
print(da.get_name())  # "dynamic get_name" — 正常工作

# 但特殊方法不会触发 __getattr__！
# len(da)  # TypeError: object of type 'DynamicAttrs' has no len()
# da + 1   # TypeError: unsupported operand type(s)

# 原因：特殊方法在类型上查找，不经过 __getattr__
# 必须在类定义中显式定义特殊方法

class DynamicAttrsWithSpecialMethods:
    def __init__(self, data):
        self._data = data

    def __getattr__(self, name):
        if name.startswith('get_'):
            return lambda: f"dynamic {name}"
        raise AttributeError(f"'{type(self).__name__}' 对象没有属性 '{name}'")

    def __len__(self):
        return len(self._data)

    def __getitem__(self, key):
        return self._data[key]


daws = DynamicAttrsWithSpecialMethods([1, 2, 3])
print(len(daws))     # 3 — 正常
print(daws[0])        # 1 — 正常
print(daws.get_name())  # "dynamic get_name" — 正常
```

### 13.5.9 陷阱九：`__getattribute__` 的无限递归

```python
class RecursiveAttr:
    """错误示范：__getattribute__ 导致无限递归"""
    def __getattribute__(self, name):
        # 错误！self.name 会再次调用 __getattribute__，导致无限递归
        return self.name  # RecursionError!


class FixedAttr:
    """正确：使用 object.__getattribute__ 访问属性"""
    def __getattribute__(self, name):
        if name == 'special':
            return "handled by __getattribute__"
        # 委托给基类实现，避免无限递归
        return object.__getattribute__(self, name)


fa = FixedAttr()
fa.normal = "normal value"
print(fa.normal)   # "normal value" — 正常属性查找
print(fa.special)  # "handled by __getattribute__" — 拦截
```

### 13.5.10 陷阱十：`functools.total_ordering` 的性能与正确性

```python
from functools import total_ordering
import timeit


@total_ordering
class PointWithTotalOrdering:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        return (self.x, self.y) == (other.x, other.y)

    def __lt__(self, other):
        return (self.x, self.y) < (other.x, other.y)

    # __le__, __gt__, __ge__ 由 total_ordering 自动生成


class PointWithAllMethods:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        return (self.x, other.y) == (other.x, other.y) if False else \
               (self.x, self.y) == (other.x, other.y)

    def __lt__(self, other):
        return (self.x, self.y) < (other.x, other.y)

    def __le__(self, other):
        return (self.x, self.y) <= (other.x, other.y)

    def __gt__(self, other):
        return (self.x, self.y) > (other.x, other.y)

    def __ge__(self, other):
        return (self.x, self.y) >= (other.x, other.y)


# 性能对比
p1_t = PointWithTotalOrdering(1, 2)
p2_t = PointWithTotalOrdering(3, 4)
p1_a = PointWithAllMethods(1, 2)
p2_a = PointWithAllMethods(3, 4)

time_total = timeit.timeit(lambda: p1_t <= p2_t, number=100000)
time_all = timeit.timeit(lambda: p1_a <= p2_a, number=100000)

print(f"total_ordering: {time_total:.4f}s")
print(f"手动实现:       {time_all:.4f}s")
# total_ordering 通常慢 2-3 倍，因为 __le__ 会调用 __lt__ 和 __eq__
```

**`@total_ordering` 的正确性陷阱**：

```python
# total_ordering 假设比较关系是全序的（total order）：
# 即满足自反性、反对称性、传递性
# 如果你的比较不满足这些性质，total_ordering 会产生错误结果

@total_ordering
class BadComparison:
    def __init__(self, value):
        self.value = value

    def __eq__(self, other):
        return self.value == other.value

    def __lt__(self, other):
        # 错误：不满足传递性的比较
        # 例如：a < b 且 b < c，但 a 可能不 < c
        return self.value % 3 < other.value % 3

# 这种情况下 total_ordering 推导出的其他比较方法会不可靠
```

### 13.5.11 最佳实践总结

| 实践 | 说明 | 优先级 |
|------|------|--------|
| 总是定义 `__repr__` | 调试和日志的基础 | 必需 |
| `__eq__` 和 `__hash__` 成对定义 | 或同时不定义（`__hash__ = None`） | 必需 |
| 比较方法返回 `NotImplemented` | 而非抛出异常，给反射方法机会 | 必需 |
| 不可变对象定义 `__hash__` | 可变对象设置 `__hash__ = None` | 必需 |
| `__iadd__` 等增量方法返回 `self` | 不要返回新对象 | 必需 |
| 使用上下文管理器而非 `__del__` | `__del__` 调用时机不可控 | 必需 |
| 特殊方法定义在类上 | 不要在实例上覆盖特殊方法 | 必需 |
| `__new__` 返回当前类的实例 | 避免跳过 `__init__` 的陷阱 | 推荐 |
| 手动实现所有比较方法 | 比 `@total_ordering` 性能更好 | 性能敏感 |
| 只定义 `__new__` 当需要时 | 不可变类型、单例模式等 | 按需 |

---

**本章小结**：Python 的特殊方法与协议是数据模型的核心。理解它们的查找规则（类型优先于实例）、双分派机制（正向/反射方法链）、隐式调用与显式调用的差异，是编写 Pythonic 代码的基础。在实际开发中，始终遵循"定义 `__repr__`"、"`__eq__` 与 `__hash__` 成对"、"比较方法返回 `NotImplemented`"这三条核心原则，就能避免绝大多数常见陷阱。
