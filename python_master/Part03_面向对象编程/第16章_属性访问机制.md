# 第16章 属性访问机制

Python 的属性访问远不止 `obj.attr` 这么简单。当你写下 `self.x` 时，CPython 虚拟机在底层执行了一条精密的查找链路——涉及描述符协议、MRO 解析、`__dict__` 查询和特殊方法分派。理解这条链路，是从"会用 Python"迈向"精通 Python"的关键一步。

---

## 16.1 概念与语法

### 16.1.1 四大属性访问特殊方法

Python 提供了四个核心特殊方法来拦截属性访问操作：

| 特殊方法 | 触发时机 | 典型用途 |
|---------|---------|---------|
| `__getattribute__(self, name)` | **每次**属性访问都会调用 | 拦截所有属性读取 |
| `__getattr__(self, name)` | 仅当**正常查找失败**后调用 | 提供兜底值或动态属性 |
| `__setattr__(self, name, value)` | **每次**属性赋值都会调用 | 拦截所有属性写入 |
| `__delattr__(self, name)` | **每次**属性删除都会调用 | 拦截所有属性删除 |

关键区别：`__getattribute__` 和 `__getattr__` 构成了**两级防御体系**——前者拦截所有读取，后者仅作为兜底。而 `__setattr__` 和 `__delattr__` 没有这种两级结构，它们总是拦截对应的操作。

```python
class AttributeDemo:
    def __init__(self):
        self.existing = "I exist"

    def __getattribute__(self, name):
        print(f"  __getattribute__ called for '{name}'")
        return super().__getattribute__(name)

    def __getattr__(self, name):
        print(f"  __getattr__ called for '{name}'")
        return f"fallback value for {name}"

    def __setattr__(self, name, value):
        print(f"  __setattr__ called: {name} = {value}")
        super().__setattr__(name, value)

    def __delattr__(self, name):
        print(f"  __delattr__ called for '{name}'")
        super().__delattr__(name)


demo = AttributeDemo()
# __setattr__ called: existing = I exist  (在 __init__ 中触发)

print("--- Accessing existing attribute ---")
print(demo.existing)
# __getattribute__ called for 'existing'
# I exist

print("--- Accessing missing attribute ---")
print(demo.missing)
# __getattribute__ called for 'missing'
# __getattr__ called for 'missing'
# fallback value for missing

print("--- Setting attribute ---")
demo.new_attr = "hello"
# __setattr__ called: new_attr = hello

print("--- Deleting attribute ---")
del demo.existing
# __delattr__ called for existing
```

### 16.1.2 `__getattribute__` 与 `__getattr__` 的分工

`__getattribute__` 是**第一道关卡**，每次属性访问都会触发。只有当 `__getattribute__` 抛出 `AttributeError` 时，Python 才会调用 `__getattr__` 作为第二道防线。

```python
class DualGate:
    def __getattribute__(self, name):
        if name == "blocked":
            raise AttributeError(f"'{name}' is blocked")
        if name == "intercepted":
            return "intercepted by __getattribute__"
        return super().__getattribute__(name)

    def __getattr__(self, name):
        return f"'{name}' not found, but __getattr__ caught it"


obj = DualGate()

print(obj.intercepted)   # intercepted by __getattribute__
print(obj.anything_else)  # 'anything_else' not found, but __getattr__ caught it

try:
    print(obj.blocked)   # __getattribute__ 抛出 AttributeError → __getattr__ 兜底
except AttributeError as e:
    print(f"Error: {e}")
# 但注意: __getattribute__ 主动 raise AttributeError 后，__getattr__ 仍会被调用
print(obj.blocked)
# 'blocked' not found, but __getattr__ caught it
```

### 16.1.3 `__setattr__` 的自举问题

`__setattr__` 有一个经典的陷阱：在 `__setattr__` 内部给 `self` 赋属性会再次触发 `__setattr__`，导致无限递归。解决方法是直接操作 `self.__dict__` 或调用 `object.__setattr__`。

```python
class Broken:
    def __setattr__(self, name, value):
        self.name = value  # 无限递归！再次触发 __setattr__


class Fixed1:
    def __setattr__(self, name, value):
        self.__dict__[name] = value  # 直接操作 __dict__，绕过 __setattr__


class Fixed2:
    def __setattr__(self, name, value):
        object.__setattr__(self, name, value)  # 调用基类方法


f1 = Fixed1()
f1.x = 10
print(f1.x)  # 10

f2 = Fixed2()
f2.x = 20
print(f2.x)  # 20
```

同理，`__delattr__` 中也存在相同的递归陷阱，必须使用 `del self.__dict__[name]` 或 `object.__delattr__(self, name)` 来避免。

### 16.1.4 描述符协议速览

描述符是属性查找链路的核心机制，它定义了三个可选方法：

```python
class Descriptor:
    """完整描述符协议"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self  # 类级别访问，返回描述符自身
        return obj._value

    def __set__(self, obj, value):
        obj._value = value

    def __delete__(self, obj):
        del obj._value


class DataDescriptor:
    """数据描述符（定义了 __set__ 或 __delete__）"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get('_data_desc', 0)

    def __set__(self, obj, value):
        obj.__dict__['_data_desc'] = value


class NonDataDescriptor:
    """非数据描述符（只定义了 __get__）"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get('_non_data_desc', 'default')
```

数据描述符与非数据描述符的区别决定了属性查找的优先级，这将在下一节详细展开。

## 16.2 原理与机制

### 16.2.1 属性查找完整链路

当你写下 `obj.attr` 时，CPython 遵循以下查找链路（这是 Python 语言最核心的查找规则之一）：

```
obj.attr 的完整查找链路：

1. 数据描述符（定义了 __set__ 或 __delete__ 的描述符）
   ├─ 在 type(obj).__mro__ 中查找 attr
   └─ 如果找到数据描述符 → 调用 descriptor.__get__(obj, type(obj))
       → 返回结果，查找结束

2. 实例 __dict__
   ├─ 检查 obj.__dict__.get('attr')
   └─ 如果找到 → 返回值，查找结束

3. 非数据描述符（只定义了 __get__ 的描述符）
   ├─ 在 type(obj).__mro__ 中查找 attr
   └─ 如果找到非数据描述符 → 调用 descriptor.__get__(obj, type(obj))
       → 返回结果，查找结束

4. 类 __dict__ 及基类 __dict__
   ├─ 在 type(obj).__mro__ 中查找 attr
   └─ 如果找到普通类属性 → 返回值，查找结束

5. __getattr__
   └─ 如果上述全部未找到 → 调用 obj.__getattr__('attr')
       → 返回结果（或抛出 AttributeError）
```

用代码验证这个优先级链路：

```python
# 验证属性查找优先级：数据描述符 > 实例 __dict__ > 非数据描述符 > 类属性

class DataDesc:
    """数据描述符：定义了 __set__"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return f"DataDesc.__get__ → {obj.__dict__.get('x', 'N/A')}"

    def __set__(self, obj, value):
        obj.__dict__['x'] = f"set_via_DataDesc({value})"


class NonDataDesc:
    """非数据描述符：只定义了 __get__"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return "NonDataDesc.__get__"


class MyClass:
    data_desc = DataDesc()       # 数据描述符
    non_data_desc = NonDataDesc()  # 非数据描述符
    class_attr = "I am class attribute"


obj = MyClass()

# 实验1: 数据描述符优先于实例 __dict__
obj.__dict__['data_desc'] = "instance override"
print("1. Data descriptor wins:", obj.data_desc)
# DataDesc.__get__ → N/A  (数据描述符胜出，忽略实例 __dict__)

obj.data_desc = "hello"
print("2. After __set__:", obj.data_desc)
# DataDesc.__get__ → set_via_DataDesc(hello)

# 实验2: 实例 __dict__ 优先于非数据描述符
obj.__dict__['non_data_desc'] = "instance wins"
print("3. Instance __dict__ wins over non-data desc:", obj.non_data_desc)
# instance wins  (实例 __dict__ 胜出，非数据描述符被覆盖)

# 实验3: 实例 __dict__ 优先于类属性
obj.__dict__['class_attr'] = "instance override"
print("4. Instance __dict__ wins over class attr:", obj.class_attr)
# instance override

# 实验4: 非数据描述符优先于类属性（当没有实例属性时）
del obj.__dict__['non_data_desc']
print("5. Non-data descriptor wins over class attr:", obj.non_data_desc)
# NonDataDesc.__get__
```

### 16.2.2 `__getattribute__` 的 CPython 实现分析

在 CPython 源码中，属性访问的核心函数是 `_PyObject_GetAttr`。以下是基于 CPython 3.12 源码的分析：

```c
// Objects/object.c — 简化版 _PyObject_GetAttr
PyObject *
_PyObject_GetAttr(PyObject *v, PyObject *name)
{
    PyTypeObject *tp = Py_TYPE(v);

    // 第一步：在类型对象上查找描述符
    // 使用 _PyType_Lookup 在 MRO 中查找
    PyObject *descr = _PyType_Lookup(tp, name);

    // 第二步：如果找到了描述符，检查它是否定义了 __get__
    // 并判断是数据描述符还是非数据描述符
    if (descr != NULL) {
        PyTypeObject *descr_tp = Py_TYPE(descr);
        descrgetfunc tp_descr_get = descr_tp->tp_descr_get;

        // 数据描述符检查：定义了 __set__ 或 __delete__
        // 如果是数据描述符，直接调用 __get__ 并返回
        if (_PyType_HasFeature(descr_tp, Py_TPFLAGS_HAVE_GC) &&
            _PyDescr_IsData(descr)) {
            // ★ 数据描述符优先级最高，直接返回
            return tp_descr_get(descr, v, (PyObject *)tp);
        }
    }

    // 第三步：查找实例 __dict__
    // 通过 tp_dictoffset 获取 __dict__ 的偏移量
    if (_PyType_HasFeature(tp, Py_TPFLAGS_MANAGED_DICT)) {
        // CPython 3.12 引入的 managed dict 优化
        PyObject *dict = _PyObject_ManagedDictPointer(v);
        if (dict != NULL) {
            PyObject *value = PyDict_GetItemWithError(dict, name);
            if (value != NULL) {
                return Py_NewRef(value);
            }
        }
    } else {
        // 传统方式：通过 tp_dictoffset
        Py_ssize_t dictoffset = tp->tp_dictoffset;
        if (dictoffset) {
            PyObject *dict = *((PyObject **)((char *)v + dictoffset));
            if (dict != NULL) {
                PyObject *value = PyDict_GetItemWithError(dict, name);
                if (value != NULL) {
                    return Py_NewRef(value);
                }
            }
        }
    }

    // 第四步：非数据描述符
    if (descr != NULL) {
        descrgetfunc tp_descr_get = Py_TYPE(descr)->tp_descr_get;
        if (tp_descr_get != NULL) {
            return tp_descr_get(descr, v, (PyObject *)tp);
        }
        return Py_NewRef(descr);
    }

    // 第五步：调用 __getattr__
    if (tp->tp_getattr != NULL) {
        return tp->tp_getattr(v, name_str);
    }
    // 或者通过 tp_getattro
    // ...
}
```

关键的 C 层面概念：

| CPython 层面概念 | 含义 |
|---|---|
| `tp_getattro` | 类型对象上的槽位，指向属性访问函数。默认为 `PyObject_GenericGetAttr` |
| `tp_descr_get` | 描述符的 `__get__` 槽位 |
| `_PyDescr_IsData(descr)` | 检查描述符是否定义了 `__set__` 或 `__delete__`（即是否为数据描述符） |
| `tp_dictoffset` | 实例中 `__dict__` 的字节偏移量 |
| `_PyType_Lookup` | 在 MRO 中查找属性，返回第一个匹配的结果 |

### 16.2.3 `tp_getattro` 与 `__getattribute__` 的映射

当你定义 `__getattribute__` 时，CPython 将其安装到类型对象的 `tp_getattro` 槽位：

```c
// 当类定义了 __getattribute__ 时：
// tp_getattro 指向自定义实现
// 当类没有定义 __getattribute__ 时：
// tp_getattro 指向 PyObject_GenericGetAttr（默认实现）

// PyObject_GenericGetAttr 的简化逻辑
PyObject *
PyObject_GenericGetAttr(PyObject *obj, PyObject *name)
{
    return _PyObject_GetAttr(obj, name);  // 调用上面的查找链路
}
```

验证 `tp_getattro` 的行为：

```python
import ctypes

class WithGetAttribute:
    def __getattribute__(self, name):
        return object.__getattribute__(self, name)

class WithoutGetAttribute:
    pass

# 在 CPython 内部，定义了 __getattribute__ 的类
# 其 tp_getattro 槽位指向自定义函数
# 没有定义的类，tp_getattro 指向 PyObject_GenericGetAttr

print(f"WithGetAttribute has __getattribute__: {hasattr(WithGetAttribute, '__getattribute__')}")
print(f"WithoutGetAttribute has __getattribute__: {hasattr(WithoutGetAttribute, '__getattribute__')}")

# object.__getattribute__ 就是 PyObject_GenericGetAttr 的 Python 包装
# 所有属性查找最终都经过同一条 C 代码路径
print(f"object.__getattribute__ is: {object.__getattribute__}")

# 注意：即使 WithoutGetAttribute 没有定义 __getattribute__
# 它的 __getattribute__ 仍然存在（从 object 继承）
obj = WithoutGetAttribute()
print(f"type(obj).__getattribute__: {type(obj).__getattribute__}")
# <method '__getattribute__' of 'object' objects>
```

### 16.2.4 `__setattr__` 的 CPython 实现

`__setattr__` 在 C 层面对应 `tp_setattro` 槽位：

```c
// Objects/object.c — PyObject_SetAttr 简化版
int
PyObject_SetAttr(PyObject *v, PyObject *name, PyObject *value)
{
    PyTypeObject *tp = Py_TYPE(v);

    // 第一步：查找数据描述符
    PyObject *descr = _PyType_Lookup(tp, name);
    if (descr != NULL) {
        PyTypeObject *descr_tp = Py_TYPE(descr);

        // 如果是数据描述符（定义了 __set__）
        if (_PyDescr_IsData(descr)) {
            setfunc tp_descr_set = descr_tp->tp_descr_set;
            if (tp_descr_set != NULL) {
                // ★ 数据描述符的 __set__ 优先级最高
                return tp_descr_set(descr, v, value);
            }
        }
    }

    // 第二步：如果值为 NULL（删除操作），处理 __delete__
    // ...

    // 第三步：写入实例 __dict__
    if (value != NULL) {
        PyObject *dict = _PyObject_GetDictPtr(v);
        if (dict != NULL) {
            return PyDict_SetItem(dict, name, value);
        }
    }

    // 第四步：调用 __setattr__
    if (tp->tp_setattro != NULL) {
        return tp->tp_setattro(v, name, value);
    }

    PyErr_Format(Py_TypeError, "'%.100s' object has no attribute '%U'",
                 tp->tp_name, name);
    return -1;
}
```

赋值时的优先级比读取简单：

```
obj.attr = value 的查找链路：

1. 数据描述符的 __set__
   └─ 在 type(obj).__mro__ 中查找 attr
   └─ 如果找到数据描述符 → 调用 descriptor.__set__(obj, value)
       → 赋值结束

2. 实例 __dict__
   └─ obj.__dict__['attr'] = value
       → 赋值结束

注意: __setattr__ 拦截所有赋值操作，在上述查找之前执行
```

### 16.2.6 `__slots__` 的原理与内存优化

#### 基本原理

`__slots__` 通过在类对象上声明一个固定属性集合，禁止实例动态创建 `__dict__`，从而节省内存并略微加速属性访问。

```python
import sys

class WithoutSlots:
    def __init__(self):
        self.x = 1
        self.y = 2
        self.z = 3

class WithSlots:
    __slots__ = ('x', 'y', 'z')
    def __init__(self):
        self.x = 1
        self.y = 2
        self.z = 3

obj_no_slots = WithoutSlots()
obj_with_slots = WithSlots()

print(f"WithoutSlots size: {sys.getsizeof(obj_no_slots)} bytes")
print(f"WithSlots size: {sys.getsizeof(obj_with_slots)} bytes")
print(f"WithoutSlots __dict__: {obj_no_slots.__dict__}")
print(f"WithoutSlots __dict__ size: {sys.getsizeof(obj_no_slots.__dict__)} bytes")

# WithSlots 没有 __dict__
try:
    print(obj_with_slots.__dict__)
except AttributeError as e:
    print(f"WithSlots has no __dict__: {e}")

# 总内存对比（对象 + __dict__）
total_no_slots = sys.getsizeof(obj_no_slots) + sys.getsizeof(obj_no_slots.__dict__)
print(f"WithoutSlots total: {total_no_slots} bytes")
print(f"WithSlots total: {sys.getsizeof(obj_with_slots)} bytes")
print(f"Memory saved: {total_no_slots - sys.getsizeof(obj_with_slots)} bytes")
```

典型输出（64 位 CPython）：

```
WithoutSlots size: 56 bytes
WithSlots size: 48 bytes
WithoutSlots __dict__: {'x': 1, 'y': 2, 'z': 3}
WithoutSlots __dict__ size: 184 bytes
WithoutSlots total: 240 bytes
WithSlots total: 48 bytes
Memory saved: 192 bytes
```

#### CPython 底层实现

在 C 层面，`__slots__` 的工作方式是：

1. 类创建时，`type.__new__` 检查 `__slots__`
2. 为每个 slot 在类对象上创建一个 **member descriptor**（定义了 `__get__`、`__set__`、`__delete__`）
3. 这些 member descriptor 是**数据描述符**，因此优先级高于实例 `__dict__`
4. 类的 `tp_dictoffset` 设为 0，禁止实例创建 `__dict__`

```python
# 验证 __slots__ 创建的是数据描述符
class SlotDemo:
    __slots__ = ('x', 'y')

# 查看类属性
print(type(SlotDemo.x))  # <class 'member_descriptor'>
print(SlotDemo.x)          # <member 'x' of 'SlotDemo' objects>

# member_descriptor 定义了 __get__, __set__, __delete__ → 数据描述符
print(hasattr(SlotDemo.x, '__set__'))    # True
print(hasattr(SlotDemo.x, '__delete__')) # True
print(hasattr(SlotDemo.x, '__get__'))    # True

# 这就是为什么 slots 属性不受实例 __dict__ 影响
# （数据描述符优先级 > 实例 __dict__）
```

#### 内存节省效果实测

```python
import sys
import tracemalloc

class PointNoSlots:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class PointWithSlots:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

def measure_memory(cls, count=100000):
    """测量创建大量实例的内存开销"""
    tracemalloc.start()
    before = tracemalloc.get_traced_memory()
    objects = [cls(i, i) for i in range(count)]
    after = tracemalloc.get_traced_memory()
    tracemalloc.stop()
    total = after[0] - before[0]
    return total, total / count

total_no_slots, per_no_slots = measure_memory(PointNoSlots)
total_with_slots, per_with_slots = measure_memory(PointWithSlots)

print(f"WithoutSlots: {total_no_slots / 1024 / 1024:.2f} MB total, {per_no_slots:.1f} bytes/object")
print(f"WithSlots:    {total_with_slots / 1024 / 1024:.2f} MB total, {per_with_slots:.1f} bytes/object")
print(f"Savings:      {(1 - total_with_slots/total_no_slots)*100:.1f}%")

# 典型输出：
# WithoutSlots: 17.50 MB total, 184.0 bytes/object
# WithSlots:    3.50 MB total, 36.8 bytes/object
# Savings:      80.0%
```

#### 继承中的 `__slots__` 行为

`__slots__` 在继承中的行为有一些微妙的规则：

```python
# 规则1: 子类如果没有定义 __slots__，会恢复 __dict__
class Parent:
    __slots__ = ('x', 'y')

class ChildNoSlots(Parent):
    pass  # 没有 __slots__ → 子类实例有 __dict__

parent = Parent()
child = ChildNoSlots()

try:
    parent.z = 1
except AttributeError as e:
    print(f"Parent has no __dict__: {e}")

child.z = 1  # 子类有 __dict__，可以动态添加属性
print(f"Child has __dict__: {child.__dict__}")

# 规则2: 子类定义 __slots__ 会叠加父类的 slots
class Parent2:
    __slots__ = ('x', 'y')

class Child2(Parent2):
    __slots__ = ('z',)  # 子类只定义自己新增的 slot

c2 = Child2()
c2.x = 1  # 从 Parent2 继承
c2.y = 2  # 从 Parent2 继承
c2.z = 3  # Child2 自己的 slot
print(f"Child2 slots: x={c2.x}, y={c2.y}, z={c2.z}")

# 规则3: 子类的 __slots__ 不能重复父类的 slot
class Parent3:
    __slots__ = ('x',)

class Child3(Parent3):
    __slots__ = ('x',)  # 重复定义！
    # 这不会报错，但是冗余的

c3 = Child3()
c3.x = 10
print(f"Child3 x: {c3.x}")  # 正常工作，但 x 实际上来自 Parent3

# 规则4: 多重继承中的 __slots__
class Base1:
    __slots__ = ('a',)

class Base2:
    __slots__ = ('b',)

# 这不会报错（因为空 slots）
class Multi(Base1, Base2):
    __slots__ = ('c',)

m = Multi()
m.a = 1
m.b = 2
m.c = 3
print(f"Multi: a={m.a}, b={m.b}, c={m.c}")

# 规则5: 空的 __slots__ 阻止 __dict__ 创建
class NoDict:
    __slots__ = ()  # 空 slots，不声明任何属性，但也没有 __dict__

nd = NoDict()
try:
    nd.anything = 1
except AttributeError as e:
    print(f"Empty __slots__ blocks __dict__: {e}")
```

#### `__slots__` 与 `__dict__` 共存

在某些场景下，你可能需要既使用 `__slots__` 节省内存，又保留 `__dict__` 以支持动态属性：

```python
class Hybrid:
    __slots__ = ('x', 'y', '__dict__')  # 在 __slots__ 中包含 '__dict__'

    def __init__(self, x, y):
        self.x = x
        self.y = y

h = Hybrid(1, 2)
print(f"Slots: x={h.x}, y={h.y}")

# 动态属性仍然可以添加（因为 __dict__ 在 slots 中）
h.z = 3
h.dynamic_attr = "hello"
print(f"Dict: {h.__dict__}")
# {'z': 3, 'dynamic_attr': 'hello'}

# 但注意：x 和 y 不在 __dict__ 中（它们存储在 slot 中）
print(f"x in __dict__: {'x' in h.__dict__}")   # False
print(f"z in __dict__: {'z' in h.__dict__}")     # True
```

#### `__slots__` 与 `__weakref__`

默认情况下，每个实例都可以被弱引用（`weakref`）。`__slots__` 会禁用 `__weakref__`，除非显式包含它：

```python
import weakref

class NoWeakRef:
    __slots__ = ('x',)

class WithWeakRef:
    __slots__ = ('x', '__weakref__')

# NoWeakRef 的实例不支持弱引用
nw = NoWeakRef()
try:
    wr = weakref.ref(nw)
except TypeError as e:
    print(f"No weakref: {e}")

# WithWeakRef 的实例支持弱引用
ww = WithWeakRef()
wr = weakref.ref(ww)
print(f"Weakref works: {wr() is ww}")

# 不使用 __slots__ 的类默认支持弱引用
# 因为 object 的 __dict__ 中包含 __weakref__ 槽位
```

### 16.3.1 延迟属性（Lazy Property）

`__getattr__` 只在属性不存在时调用，非常适合实现延迟加载：

```python
class LazyLoader:
    """使用 __getattr__ 实现延迟加载"""

    def __init__(self):
        self._loaded = {}

    def __getattr__(self, name):
        if name.startswith('load_'):
            # 延迟计算，只计算一次
            key = name[5:]  # 去掉 'load_' 前缀
            if key not in self._loaded:
                print(f"  Computing {key}...")
                self._loaded[key] = self._expensive_compute(key)
            return self._loaded[key]
        raise AttributeError(f"'{type(self).__name__}' has no attribute '{name}'")

    @staticmethod
    def _expensive_compute(key):
        return f"result_of({key})"


loader = LazyLoader()
print(loader.load_data)   # Computing data... → result_of(data)
print(loader.load_data)   # result_of(data) — 第二次不触发 __getattr__！
# 注意：因为第一次调用后 _loaded['data'] 被设置，
# 但 load_data 仍然会走 __getattr__，因为 'load_data' 不在实例 __dict__ 中
# 如果想缓存到 __dict__ 中避免反复调用 __getattr__，需要手动设置
```

更好的实现——将结果缓存到 `__dict__` 中：

```python
class CachedLazyProperty:
    """使用 __getattr__ + __dict__ 缓存实现真正的延迟属性"""

    def __getattr__(self, name):
        # 延迟计算工厂方法
        factories = {
            'expensive_data': lambda: self._compute_expensive_data(),
            'database_connection': lambda: self._create_connection(),
            'config': lambda: self._load_config(),
        }

        if name in factories:
            value = factories[name]()
            # 将结果写入 __dict__，下次访问不再触发 __getattr__
            self.__dict__[name] = value
            return value

        raise AttributeError(f"'{type(self).__name__}' has no attribute '{name}'")

    def _compute_expensive_data(self):
        print("  Computing expensive data...")
        return [1, 2, 3, 4, 5]

    def _create_connection(self):
        print("  Creating database connection...")
        return "connection_object"

    def _load_config(self):
        print("  Loading config...")
        return {"debug": True, "port": 8080}


obj = CachedLazyProperty()
print("First access:")
print(obj.expensive_data)  # Computing expensive data...
print("Second access:")
print(obj.expensive_data)  # 直接从 __dict__ 读取，不再调用 __getattr__

print(obj.database_connection)  # Creating database connection...
print(obj.config)               # Loading config...
```

### 16.3.2 代理模式（Proxy Pattern）

`__getattribute__` 拦截所有属性访问，适合实现透明代理：

```python
class LoggingProxy:
    """记录所有属性访问的代理"""

    def __init__(self, target):
        object.__setattr__(self, '_target', target)
        object.__setattr__(self, '_log', [])

    def __getattribute__(self, name):
        log = object.__getattribute__(self, '_log')
        log.append(f"GET {name}")
        target = object.__getattribute__(self, '_target')
        return getattr(target, name)

    def __setattr__(self, name, value):
        log = object.__getattribute__(self, '_log')
        log.append(f"SET {name} = {value!r}")
        target = object.__getattribute__(self, '_target')
        setattr(target, name, value)

    def get_log(self):
        return object.__getattribute__(self, '_log')


class RealService:
    def __init__(self):
        self.data = "original"

    def process(self, value):
        return f"processed: {value}"


service = RealService()
proxy = LoggingProxy(service)

print(proxy.data)           # GET data → original
proxy.data = "modified"     # SET data = 'modified'
print(proxy.process("x"))   # GET process → processed: x

for entry in proxy.get_log():
    print(f"  {entry}")
# GET data
# SET data = 'modified'
# GET process
```

```python
class ValidatedObject:
```

### 16.3.4 不可变对象（Immutable Objects）

结合 `__setattr__` 和 `__delattr__` 实现不可变对象：

```python
class Immutable:
    """使用 __setattr__ 和 __delattr__ 实现不可变对象"""

    __slots__ = ('_initialized',)  # 只保留一个标志位

    def __init__(self, **kwargs):
        object.__setattr__(self, '_initialized', False)
        for k, v in kwargs.items():
            object.__setattr__(self, k, v)
        object.__setattr__(self, '_initialized', True)

    def __setattr__(self, name, value):
        if object.__getattribute__(self, '_initialized'):
            raise AttributeError(
                f"'{type(self).__name__}' object is immutable, "
                f"cannot set attribute '{name}'"
            )
        object.__setattr__(self, name, value)

    def __delattr__(self, name):
        if object.__getattribute__(self, '_initialized'):
            raise AttributeError(
                f"'{type(self).__name__}' object is immutable, "
                f"cannot delete attribute '{name}'"
            )
        object.__delattr__(self, name)

    def __repr__(self):
        attrs = {k: v for k, v in self.__dict__.items() if k != '_initialized'}
        return f"{type(self).__name__}({attrs})"


point = Immutable(x=10, y=20)
print(f"Point: {point}")

try:
    point.x = 30
except AttributeError as e:
    print(f"Error: {e}")

try:
    del point.y
except AttributeError as e:
    print(f"Error: {e}")
```

### 16.3.5 动态属性映射（ORM 风格）

`__getattr__` 和 `__setattr__` 配合实现类似 ORM 的动态属性映射：

```python
class Record:
    """模拟 ORM 记录的动态属性映射"""

    __slots__ = ('_fields', '_dirty')

    def __init__(self, **fields):
        object.__setattr__(self, '_fields', dict(fields))
        object.__setattr__(self, '_dirty', set())

    def __getattr__(self, name):
        fields = object.__getattribute__(self, '_fields')
        if name in fields:
            return fields[name]
        raise AttributeError(f"'{type(self).__name__}' has no field '{name}'")

    def __setattr__(self, name, value):
        fields = object.__getattribute__(self, '_fields')
        dirty = object.__getattribute__(self, '_dirty')
        if name.startswith('_'):
            object.__setattr__(self, name, value)
        else:
            fields[name] = value
            dirty.add(name)

    def __delattr__(self, name):
        fields = object.__getattribute__(self, '_fields')
        if name in fields:
            del fields[name]
        else:
            raise AttributeError(f"'{type(self).__name__}' has no field '{name}'")

    @property
    def dirty_fields(self):
        return frozenset(object.__getattribute__(self, '_dirty'))

    def __repr__(self):
        fields = object.__getattribute__(self, '_fields')
        return f"Record({fields})"


# 使用示例
user = Record(id=1, name="Alice", email="alice@example.com")
print(f"User: {user}")
print(f"Name: {user.name}")

user.name = "Bob"           # 标记为 dirty
user.email = "bob@new.com"  # 标记为 dirty
print(f"Dirty fields: {user.dirty_fields}")
print(f"Updated: {user}")
```

### 16.3.6 `__slots__` 的大规模数据优化

在需要创建大量实例的场景下，`__slots__` 带来的内存节省非常显著：

```python
import time
import tracemalloc

class PointNoSlots:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class PointWithSlots:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

def benchmark(cls, count=1_000_000):
    """基准测试：内存和速度"""
    tracemalloc.start()
    t0 = time.perf_counter()

    t_create = time.perf_counter() - t0

    # 属性访问速度测试
    t0 = time.perf_counter()
    total = sum(p.x + p.y for p in points)
    t_access = time.perf_counter() - t0

    current, peak = tracemalloc.get_traced_memory()
    tracemalloc.stop()

    return {
        'class': cls.__name__,
        'create_time': t_create,
        'access_time': t_access,
        'memory_mb': peak / 1024 / 1024,
        'per_object_bytes': peak / count,

results = [benchmark(PointNoSlots), benchmark(PointWithSlots)]
for r in results:

          f"Create: {r['create_time']:.3f}s | "
          f"Access: {r['access_time']:.3f}s | "

          f"Per object: {r['per_object_bytes']:.1f} B")
```

### 16.4.1 完整属性访问链路追踪器

```python
"""
完整演示属性查找链路的优先级规则。
通过精心构造的示例，逐步验证每个优先级层次。
"""

class TracingDataDescriptor:
    """数据描述符：定义了 __set__，优先级最高"""

    def __init__(self, name):
        self.name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        value = obj.__dict__.get(f'_desc_{self.name}', f'<data_desc_default:{self.name}>')
        print(f"  [DataDesc.__get__] {self.name} → {value!r}")
        return value

    def __set__(self, obj, value):
        print(f"  [DataDesc.__set__] {self.name} = {value!r}")
        obj.__dict__[f'_desc_{self.name}'] = value

    def __delete__(self, obj):
        print(f"  [DataDesc.__delete__] {self.name}")
        obj.__dict__.pop(f'_desc_{self.name}', None)


class TracingNonDataDescriptor:
    """非数据描述符：只定义了 __get__，优先级低于实例 __dict__"""

    def __init__(self, name, default):
        self.name = name
        self.default = default

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        print(f"  [NonDataDesc.__get__] {self.name} → {self.default!r}")
        return self.default


class DemoClass:
    data_desc = TracingDataDescriptor('data_desc')
    non_data_desc = TracingNonDataDescriptor('non_data_desc', '<non_data_desc_default>')
    class_attr = '<class_attribute>'

    def __init__(self):
        self.regular_attr = '<instance_attribute>'

print(f"Result: {result}")
# 数据描述符仍然胜出！实例 __dict__ 中的值被忽略

print()
print("=" * 60)
print("实验2: 实例 __dict__ 优先级 > 非数据描述符")
print("=" * 60)
obj.__dict__['non_data_desc'] = '<instance_value>'
print("obj.non_data_desc =", end=' ')
result = obj.non_data_desc
print(f"Result: {result}")
# 实例 __dict__ 胜出！非数据描述符的 __get__ 没有被调用

print()
print("=" * 60)
print("实验3: 非数据描述符优先级 > 类属性（当没有实例属性时）")
print("=" * 60)
```

### 16.4.2 `__getattribute__` 拦截与 `super()` 配合

```python
"""
演示 __getattribute__ 中的常见错误和正确模式。
"""

class SafeGetAttribute:
    """正确使用 __getattribute__ 的模式"""

    # 需要绕过 __getattribute__ 的内部属性
    _INTERNAL_ATTRS = frozenset(['_data', '_cache', '_initialized'])

    def __init__(self, **kwargs):
        # 使用 object.__setattr__ 避免触发自定义 __setattr__
        object.__setattr__(self, '_data', dict(kwargs))
        object.__setattr__(self, '_cache', {})
        object.__setattr__(self, '_initialized', True)

    def __getattribute__(self, name):
        # 正确做法：通过 object.__getattribute__ 访问内部属性
        # 然后只在必要时做自定义逻辑
        if name.startswith('_'):
            return object.__getattribute__(self, name)

        # 自定义逻辑：缓存属性访问
        cache = object.__getattribute__(self, '_cache')
        if name in cache:
            print(f"  Cache hit for '{name}'")
            return cache[name]

        data = object.__getattribute__(self, '_data')
        if name in data:
            value = data[name]
            cache[name] = value
            print(f"  Computed and cached '{name}'")
            return value

        # 兜底：让正常查找链路继续
        return object.__getattribute__(self, name)


obj = SafeGetAttribute(x=10, y=20)
print(f"First access: {obj.x}")     # Computed and cached 'x'
print(f"Second access: {obj.x}")    # Cache hit for 'x'
print(f"Direct data: {obj._data}")  # 直接访问，不经过缓存逻辑
```

### 16.4.3 描述符与 `__slots__` 配合实现类型安全属性

```python
"""
使用描述符 + __slots__ 实现类型安全的属性系统。
描述符提供验证逻辑，__slots__ 提供内存优化。
"""


class TypedSlot:
    """类型安全的 slot 描述符"""

    def __init__(self, name, expected_type, default=None):
        self.name = name
        self.expected_type = expected_type
        self.default = default

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        # slot 描述符通过底层方式存储
        # 使用 getattr 触发 member descriptor 的 __get__
        try:
            return getattr(obj, self.name)
        except AttributeError:
            if self.default is not None:
                return self.default
            raise

    def __set__(self, obj, value):
        if not isinstance(value, self.expected_type):
            raise TypeError(
                f"Expected {self.expected_type.__name__} for '{self.name}', "
                f"got {type(value).__name__}: {value!r}"
            )
        # 对于 __slots__ 类，直接设置 slot
        object.__setattr__(obj, self.name, value)


def typed_class(**field_defs):
    """
    工厂函数：创建类型安全的 __slots__ 类

    用法:
        MyClass = typed_class(name=str, age=int, score=float)
    """
    slot_names = tuple(field_defs.keys())
    descriptors = {}
    for name, (type_, default) in field_defs.items():
        descriptors[name] = TypedSlot(name, type_, default)

    # 创建类字典
    namespace = {
        '__slots__': slot_names,
        **descriptors,
    }

    def __init__(self, **kwargs):
        for name in field_defs:
            value = kwargs.get(name, descriptors[name].default)
            if value is not None:
                setattr(self, name, value)

    namespace['__init__'] = __init__
    namespace['__repr__'] = lambda self: (
        f"{type(self).__name__}("
        + ", ".join(f"{n}={getattr(self, n, '?')!r}" for n in field_defs)
        + ")"
    )

    cls = type('TypedClass', (), namespace)
    return cls


# 创建类型安全的类
Person = typed_class(
    name=(str, None),
    age=(int, 0),
    score=(float, 0.0),
)

p = Person(name="Alice", age=30, score=95.5)
print(f"Person: {p}")
print(f"Name: {p.name}, Age: {p.age}, Score: {p.score}")

# 类型错误
try:
    p.age = "thirty"
except TypeError as e:
    print(f"Type error: {e}")

# 尝试添加未声明的属性
try:
    p.extra = "nope"
except AttributeError as e:
    print(f"Slot error: {e}")
```

### 16.4.6 自定义 `__getattribute__` 实现属性访问日志

```python
"""
实现一个完整的属性访问审计系统，
记录所有属性的读写删除操作。
"""

from collections import defaultdict
from functools import wraps


class AuditedObject:
    """对属性访问进行完整审计的对象"""

    __slots__ = ('_audit_log', '_target_class', '_instance_data')

    def __init__(self, **kwargs):
        object.__setattr__(self, '_audit_log', defaultdict(list))
        object.__setattr__(self, '_instance_data', dict(kwargs))

    def __getattribute__(self, name):
        # 内部属性不审计
        if name.startswith('_'):
            return object.__getattribute__(self, name)

        audit_log = object.__getattribute__(self, '_audit_log')
        instance_data = object.__getattribute__(self, '_instance_data')

        # 记录读取
        audit_log[name].append(('READ', None))

        # 查找属性
        if name in instance_data:
            return instance_data[name]

        # 尝试从类中查找
        for cls in type(self).__mro__:
            if name in cls.__dict__:
                attr = cls.__dict__[name]
                if hasattr(attr, '__get__'):
                    return attr.__get__(self, type(self))
                return attr

        raise AttributeError(
            f"'{type(self).__name__}' object has no attribute '{name}'"
        )

    def __setattr__(self, name, value):
        if name.startswith('_'):
            object.__setattr__(self, name, value)
            return

        audit_log = object.__getattribute__(self, '_audit_log')
        instance_data = object.__getattribute__(self, '_instance_data')

        # 记录写入
        old_value = instance_data.get(name, '<UNSET>')
        audit_log[name].append(('WRITE', old_value))
        instance_data[name] = value

    def __delattr__(self, name):
        if name.startswith('_'):
            object.__delattr__(self, name)
            return

        audit_log = object.__getattribute__(self, '_audit_log')
        instance_data = object.__getattribute__(self, '_instance_data')

        if name not in instance_data:
            raise AttributeError(
                f"'{type(self).__name__}' object has no attribute '{name}'"
            )

        # 记录删除
        old_value = instance_data[name]
        audit_log[name].append(('DELETE', old_value))
        del instance_data[name]

    def get_audit_log(self):
        """获取审计日志"""
        audit_log = object.__getattribute__(self, '_audit_log')
        return {name: list(ops) for name, ops in audit_log.items()}

    def __repr__(self):
        instance_data = object.__getattribute__(self, '_instance_data')
        return f"AuditedObject({instance_data})"


# 使用示例
obj = AuditedObject(name="Alice", age=30)
print(f"Object: {obj}")

# 读取
print(f"Name: {obj.name}")
print(f"Age: {obj.age}")

# 写入
obj.age = 31
obj.city = "Beijing"

# 删除
del obj.city

# 查看审计日志
print("\n审计日志:")
for attr, ops in obj.get_audit_log().items():
    for op, old_val in ops:
        if op == 'READ':
            print(f"  {attr}: READ")
        elif op == 'WRITE':
            print(f"  {attr}: WRITE (old: {old_val})")
        elif op == 'DELETE':
            print(f"  {attr}: DELETE (old: {old_val})")
```

## 16.5 常见陷阱与最佳实践

### 16.5.1 `__getattribute__` 中的无限递归

**陷阱**：在 `__getattribute__` 中使用 `self.xxx` 会导致无限递归。

```python
class InfiniteRecursion:
    def __getattribute__(self, name):
        # 错误！self._data 再次触发 __getattribute__
        return self._data.get(name)  # RecursionError!

class Correct:
    def __getattribute__(self, name):
        # 正确：使用 object.__getattribute__ 避免递归
        try:
            data = object.__getattribute__(self, '_data')
        except AttributeError:
            raise AttributeError(f"'{type(self).__name__}' object has no attribute '{name}'")
        return data.get(name)

    def __init__(self, **kwargs):
        object.__setattr__(self, '_data', kwargs)


# 正确示例
c = Correct(x=10, y=20)
print(c.x)  # 10
```

**最佳实践**：

1. 在 `__getattribute__` 内部，始终使用 `object.__getattribute__(self, name)` 来访问实例属性
2. 在 `__setattr__` 内部，始终使用 `object.__setattr__(self, name, value)` 或 `self.__dict__[name] = value`
3. 在 `__delattr__` 内部，始终使用 `object.__delattr__(self, name)` 或 `del self.__dict__[name]`
4. 将需要绕过自定义逻辑的内部属性用下划线前缀标记，在方法内部分流处理

### 16.5.2 `__getattr__` 与 `__getattribute__` 的混淆

**陷阱**：以为 `__getattr__` 会拦截所有属性访问。

```python
class Misunderstood:
    def __getattr__(self, name):
        print(f"__getattr__ called for {name}")
        return "fallback"

m = Misunderstood()
m.x = 10      # __setattr__ 拦截赋值？不是！默认赋值直接写入 __dict__
print(m.x)    # 10 — __getattr__ 不会被调用！属性已经存在
print(m.y)    # __getattr__ called for y — 只有缺失属性才触发

class Misunderstood2:
    def __getattribute__(self, name):
        print(f"__getattribute__ called for {name}")
        return object.__getattribute__(self, name)

m2 = Misunderstood2()
m2.x = 10      # 赋值正常
print(m2.x)    # __getattribute__ called for x — 每次都触发！
print(m2.y)    # __getattribute__ called for y → AttributeError → __getattr__? No!
                # __getattribute__ 没有 raise AttributeError，所以直接报错
```

**最佳实践**：

- 需要拦截所有读取 → 用 `__getattribute__`
- 需要为缺失属性提供兜底 → 用 `__getattr__`
- 不要同时定义两者，除非你明确知道你在做什么
- 如果同时定义了两者，确保 `__getattribute__` 中 `raise AttributeError` 的场景是你有意为之的

### 16.5.3 描述符与实例属性的优先级陷阱

**陷阱**：以为实例属性总能覆盖描述符。

```python
class MyDescriptor:
    """数据描述符"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get('_x', 0)

    def __set__(self, obj, value):
        obj.__dict__['_x'] = value


class MyClass:
    x = MyDescriptor()


obj = MyClass()
obj.x = 10
print(obj.x)  # 10 — 数据描述符工作正常

# 尝试用实例 __dict__ 覆盖（会失败！）
obj.__dict__['x'] = "I want to override!"
print(obj.x)  # 仍然是 10！数据描述符优先级更高

# 但是！非数据描述符可以被实例属性覆盖
class NonDataDesc:
    """非数据描述符"""
    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return "from non-data descriptor"


class MyClass2:
    x = NonDataDesc()


obj2 = MyClass2()
print(obj2.x)  # from non-data descriptor

obj2.x = "instance value"  # 实例属性覆盖了非数据描述符
print(obj2.x)  # instance value
```

**最佳实践**：

- 数据描述符（定义了 `__set__` 或 `__delete__`）优先级高于实例 `__dict__`
- 非数据描述符（只定义了 `__get__`）优先级低于实例 `__dict__`
- 如果需要描述符优先级低于实例属性，使用非数据描述符
- Python 的 `property`、`classmethod`、`staticmethod` 都是非数据描述符——所以它们可以被实例属性覆盖（虽然不建议这样做）

```python
# property 是数据描述符（有 __set__）
class WithProperty:
    @property
    def x(self):
        return self._x

    @x.setter
    def x(self, value):
        self._x = value


wp = WithProperty()
wp.x = 10
print(hasattr(type(wp).x, '__set__'))  # True — property 是数据描述符

# classmethod 是非数据描述符
class WithClassMethod:
    @classmethod
    def hello(cls):
        return "hello"


print(hasattr(type(WithClassMethod.hello), '__set__'))  # False — 非数据描述符
```

### 16.5.4 `__slots__` 的常见陷阱

**陷阱1：子类忘记定义 `__slots__`**

```python
class Parent:
    __slots__ = ('x',)

class Child(Parent):
    pass  # 没有 __slots__ → 子类实例有 __dict__

c = Child()
c.x = 1
c.dynamic = "I can add anything"  # 子类恢复了 __dict__
print(c.dynamic)  # I can add anything

# 修复：子类也定义 __slots__
class FixedChild(Parent):
    __slots__ = ('y',)

fc = FixedChild()
fc.x = 1
fc.y = 2
try:
    fc.dynamic = "nope"
except AttributeError as e:
    print(f"No dynamic attrs: {e}")
```

**陷阱2：`__slots__` 中重复父类的 slot**

```python
class Parent2:
    __slots__ = ('x', 'y')

class Child2(Parent2):
    __slots__ = ('x', 'z')  # 'x' 重复了

# 这不会报错，但是 'x' 的 slot 来自父类
# 子类中的 'x' 实际上是冗余的
c2 = Child2()
c2.x = 10  # 使用的是 Parent2 的 slot
print(f"Slot 'x' works: {c2.x}")

# 最佳实践：子类只定义自己新增的 slot
class BestChild(Parent2):
    __slots__ = ('z',)  # 只定义新增的，不重复 'x' 和 'y'
```

**陷阱3：`__slots__` 与 `__dict__` 的微妙交互**

```python
class SlotWithDict:
    __slots__ = ('x', '__dict__')

    def __init__(self):
        self.x = 1

obj = SlotWithDict()
obj.x = 10        # 使用 slot
obj.y = 20        # 使用 __dict__
obj.z = 30        # 使用 __dict__

# x 不在 __dict__ 中
print(f"'x' in __dict__: {'x' in obj.__dict__}")   # False
print(f"'y' in __dict__: {'y' in obj.__dict__}")   # True

# 但注意：如果先设置实例属性，再定义同名的数据描述符
# 数据描述符会"遮蔽"实例属性
```

**陷阱4：`__slots__` 与 `__class__` 赋值**

```python
class A:
    __slots__ = ('x',)

class B:
    __slots__ = ('x',)

a = A()
a.x = 10

# 在某些 Python 版本中，__class__ 赋值受限于 slot 兼容性
try:
    a.__class__ = B
    print(f"__class__ assignment worked: {type(a)}")
except TypeError as e:
    print(f"__class__ assignment failed: {e}")

# 只有当两个类的 slot 布局兼容时才能这样做
class C:

try:
```

### 16.5.5 `__setattr__` 中的属性名冲突

**陷阱**：在 `__setattr__` 中使用 `self.name = value` 导致递归。

```python
# 错误示范
class Bad:
    def __setattr__(self, name, value):
        if name == 'protected':
            raise AttributeError("Cannot set protected attribute")
        self.name = value  # RecursionError! 每次赋值都触发 __setattr__

# 正确做法
class Good:
    def __setattr__(self, name, value):
        if name == 'protected':
            raise AttributeError("Cannot set protected attribute")
        object.__setattr__(self, name, value)

# 或者使用 __dict__
class AlsoGood:
    def __setattr__(self, name, value):
        if name == 'protected':
            raise AttributeError("Cannot set protected attribute")
        self.__dict__[name] = value

# 但注意！如果类使用了 __slots__，self.__dict__ 可能不存在
class WithSlots:
    __slots__ = ('x',)

    def __setattr__(self, name, value):
        # self.__dict__[name] = value  # AttributeError! __slots__ 禁用了 __dict__
        object.__setattr__(self, name, value)  # 正确做法

ws = WithSlots()
ws.x = 10
print(ws.x)
```

**最佳实践**：始终使用 `object.__setattr__` 而非 `self.__dict__[name] = value`，因为前者在所有情况下都有效（包括 `__slots__` 类）。

### 16.5.6 性能陷阱：`__getattribute__` 的开销

```python
import timeit

class Plain:
    def __init__(self):
        self.x = 1

class WithGetattribute:
    def __getattribute__(self, name):
        return object.__getattribute__(self, name)

class WithGetattributeLogging:
    def __getattribute__(self, name):
        # 典型的审计/日志实现
        value = object.__getattribute__(self, name)
        return value


plain = Plain()
plain.x = 1
with_ga = WithGetattribute()
with_ga.x = 1
with_ga_log = WithGetattributeLogging()
with_ga_log.x = 1

N = 5_000_000

t_plain = timeit.timeit('plain.x', globals=globals(), number=N)
t_with_ga = timeit.timeit('with_ga.x', globals=globals(), number=N)
t_with_ga_log = timeit.timeit('with_ga_log.x', globals=globals(), number=N)

print(f"Plain:           {t_plain:.4f}s ({t_plain/t_plain:.1f}x)")
print(f"__getattribute__: {t_with_ga:.4f}s ({t_with_ga/t_plain:.1f}x)")
print(f"__getattribute__+log: {t_with_ga_log:.4f}s ({t_with_ga_log/t_plain:.1f}x)")

# 典型输出:
# Plain:             0.15s (1.0x)
# __getattribute__:  0.25s (1.7x)
# __getattribute__+log: 0.25s (1.7x)
#
# 关键发现：即使 __getattribute__ 内部只是调用 object.__getattribute__
# 也引入了约 70% 的额外开销，因为每次属性访问都多了一层 Python 函数调用
```

**最佳实践**：

1. 除非必要，不要定义 `__getattribute__`。它的性能开销显著
2. 如果只需要为缺失属性提供兜底值，用 `__getattr__`
3. 如果确实需要拦截所有属性访问，在 `__getattribute__` 中尽量减少逻辑——将内部属性访问委托给 `object.__getattribute__`
4. 在性能关键路径上，考虑使用描述符或 `property` 替代 `__getattribute__`

### 16.5.7 `__slots__` 最佳实践总结

```python
"""
__slots__ 使用决策树和最佳实践。
"""

# 决策树：
#
# 是否需要创建大量实例？（>1000个）
#   ├─ 是 → 使用 __slots__
#   │   ├─ 是否需要动态属性？
#   │   │   ├─ 是 → __slots__ = ('fixed_attr', '__dict__')
#   │   │   └─ 否 → __slots__ = ('attr1', 'attr2', ...)
#   │   └─ 是否需要弱引用？
#   │       ├─ 是 → __slots__ = ('attr1', '__weakref__')
#   │       └─ 否 → __slots__ = ('attr1',)
#   └─ 否 → 一般不需要 __slots__

# 最佳实践清单：

# 1. 每个子类都必须定义自己的 __slots__（除非你想要 __dict__）
class GoodParent:
    __slots__ = ('a',)

class GoodChild(GoodParent):
    __slots__ = ('b',)  # 只定义新增的 slot

# 2. 在 __slots__ 中包含 '__dict__' 来允许动态属性
class Flexible:
    __slots__ = ('fixed', '__dict__')

    def __init__(self):
        self.fixed = "slot attribute"
        self.dynamic = "dict attribute"

f = Flexible()
print(f"fixed: {f.fixed}, dynamic: {f.dynamic}")
print(f"fixed in __dict__: {'fixed' in f.__dict__}")  # False
print(f"dynamic in __dict__: {'dynamic' in f.__dict__}")  # True

# 3. 需要弱引用时包含 '__weakref__'
class WeakRefable:
    __slots__ = ('data', '__weakref__')

import weakref
wr_obj = WeakRefable()
wr_obj.data = "important"
wr = weakref.ref(wr_obj)
print(f"Weakref: {wr().data}")

# 4. 不要在 __slots__ 中重复父类的 slot
# 错误：
class BadChild(GoodParent):
    __slots__ = ('a', 'b')  # 'a' 重复了

# 正确：
class ProperChild(GoodParent):
    __slots__ = ('b',)  # 'a' 已在父类中定义

# 5. __slots__ 与 __init__ 的配合
class SlotInit:
    __slots__ = ('x', 'y', 'z')

    def __init__(self, x, y, z):
        # 直接赋值，__slots__ 使得赋值非常高效
        self.x = x
        self.y = y
        self.z = z

    def __repr__(self):
        return f"SlotInit({self.x}, {self.y}, {self.z})"

si = SlotInit(1, 2, 3)
print(si)
```

### 16.5.8 属性访问特殊方法的选择指南

```python
"""
根据不同需求选择合适的属性访问拦截方法。
"""

# ┌─────────────────────────────────────────────────────────────────┐
# │                      选择指南                                    │
# ├──────────────────────┬──────────────────────────────────────────┤
# │ 需求                  │ 推荐方法                                 │
# ├──────────────────────┼──────────────────────────────────────────┤
# │ 为缺失属性提供兜底值   │ __getattr__                              │
# │ 动态属性（代理/包装器）│ __getattr__ + __setattr__               │
# │ 拦截所有属性读取      │ __getattribute__（注意性能开销）         │
# │ 属性赋值验证          │ __setattr__ 或 property + 描述符        │
# │ 只读属性              │ property（getter only）                 │
# │ 可验证属性            │ 描述符（定义 __set__）                  │
# │ 审计所有属性访问      │ __getattribute__ + __setattr__          │
# │ 大量实例内存优化      │ __slots__                                │
# │ 不可变对象            │ __setattr__ + __delattr__ 拦截          │
# │ 延迟计算属性          │ __getattr__（缓存到 __dict__）          │
# └──────────────────────┴──────────────────────────────────────────┘

# 性能对比（从快到慢）：
# 1. 直接属性访问 (obj.x)          — 最快
# 2. __slots__ 属性访问             — 与直接访问相当，略快（无 __dict__ 查找）
# 3. property 访问                  — 略慢（需要调用 getter 函数）
# 4. 描述符访问                     — 与 property 相当
# 5. __getattr__ (命中时)           — 与直接访问相当（不触发）
# 6. __getattr__ (未命中时)          — 较慢（触发异常处理）
# 7. __getattribute__               — 慢（每次都调用 Python 函数）

import timeit

class Benchmark:
    __slots__ = ('slot_attr',)
    regular_attr = "regular"

    def __init__(self):
        self.slot_attr = "slot"
        self.regular_attr = "regular"

    @property
    def prop_attr(self):
        return "property"


b = Benchmark()

N = 2_000_000

results = {
    'direct': timeit.timeit('b.regular_attr', globals=globals(), number=N),
    'slot': timeit.timeit('b.slot_attr', globals=globals(), number=N),
    'property': timeit.timeit('b.prop_attr', globals=globals(), number=N),
}

print("性能对比（2M 次访问）:")
baseline = results['direct']
for name, t in sorted(results.items(), key=lambda x: x[1]):
    print(f"  {name:12s}: {t:.4f}s ({t/baseline:.2f}x baseline)")
```

## 本章小结

属性访问机制是 Python 面向对象编程中最精细的控制机制之一。核心要点：

1. **查找优先级**：数据描述符 > 实例 `__dict__` > 非数据描述符 > 类属性 > `__getattr__`。这个优先级链路是 Python 对象模型的基础。

2. **`__getattribute__` vs `__getattr__`**：前者拦截所有属性读取，后者仅在正常查找失败后兜底。除非确实需要拦截所有访问，否则优先使用 `__getattr__`。

3. **`__setattr__` / `__delattr__` 的递归陷阱**：在方法内部必须使用 `object.__setattr__` 或直接操作 `self.__dict__`，否则无限递归。

4. **`__slots__` 的本质**：通过 member descriptor（数据描述符）固定属性集合，移除实例 `__dict__`，在大量实例场景下可节省 40%~80% 内存。继承中子类必须显式声明自己的 `__slots__`。

5. **CPython 实现**：属性访问通过 `_PyObject_GetAttr` → `_PyType_Lookup` → 描述符检查 → `tp_dictoffset` 查找的 C 代码路径完成。理解这条路径有助于预测属性访问的性能特征。

6. **性能考量**：自定义 `__getattribute__` 引入约 70% 的额外开销；`__getattr__` 在属性命中时零开销；`__slots__` 不仅节省内存，还因跳过 `__dict__` 哈希查找而略微加速属性访问。
