## 14.1 概念与语法

### 14.1.1 什么是描述符

描述符（descriptor）是一个实现了描述符协议中任意方法的类。描述符协议由三个特殊方法组成：

```python
__get__(self, instance, owner)
__set__(self, instance, value)
__delete__(self, instance)
```

只要一个类定义了 `__get__`、`__set__` 或 `__delete__` 中的**至少一个**，该类的实例就是一个描述符。这三个方法的不同组合决定了描述符的类型和行为。

### 14.1.2 数据描述符与非数据描述符

Python 将描述符分为两类，二者的区别在于属性查找的优先级：

| 类型 | 定义的协议方法 | 查找优先级 |
|------|---------------|-----------|
| **数据描述符**（data descriptor） | `__get__` + `__set__`（或 `__delete__`） | 高于实例属性 |
| **非数据描述符**（non-data descriptor） | 仅 `__get__` | 低于实例属性 |

关键区分：只要定义了 `__set__` 或 `__delete__`，就是数据描述符。仅定义 `__get__` 则是非数据描述符。

```python
# 数据描述符 —— 同时定义 __get__ 和 __set__
class DataDescriptor:
    def __get__(self, instance, owner):
        return "data descriptor value"

    def __set__(self, instance, value):
        raise AttributeError("cannot set")

# 非数据描述符 —— 仅定义 __get__
class NonDataDescriptor:
    def __get__(self, instance, owner):
        return "non-data descriptor value"
```

### 14.1.3 协议方法的签名详解

```python
def __get__(self, instance, owner):
    """
    instance: 访问描述符的实例对象。
              通过类访问时（如 MyClass.attr），instance 为 None。
    owner:    instance 的类（即描述符被定义所在的类）。
              通过类访问时，owner 始终是那个类本身。
    返回值:   通过实例访问时通常返回计算后的值；
              通过类访问时通常返回描述符自身。
    """

def __set__(self, instance, value):
    """
    instance: 属性被设置的实例对象。
    value:    赋予属性的新值。
    无返回值。
    """

def __delete__(self, instance):
    """
    instance: 属性被删除的实例对象。
    无返回值。
    """
```

### 14.1.4 `__set_name__` 方法（Python 3.6+）

Python 3.6 引入了 `__set_name__(self, owner, name)` 协议方法。当描述符在类定义体中被赋值时，Python 会自动调用此方法，将属性名传递给描述符：

```python
class NamedDescriptor:
    def __set_name__(self, owner, name):
        self.public_name = name
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, None)

    def __set__(self, instance, value):
        setattr(instance, self.private_name, value)

class Person:
    name = NamedDescriptor()  # __set_name__ 被自动调用，name='name'
    age = NamedDescriptor()   # __set_name__ 被自动调用，name='age'

p = Person()
p.name = "Alice"
p.age = 30
print(p.name)  # Alice
print(p.age)   # 30
```

在 Python 3.6 之前，描述符必须手动传递属性名，通常通过元类或显式初始化来完成。`__set_name__` 的引入大大简化了描述符的编写。

## 14.2 原理与机制

### 14.2.1 属性查找优先级

这是描述符协议中最核心的知识点。当你通过 `obj.attr` 访问一个属性时，Python 遵循以下查找顺序：

```
1. 数据描述符（定义了 __get__ + __set__/__delete__ 的类属性）
2. 实例属性（obj.__dict__ 中的键）
3. 非数据描述符（仅定义了 __get__ 的类属性）
4. 类属性（普通类属性，包括从基类继承的）
5. __getattr__ 钩子方法
```

用 CPython 源码级别的伪代码表示：

```python
# 伪代码：属性查找的完整逻辑（等价于 CPython 的 _PyObject_GenericGetAttrWithDict）
def attribute_lookup(obj, name):
    # Step 1: 在 type(obj) 的 MRO 中查找数据描述符
    for cls in type(obj).__mro__:
        if name in cls.__dict__:
            attr = cls.__dict__[name]
            if hasattr(type(attr), '__get__') and (
                hasattr(type(attr), '__set__') or hasattr(type(attr), '__delete__')
            ):
                # 找到数据描述符，优先级最高
                return attr.__get__(obj, type(obj))

    # Step 2: 查找实例属性
    if name in obj.__dict__:
        return obj.__dict__[name]

    # Step 3: 在 type(obj) 的 MRO 中查找非数据描述符
    for cls in type(obj).__mro__:
        if name in cls.__dict__:
            attr = cls.__dict__[name]
            if hasattr(type(attr), '__get__'):
                # 非数据描述符
                return attr.__get__(obj, type(obj))
            else:
                # 普通类属性
                return attr

    # Step 4: __getattr__ 钩子
    if hasattr(type(obj), '__getattr__'):
        return type(obj).__getattr__(obj, name)

    raise AttributeError(f"'{type(obj).__name__}' object has no attribute '{name}'")
```

### 2.2 优先级验证实验

以下代码通过严格的实验证明查找优先级：

```python
class DataDesc:
    """数据描述符：定义了 __get__ 和 __set__"""
    def __get__(self, instance, owner):
        return "数据描述符的值"

    def __set__(self, instance, value):
        instance.__dict__['attr'] = "被数据描述符拦截的值"

class NonDataDesc:
    """非数据描述符：仅定义了 __get__"""
    def __get__(self, instance, owner):
        return "非数据描述符的值"

class Demo:
    data_attr = DataDesc()    # 数据描述符
    nondata_attr = NonDataDesc()  # 非数据描述符
    class_attr = "普通类属性"       # 普通类属性

d = Demo()

# --- 实验 1：数据描述符 vs 实例属性 ---
d.__dict__['data_attr'] = "实例属性覆盖"  # 直接写入 __dict__
print(d.data_attr)  # 输出: 数据描述符的值
# 数据描述符优先级 > 实例属性，即使 __dict__ 中有同键

# --- 实验 2：非数据描述符 vs 实例属性 ---
d.__dict__['nondata_attr'] = "实例属性覆盖"
print(d.nondata_attr)  # 输出: 实例属性覆盖
# 实例属性优先级 > 非数据描述符

# 删除实例属性后
del d.__dict__['nondata_attr']
print(d.nondata_attr)  # 输出: 非数据描述符的值

# --- 实验 3：实例属性 vs 普通类属性 ---
d.__dict__['class_attr'] = "实例属性覆盖类属性"
print(d.class_attr)  # 输出: 实例属性覆盖类属性
# 实例属性 > 普通类属性（因为普通类属性既不是数据描述符也不是非数据描述符，
# 而非数据描述符的优先级低于实例属性）

print("--- 优先级总结 ---")
print("数据描述符 > 实例属性 > 非数据描述符 > 普通类属性")
```

### 2.3 CPython 源码分析

在 CPython 中，属性访问的核心路径经过以下函数链：

```
PyObject_GetAttr()  →  _PyObject_GenericGetAttrWithDict()
```

在 `Objects/object.c` 中，`_PyObject_GenericGetAttrWithDict` 的核心逻辑（简化后的 C 代码）：

```c
/* Objects/object.c - 简化版 _PyObject_GenericGetAttrWithDict */
PyObject *
_PyObject_GenericGetAttrWithDict(PyObject *obj, PyObject *name,
                                  PyObject *dict, int suppress)
{
    PyTypeObject *tp = Py_TYPE(obj);
    PyObject *descr = NULL;
    PyObject *res = NULL;
    descrgetfunc f;

    /* Step 1: 在类型对象中查找描述符 */
    descr = _PyType_Lookup(tp, name);
    if (descr != NULL) {
        PyTypeObject *descr_type = Py_TYPE(descr);
        /* 检查是否为数据描述符 */
        f = descr_type->tp_descr_get;
        if (f != NULL && (descr_type->tp_descr_set != NULL ||
            PyDescr_IsData(descr))) {
            /* 数据描述符：最高优先级 */
            res = f(descr, obj, (PyObject *)tp);
            return res;
        }
    }

    /* Step 2: 查找实例 __dict__ */
    if (dict != NULL) {
        res = PyDict_GetItemWithError(dict, name);
        if (res != NULL) {
            Py_INCREF(res);
            return res;
        }
    }

    /* Step 3: 非数据描述符或普通类属性 */
    if (descr != NULL) {
        f = Py_TYPE(descr)->tp_descr_get;
        if (f != NULL) {
            /* 非数据描述符 */
            res = f(descr, obj, (PyObject *)tp);
            return res;
        }
        /* 普通类属性 */
        Py_INCREF(descr);
        return descr;
    }

    /* Step 4: __getattr__ */
    if (suppress != 0)
        return NULL;

    PyErr_Format(PyExc_AttributeError,
                 "'%.200s' object has no attribute '%U'",
                 tp->tp_name, name);
    return NULL;
}
```

关键数据结构 `PyTypeObject` 中与描述符相关的字段：

```c
/* Include/cpython/object.h - 类型对象中的描述符相关字段 */
typedef struct _typeobject {
    /* ... */
    descrgetfunc tp_descr_get;   /* 对应 __get__ */
    descrsetfunc tp_descr_set;   /* 对应 __set__ 和 __delete__ */
    /* ... */
} PyTypeObject;
```

`PyDescr_IsData` 宏的定义揭示了判断数据描述符的本质：

```c
/* Include/object.h */
#define PyDescr_IsData(d) (Py_TYPE(d)->tp_descr_set != NULL)
```

也就是说，`tp_descr_set != NULL` 就足以判定一个类型为数据描述符。这对应 Python 层面的 `hasattr(cls, '__set__') or hasattr(cls, '__delete__')`。

### 2.4 描述符协议中的 `self` 与 `instance` 参数

初学者最常困惑的问题之一：`__get__` 的参数中，`self` 和 `instance` 分别是什么？

```python
class MyDescriptor:
    def __get__(self, instance, owner):
        print(f"  self = {self!r}")
        print(f"  instance = {instance!r}")
        print(f"  owner = {owner!r}")
        return "描述符返回值"

class MyClass:
    attr = MyDescriptor()

obj = MyClass()

print("通过实例访问:")
result = obj.attr
# self = <MyDescriptor object>  —— 描述符实例本身
# instance = <MyClass object>     —— 调用者实例
# owner = <class 'MyClass'>      —— 定义描述符的类

print("\n通过类访问:")
result = MyClass.attr
# self = <MyDescriptor object>  —— 描述符实例本身
# instance = None                —— 没有实例上下文
# owner = <class 'MyClass'>      —— 定义描述符的类
```

理解要点：
- **`self`**：描述符对象自身。因为描述符是**类属性**，整个类只有一份描述符实例，所有类实例共享同一个描述符。`self` 就是那个唯一的描述符实例。
- **`instance`**：触发属性访问的类实例。当通过 `obj.attr` 访问时，`instance` 是 `obj`；当通过 `MyClass.attr` 访问时，`instance` 是 `None`。
- **`owner`**：始终是定义描述符的类（即 `type(instance)`）。

由于所有实例共享同一个描述符，`__set__` 和 `__delete__` 中必须使用 `instance` 参数来将数据存储到实例的 `__dict__` 中，而不是描述符自身的属性中：

```python
class BadDescriptor:
    """错误示范：将数据存在描述符自身——所有实例共享同一个值！"""
    def __get__(self, instance, owner):
        return self._value

    def __set__(self, instance, value):
        self._value = value  # 危险！所有实例共享同一个 _value

class GoodDescriptor:
    """正确做法：将数据存在实例的 __dict__ 中"""
    def __set_name__(self, owner, name):
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, None)

    def __set__(self, instance, value):
        setattr(instance, self.private_name, value)  # 存到实例上
```

### 2.5 `__set__` 的查找路径与写入优先级

属性的**写入**（赋值）路径与读取略有不同。`obj.attr = value` 的查找顺序：

```
1. 数据描述符的 __set__
2. 实例属性（直接写入 __dict__）
```

注意：非数据描述符**不会拦截**赋值操作。如果类属性是非数据描述符，对实例的同名属性赋值会直接覆盖实例属性：

```python
class NonDataDesc:
    def __get__(self, instance, owner):
        return "非数据描述符的值"

class MyClass:
    attr = NonDataDesc()

obj = MyClass()
obj.attr = "新值"  # 直接写入 obj.__dict__['attr'] = "新值"
print(obj.attr)     # 输出: 新值（实例属性覆盖了非数据描述符）
print(MyClass.attr) # 输出: 非数据描述符的值（类属性不变）
```

CPython 中属性赋值的核心路径：

```c
/* Objects/object.c - 简化版 _PyObject_GenericSetAttrWithDict */
int
_PyObject_GenericSetAttrWithDict(PyObject *obj, PyObject *name,
                                  PyObject *value, PyObject *dict)
{
    PyTypeObject *tp = Py_TYPE(obj);
    PyObject *descr;
    descrsetfunc f;

    /* Step 1: 查找数据描述符 */
    descr = _PyType_Lookup(tp, name);
    if (descr != NULL) {
        f = Py_TYPE(descr)->tp_descr_set;
        if (f != NULL) {
            /* 数据描述符拦截赋值 */
            return f(descr, obj, value);
        }
    }

    /* Step 2: 写入实例 __dict__ */
    if (dict != NULL) {
        if (value != NULL) {
            return PyDict_SetItem(dict, name, value);
        } else {
            /* del 操作 */
            return PyDict_DelItem(dict, name);
        }
    }
    /* ... 错误处理 ... */
}
```

### 2.6 `__delete__` 的行为

`del obj.attr` 的查找路径与赋值类似，数据描述符的 `__delete__` 优先于实例 `__dict__` 的删除：

```python
class ProtectedAttr:
    def __set_name__(self, owner, name):
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, None)

    def __set__(self, instance, value):
        setattr(instance, self.private_name, value)

    def __delete__(self, instance):
        raise AttributeError("此属性不可删除")

class Person:
    name = ProtectedAttr()

p = Person()
p.name = "Alice"
try:
    del p.name  # 触发 __delete__，抛出 AttributeError
except AttributeError as e:
    print(e)  # 此属性不可删除
```

### 2.7 描述符与 `__slots__` 的关系

`__slots__` 的底层机制也依赖描述符。当你定义 `__slots__ = ['x', 'y']` 时，Python 为每个 slot 创建一个数据描述符：

```python
class Point:
    __slots__ = ('x', 'y')

# 查看 Point.x 的类型
print(type(Point.x))   # <class 'member_descriptor'>
print(type(Point.y))   # <class 'member_descriptor'>

# member_descriptor 实现了完整的描述符协议
print(hasattr(type(Point.x), '__get__'))   # True
print(hasattr(type(Point.x), '__set__'))   # True
print(hasattr(type(Point.x), '__delete__')) # True
```

`member_descriptor` 是 C 层面实现的描述符，定义在 `Objects/descrobject.c` 中。它比普通 Python 描述符更快，因为它直接操作 C 结构体中的偏移量，避免了 `__dict__` 查找的开销。

## 3. 使用场景

### 3.1 property 的实现原理

`property` 是 Python 内置的数据描述符。它的实现揭示了描述符的典型模式：

```python
# property 的等价 Python 实现（简化版）
class Property:
    """模拟内置 property 的核心逻辑"""

    def __init__(self, fget=None, fset=None, fdel=None, doc=None):
        self.fget = fget
        self.fset = fset
        self.fdel = fdel
        if doc is None and fget is not None:
            doc = fget.__doc__
        self.__doc__ = doc

    def __get__(self, instance, owner):
        if instance is None:
            return self
        if self.fget is None:
            raise AttributeError("unreadable attribute")
        return self.fget(instance)

    def __set__(self, instance, value):
        if self.fset is None:
            raise AttributeError("can't set attribute")
        self.fset(instance, value)

    def __delete__(self, instance):
        if self.fdel is None:
            raise AttributeError("can't delete attribute")
        self.fdel(instance)

    def getter(self, fget):
        return type(self)(fget, self.fset, self.fdel, self.__doc__)

    def setter(self, fset):
        return type(self)(self.fget, fset, self.fdel, self.__doc__)

    def deleter(self, fdel):
        return type(self)(self.fget, self.fset, fdel, self.__doc__)
```

关键观察：

1. **`property` 是数据描述符**——它定义了 `__get__` 和 `__set__`（即使 `fset=None`，`__set__` 方法仍然存在，只是会抛出 `AttributeError`）。这意味着即使没有 setter，`property` 的优先级也高于实例属性。

2. **通过实例访问返回计算值，通过类访问返回 property 对象**——当 `instance is None` 时，`__get__` 返回 `self`，这允许你在类层面检查 property 对象本身。

3. **setter/deleter 装饰器返回新的 property 对象**——遵循不可变模式，每次修改都创建新的 property 实例。

验证 property 是数据描述符：

```python
class Demo:
    @property
    def value(self):
        return self._value

# property 类型有 __set__ 方法，因此是数据描述符
print(hasattr(property, '__set__'))    # True
print(hasattr(property, '__delete__'))  # True

# 即使只定义了 getter，property 仍然是数据描述符
d = Demo()
d.__dict__['value'] = "实例属性"
# 访问 d.value 时，数据描述符优先级高于实例属性
# 但因为 getter 尝试访问 self._value（而非 self.__dict__['value']），
# 所以行为取决于 _value 是否存在
```

### 3.2 ORM 中的字段验证

描述符在 ORM（对象关系映射）中最经典的应用是字段验证和类型检查。以下是完整的实现：

```python
class ValidatedField:
    """带类型检查和验证的 ORM 字段描述符基类"""

    def __init__(self, *, name=None, nullable=True, default=None):
        self.name = name
        self.nullable = nullable
        self.default = default

    def __set_name__(self, owner, name):
        if self.name is None:
            self.name = name
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, self.default)

    def __set__(self, instance, value):
        self.validate(value)
        setattr(instance, self.private_name, value)

    def validate(self, value):
        if value is None:
            if not self.nullable:
                raise ValueError(f"{self.name} cannot be None")
            return
        self._validate_type(value)
        self._validate_value(value)

    def _validate_type(self, value):
        pass  # 子类覆盖

    def _validate_value(self, value):
        pass  # 子类覆盖


class IntegerField(ValidatedField):
    def __init__(self, *, min_value=None, max_value=None, **kwargs):
        super().__init__(**kwargs)
        self.min_value = min_value
        self.max_value = max_value

    def _validate_type(self, value):
        if not isinstance(value, int) or isinstance(value, bool):
            raise TypeError(
                f"{self.name} must be an integer, got {type(value).__name__}"
            )

    def _validate_value(self, value):
        if self.min_value is not None and value < self.min_value:
            raise ValueError(
                f"{self.name} must be >= {self.min_value}, got {value}"
            )
        if self.max_value is not None and value > self.max_value:
            raise ValueError(
                f"{self.name} must be <= {self.max_value}, got {value}"
            )


class StringField(ValidatedField):
    def __init__(self, *, min_length=0, max_length=None, pattern=None, **kwargs):
        super().__init__(**kwargs)
        self.min_length = min_length
        self.max_length = max_length
        self.pattern = pattern

    def _validate_type(self, value):
        if not isinstance(value, str):
            raise TypeError(
                f"{self.name} must be a string, got {type(value).__name__}"
            )

    def _validate_value(self, value):
        if len(value) < self.min_length:
            raise ValueError(
                f"{self.name} too short: {len(value)} < {self.min_length}"
            )
        if self.max_length is not None and len(value) > self.max_length:
            raise ValueError(
                f"{self.name} too long: {len(value)} > {self.max_length}"
            )
        if self.pattern is not None:
            import re
            if not re.match(self.pattern, value):
                raise ValueError(
                    f"{self.name} doesn't match pattern: {self.pattern}"
                )


# --- ORM 模型定义 ---
class Model:

        super().__init_subclass__(**kwargs)
        cls._fields = {}
        for name, value in cls.__dict__.items():
            if isinstance(value, ValidatedField):
                cls._fields[name] = value
```

### 3.3 懒加载描述符

描述符非常适合实现懒加载（lazy loading）——仅在首次访问时计算值，之后缓存结果：

```python
class LazyProperty:
    """懒加载属性描述符。

    首次访问时调用函数计算值，之后将结果存储在实例的 __dict__ 中。
    后续访问直接从 __dict__ 读取，不再经过描述符。

    由于这是非数据描述符（仅定义 __get__），
    一旦实例 __dict__ 中存在同名键，描述符就不会再被调用。
    """

    def __init__(self, func):
        self.func = func
        self.attr_name = func.__name__

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 计算值
        value = self.func(instance)
        # 存入实例 __dict__，后续访问直接命中，不经过描述符
        instance.__dict__[self.attr_name] = value
        return value


class ExpensiveResource:
    @LazyProperty
    def database_connection(self):
        print("建立数据库连接...（耗时操作）")
        return {"connection": "established", "id": 42}

    @LazyProperty
    def config(self):
        print("加载配置文件...（耗时操作）")
        return {"debug": False, "timeout": 30}

resource = ExpensiveResource()
print("首次访问 database_connection:")
conn = resource.database_connection  # 触发计算
# 建立数据库连接...（耗时操作）

print("再次访问 database_connection:")
conn2 = resource.database_connection  # 直接从 __dict__ 读取
# 无输出——不触发计算

print(conn is conn2)  # True，同一个对象
```

### 3.5 标准库中的描述符

#### classmethod

`classmethod` 是非数据描述符——它只定义了 `__get__`：

```python
class ClassMethodDescriptor:
    """模拟 classmethod 的核心逻辑"""
    def __init__(self, func):
        self.func = func

    def __get__(self, instance, owner):
        if owner is None:
            owner = type(instance)
        # 返回绑定了类的方法
        return self.func.__get__(owner)  # 利用函数自身的描述符协议


class MyClass:
    count = 0

    @classmethod
    def get_count(cls):
        return cls.count

# 验证 classmethod 是非数据描述符
print(hasattr(classmethod, '__get__'))    # True
print(hasattr(classmethod, '__set__'))    # False
print(hasattr(classmethod, '__delete__'))  # False
```

在 CPython 中，`classmethod` 对应 `PyClassMethod_Type`，定义在 `Objects/classobject.c`：

```c
/* Objects/classobject.c - classmethod_descr_get 的核心逻辑 */
static PyObject *
classmethod_descr_get(PyObject *self, PyObject *obj, PyObject *type)
{
    /* 如果 type 为 NULL，从 obj 推断 */
    if (type == NULL) {
        type = (PyObject *)(Py_TYPE(obj));
    }
    /* 获取底层函数 */
    PyObject *func = PyClassMethod_GET_FUNCTION(self);
    /* 将函数绑定到类 */
    return PyMethod_New(func, type);
}
```

#### staticmethod

`staticmethod` 也是非数据描述符，它返回原始函数，不做任何绑定：

```python
class StaticMethodDescriptor:
    """模拟 staticmethod 的核心逻辑"""
    def __init__(self, func):
        self.func = func

    def __get__(self, instance, owner):
        # 直接返回函数，不做任何绑定
        return self.func


class Math:
    @staticmethod
    def add(a, b):
        return a + b

# 验证 staticmethod 是非数据描述符
print(hasattr(staticmethod, '__get__'))    # True
print(hasattr(staticmethod, '__set__'))    # False
```

CPython 中 `staticmethod` 的 `__get__` 实现极其简单：

```c
/* Objects/classobject.c - staticmethod_descr_get */
static PyObject *
staticmethod_descr_get(PyObject *self, PyObject *obj, PyObject *type)
{
    /* 直接返回底层函数，忽略 instance 和 owner */
    PyObject *func = PyStaticMethod_GET_FUNCTION(self);
    Py_INCREF(func);
    return func;
}
```

#### functools.cached_property（Python 3.8+）

`functools.cached_property` 是一个精巧的非数据描述符。它的工作原理与前面的 `LazyProperty` 类似，但增加了线程安全：

```python
from functools import cached_property

class DataLoader:
    @cached_property
    def data(self):
        print("计算中...")
        return list(range(1000000))

loader = DataLoader()
print("首次访问:", len(loader.data))  # 触发计算
# 计算中...
# 首次访问: 1000000

print("再次访问:", len(loader.data))  # 从 __dict__ 读取
# 再次访问: 1000000（无"计算中..."输出）

# cached_property 是非数据描述符
# 可以被实例属性覆盖
print(hasattr(cached_property, '__set__'))    # False
print(hasattr(cached_property, '__delete__'))  # True（Python 3.8+ 支持删除缓存）
```

Python 3.12 对 `cached_property` 做了重要修改：它在首次访问时将自己从类中"替换"为实例属性。这是通过 `__set_name__` 记录属性名，然后在 `__get__` 中将计算结果直接写入 `instance.__dict__` 实现的。

### 3.6 方法本身就是非数据描述符

Python 函数对象天然实现了 `__get__`，因此每个方法都是非数据描述符。这是 Python 绑定方法（bound method）机制的核心：

```python
def demo_func(self):
    return 42

# 函数对象有 __get__
print(hasattr(demo_func, '__get__'))  # True

class MyClass:
    def method(self):
        return self

# MyClass.method 是一个函数（非数据描述符）
print(type(MyClass.method))  # <class 'function'>

# obj.method 触发 __get__，返回绑定方法
obj = MyClass()
print(type(obj.method))      # <class 'method'>
print(obj.method() is obj)   # True

# 绑定方法的 __func__ 和 __self__
bound = obj.method
print(bound.__func__ is MyClass.method)  # True
print(bound.__self__ is obj)              # True
```

CPython 中函数的 `__get__` 实现（`Objects/classobject.c`）：

```c
/* Objects/classobject.c - PyMethod_New 的核心逻辑 */
PyObject *
PyMethod_New(PyObject *func, PyObject *self)
{
    /* 如果 instance 为 NULL，返回函数本身（类访问） */
    if (self == NULL) {
        Py_INCREF(func);
        return func;
    }
    /* 否则创建绑定方法对象 */
    PyMethodObject *method = PyObject_GC_New(
        PyMethodObject, &PyMethod_Type
    );
    method->im_func = func;    /* __func__ */
    method->im_self = self;    /* __self__ */
    return (PyObject *)method;
}
```

这就是为什么实例方法可以被实例属性覆盖：

```python
class MyClass:
    def method(self):
        return "original"

obj = MyClass()
print(obj.method())  # "original"

# 方法是非数据描述符，可以被实例属性覆盖
obj.method = lambda: "overridden"
print(obj.method())  # "overridden"
```

### 3.7 描述符实现方法的统一调度（多分派）

描述符可以用于实现基于参数类型的运行时方法分派：

```python
class MultiMethod:
    """基于参数类型的运行时方法分派描述符"""

    def __init__(self, name):
        self.name = name
        self._signatures = {}  # {参数类型元组: 函数}

    def register(self, *types):
        """装饰器：注册特定参数类型组合对应的函数"""
        def decorator(func):
            self._signatures[types] = func
            return self
        return decorator

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 返回一个可调用对象，它会在运行时选择正确的函数
        def dispatcher(*args, **kwargs):
            # 获取参数的类型
            arg_types = tuple(type(a) for a in args)
            if arg_types not in self._signatures:
                raise TypeError(
                    f"No implementation of {self.name} for types {arg_types}"
                )
            return self._signatures[arg_types](instance, *args, **kwargs)
        return dispatcher


class Calculator:
    compute = MultiMethod("compute")

# 注册不同类型的方法实现
@Calculator.compute.register(int, int)
def compute_int_int(self, a, b):
    return a + b

@Calculator.compute.register(float, float)
def compute_float_float(self, a, b):
    return a * 1.0 + b * 1.0

@Calculator.compute.register(str, str)
def compute_str_str(self, a, b):
    return f"{a} + {b}"

calc = Calculator()
print(calc.compute(1, 2))       # 3 (int + int)
print(calc.compute(1.0, 2.0))    # 3.0 (float + float)
print(calc.compute("a", "b"))   # a + b (str + str)

try:
    calc.compute(1, "x")
except TypeError as e:
    print(e)  # No implementation of compute for types (<class 'int'>, <class 'str'>)
```

```python
class ReadOnly:
    """只读属性描述符。

    一旦设置值，就不能再修改或删除。
    这比 @property（不带 setter）更强：即使通过 __dict__ 也不行。
    """

    def __set_name__(self, owner, name):
        self.public_name = name
        self.private_name = f'_{name}'
        self._initialized = f'_{name}_initialized'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        if not getattr(instance, self._initialized, False):
            raise AttributeError(
                f"'{self.public_name}' has not been set"
            )
        return getattr(instance, self.private_name)

    def __set__(self, instance, value):
        if getattr(instance, self._initialized, False):
            raise AttributeError(
                f"'{self.public_name}' is read-only"
            )
        setattr(instance, self.private_name, value)
        setattr(instance, self._initialized, True)

    def __delete__(self, instance):
        raise AttributeError(
            f"'{self.public_name}' cannot be deleted"
        )
```

### 4.2 带缓存和过期机制的描述符

```python
import time

class CachedWithExpiry:
    """带过期时间的缓存描述符"""

    def __init__(self, ttl_seconds=60):
        self.ttl = ttl_seconds

    def __set_name__(self, owner, name):
        self.attr_name = name
        self.cache_key = f'_{name}_cache'
        self.timestamp_key = f'_{name}_timestamp'

    def __get__(self, instance, owner):
        if instance is None:
            return self

        # 检查缓存是否存在且未过期
        timestamp = getattr(instance, self.timestamp_key, 0)
        now = time.time()

        if now - timestamp < self.ttl:
            cache = getattr(instance, self.cache_key, None)
            if cache is not None:
                return cache

        # 缓存不存在或已过期，重新计算
        compute_method = getattr(instance, f'_compute_{self.attr_name}', None)
        if compute_method is None:
            raise NotImplementedError(
                f"Must define _compute_{self.attr_name} method"
            )

        value = compute_method()
        setattr(instance, self.cache_key, value)
        setattr(instance, self.timestamp_key, now)
        return value

    def __set__(self, instance, value):
        setattr(instance, self.cache_key, value)
        setattr(instance, self.timestamp_key, time.time())

    def __delete__(self, instance):
        """强制使缓存失效"""
        if hasattr(instance, self.cache_key):
            delattr(instance, self.cache_key)
        if hasattr(instance, self.timestamp_key):
            delattr(instance, self.timestamp_key)


class APIClient:
    token = CachedWithExpiry(ttl_seconds=2)
    user_info = CachedWithExpiry(ttl_seconds=5)

    def _compute_token(self):
        print("  重新生成 token...")
        return f"token_{hash(id(self))}"

    def _compute_user_info(self):
        print("  重新加载用户信息...")
        return {"name": "Alice", "role": "admin"}

client = APIClient()
print("首次访问 token:", client.token)     # 触发计算
print("缓存命中 token:", client.token)     # 缓存命中
time.sleep(3)
print("过期后访问 token:", client.token)    # 再次触发计算

client.token = "manual_token"              # 手动设置
print("手动设置 token:", client.token)     # "manual_token"
del client.token                            # 使缓存失效
print("失效后访问 token:", client.token)     # 触发计算
```

### 4.3 弱引用描述符（避免内存泄漏）

当描述符需要维护实例到值的映射时，使用普通字典会导致内存泄漏。使用 `weakref.WeakKeyDictionary` 是标准解法：

```python
import weakref

class WeakReferencedDescriptor:
    """使用弱引用的描述符，避免内存泄漏。

    适用场景：描述符需要为每个实例维护独立状态，
    但又不想在实例 __dict__ 中存储（例如实例使用了 __slots__）。
    """

    def __init__(self, default=None):
        self.default = default
        self._storage = weakref.WeakKeyDictionary()

    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return self._storage.get(instance, self.default)

    def __set__(self, instance, value):
        self._storage[instance] = value

    def __delete__(self, instance):
        try:
            del self._storage[instance]
        except KeyError:
            raise AttributeError(
                f"'{self.name}' has not been set"
            )


class Vector:
    """使用 __slots__ 的类，实例没有 __dict__"""
    __slots__ = ()  # 没有 __dict__，无法使用实例属性存储

    x = WeakReferencedDescriptor(default=0.0)
    y = WeakReferencedDescriptor(default=0.0)

v1 = Vector()
v1.x = 1.0
v1.y = 2.0

v2 = Vector()
v2.x = 10.0
v2.y = 20.0

print(f"v1: ({v1.x}, {v1.y})")  # (1.0, 2.0)
print(f"v2: ({v2.x}, {v2.y})")  # (10.0, 20.0)

# 当 v1 被垃圾回收后，弱引用自动清除
import gc
del v1
gc.collect()
# WeakKeyDictionary 中的 v1 条目已被自动移除
```

**注意**：`WeakKeyDictionary` 要求键（即实例）是可哈希的，且不是基础类型（如 int、str、tuple 的子类）。如果实例不可哈希，需要改用 `WeakValueDictionary` 配合唯一标识符的方案。

```python
class Computed:
    """计算属性描述符：值由其他属性自动计算得出"""

    def __init__(self, *dependencies, compute_fn=None):
        self.dependencies = dependencies
        self.compute_fn = compute_fn

    def __set_name__(self, owner, name):
        self.name = name
        self.private_name = f'_{name}_cached'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 每次都重新计算（保证依赖变化时结果正确）
        if self.compute_fn is not None:
            return self.compute_fn(instance)
        # 或者收集依赖值并计算
        values = [
            getattr(instance, dep) for dep in self.dependencies
        ]
        return sum(values)  # 默认行为：求和

    def __set__(self, instance, value):
        raise AttributeError(

        )

        self.width = width
        self.height = height
```

### 4.5 ORM 关系描述符（外键与懒加载）

```python
class ForeignKey:
    """ORM 风格的外键描述符，支持懒加载关联对象"""

    def __init__(self, related_class, lazy=True):
        self.related_class = related_class
        self.lazy = lazy

    def __set_name__(self, owner, name):
        self.name = name
        self.id_column = f'{name}_id'
        self.cache_column = f'_{name}_cached'

    def __get__(self, instance, owner):
        if instance is None:
            return self

        # 检查缓存
        cached = getattr(instance, self.cache_column, None)
        if cached is not None:
            return cached

        # 懒加载：根据 id 获取关联对象
        related_id = getattr(instance, self.id_column, None)
        if related_id is None:
            return None

        # 模拟从数据库加载
        related = self.related_class._load(related_id)
        if not self.lazy:
            # 非懒加载模式，缓存结果
            setattr(instance, self.cache_column, related)
        return related

    def __set__(self, instance, value):
        if isinstance(value, self.related_class):
            setattr(instance, self.id_column, value.id)
            setattr(instance, self.cache_column, value)
        elif value is None:
            setattr(instance, self.id_column, None)
            setattr(instance, self.cache_column, None)
        else:
            raise TypeError(
                f"Expected {self.related_class.__name__} or None, "
                f"got {type(value).__name__}"
            )


class Author:
    _registry = {}

    def __init__(self, id, name):
        self.id = id
        self.name = name
        Author._registry[id] = self

    @classmethod
    def _load(cls, id):
        return cls._registry.get(id)

    def __repr__(self):
        return f"Author(id={self.id}, name={self.name!r})"


class Book:
    _registry = {}

    author = ForeignKey(Author, lazy=True)

    def __init__(self, id, title, author_id):
        self.id = id
        self.title = title
        self.author_id = author_id
        Book._registry[id] = self

    @classmethod
    def _load(cls, id):
        return cls._registry.get(id)

    def __repr__(self):
        return f"Book(id={self.id}, title={self.title!r})"


# 创建测试数据
```

### 4.6 描述符与元类协作：自动字段注册

```python
class FieldRegistry(type):
    """元类：自动收集所有字段描述符"""
    def __new__(mcs, name, bases, namespace):
        cls = super().__new__(mcs, name, bases, namespace)
        cls._fields = {}
        cls._field_order = []

        # 遍历 MRO，收集所有描述符字段
        for klass in reversed(cls.__mro__):
            for attr_name, attr_value in vars(klass).items():
                if hasattr(type(attr_value), '__get__'):
                    if attr_name not in cls._fields:
                        cls._fields[attr_name] = attr_value
                        cls._field_order.append(attr_name)

        return cls


class Field(metaclass=FieldRegistry):
    """字段描述符基类，由元类自动注册"""
    def __init__(self, *, required=False, default=None):
        self.required = required
        self.default = default

    def __set_name__(self, owner, name):
        self.name = name
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, self.default)

    def __set__(self, instance, value):
        self._validate(instance, value)
        setattr(instance, self.private_name, value)

    def _validate(self, instance, value):
        if value is None and self.required:
            raise ValueError(f"'{self.name}' is required")


class Schema(metaclass=FieldRegistry):
    """模式基类，利用元类自动注册字段"""
    pass


class UserSchema(Schema):
    id = Field(required=True)
    name = Field(required=True)
    email = Field(required=False, default="")
    age = Field(required=False, default=0)

# 元类自动收集了所有字段
print("Fields:", UserSchema._field_order)
# ['id', 'name', 'email', 'age']

for name, field in UserSchema._fields.items():
    print(f"  {name}: required={field.required}, default={field.default}")
```

## 5. 常见陷阱与最佳实践

### 5.1 陷阱：描述符存储位置错误

**错误示范**：将实例数据存储在描述符自身属性中。

```python
class BadDescriptor:
    """所有实例共享同一个值！"""
    def __get__(self, instance, owner):
        return self._value

    def __set__(self, instance, value):
        self._value = value  # 错误！存储在描述符自身

class MyClass:
    attr = BadDescriptor()

a = MyClass()
b = MyClass()
a.attr = "first"
print(b.attr)  # "first" —— b 的值被 a 的赋值覆盖了！
```

**正确做法**：存储在实例的 `__dict__` 中。

```python
class GoodDescriptor:
    def __set_name__(self, owner, name):
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, None)

    def __set__(self, instance, value):
        setattr(instance, self.private_name, value)

class MyClass:
    attr = GoodDescriptor()

a = MyClass()
b = MyClass()
a.attr = "first"
b.attr = "second"
print(a.attr)  # "first"
print(b.attr)  # "second"
```

**例外**：如果描述符的值对所有实例都相同（如常量），存储在描述符自身是可以的。或者使用 `WeakKeyDictionary` 存储实例到值的映射。

### 5.2 陷阱：忘记处理 `instance is None` 的情况

```python
class ForgetfulDescriptor:
    def __get__(self, instance, owner):
        # 忘记检查 instance is None
        return instance.some_value  # 通过类访问时崩溃！

class MyClass:
    attr = ForgetfulDescriptor()

try:
    MyClass.attr  # AttributeError: 'NoneType' object has no attribute 'some_value'
except AttributeError as e:
    print(f"错误: {e}")
```

**正确做法**：

```python
class CarefulDescriptor:
    def __get__(self, instance, owner):
        if instance is None:
            return self  # 返回描述符自身
        return getattr(instance, '_attr', None)
```

### 5.3 陷阱：`__set_name__` 在非类定义场景中不会被调用

`__set_name__` 只在类定义体中赋值时自动调用。如果你在类定义之后动态添加描述符，必须手动调用 `__set_name__`：

```python
class MyDescriptor:
    def __set_name__(self, owner, name):
        print(f"__set_name__ called: name={name}")
        self.name = name

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, f'_{self.name}', None)

    def __set__(self, instance, value):
        setattr(instance, f'_{self.name}', value)

# 在类定义中赋值 —— __set_name__ 自动调用
class MyClass:
    attr = MyDescriptor()  # 输出: __set_name__ called: name=attr

# 在类定义后动态添加 —— __set_name__ 不会自动调用！
class AnotherClass:
    pass

AnotherClass.dynamic_attr = MyDescriptor()  # 无输出！__set_name__ 未调用

# 必须手动调用
desc = MyDescriptor()
desc.__set_name__(AnotherClass, 'dynamic_attr')
AnotherClass.dynamic_attr = desc
# 输出: __set_name__ called: name=dynamic_attr
```

### 5.4 陷阱：非数据描述符被实例属性覆盖

```python
class NonDataDescriptor:
    def __get__(self, instance, owner):
        return "来自描述符的值"

class MyClass:
    attr = NonDataDescriptor()

obj = MyClass()
print(obj.attr)  # "来自描述符的值"

# 直接写入 __dict__ 就可以覆盖非数据描述符
obj.__dict__['attr'] = "实例属性的值"
print(obj.attr)  # "实例属性的值" —— 非数据描述符被覆盖！
```

如果你需要防止实例属性覆盖，应该使用数据描述符（定义 `__set__`）。

### 5.5 陷阱：描述符与 `__slots__` 的冲突

当类使用 `__slots__` 时，实例没有 `__dict__`，描述符无法使用 `setattr(instance, private_name, value)`：

```python
class MyDescriptor:
    def __set_name__(self, owner, name):
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return getattr(instance, self.private_name, None)

    def __set__(self, instance, value):
        setattr(instance, self.private_name, value)  # 对 __slots__ 类无效

class SlottedClass:
    __slots__ = ('x', 'y')  # 没有 __dict__！

    attr = MyDescriptor()  # 尝试使用描述符

obj = SlottedClass()
try:
    obj.attr = "value"  # AttributeError: 'SlottedClass' object has no attribute '_attr'
except AttributeError as e:
    print(f"错误: {e}")
```

**解决方案**：

方案 1：将私有属性加入 `__slots__`：

```python
class SlottedClass:
    __slots__ = ('x', 'y', '_attr')  # 加入 _attr

    attr = MyDescriptor()

obj = SlottedClass()
obj.attr = "value"  # 现在可以工作了
print(obj.attr)      # "value"
```

方案 2：使用 `WeakKeyDictionary` 在描述符中存储数据（参见 4.3 节）。

### 5.6 陷阱：描述符在元类中的行为

描述符定义在元类上时，它们控制的是**类属性**的访问，而不是实例属性的访问：

```python
class MetaDescriptor:
    def __get__(self, instance, owner):
        print(f"  __get__: instance={instance}, owner={owner}")
        return "meta descriptor value"

    def __set__(self, instance, value):
        print(f"  __set__: instance={instance}, value={value}")

class MyMeta(type):
    # 定义在元类上的描述符控制类的属性访问
    class_attr = MetaDescriptor()

class MyClass(metaclass=MyMeta):
    pass

print(MyClass.class_attr)
# __get__: instance=None, owner=<class 'MyClass'>
# 输出: "meta descriptor value"

# 赋值时
MyClass.class_attr = "new value"
# __set__: instance=<class 'MyClass'>, value=new value
```

在元类描述符的 `__get__` 中，`instance` 是被访问属性的**类**（而非类的实例），`owner` 是**元类**。

### 5.7 陷阱：`__getattribute__` 与描述符的交互

`__getattribute__` 是属性访问的最终拦截点。如果你重写了 `__getattribute__`，需要注意不要破坏描述符协议：

```python
class Broken:
    def __getattribute__(self, name):
        # 错误：直接从 __dict__ 读取，跳过了数据描述符！
        try:
            return object.__getattribute__(self).__dict__[name]
        except KeyError:
            raise AttributeError(name)

class Correct:
    def __getattribute__(self, name):
        # 正确：调用 object.__getattribute__ 会遵循完整的描述符协议
        return super().__getattribute__(name)
```

### 5.8 性能对比

描述符引入了额外的函数调用开销。以下是不同属性访问方式的性能对比：

```python
import timeit

# --- 测试代码 ---
class PlainAttr:
    """普通属性"""
    def __init__(self):
        self.x = 0

class PropertyAttr:
    """@property"""
    @property
    def x(self):
        return self._x

    @x.setter
    def x(self, value):
        self._x = value

class DescriptorAttr:
    """自定义描述符"""
    class XDescriptor:
        def __set_name__(self, owner, name):
            self.private_name = f'_x'

        def __get__(self, instance, owner):
            if instance is None:
                return self
            return getattr(instance, self.private_name)

        def __set__(self, instance, value):
            setattr(instance, self.private_name, value)

    x = XDescriptor()

class SlotAttr:
    """__slots__"""
    __slots__ = ('x',)

# --- 性能测试 ---
setup = """
from __main__ import PlainAttr, PropertyAttr, DescriptorAttr, SlotAttr
obj_plain = PlainAttr()
obj_prop = PropertyAttr()
obj_prop.x = 0
obj_desc = DescriptorAttr()
obj_desc.x = 0
obj_slot = SlotAttr()
obj_slot.x = 0
"""

results = {}
for label, stmt in [
    ("普通属性读", "x = obj_plain.x"),
    ("普通属性写", "obj_plain.x = 1"),
    ("@property 读", "x = obj_prop.x"),
    ("@property 写", "obj_prop.x = 1"),
    ("描述符读", "x = obj_desc.x"),
    ("描述符写", "obj_desc.x = 1"),
    ("__slots__ 读", "x = obj_slot.x"),
    ("__slots__ 写", "obj_slot.x = 1"),
]:
    time = timeit.timeit(stmt, setup, number=10_000_000)
    results[label] = time
    print(f"{label}: {time:.4f}s")
```

典型运行结果（CPython 3.12，仅供参考）：

```
普通属性读: 0.18s
普通属性写: 0.20s
@property 读: 0.52s
@property 写: 0.60s
描述符读:   1.05s
描述符写:   1.15s
__slots__ 读: 0.15s
__slots__ 写: 0.16s
```

关键发现：
- **`__slots__`** 最快，因为直接通过 C 结构体偏移量访问，无字典查找。
- **普通属性** 第二快，需要一次 `__dict__` 哈希查找。
- **`@property`** 约为普通属性的 2.5-3 倍，因为每次访问涉及一次方法调用。
- **自定义描述符** 最慢，约为 `@property` 的 2 倍，因为 `__get__` 中使用了 `getattr`/`setattr`（间接调用描述符），产生了递归式属性查找。
- 优化描述符性能的方法：避免在 `__get__`/`__set__` 中使用 `getattr`/`setattr`，直接操作 `instance.__dict__`。

优化后的描述符：

```python
class FastDescriptor:
    def __set_name__(self, owner, name):
        self.private_name = f'_{name}'

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 直接操作 __dict__，避免递归属性查找
        try:
            return instance.__dict__[self.private_name]
        except KeyError:
            raise AttributeError(self.private_name)

    def __set__(self, instance, value):
        # 直接操作 __dict__，避免递归属性查找
        instance.__dict__[self.private_name] = value
```

这种优化可以将描述符的访问速度提升到接近 `@property` 的水平。

### 5.9 最佳实践总结

1. **始终在 `__get__` 中处理 `instance is None`**。通过类访问时（`MyClass.attr`），`instance` 为 `None`，此时应返回描述符自身。

2. **使用 `__set_name__` 自动获取属性名**（Python 3.6+）。避免手动传递属性名。

3. **将实例数据存储在实例的 `__dict__` 中**，而非描述符自身。如果实例使用 `__slots__`，使用 `WeakKeyDictionary`。

4. **明确区分数据描述符和非数据描述符**。如果你需要属性赋值不被实例属性覆盖，必须定义 `__set__`（即使只是抛出异常）。

5. **优先使用 `@property`**。对于简单的 getter/setter 逻辑，`@property` 比自定义描述符更简洁、更高效。描述符适用于需要在多个属性间复用逻辑的场景。

6. **在 `__set__` 和 `__delete__` 中直接操作 `instance.__dict__`**，而非使用 `setattr`/`getattr`，以避免递归属性查找和性能损失。

7. **描述符类应该只定义 `__get__`、`__set__`、`__delete__` 和 `__set_name__`**。不要添加其他公共方法，因为这会与描述符管理的属性混淆。

8. **考虑使用 `__init_subclass__` 或元类来集中注册描述符**，而不是在每个描述符的 `__set_name__` 中分散处理。

9. **为只读描述符同时定义 `__set__`（抛出 `AttributeError`）和 `__get__`**，确保它是数据描述符，防止被实例属性覆盖。

10. **文档化描述符的行为**。明确说明描述符是数据描述符还是非数据描述符，是否支持删除，是否使用缓存等。
