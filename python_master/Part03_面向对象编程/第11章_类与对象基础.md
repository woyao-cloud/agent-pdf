# 第11章 类与对象基础

Python 的面向对象编程（Object-Oriented Programming, OOP）是其核心范式之一。与 Java 或 C++ 不同，Python 的对象模型在运行时是高度动态的——类本身也是对象，属性可以在运行时增删，方法绑定发生在访问时刻而非定义时刻。理解这些机制背后的原理，是从"会用 Python"迈向"精通 Python"的关键一步。

本章将从 CPython 源码层面深入剖析类定义、实例化过程、属性查找链路、self 机制、静态方法与类方法，以及 `__slots__` 的内存优化原理。

---

## 11.1 概念与语法

### 11.1.1 类的定义

Python 使用 `class` 关键字定义类。类体在定义时立即执行，形成一个局部命名空间，随后该命名空间被传给元类（默认为 `type`）以构造类对象。

```python
class MyClass:
    """类的文档字符串"""
    class_var = 0  # 类属性

    def __init__(self, value):
        """实例初始化器"""
        self.instance_var = value  # 实例属性

    def method(self):
        """实例方法"""
        return self.instance_var
```

类定义的本质语法等价于：

```python
# class 语句的底层等价形式
def __init__(self, value):
    self.instance_var = value

def method(self):
    return self.instance_var

# 类体执行后形成的命名空间
namespace = {
    '__module__': '__main__',
    '__doc__': '类的文档字符串',
    'class_var': 0,
    '__init__': __init__,
    'method': method,
}

# 调用 type 元类构造类对象
MyClass = type('MyClass', (object,), namespace)
```

### 11.1.2 实例化语法

创建实例的标准语法是调用类对象：

```python
obj = MyClass(42)
```

这看似简单的调用背后，实际上触发了 `__new__` 和 `__init__` 的两阶段过程。

### 11.1.3 属性绑定语法

```python
# 实例属性绑定
obj.attr = value       # 绑定到实例的 __dict__

# 类属性绑定
MyClass.attr = value   # 绑定到类的 __dict__

# 动态方法绑定
def new_method(self):
    pass
MyClass.new_method = new_method  # 动态给类添加方法
```

### 11.1.4 静态方法与类方法

```python
class MyClass:
    @staticmethod
    def static_func():
        """不接收隐式第一参数"""
        return 'static'

    @classmethod
    def class_func(cls):
        """接收类作为第一参数"""
        return f'class: {cls.__name__}'
```

### 11.1.5 `__slots__` 语法

```python
class Point:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

## 11.2 原理与机制

### 11.2.1 类定义的底层机制

当 Python 解释器执行 `class MyClass:` 语句时，CPython 的处理流程如下：

1. **创建临时命名空间**：调用 `PyDict_New()` 创建一个新的字典对象，作为类体的局部命名空间。

2. **执行类体**：将类体编译为代码对象（`PyCodeObject`），然后以该临时命名空间为 locals 执行。所有赋值语句、函数定义、类属性定义的结果都存入该字典。

3. **确定元类**：按照 CPython 的元类确定算法：
   - 若显式指定 `metaclass=MC`，则使用 `MC`
   - 否则，取所有基类的元类。若无基类，使用 `type`
   - 若存在元类冲突（多个基类的元类不同），则选择最派生的元类

4. **调用元类构造类对象**：以 `metaclass(name, bases, namespace)` 的形式调用元类，返回类对象。默认元类 `type` 的调用对应 CPython 中的 `type_new()` 函数（位于 `Objects/typeobject.c`）。

`type_new()` 的关键步骤：

```c
// Objects/typeobject.c - type_new() 简化逻辑
static PyObject *
type_new(PyTypeObject *metatype, PyObject *args, PyObject *kwds)
{
    PyObject *name, *bases, *dict;
    // 解析参数: name, bases, dict
    PyArg_ParseTuple(args, "UO!O!",
                     &name, &PyTuple_Type, &bases,
                     &PyDict_Type, &dict);

    // 1. 调用 type.__new__ 分配 PyTypeObject 结构
    // 2. 调用 type.__init__ 初始化类型
    // 3. 处理 __slots__、__qualname__ 等
    // 4. 调用 PyType_Ready() 完成类型的就绪
}
```

`PyType_Ready()` 完成以下关键工作：
- 设置 `tp_base`（继承关系）
- 构建 MRO（Method Resolution Order，方法解析顺序）
- 将基类的描述符和属性复制到子类的 `tp_dict`
- 设置 `tp_mro` 指向 MRO 元组

### 11.2.2 实例化过程：`__new__` 与 `__init__`

实例化 `MyClass(42)` 的完整过程：

**第一阶段：调用 `type.__call__`**

当我们写 `MyClass(42)` 时，实际调用的是 `type.__call__`。在 CPython 中对应 `type_call()` 函数：

```c
// Objects/typeobject.c - type_call() 简化逻辑
static PyObject *
type_call(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    // 第一步：调用 type.__new__(type, args, kwds) 创建实例
    obj = type->tp_new(type, args, kwds);

    // 如果 __new__ 返回的对象类型不是 type 的实例，直接返回
    // 不调用 __init__
    if (obj != NULL && !PyType_IsSubtype(Py_TYPE(obj), type)) {
        return obj;
    }

    // 第二步：调用 type.__init__(obj, args, kwds) 初始化实例
    if (type->tp_init != NULL) {
        type->tp_init(obj, args, kwds);
    }

    return obj;
}
```

**第二阶段：`__new__` 创建实例**

`__new__` 是静态方法（尽管定义时不需要 `@staticmethod` 装饰器），负责分配内存并创建实例对象。默认实现 `object.__new__` 对应 CPython 中的 `object_new()`：

```c
// Objects/typeobject.c - object_new() 简化逻辑
static PyObject *
object_new(PyTypeObject *type, PyObject *args, PyObject *kwds)
{
    // 检查是否是可变类型且 __new__ 没有被重写
    // 如果是，且传入了多余参数，发出 DeprecationWarning

    // 调用 tp_alloc 分配内存
    return type->tp_alloc(type, 0);
}
```

`tp_alloc` 最终调用 `PyObject_Malloc()` 从 Python 的内存分配器分配内存：

```c
// Objects/typeobject.c - PyType_GenericAlloc()
PyObject *
PyType_GenericAlloc(PyTypeObject *type, Py_ssize_t nitems)
{
    PyObject *obj;
    size_t size;

    // 计算对象大小 = 基础大小 + 额外项大小
    size = type->tp_basicsize;

    // 分配内存
    obj = (PyObject *)PyObject_Malloc(size);

    // 初始化引用计数和类型指针
    Py_INCREF(type);
    Py_SET_TYPE(obj, type);
    Py_SET_REFCOUNT(obj, 1);

    // 如果类型使用 __dict__，为其创建字典
    if (type->tp_dictoffset != 0) {
        PyObject **dictptr = (PyObject **)((char *)obj + type->tp_dictoffset);
        *dictptr = NULL;  // 延迟创建
    }

    return obj;
}
```

**第三阶段：`__init__` 初始化实例**

`__init__` 在实例已创建后对其进行初始化。它是一个实例方法，第一个参数是 `__new__` 返回的实例对象。

**关键规则总结：**

| 特性 | `__new__` | `__init__` |
|------|-----------|------------|
| 调用时机 | 实例创建之前 | 实例创建之后 |
| 第一个参数 | 类（cls） | 实例（self） |
| 方法类型 | 静态方法（隐式） | 实例方法 |
| 返回值 | 必须返回创建的实例 | 必须返回 None（或不返回） |
| 能否控制创建 | 可以决定是否创建、返回不同对象 | 不能，实例已存在 |
| 不返回实例时 | 后续 `__init__` 不被调用 | N/A |

**`__new__` 返回值的影响：**

```python
class Strange:
    def __new__(cls, *args, **kwargs):
        # 返回一个已存在的对象——完全跳过 __init__
        return 42  # 返回一个 int 对象

    def __init__(self, *args, **kwargs):
        print("这行代码永远不会执行")

obj = Strange()
print(type(obj))  # <class 'int'>
print(obj)         # 42
```

当 `__new__` 返回的对象类型不是当前类的实例时，`type_call()` 不会调用 `__init__`。这可以用于实现单例模式：

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, value=None):
        # 注意：每次 Singleton() 调用都会执行 __init__
        # 即使返回的是同一个实例！
        if not hasattr(self, '_initialized'):
            self.value = value
            self._initialized = True
```

**`__init__` 的返回值限制：**

```python
class BadInit:
    def __init__(self):
        return 42  # TypeError: __init__ should return None

obj = BadInit()  # TypeError!
```

CPython 在 `type_call()` 中检查 `__init__` 的返回值：

```c
// Objects/typeobject.c - type_call() 中的检查
res = type->tp_init(obj, args, kwds);
if (res != NULL && res != Py_None) {
    PyErr_Format(PyExc_TypeError,
                 "__init__ should return None, not '%s'",
                 Py_TYPE(res)->tp_name);
    Py_DECREF(res);
    // 清理并返回错误
}
```

### 11.2.3 属性绑定与 `__dict__`

每个 Python 对象的核心数据结构中，最关键的字段之一是 `__dict__`。在 CPython 层面：

```c
// Include/cpython/object.h - PyObject 定义（简化）
typedef struct _object {
    Py_ssize_t ob_refcnt;      // 引用计数
    PyTypeObject *ob_type;      // 类型指针
} PyObject;

// 带有 __dict__ 的对象（简化）
typedef struct {
    PyObject_HEAD               // ob_refcnt + ob_type
    PyObject *ob_dict;          // 指向实例的 __dict__
    // ... 其他字段
} PyDictKeysObject;
```

当我们在实例上设置属性 `obj.x = 10` 时：

1. CPython 调用 `PyObject_SetAttr(obj, "x", value)`
2. 首先检查类中是否有数据描述符（定义了 `__set__` 的描述符），如果有则调用描述符的 `__set__`
3. 如果没有数据描述符，将属性存入 `obj.__dict__["x"] = 10`

实例属性存储在 `obj.__dict__` 中，类属性存储在 `type(obj).__dict__` 中。两者都是字典对象。

### 11.2.4 属性查找链路

当访问 `obj.attr` 时，CPython 执行以下查找过程（对应 `PyObject_GetAttr`）：

```
1. 检查 type(obj).__dict__ 中是否有名为 attr 的数据描述符
   (即定义了 __get__ 和 __set__ 的描述符)
   → 如果有，调用描述符的 __get__，返回结果

2. 检查 obj.__dict__ 中是否有 attr
   → 如果有，返回 obj.__dict__["attr"]

3. 检查 type(obj).__dict__ 中是否有名为 attr 的非数据描述符或普通属性
   → 按照MRO顺序在类层次结构中查找：
     type(obj).__dict__["attr"]
     type(obj).__bases__[0].__dict__["attr"]
     type(obj).__bases__[1].__dict__["attr"]
     ...
   → 如果找到的是描述符，调用其 __get__
   → 如果找到的是普通值，直接返回

4. 如果以上都没有找到，检查 type(obj) 是否定义了 __getattr__
   → 如果定义了，调用 __getattr__(obj, "attr")

5. 抛出 AttributeError
```

用代码验证查找链路：

```python
class DataDescriptor:
    """数据描述符：定义了 __get__ 和 __set__"""
    def __get__(self, obj, objtype=None):
        return 'data descriptor value'

    def __set__(self, obj, value):
        pass

class A:
    desc = DataDescriptor()  # 数据描述符

a = A()

# 数据描述符的优先级高于实例 __dict__
a.__dict__['desc'] = 'instance dict value'
print(a.desc)  # 'data descriptor value' ← 数据描述符胜出

# 但非数据描述符（只定义 __get__）优先级低于实例 __dict__
class NonDataDescriptor:
    def __get__(self, obj, objtype=None):
        return 'non-data descriptor value'

class B:
    desc = NonDataDescriptor()

b = B()
b.__dict__['desc'] = 'instance dict value'
print(b.desc)  # 'instance dict value' ← 实例属性胜出
```

完整的查找优先级（从高到低）：

```
数据描述符.__get__()  >  实例.__dict__  >  非数据描述符.__get__()  >  类.__dict__  >  __getattr__()
```

### 11.2.5 self 机制的底层原理

**self 不是关键字**

`self` 在 Python 中只是一个约定俗成的变量名，不是语言关键字。你可以用任何合法标识符替代它：

```python
class MyClass:
    def __init__(this_is_not_self, value):
        this_is_not_self.value = value

    def method(me):
        return me.value

obj = MyClass(42)
print(obj.method())  # 42
```

**隐式传递机制**

当我们调用 `obj.method(arg)` 时，Python 自动将 `obj` 作为第一个参数传入。这个过程发生在函数绑定时。在 CPython 中，实例方法的调用路径如下：

```
obj.method(arg)
    ↓
查找 obj.__dict__['method']     → 未找到
查找 type(obj).__dict__['method'] → 找到函数对象
    ↓
创建绑定方法对象（bound method）
    ↓
调用 bound_method(arg)
    ↓ 等价于
func(obj, arg)
```

CPython 中方法对象的数据结构：

```c
// Include/cpython/methodobject.h
typedef struct {
    PyObject_HEAD
    PyObject *im_func;    // 底层函数对象
    PyObject *im_self;    // 绑定的实例（self）
    PyObject *im_weakreflist;
    unsigned char im_vectorcall;  // vectorcall 优化标志
} PyMethodObject;
```

**绑定方法与非绑定方法**

在 Python 2 中，通过类直接调用方法会得到"非绑定方法"（unbound method），它要求第一个参数显式传入实例。Python 3 移除了非绑定方法的概念：

```python
class MyClass:
    def method(self):
        return self

obj = MyClass()

# 通过实例调用 —— 自动绑定 self
bound = obj.method
print(bound)         # <bound method MyClass.method of <MyClass object>>
print(bound())       # 等价于 MyClass.method(obj)
print(bound.__self__ is obj)  # True
print(bound.__func__ is MyClass.__dict__['method'])  # True

# 通过类调用 —— 返回普通函数
unbound = MyClass.method
print(unbound)       # <function MyClass.method at 0x...>
print(unbound(obj))  # 显式传入 self

# 直接检查类型
print(type(bound))   # <class 'method'>
print(type(unbound)) # <class 'function'>
```

当通过实例访问方法时，CPython 的描述符协议将函数对象（`function`）转换为绑定方法（`method`）。函数对象实现了描述符协议的 `__get__` 方法：

```python
# 函数的 __get__ 方法（简化逻辑）
def __get__(self, obj, objtype=None):
    if obj is None:
        return self  # 通过类访问 → 返回原函数
    return MethodType(self, obj)  # 通过实例访问 → 返回绑定方法
```

**vectorcall 优化**

从 Python 3.9 开始（3.12 默认启用），CPython 对方法调用引入了 vectorcall 协议。当解释器检测到调用的是绑定方法时，可以直接将 `self` 和参数打包成一个数组传递给底层函数，避免了创建中间的 `PyMethodObject`：

```c
// Python/methoddescriptor.c - vectorcall 路径（简化）
static PyObject *
method_vectorcall(PyObject *method, PyObject *const *args,
                  size_t nargsf, PyObject *kwnames)
{
    // 直接提取 im_func 和 im_self
    // 将 im_self 插入参数列表头部
    // 调用 im_func 的 vectorcall
}
```

这意味着在热路径上，`obj.method(arg)` 可能完全不需要创建 method 对象，直接以 `func(obj, arg)` 的形式调用。

### 11.2.6 类属性与实例属性的查找链

属性查找的完整链路按照 MRO（C3 线性化算法）遍历整个类层次结构：

```python
class A:
    x = 'A'

class B(A):
    x = 'B'

class C(A):
    x = 'C'

class D(B, C):
    pass

d = D()
print(d.x)  # 'B'

# MRO: D -> B -> C -> A -> object
print(D.__mro__)
# (<class '__main__.D'>, <class '__main__.B'>,
#  <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

# 查找 d.x:
# 1. d.__dict__ 中查找 'x' → 未找到（d 没有 __dict__ 中的 x）
# 2. D.__dict__ 中查找 'x' → 未找到
# 3. B.__dict__ 中查找 'x' → 找到！返回 'B'
```

C3 线性化算法保证了：
- 子类优先于父类
- 多个父类按声明顺序排列
- 单调性：如果某类在 A 的 MRO 中先于 B，则在所有子类的 MRO 中也先于 B

**`__dict__` 查找链的精确路径：**

```
实例.__dict__
  → 类.__dict__
    → 父类1.__dict__
      → 父类1的父类.__dict__
    → 父类2.__dict__
      → ...
  → object.__dict__
```

每一步查找都是对字典的 `PyDict_GetItem` 操作，时间复杂度 O(1)。但如果有 N 层继承，最坏情况需要查找 N 个字典，总体为 O(N)。

### 11.2.7 静态方法与类方法的底层实现

**`@staticmethod`**

`staticmethod` 是一个描述符类。在 CPython 中定义在 `Objects/func_objects.c`：

```c
// Objects/func_objects.c - staticmethod 结构
typedef struct {
    PyObject_HEAD
    PyObject *sm_callable;  // 被包装的函数
} staticmethod;
```

`staticmethod` 实现了描述符协议的 `__get__` 方法，但它的行为是"什么都不做"——无论通过实例还是类访问，都返回原始函数：

```python
# staticmethod.__get__ 的逻辑（简化）
def __get__(self, obj, objtype=None):
    return self.sm_callable  # 直接返回原始函数
```

这意味着：

```python
class MyClass:
    @staticmethod
    def func():
        return 'hello'

obj = MyClass()

# 两种访问方式都返回同一个函数
print(MyClass.func)   # <function MyClass.func at 0x...>
print(obj.func)       # <function MyClass.func at 0x...>
print(MyClass.func is obj.func)  # True
```

**`@classmethod`**

`classmethod` 也是描述符，但它的 `__get__` 会创建一个绑定方法，将类作为第一个参数：

```c
// Objects/func_objects.c - classmethod 结构
typedef struct {
    PyObject_HEAD
    PyObject *cm_callable;  // 被包装的函数
} classmethod;
```

`classmethod.__get__` 的逻辑：

```python
# classmethod.__get__ 的逻辑（简化）
def __get__(self, obj, objtype=None):
    if objtype is None:
        objtype = type(obj)
    return MethodType(self.cm_callable, objtype)  # 绑定类而非实例
```

关键行为差异：

```python
class Base:
    @classmethod
    def create(cls, value):
        return cls(value)

class Derived(Base):
    pass

# 通过实例或类调用，cls 始终是 Derived
d = Derived(1)
print(type(d.create(2)))  # <class '__main__.Derived'>, 而非 Base
print(type(Derived.create(2)))  # <class '__main__.Derived'>
```

当通过 `obj.create(2)` 调用时，`__get__` 接收到的 `objtype` 是 `type(obj)` 即 `Derived`，因此 `cls` 被绑定为 `Derived` 而非 `Base`。这是类方法实现工厂模式的核心机制。

**描述符协议对比：**

| 类型 | `__get__` 行为 | `__set__` | 优先级 |
|------|---------------|-----------|--------|
| `function` | 通过实例→绑定方法；通过类→原函数 | 无 | 非数据描述符 |
| `staticmethod` | 直接返回原函数 | 无 | 非数据描述符 |
| `classmethod` | 返回绑定类的方法 | 无 | 非数据描述符 |
| `property` | 调用 getter | 调用 setter | 数据描述符 |

因为 `staticmethod`、`classmethod` 和普通函数都是非数据描述符（没有 `__set__`），它们的优先级低于实例 `__dict__`：

```python
class MyClass:
    @staticmethod
    def func():
        return 'static'

obj = MyClass()
obj.func = 'overwritten'
print(obj.func)      # 'overwritten' — 实例属性覆盖了静态方法
print(MyClass.func)  # <function MyClass.func> — 类层面的访问不受影响
del obj.func
print(obj.func)      # 回到静态方法: 'static'
```

### 11.2.8 `__slots__` 的原理与内存优化

**基本原理**

默认情况下，每个 Python 实例都有一个 `__dict__` 字典来存储属性。Python 字典基于哈希表实现，内存开销较大：

```python
import sys

class Normal:
    def __init__(self, x, y):
        self.x = x
        self.y = y

n = Normal(1, 2)
print(sys.getsizeof(n))         # 实例本身：48 字节（64位系统）
print(sys.getsizeof(n.__dict__)) # 实例的字典：184 字节！
# 总计约 232 字节，仅存储两个整数
```

`__slots__` 通过在类定义中声明固定属性集合来优化：

```python
class Slotted:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y

s = Slotted(1, 2)
print(sys.getsizeof(s))  # 48 字节（64位系统）
# 没有 __dict__，属性直接存储在实例结构体中
```

**CPython 底层实现**

当类定义了 `__slots__`，CPython 在创建类型对象时执行以下操作：

1. **计算偏移量**：为每个 slot 计算在实例内存布局中的偏移量
2. **创建描述符**：为每个 slot 创建 `PyMemberDef` 描述符，存储在类的 `tp_dict` 中
3. **禁用 `__dict__`**：不设置 `tp_dictoffset`，实例不再分配字典空间

CPython 中 slot 描述符的定义：

```c
// Objects/typeobject.c - slot 描述符
// __slots__ 中的每个属性生成一个 member 描述符
typedef struct {
    PyObject_HEAD
    PyMemberDef member;    // 包含 name, type, offset, flags, doc
} PyMemberDescrObject;
```

`PyMemberDef` 的关键字段：

```c
// Include/structmember.h
typedef struct {
    const char *name;    // 属性名
    int type;            // 类型代码 (T_INT, T_OBJECT, ...)
    Py_ssize_t offset;  // 在实例中的偏移量
    int flags;           // READONLY 等
    const char *doc;     // 文档字符串
} PyMemberDef;
```

当访问 `obj.x` 时，描述符直接通过偏移量从实例内存中读取值，无需字典查找：

```c
// 简化的 slot 属性访问
PyObject *
member_get(PyObject *obj, PyMemberDef *member)
{
    // 直接通过偏移量从实例内存读取
    char *addr = (char *)obj + member->offset;
    // 根据类型代码解析值
    switch (member->type) {
        case T_OBJECT:
            return *(PyObject **)addr;
        case T_INT:
            return PyLong_FromLong(*(int *)addr);
        // ...
    }
}
```

**内存节省效果**

```python
import sys

class Normal:
    def __init__(self, x, y, z):
        self.x = x
        self.y = y
        self.z = z

class Slotted:
    __slots__ = ('x', 'y', 'z')
    def __init__(self, x, y, z):
        self.x = x
        self.y = y
        self.z = z

n = Normal(1, 2, 3)
s = Slotted(1, 2, 3)

# 对比内存使用
normal_size = sys.getsizeof(n) + sys.getsizeof(n.__dict__)
slotted_size = sys.getsizeof(s)

print(f"Normal: {normal_size} bytes")
print(f"Slotted: {slotted_size} bytes")
print(f"节省: {normal_size - slotted_size} bytes")

# 大量实例的对比
normals = [Normal(1, 2, 3) for _ in range(100000)]
slotteds = [Slotted(1, 2, 3) for _ in range(100000)]

# 通过进程内存或 pymalloc 统计的方式估算总内存
# 每个 Normal 实例约 232 字节（48 + 184 字典）
# 每个 Slotted 实例约 48 字节
# 100,000 个实例: 23.2 MB vs 4.8 MB
```

**`__slots__` 的继承行为**

```python
class Base:
    __slots__ = ('x',)

class Derived(Base):
    __slots__ = ('y',)

d = Derived()
d.x = 1  # 来自 Base 的 slot
d.y = 2  # 来自 Derived 的 slot

# Derived 实例的内存布局包含 Base 和 Derived 的所有 slot
# 但不包含 __dict__（除非某个类显式添加了 '__dict__' 到 slots）
```

继承规则：
- 子类的 `__slots__` 只需声明新增的属性，继承自父类的 slot 自动可用
- 如果父类没有 `__slots__`，子类定义 `__slots__` 也无法禁用 `__dict__`（因为父类已经开启了字典）
- 只有在继承链中所有类都定义了 `__slots__` 时，实例才没有 `__dict__`

```python
class NoSlots:
    pass  # 没有 __slots__，实例有 __dict__

class WithSlots(NoSlots):
    __slots__ = ('x',)

obj = WithSlots()
obj.x = 1    # OK, 使用 slot
obj.y = 2    # OK! 存入 __dict__（继承自 NoSlots）
print(obj.__dict__)  # {'y': 2}
```

**`__slots__` 中的 `'__dict__'`**

可以在 `__slots__` 中显式包含 `'__dict__'`，这样实例同时拥有 slot 和字典：

```python
class Hybrid:
    __slots__ = ('x', '__dict__')

h = Hybrid()
h.x = 1       # 存入 slot
h.y = 2       # 存入 __dict__
print(h.__dict__)  # {'y': 2}
print(h.x)         # 1, 从 slot 读取
```

## 11.3 使用场景

### 11.3.1 类定义与实例化的使用场景

**场景一：自定义 `__new__`——不可变类型**

`__new__` 的典型使用场景是创建不可变类型（如 `int`、`str`、`tuple` 的子类），因为不可变类型的值必须在对象创建时确定，而 `__init__` 在创建之后执行，为时已晚：

```python
class PositiveInt(int):
    """正整数类型，值必须 > 0"""
    def __new__(cls, value):
        if value <= 0:
            raise ValueError(f"PositiveInt 要求值 > 0, 收到 {value}")
        return super().__new__(cls, value)

    # 不需要 __init__——int 是不可变类型，值在 __new__ 中已确定

n = PositiveInt(42)
print(n + 8)       # 50, 继承了 int 的所有运算
print(type(n))      # <class '__main__.PositiveInt'>
PositiveInt(-1)     # ValueError!
```

**场景二：单例模式**

利用 `__new__` 控制实例创建：

```python
class DatabaseConnection:
    _instance = None
    _initialized = False

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, host='localhost', port=5432):
        if not self._initialized:
            self.host = host
            self.port = port
            self._initialized = True

db1 = DatabaseConnection('server1', 5432)
db2 = DatabaseConnection('server2', 3306)  # __init__ 被调用但 _initialized 阻止了重复初始化
print(db1 is db2)  # True
print(db1.host)     # 'localhost' — 第二次 __init__ 被跳过
```

**场景三：工厂模式——`__new__` 返回不同类型**

```python
class Shape:

        if shape_type == 'circle':
            return Circle(*args, **kwargs)
        elif shape_type == 'rectangle':
            return Rectangle(*args, **kwargs)
        raise ValueError(f"Unknown shape: {shape_type}")

class Circle:
    def __init__(self, radius):
        self.radius = radius

class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

c = Shape('circle', 5)
r = Shape('rectangle', 3, 4)

print(type(r))  # <class '__main__.Rectangle'>
```

### 11.3.3 `@classmethod` 的使用场景

**场景一：替代构造器（工厂方法）**

```python
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    @classmethod
    def from_string(cls, date_str):
        """从字符串 'YYYY-MM-DD' 创建日期"""
        year, month, day = map(int, date_str.split('-'))
        return cls(year, month, day)

    @classmethod
    def today(cls):
        """创建今天的日期"""
        import datetime
        t = datetime.date.today()
        return cls(t.year, t.month, t.day)

    def __repr__(self):
        return f'Date({self.year}, {self.month}, {self.day})'

d = Date.from_string('2024-03-15')
print(d)  # Date(2024, 3, 15)

# 继承时 cls 自动指向子类
class DateTime(Date):
    def __init__(self, year, month, day, hour=0, minute=0):
        super().__init__(year, month, day)
        self.hour = hour
        self.minute = minute

dt = DateTime.from_string('2024-03-15')
print(type(dt))  # <class '__main__.DateTime'> — 返回子类实例
```

**场景二：注册模式**

```python
class Plugin:
    _registry = {}

    @classmethod
    def register(cls, name):
        """装饰器：注册插件子类"""
        def decorator(subclass):
            cls._registry[name] = subclass
            return subclass
        return decorator

    @classmethod
    def create(cls, name, *args, **kwargs):
        """根据名称创建插件实例"""
        if name not in cls._registry:
            raise ValueError(f"Unknown plugin: {name}")
        return cls._registry[name](*args, **kwargs)

@Plugin.register('encoder')
class Encoder(Plugin):
    def __init__(self, charset='utf-8'):
        self.charset = charset

@Plugin.register('decoder')
class Decoder(Plugin):
    def __init__(self, charset='utf-8'):
        self.charset = charset

enc = Plugin.create('encoder', 'ascii')
dec = Plugin.create('decoder')
print(type(enc))  # <class '__main__.Encoder'>
```

### 11.4.2 属性查找链路完整演示

```python
"""
演示属性查找的完整优先级链路
"""

# ---- 数据描述符 vs 实例属性 ----
print("=== 1. 数据描述符优先于实例属性 ===")

class DataDescriptor:
    def __set_name__(self, owner, name):
        self.name = name
        self.private_name = f'_desc_{name}'

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return getattr(obj, self.private_name, f'<{self.name} from descriptor>')

    def __set__(self, obj, value):
        setattr(obj, self.private_name, value)

class A:
    desc = DataDescriptor()  # 数据描述符

a = A()
# 即使在实例 __dict__ 中写入同名属性
a.__dict__['desc'] = 'from instance __dict__'
print(f"a.desc = {a.desc}")  # 数据描述符胜出
print(f"a.__dict__ = {a.__dict__}")  # 实例 __dict__ 中的值被忽略

# ---- 非数据描述符 vs 实例属性 ----
print("\n=== 2. 实例属性优先于非数据描述符 ===")

class NonDataDescriptor:
    def __get__(self, obj, objtype=None):
        return '<from non-data descriptor>'

class B:
    desc = NonDataDescriptor()

b = B()
print(f"b.desc = {b.desc}")  # 非数据描述符

b.__dict__['desc'] = 'from instance __dict__'
print(f"b.desc = {b.desc}")  # 实例属性胜出

del b.__dict__['desc']
print(f"b.desc = {b.desc}")  # 回到非数据描述符

# ---- __getattr__ 兜底 ----
print("\n=== 3. __getattr__ 作为兜底 ===")

class C:
    def __getattr__(self, name):
        return f'<{name} via __getattr__>'

c = C()
print(f"c.nonexistent = {c.nonexistent}")  # 触发 __getattr__

# ---- 完整优先级总结 ----
print("\n=== 属性查找优先级（从高到低）===")
print("1. 数据描述符的 __get__")
print("2. 实例 __dict__")
print("3. 非数据描述符的 __get__ / 类 __dict__")
print("4. __getattr__")
print("5. AttributeError")
```

```python
    """测量创建 count 个实例的内存增量"""
    tracemalloc.start()
    snapshot1 = tracemalloc.take_snapshot()

    objects = [cls(*args) for _ in range(count)]

    snapshot2 = tracemalloc.take_snapshot()
    tracemalloc.stop()

    stats = snapshot2.compare_to(snapshot1, 'lineno')
    total = sum(stat.size_diff for stat in stats)
    return total, objects

count = 10000

normal_mem, _ = measure_memory(PointNormal, count, 1, 2)
slotted_mem, _ = measure_memory(PointSlotted, count, 1, 2)

print(f"{count} 个 Normal 实例: {normal_mem / 1024:.1f} KB")
print(f"{count} 个 Slotted 实例: {slotted_mem / 1024:.1f} KB")
print(f"节省: {(normal_mem - slotted_mem) / 1024:.1f} KB ({(1 - slotted_mem/normal_mem)*100:.1f}%)")
```

```python
"""
使用 @classmethod 和 @staticmethod 实现灵活的工厂模式

import json

class User:
    """用户类，演示多种构造方式"""

    def __init__(self, username, email, age):
        self.username = username
        self.email = email
        self.age = age

    @classmethod
    def from_dict(cls, data):
        """从字典创建用户"""
        return cls(
            username=data.get('username'),
            email=data.get('email'),
            age=data.get('age', 0)
        )

    @classmethod
    def from_json(cls, json_str):
        """从 JSON 字符串创建用户"""
        data = json.loads(json_str)
        return cls.from_dict(data)

    @classmethod
    def from_csv(cls, csv_line):
        """从 CSV 行创建用户"""
        parts = csv_line.strip().split(',')
        return cls(username=parts[0], email=parts[1], age=int(parts[2]))

    @staticmethod
    def validate_email(email):
        """验证邮箱格式——不依赖实例或类"""
        return '@' in email and '.' in email.split('@')[-1]

    @staticmethod
    def validate_age(age):
        """验证年龄——不依赖实例或类"""
        return isinstance(age, int) and 0 <= age <= 150

    def __repr__(self):
        return f"User(username={self.username!r}, email={self.email!r}, age={self.age})"
```

### 11.4.6 描述符协议深度实验

```python
"""
验证描述符协议在属性查找中的完整行为
"""

class Descriptor:
    """完整的数据描述符示例"""
    def __set_name__(self, owner, name):
        self.name = name
        self.storage_name = f'_descriptor_{name}'

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        value = getattr(obj, self.storage_name, None)
        print(f"  Descriptor.__get__('{self.name}', {type(obj).__name__}) -> {value}")
        return value

    def __set__(self, obj, value):
        print(f"  Descriptor.__set__('{self.name}', {type(obj).__name__}, {value})")
        setattr(obj, self.storage_name, value)

    def __delete__(self, obj):
        print(f"  Descriptor.__delete__('{self.name}', {type(obj).__name__})")
        delattr(obj, self.storage_name)


class ReadOnlyDescriptor:
    """只读数据描述符"""
    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return f'<readonly {self.name}>'

    def __set__(self, obj, value):
        raise AttributeError(f"'{self.name}' is read-only")


class MyClass:
    rw = Descriptor()       # 读写数据描述符
    ro = ReadOnlyDescriptor()  # 只读数据描述符

    def __init__(self):
        # 赋值触发 __set__
        self.rw = 'initial value'

obj = MyClass()

print("\n=== 读写描述符 ===")
print(f"obj.rw = {obj.rw}")
obj.rw = 'new value'
print(f"obj.rw = {obj.rw}")

print("\n=== 只读描述符 ===")
print(f"obj.ro = {obj.ro}")
try:
    obj.ro = 'try to write'
except AttributeError as e:
    print(f"AttributeError: {e}")

print("\n=== 描述符与实例 __dict__ 的交互 ===")
# 尝试在 __dict__ 中覆盖描述符
obj.__dict__['rw'] = 'hidden in __dict__'
print(f"obj.rw = {obj.rw}")  # 数据描述符仍然优先！
print(f"obj.__dict__['rw'] = {obj.__dict__['rw']}")  # 值在 __dict__ 中但被忽略

# 删除描述符
print("\n=== 删除描述符属性 ===")
del obj.rw  # 触发 __delete__
try:
    print(obj.rw)
except AttributeError:
    print("属性已删除")
```

## 11.5 常见陷阱与最佳实践

### 11.5.1 陷阱：类属性的可变默认值

```python
# ❌ 错误：类属性使用可变对象
class TeamBad:
    members = []  # 所有实例共享同一个列表！

    def add(self, name):
        self.members.append(name)

t1 = TeamBad()
t2 = TeamBad()
t1.add('Alice')
print(t2.members)  # ['Alice'] — t2 也看到了 Alice！

# ✅ 正确：在 __init__ 中初始化实例属性
class TeamGood:
    def __init__(self):
        self.members = []  # 每个实例有自己的列表

t1 = TeamGood()
t2 = TeamGood()
t1.add('Alice') if hasattr(t1, 'add') else t1.members.append('Alice')
print(t2.members)  # [] — t2 不受影响
```

**原因**：类属性 `members = []` 在类定义时创建，存储在 `TeamBad.__dict__['members']` 中。当调用 `self.members.append(name)` 时，属性查找先查实例 `__dict__`（未找到），再查类 `__dict__`（找到列表），对同一个列表对象执行 `append`。所有实例共享这个列表。

### 11.5.2 陷阱：`__new__` 与 `__init__` 的交互

```python
# ❌ 陷阱：__new__ 返回不同类型时 __init__ 不被调用
class Broken:
    def __new__(cls):
        return object()  # 返回 object() 而非 Broken 实例

    def __init__(self):
        self.data = 'initialized'  # 永远不会执行！

b = Broken()
print(type(b))  # <class 'object'>
# b.data  → AttributeError: 'object' 对象没有 'data'

# ✅ 正确：确保 __new__ 返回当前类的实例
class Fixed:
    def __new__(cls):
        instance = super().__new__(cls)  # 正确创建实例
        return instance

    def __init__(self):
        self.data = 'initialized'

f = Fixed()
print(f.data)  # 'initialized'
```

### 11.5.3 陷阱：单例模式中 `__init__` 的重复调用

```python
# ❌ 陷阱：单例模式中 __init__ 每次都被调用
class SingletonBad:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, value):
        self.value = value

s1 = SingletonBad(1)
s2 = SingletonBad(2)  # __init__ 再次执行！覆盖了 value
print(s1.value)  # 2 — 被第二次初始化覆盖了！
print(s1 is s2)  # True

# ✅ 正确：使用 _initialized 标志
class SingletonGood:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, value=None):
        if not hasattr(self, '_initialized') or not self._initialized:
            self.value = value
            self._initialized = True

s1 = SingletonGood(1)
s2 = SingletonGood(2)  # __init__ 被调用但 _initialized 阻止了重复初始化
print(s1.value)  # 1 — 未被覆盖
print(s1 is s2)  # True
```

### 11.5.4 陷阱：`__slots__` 的继承问题

```python
# ❌ 陷阱1：父类没有 __slots__，子类定义 __slots__ 无效

    pass  # 没有 __slots__，实例有 __dict__

    __slots__ = ('x',)

    __slots__ = ('y',)
    # 忘记包含 x — 但 x 的 slot 从 Base 继承，仍然可用

class DerivedRedundant(Base):
    __slots__ = ('x', 'y')  # 重复声明 x 是多余的但无害
    # x 的 slot 已从 Base 继承，这里只是多声明了一个同名描述符

# ✅ 正确：子类只需声明新增的属性

class FlexiblePoint:
    __slots__ = ('x', 'y', '__dict__')

        self.y = y

fp = FlexiblePoint(1, 2)
```

### 11.5.5 陷阱：描述符与实例属性的优先级混淆

```python
# ❌ 陷阱：以为实例属性总是优先于类属性
class Descriptor:
    def __get__(self, obj, objtype=None):
        return 'from descriptor'

        # 数据描述符的 __set__ 即使是空实现也会影响优先级
        pass

# 非数据描述符（没有 __set__）的情况
class NonDataDesc:
    def __get__(self, obj, objtype=None):
        return 'from non-data descriptor'

class B:
    desc = NonDataDesc()

b = B()
b.__dict__['desc'] = 'from instance dict'
```

### 11.5.6 陷阱：`@classmethod` 与 `@staticmethod` 的误用

```python
# ❌ 陷阱：在类方法中误用 self
```

### 11.5.7 陷阱：修改类属性影响所有实例

```python
# ❌ 陷阱：通过实例修改类属性
class Counter:
    count = 0

    def increment(self):
        self.count += 1  # 这行有问题！

c1 = Counter()
c2 = Counter()

c1.increment()

print(f"c2.count = {c2.count}")  # 0
```

### 11.5.8 陷阱：`__slots__` 与 `pickle`/`copy` 不兼容

```python
# ❌ 陷阱：__slots__ 对象默认不能 pickle
import pickle

class Point:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
try:
    data = pickle.dumps(p)  # 默认不能 pickle
except TypeError as e:
    print(f"pickle 失败: {e}")

# ✅ 正确：实现 __getstate__ 和 __setstate__
class PicklablePoint:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __getstate__(self):
        return {slot: getattr(self, slot) for slot in self.__slots__}

    def __setstate__(self, state):
        for slot, value in state.items():
            setattr(self, slot, value)

pp = PicklablePoint(1, 2)
data = pickle.dumps(pp)
restored = pickle.loads(data)
print(f"恢复后: x={restored.x}, y={restored.y}")
```

### 11.5.9 最佳实践总结

**实例化相关：**

1. 优先使用 `__init__` 进行初始化，只在需要控制实例创建时使用 `__new__`
2. `__new__` 返回值必须是当前类的实例（或其子类），否则 `__init__` 不会被调用
3. `__init__` 不应返回任何值（隐式返回 None）
4. 单例模式中使用 `_initialized` 标志防止重复初始化

**属性相关：**

5. 避免使用可变对象作为类属性默认值（列表、字典、集合等）
6. 理解属性查找优先级：数据描述符 > 实例 `__dict__` > 非数据描述符 > 类 `__dict__` > `__getattr__`
7. 修改类属性时始终使用 `ClassName.attr = value`，而非 `self.attr = value`

**方法相关：**

8. 工厂方法使用 `@classmethod`，确保继承时 `cls` 正确指向子类
9. 与类逻辑相关但不依赖实例或类状态的辅助函数使用 `@staticmethod`
10. `self` 只是约定，但不要使用其他名字——保持代码可读性

**`__slots__` 相关：**

11. 只在大量实例（>10,000）或内存敏感场景使用 `__slots__`
12. 确保继承链中所有类都定义了 `__slots__`，否则实例仍然有 `__dict__`
13. 子类的 `__slots__` 只需声明新增属性，不需要重复父类的 slot
14. 如需 pickle 支持，实现 `__getstate__` 和 `__setstate__`
15. 如需弱引用支持，在 `__slots__` 中加入 `'__weakref__'`
16. 如需动态属性支持，在 `__slots__` 中加入 `'__dict__'`（但这会部分抵消内存优化效果）

**性能考量：**

17. 属性查找链越长（继承层次越深），查找越慢；`__slots__` 可跳过 `__dict__` 查找
18. `@staticmethod` 和 `@classmethod` 的调用开销略高于普通函数调用，但差异可忽略
19. 方法查找涉及描述符协议，热路径代码可缓存方法引用：`method = obj.method` 然后循环调用 `method()`

