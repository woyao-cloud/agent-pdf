# 第04章 函数基础

函数是 Python 程序的基本构建单元。本章将从 CPython 源码层面深入剖析函数的定义、调用、参数体系、作用域规则以及函数对象的内部结构，帮助你建立对函数机制的完整认知。

---

## 4.1 概念与语法

### 4.1.1 函数的定义与调用

Python 中通过 `def` 语句定义函数：

```python
def add(a, b):
    """返回 a 与 b 的和。"""
    return a + b

result = add(3, 5)  # 调用函数，result 为 8
```

`def` 是一条可执行语句，而非编译期声明。当 Python 解释器执行到 `def` 语句时，它会：

1. 构造一个函数对象（`PyFunctionObject` 实例）
2. 将函数对象绑定到当前作用域中的函数名

这意味着 `def` 可以出现在任何可执行语句能出现的位置——包括 `if` 分支内、循环体内、甚至其他函数体内：

```python
def create_multiplier(factor):
    if factor == 0:
        def identity(x):
            return x
        return identity
    else:
        def multiply(x):
            return x * factor
        return multiply

double = create_multiplier(2)
print(double(5))  # 10
```

### 4.1.2 `def` 语句的字节码实现

使用 `dis` 模块可以观察 `def` 语句编译后的字节码：

```python
import dis

def demo():
    def inner():
        return 42
    return inner

dis.dis(demo)
```

输出（Python 3.12+）：

```
  2           RESUME                   0

  3           LOAD_CONST               1 (<code object inner>)
              MAKE_FUNCTION            0
              STORE_FAST               0 (inner)

  4           LOAD_FAST                0 (inner)
              RETURN_VALUE
```

关键字节码指令解析：

| 指令 | 作用 |
|------|------|
| `LOAD_CONST` | 将 `inner` 函数的 `PyCodeObject`（编译产物）压入操作数栈 |
| `MAKE_FUNCTION` | 从栈上的 `PyCodeObject` 构造 `PyFunctionObject`，并处理默认参数、注解等 |
| `STORE_FAST` | 将构造好的函数对象存入局部变量 `inner` |

`MAKE_FUNCTION` 的参数（flags）决定了它需要从栈上消费哪些额外数据：

- `0x01`：有默认参数（需从栈上加载 `__defaults__` 元组）
- `0x02`：有关键字默认参数（需加载 `__kwdefaults__` 字典）
- `0x04`：有注解（需加载 `__annotations__` 字典）
- `0x08`：有闭包自由变量（需加载闭包单元列表）

### 4.1.3 PyFunctionObject 的内部结构

在 CPython 源码（`Objects/funcobject.c` 和 `Include/cpython/funcobject.h`）中，`PyFunctionObject` 的定义如下：

```c
typedef struct {
    PyObject_HEAD                            // 引用计数 ob_refcnt, 类型指针 ob_type
    PyObject *func_globals;                  // 全局命名空间字典
    PyObject *func_locals;                   // 局部命名空间（通常为 NULL）
    PyObject *func_code;                     // PyCodeObject，编译后的字节码
    PyObject *func_defaults;                 // 位置默认参数元组，或 NULL
    PyObject *func_kwdefaults;               // 关键字默认参数字典，或 NULL
    PyObject *func_closure;                  // 闭包单元元组，或 NULL
    PyObject *func_doc;                      // __doc__，文档字符串
    PyObject *func_name;                     // __name__，函数名
    PyObject *func_dict;                     // __dict__，函数属性字典
    PyObject *func_weakreflist;              // 弱引用列表
    PyObject *func_module;                   // 所属模块名
    PyObject *func_annotations;              // 注解字典
    PyObject *func_typeparams;               // PEP 695 类型参数（3.12+）
    vectorcallfunc vectorcall;               // 向量化调用入口（3.12+）
} PyFunctionObject;
```

关键字段解读：

- **`func_code`**：核心字段，指向编译时生成的 `PyCodeObject`，包含字节码、常量表、变量名表等。函数的"逻辑"全部在此。
- **`func_globals`**：指向模块的全局命名空间字典。函数在被定义时捕获当前模块的 `globals()`，后续调用时通过此字典解析全局变量。
- **`func_defaults`** / **`func_kwdefaults`**：默认参数值在函数定义时求值并存储，这是可变默认参数陷阱的根源（详见 4.4 节）。
- **`func_closure`**：闭包捕获的自由变量存储为 cell 对象元组，每个 cell 对象包含对变量的间接引用。

你可以通过 Python 层面的属性直接访问这些字段：

```python
def example(a, b=10, *, c=20):
    return a + b + c

print(example.__name__)         # 'example'
print(example.__defaults__)     # (10,)
print(example.__kwdefaults__)   # {'c': 20}
print(example.__code__.co_varnames)  # ('a', 'b', 'c')
print(type(example.__globals__))     # <class 'dict'>
```

### 4.1.4 PyCodeObject 与函数的关系

函数对象是"代码 + 上下文"的组合。`PyCodeObject` 是纯粹的编译产物，不包含运行时信息；`PyFunctionObject` 则在 `PyCodeObject` 的基础上绑定了 `globals`、`defaults`、`closure` 等运行时上下文。同一个 `PyCodeObject` 可以被多个函数对象共享：

```python
def make_adder(n):
    def adder(x):
        return x + n
    return adder

add5 = make_adder(5)
add10 = make_adder(10)

print(add5.__code__ is add10.__code__)  # True！共享同一个 PyCodeObject
print(add5 is add10)                      # False，不同的函数对象
print(add5.__closure__)                   # (<cell at ...: int object 5>,)
print(add10.__closure__)                  # (<cell at ...: int object 10>,)
```

两个函数对象共享相同的代码逻辑（`__code__`），但各自的闭包单元（`__closure__`）捕获了不同的 `n` 值。

## 4.2 原理与机制

### 4.2.1 函数调用的完整过程

当 `result = add(3, 5)` 执行时，CPython 内部发生以下步骤：

1. **参数解析**：`add.__code__.co_argcount` 确定位置参数数量；参数值与形参名建立映射。
2. **帧对象创建**：`PyFrameObject`（Python 3.11+ 为 `struct _PyInterpreterFrame`）被创建，用于保存局部变量、操作数栈、返回地址等。
3. **局部变量初始化**：实参值被写入帧的局部变量槽位。
4. **字节码执行**：解释器在新帧中执行函数的字节码。
5. **返回值传递**：遇到 `RETURN_VALUE` 指令时，栈顶值作为返回值，帧被销毁（或缓存），控制权返回调用者。

用 `dis.dis(add)` 可以看到 `return a + b` 编译为：

```
  2           LOAD_FAST                0 (a)
              LOAD_FAST                1 (b)
              BINARY_ADD
              RETURN_VALUE
```

`LOAD_FAST` 从帧的局部变量数组中按索引直接取值，这是 Python 中最快的变量访问方式（比 `LOAD_GLOBAL` 快得多，也比 `LOAD_DEREF` 快）。

### 4.2.2 调用栈帧的生命周期

Python 3.11 对帧结构进行了重大重构（PEP 659 配合的内部优化）。旧版 `PyFrameObject` 是堆分配的重量级对象；新版本使用紧凑的 `_PyInterpreterFrame` 结构，并在帧栈上连续分配，显著降低了函数调用的内存开销。

```python
import sys

def outer():
    frame = sys._getframe(0)  # 获取当前帧
    print(f"函数名: {frame.f_code.co_name}")
    print(f"局部变量: {frame.f_locals}")
    print(f"调用者: {frame.f_back.f_code.co_name if frame.f_back else 'None'}")

def caller():
    outer()

caller()
# 输出：
# 函数名: outer
# 局部变量: {'frame': <frame object at ...>}
# 调用者: caller
```

帧对象的关键属性：

| 属性 | 含义 |
|------|------|
| `f_code` | 当前执行的 `PyCodeObject` |
| `f_locals` | 局部命名空间（字典） |
| `f_globals` | 全局命名空间（字典） |
| `f_builtins` | 内建命名空间（字典） |
| `f_back` | 调用者的帧（链向栈底） |
| `f_lineno` | 当前执行行号 |

## 4.3 使用场景

函数作为程序组织的基本单位，其使用场景包括但不限于：

- **代码复用**：将重复逻辑封装为函数，避免 copy-paste
- **逻辑分层**：通过函数调用栈实现计算的层次化分解
- **回调模式**：将函数作为参数传递给其他函数（事件处理、排序键等）
- **闭包与工厂**：利用嵌套函数和闭包创建定制化行为
- **装饰器基础**：函数作为一等公民是装饰器语法的前提

## 4.4 示例代码

### 4.4.1 参数体系完整讲解

Python 的参数体系是所有语言中最为灵活和强大的之一。本节完整覆盖所有参数类型。

#### 4.4.1.1 位置参数（Positional Arguments）

最基本的参数形式，调用时按位置顺序传递：

```python
def greet(name, message):
    print(f"{message}, {name}!")

greet("Alice", "Hello")  # 位置参数，按顺序匹配
```

#### 4.4.1.2 关键字参数（Keyword Arguments）

调用时通过形参名显式指定，可以打乱顺序：

```python
def greet(name, message):
    print(f"{message}, {name}!")

greet(message="Hi", name="Bob")  # 关键字参数，顺序无关
greet("Bob", message="Hi")        # 混合：位置参数必须在关键字参数前
```

#### 4.4.1.3 默认参数（Default Arguments）

为形参提供默认值，调用时可省略：

```python
def greet(name, message="Hello"):
    print(f"{message}, {name}!")

greet("Alice")              # 使用默认值 "Hello"
greet("Alice", "Hi")        # 覆盖默认值
greet("Alice", message="Hi")  # 也可以关键字形式覆盖
```

默认参数值在函数定义时求值一次，存储在函数对象的 `__defaults__` 属性中：

```python
def greet(name, message="Hello"):
    pass

print(greet.__defaults__)  # ('Hello',)
```

#### 4.4.1.4 *args：可变位置参数

使用 `*` 前缀收集多余的位置参数为元组：

```python
def variadic(*args):
    print(f"type: {type(args)}, value: {args}")

variadic(1, 2, 3)
# type: <class 'tuple'>, value: (1, 2, 3)

def mixed(required, *args):
    print(f"required: {required}, args: {args}")

mixed(10, 20, 30)
# required: 10, args: (20, 30)
```

`*args` 之后不能再有位置参数，但可以有关键字参数：

```python
def demo(*args, suffix):
    print(f"args: {args}, suffix: {suffix}")

demo(1, 2, 3, suffix="end")  # suffix 必须以关键字形式传递
# args: (1, 2, 3), suffix: end
```

#### 4.4.1.5 **kwargs：可变关键字参数

使用 `**` 前缀收集多余的关键字参数为字典：

```python
def keywords(**kwargs):
    print(f"type: {type(kwargs)}, value: {kwargs}")

keywords(a=1, b=2, c=3)
# type: <class 'dict'>, value: {'a': 1, 'b': 2, 'c': 3}
```

#### 4.4.1.6 参数解包

调用时可以使用 `*` 和 `**` 进行参数解包：

```python
def add(a, b, c):
    return a + b + c

nums = [1, 2, 3]
print(add(*nums))  # 等价于 add(1, 2, 3)

config = {'a': 10, 'b': 20, 'c': 30}
print(add(**config))  # 等价于 add(a=10, b=20, c=30)
```

#### 4.4.1.7 仅位置参数（Positional-Only Parameters）：`/` 分隔符

Python 3.8 引入的语法，`/` 左侧的参数只能按位置传递，不能以关键字形式传递：

```python
def power(base, exp, /):
    return base ** exp

print(power(2, 10))        # 合法
# print(power(base=2, exp=10))  # TypeError！不能以关键字传递
```

`/` 的语义本质：`/` 之前的参数名不属于函数的关键字命名空间，调用者无需知道参数名，实现者也可以自由修改参数名而不影响调用方。典型应用场景是那些参数顺序有自然语义的函数，例如 `int(x=255, base=16)` 读起来令人困惑，而 `int(255, 16)` 则直觉上更合理。

内置函数大量使用仅位置参数：

```python
# len() 的签名实际上是 len(obj, /)，你无法写 len(obj=xxx)
help(len)  # len(obj, /)
```

#### 4.4.1.8 仅关键字参数（Keyword-Only Parameters）：`*` 分隔符

`*` 之后的参数只能以关键字形式传递，不能按位置传递：

```python
def fetch(url, *, timeout=30, retries=3):
    print(f"Fetching {url} with timeout={timeout}, retries={retries}")

fetch("https://example.com")                      # 使用默认值
fetch("https://example.com", timeout=60)          # 部分覆盖
fetch("https://example.com", timeout=60, retries=5)  # 全部覆盖
# fetch("https://example.com", 60)  # TypeError！不能按位置传递
```

`*` 的语义本质：`*` 本身不消费任何参数，它只是一个分界标记，声明"从今往后，所有参数必须以关键字形式显式传递"。这增强了 API 的可读性和向前兼容性——新增的仅关键字参数不会破坏现有调用。

#### 4.4.1.9 完整参数顺序

一个函数可以组合所有参数类型，但必须遵循严格的顺序：

```python
def full_example(a, b, /, c, d=10, *args, e, f=20, **kwargs):
    """
    a, b     : 仅位置参数（/ 之前）
    c        : 可位置也可关键字的参数
    d=10     : 带默认值的位置/关键字参数
    *args    : 收集多余位置参数
    e        : 仅关键字参数（* 之后，无默认值，调用时必须提供）
    f=20     : 仅关键字参数（有默认值）
    **kwargs : 收集多余关键字参数
    """
    print(f"a={a}, b={b}, c={c}, d={d}, args={args}, e={e}, f={f}, kwargs={kwargs}")

# 调用示例：
# full_example(1, 2, 3)                         # a=1, b=2, c=3, d=10, args=(), e=? TypeError!
full_example(1, 2, 3, 4, 5, 6, e=7)           # a=1, b=2, c=3, d=4, args=(5,6), e=7, f=20
full_example(1, 2, c=3, e=7, f=30, x=99)      # a=1, b=2, c=3, d=10, e=7, f=30, kwargs={'x': 99}
```

参数声明的完整顺序规则：

```
仅位置参数 -> / -> 普通参数 -> *args -> 仅关键字参数 -> **kwargs
```

### 4.4.2 默认参数的陷阱

#### 4.4.2.1 可变默认参数的延迟绑定问题

这是 Python 最著名的陷阱之一：

```python
def append_item(item, container=[]):
    container.append(item)
    return container

print(append_item(1))  # [1]
print(append_item(2))  # [1, 2]  ← 不是 [2]！
print(append_item(3))  # [1, 2, 3]  ← 持续累积！
```

原因剖析：默认参数值在函数定义时（`def` 语句执行时）求值一次，而非每次调用时重新求值。`container=[]` 中的 `[]` 在定义时创建了一个列表对象，并被存入 `append_item.__defaults__`。此后每次调用都共享同一个列表对象。

```python
print(append_item.__defaults__)  # ([1, 2, 3],) — 同一个列表对象
```

通过 `id()` 可以确认：

```python
def buggy(lst=[]):
    print(id(lst))
    return lst

buggy()  # 某个内存地址，例如 140234567890880
buggy()  # 相同的内存地址
```

#### 4.4.2.2 修复方式

使用 `None` 作为哨兵值：

```python
def append_item(item, container=None):
    if container is None:
        container = []
    container.append(item)
    return container

print(append_item(1))  # [1]
print(append_item(2))  # [2]
print(append_item(3))  # [3]
```

#### 4.4.2.3 不可变默认参数是安全的

不可变类型（`int`、`str`、`tuple`、`frozenset`、`None`）作为默认值不会产生此问题，因为它们无法被原地修改：

```python
def safe(x=10, s="hello", t=(1, 2)):
    # 这些值在每次"重新使用"时仍然是原始值
    # 因为任何"修改"都会创建新对象
    pass
```

但需注意元组内包含可变对象的边界情况：

```python
def tricky(t=([1, 2], [3, 4])):
    # 元组本身不可变，但其内列表可被修改！
    t[0].append(999)
    return t

print(tricky())  # ([1, 2, 999], [3, 4])
print(tricky())  # ([1, 2, 999, 999], [3, 4])  ← 内部列表被修改了！
```

### 4.4.3 作用域与 LEGB 规则

#### 4.4.3.1 LEGB 规则概述

Python 变量名的查找遵循 LEGB 规则，依次搜索四个作用域：

1. **L**ocal —— 当前函数的局部命名空间
2. **E**nclosing —— 所有外层嵌套函数的局部命名空间（由内向外）
3. **G**lobal —— 模块级别的全局命名空间
4. **B**uiltin —— 内建命名空间（`builtins` 模块）

```python
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)      # 查找顺序: Local -> 命中 "local"
    inner()

outer()  # local
```

如果去掉 `inner` 中的赋值：

```python
x = "global"

def outer():
    x = "enclosing"
    def inner():
        print(x)      # Local 无 x -> Enclosing 命中 "enclosing"
    inner()

outer()  # enclosing
```

#### 4.4.3.2 作用域的实现——帧的 locals/globals/builtins

每个栈帧持有三个字典引用：

```python
import sys

def demo():
    local_var = 42
    frame = sys._getframe(0)
    print(type(frame.f_locals))   # <class 'dict'>
    print(type(frame.f_globals))  # <class 'dict'>
    print(type(frame.f_builtins)) # <class 'dict'>
    print('local_var' in frame.f_locals)   # True
    print('__name__' in frame.f_globals)   # True
    print('print' in frame.f_builtins)     # True

demo()
```

- **`f_locals`**：局部变量字典。在函数内部，局部变量通过 `LOAD_FAST` / `STORE_FAST` 按索引高速访问，`f_locals` 的访问实际上是延迟构建的（Python 3.11+ 的帧优化）。
- **`f_globals`**：即模块的 `globals()` 字典，与 `func_globals` 相同。
- **`f_builtins`**：指向 `builtins` 模块的 `__dict__`，包含 `print`、`len`、`int` 等内建名称。

#### 4.4.3.3 `global` 和 `nonlocal` 语句

`global` 声明变量来自全局作用域，`nonlocal` 声明变量来自最近的外层函数作用域：

```python
counter = 0

def increment():
    global counter
    counter += 1    # 修改全局变量
    return counter

print(increment())  # 1
print(increment())  # 2
print(counter)       # 2

def make_counter():
    count = 0
    def inc():
        nonlocal count
        count += 1  # 修改外层函数的局部变量
        return count
    return inc

c = make_counter()
print(c())  # 1
print(c())  # 2
```

不使用 `global` 或 `nonlocal` 时，函数内的赋值语句会创建一个新的局部变量，即使该变量在更外层作用域中已存在：

```python
x = 10

def shadow():
    x = 20      # 创建局部变量 x，遮蔽了全局 x
    print(x)    # 20

def error_example():
    # print(x)    # 先读取，此时 x 引用全局变量
    # x = 30    # 如果取消注释，上面的 print(x) 会报 UnboundLocalError
                # 因为 Python 编译器在编译时检测到赋值语句，
                # 将 x 标记为局部变量，但在 print 时还未赋值
    pass

shadow()
```

#### 4.4.3.4 字节码层面的变量访问

不同作用域的变量使用不同的字节码指令：

```python
import dis

x = 10

def scope_demo():
    y = 20              # 局部变量
    z = x               # 全局变量
    w = len             # 内建变量

dis.dis(scope_demo)
```

关键指令对比：

| 指令 | 作用 | 速度 |
|------|------|------|
| `LOAD_FAST` / `STORE_FAST` | 访问局部变量（数组索引） | 最快 |
| `LOAD_DEREF` / `STORE_DEREF` | 访问闭包变量（cell 对象间接引用） | 较快 |
| `LOAD_GLOBAL` | 访问全局变量（字典查找） | 较慢 |
| `LOAD_NAME` | 按 LEGB 顺序查找（模块顶层代码使用） | 最慢 |

性能差异的根本原因：`LOAD_FAST` 直接通过数组索引 `frame->localsplus[i]` 访问，时间复杂度 O(1)；`LOAD_GLOBAL` 需要在字典中查找键，时间复杂度平均 O(1) 但常数因子更大。

### 4.4.4 递归

#### 4.4.4.1 递归基础与深度限制

Python 默认递归深度上限为 1000：

```python
def recurse(n):
    if n <= 0:
        return "done"
    return recurse(n - 1)

print(recurse(999))   # 正常
# print(recurse(1000))  # RecursionError: maximum recursion depth exceeded
```

可以使用 `sys.setrecursionlimit()` 调整：

```python
import sys
print(sys.getrecursionlimit())  # 1000（默认）

sys.setrecursionlimit(5000)
print(recurse(4000))  # 现在可以了
```

**警告**：盲目增大递归限制是危险的。每个递归调用消耗一个栈帧，深度过大可能导致栈溢出（segmentation fault）。CPython 的栈空间受操作系统限制，通常在几千到几万层之间就会崩溃。

#### 4.4.4.2 尾递归优化缺失

某些语言（如 Scheme、Haskell）支持尾递归优化（Tail Call Optimization, TCO），将尾递归转换为等价的循环，从而消除栈增长。Python **不支持 TCO**，这是 Guido van Rossum 的明确设计决策，原因包括：

1. TCO 会破坏调试时的栈追踪信息
2. Python 的异常追踪机制依赖完整的调用栈
3. TCO 使得某些逻辑错误的诊断变得困难

```python
# 尾递归形式的阶乘——Python 中仍会栈溢出
def factorial(n, acc=1):
    if n <= 1:
        return acc
    return factorial(n - 1, acc * n)  # 尾递归调用

print(factorial(5))     # 120
# print(factorial(1500))  # RecursionError！即使形式是尾递归
```

#### 4.4.4.3 递归到迭代的转换策略

策略一：手动转换为循环

```python
# 递归版本
def sum_recursive(n):
    if n == 0:
        return 0
    return n + sum_recursive(n - 1)

# 迭代版本
def sum_iterative(n):
    total = 0
    for i in range(1, n + 1):
        total += i
    return total

print(sum_iterative(10000))  # 50005000，无递归限制问题
```

策略二：使用显式栈模拟递归

```python
# 递归版本的树遍历
def tree_sum_recursive(node):
    if node is None:
        return 0
    return node.value + tree_sum_recursive(node.left) + tree_sum_recursive(node.right)

# 迭代版本：使用显式栈
def tree_sum_iterative(root):
    if root is None:
        return 0
    total = 0
    stack = [root]
    while stack:
        node = stack.pop()
        total += node.value
        if node.left:
            stack.append(node.left)
        if node.right:
            stack.append(node.right)
    return total
```

策略三：使用 `functools.lru_cache` 优化重叠子问题（记忆化）

```python
from functools import lru_cache

# 无缓存：指数级时间复杂度
def fib_slow(n):
    if n < 2:
        return n
    return fib_slow(n - 1) + fib_slow(n - 2)

# 有缓存：线性时间复杂度
@lru_cache(maxsize=None)
def fib_cached(n):
    if n < 2:
        return n
    return fib_cached(n - 1) + fib_cached(n - 2)

import time

start = time.perf_counter()
fib_slow(35)
print(f"无缓存: {time.perf_counter() - start:.4f}s")

start = time.perf_counter()
fib_cached(1000)
print(f"有缓存: {time.perf_counter() - start:.6f}s")
```

### 4.4.5 函数注解（Function Annotations）

#### 4.4.5.1 语法与运行时角色

```python
def add(a: int, b: int) -> int:
```

### 4.4.6 函数是一等公民

Python 中函数是一等公民（first-class citizen），意味着函数可以：

#### 4.4.6.1 赋值给变量

```python
def shout(text):
```

```python
def apply(func, value):

```

经典应用：排序的自定义键函数

```python
students = [
    {"name": "Alice", "grade": 92},
    {"name": "Bob", "grade": 85},
    {"name": "Charlie", "grade": 98},

sorted_students = sorted(students, key=lambda s: s["grade"], reverse=True)
print([s["name"] for s in sorted_students])  # ['Charlie', 'Alice', 'Bob']
```

```python
def make_multiplier(factor):
    def multiplier(x):
        return x * factor
    return multiplier

times_3 = make_multiplier(3)
times_5 = make_multiplier(5)

print(times_3(10))  # 30
print(times_5(10))  # 50
```

#### 4.4.6.4 存储在数据结构中

```python
operations = {

def calculate(op, a, b):

print(calculate('+', 3, 4))   # 7
print(calculate('*', 3, 4))   # 12
```

#### 4.4.6.6 lambda 表达式及其局限

`lambda` 创建匿名函数，语法为 `lambda 参数: 表达式`：

```python
square = lambda x: x ** 2
print(square(5))  # 25

# 等价的 def 定义
def square_def(x):
    return x ** 2
```

lambda 的局限性：

1. **只能包含单个表达式**，不能包含语句（赋值、if-else 块、for 循环等）
2. **没有语句体**，无法包含 `try`/`except`、`with` 等
3. **可读性差**，复杂的 lambda 严重损害代码可维护性
4. **没有 `__name__`**，调试时显示为 `<lambda>`

```python
add = lambda a, b: a + b
print(add.__name__)  # '<lambda>'，不如 def 定义的函数名有信息量
```

Python 3.12 引入了类型参数语法（PEP 695），但 lambda 仍不支持类型注解：

```python
# 不允许：
# typed_lambda = lambda x: int -> x + 1  # SyntaxError

# 如果需要类型注解，必须使用 def
def typed_add(a: int, b: int) -> int:
    return a + b
```

**最佳实践**：当 lambda 超过一行时，改用 `def` 定义命名函数。PEP 8 建议"始终使用 def 而非赋值语句将 lambda 表达式绑定到标识符"。

### 4.4.7 函数对象的内部属性

每个函数对象都携带着丰富的元信息，以下通过完整示例逐一展示：

```python
def outer(x):
    y = 10
    def inner(a, b=5, *, c=15):
        """Inner function docstring."""
        return a + b + c + x + y
    return inner

func = outer(100)

# __name__：函数的名称字符串
print(f"__name__: {func.__name__}")  # 'inner'

# __doc__：文档字符串
print(f"__doc__: {func.__doc__}")    # 'Inner function docstring.'

# __code__：PyCodeObject，包含字节码和编译信息
code = func.__code__
print(f"co_name: {code.co_name}")           # 'inner'
print(f"co_argcount: {code.co_argcount}")   # 3 (a, b, c)
print(f"co_varnames: {code.co_varnames}")   # ('a', 'b', 'c')
print(f"co_freevars: {code.co_freevars}")   # ('x', 'y') — 闭包自由变量
print(f"co_nlocals: {code.co_nlocals}")     # 3

# __defaults__：位置默认参数值的元组
print(f"__defaults__: {func.__defaults__}")  # (5,) — b 的默认值

# __kwdefaults__：仅关键字默认参数值的字典
print(f"__kwdefaults__: {func.__kwdefaults__}")  # {'c': 15}

# __globals__：函数定义时的全局命名空间
print(f"__globals__ is module globals: {func.__globals__ is globals()}")  # True

# __closure__：闭包单元元组，每个 cell 对象包含一个自由变量的引用
print(f"__closure__: {func.__closure__}")  # (<cell ...>, <cell ...>)
print(f"closure[0].cell_contents: {func.__closure__[0].cell_contents}")  # 100 (x)
print(f"closure[1].cell_contents: {func.__closure__[1].cell_contents}")  # 10 (y)
```

```python
# __annotations__：参数和返回值的类型注解
def annotated(a: int, b: str = "hi") -> bool:
    pass

print(f"__annotations__: {annotated.__annotations__}")
# {'a': <class 'int'>, 'b': <class 'str'>, 'return': <class 'bool'>}

# __dict__：函数的自定义属性字典
annotated.custom_attr = "hello"
print(f"__dict__: {annotated.__dict__}")  # {'custom_attr': 'hello'}
```


#### 4.4.7.2 动态修改函数属性

理解内部结构后，你可以做一些"黑魔法"——例如动态修改默认参数：

```python
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))  # Hello, Alice

# 修改 __defaults__ 改变默认值
greet.__defaults__ = ("Hi",)
print(greet("Alice"))  # Hi, Alice

# 警告：这种操作极其危险，仅用于理解内部机制，生产代码中不要这样做！
```

甚至可以替换函数的 `__code__`：

```python
def original():
    return 1

def replacement():
    return 42

print(original())  # 1
original.__code__ = replacement.__code__
print(original())  # 42
```

**再次警告**：以上操作仅供学习，生产环境中使用会导致难以调试的诡秘行为。

## 4.5 常见陷阱与最佳实践

### 4.5.1 可变默认参数陷阱

**陷阱**：可变对象作为默认参数值在多次调用间共享。

```python
# 错误
def append_to(item, target=[]):
    target.append(item)
    return target

# 正确：使用 None 哨兵
def append_to(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target
```

**根因**：默认值在 `def` 执行时求值，存储在 `__defaults__` 元组中，每次调用共享同一对象。

**判别标准**：`None` vs `[]`/`{}`/`set()`。凡是可变类型，一律使用 `None` 哨兵模式。

### 4.5.2 闭包中的变量绑定延迟

**陷阱**：闭包捕获的是变量的引用，而非值的快照。

```python
funcs = []
for i in range(5):
    def func():
        return i
    funcs.append(func)

print([f() for f in funcs])  # [4, 4, 4, 4, 4]，而非 [0, 1, 2, 3, 4]
```

所有闭包引用同一个 `i`，循环结束时 `i` 为 4。

**修复方式一**：默认参数立即绑定当前值

```python
funcs = []
for i in range(5):
    def func(i=i):  # 默认参数在定义时求值
        return i
    funcs.append(func)

print([f() for f in funcs])  # [0, 1, 2, 3, 4]
```

**修复方式二**：工厂函数创建独立作用域

```python
def make_func(i):
    def func():
        return i
    return func

funcs = [make_func(i) for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]
```

### 4.5.3 UnboundLocalError 与变量遮蔽

**陷阱**：函数内部存在对变量的赋值语句时，Python 编译器会将该变量标记为局部变量，即使赋值发生在使用之后。

```python
x = 10

def wrong():
    # print(x)   # UnboundLocalError: local variable 'x' referenced before assignment
    # x = 20     # 这行导致 x 被标记为局部变量
    pass

# 修复：使用 global 声明
def correct():
    global x
    print(x)   # 10
    x = 20
    print(x)   # 20

correct()
```

### 4.5.4 修改全局可变对象不需要 global

**陷阱/误解**：修改全局可变对象（列表、字典等）不需要 `global` 声明，但重新赋值需要。

```python
items = [1, 2, 3]

def modify_list():
    items.append(4)    # 合法！修改对象本身，不需要 global
    # items = [4]      # 如果这行取消注释，需要 global 声明

modify_list()
print(items)  # [1, 2, 3, 4]
```

`global` 声明的是"我要重新绑定这个名字"，而非"我要修改这个名字引用的对象"。

### 4.5.5 参数顺序的严格规则

**最佳实践**：遵循参数声明的标准顺序，避免 API 使用者困惑。

```python
# 正确的参数顺序
def func(pos_only, /, pos_or_kw, kw_only, *, kw_only_2, **kwargs):
    pass

# 错误：默认参数在非默认参数之前
# def bad(a=1, b):  # SyntaxError: non-default argument follows default argument
#     pass
```

在设计 API 时：
- 使用 `/` 分隔符使参数成为仅位置参数，保护参数名不被外部依赖
- 使用 `*` 分隔符使参数成为仅关键字参数，提高调用可读性
- 可变参数 `*args` 和 `**kwargs` 放在最后

### 4.5.6 递归深度与栈溢出

**陷阱**：超过递归深度限制时抛出 `RecursionError`，但 `setrecursionlimit` 设置过大会导致段错误。

**最佳实践**：
1. 优先使用迭代替代递归
2. 对递归深度有合理预估，仅在确定深度有限时使用递归
3. 对于树/图遍历，使用显式栈替代递归
4. 对于数值计算，使用数学公式或迭代替代

### 4.5.7 `__defaults__` 是元组而非列表

**细节**：`__defaults__` 是元组，不能直接修改其中的可变对象引用，但可以修改可变对象的内容：

```python
def example(lst=[1, 2, 3]):
    pass

# __defaults__ 是元组
print(type(example.__defaults__))  # <class 'tuple'>

# 不能替换元组中的元素（元组不可变）
# example.__defaults__[0] = [4, 5, 6]  # TypeError

# 但可以修改元组中可变对象的内容（正是默认参数陷阱的根因）
example.__defaults__[0].append(4)
print(example.__defaults__)  # ([1, 2, 3, 4],)
```

### 4.5.8 函数注解不是类型强制

**陷阱**：注解不会在运行时进行类型检查或转换。

```python
def add(a: int, b: int) -> int:
    return a + b

result = add("3", "4")  # 不会报错，返回 "34"（字符串拼接）
print(result)  # "34"
```

**最佳实践**：
1. 使用 mypy/pyright 进行静态类型检查
2. 在 CI 流程中集成类型检查步骤
3. 需要运行时类型验证时，使用 `pydantic`、`typeguard` 等库
4. 不要试图用注解实现运行时类型强制转换

### 4.5.9 lambda 的可读性边界

**最佳实践**：
- 简单的单行表达式适合 lambda：`sorted(items, key=lambda x: x.name)`
- 超过一个表达式时改用 `def`
- 永远不要将 lambda 赋值给命名变量（PEP 8 明确反对）

```python
# 反模式
f = lambda x: x * 2  # PEP 8 不推荐

# 推荐
def f(x):
    return x * 2
```

### 4.5.10 性能对比：局部变量 vs 全局变量 vs 闭包变量

```python
import timeit

# 局部变量访问
setup_local = """
def test():
    x = 1
    for _ in range(1000):
        x += 1
"""

# 全局变量访问
setup_global = """
x = 1
def test():
    global x
    for _ in range(1000):
        x += 1
"""

# 闭包变量访问
setup_closure = """
def outer():
    x = 1
    def inner():
        nonlocal x
        for _ in range(1000):
            x += 1
    return inner
test = outer()
"""

local_time = timeit.timeit("test()", setup=setup_local, number=10000)
global_time = timeit.timeit("test()", setup=setup_global, number=10000)
closure_time = timeit.timeit("test()", setup=setup_closure, number=10000)

print(f"局部变量: {local_time:.4f}s")
print(f"全局变量: {global_time:.4f}s")
print(f"闭包变量: {closure_time:.4f}s")
```

典型输出：

```
局部变量: 0.85s
全局变量: 1.52s
闭包变量: 1.15s
```

局部变量访问最快（`LOAD_FAST`），闭包变量次之（`LOAD_DEREF` 需要间接寻址），全局变量最慢（`LOAD_GLOBAL` 需要字典查找）。性能敏感的循环内应优先使用局部变量。
