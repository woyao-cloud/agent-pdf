## 1. 概念与语法

### 1.1 变量与绑定

Python 中的变量与传统语言（如 C/Java）有根本性差异。在 C 中，变量是内存"盒子"——声明 `int a = 1` 会分配一块 sizeof(int) 的内存，将值 1 放入其中。而在 Python 中，**变量不是盒子，而是标签（name tag）**，绑定到对象上。

```python
# Python 的变量是标签，不是盒子
a = [1, 2, 3]
b = a           # b 和 a 指向同一个 list 对象
b.append(4)
print(a)        # [1, 2, 3, 4] — a 也变了，因为 a、b 是同一对象的两个标签
```

核心语法规则：

- **赋值语句** `x = value` 创建或重新绑定名称 `x` 到右侧求值结果对象
- **多重赋值** `a = b = 0` 将同一对象绑定到多个名称
- **解包赋值** `a, b = 1, 2` 将可迭代对象的元素分别绑定
- **增量赋值** `a += 1` 对可变对象可能原地修改，对不可变对象等价于 `a = a + 1`（创建新对象）
- **删除绑定** `del x` 解除名称与对象的绑定

名称绑定发生在当前作用域的最内层命名空间（local），除非使用 `global` 或 `nonlocal` 声明。

### 1.2 数据类型概览

Python 内置类型构成了一个完整的层次结构。所有类型的根基是 `object`：

```
object
├── NoneType            # None，单例
├── bool                # True / False，int 的子类
├── int                 # 任意精度整数
├── float               # IEEE 754 双精度浮点
├── complex             # 复数 (real + imag*1j)
├── str                 # Unicode 字符串
├── tuple               # 不可变序列
├── list                # 可变序列
├── dict                # 键值映射（3.7+ 有序）
├── set                 # 可变无序集合
├── frozenset           # 不可变集合
├── bytes               # 不可变字节序列
├── bytearray           # 可变字节序列
├── range               # 不可变数字序列
├── memoryview          # 内存视图
├── ...                 # 其他内置类型
```

类型分类维度：

| 维度 | 不可变 (Immutable) | 可变 (Mutable) |
|------|---------------------|----------------|
| 序列 | `str`, `tuple`, `bytes`, `range` | `list`, `bytearray` |
| 映射 | — | `dict` |
| 集合 | `frozenset` | `set` |
| 标量 | `int`, `float`, `complex`, `bool`, `NoneType` | — |

**`bool` 是 `int` 的子类**：`True == 1`，`False == 0`，但 `True is 1` 为 `False`。这意味着 `bool` 可以参与所有整数运算。

**`None` 是单例**：整个 Python 进程中只有一个 `None` 对象，用 `x is None` 判断而非 `x == None`。

### 1.4 表达式求值顺序

Python 对表达式求值顺序有严格定义：

1. **从左到右原则**：在每一优先级内，操作数从左到右求值
2. **短路求值**：`and` 和 `or` 在能确定结果时立即停止求值
3. **函数调用**：参数从左到右求值
4. **赋值语句右侧**先求值，再绑定到左侧

```python
# 短路求值
x = 0
result = x != 0 and 10 / x   # 不会触发 ZeroDivisionError，因为 x != 0 为 False

# 从左到右
def side_effect(name):
    print(name, end=" ")
    return name

# a() + b() * c()  → 先 a()，再 b()，再 c()，再乘，再加
side_effect("a") + side_effect("b") * side_effect("c")
# 输出：a b c（操作数求值顺序严格左到右）
```

### 1.6 注释与文档字符串

**注释**（comment）以 `#` 开头，持续到行尾。Python 没有多行注释语法，连续 `#` 行是惯例。

**文档字符串**（docstring）是模块、函数、类、方法体内第一个字符串字面量。它是**运行时对象**，存储在 `__doc__` 属性中，可被 `help()` 函数和文档工具读取。

```python
def factorial(n):
    """计算 n 的阶乘。

    Args:
        n: 非负整数

    Returns:
        n! 的值

    Raises:
        ValueError: 当 n < 0 时
    """
    if n < 0:
        raise ValueError("n must be non-negative")
    return 1 if n <= 1 else n * factorial(n - 1)

# 运行时访问
print(factorial.__doc__)   # 输出文档字符串
help(factorial)            # 格式化显示帮助
```

Docstring 约定（PEP 257）：
- 单行 docstring：一行，首字母大写，句号结尾
- 多行 docstring：首行摘要，空行，详细描述
- 类的 docstring 描述行为，不是实现细节

## 2. 原理与机制

### 2.1 变量绑定的 CPython 实现原理

在 CPython 中，"变量"实际上由三个层次组成：

**层次一：PyObject — 一切对象的根基**

CPython 中每个对象都是 `PyObject` 结构体的实例（或扩展）：

```c
// Include/object.h — CPython 3.12 简化
typedef struct _object {
    Py_ssize_t ob_refcnt;    // 引用计数
    PyTypeObject *ob_type;   // 类型指针
} PyObject;
```

每个对象（整数 42、字符串 "hello"、列表 [1,2]）在堆上分配，头部包含引用计数和类型指针。变量名不持有数据，而是持有指向 `PyObject` 的指针。

**层次二：名字空间 — 字典实现**

Python 的局部变量存储在**帧对象 (frame object)** 的 `f_locals` 中，本质是一个字典（或优化后的数组）。当执行 `x = 42` 时：

1. CPython 先在堆上查找或创建整数 42 对应的 `PyLongObject`
2. 在当前帧的局部名字空间中插入/更新键 `"x"` → 值为指向该 `PyLongObject` 的 `PyObject*`

```c
// Python/ceval.c 中 STORE_NAME 的核心逻辑（简化）
case TARGET(STORE_NAME): {
    PyObject *name = GETITEM(names, oparg);  // 变量名 "x"
    PyObject *v = POP();                       // 栈顶值 (PyObject* to 42)
    int err = PyDict_SetItem(f->f_locals, name, v);  // 名字空间["x"] = v
    Py_DECREF(v);
    break;
}
```

**层次三：引用计数与对象生命周期**

绑定增加引用计数，解绑减少引用计数。当 `ob_refcnt` 降为 0，对象被回收：

```c
// Macros in Include/object.h
#define Py_INCREF(op) ((op)->ob_refcnt++)
#define Py_DECREF(op) \
    if (--((op)->ob_refcnt) == 0) \
        _Py_Dealloc(op)
```

```python
# 验证引用计数
import sys

a = [1, 2, 3]       # refcnt = 1
b = a                # refcnt = 2 (a 和 b 两个名字)
print(sys.getrefcount(a))  # 3 (a, b, 以及 getrefcount 的参数)
del b                # refcnt = 1
print(sys.getrefcount(a))  # 2 (a, 以及 getrefcount 的参数)
```

> 注意：`sys.getrefcount()` 返回值比预期多 1，因为函数调用本身临时增加了一次引用。

**变量不是"盒子"的完整证明**：

```python
a = [1, 2, 3]
b = a
print(id(a), id(b))          # 相同 — 同一个 PyObject*
print(a is b)                  # True
a = [1, 2, 3, 4]              # a 重新绑定到新对象，b 不受影响
print(a is b)                  # False — a 指向新对象，b 仍指向旧对象
print(b)                       # [1, 2, 3] — 旧对象不变
```

`a = [1, 2, 3, 4]` 不是"修改 a 的盒子内容"，而是"让标签 a 贴到新对象上"。旧对象仍被 b 引用。

### 2.2 小整数缓存与大整数机制

CPython 在启动时预创建 -5 到 256 的整数对象，所有该范围内的整数绑定共享同一对象：

```c
// Objects/longobject.c (CPython 3.12 简化)
#define NSMALLNEGINTS 5
#define NSMALLPOSINTS 257
static PyLongObject small_ints[NSMALLNEGINTS + NSMALLPOSINTS];
```

```python
# 小整数缓存
a = 256; b = 256
print(a is b)   # True — 同一对象

a = 257; b = 257
print(a is b)   # False — 不同对象（交互模式下；同一代码块内 CPython 可能合并常量）
```

大整数（超出缓存范围）使用 `PyLongObject` 表示，内部以 30 位或 15 位为一段的数组存储（取决于平台），实现任意精度。

### 2.3 数据类型在 CPython 中的结构

**int — PyLongObject**

```c
// Include/longintrepr.h
struct _longobject {
    PyObject_VAR_HEAD       // ob_refcnt, ob_type, ob_size
    digit ob_digit[1];      // 数字数组，每个 digit 为 30 位
};
```

`ob_size` 的绝对值表示数组长度，符号表示正负。整数 0 的 `ob_size` 为 0。

**float — PyFloatObject**

```c
// Include/floatobject.h
typedef struct {
    PyObject_HEAD
    double ob_fval;    // 标准 C double，IEEE 754
} PyFloatObject;
```

**str — PyUnicodeObject**

Python 3 字符串在 CPython 中有紧凑表示（PEP 393）：

```c
// Include/cpython/unicodeobject.h
typedef struct {
    PyASCIIObject base;     // 长度、hash、状态
    union {
        unsigned int latin1[1];   // 1字节/字符 (Latin-1)
        unsigned int ucs1[1];      // 1字节/字符 (ASCII/Latin-1)
        unsigned int ucs2[1];     // 2字节/字符 (UCS-2/BMP)
        unsigned int ucs4[1];     // 4字节/字符 (UCS-4)
    } data;
} PyUnicodeObject;
```

根据字符串内容自动选择最紧凑的编码：纯 ASCII 用 1 字节/字符，含 BMP 字符用 2 字节/字符，含补充字符用 4 字节/字符。

**list — PyListObject**

```c
// Include/cpython/listobject.h
typedef struct {
    PyObject_VAR_HEAD
    PyObject **ob_item;     // 指向 PyObject* 数组的指针
    Py_ssize_t allocated;   // 已分配容量
} PyListObject;
```

列表底层是 `PyObject*` 指针数组，采用**过分配策略**：`allocated > ob_size`，以摊销 append 的开销。扩容公式约为 `new_allocated = (newsize >> 3) + (newsize < 9 ? 3 : 6) + newsize`。

**dict — PyDictObject**

Python 3.6+ 的字典采用紧凑字典（compact dict）实现，键值对存储在连续数组中，索引存储在稀疏哈希表中，插入顺序被保留。

**tuple — PyTupleObject**

```c
typedef struct {
    PyObject_VAR_HEAD
    PyObject *ob_item[1];   // 内联指针数组（变长）
} PyTupleObject;
```

元组不可变，无需过分配。单元素元组（singleton tuple）有特殊优化路径。

### 2.4 运算符调度的 CPython 机制

当 Python 执行 `a + b` 时，CPython 的调度流程：

1. 生成字节码 `BINARY_OP`（3.11+ 合并为统一指令，旧版为 `BINARY_ADD`）
2. 调用 `PyObject_Add(a, b)`
3. 先尝试 `a->ob_type->tp_as_number->nb_add(a, b)` — 即 `a.__add__(b)`
4. 若返回 `Py_NotImplemented`，尝试 `b->ob_type->tp_as_number->nb_radd(b, a)` — 即 `b.__radd__(a)`
5. 若仍返回 `Py_NotImplemented`，抛出 `TypeError`

```c
// Objects/abstract.c (简化)
PyObject *
PyNumber_Add(PyObject *v, PyObject *w)
{
    PyObject *result = binary_op1(v, w, NB_SLOT(nb_add));
    if (result == Py_NotImplemented) {
        Py_DECREF(result);
        result = binary_op1(w, v, NB_SLOT(nb_radd));
    }
    // ...
}
```

**反射方法（reflected method）的调用条件**：

- 当右操作数是左操作数子类的实例时，**直接跳过**左操作数的方法，优先尝试右操作数的反射方法——这确保子类能覆盖父类的行为
- 当两操作数类型不同且右操作数不是左操作数的子类时，按正常顺序尝试

### 2.5 整除与取模的数学原理

Python 的整除 `//` 和取模 `%` 遵循数学恒等式：

```
a = (a // b) * b + (a % b)
```

关键：**整除向负无穷取整（floor division）**，而非向零取整（truncation）。这与 C/Java 的 `/` 行为不同：

```python
# Python
print(7 // 2)     # 3
print(-7 // 2)    # -4 (向负无穷)
print(7 // -2)    # -4 (向负无穷)
print(-7 // -2)   # 3

# 取模结果符号与除数一致
print(7 % 2)      # 1
print(-7 % 2)     # 1  (因为 -7 = (-4) * 2 + 1)
print(7 % -2)     # -1 (因为 7 = (-4) * (-2) + (-1))
print(-7 % -2)    # -1
```

### 2.6 幂运算的右结合性与实现

`**` 是 Python 中唯一的右结合中缀运算符：

```python
2 ** 3 ** 2   # 等价于 2 ** (3 ** 2) = 2 ** 9 = 512，而非 (2 ** 3) ** 2 = 64
```

CPython 中幂运算通过 `long_pow` 实现，使用**快速幂算法（exponentiation by squaring）**：

```c
// Objects/longobject.c (简化)
static PyLongObject *
long_pow(PyLongObject *a, PyLongObject *b, PyLongObject *c)
{
    // 快速幂：O(log n) 次乘法
    while (exponent > 0) {
        if (exponent & 1)
            result = result * base;
        base = base * base;
        exponent >>= 1;
    }
}
```

### 2.7 增量赋值的原地语义

增量赋值（如 `+=`）在 CPython 中的行为取决于类型是否实现了 `__iadd__`：

```c
// Python/ceval.c — BINARY_OP 中 INPLACE_ADD 分支
case INPLACE_ADD: {
    // 先尝试 nb_inplace_add (即 __iadd__)
    // 若不存在或返回 NotImplemented，回退到 nb_add + STORE
}
```

- **list 的 `+=`**：调用 `list.__iadd__`，等价于 `list.extend()`，**原地修改**，返回 `self`
- **tuple/str/int 的 `+=`**：不可变类型没有 `__iadd__`，回退为 `__add__` + 赋值，**创建新对象**

```python
# 可变类型：原地修改
a = [1, 2]
b = a
a += [3]        # 调用 list.__iadd__，原地修改
print(a is b)   # True — 同一对象
print(a)         # [1, 2, 3]

# 不可变类型：创建新对象
c = (1, 2)
d = c
c += (3,)       # 等价于 c = c + (3,)，创建新元组
print(c is d)   # False — 不同对象
print(c)         # (1, 2, 3)
print(d)         # (1, 2) — 旧对象不受影响
```

### 2.8 `is` 运算符的底层机制

`is` 比较的是两个变量是否指向同一个 `PyObject`，在 C 层等价于指针比较：

```c
// Python/ceval.c
case COMPARE_OP: {
    // ...
    if (op == Py_IS) {
        res = (v == w);  // 直接指针比较
    }
}
```

`id()` 函数返回对象的内存地址（CPython 实现中）：

```c
// Objects/objectmodule.c
static PyObject *
builtin_id(PyObject *self, PyObject *v)
{
    return PyLong_FromVoidPtr((void *)v);  // 直接返回对象地址
}
```

### 2.9 缩进的底层实现 — INDENT/DEDENT Token

Python 的缩进在词法分析阶段被转换为 `INDENT` 和 `DEDENT` token，扮演与 C 语言花括号类似的角色。

**词法分析器的缩进栈**：

```c
// Parser/tokenizer.c (简化)
// 缩进栈初始为 [0]
// 每行开头：
//   1. 计算前导空白列数 (indent_level)
//   2. 若 indent_level > 栈顶：
//        - 压入 indent_level
//        - 生成 INDENT token
//   3. 若 indent_level < 栈顶：
//        - 弹出栈顶直到匹配
//        - 每弹出一个生成 DEDENT token
//   4. 若 indent_level == 栈顶：
//        - 不生成缩进 token（同层级继续）
```

```python
# 示例代码及其 token 序列
if True:
    print("a")
    if True:
        print("b")
    print("c")
```

对应 token 序列（简化）：

```
NAME 'if'    TRUE         NEWLINE
INDENT
NAME 'print'    STRING '"a"'    NEWLINE
NAME 'if'    TRUE         NEWLINE
INDENT
NAME 'print'    STRING '"b"'    NEWLINE
DEDENT
NAME 'print'    STRING '"c"'    NEWLINE
DEDENT
```

`INDENT` 和 `DEDENT` 在语法分析器中等价于 `{` 和 `}`——它们界定代码块的开始和结束。混用 Tab 和空格会导致 `TabError`，因为缩进列数计算结果不一致。

### 2.10 Docstring 的运行时角色

Docstring 不是注释——它是**运行时对象**，存储在模块/类/函数的 `__doc__` 属性中：

```c
// Python/bltinmodule.c
static PyObject *
builtin_help(PyObject *self, PyObject *args)
{
    // help() 函数读取对象的 __doc__ 属性并格式化输出
    // PyDoc_GetAttr() -> 调用 PyObject_GetAttr(obj, __doc_str)
}
```

CPython 编译器将模块/函数/类体内的第一个字符串字面量识别为 docstring，生成 `STORE_NAME __doc__` 字节码：

```python
import dis

def example():
    """This is a docstring."""
    pass

dis.dis(example)
# 输出中可见：
#   0 LOAD_CONST    'This is a docstring.'
#   2 STORE_NAME    '__doc__'
#   ...
```

这意味着 docstring 在模块加载时就被执行和存储，会产生微小的启动开销。在性能极其敏感的热路径上，可以省略 docstring 以避免该开销（但这几乎永远不值得）。

### 2.11 字节码层面的表达式求值

```python
import dis

# a + b * c
expr = compile("a + b * c", "<string>", "eval")
dis.dis(expr)
#   0 LOAD_NAME    0 (a)
#   2 LOAD_NAME    1 (b)
#   4 LOAD_NAME    2 (c)
#   6 BINARY_MULTIPLY     # b * c 先执行
#   8 BINARY_ADD           # a + (b*c) 后执行
#  10 RETURN_VALUE
```

操作数求值顺序（LOAD_NAME 的顺序）严格从左到右：先加载 `a`，再加载 `b`，再加载 `c`。但运算执行顺序由优先级决定：先 `*` 后 `+`。这证明了"操作数求值"与"运算执行"是两个不同阶段。

### 2.12 短路求值的字节码实现

```python
import dis

dis.dis(compile("a and b", "<string>", "eval"))
#   0 LOAD_NAME    0 (a)
#   2 JUMP_IF_FALSE_OR_POP  6    # 若 a 为假，跳到偏移 6，栈上保留 a
#   4 LOAD_NAME    1 (b)
#   6 RETURN_VALUE
```

`JUMP_IF_FALSE_OR_POP`：如果栈顶为假值，直接跳到目标（短路），不弹出也不加载 b；如果为真值，弹出它并继续加载 b。

`JUMP_IF_TRUE_OR_POP` 用于 `or`：如果栈顶为真值，跳到目标（短路），不加载 b。

## 3. 使用场景

### 3.1 变量绑定的典型场景

**场景一：避免不必要的数据拷贝**

```python
# 处理大列表时，传递引用而非拷贝
def process_data(data):
    # data 是对原始列表的引用，无需拷贝
    for item in data:
        print(item)

big_list = list(range(1_000_000))
process_data(big_list)  # 零拷贝传递
```

**场景二：理解可变默认参数陷阱**

```python
# 错误：可变默认值在函数定义时创建一次，所有调用共享同一对象
def append_to(item, target=[]):
    target.append(item)
    return target

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] — 意外！

# 正确：使用 None 作为哨兵
def append_to(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target
```

**场景三：交换变量**

```python
# Pythonic 的交换 — 基于元组解包
a, b = b, a
# 等价于：先构造 (b, a) 元组，再解包到 a, b
# 无需临时变量，也不存在并发问题
```

### 3.2 运算符的典型场景

**场景一：位运算用于标志位管理**

```python
# 权限系统
READ    = 0b0001   # 1
WRITE   = 0b0010   # 2
EXECUTE = 0b0100   # 4
ADMIN   = 0b1000   # 8

# 组合权限
permission = READ | WRITE              # 0b0011 = 3
has_read = bool(permission & READ)     # True
has_exec = bool(permission & EXECUTE)  # False

# 添加权限
permission |= EXECUTE                   # 0b0111 = 7
# 移除权限
permission &= ~WRITE                    # 0b0101 = 5
```

**场景二：短路求值用于默认值和条件执行**

```python
# 默认值模式
name = user_input or "anonymous"   # 空字符串时使用默认值

# 条件执行模式
is_debug and print("debug info")    # 仅调试时输出，避免 if 语句
result = condition and compute()    # 条件为真才执行计算

# 链式默认值
config = env_value or file_value or default_value
```

```python
# 传统写法
line = input()

while (line := input()) != "quit":
```

### 3.3 类型选择的典型场景

| 场景 | 推荐类型 | 原因 |
|------|---------|------|
| 计数/索引 | `int` | 任意精度，无溢出 |
| 科学计算 | `float` / `complex` | IEEE 754，兼容 NumPy |
| 文本处理 | `str` | Unicode 原生支持 |
| 有序集合 | `list` | 动态数组，随机访问 O(1) |
| 不可变序列 | `tuple` | 可哈希，可作为 dict 键 |
| 键值映射 | `dict` | O(1) 查找，3.7+ 有序 |
| 去重/成员检测 | `set` | O(1) 成员检测 |
| 函数返回多值 | `tuple` | 惯用法，轻量 |
| 哨兵值 | `None` | 单例，`is None` 判断 |

### 3.4 `is` vs `==` 的使用场景

- **用 `is` 判断**：`None`、`True`、`False`、类型检查（`type(x) is int`）
- **用 `==` 判断**：值相等比较（`x == 42`、`s == "hello"`）
- **永远不要用 `is` 比较值**：`x is 257` 可能因缓存失效而错误

```python
# 正确用法
if response is None:
    ...
if flag is True:    # 比 == 更严格，不接受 1 等真值
    ...

# 错误用法
if x is 1000:       # 实现依赖，可能为 False 即使值相等
    ...
```

## 4. 示例代码

### 4.1 变量绑定完整演示

```python
"""
变量绑定机制完整演示
展示 Python 变量是"标签"而非"盒子"的核心概念
"""

import sys


def demonstrate_binding():
    """演示变量绑定的核心行为。"""

    # 1. 基本绑定
    x = 42
    print(f"x = {x}, id(x) = {id(x)}")

    # 2. 多个名称绑定到同一对象
    a = [1, 2, 3]
    b = a               # b 和 a 是同一对象的两个标签
    print(f"a is b: {a is b}")          # True
    print(f"id(a) = {id(a)}, id(b) = {id(b)}")  # 相同

    # 3. 重新绑定不改变旧对象
    a = [1, 2, 3, 4]    # a 绑定到新对象，b 不受影响
    print(f"a is b: {a is b}")          # False
    print(f"b = {b}")                   # [1, 2, 3]

    # 4. 小整数缓存
    c = 256; d = 256
    print(f"256: c is d = {c is d}")     # True
    e = 257; f = 257
    print(f"257: e is f = {e is f}")     # False (交互模式；同代码块可能 True)

    # 5. 字符串驻留 (interning)
    s1 = "hello_world"    # 下划线连接，符合标识符规则，可能驻留
    s2 = "hello_world"
    print(f"interned: s1 is s2 = {s1 is s2}")  # True (通常)

    # 包含特殊字符的字符串不自动驻留
    s3 = "hello world!"
    s4 = "hello world!"
    print(f"not interned: s3 is s4 = {s3 is s4}")  # 通常 False

    # 6. 引用计数
    g = [10, 20]
    print(f"refcount of g: {sys.getrefcount(g)}")  # 2 (g + getrefcount 参数)
    h = g
    print(f"refcount after h=g: {sys.getrefcount(g)}")  # 3

    # 7. 增量赋值对可变/不可变类型的不同行为
    list_a = [1, 2]
    list_b = list_a
    list_a += [3]        # 原地修改 (list.__iadd__)
    print(f"list: list_a is list_b = {list_a is list_b}")  # True

    tuple_a = (1, 2)
    tuple_b = tuple_a
    tuple_a += (3,)      # 创建新对象 (tuple 无 __iadd__)
    print(f"tuple: tuple_a is tuple_b = {tuple_a is tuple_b}")  # False


if __name__ == "__main__":
    demonstrate_binding()
```

### 4.2 运算符完整演示

```python
"""
运算符完整演示
覆盖所有运算符类别及其边界行为
"""


def demonstrate_arithmetic():
    """算术运算符及边界情况。"""

    # 基本算术
    print("=== 算术运算符 ===")
    print(f"7 / 2 = {7 / 2}")        # 3.5 (真除法，始终 float)
    print(f"7 // 2 = {7 // 2}")      # 3 (整除，向负无穷)
    print(f"-7 // 2 = {-7 // 2}")     # -4 (不是 -3!)
    print(f"7 % 2 = {7 % 2}")        # 1
    print(f"-7 % 2 = {-7 % 2}")      # 1 (不是 -1!)

    # 幂运算右结合
    print(f"2 ** 3 ** 2 = {2 ** 3 ** 2}")  # 512 = 2^(3^2)

    # 大整数 — Python 不会溢出
    big = 2 ** 1000
    print(f"2^1000 has {len(str(big))} digits")  # 302 digits

    # 浮点精度
    print(f"0.1 + 0.2 = {0.1 + 0.2}")           # 0.30000000000000004
    print(f"0.1 + 0.2 == 0.3 = {0.1 + 0.2 == 0.3}")  # False!

    # 使用 decimal 模块处理精确小数
    from decimal import Decimal
    print(f"Decimal('0.1') + Decimal('0.2') = {Decimal('0.1') + Decimal('0.2')}")

    # 复数运算
    z = 3 + 4j
    print(f"abs({z}) = {abs(z)}")  # 5.0


def demonstrate_comparison():
    """比较运算符。"""
    print("\n=== 比较运算符 ===")

    # 值比较 vs 身份比较
    a = [1, 2, 3]
    b = [1, 2, 3]
    print(f"[1,2,3] == [1,2,3]: {a == b}")  # True (值相等)
    print(f"[1,2,3] is [1,2,3]: {a is b}")  # False (不同对象)

    # None 比较
    x = None
    print(f"x is None: {x is None}")     # True (推荐)

    # bool 是 int 的子类
    print(f"True == 1: {True == 1}")     # True
    print(f"False == 0: {False == 0}")   # True
    print(f"True is 1: {True is 1}")     # False — 不同对象

    # 链式比较
    x = 5
    print(f"1 < x < 10: {1 < x < 10}")   # True (等价于 (1 < x) and (x < 10))
    print(f"1 < x > 10: {1 < x > 10}")   # False


def demonstrate_logical():
    """逻辑运算符与短路求值。"""
    print("\n=== 逻辑运算符 ===")

    # and/or 返回操作数，不一定是 bool
    print(f"'hello' and 42: {'hello' and 42}")      # 42
    print(f"'' and 42: {'' and 42}")                # ''
    print(f"0 or 'default': {0 or 'default'}")      # 'default'
    print(f"not 0: {not 0}")                        # True


def demonstrate_bitwise():
    """位运算符。"""
    print("\n=== 位运算符 ===")

    a = 0b1100   # 12
    b = 0b1010   # 10

    print(f"{a} & {b} = {a & b}  (0b{a & b:04b})")    # 8  (0b1000)
    print(f"{a} | {b} = {a | b}  (0b{a | b:04b})")    # 14 (0b1110)
    print(f"{a} ^ {b} = {a ^ b}  (0b{a ^ b:04b})")    # 6  (0b0110)

    # 移位
    print(f"1 << 10 = {1 << 10}")     # 1024
    print(f"1024 >> 3 = {1024 >> 3}") # 128


if __name__ == "__main__":
    demonstrate_arithmetic()
    demonstrate_comparison()
    demonstrate_logical()
    demonstrate_bitwise()
```

## 5. 常见陷阱与最佳实践

### 5.1 变量绑定陷阱

**陷阱一：误以为变量是独立的拷贝**

```python
# 陷阱：修改可变对象影响所有绑定
shared_list = [1, 2, 3]
a = shared_list
a.append(4)
# shared_list 也变成了 [1, 2, 3, 4]!

# 最佳实践：需要独立拷贝时显式复制
import copy
a = copy.copy(shared_list)       # 浅拷贝
a = copy.deepcopy(shared_list)   # 深拷贝（嵌套结构时）
```

**陷阱二：可变默认参数**

```python
# 陷阱
def append_to(element, lst=[]):
    lst.append(element)
    return lst

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] — 非预期！

# 最佳实践：使用 None 哨兵
def append_to(element, lst=None):
    if lst is None:
        lst = []
    lst.append(element)
    return lst
```

**陷阱三：增量赋值对可变对象的意外原地修改**

```python
# 陷阱：+= 对列表的原地修改
original = [1, 2, 3]
reference = original
original += [4]  # 原地修改！reference 也变了

# 如果想避免，使用拼接赋值
original = [1, 2, 3]
reference = original
original = original + [4]  # 创建新对象！reference 不变
```

### 5.2 运算符陷阱

**陷阱一：浮点精度**

```python
# 陷阱
print(0.1 + 0.2 == 0.3)   # False

# 最佳实践：使用 math.isclose 或 decimal
import math
print(math.isclose(0.1 + 0.2, 0.3))  # True

from decimal import Decimal
print(Decimal('0.1') + Decimal('0.2') == Decimal('0.3'))  # True
```

**陷阱二：整除的负数行为**

```python
# 陷阱：C/Java 程序员预期 -7 / 2 = -3，但 Python 中
print(-7 // 2)  # -4 (向负无穷取整)

# 最佳实践：若需向零取整，使用 math.trunc 或 int()
import math
print(math.trunc(-7 / 2))  # -3
```

**陷阱三：`is` 误用于值比较**

```python
# 陷阱
a = 256; b = 256
print(a is b)  # True (缓存范围内)

c = 257; d = 257
print(c is d)  # 可能 False (超出缓存)

# 最佳实践：仅对 None/True/False 使用 is，值比较用 ==
```

**陷阱四：`and`/`or` 返回值不是布尔值**

```python
# 陷阱
result = "hello" and "world"
print(result)  # "world" — 不是 True!

# 最佳实践：如果需要布尔结果，显式使用 bool()
is_both_truthy = bool("hello" and "world")  # True
```

### 5.3 类型相关陷阱

**陷阱一：元组单元素的逗号**

```python
# 陷阱
single = (1)     # 这只是整数 1，不是元组！
print(type(single))  # <class 'int'>

# 最佳实践：单元素元组必须加逗号
single = (1,)    # 元组
print(type(single))  # <class 'tuple'>
```

**陷阱二：Tab 与空格混用**

```python
# 陷阱（Python 3 会抛出 TabError）
# if True:
#     print("a")      # 4 空格
# 	print("b")       # Tab — 混用！

# 最佳实践：始终使用 4 空格，编辑器设置 Tab 自动转空格
```

### 5.4 最佳实践总结

| 领域 | 最佳实践 | 原因 |
|------|---------|------|
| 变量绑定 | 理解"标签"模型，而非"盒子"模型 | 避免 mutable 共享引用的意外修改 |
| 值比较 | 使用 `==` 比较值，`is` 仅用于 None/True/False | `is` 对值比较依赖实现细节 |
| 可变默认参数 | 使用 `None` 哨兵模式 | 避免跨调用共享状态 |
| 浮点比较 | 使用 `math.isclose()` | 避免 IEEE 754 精度问题 |
| 整除取模 | 注意负数的 floor 行为 | 与 C/Java 的 truncation 行为不同 |
| 缩进 | 统一 4 空格，禁用 Tab | 避免混用导致的 TabError |
| 命名 | 遵循 PEP 8 | 团队一致性，可读性 |
| Docstring | 为公开 API 编写 docstring | 运行时可通过 `__doc__` 和 `help()` 访问 |
| 短路求值 | 利用短路做条件执行和默认值 | 简洁高效 |
| 类型选择 | 优先使用不可变类型（tuple/frozenset） | 安全性、可哈希、线程安全 |
| 拷贝 | 需要独立副本时显式 copy | 防止共享引用的意外修改 |

> **关键启示**：Python 的基础语法看似简单，但每一个"简单"的概念背后都有深层的实现原理——变量绑定是 `PyObject*` 指针的标签，缩进是 INDENT/DEDENT token，运算符调度经过双分派协议，表达式求值区分操作数求值顺序和运算执行顺序。理解这些底层机制，是避免陷阱、写出正确高效代码的根基。
