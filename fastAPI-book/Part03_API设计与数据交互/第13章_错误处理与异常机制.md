# 第13章 错误处理与异常机制

错误处理是构建健壮 API 的关键环节。本章从 FastAPI 的异常处理到自定义异常，从 HTTPException 到全局错误处理器，深入剖析 FastAPI 错误处理机制的底层原理。

---

## 13.1 概念与语法

### 13.1.1 HTTPException

```python
from fastapi import FastAPI, HTTPException, status

app = FastAPI()

# 基本 HTTP 异常
@app.get("/items/{item_id}")
async def get_item(item_id: int):
    item = find_item(item_id)
    if not item:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="Item not found",
        )
    return item

# 带自定义头
@app.get("/rate-limited")
async def rate_limited():
    raise HTTPException(
        status_code=429,
        detail="Too many requests",
        headers={"Retry-After": "60"},
    )

# HTTPException 结构：
# status_code — HTTP 状态码（必需）
# detail      — 错误详情（可以是字符串、字典、列表）
# headers     — 额外响应头（可选）

# detail 支持多种类型
raise HTTPException(400, detail={"field": "email", "message": "Invalid format"})
raise HTTPException(422, detail=[
    {"loc": ["body", "email"], "msg": "Invalid email", "type": "value_error"},
])
```

### 13.1.2 自定义异常类

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

app = FastAPI()

# 业务异常基类
class AppException(Exception):
    def __init__(self, code: str, message: str, status_code: int = 400):
        self.code = code
        self.message = message
        self.status_code = status_code

class NotFoundException(AppException):
    def __init__(self, resource: str, resource_id: int):
        super().__init__(
            code="NOT_FOUND",
            message=f"{resource} with id {resource_id} not found",
            status_code=404,
        )

class ConflictException(AppException):
    def __init__(self, resource: str, field: str, value: str):
        super().__init__(
            code="CONFLICT",
            message=f"{resource} with {field}='{value}' already exists",
            status_code=409,
        )

class UnauthorizedException(AppException):
    def __init__(self, message: str = "Authentication required"):
        super().__init__(code="UNAUTHORIZED", message=message, status_code=401)

# 注册异常处理器
@app.exception_handler(AppException)
async def app_exception_handler(request: Request, exc: AppException):
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "code": exc.code,
            "message": exc.message,
            "path": str(request.url),
        },
    )

# 在路由中使用
@app.get("/users/{user_id}")
async def get_user(user_id: int):
    user = find_user(user_id)
    if not user:
        raise NotFoundException("User", user_id)
    return user
```

### 13.1.3 请求验证异常

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

app = FastAPI()

# 自定义验证错误响应
@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    errors = []
    for error in exc.errors():
        errors.append({
            "field": ".".join(str(loc) for loc in error["loc"]),
            "message": error["msg"],
            "type": error["type"],
        })

    return JSONResponse(
        status_code=422,
        content={
            "code": "VALIDATION_ERROR",
            "message": "Request validation failed",
            "errors": errors,
        },
    )
```

### 13.1.4 全局异常处理

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
import logging
import traceback

app = FastAPI()
logger = logging.getLogger(__name__)

# 处理所有未捕获异常
@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    logger.error(f"Unhandled exception: {exc}", exc_info=True)
    return JSONResponse(
        status_code=500,
        content={
            "code": "INTERNAL_ERROR",
            "message": "An unexpected error occurred",
            "path": str(request.url),
        },
    )

# 处理特定异常
@app.exception_handler(ValueError)
async def value_error_handler(request: Request, exc: ValueError):
    return JSONResponse(
        status_code=400,
        content={"code": "BAD_REQUEST", "message": str(exc)},
    )

@app.exception_handler(PermissionError)
async def permission_error_handler(request: Request, exc: PermissionError):
    return JSONResponse(
        status_code=403,
        content={"code": "FORBIDDEN", "message": str(exc)},
    )

# 异常处理器的优先级：
# 1. 最具体的异常类优先
# 2. 父类异常处理器兜底
# 3. @app.exception_handler 优先于 FastAPI 默认处理器
# 4. Starlette HTTPException 先于 Python Exception
```

## 13.2 原理与机制

### 13.2.1 FastAPI 异常处理流程

```python
"""
FastAPI 异常处理的完整流程：

1. 请求进入路由函数
   ↓
2. 路由函数执行过程中抛出异常
   ↓
3. Starlette 的 ExceptionMiddleware 捕获异常
   ↓
4. 查找匹配的 exception_handler：
   a. 检查用户注册的 @exception_handler
   b. 从最具体的异常类开始匹配
   c. 沿继承链向上查找
   d. 找不到则使用默认处理器
   ↓
5. 执行异常处理器
   - 用户自定义处理器 → 返回自定义响应
   - 默认处理器 → 返回标准 JSON 错误
   ↓
6. 返回错误响应给客户端

源码位置：
- starlette/middleware/errors.py: ServerErrorMiddleware
- starlette/middleware/exceptions.py: ExceptionMiddleware
- fastapi/exception_handlers.py: 默认异常处理器

关键类：
- ExceptionMiddleware: 捕获所有异常并路由到处理器
- HTTPException: FastAPI 内置的 HTTP 异常类
- RequestValidationError: Pydantic 验证错误
"""
```

### 13.2.2 HTTPException 的实现

```python
"""
HTTPException 的源码实现（starlette/exceptions.py）：

class HTTPException(Exception):
    def __init__(self, status_code: int, detail: Any = None, headers: dict | None = None):
        self.status_code = status_code
        self.detail = detail
        self.headers = headers

    def __repr__(self):
        return f"{self.status_code}: {self.detail}"

默认处理器（fastapi/exception_handlers.py）：

async def http_exception_handler(request, exc):
    return JSONResponse(
        {"detail": exc.detail},
        status_code=exc.status_code,
        headers=exc.headers,
    )

关键特性：
1. HTTPException 不是 Python 内置异常
2. 继承自 Exception，被 Starlette 中间件捕获
3. detail 可以是任意可序列化的值
4. headers 允许设置自定义响应头
5. 默认被 JSONResponse 处理器处理
"""
```

### 13.2.3 验证错误的内部结构

```python
"""
RequestValidationError 的错误结构：

exc.errors() 返回列表，每个元素：
{
    "type": "missing",          # 错误类型
    "loc": ["body", "name"],    # 错误位置
    "msg": "Field required",    # 错误消息
    "input": None,               # 输入值
}

常见错误类型：
- missing:          必填字段缺失
- value_error:      自定义验证器抛出 ValueError
- string_type:      字符串类型不匹配
- int_parsing:      整数解析失败
- greater_than:     gt 约束失败
- less_than:        lt 约束失败
- too_short:        min_length 约束失败
- too_long:         max_length 约束失败
- url_parsing:      URL 格式错误
- json_invalid:     JSON 解析失败
- unexpected:       多余字段

loc 位置格式：
- ("path", "param_name")     → 路径参数
- ("query", "param_name")    → 查询参数
- ("body",)                  → 请求体
- ("body", "field_name")     → 请求体字段
- ("body", "field", 0)       → 嵌套列表索引
"""
```

## 13.3 使用场景

### 13.3.1 统一错误响应格式

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError
from pydantic import BaseModel
from datetime import datetime
import traceback

app = FastAPI()

class ErrorResponse(BaseModel):
    code: str
    message: str
    details: list | None = None
    path: str
    timestamp: str
    request_id: str | None = None

@app.middleware("http")
async def add_request_id(request: Request, call_next):
    import uuid
    request.state.request_id = str(uuid.uuid4())
    response = await call_next(request)
    return response

@app.exception_handler(RequestValidationError)
async def validation_handler(request: Request, exc: RequestValidationError):
    details = [
        {"field": ".".join(str(l) for l in e["loc"]), "message": e["msg"], "type": e["type"]}
        for e in exc.errors()
    ]
    return JSONResponse(status_code=422, content=ErrorResponse(
        code="VALIDATION_ERROR",
        message="Request validation failed",
        details=details,
        path=str(request.url.path),
        timestamp=datetime.utcnow().isoformat(),
        request_id=getattr(request.state, "request_id", None),
    ).model_dump())

@app.exception_handler(Exception)
async def global_handler(request: Request, exc: Exception):
    return JSONResponse(status_code=500, content=ErrorResponse(
        code="INTERNAL_ERROR",
        message="An unexpected error occurred",
        path=str(request.url.path),
        timestamp=datetime.utcnow().isoformat(),
        request_id=getattr(request.state, "request_id", None),
    ).model_dump())
```

### 13.3.2 业务异常体系

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

app = FastAPI()

class DomainException(Exception):
    """业务异常基类。"""
    status_code: int = 400
    code: str = "DOMAIN_ERROR"
    message: str = "Domain error occurred"

class NotFoundError(DomainException):
    status_code = 404
    code = "NOT_FOUND"
    def __init__(self, resource: str, id: int):
        self.message = f"{resource} with id={id} not found"

class ConflictError(DomainException):
    status_code = 409
    code = "CONFLICT"
    def __init__(self, resource: str, field: str, value: str):
        self.message = f"{resource} with {field}='{value}' already exists"

class ForbiddenError(DomainException):
    status_code = 403
    code = "FORBIDDEN"
    message = "You don't have permission to perform this action"

class RateLimitError(DomainException):
    status_code = 429
    code = "RATE_LIMIT_EXCEEDED"
    def __init__(self, retry_after: int):
        self.message = f"Rate limit exceeded. Retry after {retry_after}s"
        self.retry_after = retry_after

@app.exception_handler(DomainException)
async def domain_exception_handler(request: Request, exc: DomainException):
    response = JSONResponse(
        status_code=exc.status_code,
        content={"code": exc.code, "message": exc.message, "path": str(request.url.path)},
    )
    if hasattr(exc, "retry_after"):
        response.headers["Retry-After"] = str(exc.retry_after)
    return response
```

## 13.4 示例代码

### 13.4.1 完整的错误处理中间件

```python
"""生产级错误处理中间件"""
import logging
import traceback
import uuid
from datetime import datetime
from fastapi import FastAPI, Request, HTTPException
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

app = FastAPI()
logger = logging.getLogger("app.errors")

class ErrorCodes:
    VALIDATION_ERROR = "VALIDATION_ERROR"
    NOT_FOUND = "NOT_FOUND"
    UNAUTHORIZED = "UNAUTHORIZED"
    FORBIDDEN = "FORBIDDEN"
    CONFLICT = "CONFLICT"
    RATE_LIMITED = "RATE_LIMITED"
    INTERNAL_ERROR = "INTERNAL_ERROR"

@app.exception_handler(RequestValidationError)
async def validation_handler(request: Request, exc: RequestValidationError):
    errors = [
        {
            "field": ".".join(str(l) for l in err["loc"]),
            "message": err["msg"],
            "type": err["type"],
        }
        for err in exc.errors()
    ]
    logger.warning(f"Validation error on {request.url.path}: {errors}")
    return JSONResponse(status_code=422, content={
        "code": ErrorCodes.VALIDATION_ERROR,
        "message": "Request validation failed",
        "details": errors,
        "timestamp": datetime.utcnow().isoformat(),
        "request_id": getattr(request.state, "request_id", None),
    })

@app.exception_handler(HTTPException)
async def http_exception_handler(request: Request, exc: HTTPException):
    return JSONResponse(status_code=exc.status_code, content={
        "code": f"HTTP_{exc.status_code}",
        "message": str(exc.detail),
        "timestamp": datetime.utcnow().isoformat(),
        "request_id": getattr(request.state, "request_id", None),
    })

@app.exception_handler(Exception)
async def unhandled_exception_handler(request: Request, exc: Exception):
    error_id = str(uuid.uuid4())
    logger.error(f"Unhandled exception [{error_id}]: {traceback.format_exc()}")
    return JSONResponse(status_code=500, content={
        "code": ErrorCodes.INTERNAL_ERROR,
        "message": "An unexpected error occurred",
        "error_id": error_id,
        "timestamp": datetime.utcnow().isoformat(),
    })

@app.middleware("http")
async def error_context_middleware(request: Request, call_next):
    request.state.request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))
    try:
        response = await call_next(request)
        response.headers["X-Request-ID"] = request.state.request_id
        return response
    except Exception as exc:
        raise  # 让异常处理器处理
```

## 13.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 裸 raise Exception | 使用 HTTPException 或自定义异常类 |
| 错误信息泄露内部细节 | 生产环境返回通用错误信息 |
| 不记录异常日志 | 所有异常都应记录到日志系统 |
| 422 错误格式不友好 | 自定义 RequestValidationError 处理器 |
| 异常处理器不处理 headers | HTTPException 可能包含自定义 headers |
| 不区分开发和生产错误 | 开发环境显示堆栈，生产环境隐藏 |
| 异常继承层次太深 | 保持异常体系扁平，2-3 层即可 |
| 不使用 status_code 常量 | 使用 `status.HTTP_404_NOT_FOUND` 等常量 |
