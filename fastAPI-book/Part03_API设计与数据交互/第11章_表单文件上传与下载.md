# 第11章 表单、文件上传与下载

表单和文件处理是 Web 应用的常见需求。本章从 Form 参数到文件上传，从多文件处理到流式响应，深入剖析 FastAPI 处理表单和文件的机制与实践。

## 11.1 概念与语法

### 11.1.1 Form 表单参数

```python
from fastapi import FastAPI, Form

app = FastAPI()

# 基本表单参数
@app.post("/login")
async def login(username: str = Form(), password: str = Form()):
    return {"username": username}

# 带默认值的表单参数
@app.post("/contact")
async def contact(
    name: str = Form(..., min_length=1, max_length=100),
    email: str = Form(..., pattern=r"^[\w.-]+@[\w.-]+\.\w+$"),
    message: str = Form(..., min_length=10),
    subject: str = Form("General Inquiry"),
):
    return {"name": name, "email": email}

# 表单与 JSON 的区别：
# JSON 请求：Content-Type: application/json
# 表单请求：Content-Type: application/x-www-form-urlencoded
# FastAPI 根据 Content-Type 决定解析方式
```

### 11.1.2 文件上传

```python
from fastapi import FastAPI, File, UploadFile
from fastapi.responses import JSONResponse

app = FastAPI()

# 单文件上传（UploadFile 推荐）
@app.post("/upload")
async def upload_file(file: UploadFile = File(...)):
    contents = await file.read()
    return {
        "filename": file.filename,
        "content_type": file.content_type,
        "size": len(contents),
    }

# UploadFile 属性和方法：
# file.filename     — 原始文件名
# file.content_type — MIME 类型
# file.size         — 文件大小（Python 3.9+）
# await file.read() — 读取内容
# await file.write()— 写入内容
# await file.seek() — 移动指针
# await file.close()— 关闭文件

# 保存上传文件
import aiofiles

@app.post("/upload/save")
async def upload_and_save(file: UploadFile = File(...)):
    save_path = f"uploads/{file.filename}"
    async with aiofiles.open(save_path, "wb") as f:
        content = await file.read()
        await f.write(content)
    return {"filename": file.filename, "saved_to": save_path}

# 多文件上传
@app.post("/uploads")
async def upload_files(files: list[UploadFile] = File(...)):
    results = []
    for file in files:
        content = await file.read()
        results.append({
            "filename": file.filename,
            "size": len(content),
        })
    return {"uploaded": results}
```

### 11.1.4 文件下载与流式响应

```python
from fastapi import FastAPI
from fastapi.responses import (
    FileResponse,
    StreamingResponse,
    Response,
)

app = FastAPI()

# FileResponse — 发送文件
@app.get("/download/{filename}")
async def download_file(filename: str):
    file_path = f"uploads/{filename}"
    return FileResponse(
        path=file_path,
        filename=filename,
        media_type="application/octet-stream",
    )

# StreamingResponse — 流式发送大文件
@app.get("/stream/{filename}")
async def stream_file(filename: str):
    file_path = f"uploads/{filename}"

    async def file_generator():
        async with aiofiles.open(file_path, "rb") as f:
            while chunk := await f.read(64 * 1024):  # 64KB chunks
                yield chunk

    return StreamingResponse(
        file_generator(),
        media_type="application/octet-stream",
        headers={"Content-Disposition": f"attachment; filename={filename}"},
    )

# 生成 CSV 下载
import io
import csv

@app.get("/export/csv")
async def export_csv():
    output = io.StringIO()
    writer = csv.writer(output)
    writer.writerow(["id", "name", "price"])
```

## 11.2 原理与机制

### 11.2.1 文件上传的内部处理

```python
"""
UploadFile 的内部机制：

1. 请求到达时，FastAPI 检测 Content-Type：
   - multipart/form-data → 解析为表单 + 文件
   - application/x-www-form-urlencoded → 解析为表单
   - application/json → 解析为 JSON

2. multipart 解析流程：
   - 使用 python-multipart 库解析边界
   - 每个 part 根据 Content-Disposition 判断类型
   - name 字段匹配 Form 参数 → 存入 form 数据
   - filename 字段 → 创建 UploadFile 对象

3. UploadFile 的文件存储：
   - 小文件（< 2MB）→ 存入内存（SpooledTemporaryFile）
   - 大文件 → 写入磁盘临时文件
   - SpooledTemporaryFile 的 max_size 默认 2MB

4. 源码位置：
   - starlette/requests.py: Request.form()
   - starlette/datastructures.py: UploadFile
   - fastapi/params.py: File()

5. UploadFile 内部结构：
   class UploadFile:
       filename: str | None
       size: int | None
       headers: Headers
       file: SpooledTemporaryFile  # 核心存储
"""
```

### 11.2.2 StreamingResponse 机制

```python
"""
StreamingResponse 的工作原理：

1. ASGI 协议的流式响应：
   await send({
       "type": "http.response.start",
       "status": 200,
       "headers": [...],
   })
   # 然后循环发送 body
   async for chunk in body_iterator:
       await send({
           "type": "http.response.body",
           "body": chunk,
           "more_body": True,  # 还有更多数据
       })
   # 最后一块
   await send({
       "type": "http.response.body",
       "body": b"",
       "more_body": False,
   })

2. FileResponse vs StreamingResponse：
   - FileResponse：自动检测文件大小，设置 Content-Length
   - StreamingResponse：不预知大小，使用分块传输编码

3. 分块传输编码（Chunked Transfer Encoding）：
   - 无需 Content-Length 头
   - 每个分块前标记大小
   - 最后发送空分块表示结束
   - 适合大文件或实时数据
"""
```

## 11.3 使用场景

### 11.3.1 图片上传与缩略图

```python
from fastapi import FastAPI, UploadFile, File, HTTPException
from pathlib import Path
import aiofiles
from PIL import Image
import io

app = FastAPI()
UPLOAD_DIR = Path("uploads")
UPLOAD_DIR.mkdir(exist_ok=True)

ALLOWED_TYPES = {"image/jpeg", "image/png", "image/webp"}
MAX_SIZE = 5 * 1024 * 1024  # 5MB

@app.post("/images/upload")
async def upload_image(file: UploadFile = File(...)):
    # 验证文件类型
    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(400, detail=f"Invalid file type: {file.content_type}")

    # 读取内容并验证大小
    content = await file.read()
    if len(content) > MAX_SIZE:
        raise HTTPException(400, detail="File too large")

    # 保存原始文件
    file_path = UPLOAD_DIR / file.filename
    async with aiofiles.open(file_path, "wb") as f:
        await f.write(content)

    # 生成缩略图
    img = Image.open(io.BytesIO(content))
    img.thumbnail((200, 200))
    thumb_path = UPLOAD_DIR / f"thumb_{file.filename}"
    img.save(thumb_path)

    return {
        "filename": file.filename,
        "size": len(content),
        "thumbnail": f"thumb_{file.filename}",
    }
```

### 11.3.2 大文件分片上传

```python
from fastapi import FastAPI, UploadFile, File, Form, HTTPException
from pathlib import Path
import aiofiles

app = FastAPI()
CHUNK_DIR = Path("chunks")
CHUNK_DIR.mkdir(exist_ok=True)

@app.post("/upload/chunk")
async def upload_chunk(
    file_id: str = Form(...),
    chunk_index: int = Form(...),
    total_chunks: int = Form(...),
    chunk: UploadFile = File(...),
):
    # 保存分片
    chunk_path = CHUNK_DIR / f"{file_id}_{chunk_index}"
    content = await chunk.read()
    async with aiofiles.open(chunk_path, "wb") as f:
        await f.write(content)

    # 检查是否所有分片都已上传
    uploaded = len(list(CHUNK_DIR.glob(f"{file_id}_*")))
    if uploaded == total_chunks:
        # 合并分片
        final_path = Path("uploads") / file_id
        async with aiofiles.open(final_path, "wb") as f:
            for i in range(total_chunks):
                chunk_path = CHUNK_DIR / f"{file_id}_{i}"
                async with aiofiles.open(chunk_path, "rb") as cf:
                    await f.write(await cf.read())
                chunk_path.unlink()  # 删除分片

        return {"status": "completed", "file_id": file_id}

    return {"status": "partial", "uploaded_chunks": uploaded, "total_chunks": total_chunks}
```

## 11.4 示例代码

### 11.4.1 完整的文件管理 API

```python
"""文件管理 API — 上传、下载、列表、删除"""
import os
import aiofiles
from pathlib import Path
from fastapi import FastAPI, UploadFile, File, HTTPException, Query
from fastapi.responses import FileResponse, StreamingResponse

app = FastAPI(title="File Manager API")
UPLOAD_DIR = Path("uploads")
UPLOAD_DIR.mkdir(exist_ok=True)
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
ALLOWED_EXTENSIONS = {".jpg", ".jpeg", ".png", ".gif", ".pdf", ".txt", ".zip"}

@app.post("/files/upload")
async def upload_file(file: UploadFile = File(...)):
    # 验证扩展名
    ext = Path(file.filename).suffix.lower()
    if ext not in ALLOWED_EXTENSIONS:
        raise HTTPException(400, detail=f"File type not allowed: {ext}")

    # 读取内容
    content = await file.read()
    if len(content) > MAX_FILE_SIZE:
        raise HTTPException(400, detail="File too large")

    # 生成唯一文件名
    import uuid
    unique_name = f"{uuid.uuid4().hex}{ext}"
    file_path = UPLOAD_DIR / unique_name

    # 保存文件
    async with aiofiles.open(file_path, "wb") as f:
        await f.write(content)

    return {
        "id": unique_name,
        "original_name": file.filename,
        "size": len(content),
        "content_type": file.content_type,
    }

@app.get("/files/{file_id}")
async def download_file(file_id: str):
    file_path = UPLOAD_DIR / file_id
    if not file_path.exists():
        raise HTTPException(404, detail="File not found")
    return FileResponse(
        path=file_path,
        filename=file_id,
        media_type="application/octet-stream",
    )

@app.get("/files")
async def list_files(page: int = Query(1, ge=1), page_size: int = Query(20, ge=1, le=100)):
    files = list(UPLOAD_DIR.iterdir())
    total = len(files)
    start = (page - 1) * page_size
    items = [
        {"id": f.name, "size": f.stat().st_size}
        for f in files[start:start + page_size]
    ]
    return {"items": items, "total": total, "page": page, "page_size": page_size}

@app.delete("/files/{file_id}", status_code=204)
async def delete_file(file_id: str):
    file_path = UPLOAD_DIR / file_id
    if not file_path.exists():
        raise HTTPException(404, detail="File not found")
    file_path.unlink()
```

## 11.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 不限制上传文件大小 | 设置 MAX_FILE_SIZE 并在读取后验证 |
| 不验证文件类型 | 检查 content_type 和文件扩展名 |
| 大文件全部读入内存 | 使用流式读取和 SpooledTemporaryFile |
| 文件名冲突 | 使用 UUID 生成唯一文件名 |
| 不清理临时文件 | 使用 try/finally 或 async with 确保清理 |
| UploadFile 不关闭 | 使用 async with 或手动 await file.close() |
| 不设置 Content-Disposition | 文件下载时设置 attachment 和 filename |
| 路径遍历攻击 | 验证文件名不含 ../ 等路径操作符 |
