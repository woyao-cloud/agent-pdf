# 第44章 依赖注入系统源码分析

依赖注入是 FastAPI 最强大的特性之一。本章从 Depends 的声明到 solve_dependencies 的执行，从依赖缓存到 yield 依赖的资源管理，逐层剖析 FastAPI 依赖注入系统的源码实现。

---

## 44.1 概念与语法

### 44.1.1 依赖注入架构

```python
"""
FastAPI 依赖注入系统的核心概念：

1. Depends() — 声明依赖
   从函数签名声明依赖关系

2. Dependant — 依赖描述对象
   描述一个依赖的所有信息（函数、参数、缓存等）

3. solve_dependencies — 解析依赖
   递归解析依赖图，执行依赖函数

4. 依赖图 — 依赖之间的引用关系
   Dependant.dependencies 列表存储子依赖

数据流：
路由函数签名 → 收集 Depends() → 构建 Dependant → solve_dependencies → 执行依赖 → 注入参数
"""
```

### 44.1.2 核心数据结构

```python
"""
核心数据结构（fastapi/dependencies/models.py）：

class Dependant:
    name: str                    # 参数名
    call: Callable               # 依赖函数
    use_cache: bool              # 是否缓存结果
    path: str                    # 路由路径
    cache_key: tuple             # 缓存键
    dependencies: list[Dependant]  # 子依赖列表
    security_scopes: list        # 安全范围

class Dependency:
    """Depends() 的返回类型"""
    dependency: Callable         # 依赖函数
    use_cache: bool              # 是否缓存

class SecurityBase:
    """安全依赖的基类"""
    model: Any                   # OpenAPI 模型
    scheme_name: str             # 方案名称

依赖图的构建过程：
1. 遍历路由函数签名
2. 对每个 Depends() 参数创建 Dependant
3. 递归解析 Dependant 的子依赖
4. 构建 Dependant.dependencies 列表
5. 形成有向无环图（DAG）

示例：
async def get_db(): ...
async def get_user(db=Depends(get_db)): ...

Dependant(call=get_user)
  └── Dependant(call=get_db)
"""
```

## 44.2 原理与机制

### 44.2.1 依赖收集与构建

```python
"""
依赖收集源码（fastapi/dependencies/utils.py）：

def get_dependant(
    depends: Depends,
    path: str,
) -> Dependant:
    # 1. 创建 Dependant 对象
    dependant = Dependant(
        call=depends.dependency,
        use_cache=depends.use_cache,
        path=path,
    )

    # 2. 检查依赖覆盖
    # dependency_overrides_provider 检查

    # 3. 解析依赖函数的参数签名
    sig = inspect.signature(dependant.call)

    for param_name, param in sig.parameters.items():
        # 4. 根据参数注解确定参数类型
        if isinstance(param.default, Depends):
            # 4a. 嵌套依赖 — 递归解析
            sub_dependant = get_dependant(
                depends=param.default,
                path=path,
            )
            dependant.dependencies.append(sub_dependant)

        elif isinstance(param.annotation, SecurityBase):
            # 4b. 安全依赖（OAuth2 等）
            ...

        else:
            # 4c. 请求参数（Query, Path, Body 等）
            # 添加到 dependant.query_params, dependant.body_params 等
            ...

    return dependant

关键点：
1. 递归构建依赖图
2. 子依赖存储在 dependant.dependencies 列表
3. 每个参数对应一个 Depends() 或请求参数
4. 安全依赖（OAuth2）有特殊处理
"""
```

### 44.2.2 solve_dependencies 源码分析

```python
"""
solve_dependencies 源码（fastapi/dependencyInjection.py）：

async def solve_dependencies(
    request: Request,
    dependant: Dependant,
    dependency_overrides_provider: Any = None,
    dependency_cache: dict[tuple, Any] | None = None,
) -> tuple[dict[str, Any], list[ValidationError]]:
    # 1. 初始化
    values: dict[str, Any] = {}
    errors: list[ValidationError] = []

    # 2. 初始化依赖缓存
    if dependency_cache is None:
        dependency_cache = {}

    # 3. 遍历依赖图（深度优先）
    for sub_dependant in dependant.dependencies:
        # 4. 递归解析子依赖
        sub_values, sub_errors = await solve_dependencies(
            request=request,
            dependant=sub_dependant,
            dependency_overrides_provider=dependency_overrides_provider,
            dependency_cache=dependency_cache,
        )

        # 5. 收集子依赖的错误
        errors.extend(sub_errors)

        # 6. 如果子依赖没有错误，执行依赖函数
        if not sub_errors:
            # 7. 检查依赖覆盖
            call = sub_dependant.call
            if dependency_overrides_provider:
                overrides = dependency_overrides_provider.dependency_overrides
                if call in overrides:
                    call = overrides[call]

            # 8. 检查缓存
            cache_key = sub_dependant.cache_key
            if sub_dependant.use_cache and cache_key in dependency_cache:
                values[sub_dependant.name] = dependency_cache[cache_key]
                continue

            # 9. 执行依赖函数
            value = await call(**sub_values)

            # 10. 缓存结果
            if sub_dependant.use_cache:
                dependency_cache[cache_key] = value

            values[sub_dependant.name] = value

    # 11. 处理路由函数的直接参数（Query, Path, Body 等）
    for param in dependant.query_params:
        # 从 request.query_params 提取
        ...
    for param in dependant.path_params:
        # 从 request.path_params 提取
        ...
    for param in dependant.body_params:
        # 从 request body 提取并验证
        ...

    return values, errors

关键特性：
1. 递归解析依赖图（深度优先遍历）
2. 同一请求内依赖缓存（use_cache=True）
3. dependency_overrides 机制（测试用）
4. yield 依赖的资源管理（try/finally）
5. 验证错误收集（不中断，收集所有错误）
"""
```

### 44.2.3 yield 依赖的资源管理

```python
"""
yield 依赖的实现（fastapi/dependencyInjection.py）：

async def solve_dependencies(...):
    ...
    # yield 依赖使用 async with 上下文管理
    stack = AsyncExitStack()
    # stack 会跟踪所有 yield 依赖
    # 在请求结束时（finally），调用所有 yield 依赖的 finally 块

    for sub_dependant in dependant.dependencies:
        # 检查是否是 yield 依赖
        if inspect.isasyncgenfunction(call) or inspect.isgeneratorfunction(call):
            # 使用 AsyncExitStack 管理
            value = await stack.enter_async_context(
                contextmanager_or_asynccontextmanager(call)(**sub_values)
            )
        else:
            value = await call(**sub_values)

    # 请求处理完成后
    # await stack.aclose()  → 触发所有 yield 依赖的 finally 块

yield 依赖的执行流程：

async def get_db():
    db = await create_connection()   # __aenter__
    try:
        yield db                       # yield
    finally:
        await db.close()               # __aexit__

1. solve_dependencies 执行到 yield
2. 返回 db 连接给路由函数
3. 路由函数执行完毕
4. AsyncExitStack.aclose() 触发 finally
5. 数据库连接关闭

源码位置：
- fastapi/dependencyInjection.py: solve_dependencies()
- contextlib.AsyncExitStack: yield 依赖管理
"""
```

### 44.2.4 依赖缓存机制

```python
"""
依赖缓存机制：

1. 缓存键（cache_key）：
   - 由依赖函数的 id 和路径组成
   - 同一函数在同一请求中共享缓存

2. 缓存范围：
   - 单次请求内有效
   - 每个请求创建新的 dependency_cache
   - use_cache=True 时缓存

3. 缓存行为：
   async def get_db():
       return Database()

   # 同一请求中多次 Depends(get_db) 只执行一次
   @app.get("/items")
   async def get_items(db1=Depends(get_db), db2=Depends(get_db)):
       assert db1 is db2  # True，缓存生效

   # 不同请求中 Depends(get_db) 各执行一次
   # 请求1 → 执行 get_db()
   # 请求2 → 执行 get_db()（新缓存）

4. use_cache=False：
   async def get_timestamp():
       return time.time()

   # 不缓存，每次都执行
   @app.get("/time")
   async def get_time(ts=Depends(get_timestamp, use_cache=False)):
       return {"time": ts}

5. 缓存键冲突：
   - 不同路径的相同依赖函数使用不同缓存键
   - 路径前缀确保缓存隔离
"""
```

## 44.3 使用场景

### 44.3.1 复杂依赖图分析

```python
from fastapi import FastAPI, Depends

app = FastAPI()

# 依赖图：
# get_db → get_session
# get_user → get_db
# get_current_user → get_db, get_user
# admin_required → get_current_user

async def get_db():
    return Database()

async def get_session(db=Depends(get_db)):
    return Session(db)

async def get_user(session=Depends(get_session), user_id: int = None):
    return User(id=user_id)

async def get_current_user(session=Depends(get_session), token: str = None):
    return authenticate(token)

async def admin_required(user=Depends(get_current_user)):
    if user.role != "admin":
        raise HTTPException(403)
    return user

@app.get("/admin/dashboard")
async def admin_dashboard(
    admin=Depends(admin_required),
    session=Depends(get_session),
):
    # 依赖执行顺序：
    # 1. get_db()
    # 2. get_session(db=get_db_result)  # 缓存 get_db
    # 3. get_current_user(session=get_session_result)
    # 4. admin_required(user=get_current_user_result)
    # 5. get_session() → 使用缓存
    return {"admin": admin}
```

## 44.4 示例代码

### 44.4.1 依赖注入追踪

```python
"""追踪依赖注入的完整流程"""
from fastapi import FastAPI, Depends, Request

app = FastAPI()

# 添加追踪中间件
@app.middleware("http")
async def trace_middleware(request: Request, call_next):
    print(f"\n=== Request: {request.method} {request.url.path} ===")
    request.state.trace_log = []
    response = await call_next(request)
    for log in request.state.trace_log:
        print(f"  {log}")
    return response

async def trace_dependency(name, func):
    """包装依赖以追踪执行。"""
    async def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = await func(*args, **kwargs) if inspect.iscoroutinefunction(func) else func(*args, **kwargs)
        duration = time.time() - start
        print(f"  [Dep] {name}: {duration:.4f}s")
        return result
    return wrapper

# 定义可追踪的依赖
async def get_db():
    print("  [Dep] get_db: executing")
    return "db_connection"

async def get_user(db=Depends(get_db)):
    print("  [Dep] get_user: executing")
    return {"id": 1, "name": "Alice"}

@app.get("/items")
async def get_items(user=Depends(get_user), db=Depends(get_db)):
    # get_db 只执行一次（缓存）
    print("  [Handler] get_items: executing")
    return {"user": user}
```

## 44.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 循环依赖 | 检查依赖图，避免 A→B→A |
| 忘记 await 异步依赖 | 异步依赖必须用 async def |
| 不使用 use_cache=False | 时间戳等每次都应重新计算的依赖设 use_cache=False |
| yield 依赖不释放资源 | 使用 try/finally 确保资源释放 |
| 依赖过深影响性能 | 控制依赖层级，超过 5 层考虑简化 |
| 测试不覆盖依赖 | 使用 dependency_overrides 替换测试依赖 |
| 全局状态在依赖中 | 依赖使用 request.state 而非全局变量 |
