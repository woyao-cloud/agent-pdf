# 第26章 中间件机制

中间件是 ASGI 应用的核心扩展机制。本章从 Starlette 中间件到 FastAPI 的中间件系统，从内置中间件到自定义中间件，深入剖析中间件的机制与实践。

---

## 26.1 概念与语法

### 26.1.1 中间件基础

```python
from fastapi import FastAPI, Request
from starlette.middleware.base import BaseHTTPMiddleware

app = FastAPI()

# 方式1：BaseHTTPMiddleware（推荐简单场景）
class TimingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        start = time.time()
        response = await call_next(request)
        duration = time.time() - start
        response.headers["X-Process-Time"] = f"{duration:.4f}s"
        return response

app.add_middleware(TimingMiddleware)

# 方式2：纯 ASGI 中间件（推荐高性能场景）
class RawMiddleware:
    def __init__(self, app):
        self.app = app

    async def __call__(self, scope, receive, send):
        if scope["type"] != "http":
            await self.app(scope, receive, send)
            return

        # 请求前处理
        start = time.time()

        # 自定义 send 函数
        async def send_with_timing(message):
            if message["type"] == "http.response.start":
                headers = dict(message.get("headers", []))
                headers[b"x-process-time"] = f"{time.time() - start:.4f}s".encode()
                message["headers"] = list(headers.items())
            await send(message)

        await self.app(scope, receive, send_with_timing)

app.add_middleware(RawMiddleware)

# 方式3：@app.middleware 装饰器
@app.middleware("http")
async def add_process_time(request: Request, call_next):
    start = time.time()
    response = await call_next(request)
    response.headers["X-Process-Time"] = f"{time.time() - start:.4f}s"
    return response
```

### 26.1.2 内置中间件

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.middleware.gzip import GZipMiddleware
from fastapi.middleware.httpsredirect import HTTPSRedirectMiddleware
from fastapi.middleware.trustedhost import TrustedHostMiddleware

app = FastAPI()

# CORS 中间件
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://example.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Gzip 压缩
app.add_middleware(GZipMiddleware, minimum_size=1000)  # 超过1KB才压缩

# HTTPS 重定向
app.add_middleware(HTTPSRedirectMiddleware)

# 可信主机
app.add_middleware(TrustedHostMiddleware, allowed_hosts=["example.com", "*.example.com"])
```

### 26.1.3 中间件依赖注入

```python
from fastapi import FastAPI, Request, Depends

app = FastAPI()

# 中间件中访问依赖
async def get_db():
    return database_session

@app.middleware("http")
async def db_middleware(request: Request, call_next):
    # 中间件中不能直接使用 Depends
    # 但可以手动获取数据库连接
    db = next(get_db())
    request.state.db = db
    try:
        response = await call_next(request)
    finally:
        db.close()
    return response

# 更好的方式：在路由中通过依赖注入
@app.get("/items")
async def get_items(db=Depends(get_db)):
    return db.query(Item).all()
```

## 26.2 原理与机制

### 26.2.1 ASGI 中间件协议

```python
"""
ASGI 中间件的核心协议：

1. 中间件是一个可调用对象，接受三个参数：
   async def middleware(scope, receive, send):
       # scope:  连接信息字典
       # receive: 接收消息的协程
       # send:   发送消息的协程

2. 中间件通过包装 send 和 receive 来修改请求和响应：
   - 修改 scope → 修改请求信息
   - 包装 receive → 修改请求体
   - 包装 send   → 修改响应

3. 洋葱模型执行顺序：
   Request → Middleware1 → Middleware2 → Route Handler
                                                ↓
   Response ← Middleware1 ← Middleware2 ← Route Handler

4. 中间件注册顺序与执行顺序：
   app.add_middleware(A)  # 最后注册，最外层（请求最先经过）
   app.add_middleware(B)  # 中间
   app.add_middleware(C)  # 最先注册，最内层

   请求：A → B → C → Handler
   响应：Handler → C → B → A

5. Starlette 中间件源码：
   starlette/middleware/base.py: BaseHTTPMiddleware
   starlette/middleware/cors.py: CORSMiddleware
   starlette/middleware/errors.py: ServerErrorMiddleware
   starlette/middleware/exceptions.py: ExceptionMiddleware
"""
```

### 26.2.2 BaseHTTPMiddleware 的实现

```python
"""
BaseHTTPMiddleware 的工作原理：

class BaseHTTPMiddleware:
    async def __call__(self, scope, receive, send):
        if scope["type"] != "http":
            await self.app(scope, receive, send)
            return

        # 创建 Request 对象
        request = Request(scope, receive, send)

        # 调用子类的 dispatch 方法
        response = await self.dispatch(request, call_next)

        # 发送响应
        await response(scope, receive, send)

    async def call_next(self, request):
        # 创建接收队列
        recv = asyncio.Queue()
        # 将原始 receive 消息放入队列
        # 创建新的 send 函数来收集响应
        # 等待下一个中间件或路由处理器完成

关键限制：
1. call_next 内部使用 asyncio.Queue 和 TaskGroup
2. 请求体只能读取一次
3. 响应必须返回 Response 对象
4. 不适用于 WebSocket 和 lifespan 事件
5. 性能比纯 ASGI 中间件略差（额外的 Request/Response 对象创建）

对于高性能场景，推荐纯 ASGI 中间件：
- 避免创建 Request 对象
- 直接操作 scope/receive/send
- 减少 asyncio 开销
"""
```

### 26.2.3 中间件与异常处理

```python
"""
中间件中的异常处理：

1. 中间件可以捕获下游的异常
2. 但要注意异常传播顺序

@app.middleware("http")
async def error_handler(request, call_next):
    try:
        response = await call_next(request)
    except CustomException as e:
        return JSONResponse(status_code=400, content={"error": str(e)})
    except Exception as e:
        return JSONResponse(status_code=500, content={"error": "Internal error"})
    return response

3. ServerErrorMiddleware（内置）：
   - 捕获所有未处理的异常
   - 开发模式返回 HTML 错误页面
   - 生产模式返回 500 Internal Server Error
   - 位于中间件链最外层

4. ExceptionMiddleware（内置）：
   - 处理 HTTPException
   - 查找注册的 exception_handler
   - 位于 ServerErrorMiddleware 下方

中间件栈（从外到内）：
ServerErrorMiddleware → ExceptionMiddleware → CORS → 自定义中间件 → 路由
"""
```

## 26.3 使用场景

### 26.3.1 请求日志中间件

```python
import time
import logging
from fastapi import FastAPI, Request
from starlette.middleware.base import BaseHTTPMiddleware

logger = logging.getLogger("api.access")

class LoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        start = time.time()
        request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))

        # 请求日志
        logger.info(
            f"→ {request.method} {request.url.path} "
            f"client={request.client.host} request_id={request_id}"
        )

        response = await call_next(request)
        duration = time.time() - start

        # 响应日志
        logger.info(
            f"← {request.method} {request.url.path} "
            f"status={response.status_code} "
            f"duration={duration:.3f}s request_id={request_id}"
        )

        response.headers["X-Request-ID"] = request_id
        return response

app.add_middleware(LoggingMiddleware)
```

### 26.3.2 数据库会话中间件

```python
from fastapi import FastAPI, Request
from starlette.middleware.base import BaseHTTPMiddleware

app = FastAPI()

class DatabaseMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        request.state.db = create_db_session()
        try:
            response = await call_next(request)
            request.state.db.commit()
        except Exception:
            request.state.db.rollback()
            raise
        finally:
            request.state.db.close()
        return response

app.add_middleware(DatabaseMiddleware)

# 路由中使用
@app.get("/items")
async def get_items(request: Request):
    db = request.state.db
    return db.query(Item).all()
```

## 26.4 示例代码

### 26.4.1 完整的中间件体系

```python
"""生产级中间件体系"""
import time
import uuid
import logging
from fastapi import FastAPI, Request, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from fastapi.middleware.gzip import GZipMiddleware
from fastapi.responses import JSONResponse

app = FastAPI(title="API with Middleware")
logger = logging.getLogger("app")

# 1. 请求追踪（最外层）
@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    request_id = request.headers.get("X-Request-ID") or str(uuid.uuid4())
    request.state.request_id = request_id
    response = await call_next(request)
    response.headers["X-Request-ID"] = request_id
    return response

# 2. 请求计时
@app.middleware("http")
async def timing_middleware(request: Request, call_next):
    start = time.time()
    response = await call_next(request)
    response.headers["X-Response-Time"] = f"{time.time() - start:.3f}s"
    return response

# 3. CORS
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://app.example.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 4. Gzip 压缩
app.add_middleware(GZipMiddleware, minimum_size=1000)

# 5. 安全头
@app.middleware("http")
async def security_headers(request: Request, call_next):
    response = await call_next(request)
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["X-XSS-Protection"] = "1; mode=block"
    return response

@app.get("/health")
async def health():
    return {"status": "ok"}
```

## 26.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| BaseHTTPMiddleware 读取请求体 | 请求体只能读取一次，中间件中避免读取 |
| 中间件顺序错误 | 日志和追踪最外层，CORS 和安全头次之 |
| 中间件中抛异常 | 中间件异常不经过 exception_handler |
| 不处理 WebSocket | 中间件需检查 scope["type"] |
| 全局异常处理放中间件 | 使用 @app.exception_handler 替代 |
| 中间件影响性能 | 高频路径使用纯 ASGI 中间件 |
| 中间件不记录响应时间 | 在 call_next 前后计算时间差 |
