# 第27章 请求生命周期与事件处理

理解请求从接收到响应的完整生命周期是深入掌握 FastAPI 的关键。本章从 ASGI 连接生命周期到 FastAPI 的事件处理，从 startup/shutdown 到 lifespan，深入剖析请求生命周期的底层机制。

---

## 27.1 概念与语法

### 27.1.1 请求生命周期

```python
"""
HTTP 请求在 FastAPI 中的完整生命周期：

1. TCP 连接建立
   uvicorn accept() → 创建连接对象

2. ASGI 事件创建
   scope = {
       "type": "http",
       "method": "GET",
       "path": "/items/1",
       "query_string": b"",
       "headers": [(b"host", b"localhost:8000"), ...],
       "server": ("localhost", 8000),
   }
   receive = coroutine  # 接收请求体
   send = coroutine      # 发送响应

3. 中间件链处理
   ServerErrorMiddleware → ExceptionMiddleware → CORS → 自定义中间件

4. 路由匹配
   遍历 routes → 匹配路径和方法 → 提取路径参数

5. 依赖注入
   solve_dependencies() → 执行依赖链

6. 请求验证
   Pydantic 模型验证 → 422 错误或继续

7. 路由函数执行
   async def endpoint(...) → 业务逻辑

8. 响应处理
   response_model 过滤 → jsonable_encoder 序列化

9. 响应发送
   ASGI send() → HTTP 响应发送给客户端

10. 连接关闭（或 keep-alive）
"""
```

### 27.1.2 Lifespan 事件

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI

# 推荐方式：lifespan 上下文管理器
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    await connect_to_database()
    await initialize_cache()
    print("Application started")
    yield  # 应用运行期间
    # Shutdown
    await close_database()
    await close_cache()
    print("Application stopped")

app = FastAPI(lifespan=lifespan)

# 旧方式（已弃用）：on_event
@app.on_event("startup")
async def startup():
    await connect_to_database()

@app.on_event("shutdown")
async def shutdown():
    await close_database()
```

### 27.1.3 Lifespan 与状态管理

from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):

    app.state.redis = await create_redis_connection()

### 27.1.4 请求状态（request.state）

```python
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def auth_middleware(request: Request, call_next):
    # 在中间件中设置请求级状态
    request.state.user = authenticate(request)
    request.state.start_time = time.time()
    response = await call_next(request)
    return response

@app.get("/me")
async def get_me(request: Request):
    # 在路由中访问请求级状态
    return {"user": request.state.user}

@app.get("/timing")
async def get_timing(request: Request):
    duration = time.time() - request.state.start_time
    return {"duration": duration}

# request.state 是 Starlette State 对象
# 支持任意属性设置和访问
# 生命周期仅限单个请求
```

## 27.2 原理与机制

### 27.2.1 ASGI Lifespan 协议

```python
"""
ASGI Lifespan 协议：

1. ASGI 服务器发送 lifespan 事件：
   {"type": "lifespan.startup"}  → 启动

2. 应用处理完成后回复：
   {"type": "lifespan.startup.complete"}  → 启动成功
   {"type": "lifespan.startup.failed", "message": "..."}  → 启动失败

3. 关闭时发送：
   {"type": "lifespan.shutdown"}  → 关闭

4. 应用处理完成后回复：
   {"type": "lifespan.shutdown.complete"}  → 关闭完成
   {"type": "lifespan.shutdown.failed", "message": "..."}  → 关闭失败

FastAPI lifespan 实现（源码：fastapi/routing.py）：

class LifespanContext:
    async def __aenter__(self):
        # 执行所有 startup 事件处理器
        for handler in self.startup_handlers:
            await handler()
        return self

    async def __aexit__(self, *args):
        # 执行所有 shutdown 事件处理器
        for handler in self.shutdown_handlers:
            await handler()

LifespanOnEvent 方式（已弃用）：
- startup 事件收集在 self.startup_handlers 列表
- shutdown 事件收集在 self.shutdown_handlers 列表
- 按注册顺序执行

Lifespan 方式（推荐）：
- 使用 @asynccontextmanager
- yield 之前是 startup，yield 之后是 shutdown
- 更清晰的生命周期管理
"""
```

### 27.2.2 请求处理流程源码

```python
"""
FastAPI 请求处理的核心流程（源码分析）：

1. ASGI 入口：FastAPI.__call__(scope, receive, send)
   → 调用 Starlette 的路由分发

2. 路由匹配：Router.__call__()
   → 遍历 self.routes 查找匹配的路由
   → Route.match(scope) 返回路径参数

3. 依赖注入：solve_dependencies()
   → 位置：fastapi/dependencyInjection.py
   → 递归解析依赖图
   → 执行依赖函数，缓存结果

4. 请求体解析：get_request_handler()
   → 位置：fastapi/routing.py
   → 根据 Depends/Query/Path/Body 解析参数
   → Pydantic 模型验证

5. 路由执行：endpoint(**params)
   → 调用用户定义的路由函数
   → 返回结果

6. 响应处理：
   → response_model 过滤字段
   → jsonable_encoder 序列化
   → 设置响应头和状态码

7. 异常处理：
   → RequestValidationError → 422
   → HTTPException → 对应状态码
   → 未捕获异常 → 500

关键源码文件：
- fastapi/routing.py: APIRoute.get_route_handler()
- fastapi/dependencyInjection.py: solve_dependencies()
- fastapi/params.py: 参数解析
- fastapi/response.py: 响应处理
- starlette/routing.py: Router, Route
"""
```

## 27.3 使用场景

### 27.3.1 数据库连接池初始化

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI, Depends
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    engine = create_async_engine("postgresql+asyncpg://user:pass@localhost/db")
    async_session = sessionmaker(engine, class_=AsyncSession)
    app.state.db_engine = engine
    app.state.async_session = async_session

    yield

    # Shutdown
    await engine.dispose()

app = FastAPI(lifespan=lifespan)

async def get_db():
    async with app.state.async_session() as session:
        yield session

@app.get("/items")
async def get_items(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(Item))
    return result.scalars().all()
```

### 27.3.2 优雅关闭

```python
import signal
import asyncio
from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    app.state.connections = set()
    app.state.shutting_down = False
    print("Server started")
    yield
    # Shutdown — 等待活跃连接完成
    app.state.shutting_down = True
    while app.state.connections:
        print(f"Waiting for {len(app.state.connections)} active connections...")
        await asyncio.sleep(0.5)
    print("All connections closed")

app = FastAPI(lifespan=lifespan)

@app.get("/slow")
async def slow_endpoint(request: Request):
    request.app.state.connections.add(request)
    try:
        await asyncio.sleep(10)  # 模拟慢请求
        return {"status": "done"}
    finally:
        request.app.state.connections.discard(request)
```

## 27.4 示例代码

### 27.4.1 完整的应用生命周期管理

```python
"""生产级应用生命周期管理"""
from contextlib import asynccontextmanager
from fastapi import FastAPI, Request
import redis.asyncio as aioredis
import logging

logger = logging.getLogger("app")

@asynccontextmanager
async def lifespan(app: FastAPI):
    """应用生命周期管理。"""
    # === Startup ===
    logger.info("Starting application...")

    # 数据库连接
    from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
    from sqlalchemy.orm import sessionmaker
    app.state.db_engine = create_async_engine(
        "postgresql+asyncpg://user:pass@localhost/db",
        pool_size=20, max_overflow=10,
    )
    app.state.db_session = sessionmaker(
        app.state.db_engine, class_=AsyncSession,
    )
    logger.info("Database connected")

    # Redis 连接
    app.state.redis = aioredis.from_url("redis://localhost:6379")
    logger.info("Redis connected")

    # 加载 ML 模型
    app.state.model = load_model()
    logger.info("ML model loaded")

    # 预热缓存
    await warm_up_cache(app.state.redis)
    logger.info("Cache warmed up")

    yield  # === 应用运行中 ===

    # === Shutdown ===
    logger.info("Shutting down application...")

    # 关闭数据库连接
    await app.state.db_engine.dispose()
    logger.info("Database disconnected")

    # 关闭 Redis 连接
    await app.state.redis.close()
    logger.info("Redis disconnected")

    logger.info("Application stopped")

app = FastAPI(lifespan=lifespan, title="Production API")

@app.get("/health")
async def health():
    return {"status": "healthy"}

@app.get("/predict")
async def predict(request: Request):
    model = request.app.state.model
    return {"prediction": model.predict()}
```

## 27.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 使用 on_event 而非 lifespan | 使用 lifespan 上下文管理器（on_event 已弃用） |
| startup 中创建的连接不关闭 | yield 后确保所有资源释放 |
| 不处理 shutdown 异常 | shutdown 代码用 try/except 包裹 |
| 资源初始化顺序依赖 | 按依赖顺序初始化，逆序关闭 |
| request.state 和 app.state 混淆 | request.state 单次请求，app.state 全局 |
| 不做优雅关闭 | 等待活跃请求完成后关闭 |
| 多 Worker 时共享状态 | app.state 在每个 Worker 独立，共享用 Redis |
