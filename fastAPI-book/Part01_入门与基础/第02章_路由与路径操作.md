# 第02章 路由与路径操作

路由是 FastAPI 应用的骨架。本章从路径操作装饰器的注册机制到 Starlette 的路由匹配算法，深入剖析 FastAPI 路由系统的底层原理。

---

## 2.1 概念与语法

### 2.1.1 路径操作装饰器

```python
from fastapi import FastAPI

app = FastAPI()

# HTTP 方法装饰器
@app.get("/")                    # GET 请求
async def read_root():
    return {"message": "Hello"}

@app.post("/items/")             # POST 请求
async def create_item(item: dict):
    return {"created": item}

@app.put("/items/{item_id}")     # PUT 请求（完整更新）
async def update_item(item_id: int, item: dict):
    return {"id": item_id, **item}

@app.patch("/items/{item_id}")   # PATCH 请求（部分更新）
async def partial_update(item_id: int, item: dict):
    return {"id": item_id, "updated": item}

@app.delete("/items/{item_id}")  # DELETE 请求
async def delete_item(item_id: int):
    return {"deleted": item_id}

@app.options("/items/")          # OPTIONS 请求
async def options_items():
    return {}

@app.head("/items/")             # HEAD 请求
async def head_items():
    return {}

# 多种方法
from fastapi import APIRouter

router = APIRouter()

@router.api_route("/items/", methods=["GET", "POST"])
async def handle_items():
    return {"method": "multi"}
```

### 2.1.2 路径参数

```python
from fastapi import FastAPI
from enum import Enum

app = FastAPI()

# 基本路径参数
@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

# 枚举路径参数
class ModelName(str, Enum):
    alexnet = "alexnet"
    resnet = "resnet"
    lenet = "lenet"

@app.get("/models/{model_name}")
async def get_model(model_name: ModelName):
    if model_name is ModelName.alexnet:
        return {"model": model_name, "message": "Deep Learning"}
    return {"model": model_name}

# 路径参数包含斜杠（使用 Path）
from fastapi import Path

@app.get("/files/{file_path:path}")
async def read_file(file_path: str = Path(...)):
    return {"file_path": file_path}

# 路径参数验证
@app.get("/items/{item_id}")
async def read_item(
    item_id: int = Path(..., title="Item ID", description="The ID of the item", ge=1, le=1000),
):
    return {"item_id": item_id}

# Path 验证参数：
# gt/ge — 大于/大于等于
# lt/le — 小于/小于等于
# title — OpenAPI 文档标题
# description — OpenAPI 文档描述
# alias — 别名
```

### 2.1.3 路由顺序

```python
from fastapi import FastAPI

app = FastAPI()

# 路由匹配顺序：先注册先匹配
# 以下路由顺序是正确的
@app.get("/users/me")       # 固定路径在前
async def read_user_me():
    return {"user": "me"}

@app.get("/users/{user_id}")  # 动态路径在后
async def read_user(user_id: int):
    return {"user_id": user_id}

# 如果顺序反过来，/users/me 会被 /users/{user_id} 匹配
# user_id = "me"，会导致类型转换错误
```

## 2.2 原理与机制

### 2.2.1 路由注册机制

```python
"""
FastAPI 路由注册的源码流程：

1. @app.get("/items/{item_id}") 调用 FastAPI.get()
2. FastAPI.get() 调用 APIRoute.add_api_route()
3. 创建 Route 对象（Starlette 路由）
4. 路由信息存储在 app.routes 列表中
5. app.include_router() 将子路由器的路由合并到主路由

关键源码（fastapi/routing.py）：

class APIRoute(routing.Route):
    def __init__(self, path, endpoint, ...):
        # 1. 依赖收集：从函数签名提取依赖
        self.dependant = get_dependant(path=path, call=endpoint)
        
        # 2. 请求体解析：从依赖中提取请求体模型
        self.body_field = get_body_field(dependant=self.dependant)
        
        # 3. OpenAPI schema 生成
        self.openapi_schema = self._generate_schema()
        
        # 4. 创建请求处理函数
        self.app = get_request_handler(
            dependant=self.dependant,
            body_field=self.body_field,
            ...
        )

# 路由注册顺序保存在 self.routes 列表中
# 请求匹配时按顺序遍历，第一个匹配的路由胜出
"""
```

### 2.2.2 路径匹配算法

```python
"""
Starlette 的路由匹配流程：

1. 请求到达 app.__call__(scope, receive, send)
2. 遍历 app.routes 列表
3. 对每个路由调用 route.matches(scope)
4. 第一个匹配的路由处理请求

Route.matches() 的实现：
- 路径模板编译为正则表达式
- 例如 /items/{item_id} 编译为 /^items/(?P<item_id>[^/]+)/
- 路径参数通过正则捕获组提取
- {item_id:int} 编译为 /^items/(?P<item_id>\d+)/
- {file_path:path} 编译为 /^files/(?P<file_path>.+)$/

路由匹配的优先级：
1. 精确匹配优先于模式匹配
2. 先注册的路由优先
3. Starlette 使用 Router 按注册顺序遍历所有路由

性能考虑：
- 路由数量较多时（>1000），线性遍历可能成为瓶颈
- 可考虑使用 Starlette 的 Mount 进行路由分组
- 或使用第三方路由库（如 fastapi-routers）优化
"""

# 路由编译示例
from starlette.routing import Route, Router

route = Route("/items/{item_id}", endpoint=read_item)
# route.path == "/items/{item_id}"
# route.path_regex 编译为正则表达式
# route.param_convertors == {"item_id": PathConvertor()}
```

### 2.2.3 OpenAPI 自动生成原理

```python
"""
FastAPI 自动生成 OpenAPI 文档的流程：

1. 路由注册时，收集每个端点的元数据：
   - 路径和方法 (path, method)
   - 请求参数 (path, query, header, cookie)
   - 请求体 (body, content-type)
   - 响应模型 (response_model, responses)
   - 状态码 (status_code)
   - 描述信息 (summary, description, tags)

2. FastAPI.openapi_schema 属性首次访问时生成完整 schema：
   - 遍历所有路由
   - 对每个路由调用 get_openapi_path()
   - 收集所有 schema 定义到 components/schemas
   - 生成 OpenAPI 3.1 规范文档

3. Swagger UI 和 ReDoc 端点：
   - /docs — Swagger UI（默认启用）
   - /redoc — ReDoc（默认启用）
   - /openapi.json — OpenAPI JSON 规范

自定义 OpenAPI：
"""

from fastapi import FastAPI

app = FastAPI()

def custom_openapi():
    if app.openapi_schema:
        return app.openapi_schema
    
    from fastapi.openapi.utils import get_openapi
    openapi_schema = get_openapi(
        title="My API",
        version="1.0.0",
        description="My API description",
        routes=app.routes,
    )
    
    # 自定义 schema
    openapi_schema["info"]["x-logo"] = {"url": "https://example.com/logo.png"}
    
    app.openapi_schema = openapi_schema
    return app.openapi_schema

app.openapi = custom_openapi
```

### 2.3.1 模块化路由组织

```python

# app/
# ├── main.py
# ├── api/
# │   ├── __init__.py
# │   ├── v1/
# │   │   ├── __init__.py
# │   │   ├── users.py
# │   │   ├── items.py
# │   │   └── auth.py
# │   └── v2/
# │       ├── __init__.py
# │       └── users.py
# └── dependencies.py

# app/api/v1/users.py
from fastapi import APIRouter, Depends

router = APIRouter()

@router.get("/", response_model=list[UserResponse])
async def list_users():
    ...

@router.get("/{user_id}", response_model=UserResponse)
async def get_user(user_id: int):
    ...

@router.post("/", response_model=UserResponse, status_code=201)

    ...

from .users import router as users_router
from .items import router as items_router

router.include_router(items_router, prefix="/items", tags=["items"])
router.include_router(auth_router, prefix="/auth", tags=["auth"])

from fastapi import FastAPI

app.include_router(v1_router, prefix="/api/v1")
```

## 2.4 示例代码

### 2.4.1 版本化 API 路由

```python
from fastapi import FastAPI, APIRouter
from pydantic import BaseModel

app = FastAPI(title="Versioned API")

# V1 模型
class UserV1(BaseModel):
    id: int
    name: str

# V2 模型（增加了 email 字段）

    id: int
    name: str
    email: str

@v1_router.get("/users/{user_id}", response_model=UserV1)

    return UserV1(id=user_id, name="Alice")

v2_router = APIRouter(prefix="/api/v2", tags=["v2"])

@v2_router.get("/users/{user_id}", response_model=UserV2)
async def get_user_v2(user_id: int):
    return UserV2(id=user_id, name="Alice", email="alice@example.com")

app.include_router(v1_router)
app.include_router(v2_router)
```

## 2.5 常见陷阱与最佳实践

### 2.5.1 路由顺序问题

```python
# 陷阱：动态参数路由在固定路径之前注册
@app.get("/users/{user_id}")  # 这会匹配 /users/me
async def get_user(user_id: str):
    return {"user_id": user_id}

@app.get("/users/me")  # 永远不会被匹配到
async def get_current_user():
    return {"user": "current"}

# 修复：固定路径放在动态参数之前
@app.get("/users/me")
async def get_current_user():
    return {"user": "current"}

@app.get("/users/{user_id}")
async def get_user(user_id: str):
    return {"user_id": user_id}
```

### 2.5.2 尾部斜杠处理

```python
from fastapi import FastAPI

app = FastAPI()

# FastAPI 默认不自动重定向尾部斜杠
@app.get("/items")      # 匹配 /items
async def read_items():
    return {"items": []}

@app.get("/items/")     # 匹配 /items/
async def read_items_slash():
    return {"items": []}

# 最佳实践：统一风格，选择一种并保持一致
# 推荐：API 不使用尾部斜杠，文档页面使用尾部斜杠
```

### 2.5.3 总结

| 陷阱 | 最佳实践 |
|------|---------|
| 固定路径被动态参数拦截 | 固定路径注册在动态参数之前 |
| 路由前缀重复定义 | 使用 APIRouter 的 prefix 参数 |
| 尾部斜杠不一致 | 统一风格，推荐 API 不用尾部斜杠 |
| 路由函数缺少类型注解 | 所有参数必须添加类型注解 |
| 大量路由影响性能 | 使用 Mount 分组或第三方路由优化 |
