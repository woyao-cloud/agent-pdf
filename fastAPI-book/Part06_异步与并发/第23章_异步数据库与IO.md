# 第23章 异步数据库与I/O

异步 I/O 是 FastAPI 高性能的基石。本章从异步数据库操作到异步 HTTP 客户端，深入剖析 FastAPI 中异步 I/O 的实践与原理。

## 23.1 概念与语法

### 23.1.1 async/await 在 FastAPI 中的使用

```python
from fastapi import FastAPI

app = FastAPI()

# 异步路由（推荐）
@app.get("/items/")
async def read_items():
    data = await fetch_items_async()
    return data

# 同步路由（阻塞操作）
@app.get("/items/sync/")
def read_items_sync():
    data = fetch_items_sync()  # 会阻塞事件循环！
    return data

# 在异步路由中调用同步代码
import asyncio
from concurrent.futures import ThreadPoolExecutor

executor = ThreadPoolExecutor(max_workers=10)

@app.get("/items/mixed/")
async def read_items_mixed():
    loop = asyncio.get_event_loop()
    data = await loop.run_in_executor(executor, fetch_items_sync)
    return data

# 或使用 fastapi 的 SyncToAsync
from starlette.concurrency import run_in_threadpool

@app.get("/items/threadpool/")
async def read_items_threadpool():
    data = await run_in_threadpool(fetch_items_sync)
    return data
```

### 23.1.2 异步 HTTP 客户端（httpx）

```python
import httpx
from fastapi import FastAPI

app = FastAPI()

# 异步 HTTP 客户端
@app.get("/proxy/{url:path}")
async def proxy_request(url: str):
    async with httpx.AsyncClient() as client:
        response = await client.get(f"https://{url}")
        return response.json()

# 带超时和重试
@app.get("/api/data")
async def fetch_data():
    async with httpx.AsyncClient(
        timeout=httpx.Timeout(10.0, connect=5.0),
        follow_redirects=True,
    ) as client:
        response = await client.get("https://api.example.com/data")
        response.raise_for_status()
        return response.json()

# 并发请求
@app.get("/api/multi")
async def fetch_multiple():
    async with httpx.AsyncClient() as client:
        # 并发发送多个请求
        tasks = [
            client.get(f"https://api.example.com/items/{i}")
            for i in range(1, 6)
        ]
        responses = await asyncio.gather(*tasks)
        return [r.json() for r in responses]
```

### 23.1.3 异步任务编排

```python
import asyncio
from fastapi import FastAPI

app = FastAPI()

# asyncio.gather — 并发执行
@app.get("/dashboard")
async def dashboard():
    results = await asyncio.gather(
        fetch_user_stats(),
        fetch_order_stats(),
        fetch_revenue_stats(),
    )
    return {
        "users": results[0],
        "orders": results[1],
        "revenue": results[2],
    }

# TaskGroup（Python 3.11+）— 更安全的并发
@app.get("/dashboard/safe")
async def dashboard_safe():
    results = {}
    async with asyncio.TaskGroup() as tg:
        tg.create_task(fetch_user_stats())
        tg.create_task(fetch_order_stats())
        tg.create_task(fetch_revenue_stats())
    return results

# asyncio.wait — 更细粒度控制
@app.get("/search")
async def search():
    tasks = [search_source(i) for i in range(5)]
    done, pending = await asyncio.wait(
        tasks, timeout=2.0, return_when=asyncio.FIRST_COMPLETED
    )
    for task in pending:
        task.cancel()
    return [task.result() for task in done]
```

## 23.2 原理与机制

### 23.2.1 同步代码在异步框架中的影响

```python
"""
FastAPI 运行在 asyncio 事件循环上：
- 每个请求是一个协程（coroutine）
- 事件循环调度所有协程
- 如果一个协程阻塞（如同步 I/O），整个事件循环暂停
- 其他请求无法处理，直到阻塞操作完成

阻塞的影响：
1. 同步数据库查询（如 psycopg2）→ 阻塞事件循环
2. 同步 HTTP 请求（如 requests）→ 阻塞事件循环
3. CPU 密集型计算 → 阻塞事件循环

解决方案：
1. 使用异步库（asyncpg/aiosqlite/httpx）
2. 使用 run_in_threadpool() 将同步操作放入线程池
3. 使用 ProcessPoolExecutor 处理 CPU 密集型任务
"""
```

## 23.5 常见陷阱与最佳实践

| 陷阱 | 最佳实践 |
|------|---------|
| 异步路由中使用同步 I/O | 使用异步库或 run_in_threadpool |
| 忘记 await | 确保所有异步操作都 await |
| 创建过多并发任务 | 使用 asyncio.Semaphore 限制并发 |
| 不处理超时 | 使用 asyncio.wait_for 设置超时 |
| httpx 不复用连接 | 使用依赖注入共享 AsyncClient |
